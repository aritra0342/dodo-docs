## So funktioniert es

Zahlungsmethoden werden automatisch basierend auf Folgendem angezeigt:

- **Standort des Kunden** — Regionale Methoden wie iDEAL (Niederlande) oder UPI (Indien) erscheinen nur für Kunden in diesen Regionen
- **Transaktionswährung** — Zahlungsmethoden werden nach unterstützten Währungen gefiltert
- **Gerätetyp** — Apple Pay erscheint nur auf kompatiblen Apple-Geräten und -Browserversionen
- **Transaktionstyp** — Einige Methoden unterstützen keine Abonnements oder haben Mindestbeträge

<Info>
**Automatische Aktivierung:** Alle Zahlungsmethoden sind vorkonfiguriert und werden von Dodo Payments verwaltet. Es ist keine zusätzliche Einrichtung erforderlich – sie stehen zur Verfügung, sobald dein Konto verifiziert ist und der Live-Modus aktiviert wurde.
</Info>

## Konfiguration von Zahlungsmethoden

Standardmäßig erscheinen alle geeigneten Zahlungsmethoden automatisch. Verwende `allowed_payment_method_types`, um einzuschränken, welche Methoden angezeigt werden:

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://test.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

| Typ | Beschreibung |
| :--- | :------------ |
| `credit` | Kreditkarten (Visa, Mastercard, Amex usw.) |
| `debit` | Debitkarten |
| `apple_pay` | Apple Pay |

| `amazon_pay` | Amazon Pay |
| `cashapp` | Cash App Pay |
| `klarna` | Klarna BNPL |
| `afterpay_clearpay` | Afterpay / Clearpay |
| `upi_collect` | UPI Collect (QR-Code) |
| `ideal` | iDEAL (Niederlande) |
| `bancontact_card` | Bancontact (Belgien) |
| `eps` | EPS (Österreich) |
| `multibanco` | Multibanco (Portugal) |
| `revolut_pay` | RevolutPay |
| `billie` | Billie B2B BNPL |

<Warning>
Behalte `credit` und `debit` immer als Fallbacks bei. Wenn für einen Kunden keine der angegebenen Zahlungsmethoden verfügbar ist, schlägt der Checkout fehl.
</Warning>

## Beste Praktiken

<AccordionGroup>
{/* LOCKED_PATTERN_63da960e31e014d866efe70101f28b99 */}
Füge regionale Zahlungsmethoden für deine Zielmärkte hinzu. iDEAL deckt etwa 60 % der Online-Zahlungen in den Niederlanden ab; UPI dominiert in Indien. Das Weglassen dieser Methoden beeinträchtigt die Conversion erheblich.
</Accordion>

{/* LOCKED_PATTERN_78f59c035688fa14218708818c9f0fca */}
Selbst wenn du bestimmte Methoden gezielt ansprichst, behalte `credit` und `debit` als Fallbacks. Nicht alle Kunden haben Zugriff auf jede Zahlungsmethode.
</Accordion>

{/* LOCKED_PATTERN_564640b27644e47b9e4e7faaec6c2d88 */}
BNPL-Methoden (Klarna, Afterpay) erfordern mindestens $50,01. Schließe sie bei Produkten mit niedrigem Wert nicht ein – sie tauchen dann schlichtweg nicht auf, was akzeptabel ist.
</Accordion>

{/* LOCKED_PATTERN_ea5461a635a5291807627a6b40d7922e */}
Verwende den Testmodus, um die Verfügbarkeit von Zahlungsmethoden für verschiedene Kundenstandorte, Währungen und Gerätetypen zu überprüfen, bevor du live gehst.
</Accordion>
</AccordionGroup>

## Fehlersuche

<AccordionGroup>
{/* LOCKED_PATTERN_fbf2d38e7725fb97384bef01324b59a1 */}
**Überprüfe diese Punkte:**
1. Ist die Methode in `allowed_payment_method_types` enthalten?
2. Passt der Kundenstandort zur unterstützten Region der Methode?
3. Wird die Transaktionswährung von der Methode unterstützt?
4. Entspricht der Transaktionsbetrag dem Minimum (z. B. $50,01 für BNPL)?
5. Unterstützt die Methode bei Abonnements wiederkehrende Zahlungen?

**Lösung:** Entferne `allowed_payment_method_types` vorübergehend, um alle geeigneten Methoden anzuzeigen, und fahre dann mit der Eingrenzung des Problems fort.
</Accordion>

{/* LOCKED_PATTERN_57385d59bd851fb1e8c26266b489efd4 */}
**Mögliche Ursachen:**
- `allowed_payment_method_types` wird nicht korrekt übergeben
- Parameter falsch formatiert (muss ein Array von Strings sein)


**Lösung:** Protokolliere die API-Anfrage und prüfe die Struktur der Parameter.
</Accordion>

{/* LOCKED_PATTERN_6d1a1b1a59cbe2789bbc9434eb1ec153 */}
**Überprüfung:**
- Stimmen die Rechnungsadresse des Kunden und die Region der Methode überein?
- Ist die Währung korrekt gesetzt (z. B. EUR für iDEAL)?
- Ist Adaptive Currency aktiviert, falls der Kunde international ist?


**Lösung:** Überprüfe den Ländercode der Rechnungsadresse und die Währung in deiner API-Anfrage.
</Accordion>

{/* LOCKED_PATTERN_52f3532b80d2def876ef3cc78b55f9f6 */}
UPI- und Rupay-Abonnements haben spezifische Anforderungen, darunter 48-stündige Verzögerungen bei der Verarbeitung und RBI-Mandatslimits.

**Siehe:** <a href="/features/payment-methods/india">India Payment Methods</a> für vollständige Anleitungen.
</Accordion>
</AccordionGroup>
