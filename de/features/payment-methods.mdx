Dodo Payments unterstützt eine Vielzahl von Zahlungsmethoden, um Ihnen zu helfen, Kunden weltweit zu bedienen. Alle aufgeführten Zahlungsmethoden werden automatisch verfügbar, sobald Ihr Händlerkonto verifiziert und der Live-Modus aktiviert ist.

<Info>
**Automatische Aktivierung:** Es sind keine zusätzlichen Konfigurationen von Ihrer Seite erforderlich. Alle Zahlungsmethoden sind vorkonfiguriert und werden von Dodo Payments verwaltet. Sie können optional einschränken, welche Methoden beim Checkout angezeigt werden, indem Sie die unten beschriebene API-Konfiguration verwenden.
</Info>

  Dodo Payments akzeptiert alle großen Kredit- und Debitkartennetzwerke weltweit. Sie können akzeptieren:

  - **Visa** – Zuverlässiges, weltweit akzeptiertes Netzwerk
  - **Mastercard** – Robustes, sicheres Netzwerk mit globaler Reichweite
  - **American Express** – Premium-Vorteile und starker Kundenservice
  - **Discover** – Wettbewerbsfähige Belohnungen, starke Präsenz in den USA
  - **JCB** – Führendes asiatisches Kartennetzwerk mit wachsender Reichweite
  - **UnionPay** – Dominant in China und Asien, mit wachsender globaler Akzeptanz
  - **Interac** – Kanadas sicheres Debit-Zahlungsnetzwerk
  - **Cartes Bancaires** – Weit verbreitet in Frankreich
  - **Diners Club** – Premium-Internationale Karte mit exklusiven Vorteilen
  - **Koreanische lokale Karten** – Unterstützung für koreanische Inlands-Kartennetzwerke
</Accordion>

<Accordion title="Digitale Geldbörsen">
  Bieten Sie Kunden einen schnellen und sicheren Checkout mit digitalen Geldbörsen an:

  - **Apple Pay** – Einfaches Bezahlen auf iOS, macOS und watchOS
  - **Google Pay** – Sichere Zahlungen auf Android, im Web und im Geschäft
  - **PayPal** – Globale Zahlungen in EUR und GBP (nicht verfügbar für Abonnements). <strong>Hinweis: Der PayPal-Support ist derzeit pausiert.</strong>
  - **Cash App Pay** – Mobile Zahlungen für US-Kunden (nicht verfügbar für Abonnements)
  - **Amazon Pay** – Verwenden Sie Amazon-Anmeldeinformationen für einen einfachen Checkout (nicht verfügbar für Abonnements)
</Accordion>

<Accordion title="Jetzt kaufen, später bezahlen (BNPL)">
  Erhöhen Sie die Konversion, indem Sie flexible Ratenzahlungsoptionen anbieten:

  - **Klarna** – Ermöglichen Sie flexible Zahlungen mit Klarna’s Optionen: in 4 Raten zahlen, in 30 Tagen zahlen oder Ihren Kauf finanzieren. Klarna ist in den USA und 19 europäischen Ländern verfügbar, darunter Österreich, Belgien, Tschechische Republik, Dänemark, Finnland, Frankreich, Deutschland, Griechenland, Irland, Italien, Niederlande, Norwegen, Polen, Portugal, Rumänien, Spanien, Schweden, Schweiz und das Vereinigte Königreich. Unterstützte Währungen sind USD, EUR, GBP, DKK, NOK, SEK, CZK, RON, PLN und CHF.
  - **Afterpay** – Ermöglichen Sie es Kunden, Käufe in vier zinsfreien Zahlungen aufzuteilen. Afterpay ist in den USA und im Vereinigten Königreich verfügbar und unterstützt Zahlungen in USD und GBP.
</Accordion>

<Accordion title="UPI und Rupay (Indien)" id="upi-and-rupay-india">

  - **UPI** – Kunden scannen einen QR-Code oder verwenden Apps (Google Pay, PhonePe, Paytm, CRED) für sofortige, kontaktlose Banküberweisungen
  - **Rupay-Karten** – Sichere Kredit- und Debit-Zahlungen über Indiens Inlands-Kartennetzwerk

  - **Bancontact** (Belgien) – Belgiens vertrauenswürdige Debitkartenmethode

## Konfigurieren von Zahlungsmethoden

Während alle Zahlungsmethoden automatisch basierend auf dem Standort und der Währung des Kunden verfügbar sind, können Sie optional einschränken, welche Zahlungsoptionen während des Checkouts angezeigt werden, indem Sie das `allowed_payment_method_types` Parameter verwenden.

### Wann Zahlungsmethoden konfigurieren

Sie möchten möglicherweise die erlaubten Zahlungsmethoden konfigurieren, um:

- Zu steuern, welche Zahlungsoptionen den Kunden basierend auf Ihren Geschäftsanforderungen angezeigt werden
- Zahlungsmethoden für bestimmte Produkttypen oder Transaktionsabläufe einzuschränken
- Konsistenz über verschiedene Zahlungserlebnisse hinweg aufrechtzuerhalten
- Das Checkout-Erlebnis für Ihre Zielgruppe zu optimieren

### Implementierung

<Steps>
<Step title="Parameter zur API-Anfrage hinzufügen">
Fügen Sie das `allowed_payment_method_types` Feld in Ihre Zahlungs- oder Abonnementerstellungsanfrage ein:

```json
{
  "allowed_payment_method_types": [
    "credit",
    "debit",
    "apple_pay"
  ]
}
```

</Step>

<Step title="Konfiguration testen">
Überprüfen Sie, ob nur die angegebenen Zahlungsmethoden an der Kasse angezeigt werden.

<Tip>
Testen Sie die Verfügbarkeit von Zahlungsmethoden mit verschiedenen Kundenstandorten und Währungen, um sicherzustellen, dass die Konfiguration wie erwartet funktioniert.
</Tip>
</Step>
</Steps>

### Best Practices

<AccordionGroup>
<Accordion title="Standardverhalten">
Wenn das `allowed_payment_method_types` nicht angegeben ist, sind alle berechtigten Zahlungsmethoden basierend auf dem Standort, der Währung und dem Transaktionstyp des Kunden verfügbar. Verwenden Sie dieses Parameter nur, wenn Sie Zahlungsmethoden einschränken müssen.
</Accordion>

<Accordion title="Regionale Überlegungen">
Berücksichtigen Sie Ihren Zielmarkt bei der Auswahl von Zahlungsmethoden. Die Einbeziehung regionsspezifischer Optionen (wie UPI für Indien oder iDEAL für die Niederlande) kann die Konversionsraten in diesen Märkten erheblich verbessern.
</Accordion>

<Accordion title="Überwachen und optimieren">
Verfolgen Sie, welche Zahlungsmethoden Ihre Kunden am häufigsten verwenden, und passen Sie Ihre erlaubten Methoden basierend auf tatsächlichen Nutzungsmustern und Konversionsdaten an.
</Accordion>
</AccordionGroup>

### Wichtige Überlegungen

Die Verfügbarkeit von Zahlungsmethoden hängt von mehreren Faktoren ab:
- **Kundenstandort**: Einige Zahlungsmethoden sind regionsspezifisch
- **Händler-Einstellungen**: Ihre Kontokonfiguration und -fähigkeiten
- **Währung**: Unterstützte Zahlungsmethoden variieren je nach Währung
- **Transaktionstyp**: Einige Methoden unterstützen keine Abonnements oder haben Mindestbeträge

<Note>
Das Hinzufügen einer Zahlungsmethode zu `allowed_payment_method_types` garantiert nicht, dass sie angezeigt wird - sie muss auch basierend auf diesen Faktoren berechtigt sein.
</Note>

## Fehlersuche

<AccordionGroup>
<Accordion title="Zahlungsmethode erscheint nicht beim Checkout">
**Mögliche Ursachen:**
- Die Zahlungsmethode ist nicht in `allowed_payment_method_types` enthalten
- Die Zahlungsmethode wird für den Standort des Kunden nicht unterstützt
- Die Zahlungsmethode unterstützt die Transaktionswährung nicht
- Für Abonnements: die Zahlungsmethode unterstützt keine wiederkehrenden Zahlungen
- Der Transaktionsbetrag liegt unter dem Mindestbetrag (z. B. BNPL-Methoden)

**Lösung:** Überprüfen Sie, ob alle Berechtigungsanforderungen erfüllt sind, und überprüfen Sie Ihre API-Anfrage-Konfiguration.
</Accordion>

<Accordion title="Unerwartete Zahlungsmethoden erscheinen">
**Mögliche Ursachen:**
- Das `allowed_payment_method_types` Parameter wird nicht korrekt übergeben
- Es kann Konflikte in Ihren Händler-Dashboard-Konfigurationen geben

**Lösung:** Überprüfen Sie Ihre API-Anfrage-Nutzlast und stellen Sie sicher, dass der Parameter korrekt als Array von Strings formatiert ist.
</Accordion>

<Accordion title="Regionale Zahlungsmethode nicht verfügbar">
**Mögliche Ursachen:**
- Der Standort des Kunden stimmt nicht mit der unterstützten Region der Zahlungsmethode überein
- Die Währung wird von der regionalen Zahlungsmethode nicht unterstützt
- Ihr Händlerkonto ist möglicherweise nicht für diese spezifische Zahlungsmethode aktiviert

**Lösung:** Überprüfen Sie die regionale und Währungs-Kompatibilität und kontaktieren Sie den Support, wenn Sie glauben, dass die Methode verfügbar sein sollte.
</Accordion>

<Accordion title="Probleme mit indischen Zahlungsmethoden bei Abonnements">
**Abonnementgebühr nach Upgrade/Downgrade fehlgeschlagen:**
- Der neue Gebührenbetrag kann das bestehende Mandatslimit (Schwelle von Rs 15.000) überschreiten
- Der Kunde muss möglicherweise seine Zahlungsmethode aktualisieren, um ein neues Mandat zu erstellen

**Lösung:** Fordern Sie den Kunden auf, seine Zahlungsmethode zu aktualisieren oder den Abonnementbetrag anzupassen, um innerhalb der Mandatsgrenzen zu bleiben.

**Abonnement pausiert nach Initiierung der Gebühr:**
- Der Kunde hat möglicherweise das Mandat während des 48-stündigen Bearbeitungsfensters storniert
- Der Kunde hat möglicherweise eine hochpreisige Gebühr (≥Rs 15.000) nicht autorisiert

**Lösung:** Überwachen Sie Webhook-Ereignisse für Mandatsstornierungen und Zahlungsfehler. Implementieren Sie eine Wiederholungslogik oder fordern Sie die Kunden auf, ihre Zahlungsmethode erneut zu etablieren.

**Zahlungsabzug verzögert:**
- Normales Verhalten: Indische Karten- und UPI-Abonnements haben eine 48-stündige Verzögerung zwischen der Initiierung der Gebühr und dem tatsächlichen Abzug
- Diese Verzögerung kann je nach Bank-APIs um 2-3 zusätzliche Stunden verlängert werden

**Lösung:** Dies ist erwartetes Verhalten. Aktivieren Sie keine Vorteile oder Gutschriften, bis die Zahlungsbestätigung über Webhooks erhalten wurde.
</Accordion>

<Accordion title="Zahlungsmethoden für Abonnements aktualisieren">
**Wann Zahlungsmethoden aktualisiert werden sollten:**
- Die Karte des Kunden läuft ab oder wird ersetzt
- Der Kunde möchte eine andere Zahlungsmethode verwenden
- Das Abonnement befindet sich im `on_hold` Zustand aufgrund einer fehlgeschlagenen Zahlung
- Die Zahlungsmethode muss aus irgendeinem Grund geändert werden

**Für aktive Abonnements:**
Das Aktualisieren der Zahlungsmethode für ein aktives Abonnement aktualisiert einfach die Zahlungsmethode für zukünftige Erneuerungen. Es wird keine sofortige Gebühr erstellt.

**Für on_hold Abonnements:**
Wenn sich ein Abonnement im `on_hold` Zustand befindet (aufgrund einer fehlgeschlagenen Verlängerung oder einer fehlgeschlagenen Planänderungsgebühr), wird die Aktualisierung der Zahlungsmethode:
- Automatisch eine Gebühr für ausstehende Beträge erstellen
- Eine Rechnung für die Gebühr generieren
- Die Zahlung mit der neuen Zahlungsmethode verarbeiten
- Das Abonnement in den `active` Zustand reaktivieren, sobald die Zahlung erfolgreich war

<CodeGroup>

```javascript Node.js
// Update payment method for active subscription
await client.subscriptions.updatePaymentMethod('sub_123', {
  type: 'new',
  return_url: 'https://example.com/return'
});

// Or use existing payment method
await client.subscriptions.updatePaymentMethod('sub_123', {
  type: 'existing',
  payment_method_id: 'pm_abc123'
});

// For on_hold subscription - automatically creates charge
const response = await client.subscriptions.updatePaymentMethod('sub_123', {
  type: 'new',
  return_url: 'https://example.com/return'
});

if (response.payment_id) {
  // Charge created for remaining dues
  // Redirect customer to response.payment_link
  // Monitor webhooks: payment.succeeded → subscription.active
}
```

```python Python
# Update payment method for active subscription
client.subscriptions.update_payment_method(
    subscription_id="sub_123",
    type="new",
    return_url="https://example.com/return"
)

# Or use existing payment method
client.subscriptions.update_payment_method(
    subscription_id="sub_123",
    type="existing",
    payment_method_id="pm_abc123"
)

# For on_hold subscription - automatically creates charge
response = client.subscriptions.update_payment_method(
    subscription_id="sub_123",
    type="new",
    return_url="https://example.com/return"
)

if response.payment_id:
    # Charge created for remaining dues
    # Redirect customer to response.payment_link
    # Monitor webhooks: payment.succeeded → subscription.active
```

</CodeGroup>

**Webhook-Ereignisse:**
- Für `on_hold` Abonnements: Sie erhalten `payment.succeeded` gefolgt von `subscription.active` nach erfolgreicher Aktualisierung der Zahlungsmethode
- Überwachen Sie diese Ereignisse, um den Status der Abonnementreaktivierung zu verfolgen
