Dodo Payments unterstützt eine Vielzahl von Zahlungsmethoden, um Ihnen zu helfen, Kunden weltweit zu bedienen. Aktivieren Sie neue Zahlungsmethoden sofort – keine zusätzlichen Verträge, Integrationen oder Händlerkonten erforderlich.

## Warum mehrere Zahlungsmethoden anbieten?

</Card>

</Card>

</Card>

## So funktioniert es

Zahlungsmethoden werden automatisch basierend auf Folgendem angezeigt:

- **Standort des Kunden** — Regionale Methoden wie iDEAL (Niederlande) oder UPI (Indien) erscheinen nur für Kunden in diesen Regionen
- **Transaktionswährung** — Zahlungsmethoden werden nach unterstützten Währungen gefiltert
- **Gerätetyp** — Apple Pay erscheint nur auf kompatiblen Apple-Geräten und -Browserversionen
- **Transaktionstyp** — Einige Methoden unterstützen keine Abonnements oder haben Mindestbeträge

<Info>
**Automatische Aktivierung:** Alle Zahlungsmethoden sind vorkonfiguriert und werden von Dodo Payments verwaltet. Es ist keine zusätzliche Einrichtung erforderlich — sie sind verfügbar, sobald Ihr Konto verifiziert und der Live-Modus aktiviert ist.
</Info>

## Konfiguration von Zahlungsmethoden

Standardmäßig werden alle berechtigten Zahlungsmethoden automatisch angezeigt. Verwenden Sie `allowed_payment_method_types`, um festzulegen, welche Methoden angezeigt werden:

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://test.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

</Warning>

## Beste Praktiken

<AccordionGroup>
<Accordion title="Methoden an Ihren Markt anpassen">
Berücksichtigen Sie regionale Zahlungsmethoden für Ihre Zielmärkte. iDEAL erfasst ~60 % der Online-Zahlungen in den Niederlanden; UPI dominiert in Indien. Das Weglassen dieser Methoden schadet der Conversion erheblich.
</Accordion>

<Accordion title="Immer Karten-Fallbacks bereitstellen">
Auch wenn bestimmte Methoden angestrebt werden, fügen Sie `credit` und `debit` als Fallbacks hinzu. Nicht alle Kunden haben Zugang zu jeder Zahlungsmethode.
</Accordion>

<Accordion title="Berücksichtigen Sie Transaktionsminimums">
BNPL-Methoden (Klarna, Afterpay) erfordern einen Minimalbetrag von 50,01 $. Schließen Sie sie nicht für Produkte mit niedrigem Wert ein – sie werden einfach nicht angezeigt, was in Ordnung ist.
</Accordion>

<Accordion title="Über Regionen und Geräte testen">
Verwenden Sie den Testmodus, um die Verfügbarkeit von Zahlungsmethoden für verschiedene Kundenstandorte, Währungen und Gerätetypen vor dem Live-Gang zu überprüfen.
</Accordion>
</AccordionGroup>

## Fehlersuche

<AccordionGroup>
<Accordion title="Zahlungsmethode erscheint nicht">
**Überprüfen Sie diese Faktoren:**
1. Ist die Methode in `allowed_payment_method_types` enthalten?
2. Passt der Standort des Kunden zur unterstützten Region der Methode?
3. Ist die Transaktionswährung von der Methode unterstützt?
4. Erfüllt der Transaktionsbetrag das Minimum (z. B. 50,01 $ für BNPL)?
5. Für Abonnements: Unterstützt die Methode wiederkehrende Zahlungen?

**Lösung:** Entfernen Sie `allowed_payment_method_types` vorübergehend, um alle berechtigten Methoden anzuzeigen, und schränken Sie dann das Problem ein.
</Accordion>

<Accordion title="Unerwartete Methoden erscheinen">
**Mögliche Ursachen:**
- `allowed_payment_method_types` wird nicht korrekt übermittelt
- Parameter ist falsch formatiert (muss ein Array von Strings sein)

**Lösung:** Protokollieren Sie Ihre API-Anforderungsnutzlast und überprüfen Sie die Parameterstruktur.
</Accordion>

<Accordion title="Regionale Methode nicht verfügbar">
**Überprüfen Sie:**
- Entspricht die Rechnungsadresse des Kunden der Region der Methode?
- Ist die Währung korrekt festgelegt (z. B. EUR für iDEAL)?
- Ist Adaptive Currency aktiviert, wenn der Kunde international ist?

**Lösung:** Überprüfen Sie den Länder-Code der Rechnungsadresse und die Währung in Ihrer API-Anforderung.
</Accordion>

<Accordion title="Probleme mit indischen Zahlungsmethoden">
UPI- und Rupay-Abonnements haben besondere Anforderungen, die 48-Stunden-Verzögerungen bei der Verarbeitung und RBI-Mandatsgrenzen umfassen.

**Siehe:** <a href="/features/payment-methods/india">Indische Zahlungsmethoden</a> für umfassende Anleitungen.
</Accordion>
</AccordionGroup>
