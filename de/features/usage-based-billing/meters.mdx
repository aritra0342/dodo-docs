<Info>
Zähler wandeln Rohereignisse in abrechenbare Mengen um. Sie filtern Ereignisse und wenden Aggregationsfunktionen (Count, Sum, Max, Last) an, um die Nutzung pro Kunde zu berechnen.
</Info>

## Einen Zähler erstellen

<Steps>
{/* LOCKED_PATTERN_148f3cd6016b0fbbcdca7a493126f782 */}
{/* LOCKED_PATTERN_29b9e112369bf16db8b48662b2a38ae8 */}
Beschreibender Name (z. B. „API Requests“, „Token Usage“)
</ParamField>

<ParamField path="Event Name" type="string" required>
Exakter Ereignisname, der übereinstimmen muss (Groß-/Kleinschreibung beachten). Beispiele: `api.call`, `image.generated`
</ParamField>
</Step>

<Step title="Aggregation">
{/* LOCKED_PATTERN_97313bb6bf41a89f8d2567ad673466f3 */}
Wählen Sie aus, wie Ereignisse aggregiert werden:

- **Count**: Gesamtanzahl von Ereignissen (API-Aufrufe, Uploads)
- **Sum**: Addition numerischer Werte (Tokens, Bytes)
- **Max**: Höchster Wert im Zeitraum (Spitzenbenutzer)
- **Last**: Aktuellster Wert
</ParamField>

<ParamField path="Over Property" type="string">
Metadaten-Schlüssel zur Aggregation (erforderlich für alle Typen außer Count). Beispiele: `tokens`, `bytes`, `duration_ms`
</ParamField>

<ParamField path="Measurement Unit" type="string" required>
Einheitenbezeichnung für Rechnungen. Beispiele: `calls`, `tokens`, `GB`, `hours`
</ParamField>
</Step>

{/* LOCKED_PATTERN_709e3e0ef72c95833fd8d928d7a0591a */}
<Frame>
<img src="/images/usage-based/UBB-3.png" alt="Ereignisfilterung" />
</Frame>

Fügen Sie Bedingungen hinzu, um zu filtern, welche Ereignisse gezählt werden:
- **UND-Logik**: Alle Bedingungen müssen übereinstimmen
- **ODER-Logik**: Jede Bedingung kann übereinstimmen

**Vergleichsoperatoren**: gleich, ungleich, größer als, kleiner als, enthält

Aktivieren Sie Filter, wählen Sie die Logik, und fügen Sie Bedingungen mit Property-Key, Vergleichsoperator und Wert hinzu.
</Step>

{/* LOCKED_PATTERN_314e5b01027621505255760af9937f80 */}
Überprüfen Sie die Konfiguration und klicken Sie auf **Create Meter**.
</Step>
</Steps>

## Fehlersuche

<AccordionGroup>
{/* LOCKED_PATTERN_fc1788ddf0982a3a2fe9f6e81812c541 */}
- Der Ereignisname muss exakt übereinstimmen (Groß-/Kleinschreibung beachten)
- Stellen Sie sicher, dass Meterfilter Ereignisse nicht ausschließen
- Verifizieren Sie, dass Kunden-IDs existieren
- Deaktivieren Sie Filter vorübergehend zum Testen
</Accordion>

{/* LOCKED_PATTERN_ddafb5b33ecc6ac484aee2e1a2c4cd7d */}
- Stellen Sie sicher, dass Over Property genau dem Metadaten-Schlüssel entspricht
- Verwenden Sie Zahlen, keine Strings: `tokens: 150` nicht `"150"`
- Schließen Sie erforderliche Eigenschaften in allen Ereignissen ein
</Accordion>

{/* LOCKED_PATTERN_7673f2ad014dbdbdfd53c822f111f71d */}
- Achten Sie auf exakte Groß-/Kleinschreibung
- Verwenden Sie die richtigen Operatoren für den Datentyp
- Stellen Sie sicher, dass Ereignisse die gefilterten Eigenschaften enthalten
</Accordion>

{/* LOCKED_PATTERN_9641c2e1836928ca6e48f68372e5b21d */}
- Prüfen Sie die Registerkarte „Events“, um die tatsächlich empfangenen Ereignisse zu zählen
- Vergewissern Sie sich, dass der Aggregationstyp korrekt ist (Count vs. Sum)
- Stellen Sie sicher, dass Werte für Sum/Max numerisch sind
</Accordion>
</AccordionGroup>
