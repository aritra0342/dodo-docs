Ereignisse sind die Grundlage der nutzungsbasierten Abrechnung. Senden Sie Ereignisse, wenn abrechnungsfähige Aktionen stattfinden, und Zähler aggregieren sie in Gebühren.

## Ereignisstruktur

{/* LOCKED_PATTERN_4d7f2e9ce45e7323dd2d626df4a7c5b8 */}
Einzigartige Kennung. Verwenden Sie UUIDs oder kombinieren Sie Kunden-ID + Zeitstempel + Aktion.
</ParamField>

<ParamField body="customer_id" type="string" required>
Dodo Payments Kunden-ID. Muss ein gültiger bestehender Kunde sein.
</ParamField>

<ParamField body="event_name" type="string" required>
Ereignistyp, der genau mit dem Ereignisnamen Ihres Zählers übereinstimmt (Groß-/Kleinschreibung beachten). Beispiele: `api.call`, `image.generated`
</ParamField>
</Accordion>

{/* LOCKED_PATTERN_94ee55474fc7877494948c444fe08c91 */}

<ParamField body="timestamp" type="string">
ISO-8601-Zeitstempel. Standardmäßig Serverzeit, wenn ausgelassen. Für genaue Abrechnung bei verzögerten/Batch-Ereignissen angeben.
</ParamField>

<ParamField body="metadata" type="object">
Zusätzliche Eigenschaften für Aggregation und Filterung:
- Numerische Werte: `bytes`, `tokens`, `duration_ms`
- Filter: `endpoint`, `method`, `quality`

```javascript
metadata: {
  endpoint: "/v1/orders",
  method: "POST",
  tokens: 1024
}
```

</ParamField>
</Accordion>

## Ereignisse senden

<CodeGroup>

```javascript Single Event
await fetch('https://test.dodopayments.com/events/ingest', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.DODO_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    events: [{
      event_id: "api_call_1234",
      customer_id: "cus_abc123",
      event_name: "api.call",
      metadata: { endpoint: "/v1/orders" }
    }]
  })
});
```

```javascript Batch Events
await fetch('https://test.dodopayments.com/events/ingest', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.DODO_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    events: [
      { event_id: "call_1", customer_id: "cus_abc", event_name: "api.call" },
      { event_id: "call_2", customer_id: "cus_abc", event_name: "api.call" },
      { event_id: "call_3", customer_id: "cus_abc", event_name: "api.call" }
    ]
  })
});
```

```python Python
import requests

requests.post(
    'https://test.dodopayments.com/events/ingest',
    headers={'Authorization': f'Bearer {api_key}', 'Content-Type': 'application/json'},
    json={'events': [{'event_id': 'call_1', 'customer_id': 'cus_abc', 'event_name': 'api.call'}]}
)
```

```curl cURL
curl -X POST 'https://test.dodopayments.com/events/ingest' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{"events": [{"event_id": "call_1", "customer_id": "cus_abc", "event_name": "api.call"}]}'
```

</CodeGroup>

<Tip>
Batchen Sie bis zu 100 Ereignisse pro Anfrage für bessere Performance.
</Tip>

## Best Practices

<AccordionGroup>
{/* LOCKED_PATTERN_c11962fb3ea9e1a82d1994f0860ee77a */}
Verwenden Sie deterministische IDs, um Duplikate zu vermeiden: `${customerId}_${action}_${timestamp}`
</Accordion>

{/* LOCKED_PATTERN_d4b029e15b52ca6ac907535b9dad9ee9 */}
Wiederholen Sie bei 5xx-Fehlern mit exponentiellem Backoff. Wiederholen Sie nicht bei 4xx-Fehlern.
</Accordion>

{/* LOCKED_PATTERN_90e261cbcc49667becacc297210d42c3 */}
Für Echtzeit-Ereignisse weglassen. Für verzögerte/Batch-Ereignisse zur Genauigkeit einbeziehen.
</Accordion>

{/* LOCKED_PATTERN_fa83e100effc6c87ff1c68c56ea8afa9 */}
Verfolgen Sie Erfolgsraten und fügen Sie fehlgeschlagene Ereignisse zur erneuten Verarbeitung in die Warteschlange.
</Accordion>
</AccordionGroup>

## Fehlersuche

<AccordionGroup>
{/* LOCKED_PATTERN_fc1788ddf0982a3a2fe9f6e81812c541 */}
- Der Ereignisname muss exakt mit dem Zähler übereinstimmen (Groß-/Kleinschreibung beachten)
- Die Kunden-ID muss existieren
- Überprüfen Sie, dass Zählerfilter Ereignisse nicht ausschließen
- Überprüfen Sie, ob Zeitstempel aktuell sind
</Accordion>

{/* LOCKED_PATTERN_1af6f154c82eaebbb38a748cef2be54a */}
Stellen Sie sicher, dass der API-Schlüssel korrekt ist, und verwenden Sie das Format: `Bearer YOUR_API_KEY`
</Accordion>

{/* LOCKED_PATTERN_bfe0427c6207c44242acb102b2467be5 */}
Stellen Sie sicher, dass alle erforderlichen Felder vorhanden sind: `event_id`, `customer_id`, `event_name`
</Accordion>

{/* LOCKED_PATTERN_61cd934926cd73f663eac6935ef56a1b */}
- Metadaten-Schlüssel müssen genau mit der "Over Property" des Zählers übereinstimmen
- Verwenden Sie Zahlen, keine Zeichenketten: `tokens: 150` nicht `tokens: "150"`
</Accordion>
</AccordionGroup>
