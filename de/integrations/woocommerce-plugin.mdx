## Einführung

Das Dodo Payments WooCommerce-Plugin bietet eine nahtlose Integration mit Ihrem Shop, sodass Sie Zahlungen über mehrere Methoden, einschließlich UPI, Karten, Online-Banking und digitale Geldbörsen, akzeptieren können. Diese umfassende Anleitung führt Sie durch den Installations- und Konfigurationsprozess.

## Konfigurationsanleitung

### API-Schlüssel und Webhooks einrichten

1. Navigieren Sie zu **WooCommerce → Einstellungen → Zahlungen** oder klicken Sie auf die Schaltfläche **Zahlungen** in der linken Seitenleiste unter dem WooCommerce-Menüpunkt
    
    <Frame>
        <img src="/images/woocommerce/6.webp" alt="Dodo Payments for WooCommerce Plugin" />
    </Frame>
    
    <Frame>
        <img src="/images/woocommerce/7.webp" alt="Dodo Payments for WooCommerce Plugin" />
    </Frame>
    
2. Aktivieren Sie den Zahlungsanbieter **Dodo Payments**, falls noch nicht aktiviert, und klicken Sie dann auf **Verwalten**, um das Plugin zu konfigurieren
3. Sie sehen verschiedene Konfigurationsoptionen mit hilfreichem erläuterndem Text. Beginnen Sie mit der Einrichtung Ihres Live-API-Schlüssels
    
    <Frame>
        <img src="/images/woocommerce/8.webp" alt="Dodo Payments for WooCommerce Plugin" />
    </Frame>
    
4. Melden Sie sich im **Live-Modus** Ihres Dodo Payments-Dashboards an, navigieren Sie dann zu **Dodo Payments (Live Mode) > Developer > API Keys** oder besuchen Sie [diesen Direktlink](https://app.dodopayments.com/developer/api-keys) und klicken Sie auf **API-Schlüssel hinzufügen**
    
    <Frame>
        <img src="/images/woocommerce/9.webp" alt="Dodo Payments for WooCommerce Plugin" />
    </Frame>
    
5. Geben Sie Ihrem API-Schlüssel einen aussagekräftigen Namen und klicken Sie auf **Erstellen**
    
    <Frame>
        <img src="/images/woocommerce/10.webp" alt="Dodo Payments for WooCommerce Plugin" />
    </Frame>
    
6. Kopieren Sie den generierten API-Schlüssel und fügen Sie ihn in das Feld **Live API Key** in Ihren WooCommerce-Plugin-Einstellungen ein
    
    <Frame>
        <img src="/images/woocommerce/11.webp" alt="Dodo Payments for WooCommerce Plugin" />
    </Frame>
    
    <Frame>
        <img src="/images/woocommerce/12.webp" alt="Dodo Payments for WooCommerce Plugin" />
    </Frame>
    
7. Richten Sie als Nächstes den Webhook-Signaturschlüssel ein, um die Synchronisierung des Zahlungsstatus zwischen Dodo Payments und WooCommerce zu ermöglichen
8. Scrollen Sie bis zum Ende der Plugin-Einstellungsseite und kopieren Sie die dort angezeigte Webhook-URL
    
    <Frame>
        <img src="/images/woocommerce/13.webp" alt="Dodo Payments for WooCommerce Plugin" />
    </Frame>
    
9. Kehren Sie zu Ihrem Dodo Payments-Dashboard zurück, navigieren Sie zu **Dodo Payments (Live Mode) > Developer > Webhooks** und klicken Sie auf **Webhook hinzufügen**
    
    <Frame>
        <img src="/images/woocommerce/14.webp" alt="Dodo Payments for WooCommerce Plugin" />
    </Frame>
    
10. Fügen Sie die in Schritt 8 kopierte URL in das Dialogfeld ein und klicken Sie auf **Webhook hinzufügen**
    
    <Frame>
        <img src="/images/woocommerce/15.webp" alt="Dodo Payments for WooCommerce Plugin" />
    </Frame>
    
11. Klicken Sie nach dem Erstellen des Webhooks auf das Augensymbol neben dem geschwärzten **Signing Key**, um ihn anzuzeigen und zu kopieren
    
    <Frame>
        <img src="/images/woocommerce/16.webp" alt="Dodo Payments for WooCommerce Plugin" />
    </Frame>
    
12. Fügen Sie den Signing Key in das Feld **Live Webhook Signing Key** in Ihren Plugin-Einstellungen ein und speichern Sie die Änderungen
    
    <Frame>
        <img src="/images/woocommerce/17.webp" alt="Dodo Payments for WooCommerce Plugin" />
    </Frame>
    
13. Überprüfen Sie die verbleibenden Einstellungen wie **Global Tax Category** und **All Prices are Tax Inclusive**, da diese Optionen bestimmen, wie Produkte von WooCommerce zu Dodo Payments synchronisiert werden. Beachten Sie, dass **Test API Key** und **Test Webhook Signing Key** nur erforderlich sind, wenn Sie den Testmodus verwenden möchten
    
    <Frame>

## Hauptmerkmale

- **Nahtloses Checkout-Erlebnis**: Leiten Sie Kunden zu einer sicheren, optimierten Checkout-Seite weiter
- **Mehrere Zahlungsmethoden**: Unterstützung für verschiedene lokale und internationale Zahlungsoptionen
- **Echtzeit-Statusaktualisierungen**: Sofortige Synchronisierung des Bestellstatus über Webhooks
- **Detaillierte Transaktionsaufzeichnungen**: Umfassende Zahlungshistorie in Ihrem WooCommerce-Dashboard
- **Mehrwährungsunterstützung**: Akzeptieren Sie Zahlungen in mehreren Währungen

## Fehlersuche

### Häufige Probleme

1. **Zahlungsverarbeitungsfehler**
   - Überprüfen Sie, ob die API-Schlüssel korrekt konfiguriert sind
   - Stellen Sie sicher, dass die Webhook-URL richtig eingerichtet ist
   - Überprüfen Sie, ob Ihr SSL-Zertifikat gültig ist

2. **Webhook-Synchronisierungsprobleme**
   - Bestätigen Sie, dass der Webhook-Signierschlüssel übereinstimmt
   - Überprüfen Sie die Serverprotokolle auf mögliche Fehler
   - Stellen Sie sicher, dass Ihr Server eingehende Webhook-Anfragen empfangen kann

## Wartungsbest Practices

1. Aktualisieren Sie das Plugin regelmäßig für Sicherheitsupdates und neue Funktionen
2. Testen Sie alle Zahlungsflüsse im Sandbox-Modus, bevor Sie Live-Transaktionen verarbeiten
3. Halten Sie WordPress und WooCommerce auf die neuesten Versionen aktualisiert
4. Implementieren Sie regelmäßige Site-Backups
5. Überwachen Sie Transaktionsprotokolle auf ungewöhnliche Aktivitäten
6. Bewahren Sie API-Schlüssel sicher auf und teilen Sie sie niemals öffentlich
