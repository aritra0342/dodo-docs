## Voraussetzungen

Um die Dodo Payments API zu integrieren, benötigen Sie:
- Ein Dodo Payments Händlerkonto
- API-Anmeldeinformationen (API-Schlüssel und Webhook-Geheimschlüssel) aus dem Dashboard

## Dashboard-Einrichtung

1. Navigieren Sie zum [Dodo Payments Dashboard](https://app.dodopayments.com/)

2. Erstellen Sie ein Produkt (Einmalzahlung oder Abonnement)

4. Generieren Sie Ihren API-Schlüssel:
   - Gehen Sie zu Entwickler > API
   - [Detaillierte Anleitung](/api-reference/introduction#api-key-generation)
   - Kopieren Sie den API-Schlüssel in die Umgebungsvariable DODO_PAYMENTS_API_KEY

5. Konfigurieren Sie Webhooks:
   - Gehen Sie zu Entwickler > Webhooks
   - Erstellen Sie eine Webhook-URL für Zahlungsbenachrichtigungen
   - Kopieren Sie den Webhook-Geheimschlüssel in die Umgebungsvariable

## Integration

### Zahlungslinks

Wählen Sie den Integrationspfad, der zu Ihrem Anwendungsfall passt:
- **Checkout-Sitzungen (empfohlen)**: Am besten für die meisten Integrationen. Erstellen Sie eine Sitzung auf Ihrem Server und leiten Sie die Kunden zu einem sicheren, gehosteten Checkout weiter.
- **Overlay-Checkout (eingebettet)**: Verwenden Sie dies, wenn Sie eine In-Page-Erfahrung benötigen, die den gehosteten Checkout auf Ihrer Website einbettet.
- **Statische Zahlungslinks**: No-Code, sofort teilbare URLs zur schnellen Zahlungsabwicklung.
- **Dynamische Zahlungslinks**: Programmgesteuert erstellte Links. Checkout-Sitzungen werden jedoch empfohlen und bieten mehr Flexibilität.

#### 1. Checkout-Sitzungen

Verwenden Sie Checkout-Sitzungen, um eine sichere, gehostete Checkout-Erfahrung für Einmalzahlungen oder Abonnements zu schaffen. Sie erstellen eine Sitzung auf Ihrem Server und leiten den Kunden dann zur zurückgegebenen `checkout_url` weiter.

<Info>
Checkout-Sitzungen sind standardmäßig 24 Stunden gültig. Wenn Sie <code>confirm=true</code> übergeben, sind die Sitzungen 15 Minuten gültig und alle erforderlichen Felder müssen bereitgestellt werden.
</Info>

<Steps>
<Step title="Erstellen Sie eine Checkout-Sitzung">
  Wählen Sie Ihr bevorzugtes SDK oder rufen Sie die REST-API auf.

  <Tabs>
  <Tab title="Node.js SDK">

  ```javascript
  import DodoPayments from 'dodopayments';

  const client = new DodoPayments({
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    environment: 'test_mode', // defaults to 'live_mode'
  });

  const session = await client.checkoutSessions.create({
    product_cart: [{ product_id: 'prod_123', quantity: 1 }],
    customer: { email: 'customer@example.com', name: 'John Doe' },
    return_url: 'https://yourapp.com/checkout/success',
  });
  ```

  </Tab>

#### 2. Overlay-Checkout

Für eine nahtlose In-Page-Checkout-Erfahrung erkunden Sie unsere [Overlay-Checkout](/developer-resources/overlay-checkout) Integration, die es Kunden ermöglicht, Zahlungen abzuschließen, ohne Ihre Website zu verlassen.

#### 3. Statische Zahlungslinks

Statische Zahlungslinks ermöglichen es Ihnen, Zahlungen schnell zu akzeptieren, indem Sie eine einfache URL teilen. Sie können das Checkout-Erlebnis anpassen, indem Sie Abfrageparameter übergeben, um Kundendetails vorauszufüllen, Formularfelder zu steuern und benutzerdefinierte Metadaten hinzuzufügen.

<Steps>
<Step title="Konstruktion Ihres Zahlungslinks">
  Beginnen Sie mit der Basis-URL und fügen Sie Ihre Produkt-ID hinzu:

  ```text
  https://checkout.dodopayments.com/buy/{productid}
  ```

</Step>
<Step title="Fügen Sie Kernparameter hinzu">
  Fügen Sie wesentliche Abfrageparameter hinzu:
  - <ParamField query="quantity" type="integer" default="1">Anzahl der zu kaufenden Artikel.</ParamField>
  - <ParamField query="redirect_url" type="string" required>URL, um nach Abschluss der Zahlung weiterzuleiten.</ParamField>
  <Note>
  Die Weiterleitungs-URL enthält Zahlungsdetails als Abfrageparameter, zum Beispiel:<br/>
  <code>https://example.com/?payment_id=pay_ts2ySpzg07phGeBZqePbH&status=succeeded</code>
  </Note>
</Step>
<Step title="Kundendaten vorausfüllen (optional)">
  Fügen Sie Kunden- oder Rechnungsfelder als Abfrageparameter hinzu, um den Checkout zu optimieren.
  <AccordionGroup>
    <Accordion title="Unterstützte Kundenfelder">
      - <ParamField query="fullName" type="string">Vollständiger Name des Kunden (wird ignoriert, wenn firstName oder lastName angegeben sind).</ParamField>
      - <ParamField query="firstName" type="string">Vorname des Kunden.</ParamField>
      - <ParamField query="lastName" type="string">Nachname des Kunden.</ParamField>
      - <ParamField query="email" type="string">E-Mail-Adresse des Kunden.</ParamField>
      - <ParamField query="country" type="string">Land des Kunden.</ParamField>
      - <ParamField query="addressLine" type="string">Straßenadresse.</ParamField>
      - <ParamField query="city" type="string">Stadt.</ParamField>
      - <ParamField query="state" type="string">Bundesland oder Provinz.</ParamField>
      - <ParamField query="zipCode" type="string">Postleitzahl.</ParamField>
      - <ParamField query="showDiscounts" type="boolean">true oder false</ParamField>
    </Accordion>
  </AccordionGroup>
</Step>
<Step title="Formularfelder steuern (optional)">
  <Info>
  Sie können bestimmte Felder deaktivieren, um sie für den Kunden schreibgeschützt zu machen. Dies ist nützlich, wenn Sie bereits die Kundendaten haben (z. B. bei angemeldeten Benutzern).
  </Info>
  Um ein Feld zu deaktivieren, geben Sie seinen Wert an und setzen Sie das entsprechende <code>disable...</code> Flag auf <code>true</code>:
  <CodeGroup>

  ```text Example
  &email=alice@example.com&disableEmail=true
  ```

  </CodeGroup>
  <Tabs>
    <Tab title="Tabelle der Deaktivierungsflags">
      | Feld          | Deaktivierungsflag    | Erforderlicher Parameter |
      |---------------|---------------------|-------------------|
      | Vollständiger Name | `disableFullName`   | `fullName`        |
      | Vorname      | `disableFirstName`  | `firstName`       |
      | Nachname     | `disableLastName`   | `lastName`        |
      | E-Mail       | `disableEmail`      | `email`           |
      | Land         | `disableCountry`    | `country`         |
      | Adresszeile  | `disableAddressLine`| `addressLine`     |
      | Stadt        | `disableCity`       | `city`            |
      | Bundesland   | `disableState`      | `state`           |
      | PLZ          | `disableZipCode`    | `zipCode`         |
    </Tab>
  </Tabs>
  <Tip>
  Das Deaktivieren von Feldern hilft, versehentliche Änderungen zu verhindern und die Datenkonsistenz sicherzustellen.
  </Tip>
  <Note>
  Das Setzen von <code>showDiscounts=false</code> deaktiviert und verbirgt den Rabattbereich im Checkout-Formular. Verwenden Sie dies, wenn Sie verhindern möchten, dass Kunden während des Checkouts Gutscheine oder Aktionscodes eingeben.
  </Note>

</Step>
<Step title="Fügen Sie erweiterte Steuerungen hinzu (optional)">
  - <ParamField query="paymentCurrency" type="string">Gibt die Zahlungswährung an. Standardmäßig die Währung des Rechnungslandes.</ParamField>
  - <ParamField query="showCurrencySelector" type="boolean" default="true">Währungsauswahl anzeigen oder ausblenden.</ParamField>
  - <ParamField query="paymentAmount" type="integer">Betrag in Cent (nur für Pay What You Want-Preise).</ParamField>
  - <ParamField query="metadata_*" type="string">Benutzerdefinierte Metadatenfelder (z. B. <code>metadata_orderId=123</code>).</ParamField>
</Step>
<Step title="Teilen Sie den Link">
  Senden Sie den vollständigen Zahlungslink an Ihren Kunden. Wenn sie ihn besuchen, werden alle Abfrageparameter gesammelt und mit einer Sitzungs-ID gespeichert. Die URL wird dann vereinfacht, um nur den Sitzungsparameter zu enthalten (z. B. <code>?session=sess_1a2b3c4d</code>). Die gespeicherten Informationen bleiben bei Seitenaktualisierungen erhalten und sind während des gesamten Checkout-Prozesses zugänglich.
  <Check>
  Das Checkout-Erlebnis des Kunden ist jetzt optimiert und basierend auf Ihren Parametern personalisiert.
  </Check>
</Step>
</Steps>

    ```javascript
export async function POST(request: NextRequest) {

    const response = await fetch(`${process.env.NEXT_PUBLIC_DODO_TEST_API}/payments`, {

          state: formData.state,
          street: formData.addressLine,
          zipcode: parseInt(formData.zipCode),
        },

          product_id: id,
          quantity: 1,
        })),
        return_url: process.env.NEXT_PUBLIC_RETURN_URL,
      }),
    });

    if (!response.ok) {
      const errorData = await response.json().catch(() => null);
      return NextResponse.json(
        { error: "Payment link creation failed", details: errorData },
        { status: response.status }
      );
    }

    const data = await response.json();
    return NextResponse.json({ paymentLink: data.payment_link });
  } catch (err) {
    console.error("Payment error:", err);
    return NextResponse.json(
      {
        error: err instanceof Error ? err.message : "An unknown error occurred",
      },
      { status: 500 }
    );
  }
}
```

### Implementierung von Webhooks

Richten Sie einen API-Endpunkt ein, um Zahlungsbenachrichtigungen zu empfangen. Hier ist ein Beispiel mit Next.js:

```javascript
import { Webhook } from "standardwebhooks";
import { headers } from "next/headers";
import { WebhookPayload } from "@/types/api-types";

const webhook = new Webhook(process.env.DODO_WEBHOOK_KEY!); // Replace with your secret key generated from the Dodo Payments Dashboard

export async function POST(request: Request) {
  const headersList = headers();
  const rawBody = await request.text();

  const webhookHeaders = {
    "webhook-id": headersList.get("webhook-id") || "",
    "webhook-signature": headersList.get("webhook-signature") || "",
    "webhook-timestamp": headersList.get("webhook-timestamp") || "",
  };

  await webhook.verify(rawBody, webhookHeaders);
  const payload = JSON.parse(rawBody) as WebhookPayload;
  
  // Process the payload according to your business logic
}
```

Unsere Webhook-Implementierung folgt der [Standard-Webhooks](https://standardwebhooks.com/) Spezifikation. Für die Definitionen der Webhook-Typen siehe unseren [Webhook-Ereignisleitfaden](/developer-resources/webhooks/intents/webhook-events-guide).

Sie können dieses Projekt mit einer Demo-Implementierung auf [GitHub](https://github.com/dodopayments/dodo-checkout-demo) mit Next.js und TypeScript einsehen.

Sie können die Live-Implementierung [hier](https://atlas.dodopayments.com/) ansehen.
