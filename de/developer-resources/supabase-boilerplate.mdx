## Übersicht

Ein produktionsbereites Boilerplate für Abonnements mit Next.js 15, React 19, Supabase, Drizzle ORM und Dodo Payments. Es wird mit Google OAuth, Abonnement-Checkout, Webhook-Verarbeitung, Datenbankschema und einem grundlegenden Dashboard geliefert.

<Info>
Wenn du nur Route-Handler für eine bestehende App benötigst, siehe die dedizierten Adapter: <a href="/developer-resources/nextjs-adaptor">Next.js Adaptor</a> und <a href="/developer-resources/express-adaptor">Express Adaptor</a>.
</Info>

## Voraussetzungen

- Node.js 18+ (oder Bun 1.0+)
- Supabase-Projekt (URL, Anon-Schlüssel, Service-Rollen-Schlüssel, Datenbank-URL)
- Dodo Payments-Konto (API-Schlüssel, Webhook-Geheimnis)
- Google Cloud OAuth-Client (Client-ID und Geheimnis)

## Schnellstart

<Steps>
{/* LOCKED_PATTERN_587a3a4c926aec89f48dfc1e7c21054e */}

```bash
git clone https://github.com/dodopayments/dodo-supabase-subscription-starter.git
cd dodo-supabase-subscription-starter
# choose one
bun install
# or
npm install
# or
pnpm install
```

</Step>

{/* LOCKED_PATTERN_ea54c61e84e28cd275c0eed6d7634e0f */}
Erstelle ein Supabase-Projekt und kopiere:
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
- SUPABASE_SERVICE_ROLE_KEY
- DATABASE_URL (Connection string)
</Step>

{/* LOCKED_PATTERN_42bf2f0f0010fd47b77c88325d34313c */}
Setze die Redirect-URI auf: `https://[your-project-ref].supabase.co/auth/v1/callback` in Google Cloud und aktiviere dann den Google-Provider in Supabase Auth mit deiner Client-ID und deinem Secret.
</Step>

{/* LOCKED_PATTERN_f32b65d8a917a558befa771bede82115 */}
Generiere einen API-Schlüssel und Webhook-Secret über das Dodo-Dashboard. Setze die Umgebung während der Entwicklung auf `test_mode`.
</Step>

{/* LOCKED_PATTERN_a34a8e162789e5a79d108d9eb3c86df5 */}

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role

# Database
DATABASE_URL=postgresql://postgres:[password]@db.[project-ref].supabase.co:5432/postgres

# Dodo Payments
DODO_PAYMENTS_API_KEY=your-dodo-api-key
DODO_WEBHOOK_SECRET=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT=test_mode
```

<Warning>
Geheimnisse niemals commiten. Nutze Umgebungsvariablen in Deployments.
</Warning>
</Step>

{/* LOCKED_PATTERN_c35fe33eae6aa68b4bc39bd740632964 */}

```bash
bun run db:push
# or
npm run db:push
# or
pnpm run db:push
```

<Check>
Erstellte Tabellen: `users`, `subscriptions`, `payments`.
</Check>
</Step>

{/* LOCKED_PATTERN_560d1d81fad22bd0622f0c8dcb3c706d */}

```bash
# login (one-time)
bunx supabase login
# or
npx supabase login

# deploy the edge function
bun run deploy:webhook --project-ref [your-project-ref]
# or
npm run deploy:webhook -- --project-ref [your-project-ref]
# or
pnpm run deploy:webhook --project-ref [your-project-ref]
```

```bash cURL
curl -X POST \
  'https://[your-project-ref].supabase.co/functions/v1/dodo-webhook' \
  -H 'Content-Type: application/json' \
  -H 'Dodo-Signature: <signature>' \
  -d '{"type":"payment.succeeded","data":{}}'
```

</Step>

{/* LOCKED_PATTERN_dd5db8f3734010c91f70fdf794c4e2a2 */}
Setze die Endpoint-URL auf:

```text
https://[your-project-ref].supabase.co/functions/v1/dodo-webhook
```

Wähle Zahlungs- und Abonnement-Events aus.
</Step>

{/* LOCKED_PATTERN_43dfdbde6ac180daa9987aee2baf2da7 */}
Im Dodo-Dashboard → Products → Create Product. Optional Metadaten hinzufügen:
</Step>

```json
{
  "features": ["Feature 1", "Feature 2", "Feature 3"]
}
```

Die Preisanzeige liest dieses `features` Array und rendert es dynamisch.
</Step>

{/* LOCKED_PATTERN_e437399605995e5a4845b4d749b71d01 */}

```bash
bun run dev
# or
npm run dev
# or
pnpm run dev
```

Öffne http://localhost:3000.
</Step>
</Steps>

## Was ist enthalten

- Authentifizierung über Supabase (Google OAuth konfiguriert)
- Abonnement-Checkout über Dodo Payments
- Supabase Edge Function für Webhooks (`dodo-webhook`)
- Drizzle ORM Schema und Migrationen
- Dashboard mit Rechnungen, Abonnementstatus und Planfunktionen

<Tip>
Behalte `DODO_PAYMENTS_ENVIRONMENT` als `test_mode`, bis du End-to-End-Tests abgeschlossen hast.
</Tip>

## Überprüfung und Fehlersuche

<AccordionGroup>
{/* LOCKED_PATTERN_6b286e9e2bbacd5c7684f8442203f10e */}
- Stelle sicher, dass `DODO_WEBHOOK_SECRET` mit dem Wert aus dem Dodo-Dashboard übereinstimmt
- Bestätige, dass du die neueste `dodo-webhook` Funktion bereitgestellt hast
- Überprüfe, ob der Header-Name in deiner Funktion korrekt ist (Dodo-Signature)
</Accordion>
{/* LOCKED_PATTERN_c270a2f8797d0168d608bb07bd40f056 */}
- Überprüfe `DATABASE_URL` Syntax und Supabase-Netzwerkegress
- Warte ca. 2–3 Minuten nach der Projekterstellung vor dem ersten Push
</Accordion>
{/* LOCKED_PATTERN_276d2f8137bb346c423586f69d6b7cae */}
- Die Redirect-URI muss `https://[ref].supabase.co/auth/v1/callback` sein
- Stelle sicher, dass dies sowohl in Google Cloud als auch im Supabase Auth Provider der Fall ist
</Accordion>
</AccordionGroup>
