Dieser Leitfaden behandelt ausschließlich **Einmalzahlungen** mit **Karten, Apple Pay und Google Pay**.
</Note>

## API-Endpunkte

| Environment | Base URL | Confirm Payment Endpoint |
| --- | --- | --- |
| **Test Mode** | `https://test.dodopayments.com` | `https://api.custom.hs.dodopayments.com/payments/<payment_id>/confirm` |
| **Live Mode** | `https://live.dodopayments.com` | `https://api.hs.dodopayments.com/payments/<payment_id>/confirm` |

## Schritt-für-Schritt-Kartenzahlungsablauf

### Schritt 1: Beschaffung des geheimen Schlüssels

Beginnen Sie mit der Erstellung einer Zahlung über folgende API:

- **Payments**: [Create Payment API](/api-reference/payments/post-payments)

Diese API liefert einen **`secret_key`**, der für die weiteren Schritte benötigt wird.

### Schritt 2: Bestätigen Sie die Zahlung

Bestätigen Sie die Zahlung, indem Sie den Endpoint `payments confirm` aufrufen:

- **Test Mode**: `POST https://api.custom.hs.dodopayments.com/payments/<payment_id>/confirm`
- **Live Mode**: `POST https://api.hs.dodopayments.com/payments/<payment_id>/confirm`

Fügen Sie folgende Details in den Request-Body ein:

- `client_secret`: Geheimer Schlüssel aus Schritt 1
- `payment_method`: Auf `card` setzen
- `payment_method_data`: Karten- und Rechnungsdaten
- `browser_info`: Browserspezifische Daten (User-Agent, Bildschirmdetails usw.)
- `payment_type`: Explizit auf `normal` setzen (für Einmalzahlungen)

Erforderliche Header:

- Holen Sie Ihren veröffentlichbaren Schlüssel vom DodoPayments-Support
- `api-key`: Ihr veröffentlichbarer Schlüssel (vom Dodo Payments Team zu beziehen)
- `content-type`: `application/json`

**Beispiel für den Request-Body:**

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "card",
  "payment_method_data": {
    "card": {
      "card_number": "<card_number>",
      "card_exp_month": "<exp_month>",
      "card_exp_year": "<exp_year>",
      "card_holder_name": "<cardholder_name>",
      "card_cvc": "<cvc>",
      "card_network": "<network>"
    },
    "billing": {
      "address": {
        "first_name": "<first_name>",
        "last_name": "<last_name>"
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
    "time_zone": "<time_zone>",
    "java_enabled": true,

```

### Verarbeitung der Antwort

- Wenn die Antwort **`status`** **`requires_customer_action`** ist, leiten Sie den Kunden zur in **`next_action`** angegebenen URL weiter, um die Authentifizierung fortzusetzen.

<Warning>
Stellen Sie sicher, dass Sie bei der Verarbeitung roher Kartendaten die PCI-Compliance-Richtlinien einhalten.
</Warning>

## Apple Pay-Integration

Um Apple Pay zu verwenden, setzen Sie `payment_method` auf `wallet` und fügen Sie die Apple-Pay-Daten in `payment_method_data` ein.

### Aufbau des Request-Bodys

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "wallet",
  "payment_method_data": {
    "wallet": {
      "apple_pay": {
        "payment_data": "<decrypted_payment_data>",
        "payment_method": {
          "display_name": "Visa 1234",
          "network": "Visa",
          "type": "debit"
        },
        "transaction_identifier": "UNIQUE_TRANSACTION_ID_FROM_APPLE"
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
    "time_zone": "<time_zone>",
    "java_enabled": true,
    "java_script_enabled": true,
    "ip_address": "<ip_address_of_client>",
    "device_model": "<device_model>",
    "os_type": "<os_type>",
    "os_version": "<version>"
  },
  "payment_type": "normal"
}
```

### Apple Pay Feldreferenz

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `payment_data` | string | Ja | Dieses Enum stellt die Apple-Pay-Zahlungsdaten dar, die entweder verschlüsselt oder entschlüsselt vorliegen können. |
| `payment_method.display_name` | string | Ja | Anzeigename, der auf der Apple-Pay-Schaltfläche angezeigt wird (z. B. „Visa 1234“) |
| `payment_method.network` | string | Ja | Kartennetzwerk (z. B. „Visa“, „Mastercard“, „Amex“) |
| `payment_method.type` | string | Ja | Zahlungskartentyp (z. B. „debit“, „credit“) |
| `payment_method.card_exp_month` | string | Nein | Ablaufmonat der Karte (z. B. „12“) |
| `payment_method.card_exp_year` | string | Nein | Ablaufjahr der Karte (z. B. „25“) |
| `transaction_identifier` | string | Ja | Eindeutiger Transaktionskennzeichner von Apple Pay |

## Google Pay-Integration

Um Google Pay zu verwenden, setzen Sie `payment_method` auf `wallet` und fügen Sie die Google-Pay-Daten in `payment_method_data` ein.

### Aufbau des Request-Bodys

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "wallet",
  "payment_method_data": {
    "wallet": {
      "google_pay": {
        "type": "CARD",
        "description": "Visa ending in 1234",
        "info": {
          "card_network": "VISA",
          "card_details": "1234"
        },
        "tokenization_data": {
          "type": "PAYMENT_GATEWAY",
          "token": "<encrypted_payment_token>"
        }
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
    "time_zone": "<time_zone>",
    "java_enabled": true,
    "java_script_enabled": true,
    "ip_address": "<ip_address_of_client>",
    "device_model": "<device_model>",
    "os_type": "<os_type>",
    "os_version": "<version>"
  },
  "payment_type": "normal"
}
```

### Google Pay Feldreferenz

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `type` | string | Ja | Zahlungsart (typischerweise „CARD“) |
| `description` | string | Ja | Benutzerfreundliche Beschreibung der Zahlungsmethode |
| `info.card_network` | string | Ja | Kartennetzwerk (z. B. „VISA“, „MASTERCARD“, „AMEX“) |
| `info.card_details` | string | Ja | Letzte 4 Ziffern der Karte |
| `info.assurance_details` | object | Nein | Details zur Authentifizierungsstärke |
| `tokenization_data.type` | string | Ja | Tokenisierungstyp (z. B. „PAYMENT_GATEWAY“) |
| `tokenization_data.token` | string | Ja | Verschlüsselter Zahlungstoken von Google Pay |
