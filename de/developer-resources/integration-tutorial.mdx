## Voraussetzungen

Bevor Sie beginnen, stellen Sie sicher, dass Sie:

- [Ein Dodo Payments-Konto erstellt haben](/quickstart)
- [Ihre API-Schlüssel erhalten haben](/api-reference/introduction#authentication)
- Grundkenntnisse in Next.js und React haben
- Node.js installiert ist (v14 oder höher)

## Boilerplate-Setup

Klonen Sie unser Next.js-Boilerplate, um schnell zu starten:

```bash
git clone https://github.com/dodopayments/dodo-nextjs-minimal-boilerplate
cd dodo-nextjs-minimal-boilerplate
npm install
```

<Note>
Das Boilerplate enthält alle notwendigen Abhängigkeiten und die grundlegende Einrichtung für die Integration von Dodo Payments.
</Note>

## Webhook-Integration

Richten Sie Webhooks ein, um asynchrone Ereignisse zu verarbeiten:

```typescript
import { verifyWebhookSignature } from '@dodopayments/node';

export default async function handler(req, res) {
  const signature = req.headers['dodo-signature'];
  const payload = req.body;
  
  try {
    verifyWebhookSignature({
      payload,
      signature,
      secret: process.env.WEBHOOK_SECRET
    });
    
    // Handle webhook event
    switch (payload.type) {
      case 'payment.succeeded':
        // Handle successful payment
        break;
      case 'subscription.created':
        // Handle new subscription
        break;
      // ... handle other events
    }
    
    res.status(200).json({ received: true });
  } catch (err) {
    res.status(400).json({ error: 'Invalid signature' });
  }
}
```
