## Overview

The Model Context Protocol (MCP) is an open standard that enables AI applications to securely connect to external data sources and APIs. The Dodo Payments MCP Server provides AI assistants like Claude, Cursor, and other MCP-compatible clients with structured access to your payment infrastructure.

Der Dodo Payments MCP Server verwendet die **Code Mode**-Architektur. Anstatt Hunderte einzelner Tools für jeden API-Endpunkt bereitzustellen, ermöglicht Code Mode KI-Agenten das Schreiben und Ausführen von TypeScript-Code gegen das Dodo Payments SDK in einer isolierten Sandbox-Umgebung.

### Key capabilities

- **Payment Operations**: Create, retrieve, and manage payments and refunds
- **Subscription Management**: Handle recurring billing, upgrades, and cancellations
- **Customer Administration**: Manage customer data and portal access
- **Product Catalog**: Create and update products, pricing, and discounts
- **License Management**: Activate, validate, and manage software licenses
- **Usage-Based Billing**: Track and bill for metered usage

## How Code Mode Works

Der Dodo Payments MCP Server stellt Ihrem KI-Agenten genau zwei Tools zur Verfügung:

1. **Docs Search Tool**: Fragt die Dokumentation zur Dodo Payments API und SDK ab, um verfügbare Operationen und Parameter zu verstehen.
2. **Code Execution Tool**: Schreibt TypeScript-Code gegen das SDK, der in einer sicheren Sandbox-Umgebung ausgeführt wird.

Diese Architektur erlaubt es Agenten, komplexe, mehrstufige Operationen in einem einzigen Aufruf durchzuführen. Ein Agent kann beispielsweise alle aktiven Abonnements auflisten, sie anhand bestimmter Kriterien filtern und jedem einen Rabatt gewähren — alles innerhalb eines Skripts.

<Tip>
Mit Code Mode können Agenten mehrere API-Aufrufe verketten, bedingte Logik verarbeiten und Berechnungen durchführen, ohne mehrere Round-Trips zum LLM zu erfordern.
</Tip>

### Schnellstart

Verbinden Sie sich mit dem Dodo Payments MCP Server in Ihrem KI-Client:

<Tabs>
<Tab title="Cursor">
Fügen Sie zu `~/.cursor/mcp.json` hinzu:

```json
{
  "mcpServers": {
    "dodopayments": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://mcp.dodopayments.com/sse"]
    }
  }
}
```

</Tab>


## Dodo Knowledge MCP

Zusätzlich zum Dodo Payments MCP Server (für die Ausführung von API-Operationen) bieten wir **Dodo Knowledge MCP** an — einen semantischen Suchserver, der KI-Assistenten sofortigen Zugriff auf die Dodo Payments-Dokumentation und Wissensdatenbank ermöglicht.

<Info>
**Gebaut mit [ContextMCP.ai](https://contextmcp.ai)**: Dodo Knowledge MCP wird von ContextMCP betrieben und ermöglicht schnelle semantische Suche in unserer Dokumentation mithilfe von Vektor-Embeddings.
</Info>

### Was ist Dodo Knowledge MCP?

Dodo Knowledge MCP ist ein Remote-MCP-Server, der bereitstellt:

- **Semantische Dokumentensuche**: Finden Sie relevante Dokumentation mit natürlichsprachlichen Anfragen.
- **Kontextuelle Antworten**: KI-Assistenten erhalten genaue, aktuelle Informationen über Dodo Payments.
- **Null Setup**: Keine API-Schlüssel oder lokale Installation erforderlich — einfach verbinden und loslegen.

### Beide MCP-Server zusammen verwenden

| Server | Zweck | Anwendungsfall |
|--------|---------|----------|

```json
    }
  }
}
```

<Tip>
Mit beiden konfigurierten Servern kann Ihr KI-Assistent zuerst die Dokumentation durchsuchen, um zu verstehen, wie ein Feature implementiert wird, und dann die tatsächlichen API-Aufrufe ausführen — alles in einem einzigen Gespräch.
</Tip>

### Fehlerbehebung Knowledge MCP

Wenn Verbindungsprobleme auftreten:

1. **MCP-Authentifizierungs-Cache leeren**: `rm -rf ~/.mcp-auth`
2. **Starten Sie Ihre Client-Anwendung neu**
3. **Überprüfen Sie Client-Protokolle** auf Fehlermeldungen
4. **Überprüfen Sie die Node.js-Version**: Erfordert Node.js 18+

### Docker

Führen Sie den MCP-Server in einer containerisierten Umgebung für konsistente Bereitstellungen aus.

<CodeGroup>

```bash Docker
# Pull the latest image
docker pull ghcr.io/dodopayments/mcp:latest

# Run the container
docker run -e DODO_PAYMENTS_API_KEY="dodo_test_..." \
  -e DODO_PAYMENTS_WEBHOOK_KEY="your_webhook_key" \
  -e DODO_PAYMENTS_ENVIRONMENT="live_mode" \
  ghcr.io/dodopayments/mcp:latest
```

```yaml Docker Compose
services:
  dodopayments-mcp:
    image: ghcr.io/dodopayments/mcp:latest
    environment:
      - DODO_PAYMENTS_API_KEY=dodo_test_...
      - DODO_PAYMENTS_WEBHOOK_KEY=your_webhook_key
      - DODO_PAYMENTS_ENVIRONMENT=live_mode
    restart: unless-stopped
```

</CodeGroup>

<Note>
Docker-Images sind auf der [GitHub Container Registry](https://github.com/orgs/dodopayments/packages/container/package/mcp) verfügbar.
</Note>

## Remote-Ausführung

Bereitstellen des MCP-Servers als Remote-HTTP-Server für webbasierte Clients oder agentische Workflows.

```bash
# Start as remote server
npx -y dodopayments-mcp --transport=http

# Specify a custom port
npx -y dodopayments-mcp --transport=http --port=3000
```

### Konfiguration des Remote-Servers

Nach der Bereitstellung können Clients über die Server-URL verbinden:

```json
{
  "mcpServers": {
    "dodopayments_api": {
      "url": "http://localhost:3000",
      "headers": {
        "Authorization": "Bearer your_api_key"
      }
    }
  }
}
```

### Autorisierungs-Header

Der Remote-Server akzeptiert Authentifizierung über folgende Header:

| Header | Beschreibung |
|--------|-------------|
| `Authorization` | Bearer-Token-Authentifizierung |
| `x-dodo-payments-api-key` | Direkter API-Key-Header |

## Sicherheitsbest Practices

Code Mode bietet inhärente Sicherheit, indem Code in einer Sandbox-Umgebung ausgeführt und API-Schlüssel serverseitig injiziert werden. Befolgen Sie diese zusätzlichen Best Practices, um Ihre Zugangsdaten zu schützen.

<AccordionGroup>
<Accordion title="API Key Management">
**Bewahren Sie niemals Zugangsdaten in der Versionskontrolle auf**

Speichern Sie API-Schlüssel in Umgebungsvariablen oder sicheren Geheimnisverwaltungs-Systemen.


## Warum Code Mode

Traditionelle MCP-Implementierungen leiden oft unter „Tool-Proliferation“, bei der jeder API-Endpunkt als separates Tool exponiert wird. Code Mode ist aus mehreren Gründen ein überlegener Ansatz:

### LLMs sind besser darin, Code zu schreiben als Tools aufzurufen
LLMs wurden mit Millionen Zeilen realen Codes trainiert, wodurch sie von Natur aus im Schreiben von Skripten geübt sind. Im Gegensatz dazu basieren Tool-Aufrufe häufig auf synthetischen Beispielen.

<Info>
„Ein LLM Aufgaben mit Tool-Aufrufen erledigen zu lassen ist, als würde man Shakespeare einen einmonatigen Mandarin-Kurs geben und ihn dann bitten, ein Stück darin zu schreiben.“ — [Cloudflare](https://blog.cloudflare.com/code-mode/)
</Info>

### Eliminierung von Kontextfenster-Aufblähung
In einem traditionellen Ansatz verbraucht jede Tool-Definition Tokens, bevor das Gespräch überhaupt begonnen hat. Wenn mehr als 50 Tools exponiert werden, können problemlos 55K–100K+ Tokens aufgebraucht werden. Anthropic stellte fest, dass Tool-Definitionen vor der Optimierung bis zu 134K Tokens einnehmen konnten.

Mit Code Mode werden nur 2 Tool-Definitionen geladen (~1K Tokens). Der Agent sucht bei Bedarf nach der benötigten Dokumentation. Anthropics Tool Search Tool bewahrte 95 % des Kontextfensters und reduzierte den Overhead von 77K auf 8,7K Tokens.

### Verringerung der Latenz durch programmatische Orchestrierung
Traditionelle Tool-Aufrufe erfordern für jede einzelne Operation einen vollständigen Modell-Inferenz-Round-Trip. Wenn eine Aufgabe 20 API-Aufrufe benötigt, sind das 20 Round-Trips. In Code Mode schreibt der Agent ein Skript, das alle Aufrufe ausführt und nur das endgültige Ergebnis zurückgibt.

Anthropic beobachtete eine Reduzierung der Tokens um 37 % und verbesserte Genauigkeit (Wissensabruf verbesserte sich von 25,6 % auf 28,5 %), indem dieser programmatische Ansatz genutzt wurde.

### Besser von Haus aus sicher
Code Mode bietet inhärente Sicherheitsvorteile:
- **Keine API-Schlüssel in Parametern**: API-Schlüssel werden serverseitig injiziert und sind in den an das LLM gesendeten Tool-Parametern nicht sichtbar.
- **Isolierte Sandbox**: Der Code läuft in einer sicheren Umgebung ohne Zugriff auf das Netzwerk oder das Host-Dateisystem.
- **Kontrolliertes SDK**: Nur autorisierte SDK-Methoden stehen dem Agenten zur Verfügung.

### Skalierbar auf jede API-Größe
Wenn eine API wächst, verschlechtert sich die MCP-Leistung traditionell, weil mehr Tools in den Kontext geladen werden müssen. Code Mode bleibt konstant mit 2 Tools, unabhängig von der Größe der API-Oberfläche. Cloudflare konsolidierte erfolgreich über 2.500 API-Endpunkte in nur 2 Tools und ungefähr 1.000 Tokens Kontext.

<Info>
Weitere Details zu den Vorteilen dieser Architektur finden Sie in den technischen Blogs von [Anthropic](https://www.anthropic.com/engineering/advanced-tool-use) und [Cloudflare](https://blog.cloudflare.com/code-mode/) sowie in der [Programmatic Tool Calling](https://platform.claude.com/docs/en/agents-and-tools/tool-use/programmatic-tool-calling)-Dokumentation von Claude.
</Info>
