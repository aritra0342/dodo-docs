Dodo Payments bietet Echtzeit-Webhooks-Benachrichtigungen, um Sie über kritische Ereignisse im Zusammenhang mit Transaktionen, Abonnements, Rückerstattungen, Streitigkeiten und der Verwaltung von Lizenzschlüsseln zu informieren. Im Folgenden finden Sie einen detaillierten Leitfaden zu diesen Ereignissen, die zur Klarheit kategorisiert sind, sowie Implementierungstipps und -überlegungen.

## **Zahlungsevents**

| **Ereignisname**        | **Ereignistyp**          | **Beschreibung**                                                                |
|--------------------------|--------------------------|---------------------------------------------------------------------------------|
| **Zahlung Erfolgreich**  | `payment.succeeded`      | Wird ausgelöst, wenn eine Zahlung erfolgreich verarbeitet wird.                  |
| **Zahlung Fehlgeschlagen** | `payment.failed`         | Tritt auf, wenn ein Zahlungsversuch aufgrund von Fehlern, abgelehnten Karten oder anderen Problemen fehlschlägt. |
| **Zahlung wird verarbeitet** | `payment.processing`     | Zeigt an, dass eine Zahlung derzeit verarbeitet wird.                            |
| **Zahlung Abgebrochen**  | `payment.cancelled`      | Wird ausgelöst, wenn eine Zahlung vor Abschluss abgebrochen wird.               |

## **Rückerstattungsevents**

| **Ereignisname**        | **Ereignistyp**          | **Beschreibung**                                                                |
|--------------------------|--------------------------|---------------------------------------------------------------------------------|
| **Rückerstattung Erfolgreich** | `refund.succeeded`       | Wird ausgelöst, wenn eine Rückerstattung erfolgreich verarbeitet wird.           |
| **Rückerstattung Fehlgeschlagen** | `refund.failed`          | Tritt auf, wenn ein Rückerstattungsversuch aufgrund von Verarbeitungsfehlern oder anderen Problemen fehlschlägt. |

## **Streitigkeitsevents**

| **Ereignisname**        | **Ereignistyp**          | **Beschreibung**                                                                |
|--------------------------|--------------------------|---------------------------------------------------------------------------------|
| **Streitigkeit Eröffnet** | `dispute.opened`         | Wird ausgelöst, wenn ein Kunde eine Streitigkeit einleitet.                     |
| **Streitigkeit Abgelaufen** | `dispute.expired`        | Zeigt an, dass eine Streitigkeit ohne Lösung abgelaufen ist.                    |
| **Streitigkeit Akzeptiert** | `dispute.accepted`       | Wird ausgelöst, wenn ein Händler die Streitigkeit akzeptiert.                   |
| **Streitigkeit Abgebrochen** | `dispute.cancelled`      | Tritt auf, wenn eine Streitigkeit vom Kunden oder System abgebrochen wird.      |
| **Streitigkeit Angefochten** | `dispute.challenged`     | Wird ausgelöst, wenn ein Händler eine Streitigkeit anfechtet.                   |
| **Streitigkeit Gewonnen** | `dispute.won`            | Zeigt an, dass der Händler eine Streitigkeit erfolgreich gewonnen hat.           |
| **Streitigkeit Verloren** | `dispute.lost`           | Tritt auf, wenn der Händler eine Streitigkeit verliert.                         |

## **Abonnementsevents**

| **Ereignisname**        | **Ereignistyp**          | **Beschreibung**                                                                |
|--------------------------|--------------------------|---------------------------------------------------------------------------------|
| **Abonnement Aktiv**     | `subscription.active`    | Zeigt an, dass ein Abonnement jetzt aktiv ist und wiederkehrende Gebühren geplant sind. |
| **Abonnement Aktualisiert** | `subscription.updated`   | Wird ausgelöst, wenn ein beliebiges Abonnementfeld aktualisiert wird (Echtzeitsynchronisierung ohne Abfragen). |
| **Abonnement Ausgesetzt** | `subscription.on_hold`   | Wird ausgelöst, wenn ein Abonnement vorübergehend aufgrund einer fehlgeschlagenen Erneuerung ausgesetzt wird. |
| **Abonnement Erneuert**  | `subscription.renewed`   | Tritt auf, wenn ein Abonnement erfolgreich erneuert wird.                       |
| **Abonnement Plan Geändert** | `subscription.plan_changed` | Wird ausgelöst, wenn ein Abonnement aufgewertet, herabgestuft oder mit anderen Add-ons modifiziert wird. |
| **Abonnement Abgebrochen** | `subscription.cancelled` | Wird ausgelöst, wenn ein Abonnement vom Händler oder Kunden abgebrochen wird.  |
| **Abonnement Fehlgeschlagen** | `subscription.failed`    |  Zeigt ein fehlgeschlagenes Abonnement an. Das bedeutet, dass wir nicht in der Lage waren, ein Mandat zu erstellen. |
| **Abonnement Abgelaufen** | `subscription.expired`   | Wird ausgelöst, wenn ein Abonnement das Ende seiner Laufzeit erreicht und abläuft. |

## **Lizenzschlüsselereignisse**

| **Ereignisname**        | **Ereignistyp**          | **Beschreibung**                                                                |
|--------------------------|--------------------------|---------------------------------------------------------------------------------|
| **Lizenzschlüssel Erstellt** | `license_key.created`    | Wird ausgelöst, wenn ein neuer Lizenzschlüssel für ein Produkt erstellt wird.    |

Diese Webhook-Ereignisse sind darauf ausgelegt, Händlern kritische Updates zu bieten, um ihre Abläufe effektiv zu verwalten. Implementieren Sie robuste Handler für jedes relevante Ereignis in Ihrer Webhook-Integration, um eine reibungslose und zuverlässige Funktionalität sicherzustellen.
