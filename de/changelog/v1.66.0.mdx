## Neue Funktionen 🚀

1. **`subscription.updated` Webhook**  
   Ein neues Webhook-Ereignis, das bei jeder Aktualisierung eines Abonnementobjekts ausgelöst wird und Ihnen Echtzeitbenachrichtigungen über Änderungen an Ihren Abonnements liefert.
   
   Verwenden Sie dieses Webhook, um:
   - Sofortige Benachrichtigungen zu erhalten, wenn sich Abonnementdetails ändern
   - Ihren Anwendungszustand mit Dodo Payments synchron zu halten
   - Auf Abonnementaktualisierungen zu reagieren, ohne die API abzufragen
   
   <Tip>
   Abonnieren Sie `subscription.updated`, um die Notwendigkeit des Pollings zu eliminieren und Ihr System durch Echtzeit-Abonnementänderungen synchron zu halten.
   </Tip>
   
   Erfahren Sie mehr: [Subscription Webhooks](/developer-resources/webhooks/intents/subscription)

2. **Minimal Address Support im Checkout**  

   <Frame>
   <img src="/images/changelog/minimal-address.png" alt="Minimal Address Support in Checkout" />
   </Frame>

   Reduzieren Sie Checkout-Reibung mit dem neuen Modus zur minimalen Adresserfassung. Anstatt vollständige Adressdaten zu erfassen, sammelt der Modus nur:
   
   - **Land**: Immer erforderlich zur Steuerbestimmung
   - **PLZ/Postalcode**: Nur in Regionen, in denen sie für die Berechnung von Umsatzsteuer, Mehrwertsteuer oder GST notwendig ist
   
   <Check>
   Ergebnis: Deutlich schnellere Checkout-Abwicklung mit weniger Formularfeldern für Kund:innen.
   </Check>
   
   **Minimaladresse aktivieren:**
   
   <Tabs>
   {/* LOCKED_PATTERN_0090a508f27e87d00759ddcd3b9fe94c */}

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [{ product_id: 'prod_123', quantity: 1 }],
     minimal_address: true
   });
   ```

   </Tab>
   
   {/* LOCKED_PATTERN_ed987f4815610e04ce90e6f11a8998dc */}
   Fügen Sie `minimalAddress=true` zu Ihrer Zahlungslink-URL hinzu:
   

   ```
   https://checkout.dodopayments.com/buy/prod_123?minimalAddress=true
   ```

   </Tab>
   </Tabs>
   
   <Info>
   Standardmäßig sammelt der Checkout weiterhin vollständige Adressen. Wir werden alle Checkout-Links schrittweise auf den Modus mit minimaler Adresse umstellen. Aktivieren Sie jetzt das Feature-Flag, um von der reduzierten Checkout-Reibung zu profitieren.
   </Info>

3. **Nur-Lese-API-Schlüssel**  
   Erstellen Sie API-Schlüssel mit Nur-Lese-Berechtigungen für sicherere Integrationen. Beim Erstellen eines neuen API-Schlüssels deaktivieren Sie **Write-Zugriff aktivieren**, um einen Nur-Lese-Schlüssel zu erstellen.
   
   Nur-Lese-Schlüssel können:
   - Zahlungs-, Abonnement- und Kundendaten abrufen
   - Produkte auflisten und Details abrufen
   - Auf Berichte und Analysen zugreifen
   
   Nur-Lese-Schlüssel **können nicht**:
   - Zahlungen erstellen oder ändern
   - Abonnements aktualisieren
   - Produktkonfigurationen ändern
   
   <Tip>
   Deaktivieren Sie „Write-Zugriff aktivieren“ für Dashboard-Integrationen, Analyse-Tools und jedes System, das nur Daten anzeigen muss, ohne Änderungen vorzunehmen.
   </Tip>

4. **UPI AutoPay & RuPay Card Subscriptions – Vollständige Einführung**  
   UPI AutoPay und RuPay-Kartenabonnements sind jetzt für alle Unternehmen verfügbar. Damit sind wiederkehrende Zahlungen für indische Kund:innen möglich mit:
   
   - **UPI AutoPay**: Automatisierte wiederkehrende Belastungen über UPI-Mandat
   - **RuPay Cards**: Abonnementunterstützung für das indische nationale Kartennetzwerk
   
   <Check>
   Ergebnis: Erweiterte Zahlungsoptionen für Abonnements im indischen Markt mit RBI-konformer Mandatsabwicklung.
   </Check>
   
   Erfahren Sie mehr: [Subscriptions with RBI-Compliant Mandates](/features/subscription#subscriptions-with-rbi-compliant-mandates)

5. **API zur Vorschau von Planänderungen**  
   Ein neuer API-Endpunkt, um Änderungen des Abonnementplans vor der endgültigen Bestätigung anzuzeigen. Erhalten Sie den genauen Betrag und sehen Sie, wie das Abonnement nach der Änderung aussehen wird - ohne tatsächliche Änderungen vorzunehmen.
   

   ```javascript
   const preview = await client.subscriptions.previewChangePlan('sub_123', {
     product_id: 'prod_pro_plan',
     proration_billing_mode: 'prorated_immediately',
     quantity: 1
   });
   
   console.log('Charge amount:', preview.immediate_charge.summary);
   console.log('New plan details:', preview.new_plan);
   ```

   
   Verwenden Sie dies, um:
   - Kunden genau zu zeigen, was sie vor der Bestätigung von Upgrades/Downgrades zahlen werden
   - Preisrechner und Planvergleichstools zu erstellen
   - Supportanfragen zu reduzieren, indem die Preistransparenz verbessert wird
   
   Erfahren Sie mehr: [API zur Vorschau von Planänderungen](/api-reference/subscriptions/preview-change-plan)
