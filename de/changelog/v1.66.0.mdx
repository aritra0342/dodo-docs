## Neue Funktionen 🚀

1. **`subscription.updated` Webhook**  
   Ein neues Webhook-Ereignis, das bei jeder Aktualisierung eines Abonnementobjekts ausgelöst wird und Ihnen Echtzeitbenachrichtigungen über Änderungen an Ihren Abonnements gibt.
   
   Verwenden Sie dieses Webhook, um:
   - Sofortige Benachrichtigungen zu erhalten, wenn sich Abonnementdetails ändern
   - Den Anwendungsstatus mit Dodo Payments synchronisiert zu halten
   - Auf Abonnementaktualisierungen zu reagieren, ohne die API abzufragen
   
   <Tip>
   Abonnieren Sie `subscription.updated`, um die Notwendigkeit der Abfrage zu beseitigen und Ihr System mit Echtzeit-Abonnementänderungen synchron zu halten.
   </Tip>
   
   Erfahren Sie mehr: [Subscription Webhooks](/developer-resources/webhooks/intents/subscription)

2. **Minimal Address Support im Checkout**  

   <Frame>
   <img src="/images/changelog/minimal-address.png" alt="Minimal Address Support im Checkout" />
   </Frame>

   Reduzieren Sie die Checkout-Reibung mit dem neuen minimalen Adressensammelmodus. Anstatt vollständige Adressdetails zu sammeln, erfasst der minimale Adressmodus nur:
   
   - **Land**: Immer erforderlich zur Steuerbestimmung
   - **PLZ/Postleitzahl**: Nur in Regionen erfasst, in denen es für die Berechnung von Verkaufssteuern, Mehrwertsteuer oder GST erforderlich ist
   
   <Check>
   Ergebnis: Deutlich schnellere Checkout-Abwicklung mit weniger Formularfeldern für die Kunden.
   </Check>
   
   **Aktivieren Sie die minimale Adresse:**
   
   <Tabs>
   <Tab title="Checkout-Sitzungen">

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [{ product_id: 'prod_123', quantity: 1 }],
     minimal_address: true
   });
   ```

   </Tab>
   
   <Tab title="Statische Zahlungslinks">
   Fügen Sie `minimalAddress=true` zu Ihrer Zahlungslink-URL hinzu:
   

   ```
   https://checkout.dodopayments.com/buy/prod_123?minimalAddress=true
   ```

   </Tab>
   </Tabs>
   
   <Info>
   Standardmäßig sammelt der Checkout weiterhin vollständige Adressen. Wir werden alle Checkout-Links schrittweise auf den minimalen Adressmodus umstellen. Aktivieren Sie jetzt das Feature-Flag, um von der reduzierten Checkout-Reibung zu profitieren.
   </Info>

3. **Schreibgeschützte API-Schlüssel**  
   Erstellen Sie API-Schlüssel mit schreibgeschützten Berechtigungen für sicherere Integrationen. Deaktivieren Sie beim Erstellen eines neuen API-Schlüssels **Schreibzugriff aktivieren**, um einen schreibgeschützten Schlüssel zu erstellen.
   
   Schreibgeschützte Schlüssel können:
   - Zahlungs-, Abonnement- und Kundendaten abrufen
   - Produkte auflisten und Details abrufen
   - Berichte und Analysen zugreifen
   
   Schreibgeschützte Schlüssel **können nicht**:
   - Zahlungen erstellen oder ändern
   - Abonnements aktualisieren
   - Produktkonfigurationen ändern
   
   <Tip>
   Deaktivieren Sie "Schreibzugriff aktivieren" für Dashboard-Integrationen, Analysetools und jedes System, das nur Daten anzeigen muss, ohne Änderungen vorzunehmen.
   </Tip>

4. **UPI AutoPay & RuPay-Karten-Abonnements - Vollständige Einführung**  
   UPI AutoPay und RuPay-Karten-Abonnements sind jetzt für alle Unternehmen verfügbar. Dies ermöglicht wiederkehrende Zahlungen für indische Kunden mit:
   
   - **UPI AutoPay**: Automatisierte wiederkehrende Belastungen über UPI-Mandat
   - **RuPay-Karten**: Abonnementsupport für Indiens nationales Kartennetzwerk
   
   <Check>
   Ergebnis: Erweiterte Abonnement-Zahlungsoptionen für den indischen Markt mit RBI-konformer Mandatsbearbeitung.
   </Check>
   
   Erfahren Sie mehr: [Abonnements mit RBI-konformen Mandaten](/features/subscription#subscriptions-with-rbi-compliant-mandates)

5. **API zur Vorschau von Planänderungen**  
   Ein neuer API-Endpunkt, um Änderungen des Abonnementplans vor der endgültigen Bestätigung anzuzeigen. Erhalten Sie den genauen Betrag und sehen Sie, wie das Abonnement nach der Änderung aussehen wird - ohne tatsächliche Änderungen vorzunehmen.
   

   ```javascript
   const preview = await client.subscriptions.previewChangePlan('sub_123', {
     product_id: 'prod_pro_plan',
     proration_billing_mode: 'prorated_immediately',
     quantity: 1
   });
   
   console.log('Charge amount:', preview.immediate_charge.summary);
   console.log('New plan details:', preview.new_plan);
   ```

   
   Verwenden Sie dies, um:
   - Kunden genau zu zeigen, was sie vor der Bestätigung von Upgrades/Downgrades zahlen werden
   - Preisrechner und Planvergleichstools zu erstellen
   - Supportanfragen zu reduzieren, indem die Preistransparenz verbessert wird
   
   Erfahren Sie mehr: [API zur Vorschau von Planänderungen](/api-reference/subscriptions/preview-change-plan)
