## Neue Funktionen 🚀

1. **Gemischter Warenkorb-Checkout: Einmalige und Abonnement-Produkte im selben Warenkorb**  
   Der Checkout unterstützt jetzt die Kombination von einmaligen Zahlungsprodukten und Abonnement-Produkten in einem einzigen Warenkorb und erschließt so leistungsstarke neue Abrechnungsfälle.
   
   <Frame>
   <img src="/images/changelog/mixed-checkout.png" alt="Mixed checkout with one-time and subscription products" />
   </Frame>
   
   ### Was dies ermöglicht
   
   - **Einrichtungsgebühren + Abonnements**: Erheben Sie neben einem wiederkehrenden Abonnement eine einmalige Einrichtungsgebühr in einem Checkout
   - **Bundles mit wiederkehrenden Komponenten**: Verkaufen Sie ein Produkt (einmalig) zusammen mit einem laufenden Servicezugang (Abonnement)
   - **Kurs- + Mitgliedschafts-Kombinationen**: Bieten Sie den Kauf eines Kurses mit einer optionalen Community-Mitgliedschaft als Abonnement an
   - **Lifetime + Add-on-Abonnements**: Kombinieren Sie lebenslangen Produktzugang mit optionalen wiederkehrenden Premium-Funktionen
   
   ### Funktionsweise
   
   Fügen Sie einfach sowohl einmalige als auch Abonnement-Produkte zu Ihrer Checkout-Session hinzu:
   

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [
       // One-time setup fee
       { product_id: 'prod_setup_fee', quantity: 1 },
       // Monthly subscription
       { product_id: 'prod_monthly_plan', quantity: 1 }
     ],
     customer: {
       email: 'customer@example.com',
       name: 'Jane Doe'
     },
     return_url: 'https://yourapp.com/success'
   });
   ```

   
   <Check>
   Ergebnis: Ein einzelner Checkout-Ablauf, der die einmalige Zahlung verarbeitet und das Abonnement initiiert – wodurch Reibung reduziert und die Conversion-Rate erhöht wird.
   </Check>
   
   <Info>
   Beim Mischen von Produkttypen wird die einmalige Zahlung sofort belastet, während das Abonnement seinem konfigurierten Abrechnungszyklus folgt (einschließlich etwaiger Testzeiträume, die Sie festgelegt haben).
   </Info>
   
   Mehr erfahren: [Checkout-Sitzungen](/developer-resources/checkout-session)

2. **Einheitliches Kundenportal**  
   
   Ein neues zentrales Kundenportal unter [customer.dodopayments.com](https://customer.dodopayments.com), über das Kunden alle ihre Käufe und Abonnements bei verschiedenen Unternehmen auf Dodo Payments anzeigen und verwalten können.
   
   <Frame>
   <img src="/images/changelog/unified-customer-portal.png" alt="Unified Customer Portal" />
   </Frame>
   
   ### Portal-Funktionen
   
   - **Geschäftsübergreifende Sichtbarkeit**: Sehen Sie alle Käufe und Abonnements jedes Unternehmens, das Dodo Payments nutzt, an einem Ort
   - **Zentralisiertes Abonnement-Management**: Verwalten Sie Abonnements verschiedener Händler über ein einziges Dashboard
   - **Einheitliche Abrechnungshistorie**: Greifen Sie auf Rechnungen und Zahlungshistorien aller Ihrer Käufe zu
   - **Single-Sign-On-Erlebnis**: Melden Sie sich einmal mit Ihrer E-Mail-Adresse an, um auf Käufe aller Dodo Payments-Händler zuzugreifen
   
   ### Funktionsweise
   
   Kunden besuchen einfach [customer.dodopayments.com](https://customer.dodopayments.com) und geben ihre E-Mail-Adresse ein. Sie erhalten einen sicheren Login-Link, der Zugang zu all ihren Käufen im Dodo Payments-Ökosystem gewährt.
   
   <Tip>
   Das einheitliche Kundenportal ergänzt geschäftsspezifische Kundenportale. Kunden können weiterhin auf einzelne Händlerportale für ein markenspezifisches Erlebnis zugreifen oder das einheitliche Portal für eine konsolidierte Ansicht nutzen.
   </Tip>
   
   <Note>
   Das einheitliche Portal verwendet die gleiche E-Mail-basierte Authentifizierung wie geschäftsspezifische Portale und sorgt so für ein konsistentes und sicheres Erlebnis.
   </Note>

3. **Rechnungs-ID in der Zahlungsantwort**  
   Die Zahlungsantwort enthält nun ein `invoice_id`-Feld, wodurch das Abgleichen von Zahlungen mit Rechnungen erleichtert und die Integration in Buchhaltungssysteme vereinfacht wird.
   

   ```json
   {
     "payment_id": "pay_2IjeQm4hqU6RA4Z4kwDee",
     "status": "succeeded",
     "invoice_id": "inv_2IsUnWGtRKFLxk7xAQeyt",
     "amount": 9900,
     "currency": "USD",
     // ... other fields
   }
   ```

   
   ### Anwendungsfälle
   
   - **Buchhaltungsintegration**: Verknüpfen Sie Zahlungen direkt mit Rechnungen in Ihrer Buchhaltungssoftware
   - **Kundensupport**: Finden Sie schnell die Rechnung zu einer Zahlung, wenn Sie Support-Anfragen bearbeiten
   - **Abstimmung**: Vereinfachen Sie die finanzielle Abstimmung, indem Sie Zahlungen programmatisch mit Rechnungen abgleichen
   - **Compliance**: Führen Sie genaue Aufzeichnungen, indem Sie Zahlungen mit den entsprechenden Steuerrechnungen verknüpfen
   
   <Info>
   Das `invoice_id`-Feld verwendet indien-spezifische Rechnungs-IDs, wenn verfügbar, und stellt so die Einhaltung lokaler Rechnungsanforderungen sicher.
   </Info>
   
   Mehr erfahren: [Get Payment Detail API](/api-reference/payments/get-payments-1)
