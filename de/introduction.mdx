## Was ist Dodo Payments?

Dodo Payments ist der **All-in-One-Motor**, um weltweit zu starten, zu skalieren und zu monetarisieren. Entwickelt für die neue Generation digitaler SaaS- und KI-Produkte, kümmern wir uns um alles von **Zahlungen über Abrechnung bis hin zu Distribution** - alles ohne zusätzliche Ingenieurleistungen.

<Info>
**Entwickelt für SaaS- und KI-Produkte**: Verarbeiten Sie Einmalzahlungen, Abonnements, Guthaben, Plätze, Token oder jede Nutzungsmetrik mit unserer optimierten Abrechnungsinfrastruktur.
</Info>

<Tip>
  <strong>Treffen Sie Sentra: Ihren KI-gestützten Abrechnungsassistenten</strong><br/>
  Sentra ist ein KI-Agent, der Abrechnung und Zahlungen direkt in Ihrer IDE integriert, analysiert und darauf reagiert. Stellen Sie Fragen, erstellen Sie Integrationen oder planen Sie Abrechnungsarchitekturen - alles, indem Sie beschreiben, was Sie möchten.
  <a href="/developer-resources/sentra">
    Mit Sentra starten &rarr;
  </a>
</Tip>

## Schnellstartanleitung

Starten Sie mit Dodo Payments in nur wenigen Schritten:

<Steps>
<Step title="Erstellen Sie Ihr Konto">
  Melden Sie sich unter [app.dodopayments.com](https://app.dodopayments.com/signup) mit Ihrer E-Mail-Adresse an.

  <Check>
  Sie erhalten eine Bestätigungs-E-Mail, um Ihr Konto zu verifizieren.
  </Check>
</Step>

<Step title="Fügen Sie Ihr erstes Produkt hinzu">
  Wählen Sie Ihr Abrechnungsmodell und fügen Sie Ihr Produkt hinzu, um Zahlungen zu akzeptieren.

  <CardGroup cols={3}>
  <Card title="Einmalige Zahlung" icon="box" href="/features/one-time-payment-products">
    Perfekt für digitale Downloads, Kurse oder Einzelkäufe
  </Card>
  
  <Card title="Abonnementprodukt" icon="repeat" href="/features/subscription">
    Ideal für SaaS, Mitgliedschaften oder wiederkehrende Dienstleistungen
  </Card>
  <Card title="Nutzungsbasierte Abrechnung" icon="chart-bar" href="/features/usage-based-billing">
    Berechnen Sie Kunden für API-Aufrufe, Guthaben, Tokens oder jede Nutzungsmetrik - ideal für KI und SaaS
  </Card>
  </CardGroup>
</Step>

<Step title="Wählen Sie Ihre Integrationsmethode">
  Wählen Sie den Integrationsansatz, der am besten zu Ihren Bedürfnissen und technischen Anforderungen passt.

  <Tabs>
  <Tab title="No-Code (Schnellste)">
    **Perfekt, um schnell zu starten:**
    
    - Erstellen Sie **Zahlungslinks** von Ihrem Dashboard
    - Teilen Sie sie per E-Mail, sozialen Medien oder betten Sie sie in Websites ein
    - Beginnen Sie sofort mit der Annahme von Zahlungen
  </Tab>

  <Tab title="Overlay-Checkout">
    **Am besten für nahtlose, markenkonforme Zahlungen:**
    
    - Starten Sie den Checkout als Overlay auf Ihrer Website oder App
    - Halten Sie die Benutzer auf Ihrer Seite für ein reibungsloses Erlebnis
    - Passen Sie das Erscheinungsbild an, um Ihrer Marke zu entsprechen

    <Card title="Overlay-Integration" icon="layer-group" href="/developer-resources/overlay-checkout">
      Fügen Sie unser Overlay-Checkout mit einer einzigen Codezeile hinzu
    </Card>
  </Tab>

  <Tab title="Inline-Checkout">
    **Am besten für vollständig integrierte, eingebettete Checkout-Erlebnisse:**
    
    - Betten Sie den Checkout direkt in Ihr Seitenlayout ein
    - Erstellen Sie benutzerdefinierte Bestellübersichten, die mit dem Checkout synchronisiert werden
    - Maximale Kontrolle über das Design und den Ablauf des Checkouts

    <Card title="Inline-Integration" icon="credit-card" href="/developer-resources/inline-checkout">
      Betten Sie den Checkout direkt in Ihre Website ein
    </Card>
  </Tab>

  <Tab title="Vollständige SDK-Integration">
    **Am besten für fortgeschrittene, vollständig angepasste Lösungen:**

    - Verwenden Sie unser TypeScript/Python/Go/Java/Kotlin/C#/Ruby SDK für nahtlose Integration
    - Programmgesteuerter Zugriff auf alle Zahlungs-, Abonnement- und Kundenportal-APIs
    - Tiefe Integration mit Ihrem Backend, Workflows und Datenmodellen
    - Vollständige Unterstützung für Webhooks, erweiterte Funktionen und Automatisierung

    <Tip>
      <strong>Überspringen Sie die manuelle Arbeit mit Sentra</strong><br />
      Installieren Sie unseren KI-Agenten in VS Code, Cursor oder Windsurf. Beschreiben Sie, was Sie möchten - SDK-Integration, Webhook-Handler, Abonnementflüsse - und Sentra generiert den Code für Sie.
      <a href="/developer-resources/sentra">
        Sentra installieren &rarr;
      </a>
    </Tip>

    <Card title="SDK-Quickstart" icon="code" href="/developer-resources/integration-guide">
      Erfahren Sie, wie Sie unser offizielles SDK installieren und verwenden
    </Card>
    <Card title="API-Referenz" icon="server" href="/api-reference/introduction">
      Erkunden Sie die vollständige API- und SDK-Dokumentation
    </Card>
  </Tab>
  </Tabs>
</Step>

</Steps>

### MCP-Server für KI-Assistenten

Verbinden Sie Ihren KI-Coding-Assistenten (Cursor, Claude Desktop, Windsurf, VS Code) direkt mit Dodo Payments über unsere MCP-Server.

<CardGroup cols={2}>
<Card title="Dodo Knowledge MCP" icon="book-open" href="/developer-resources/mcp-server#dodo-knowledge-mcp">
  Semantische Suche in der Dodo Payments-Dokumentation - erhalten Sie sofortige Antworten zu Funktionen, APIs und Best Practices
</Card>

<Card title="Dodo Payments MCP" icon="credit-card" href="/developer-resources/mcp-server">
  Führen Sie API-Operationen direkt aus - erstellen Sie Zahlungen, verwalten Sie Abonnements und bearbeiten Sie Rückerstattungen über Ihren KI-Assistenten
</Card>
</CardGroup>

<Accordion title="Schnelle MCP-Einrichtung für Cursor/Claude Desktop">
Fügen Sie beide Server zu Ihrer MCP-Konfiguration (`~/.cursor/mcp.json` oder Claude Desktop-Konfiguration) hinzu:

```json
{
  "mcpServers": {
    "dodo-knowledge": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://knowledge.dodopayments.com/mcp"]
    },
    "dodopayments": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://mcp.dodopayments.com/sse"]
    }
  }
}
```

<Info>
Erfordert Node.js 18+. Siehe die [MCP-Server-Dokumentation](/developer-resources/mcp-server) für detaillierte Einrichtungsanleitungen für alle unterstützten Clients.
</Info>
</Accordion>
