## Umgang mit Zahlungsfehlern

Verwenden Sie den `on_payment_failure`-Parameter, um zu steuern, was passiert, wenn die Zahlung für die Planänderung fehlschlägt:

| Value | Verhalten |
|-------|-----------|
| `prevent_change` | Behalten Sie das Abonnement im aktuellen Plan, bis die Zahlung erfolgreich ist. Die Planänderung bleibt ausstehend. |
| `apply_change` | Wenden Sie die Planänderung sofort an, unabhängig vom Zahlungsergebnis. Dies ist die Standardeinstellung. |
