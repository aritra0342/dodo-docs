<Accordion title="Q1: Warum habe ich die Produkt-ID im payment.succeeded Webhook für ein Abonnementprodukt nicht erhalten?">
  A: Das payment.succeeded Webhook-Ereignis für Abonnementprodukte enthält keine product_id. Stattdessen wird die `subscription_id` bereitgestellt, die mit dem Abonnement verbunden ist, zusammen mit anderen relevanten Details.
</Accordion>

<Accordion title="Q2: Wie kann ich überprüfen, ob die Webhooks, die ich erhalte, tatsächlich von Dodo Payments stammen und nicht von böswilligen Akteuren?">
  A: Um zu überprüfen, dass die Webhooks, die Sie erhalten, tatsächlich von Dodo Payments stammen und nicht von böswilligen Akteuren, befolgen Sie diese Schritte:
    1. **Überprüfen Sie die Signatur**: Jedes Webhook, das von Dodo Payments gesendet wird, enthält eine eindeutige Signatur im **`webhook-signature`** Header. Diese Signatur wird mit dem geheimen Schlüssel Ihres Webhooks und dem Payload des Ereignisses generiert.
    2. **Holen Sie sich den geheimen Schlüssel**: Stellen Sie sicher, dass Sie den geheimen Schlüssel für Ihren Webhook haben, der in Ihrem Dodo Payments-Dashboard unter den Webhook-Einstellungen zu finden ist.
    3. **Berechnen Sie die Signatur neu**: Verwenden Sie denselben Signaturalgorithmus (HMAC-SHA256) wie Dodo Payments, um die Signatur auf Ihrem Server zu berechnen. Verwenden Sie das Payload des Webhooks und Ihren geheimen Webhook-Schlüssel.
    4. **Vergleichen Sie die Signaturen**: Vergleichen Sie die neu berechnete Signatur mit der im **`webhook-signature`** Header bereitgestellten. Wenn sie übereinstimmen, ist das Webhook authentisch; wenn nicht, sollte es abgelehnt werden.
    5. **Verwenden Sie HTTPS**: Stellen Sie sicher, dass Ihr Webhook-Endpunkt HTTPS verwendet, um die Kommunikation zu sichern und Manipulationen während der Datenübertragung zu verhindern.
  Durch die Implementierung dieser Schritte können Sie die Authentizität der von Dodo Payments gesendeten Webhooks mit Zuversicht überprüfen.
</Accordion>

<Accordion title="Q3: Warum erhalte ich ein subscription.renew Webhook sogar für die erste Monatszahlung?">
  A: Bei Abonnements wird `subscription.renew` ausgelöst, wann immer der Abonnementbetrag abgebucht wird. Ja, dies ist also das erwartete Verhalten.
</Accordion>

<Accordion title="Q5: Wie kann ich Webhooks lokal testen?">
  A: Um Webhooks lokal zu testen, können Sie diese Schritte befolgen:
  
1. **Verwenden Sie ein Tunneling-Tool**: Tools wie [ngrok](https://ngrok.com/), [LocalTunnel](https://localtunnel.github.io/www/) oder [Cloudflare Tunnel](https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/) können Ihren lokalen Server ins Internet exponieren. Dies bietet eine öffentliche URL, die Dodo Payments verwenden kann, um Webhook-Ereignisse an Ihre lokale Umgebung zu senden.
2. **Richten Sie den Webhook-Endpunkt ein**: Erstellen Sie einen Webhook-Endpunkt in Ihrer lokalen Anwendung, um eingehende Webhook-Ereignisse zu verarbeiten. Stellen Sie sicher, dass er auf HTTP POST-Anfragen hört.
3. **Konfigurieren Sie den Webhook in Dodo Payments**: Gehen Sie zu Ihrem Dodo Payments-Dashboard, navigieren Sie zu den Webhook-Einstellungen und aktualisieren Sie die Webhook-URL auf die öffentliche URL, die vom Tunneling-Tool generiert wurde.
4. **Ereignisse auslösen**: Führen Sie Aktionen in Ihrer Umgebung aus, um Webhook-Ereignisse auszulösen. Zum Beispiel können Sie Zahlungen oder Abonnementverlängerungen simulieren.
5. **Überprüfen Sie eingehende Anfragen**: Verwenden Sie die Protokolle des Tunneling-Tools oder das Logging Ihrer Anwendung, um die eingehenden Webhook-Payloads zu überprüfen. Dies ermöglicht es Ihnen zu bestätigen, dass Ihr Endpunkt die Ereignisse empfängt und korrekt verarbeitet.
6. **Signatur überprüfen**: Stellen Sie sicher, dass Sie den **`webhook-signature`** Header validieren, um zu bestätigen, dass die Webhook-Ereignisse authentisch sind und von Dodo Payments gesendet wurden.

Indem Sie diese Schritte befolgen, können Sie Webhooks in Ihrer lokalen Entwicklungsumgebung einfach testen und debuggen.
</Accordion>
