## Was ist eine Abonnement-Migration?

Eine Abonnement-Migration überträgt Ihre bestehenden Abonnenten von Ihrem alten Zahlungsanbieter zu Dodo Payments und bietet Ihnen eine einzige Quelle der Wahrheit für alle Ihre Kundendaten. Ohne eine Migration bleiben die Kunden, die Sie bis heute gewonnen haben, in Ihrem alten System, und nur neue Kunden wären in Dodo Payments sichtbar.

Der Migrationsprozess sorgt für ein nahtloses Abrechnungserlebnis für Ihre Kunden – sie müssen ihre Zahlungsdaten nicht erneut eingeben und erleben keine Unterbrechung ihrer Abonnements.

<Warning>
Derzeit unterstützen wir nur die Migration von **Kartentoken**. Apple Pay, Google Pay und andere Wallet-Zahlungsmethoden können aufgrund der von diesen Anbietern auferlegten Tokenisierungsbeschränkungen nicht zwischen Zahlungsdienstleistern übertragen werden.
</Warning>

## Übersicht des Migrationsprozesses

Der Abonnement-Migrationsprozess besteht aus drei Hauptteilen:

1. **Übertragung der Zahlungsmethode** — Sicherer Transfer sensibler Kartendaten von Ihrem aktuellen Anbieter
2. **Übertragung der Abonnentendaten** — Export von Kundendaten, Abonnementplänen und Abrechnungsplänen
3. **Import und Validierung** — Vereinheitlichung der Zahlungsmethoden und Abonnentendaten in Dodo Payments

<Note>
Um eine Abonnementmigration zu starten, kontaktiere unser [Support-Team](mailto:support@dodopayments.com). Sie verbinden dich mit einem dedizierten Solutions Engineer, der dich durch den gesamten Prozess führt.
</Note>

## Anforderungen

Bevor Sie eine Migration einleiten, stellen Sie sicher, dass Sie die folgenden Anforderungen erfüllen:

<AccordionGroup>
<Accordion title="PCI DSS compliance verification">
Dodo Payments ist PCI-DSS-Level-1-konform. Um den Compliance-Anforderungen gerecht zu werden, dürfen wir Kartendaten nur von einem anderen PCI-DSS-Level-1-konformen Zahlungsdienstleister empfangen.

Ihr aktueller Anbieter muss bereitstellen:
- Ihre aktuelle PCI-Bestätigung der Konformität (AOC) oder deren Eintrag im globalen Register der Dienstanbieter von Visa
- Bestätigung, dass sie sichere Kartendaten-Exporte unterstützen

Sieh dir unsere [Attestation of Compliance (AOC)](https://pub-2a89184e4c954b599464449d23f506bb.r2.dev/dodo-payments-pci-dss-aoc.pdf) an, um sie mit deinem aktuellen Dienstleister zu teilen.
</Accordion>

<Accordion title="Dodo Payments integration">
Schließe deine Dodo Payments-Integration ab und stelle sicher, dass alle neuen Kunden zum Dodo Payments-Checkout weitergeleitet werden, bevor du mit der Migration beginnst. Dies verhindert Dateninkonsistenzen während des Übergangs.
</Accordion>

<Accordion title="Data access authorization">
Stelle sicher, dass du die erforderliche Autorisierung hast, um eine Datenexportanforderung bei deinem aktuellen Dienstleister zu stellen. Dies erfordert in der Regel:
- Verifizierung des Kontoinhabers
- Schriftliche Genehmigung für den Zugriff auf sensible Daten
- Abschluss der Sicherheitsverifizierungsmaßnahmen deines Dienstleisters
</Accordion>
</AccordionGroup>

## Schritt-für-Schritt-Migrationsanleitung

<Steps>
<Step title="Request payment method export from your current processor">
Kontaktiere das Support-Team deines aktuellen Zahlungsdienstleisters und stelle eine formelle Anfrage für einen vollständigen Export deiner Kundenkartentoken.

Sie müssen:
- Eine Überprüfung Ihrer Identität und Autorisierung bereitstellen
- Anfordern, dass sie den Export mit dem PGP-Öffentlichen Schlüssel von Dodo Payments (unten bereitgestellt) verschlüsseln
- Sie bitten, die verschlüsselte Datei direkt an unseren sicheren SFTP-Server zu übertragen

<Info>
Sichere Übertragungen erfolgen direkt von Server zu Server über SFTP. Dein Solutions Engineer wird die SFTP-Zugangsdaten bereitstellen, die du mit deinem aktuellen Dienstleister teilen kannst.
</Info>

Im Durchschnitt dauert dieser Prozess etwa **2 Wochen**, wobei die Dauer je nach Politik und Reaktionszeit deines aktuellen Dienstleisters variieren kann.
</Step>

<Step title="Prepare subscriber data export">
Solange die Übertragung der Zahlungsmethode läuft, exportiere deine Abonnentendaten aus deinem Legacy-System. Dein Solutions Engineer stellt dir eine CSV-Vorlage zur Verfügung und führt dich durch die erforderlichen Felder.

### Erforderliche Abonnentendaten

| Field | Description |
|-------|-------------|
| `customer_id` | Deine interne Kundenkennung |
| `email` | E-Mail-Adresse des Kunden |
| `name` | Vollständiger Name des Kunden |
| `country` | Zwei Buchstaben langer Ländercode (ISO 3166-1 alpha-2) |
| `currency` | Drei Buchstaben langer Währungscode (ISO 4217) |
| `payment_amount` | Abonnementbetrag in der kleinsten Währungseinheit |
| `payment_interval` | Abrechnungsfrequenz (monatlich, jährlich usw.) |
| `next_billing_date` | Nächstes geplantes Abrechnungsdatum (ISO 8601) |
| `card_token` | Eindeutiger Token deines aktuellen Dienstleisters |

<Warning>
Das Feld `card_token` ist entscheidend—es wird verwendet, um Abonnenten mit ihren Zahlungsmethoden beim Import abzugleichen. Stelle sicher, dass es genau mit dem Token im Export der Zahlungsmethode übereinstimmt.
</Warning>

### Beispiel-Datenformat

```json
{
  "customers": [
    {
      "customer_id": "cus_abc123",
      "email": "jane.doe@example.com",
      "name": "Jane Doe",
      "country": "US",
      "currency": "USD",
      "payment_amount": 2999,
      "payment_interval": "monthly",
      "next_billing_date": "2025-01-15T00:00:00Z",
      "card_token": "tok_xyz789",
      "metadata": {
        "plan": "pro",
        "signup_source": "website"
      }
    }
  ]
}
```

</Step>

<Step title="Validation and testing">
Vor dem Importtermin wird unser Team:

- Die Integrität und Verschlüsselung der Zahlungsmethodendaten überprüfen
- Ihre Abonnenten-CSV auf korrekte Formatierung validieren
- Einen Testimport in unserer Sandbox-Umgebung durchführen
- Alle Datenzuordnungsprobleme identifizieren und lösen

<Check>
Du erhältst einen Validierungsbericht, der bestätigt, dass alles für den Produktionsimport bereit ist.
</Check>
</Step>

<Step title="Production import">
Am vereinbarten Importtermin wird unser Engineering-Team:

1. Die Zahlungsmethodendaten in unseren PCI-konformen Card Vault importieren
2. Ihre Abonnentendaten importieren
3. Abonnenten mit ihren Zahlungsmethoden unter Verwendung der Kartentoken abgleichen
4. Alle migrierten Abonnements aktivieren

<Info>
Die typische Bearbeitungsdauer beträgt **2-3 Arbeitstage** für Standardmigrationen, abhängig vom Volumen der Abonnenten.
</Info>
</Step>

<Step title="Post-migration verification">
Nach Abschluss des Imports erhältst du:

- Einen umfassenden Migrationsbericht
- Eine CSV, die Ihre alten Abonnement-IDs mit neuen Dodo Payments-IDs abgleicht
- Zugriff auf alle migrierten Kunden in Ihrem Dodo Payments-Dashboard

Wir empfehlen:
- Eine Stichprobe der migrierten Abonnements in deinem Dashboard zu überprüfen
- Einige Abonnementverlängerungen im Sandbox-Modus zu testen
- Deine internen Datenbankverweise mit den neuen IDs zu aktualisieren
- Für den ersten Abrechnungszeitraum eine parallele Überwachung durchzuführen
</Step>
</Steps>

## Übertragbare Daten

Die folgenden Daten können während einer Abonnement-Migration übertragen werden:

| Datentyp | Unterstützt | Hinweise |
|-----------|-----------|-------|
| Kredit-/Debitkartennummern | ✅ | Verschlüsselt über PGP |
| Kartenauslaufdaten | ✅ | |
| Karteninhabernamen | ✅ | |
| Rechnungsadressen | ✅ | |
| E-Mail-Adressen der Kunden | ✅ | |
| Kundenmetadaten | ✅ | Benutzerdefinierte Schlüssel-Wert-Paare |
| Abonnementdetails | ✅ | Plan, Betrag, Intervall |
| Apple Pay-Token | ❌ | Können nicht übertragen werden |
| Google Pay-Token | ❌ | Können nicht übertragen werden |
| Andere Wallet-Token | ❌ | Können nicht übertragen werden |
| Zahlungshistorie | ❌ | Export von Ihrem aktuellen Anbieter |

<Note>
Zahlungshistorie und Transaktionsaufzeichnungen können nicht migriert werden. Wir empfehlen, diese Daten vor dem Abschluss der Umstellung von deinem aktuellen Dienstleister zu exportieren und für deine Unterlagen aufzubewahren.
</Note>

## Zeitrahmen-Erwartungen

| Phase | Dauer |
|-------|----------|
| Erste Einrichtung und Planung | 1-2 Tage |
| Übertragung der Zahlungsmethode | ~2 Wochen |
| Vorbereitung der Abonnentendaten | 3-5 Tage (parallel) |
| Validierung und Test | 2-3 Tage |
| Produktionsimport | 2-3 Werktage |
| Nach-Migration-Verifizierung | 1-2 Tage |

<Tip>
Die Gesamtmigration dauert in der Regel **3-4 Wochen** von Anfang bis Ende. Wenn du mit der Vorbereitung deiner Abonnentendaten beginnst, während die Übertragung der Zahlungsmethode läuft, verkürzt das den gesamten Zeitrahmen.
</Tip>
