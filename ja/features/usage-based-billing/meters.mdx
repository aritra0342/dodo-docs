<Info>
メーターは生イベントを課金対象の数量に変換します。イベントをフィルタリングし、集計関数（Count、Sum、Max、Last）を適用して顧客ごとの使用量を算出します。
</Info>

## メーターの作成

<Steps>
{/* LOCKED_PATTERN_148f3cd6016b0fbbcdca7a493126f782 */}
{/* LOCKED_PATTERN_29b9e112369bf16db8b48662b2a38ae8 */}
説明的な名前（例: «API Requests», «Token Usage»）
</ParamField>

<ParamField path="Event Name" type="string" required>
一致させる正確なイベント名（大文字小文字を区別）。例: `api.call`、`image.generated`
</ParamField>
</Step>

{/* LOCKED_PATTERN_2971978ae085c72cb2eed501c36fa0a8 */}
{/* LOCKED_PATTERN_97313bb6bf41a89f8d2567ad673466f3 */}
イベントの集計方法を選択:


- **Count**: イベントの総数（API 呼び出し、アップロードなど）
- **Sum**: 数値を合計（トークン、バイトなど）
- **Max**: 期間内の最大値（ピークユーザー数など）
- **Last**: 最新の値
</ParamField>

<ParamField path="Over Property" type="string">
Count を除くすべてのタイプで必須となる集計対象のメタデータキー。例: `tokens`、`bytes`、`duration_ms`
</ParamField>

<ParamField path="Measurement Unit" type="string" required>
請求書上の単位ラベル。例: `calls`、`tokens`、`GB`、`hours`
</ParamField>
</Step>

{/* LOCKED_PATTERN_709e3e0ef72c95833fd8d928d7a0591a */}
<Frame>
<img src="/images/usage-based/UBB-3.png" alt="イベントフィルタリング" />
</Frame>

カウントされるイベントをフィルタリングする条件を追加します：
- **ANDロジック**：すべての条件が一致する必要があります
- **ORロジック**：任意の条件が一致する可能性があります

**比較演算子**：等しい、等しくない、大きい、小さい、含む

フィルタリングを有効にし、ロジックを選択し、プロパティキー、比較演算子、値を使って条件を追加します。
</Step>

{/* LOCKED_PATTERN_314e5b01027621505255760af9937f80 */}
設定を確認し、**Create Meter** をクリックします。
</Step>
</Steps>

## トラブルシューティング

<AccordionGroup>
{/* LOCKED_PATTERN_fc1788ddf0982a3a2fe9f6e81812c541 */}
- イベント名は完全一致（大文字小文字を区別）する必要があります
- メーターのフィルターがイベントを除外していないか確認してください
- 顧客 ID が存在することを確認してください
- テストのため一時的にフィルターを無効にします
</Accordion>

{/* LOCKED_PATTERN_ddafb5b33ecc6ac484aee2e1a2c4cd7d */}
- Over Property がメタデータキーと完全に一致していることを確認します
- 文字列ではなく数値を使用します: `tokens: 150` ではなく `"150"`
- 必要なプロパティをすべてのイベントに含めます
</Accordion>

{/* LOCKED_PATTERN_7673f2ad014dbdbdfd53c822f111f71d */}
- 大文字小文字を正確に一致させます
- データ型に適した演算子を使用します
- イベントにフィルター対象のプロパティが含まれていることを確認します
</Accordion>

{/* LOCKED_PATTERN_9641c2e1836928ca6e48f68372e5b21d */}
- Events タブで実際に受信したイベント数を確認します
- 集計タイプ（Count と Sum）を確認します
- Sum/Max に対して値が数値であることを確認します
</Accordion>
</AccordionGroup>
