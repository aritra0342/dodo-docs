インジェクションブループリントは、一般的なシナリオでの使用状況を追跡するための事前構築されたパターンです。イベント構造やメーター設定をゼロから考えるのではなく、実績のあるブループリントから始めて、ニーズに合わせてカスタマイズします。

## ブループリントとは？

各ブループリントには以下が含まれます：
- **イベント構造の推奨**：最適なイベント名付けとメタデータ設計
- **メーター設定**：推奨される集約タイプとフィルター
- **サンプルコード**：複数の言語での動作実装例
- **ベストプラクティス**：精度、パフォーマンス、コスト最適化のためのヒント

<Tip>
ブループリントはセットアップ時間を何時間も節約し、一般的な落とし穴を回避するのに役立ちます。ブループリントから始めて、特定の要件に合わせて調整してください。
</Tip>

## 利用可能なブループリント

あなたのユースケースに合ったブループリントを選択してください：

<CardGroup cols={1}>
<Card title="LLM Blueprint" icon="brain-circuit" href="/developer-resources/ingestion-blueprints/llm">
OpenAI、Anthropic、Groq、Gemini、Vercel AI SDK などにわたる AI トークン使用量を追跡します。

**最適な用途**：AIチャットボット、コードジェネレーター、コンテンツ作成ツール、AI駆動アプリ。

**追跡対象**: 入力トークン、出力トークン、モデル使用量、リクエスト数。

<Card title="API Gateway Blueprint" icon="cloud" href="/developer-resources/ingestion-blueprints/api-gateway">
ゲートウェイレベルで API リクエストを計測し、エンドポイントフィルタリングとレート制限をサポートします。

**最適な用途**：REST API、GraphQLサービス、Webhookプロバイダー、APIファーストプラットフォーム。

**追跡対象**: リクエスト数、エンドポイントの利用、HTTP メソッド、応答時間。
</Card>

<Card title="Object Storage Blueprint" icon="box-archive" href="/developer-resources/ingestion-blueprints/object-storage">
クラウドストレージサービスのファイルアップロード、ストレージ消費、帯域幅を追跡します。

**最適な用途**：ファイルホスティング、メディアプラットフォーム、バックアップサービス、ドキュメント管理。

**追跡対象**: 保存されたバイト数、ファイルのアップロード、ダウンロード、ストレージスナップショット。
</Card>

<Card title="Stream Blueprint" icon="tower-broadcast" href="/developer-resources/ingestion-blueprints/stream">
ビデオ、オーディオ、リアルタイムデータ配信のストリーミング帯域幅を測定します。

**最適な用途**：ビデオストリーミング、オーディオプラットフォーム、ライブ放送、リアルタイムデータフィード。

**追跡対象**: 消費された帯域幅、ストリームの継続時間、品質レベル、同時ストリーム数。
</Card>

<Card title="Time Range Blueprint" icon="clock" href="/developer-resources/ingestion-blueprints/time-range">
コンピュートリソース、サーバーレス関数、長時間実行ジョブの経過時間で請求します。

**最適な用途**：サーバーレスプラットフォーム、コンテナホスティング、コンピュートインスタンス、バッチ処理。

**追跡対象**: 実行時間、コンピュート時間、関数の呼び出し、ジョブの継続時間。
</Card>
</CardGroup>

## ブループリントの使用方法

<Steps>
<Step title="Choose your blueprint">
請求シナリオに最も適したブループリントを選択してください。各ブループリントページには詳細な実装ガイドが含まれています。
</Step>

<Step title="Review the event structure">
推奨されるイベント命名とメタデータ設計を理解してください。イベント構造をユースケースにコピーします。
</Step>

<Step title="Create the meter">
ブループリントのメーター構成手順に従ってください。推奨される集計タイプやフィルタを使用します。
</Step>

<Step title="Implement event sending">
使用する言語/フレームワークのサンプルコードをコピーします。プレースホルダー値を実際の API キーや顧客 ID に置き換えてください。
</Step>

<Step title="Test and verify">
テストイベントを送信し、正しい集計でメーターダッシュボードに表示されることを確認します。
</Step>

<Step title="Customize as needed">
ブループリントを特定の要件に合わせて調整します—カスタムメタデータを追加したり、フィルタを調整したり、価格設定を変更したりしてください。
</Step>
</Steps>

## ブループリントの比較

どのブループリントがあなたのニーズに合うか分からない？ここに簡単な比較があります：

| ユースケース | ブループリント | 集約 | 一般的なメタデータ |
|----------|-----------|-------------|-----------------|
| AIチャットアプリ | LLM | 合計 (トークン) | model, input_tokens, output_tokens |
| REST API | APIゲートウェイ | カウント | endpoint, method, status_code |
| ファイルストレージ | オブジェクトストレージ | 合計 (バイト) | file_size, file_type, operation |
| ビデオプラットフォーム | ストリーム | 合計 (帯域幅) | duration, quality, bitrate |
| サーバーレス | 時間範囲 | 合計 (時間) | function_name, execution_time_ms |

## カスタム実装

正確なユースケースに合ったブループリントが見つからない？問題ありません：

1. **最も近いブループリントから始める**：ニーズに最も似たものを選択します
2. **イベント構造を適応させる**：メタデータフィールドを要件に合わせて変更します
3. **メーター設定を調整する**：必要に応じて集約タイプやフィルターを変更します
4. **徹底的にテストする**：サンプル使用量で請求計算を確認します

<Info>
すべてのブループリントは出発点です。異なるブループリントの概念を組み合わせたり、完全にカスタムの実装を作成したりして構いません。
</Info>
