<Info>
製品コレクションを使用すると、関連する製品（例：スターター、プロ、エンタープライズプラン）を1つの傘の下にグループ化できます。すべてのオプションを単一のチェックアウトで表示し、アップグレード/ダウングレードパスを定義し、顧客がCustomer Portalから直接プランを切り替える柔軟性を提供します。
</Info>

## 主なハイライト

- **コレクションベースの構造**: 関連する製品（プラン、ティア、価格オプション）を単一のコレクションにグループ化して、管理を整理します。
- **1つのコレクション、複数の製品**: スターター、プロ、ライフタイムなど、独自の価格モデルを持つ複数の製品を含めることができます。
- **ダイナミックなチェックアウト体験**: 収集された製品を1つのチェックアウトビューで表示し、顧客が希望のプランを選択できます。
- **マーチャントレベルの制御**: 各コレクション内で製品を有効化、無効化、および順序を変更できます。最初の製品はチェックアウト時に自動的に事前選択されます。
- **ライフサイクル認識**: 顧客がCustomer Portalを介して同じコレクション内の製品をアップグレードまたはダウングレードできるようにします。

## 製品コレクションの作成

製品コレクションは、ダッシュボードまたはAPIを介して作成および管理されます。各コレクションは、関連する製品のコンテナとして機能します。

<Steps>
<Step title="コレクションを作成する">
コレクションに名前と任意の説明を定義します。チェックアウトでコレクションを視覚的に表す画像をアップロードしてください。

<Frame>
  <img 
    src="/images/product-collection/collection-form.png" 
    alt="ダッシュボードの製品コレクション作成フォームのスクリーンショットで、名前、説明、画像アップロードのためのフィールドが表示されています" 
  />
</Frame>

**コレクションフィールド:**
- **名前** (必須): コレクションの表示名（例："SaaSプラン"、"ライセンスティア"）
- **説明** (任意): チェックアウトで表示される簡単な説明
- **画像** (任意): コレクションの視覚的ブランディング

</Step>

<Step title="コレクションに製品を追加する">
既存の製品をコレクションに追加します。製品はより良い構造のためにグループに整理できます。

<Frame>
  <img 
    src="/images/product-collection/collection-form-products.png" 
    alt="製品コレクションの製品ページのスクリーンショットで、製品のリストを表示し、それらをコレクションに追加することができる" 
  />
</Frame>

**製品の整理:**
- **グループ**: 製品を名前付きグループにオプションで整理（例："月間プラン"、"年間プラン"）
- **無グループ製品**: グループに属さない製品はコレクションレベルに表示されます
- **順序設定**: ドラッグ＆ドロップで表示順を設定

<Warning>
各製品は1つのコレクションのみに属することができます。製品が他のコレクションにすでに存在する場合、最初にそれを削除する必要があります。
</Warning>

</Step>

<Step title="表示順序と可視性を構成する">
コレクション内で製品の表示順序と可視性を制御します。

**設定オプション:**
- **製品の状態**: コレクション内の各製品を有効または無効にする
- **表示順序**: ドラッグ＆ドロップで製品がチェックアウトに表示される順序を設定

<Info>
コレクション内の最初の製品は自動的にデフォルトとしてチェックアウト時に事前選択されます。製品の順序を変更して、デフォルトで選択される製品を変更できます。
</Info>

</Step>
</Steps>

## コレクションチェックアウト

コレクションは、顧客が1つの場所で利用可能なすべての製品を表示して選択できる統一されたチェックアウト体験を提供します。

### チェックアウトタイプ

| タイプ | 説明 | ユースケース |
|------|-------------|----------|
| **コレクションチェックアウト** | コレクション内のすべてのアクティブな製品を表示 | サブスクリプションプランの選択、ティアードプライシング |
| **単一製品チェックアウト** | 特定の1つの製品のみを表示 | 直接購入、プロモーションリンク |

### コレクションチェックアウト体験

コレクションチェックアウトを使用する際：

1. **すべてのアクティブな製品が表示される**: 顧客はコレクション内のすべての有効な製品を見ることができます
2. **最初の製品が事前選択される**: コレクション順の最初の製品が自動的に選択されます
3. **製品の詳細が表示される**: 各製品はその名前、説明、および価格を表示します
4. **単一選択**: 顧客は購入する製品を1つ選択します
5. **標準フローが続く**: 選択後、チェックアウトは選択された製品の価格および請求設定に従って進行します

<Frame>
  <img 
    src="/images/product-collection/checkout-page.png" 
    alt="複数の製品が表示されている製品コレクションのチェックアウトページのスクリーンショット" 
  />
</Frame>

<Tip>
コレクションチェックアウトは、顧客が購入前にプランを並べて比較することを希望するサブスクリプションビジネスに最適です。
</Tip>

### API統合

コレクションのためのチェックアウトセッションを作成します：

```typescript
const session = await client.checkoutSessions.create({
  product_collection_id: 'pdc_abc123',
  product_cart: [], // Required: pass an empty array for collection checkout
  return_url: 'https://yoursite.com/return'
});

// Redirect customer to the checkout
window.location.href = session.url;
```

<Warning>
`product_collection_id`を使用する際には、セッションの作成時にディスカウントコードを適用できません。顧客は、チェックアウト時に有効であればディスカウントコードを入力できます。
</Warning>

## Customer Portal統合

顧客は同じコレクション内で製品をアップグレードまたはダウングレードできます。

### プラン管理アクション

| アクション | 説明 | マーチャント制御 |
|--------|-------------|------------------|
| **現在のプランを表示** | 現在の製品名、価格、更新日を表示 |
| **プランをアップグレード** | 同じコレクション内の高ティア製品に移動 |
| **プランをダウングレード** | 同じコレクション内の低ティア製品に移動 |
| **キャンセル** | サブスクリプションを完全にキャンセル |
| **常に利用可能** | 常に利用可能 |

  <img 
    src="/images/product-collection/portal.png"

  />
</Frame>

### アップグレード/ダウングレードルール

- アップグレードおよびダウングレードは、**同じコレクション内の製品間**でのみ利用可能です
- プロレーションは、サブスクリプションの設定に基づいて適用されます
- アップグレード、ダウングレード、キャンセルのたびにビジネスにメール通知が送信されます

<Info>
顧客は現在のコレクション外の製品に変更を加えることはできません。異なる製品ライン用の別々のコレクションを作成してください。
</Info>

## サブスクリプション設定

ダッシュボードのサブスクリプション設定セクションから、あなたのビジネスにおけるサブスクリプションとプラン変更の動作を設定します。

<Frame>
  <img 
    src="/images/product-collection/business-settings.png" 
    alt="サブスクリプション設定ページのスクリーンショットで、複数のサブスクリプションを許可するオプションとサブスクリプションの更新を許可するオプションが表示されています" 
  />
</Frame>

### 利用可能な設定

| 設定 | 説明 | デフォルト |
|---------|-------------|---------|
| **複数のサブスクリプションを許可** | 顧客は同時に複数のアクティブなサブスクリプションを保持できます | 有効 |
| **サブスクリプションの更新を許可** | 顧客はCustomer Portalを介していつでも既存のサブスクリプションをアップグレードまたはダウングレードできます | 無効 |

<Info>
Customer Portal経由のプラン変更は、デフォルトで無効になっています。顧客が同じコレクション内の製品をアップグレードまたはダウングレードできるようにするには、サブスクリプション設定で"サブスクリプションの更新を許可"を有効にしてください。
</Info>

## コレクションの管理

製品コレクションは、Dodo Paymentsダッシュボードを介してのみ作成および管理されます。

<Info>
コレクションの作成、更新、削除はダッシュボードを介してのみ利用可能です。APIは、統合目的のためにコレクションデータを取得するための読み取り専用アクセスを提供します。
</Info>

### ダッシュボード操作

- **作成**: 製品とグループを含む新しいコレクションを設定
- **更新**: 名前、説明、画像、製品の整理を変更
- **順序の変更**: ドラッグ＆ドロップで製品の表示順を変更
- **製品の有効/無効**: どの製品がチェックアウトに表示されるかを制御
- **削除**: コレクションを削除（製品は残りますが、リンクされなくなります）

## ベストプラクティス

- **論理的にグループ化**: 製品を請求間隔（月次/年次）または機能ティア（スターター/プロ/エンタープライズ）で整理します
- **戦略的に順序付け**: 最も人気のあるまたは推奨されるプランを最初に配置し、チェックアウト時に事前選択されるようにします
- **明確な命名を使用**: 製品名は価値の違いを明確に伝えるべきです
- **両方向を有効にする**: アップグレードとダウングレードの両方を可能にして、顧客に柔軟性を提供します
- **プロレーションを考慮する**: ビジネスモデルに合ったプロレーションモードを選択します
- **徹底的にテストする**: ライブ稼働前にテストモードでチェックアウトおよびプラン変更フローを確認します
