Dodo Paymentsは、世界中の顧客にサービスを提供するために、幅広い支払い方法をサポートしています。すべてのリストされた支払い方法は、マーチャントアカウントが確認され、ライブモードが有効になると自動的に利用可能になります。

<Info>
**自動アクティベーション:** あなたの側で追加の設定は必要ありません。すべての支払い方法はDodo Paymentsによって事前に設定され、管理されています。オプションで、以下に説明するAPI設定を使用して、チェックアウト時に表示される方法を制限することができます。
</Info>

<Note>
各取引は、<strong>各通貨に必要な最低金額</strong>を満たす必要があります。USDの場合、最低金額は<strong>0.50 USD</strong>です。他の通貨の最低金額は地域によって異なります。
[サポートされている通貨と最低金額を確認する](/features/adaptive-currency#supported-currencies)

</Info>

デフォルトでは、利用可能なすべての支払い方法が自動的に表示されます。表示する方法を制限するには`allowed_payment_method_types`を使用してください:

<Tabs>
{/* LOCKED_PATTERN_7202147f6fc990d9a8a4b3a4cdc10768 */}

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Steps>
<Step title="APIリクエストにパラメータを追加">
支払いまたはサブスクリプション作成リクエストに`allowed_payment_method_types`フィールドを含めます:

| Type | Description |
| :--- | :---------- |
| `credit` | クレジットカード（Visa、Mastercard、Amexなど） |
| `debit` | デビットカード |
| `apple_pay` | Apple Pay |
| `google_pay` | Google Pay |
| `amazon_pay` | Amazon Pay |
| `cashapp` | Cash App Pay |
| `klarna` | Klarnaの後払い |
| `afterpay_clearpay` | Afterpay / Clearpay |
| `upi_collect` | UPI Collect（QRコード） |
| `ideal` | iDEAL（オランダ） |
| `bancontact_card` | Bancontact（ベルギー） |
| `eps` | EPS（オーストリア） |
| `multibanco` | Multibanco（ポルトガル） |
| `revolut_pay` | RevolutPay |
| `billie` | BillieのB2B後払い |

<Warning>
常に`credit`と`debit`をフォールバックとして含めてください。指定したすべての支払い方法が顧客に利用不可の場合、チェックアウトは失敗します。
</Warning>

<Step title="設定をテスト">

### APIエンドポイント

<AccordionGroup>
{/* LOCKED_PATTERN_63da960e31e014d866efe70101f28b99 */}
ターゲット市場向けに地域の支払い方法を含めてください。iDEALはオランダのオンライン決済の約60%を占め、インドではUPIが支配的です。これらを省略するとコンバージョンに大きな悪影響を与えます。

{/* LOCKED_PATTERN_78f59c035688fa14218708818c9f0fca */}

</Accordion>

{/* LOCKED_PATTERN_564640b27644e47b9e4e7faaec6c2d88 */}
BNPL方法（Klarna、Afterpay）は最低$50.01が必要です。低価格の商品に含めないでください — 表示されないだけで問題ありません。
</Accordion>

<AccordionGroup>
{/* LOCKED_PATTERN_fbf2d38e7725fb97384bef01324b59a1 */}
**これらの要因を確認してください:**
1. 方法が`allowed_payment_method_types`に含まれていますか?
2. 顧客の所在地が方法のサポート地域と一致していますか?
3. 取引通貨は方法でサポートされていますか?
4. 取引金額は最低額を満たしていますか（例：BNPLは$50.01）?
5. サブスクリプションの場合：方法は定期支払いに対応していますか?

**解決策：** 一時的に`allowed_payment_method_types`を削除して利用可能なすべての方法を確認し、問題を絞り込みます。
</Accordion>

{/* LOCKED_PATTERN_57385d59bd851fb1e8c26266b489efd4 */}
**考えられる原因：**
- `allowed_payment_method_types`が正しく渡されていない
- パラメータの書式が間違っている（文字列の配列である必要があります）

**解決策：** APIリクエストのペイロードをログに記録し、パラメータ構造を確認してください。
</Accordion>

{/* LOCKED_PATTERN_6d1a1b1a59cbe2789bbc9434eb1ec153 */}
**確認:**
- 顧客の請求先住所が方法の地域と一致しているか
- 通貨が正しく設定されているか（例：iDEALではEUR）
- 顧客が海外の場合はAdaptive Currencyが有効になっているか

**解決策：** APIリクエストで請求先住所の国コードと通貨を確認してください。
</Accordion>

UPIおよびRupayのサブスクリプションには、48時間の処理遅延やRBIマンダテの制限など独自の要件があります。

**参照：** <a href="/features/payment-methods/india">India Payment Methods</a> で完全なガイダンスを確認してください。
</Accordion>
