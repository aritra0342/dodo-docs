## はじめに

**残高**セクションは、すべての通貨にわたる商人の財務活動の包括的な概要を提供します。これには、支払い、返金、手数料、税金、支払いなど、すべての取引タイプが含まれており、収益と控除の両方を完全に把握できます。

## ウォレット構造

各ビジネスアカウントは、サポートされている支払い通貨に基づいて**4つのネイティブウォレット**を維持します：

| ウォレット | 通貨コード |
| --- | --- |
| USDウォレット | USD |
| GBPウォレット | GBP |
| EURウォレット | EUR |
| INRウォレット | INR |

### ウォレットロジック

- 商人の売上は、**顧客の支払い通貨**と**地理的位置**に基づいて**適切なウォレットに入金**されます。
- このシステムは、**支払い処理業者の効率**、**取引手数料の削減**、および**通貨換算手数料の低減**を最適化します。

## 残高台帳

各ウォレットには、その特定のウォレットの通貨でのすべての関連財務活動（支払い、返金、手数料、支払い）を記録する**独立した台帳**があります。

**統合台帳**は、すべてのウォレットデータを組み合わせて、商人の全財務履歴の統一されたビューを提供します。

### フィルターとナビゲーション

- **通貨フィルター：** 商人が特定のウォレット（USD、INR、GBP、EUR）による取引を表示できるようにします。
- **すべての取引ビュー：** すべてのウォレットにわたる集約ビューを提供し、統一されたソート、フィルタリング、および検索機能を備えています。

### 取引タイプ

残高台帳に記録される取引タイプは次のとおりです：

| 取引タイプ | シンボル | 説明 |
| --- | --- | --- |
| **支払い** | クレジット (+) | 顧客の成功した支払い |
| **返金** | デビット (-) | 顧客への返金額 |
| **取引手数料** | デビット (-) | 支払いに適用される処理手数料 |
| **返金手数料** | デビット (-) | 返金処理のために差し引かれる手数料 |
| **売上税** | デビット (-) | 支払いに対して課される税額 |
| **売上税の返金** | クレジット (+) | 返金に対する売上税の返金部分 |
| **支払い** | デビット (-) | 商人の銀行口座に転送される金額 |
| **支払い手数料** | デビット (-) | 支払いから差し引かれる取引手数料 |

#### 追加フィールド

- **取引金額：** 取引に関連する総額。
- **取引タイムスタンプ：** 取引完了の日付と時刻。
- **取引ID：** トラッキングおよびサポート目的のための一意の識別子。

## 総残高

**総残高**セクションは、**USD、GBP、EUR**のすべてのウォレット残高の**統合された指標額**を表示します。

<Note>
**参考表示：** 表示されている金額は**参考値のみ**です。最終的な支払い金額は、送金時点の実際の為替レートによって変動する可能性があります。
</Note>

## 結論

**残高**は、複数通貨の資金管理において**透明性、正確性、柔軟性**を提供することを目指しています。

別々のネイティブウォレットを維持し、通貨ベースのフィルターを提供し、指標的な統合ビューを可能にすることで、Dodo Paymentsは商人が常に自分の収益と支払いの準備状況を明確に理解できるようにしています。

<Tip>
差異やご不明点がある場合は、**Transaction ID（取引ID）**と問題の詳細を添えて、**support@dodopayments.com** までご連絡ください。
</Tip>
