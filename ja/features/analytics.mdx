ストアフロントからチェックアウト、購入完了までの顧客ジャーニーを追跡します。Dodo Payments は Google Analytics 4、Google Tag Manager、Meta Pixel と連携し、ホストされたすべてのページで包括的な分析を提供します。

<Info>
ダッシュボードにトラッキング ID を追加すると、ストアフロント、チェックアウトページ、カスタマーポータルで分析トラッキングが自動的に有効になります。
</Info>

<Warning>
顧客のブラウザにインストールされた広告ブロッカーによって、一部の訪問者で分析トラッキングが機能しない場合があります。
</Warning>

## はじめに

<Steps>
<Step title="Open Business Settings">
  Dodo Payments ダッシュボードで **設定** に移動し、**Analytics** セクションまでスクロールします。
</Step>

<Step title="Enable Your Tracking Platform">
  使用する分析プラットフォームをトグルオンし、トラッキング ID を入力します:
  - **Google Analytics**: 計測 ID を入力します（形式: `G-XXXXXXXXXX`）
  - **Google Tag Manager**: コンテナ ID を入力します（形式: `GTM-XXXXXXX`）
  - **Meta Pixel**: ピクセル ID（数字、10〜20 桁）を入力します
</Step>

<Step title="Save Your Settings">
  **保存** をクリックしてトラッキング設定を反映させます。
</Step>

<Step title="Verify Tracking">
  ストアフロントを表示するかテストチェックアウトを作成し、分析ダッシュボードでイベントが追跡されていることを確認します。
</Step>
</Steps>

## Google Analytics 4

[Google Analytics 4](https://marketingplatform.google.com/about/analytics/) は e コマース追跡機能を備えた包括的な Web 分析を提供します。Dodo Payments との統合により、ストアの閲覧から購入完了まで、顧客の一連の行動を追跡できます。

### Google Analytics の計測 ID を追加

1. Google Analytics で **管理** → **データ ストリーム** → ウェブ ストリームを選択
2. **計測 ID** をコピーする（`G-` で始まります）
3. Dodo Payments ダッシュボードで **設定** → **Analytics** に移動
4. **Google Analytics** をトグルオンし、計測 ID を貼り付け
5. **保存** をクリック

<Tip>
まだ Google Analytics プロパティがない場合は、[Google のセットアップ ガイド](https://support.google.com/analytics/answer/9304153) に従って作成してください。
</Tip>

### Google Analytics で追跡されるイベント

ホストされたストアフロントとチェックアウトページでは以下のイベントを追跡します。イベントごとに商品情報、取引総額、通貨などのデータを送信します。

#### ストアフロントのイベント

| イベント | 説明 |
|-------|-------------|
| `view_item_list` | 商品を含むストアフロントページの読み込み時に発火 |
| `select_item` | 顧客が商品をクリックしてチェックアウトを開始したときに発火 |

#### チェックアウトのイベント

| イベント | 説明 |
|-------|-------------|
| `view_item` | チェックアウトページの読み込み時に発火 |
| `view_cart` | チェックアウトの読み込み時およびバリアント選択時に発火 |
| `apply_discount`* | 有効な割引コードが正常に適用されたときに発火 |
| `remove_discount`* | 割引コードが削除されたときに発火 |
| `add_payment_info` | 顧客が支払いフォームを送信したときに発火 |
| `purchase` | 支払いが成功したときに発火 |

<Note>
`apply_discount` および `remove_discount` は [Google Analytics の標準 e コマースイベント](https://support.google.com/analytics/answer/9267735) には含まれていないカスタムイベントです。
</Note>

### オーバーレイ／インラインチェックアウトでの GA4 利用

自身のウェブサイトに埋め込まれたオーバーレイまたはインラインチェックアウトで Google Analytics イベントを追跡するには、埋め込み先のウェブサイトに Google Analytics トラッキングコードを導入する必要があります。

導入後、上記のすべての追跡イベントが追加設定なしで自動的に Google Analytics アカウントに送信されます。

## Google Tag Manager

[Google Tag Manager](https://marketingplatform.google.com/about/tag-manager/) を使うと、コード変更なしでマーケティングや分析タグを管理できます。GTM を通じて配置されたタグは、すべての Dodo Payments ホストページで発火します。

### GTM コンテナ ID を追加

1. Google Tag Manager で **コンテナ ID** をコピー（`GTM-` で始まります）
2. Dodo Payments ダッシュボードで **設定** → **Analytics** に移動
3. **Google Tag Manager** をトグルオンし、コンテナ ID を貼り付け
4. **保存** をクリック

<Info>
GTM を使用する際は、Google 広告のコンバージョンなど追加のトラッキングを GTM コンテナ内で直接設定できます。Dodo Payments 統合では、GTM スクリプトと noscript フォールバックの双方を注入し、最大互換性を確保します。
</Info>

### オーバーレイ／インラインチェックアウトでの GTM 利用

ウェブサイトに埋め込まれたオーバーレイまたはインラインチェックアウトでは、サイト側に GTM を導入してください。チェックアウトは GTM 設定を継承し、埋め込みフレーム内でイベントを発火します。

## Meta Pixel

[Meta Pixel](https://www.facebook.com/business/tools/meta-pixel/) は、Facebook および Instagram 広告の効果を測定するためにストアでの顧客行動を追跡します。広告クリックから購入完了までのコンバージョンをトラッキングできます。

### Meta Pixel ID を追加

1. [Meta Events Manager](https://www.facebook.com/events_manager2/) でピクセルを作成または選択
2. **Pixel ID** をコピー（数字、通常は 15〜16 桁）
3. Dodo Payments ダッシュボードで **設定** → **Analytics** に移動
4. **Meta Pixel** をトグルオンし、Pixel ID を貼り付け
5. **保存** をクリック

<Tip>
まだピクセルを作成していない場合は、Meta の [Pixel セットアップガイド](https://www.facebook.com/business/help/952192354843755) に従ってください。
</Tip>

### Meta Pixel で追跡されるイベント

以下のイベントは [Meta Events Manager](https://www.facebook.com/events_manager2/) に表示されます。コンテンツ ID、値、通貨などのイベントデータを送信します。

<Note>
イベントが Meta Events Manager に表示されるまで数分の遅延が発生することがあります。
</Note>

#### ストアフロントのイベント

| イベント | 説明 |
|-------|-------------|
| `PageView` | ストアフロントページの読み込み時に発火 |
| `ViewContent` | 商品を表示してストアフロントが読み込まれたときに発火 |
| `InitiateCheckout` | 顧客が商品をクリックしてチェックアウトを開始したときに発火 |

#### チェックアウトのイベント

| イベント | 説明 |
|-------|-------------|
| `PageView` | チェックアウトページの読み込み時に発火 |
| `AddToCart` | チェックアウトの読み込み時およびバリアント選択時に発火 |
| `ApplyDiscount`* | 有効な割引コードが正常に適用されたときに発火 |
| `RemoveDiscount`* | 割引コードが削除されたときに発火 |
| `AddPaymentInfo` | 顧客が支払いフォームを送信したときに発火 |
| `Purchase` | 支払いが成功したときに発火 |

<Note>
`ApplyDiscount` および `RemoveDiscount` は [Meta の標準イベント](https://developers.facebook.com/docs/meta-pixel/reference#standard-events)には含まれていないカスタムイベントです。
</Note>

## イベント完全参照

この表は Google Analytics 4 と Meta Pixel の両方で追跡されるすべてのイベントを示します:

| トリガー | GA4 イベント | Meta Pixel イベント | 場所 |
|---------|-----------|------------------|----------|
| ストアフロントページの読み込み | `view_item_list` | `ViewContent` | ストアフロント |
| 顧客が商品をクリック | `select_item` | `InitiateCheckout` | ストアフロント |
| チェックアウトページの読み込み | `view_item` | `AddToCart` | チェックアウト |
| カート閲覧／バリアント選択 | `view_cart` | - | チェックアウト |
| 割引コード適用 | `apply_discount` | `ApplyDiscount` | チェックアウト |
| 割引コード削除 | `remove_discount` | `RemoveDiscount` | チェックアウト |
| 支払いフォーム送信 | `add_payment_info` | `AddPaymentInfo` | チェックアウト |
| 支払い成功 | `purchase` | `Purchase` | チェックアウト |

## プラットフォームのカバレッジ

解析トラッキングは、さまざまな Dodo Payments のインターフェースで利用できます:

| インターフェース | Google Analytics 4 | Google Tag Manager | Meta Pixel |
|---------|-------------------|-------------------|------------|
| ストアフロント | はい | はい | はい |
| チェックアウト（ホスト） | はい | はい | はい |
| チェックアウト（オーバーレイ／インライン）* | はい | はい | はい |
| カスタマーポータル | はい | はい | いいえ |

<Info>
*オーバーレイおよびインラインチェックアウトでは、完全なアトリビューションのために自サイトにトラッキングコードを導入してください。埋め込まれたチェックアウトからのイベントは自動的に追跡されます。
</Info>

## 含まれるイベントデータ

各トラッキングイベントには、顧客行動分析に役立つ関連する e コマースデータが含まれます:

### 商品データ
- 商品 ID
- 商品名
- 価格
- 通貨
- 数量

### 取引データ（購入イベント）
- 取引／支払い ID
- 合計金額
- 税金額
- 通貨
- 適用された割引

### 顧客データ
- 顧客 ID（リピーター向け）
- メール（該当する場合はプライバシー保護のためハッシュ化）

## ベストプラクティス

<AccordionGroup>
<Accordion title="Test Your Integration">
トラッキング ID を追加した後、ストアフロントにアクセスしてテストチェックアウトを完了し、分析ダッシュボードにイベントが表示されることを確認してください。Google Analytics のリアルタイム レポートや Meta Events Manager のテストイベント機能を活用してください。
</Accordion>

<Accordion title="Use UTM Parameters">
マーケティングリンクに UTM パラメータを追加して、どのキャンペーンがコンバージョンを促進しているかを追跡します。Dodo Payments はチェックアウト全体を通じて UTM パラメータを保持します。
</Accordion>

<Accordion title="Set Up Conversion Goals">
Google Analytics では `purchase` のコンバージョンイベントを作成し、コンバージョン率を追跡します。Meta 広告マネージャーでは `Purchase` イベントをキャンペーン最適化ゴールとして使用してください。
</Accordion>

<Accordion title="Enable Enhanced E-commerce">
Google Tag Manager を使用している場合は、詳細なファネル分析および商品パフォーマンス指標のために Enhanced E-commerce レポートの設定を検討してください。
</Accordion>
</AccordionGroup>

## トラブルシューティング

<AccordionGroup>
<Accordion title="Events not appearing in Google Analytics">
- 計測 ID が正しいことを確認（形式: `G-XXXXXXXXXX`）
- Google Analytics のリアルタイムレポートを確認 — イベントはそこに最初に表示されます
- Dodo Payments 設定でトグルが有効になっていることを確認
- 数分待つ（処理に遅延が生じる場合があります）
- 広告ブロッカーの影響を排除するためにシークレットウィンドウでテスト
</Accordion>

<Accordion title="Events not appearing in Meta Events Manager">
- Pixel ID が正しいことを確認（数字、10〜20 桁）
- Meta Pixel Helper ブラウザ拡張機能でデバッグ
- Events Manager の Test Events タブを確認
- イベントが表示されるまで数分かかる場合があります
- テスト中は広告ブロッカーを無効化
</Accordion>

<Accordion title="GTM tags not firing">
- コンテナ ID が正しいことを確認（形式: `GTM-XXXXXXX`）
- GTM プレビュー モードでタグの発火をデバッグ
- トリガーが GTM コンテナ内で正しく設定されているか確認
- コンテナが公開されているか確認（ドラフトではない）
</Accordion>

<Accordion title="Overlay/Inline checkout not tracking">
- まず自サイトにトラッキングコードを導入
- チェックアウトオーバーレイが開く前にトラッキングが読み込まれることを確認
- ブラウザコンソールに JavaScript エラーがないか確認
- GA4 使用時はクロスドメイントラッキングの設定を確認
</Accordion>

<Accordion title="Purchase events showing wrong amounts">
- 金額は最小通貨単位（セント）で送信されます
- アナリティクスツールが正しい通貨を表示するよう設定されているか確認
- 税金が含まれている／除外されているかを確認
</Accordion>
</AccordionGroup>
