## 一回限りの支払いとは？

一回限りの支払いは、固定の前払い購入であり、以下に最適です：

- **ソフトウェアライセンス**：ライフタイムまたはメジャーバージョンのアンロック
- **デジタルダウンロード**：電子書籍、テンプレート、グラフィックス、またはプラグイン
- **サービス**：監査、オンボーディングパッケージ、または単発コンサルティング
- **コースとチケット**：コース、ワークショップ、またはライブセッション

## 主な利点

- **シンプルなチェックアウト**：更新や按分ロジックはなし
- **即時の履行**：ライセンスやファイルをすぐに提供
- **柔軟な価格設定**：割引や税金に対応
- **開発者ファースト**：作成、返金、Webhook用の明確なAPI

## 一回限りの製品の作成

Dodo Paymentsダッシュボードで製品を作成し、ホストされたチェックアウト、支払いリンク、またはAPIを介して販売します。

### 製品設定

ダッシュボードでフィールドを設定して、製品の表示方法と販売方法を定義します。

#### 製品詳細

- **製品名**（必須）：チェックアウトおよび請求書に表示される名前
- **製品説明**（必須）：顧客が支払う前に見る明確な価値声明
- **製品画像**（必須）：PNG/JPG/WebPで最大3MB
- **ブランド**：テーマやメール用に特定のブランドに製品を関連付けます
- **税カテゴリ**（必須）：税ルールを決定するために正しいカテゴリを選択します

<Tip>
正確な税カテゴリを選択して、地域ごとの正しい税金の徴収を確保してください。
</Tip>

#### 価格設定

- **価格タイプ**：<b>一括支払い</b>を選択
- **価格**（必須）：通貨付きの固定料金
- **適用可能な割引（%）**：チェックアウトおよび請求書に表示されるオプションのパーセント割引

<Warning>
製品の価格変更は新しい購入にのみ影響します。既存の顧客には影響しません。
</Warning>

#### 高度な設定

- **税金込みの価格設定**：適用される税金を含む価格を表示
- **ライセンスキーの生成**：購入後にユニークなキーを発行します。詳細は<a href="/features/license-keys">ライセンスキー</a>を参照してください。
- **デジタル製品配信**：ファイルやコンテンツを自動的に配信します。詳細は<a href="/features/digital-product-delivery">デジタル製品配信</a>を参照してください。
- **メタデータ**：内部タグ付けや統合のためにカスタムキー–バリューのペアを添付します。詳細は<a href="/api-reference/metadata">メタデータ</a>を参照してください。

<Tip>
メタデータを使用して、システムからの識別子（例：orderId）を保存し、後でイベントや領収書を照合します。
</Tip>

## 一般的なユースケース

- **ライセンスベースのツール**：一回限りのキーで機能をアンロック
- **デジタル資産**：テーマ、テンプレート、アイコンセット、またはメディア
- **プロフェッショナルサービス**：固定料金の実装またはセットアップ
- **教育**：コースチケットや単発ワークショップ

## 統合例

### チェックアウトセッション（一括支払い）
カートに一回限りの製品を含むチェックアウトセッションを作成します：

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    { product_id: 'prod_one_time', quantity: 1 }
  ]
});
```

## ベストプラクティス

- **提供物を明確にする**：購入者に正確に何を得られるか、どのように提供されるかを伝えます
- **即時に提供する**：ライセンスキーやデジタル配信を使用して即時アクセスを提供します
- **メタデータで追跡する**：簡単に照合できるようにIDを保存します

<Info>
一回限りの支払いは、固定価格の製品を販売する最も迅速な方法です。シンプルに始め、配信を確認し、コンバージョンやフィードバックに基づいてオファーを改善してください。
</Info>
