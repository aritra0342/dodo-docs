<Accordion title="Q1: サブスクリプション製品の payment.succeeded webhook で製品 ID を受け取れなかったのはなぜですか？">
  A: サブスクリプション製品の payment.succeeded webhook イベントには product_id は含まれていません。代わりに、サブスクリプションに関連付けられた `subscription_id` とその他の関連情報が提供されます。
</Accordion>

<Accordion title="Q2: 受信した webhook が本当に Dodo Payments からのものであり、悪意のあるアクターからのものでないことをどのように確認しますか？">
  A: 受信した webhook が本当に Dodo Payments からのものであり、悪意のあるアクターからのものでないことを確認するには、以下の手順に従ってください：
    1. **署名を確認する**: Dodo Payments によって送信される各 webhook には、**`webhook-signature`** ヘッダーに一意の署名が含まれています。この署名は、あなたの webhook の秘密鍵とイベントのペイロードを使用して生成されます。
    2. **秘密鍵を取得する**: あなたの webhook の秘密鍵を確認してください。これは Dodo Payments ダッシュボードの Webhooks 設定で見つけることができます。
    3. **署名を再計算する**: Dodo Payments と同じ署名アルゴリズム (HMAC-SHA256) を使用して、あなたのサーバーで署名を計算します。Webhook のペイロードとあなたの webhook 秘密鍵を使用します。
    4. **署名を比較する**: 再計算した署名を **`webhook-signature`** ヘッダーに提供された署名と比較します。一致すれば、webhook は正当なものであり、一致しなければ拒否する必要があります。
    5. **HTTPS を使用する**: あなたの webhook エンドポイントが HTTPS を使用して通信を保護し、データ送信中の改ざんを防ぐことを確認してください。
  これらの手順を実施することで、Dodo Payments から送信された webhook の信頼性を自信を持って確認できます。
</Accordion>

<Accordion title="Q3: なぜ最初の月の支払いでも subscription.renew webhook を受信しているのですか？">
  A: サブスクリプションの場合、`subscription.renew` はサブスクリプション金額が差し引かれるたびにトリガーされます。したがって、これは予期される動作です。
</Accordion>

<Accordion title="Q5: ローカルで webhook をテストするにはどうすればよいですか？">
  A: ローカルで webhook をテストするには、以下の手順に従ってください：
  
1. **トンネリングツールを使用する**: [ngrok](https://ngrok.com/)、[LocalTunnel](https://localtunnel.github.io/www/)、または [Cloudflare Tunnel](https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/) のようなツールを使用して、ローカルサーバーをインターネットに公開します。これにより、Dodo Payments がローカル環境に webhook イベントを送信するために使用できるパブリック URL が提供されます。
2. **Webhook エンドポイントを設定する**: 受信した webhook イベントを処理するために、ローカルアプリケーションに webhook エンドポイントを作成します。HTTP POST リクエストをリッスンすることを確認してください。
3. **Dodo Payments で webhook を設定する**: Dodo Payments ダッシュボードに移動し、Webhooks 設定に進み、トンネリングツールによって生成されたパブリック URL に webhook URL を更新します。
4. **イベントをトリガーする**: 環境内でアクションを実行して webhook イベントをトリガーします。たとえば、支払いまたはサブスクリプションの更新をシミュレートできます。
5. **受信リクエストを検査する**: トンネリングツールのログまたはアプリケーションのロギングを使用して、受信した webhook ペイロードを検査します。これにより、エンドポイントがイベントを受信し、正しく処理していることを確認できます。
6. **署名を確認する**: **`webhook-signature`** ヘッダーを検証して、webhook イベントが正当であり、Dodo Payments によって送信されたものであることを確認します。

これらの手順に従うことで、ローカル開発環境で webhook を簡単にテストおよびデバッグできます。
</Accordion>
