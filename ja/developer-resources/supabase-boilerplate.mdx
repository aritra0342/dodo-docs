## 概要

Next.js 15、React 19、Supabase、Drizzle ORM、および Dodo Payments を使用した、プロダクション対応のサブスクリプションボイラープレートです。Google OAuth、サブスクリプションチェックアウト、Webhook ハンドリング、データベーススキーマ、および基本的なダッシュボードが付属しています。

<Info>
既存アプリのルートハンドラのみが必要な場合は、専用のアダプターをご参照ください: <a href="/developer-resources/nextjs-adaptor">Next.js Adaptor</a> および <a href="/developer-resources/express-adaptor">Express Adaptor</a>。
</Info>

## 前提条件

- Node.js 18+ (または Bun 1.0+)
- Supabase プロジェクト (URL、Anon キー、サービスロールキー、データベース URL)
- Dodo Payments アカウント (API キー、Webhook シークレット)
- Google Cloud OAuth クライアント (クライアント ID とシークレット)

## クイックスタート

<Steps>
{/* LOCKED_PATTERN_587a3a4c926aec89f48dfc1e7c21054e */}

```bash
git clone https://github.com/dodopayments/dodo-supabase-subscription-starter.git
cd dodo-supabase-subscription-starter
# choose one
bun install
# or
npm install
# or
pnpm install
```

</Step>

{/* LOCKED_PATTERN_ea54c61e84e28cd275c0eed6d7634e0f */}
Supabase プロジェクトを作成し、次の値をコピーします:
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
- SUPABASE_SERVICE_ROLE_KEY
- DATABASE_URL（接続文字列）
</Step>

{/* LOCKED_PATTERN_42bf2f0f0010fd47b77c88325d34313c */}
Google Cloud でリダイレクト URI を `https://[your-project-ref].supabase.co/auth/v1/callback` に設定し、Supabase Auth で Client ID と Secret を使って Google プロバイダーを有効にします。
</Step>

{/* LOCKED_PATTERN_f32b65d8a917a558befa771bede82115 */}
Dodo ダッシュボードから API キーと Webhook シークレットを生成します。開発中は環境を `test_mode` に設定してください。
</Step>

{/* LOCKED_PATTERN_a34a8e162789e5a79d108d9eb3c86df5 */}

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role

# Database
DATABASE_URL=postgresql://postgres:[password]@db.[project-ref].supabase.co:5432/postgres

# Dodo Payments
DODO_PAYMENTS_API_KEY=your-dodo-api-key
DODO_WEBHOOK_SECRET=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT=test_mode
```

<Warning>
秘密情報をコミットしないでください。デプロイ環境では環境変数を使用します。
</Warning>
</Step>

{/* LOCKED_PATTERN_c35fe33eae6aa68b4bc39bd740632964 */}

```bash
bun run db:push
# or
npm run db:push
# or
pnpm run db:push
```

<Check>
作成されるテーブル: `users`、`subscriptions`、`payments`。
</Check>
</Step>

{/* LOCKED_PATTERN_560d1d81fad22bd0622f0c8dcb3c706d */}

```bash
# login (one-time)
bunx supabase login
# or
npx supabase login

# deploy the edge function
bun run deploy:webhook --project-ref [your-project-ref]
# or
npm run deploy:webhook -- --project-ref [your-project-ref]
# or
pnpm run deploy:webhook --project-ref [your-project-ref]
```

```bash cURL
curl -X POST \
  'https://[your-project-ref].supabase.co/functions/v1/dodo-webhook' \
  -H 'Content-Type: application/json' \
  -H 'Dodo-Signature: <signature>' \
  -d '{"type":"payment.succeeded","data":{}}'
```

</Step>

{/* LOCKED_PATTERN_dd5db8f3734010c91f70fdf794c4e2a2 */}
エンドポイント URL を次に設定します:

```text
https://[your-project-ref].supabase.co/functions/v1/dodo-webhook
```

支払いおよびサブスクリプションイベントを選択します。
</Step>

{/* LOCKED_PATTERN_43dfdbde6ac180daa9987aee2baf2da7 */}
Dodo ダッシュボード → Products → Create Product で、メタデータを必要に応じて追加します:
{/* LOCKED_PATTERN_27a7507b91309aba */}

```json
{
  "features": ["Feature 1", "Feature 2", "Feature 3"]
}
```

価格表示 UI はこの `features` 配列を読み取り、動的にレンダリングします。
</Step>

{/* LOCKED_PATTERN_e437399605995e5a4845b4d749b71d01 */}

```bash
bun run dev
# or
npm run dev
# or
pnpm run dev
```

http://localhost:3000 を開きます。
</Step>
</Steps>

## 含まれているもの

- Supabase 経由の認証（Google OAuth 設定済み）
- Dodo Payments によるサブスクリプションチェックアウト
- Webhook 用の Supabase Edge Function（`dodo-webhook`）
- Drizzle ORM スキーマとマイグレーション
- 請求書、サブスクリプションステータス、プランの機能を 갖備えたダッシュボード

<Tip>
`DODO_PAYMENTS_ENVIRONMENT` は、エンドツーエンドテストが完了するまで `test_mode` のままにしてください。
</Tip>

## 検証とトラブルシューティング

<AccordionGroup>
{/* LOCKED_PATTERN_6b286e9e2bbacd5c7684f8442203f10e */}
- `DODO_WEBHOOK_SECRET` が Dodo ダッシュボードの値と一致していることを確認してください
- 最新の `dodo-webhook` 関数をデプロイしたことを確認してください
- 関数内のヘッダー名（Dodo-Signature）が正しいことを確認してください
</Accordion>
{/* LOCKED_PATTERN_c270a2f8797d0168d608bb07bd40f056 */}
- `DATABASE_URL` の構文と Supabase のネットワーク egress を確認してください
- プロジェクト作成後、最初のプッシュまでに約 2〜3 分待ってください
</Accordion>
{/* LOCKED_PATTERN_276d2f8137bb346c423586f69d6b7cae */}
- リダイレクト URI は `https://[ref].supabase.co/auth/v1/callback` にする必要があります
- Google Cloud と Supabase Auth プロバイダーの両方で同じ値を設定してください
</Accordion>
</AccordionGroup>
