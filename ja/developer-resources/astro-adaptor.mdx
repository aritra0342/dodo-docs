## インストール

<Steps>
{/* LOCKED_PATTERN_ef7a270d213ae1ccf935bda545a932bd */}
プロジェクトルートで次のコマンドを実行してください:


```bash
npm install @dodopayments/astro
```

</Step>
{/* LOCKED_PATTERN_85e536148c3d4a355f42242d1584f3dc */}
プロジェクトルートに<code>.env</code>ファイルを作成してください:


```env expandable
DODO_PAYMENTS_API_KEY=your-api-key
DODO_PAYMENTS_WEBHOOK_KEY=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT="test_mode" or "live_mode"
DODO_PAYMENTS_RETURN_URL=https://yourdomain.com/checkout/success
```

<Warning>
バージョン管理に<code>.env</code>ファイルやシークレットを絶対にコミットしないでください。
</Warning>

## 顧客ポータルルートハンドラー

顧客ポータルルートハンドラーを使用すると、Dodo Paymentsの顧客ポータルをAstroアプリケーションにシームレスに統合できます。

### クエリパラメータ
<ParamField query="customer_id" type="string" required>
ポータルセッションの顧客ID（例: <code>?customer_id=cus_123</code>）。
</ParamField>
<ParamField query="send_email" type="boolean">
<code>true</code>に設定すると、顧客にポータルリンクを含むメールが送信されます。
</ParamField>

### クエリパラメーター
<ParamField query="customer_id" type="string" required>
ポータルセッションの顧客ID（例: <code>?customer_id=cus_123</code>）。
</ParamField>
<ParamField query="send_email" type="boolean">
trueに設定すると、顧客にポータルリンク付きメールを送信します。
</ParamField>

<Warning>
<code>customer_id</code>がない場合、400を返します。
</Warning>

- **メソッド:** POSTリクエストのみがサポートされています。他のメソッドは405を返します。
- **署名検証:** <code>webhookKey</code>を使用してWebhook署名を検証します。検証に失敗した場合は401を返します。
- **ペイロード検証:** Zodで検証されます。無効なペイロードには400を返します。
- **エラーハンドリング:**
  - 401: 無効な署名
  - 400: 無効なペイロード
  - 500: 検証中の内部エラー
- **イベントルーティング:** ペイロードタイプに基づいて適切なイベントハンドラーを呼び出します。

import { Webhooks } from "@dodopayments/astro";

export const POST = Webhooks({
  webhookKey: import.meta.env.DODO_PAYMENTS_WEBHOOK_KEY,
  onPayload: async (payload) => {
    // handle the payload
  },
  // ... other event handlers for granular control
});

    401: Invalid signature

    400: Invalid payload

    500: Internal error during verification

Event Routing: Calls the appropriate event handler based on the payload type.

Supported Webhook Event Handlers:
