## 概要

Model Context Protocol (MCP)は、AIアプリケーションが外部データソースやAPIに安全に接続できるオープンスタンダードです。Dodo Payments MCPサーバーは、Claude、Cursor、その他のMCP互換クライアントに対して、あなたの支払いインフラストラクチャへの構造化されたアクセスを提供します。

Dodo Payments MCP サーバーは**コードモード**アーキテクチャを使用しています。すべての API エンドポイントごとに数百の個別ツールを公開する代わりに、コードモードでは AI エージェントが隔離されたサンドボックス環境で Dodo Payments SDK に対して TypeScript コードを書き、実行できます。

### 主な機能

- **支払い操作**: 支払いと返金の作成、取得、管理
- **サブスクリプション管理**: 定期請求、アップグレード、キャンセルの処理
- **顧客管理**: 顧客データとポータルアクセスの管理
- **製品カタログ**: 製品、価格、割引の作成と更新
- **ライセンス管理**: ソフトウェアライセンスの有効化、検証、管理
- **使用量ベースの請求**: メーター使用量の追跡と請求

## Code Modeの動作

Dodo Payments MCP サーバーは AI エージェントに正確に 2 つのツールを提供します:

1. **Docs Search Tool**: Dodo Payments API および SDK のドキュメントを照会し、利用可能な操作やパラメータを理解します。
2. **Code Execution Tool**: SDK に対して TypeScript コードを書き、安全なサンドボックス環境で実行します。

このアーキテクチャにより、エージェントは単一の呼び出しで複雑なマルチステップ操作を実行できます。たとえば、アクティブなサブスクリプションを一覧表示し、特定の条件でフィルタリングし、各サブスクリプションに割引を適用する、といった処理をすべて 1 つのスクリプトで行えます。

<Tip>
コードモードを使用すると、エージェントは複数の API コールを連鎖させ、条件ロジックを処理し、LLM への複数の往復を必要とせずに計算を実行できます。
</Tip>

### クイックセットアップ

AI クライアントで Dodo Payments MCP サーバーに接続します:

<Tabs>
<Tab title="Cursor">
以下を `~/.cursor/mcp.json` に追加します:

```json
{
  "mcpServers": {
    "dodopayments": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://mcp.dodopayments.com/sse"]
    }
  }
}
```

</Tab>

<Tab title="Claude Desktop">
Claude Desktop の設定に追加します:

```json
{
  "mcpServers": {
    "dodopayments": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://mcp.dodopayments.com/sse"]
    }
  }
}
```

設定ファイルの場所:
- **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`
- **Linux**: `~/.config/Claude/claude_desktop_config.json`
</Tab>

<Tab title="Windsurf">
`~/.codeium/windsurf/mcp_config.json` に追加:

```json
{
  "mcpServers": {
    "dodopayments": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://mcp.dodopayments.com/sse"]
    }
  }
}
```

</Tab>

<Tab title="Claude Code">
ターミナルで実行:

```bash
claude mcp add dodopayments -- npx -y mcp-remote@latest https://mcp.dodopayments.com/sse
```

</Tab>
</Tabs>

<Note>
Node.js 18 以上が必要です。リモートサーバーは OAuth による認証を使用しており、初回接続時に API キーの入力と環境の選択を求められます。
</Note>

## Dodo Knowledge MCP

Dodo Payments MCP サーバー（API 操作の実行）に加えて、**Dodo Knowledge MCP** も提供しています。これは Dodo Payments のドキュメントとナレッジベースへの高速アクセスを AI アシスタントに提供するセマンティック検索サーバーです。

<Info>
**[ContextMCP.ai](https://contextmcp.ai) によって構築**: Dodo Knowledge MCP は ContextMCP により強化されており、ベクトル埋め込みを使用してドキュメント全体の高速なセマンティック検索を可能にします。
</Info>

### Dodo Knowledge MCP とは何ですか?

Dodo Knowledge MCP は次を提供するリモート MCP サーバーです:

- **セマンティックなドキュメント検索**: 自然言語クエリで関連ドキュメントを検索します。
- **文脈に即した回答**: AI アシスタントは最新の正確な Dodo Payments 情報を得られます。
- **ゼロセットアップ**: API キーやローカルインストールは不要です — 接続してすぐにクエリを開始できます。

### 両 MCP サーバーを併用する

最高の AI 支援型開発体験のため、両 MCP サーバーの併用を推奨します:

| Server | Purpose | Use Case |
|--------|---------|----------|
| **Dodo Knowledge MCP** | Documentation search | 「Webhook の処理方法は?」「サポートされている支払い方法は?」 |
| **Dodo Payments MCP** | API operations | 決済の作成、サブスクリプションの管理、返金の処理 |

```json
{
  "mcpServers": {
    "dodo-knowledge": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://knowledge.dodopayments.com/mcp"]
    },
    "dodopayments": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://mcp.dodopayments.com/sse"]
    }
  }
}
```

<Tip>
両サーバーを設定すると、AI アシスタントはまずドキュメントを検索して機能の実装方法を把握し、その後実際の API 呼び出しを行うことができます — これらを 1 回の会話で完結できます。
</Tip>

### Knowledge MCP のトラブルシューティング

接続に問題がある場合:

1. **MCP 認証キャッシュのクリア**: `rm -rf ~/.mcp-auth`
2. **クライアントアプリケーションを再起動**
3. **クライアントログを確認** してエラーメッセージを確認
4. **Node.js バージョンを確認**: Node.js 18 以上が必要

### NPM パッケージ

NPM を使用してローカルで MCP サーバーをインストールおよび実行します。

<Tabs>
<Tab title="NPX (No Installation)">

```bash
# Set your environment variables
export DODO_PAYMENTS_API_KEY="dodo_test_..."
export DODO_PAYMENTS_WEBHOOK_KEY="your_webhook_key"
export DODO_PAYMENTS_ENVIRONMENT="live_mode"

# Run the latest version
npx -y dodopayments-mcp@latest
```

</Tab>

<Tab title="MCP Client Configuration">

```json
{
  "mcpServers": {
    "dodopayments_api": {
      "command": "npx",
      "args": ["-y", "dodopayments-mcp@latest"],
      "env": {
        "DODO_PAYMENTS_API_KEY": "dodo_test_...",
        "DODO_PAYMENTS_WEBHOOK_KEY": "your_webhook_key",
        "DODO_PAYMENTS_ENVIRONMENT": "live_mode"
      }
    }
  }
}
```

</Tab>
</Tabs>

<Tip>
最新バージョンを常に取得するには `@latest` を使用し、安定性を重視する場合は `@1.0.0` のように特定のバージョンを固定してください。
</Tip>

### Docker

一貫したデプロイのためにコンテナ化された環境で MCP サーバーを実行します。

<CodeGroup>

```bash Docker
# Pull the latest image
docker pull ghcr.io/dodopayments/mcp:latest

# Run the container
docker run -e DODO_PAYMENTS_API_KEY="dodo_test_..." \
  -e DODO_PAYMENTS_WEBHOOK_KEY="your_webhook_key" \
  -e DODO_PAYMENTS_ENVIRONMENT="live_mode" \
  ghcr.io/dodopayments/mcp:latest
```

```yaml Docker Compose
services:
  dodopayments-mcp:
    image: ghcr.io/dodopayments/mcp:latest
    environment:
      - DODO_PAYMENTS_API_KEY=dodo_test_...
      - DODO_PAYMENTS_WEBHOOK_KEY=your_webhook_key
      - DODO_PAYMENTS_ENVIRONMENT=live_mode
    restart: unless-stopped
```

</CodeGroup>

<Note>
Docker イメージは [GitHub Container Registry](https://github.com/orgs/dodopayments/packages/container/package/mcp) で利用可能です。
</Note>

| Variable | Description | Required |

| `DODO_PAYMENTS_API_KEY` | あなたの Dodo Payments API キー | Yes |
| `DODO_PAYMENTS_WEBHOOK_KEY` | あなたの Webhook 署名キー | No |
| `DODO_PAYMENTS_ENVIRONMENT` | 本番用には `live_mode` に設定 | No |

## リモート実行

Web ベースのクライアントやエージェントワークフロー向けに MCP サーバーをリモート HTTP サーバーとしてデプロイします。

```bash
# Start as remote server
npx -y dodopayments-mcp --transport=http

# Specify a custom port
npx -y dodopayments-mcp --transport=http --port=3000
```

### リモートサーバー設定

デプロイ後、クライアントは以下のサーバー URL を使用して接続できます:

```json
{
  "mcpServers": {
    "dodopayments_api": {
      "url": "http://localhost:3000",
      "headers": {
        "Authorization": "Bearer your_api_key"
      }
    }
  }
}
```

### 認証ヘッダー

リモートサーバーは次のヘッダーによる認証を受け付けます:

| Header | Description |
|--------|-------------|
| `Authorization` | Bearer トークン認証 |
| `x-dodo-payments-api-key` | 直接 API キー ヘッダー |

## セキュリティのベストプラクティス

コードモードはサンドボックス化された環境でコードを実行し、API キーをサーバー側で注入することでセキュリティを確保します。さらに以下のベストプラクティスに従って資格情報を保護してください。

<AccordionGroup>
<Accordion title="API Key Management">
**認証情報をバージョン管理にコミットしないでください**

API キーは環境変数または安全なシークレット管理システムに保存します。

```bash
# Use environment variables
export DODO_PAYMENTS_API_KEY="dodo_test_..."

# Use a .env file (add to .gitignore)
echo "DODO_PAYMENTS_API_KEY=dodo_test_..." > .env
```

**定期的にキーをローテーションする**

定期的に新しい API キーを生成し、古いキーは Dodo Payments ダッシュボードから無効化します。

**開発にはテストキーを使用**

開発中は常にテストモードの API キーを使用し、本番データに影響を与えないようにします。
</Accordion>

<Accordion title="Access Control">
**リモートサーバーには認証を実装**

リモートデプロイの際は常に `Authorization` ヘッダーまたは `x-dodo-payments-api-key` ヘッダーによる認証を要求してください。

**API 使用状況を監視**

Dodo Payments ダッシュボードで MCP サーバーのアクティビティを追跡し、異常なパターンに対するアラートを設定します。
</Accordion>

<Accordion title="Network Security">
**リモートサーバーは HTTPS を使用**

リモート MCP サーバーは常に HTTPS エンドポイントの背後にデプロイします。

**レート制限を実装**

MCP サーバーおよび API レベルでレート制限を実装し、悪用から保護します。

**ネットワークアクセスを制限**

ファイアウォールルールを設定して MCP サーバーに接続できるクライアントを制限します。
</Accordion>
</AccordionGroup>

## なぜコードモードか

従来の MCP 実装では各 API エンドポイントを個別のツールとして公開することで「ツールの爆発」が発生しがちです。コードモードが優れている理由はいくつかあります:

### LLM はツールを呼び出すよりコードを書く方が得意
LLM は実世界のコードの何百万行にも及ぶ学習を通じて自然にスクリプトを書く能力を身につけています。対してツール呼び出しは合成的な例に基づくことが多いです。

<Info>
「LLM にツール呼び出しでタスクを実行させるのは、シェイクスピアに 1 か月間中国語の授業を受けさせてからその中国語で戯曲を書かせるようなものだ。」 — [Cloudflare](https://blog.cloudflare.com/code-mode/)
</Info>

### コンテクストウィンドウの膨張を排除
従来のアプローチでは、会話開始前にツール定義ごとにトークンが消費されます。50 以上のツールを公開すると 55K〜100K+ トークンが容易に消費されます。Anthropic によると、最適化前はツール定義だけで最大 134K トークンを消費していました。

コードモードではたった 2 つのツール定義が読み込まれ（約 1K トークン）、エージェントは必要に応じてドキュメントを検索します。Anthropic の Tool Search Tool はコンテクストウィンドウの 95% を維持し、77K から 8.7K トークンへとオーバーヘッドを減らしました。

### プログラム的なオーケストレーションでレイテンシを削減
従来のツール呼び出しでは操作ごとにモデル推論の往復が必要です。20 回の API 呼び出しが必要なタスクなら 20 回の往復です。コードモードではエージェントが 1 つのスクリプトを書き、すべての呼び出しを実行して最終結果のみを返します。

Anthropic はこのプログラム的手法でトークン使用量が 37% 減少し、精度が向上しました（知識取得は 25.6% から 28.5% に改善）。

### 設計上より安全
コードモードは以下のセキュリティ利点を備えています:
- **パラメータに API キーを含めない**: API キーはサーバー側で注入され、LLM に送信されるツールパラメータには一切含まれません。
- **隔離されたサンドボックス**: コードはネットワークやホストファイルシステムへのアクセスがない安全な環境で実行されます。
- **管理された SDK**: 許可された SDK メソッドのみがエージェントに提供されます。

### どんな API 規模にも対応
API が拡大しても、従来の MCP は読み込むツールが増えるためパフォーマンスが低下します。コードモードは API の表面積に関係なく常に 2 つのツールです。Cloudflare は 2,500 を超える API エンドポイントを 2 つのツールと約 1,000 トークンのコンテクストに集約できました。

<Info>
このアーキテクチャの詳細については、[Anthropic](https://www.anthropic.com/engineering/advanced-tool-use) と [Cloudflare](https://blog.cloudflare.com/code-mode/) のエンジニアリングブログ、および Claude の [Programmatic Tool Calling](https://platform.claude.com/docs/en/agents-and-tools/tool-use/programmatic-tool-calling) ドキュメントをご覧ください。
</Info>
