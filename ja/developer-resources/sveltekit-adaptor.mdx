## インストール

<Steps>
{/* LOCKED_PATTERN_ef7a270d213ae1ccf935bda545a932bd */}
プロジェクトのルートで次のコマンドを実行します:

```bash
npm install @dodopayments/svelteKit
```

</Step>
{/* LOCKED_PATTERN_85e536148c3d4a355f42242d1584f3dc */}
プロジェクトのルートに <code>.env</code> ファイルを作成します:

```env expandable
DODO_PAYMENTS_API_KEY=your-api-key
DODO_PAYMENTS_WEBHOOK_KEY=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT="test_mode" or "live_mode"
DODO_PAYMENTS_RETURN_URL=https://yourdomain.com/checkout/success
```

<Warning>
<code>.env</code> ファイルや機密情報をバージョン管理にコミットしないでください。
</Warning>
</Step>
</Steps>

## チェックアウトルートハンドラー

<Info>
Dodo Payments はウェブサイトへの支払い統合に 3 種類の支払いフローをサポートしており、このアダプターはすべての支払いフローに対応しています。
</Info>

- **静的支払いリンク:** すぐに共有可能なURLで、迅速かつコードなしで支払いを収集できます。
- **動的支払いリンク:** APIまたはSDKを使用してカスタム詳細を持つ支払いリンクをプログラムで生成します。
- **チェックアウトセッション:** 事前に設定された商品カートと顧客詳細を持つ、安全でカスタマイズ可能なチェックアウト体験を作成します。

## 顧客ポータルルートハンドラー

顧客ポータルルートハンドラーは、Dodo Paymentsの顧客ポータルをSvelteKitアプリケーションにシームレスに統合することを可能にします。

### クエリパラメーター
<ParamField query="customer_id" type="string" required>
ポータルセッションの顧客 ID (例: <code>?customer_id=cus_123</code>)。
</ParamField>
<code>true</code> に設定すると、顧客にポータルリンクを記載したメールが送信されます。
</ParamField>

<Warning>
<code>customer_id</code> が欠落している場合は 400 を返します。
</Warning>

## ウェブフックルートハンドラー

- **メソッド:** POSTリクエストのみがサポートされています。他のメソッドは405を返します。
- **署名検証:** <code>webhookKey</code>を使用してウェブフックの署名を検証します。検証に失敗した場合は401を返します。
- **ペイロード検証:** Zodで検証されます。無効なペイロードには400を返します。
- **エラーハンドリング:**
  - 401: 無効な署名
  - 400: 無効なペイロード
  - 500: 検証中の内部エラー
- **イベントルーティング:** ペイロードタイプに基づいて適切なイベントハンドラーを呼び出します。
