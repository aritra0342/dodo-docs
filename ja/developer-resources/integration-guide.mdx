## 前提条件

Dodo Payments APIを統合するには、以下が必要です：
- Dodo Paymentsのマーチャントアカウント
- ダッシュボードからのAPI認証情報（APIキーとWebhookシークレットキー）

## ダッシュボードの設定

1. [Dodo Paymentsダッシュボード](https://app.dodopayments.com/)に移動します。

2. 商品を作成します（一回限りの支払いまたはサブスクリプション）。

4. APIキーを生成します：
   - Developer > APIに移動します。
   - [詳細ガイド](/api-reference/introduction#api-key-generation)
   - 環境変数DODO_PAYMENTS_API_KEYにAPIキーをコピーします。

5. Webhookを設定します：
   - Developer > Webhooksに移動します。
   - 支払い通知用のWebhook URLを作成します。
   - 環境変数にWebhookシークレットキーをコピーします。

## 統合

### 支払いリンク

ユースケースに合った統合パスを選択してください：
- **チェックアウトセッション（推奨）**: ほとんどの統合に最適です。サーバー上でセッションを作成し、顧客を安全なホストされたチェックアウトにリダイレクトします。
- **オーバーレイチェックアウト**: チェックアウトをモーダルオーバーレイとしてサイト上で開く必要がある場合に使用します。
- **インラインチェックアウト**: ページレイアウトに直接チェックアウトを埋め込んで、完全に統合されたブランドのチェックアウト体験を提供します。
- **静的支払いリンク**: コード不要で、迅速な支払い収集のために即座に共有可能なURLです。
- **動的支払いリンク**: プログラムで作成されたリンク。ただし、チェックアウトセッションが推奨され、より柔軟性を提供します。

#### 1. チェックアウトセッション

Checkout Sessions を使用して、1回限りの支払いまたはサブスクリプション向けに安全なホスト型チェックアウト体験を作成します。サーバーでセッションを作成し、返される `checkout_url` に顧客をリダイレクトします。

<Info>
Checkout セッションはデフォルトで24時間有効です。<code>confirm=true</code> を渡すと、セッションは15分間有効で、すべての必須フィールドを入力する必要があります。
</Info>

#### 2. オーバーレイチェックアウト

シームレスなインページチェックアウト体験のために、顧客がウェブサイトを離れることなく支払いを完了できる[オーバーレイチェックアウト](/developer-resources/overlay-checkout)統合を検討してください。

#### 3. インラインチェックアウト

ページに直接埋め込まれた完全に統合されたチェックアウト体験には、[インラインチェックアウト](/developer-resources/inline-checkout)統合を使用してください。これにより、カスタム注文サマリーを構築し、Dodo Paymentsが安全に支払い収集を処理する間、チェックアウトレイアウトを完全に制御できます。

#### 4. 静的支払いリンク

静的支払いリンクを使用すると、シンプルなURLを共有することで迅速に支払いを受け付けることができます。クエリパラメータを渡して顧客の詳細を事前に入力し、フォームフィールドを制御し、カスタムメタデータを追加することで、チェックアウト体験をカスタマイズできます。

### ウェブフックの実装

支払い通知を受信するためのAPIエンドポイントを設定します。以下はNext.jsを使用した例です：

```javascript
import { Webhook } from "standardwebhooks";
import { headers } from "next/headers";
import { WebhookPayload } from "@/types/api-types";

const webhook = new Webhook(process.env.DODO_WEBHOOK_KEY!); // Replace with your secret key generated from the Dodo Payments Dashboard

export async function POST(request: Request) {
  const headersList = headers();
  const rawBody = await request.text();

  const webhookHeaders = {
    "webhook-id": headersList.get("webhook-id") || "",
    "webhook-signature": headersList.get("webhook-signature") || "",
    "webhook-timestamp": headersList.get("webhook-timestamp") || "",
  };

  await webhook.verify(rawBody, webhookHeaders);
  const payload = JSON.parse(rawBody) as WebhookPayload;
  
  // Process the payload according to your business logic
}
```

私たちのウェブフック実装は、[Standard Webhooks](https://standardwebhooks.com/)仕様に従っています。ウェブフックタイプの定義については、[ウェブフックイベントガイド](/developer-resources/webhooks/intents/webhook-events-guide)を参照してください。

Next.jsとTypeScriptを使用したデモ実装のあるこのプロジェクトを[GitHub](https://github.com/dodopayments/dodo-checkout-demo)で参照できます。
