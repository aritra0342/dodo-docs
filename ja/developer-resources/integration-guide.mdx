## 前提条件

Dodo Payments APIを統合するには、以下が必要です：
- Dodo Paymentsのマーチャントアカウント
- ダッシュボードからのAPI認証情報（APIキーとWebhookシークレットキー）

## ダッシュボードの設定

1. [Dodo Paymentsダッシュボード](https://app.dodopayments.com/)に移動します。

2. 商品を作成します（一回限りの支払いまたはサブスクリプション）。

4. APIキーを生成します：
   - Developer > APIに移動します。
   - [詳細ガイド](/api-reference/introduction#api-key-generation)
   - 環境変数DODO_PAYMENTS_API_KEYにAPIキーをコピーします。

5. Webhookを設定します：
   - Developer > Webhooksに移動します。
   - 支払い通知用のWebhook URLを作成します。
   - 環境変数にWebhookシークレットキーをコピーします。

## 統合

### 支払いリンク

ユースケースに合った統合パスを選択してください：
- **チェックアウトセッション（推奨）**: ほとんどの統合に最適です。サーバー上でセッションを作成し、顧客を安全なホストされたチェックアウトにリダイレクトします。
- **オーバーレイチェックアウト**: チェックアウトをモーダルオーバーレイとしてサイト上で開く必要がある場合に使用します。
- **インラインチェックアウト**: ページレイアウトに直接チェックアウトを埋め込んで、完全に統合されたブランドのチェックアウト体験を提供します。
- **静的支払いリンク**: コード不要で、迅速な支払い収集のために即座に共有可能なURLです。
- **動的支払いリンク**: プログラムで作成されたリンク。ただし、チェックアウトセッションが推奨され、より柔軟性を提供します。

#### 1. チェックアウトセッション

Checkout Sessionsを使用して、一度きりの支払いまたはサブスクリプションのための安全でホストされたチェックアウト体験を作成します。サーバー上でセッションを作成し、顧客を返された`checkout_url`にリダイレクトします。

<Info>
チェックアウトセッションはデフォルトで24時間有効です。<code>confirm=true</code>を渡すと、セッションは15分間有効で、すべての必須フィールドを提供する必要があります。
</Info>

<Steps>
<Step title="チェックアウトセッションを作成する">
  お好みのSDKを選択するか、REST APIを呼び出します。

  ```javascript
  });
  ```

#### 2. オーバーレイチェックアウト

シームレスなインページチェックアウト体験のために、顧客がウェブサイトを離れることなく支払いを完了できる[オーバーレイチェックアウト](/developer-resources/overlay-checkout)統合を検討してください。

#### 3. インラインチェックアウト

ページに直接埋め込まれた完全に統合されたチェックアウト体験には、[インラインチェックアウト](/developer-resources/inline-checkout)統合を使用してください。これにより、カスタム注文サマリーを構築し、Dodo Paymentsが安全に支払い収集を処理する間、チェックアウトレイアウトを完全に制御できます。

#### 4. 静的支払いリンク

静的支払いリンクを使用すると、シンプルなURLを共有することで迅速に支払いを受け付けることができます。クエリパラメータを渡して顧客の詳細を事前に入力し、フォームフィールドを制御し、カスタムメタデータを追加することで、チェックアウト体験をカスタマイズできます。

<Steps>
<Step title="支払いリンクを構築する">
  ベースURLから始めて、製品IDを追加します：

  ```text
  https://checkout.dodopayments.com/buy/{productid}
  ```

</Step>
<Step title="コアパラメータを追加する">
  必要なクエリパラメータを含めます：
  - <ParamField query="quantity" type="integer" default="1">購入するアイテムの数。</ParamField>
  - <ParamField query="redirect_url" type="string" required>支払い完了後にリダイレクトするURL。</ParamField>
  <Note>
  リダイレクトURLには、支払い詳細がクエリパラメータとして含まれます。例えば：<br/>
  <code>https://example.com/?payment_id=pay_ts2ySpzg07phGeBZqePbH&status=succeeded</code>
  </Note>
</Step>
<Step title="顧客情報を事前入力する（オプション）">
  チェックアウトをスムーズにするために、顧客または請求フィールドをクエリパラメータとして追加します。
  <AccordionGroup>
    <Accordion title="サポートされている顧客フィールド">
      - <ParamField query="fullName" type="string">顧客のフルネーム（firstNameまたはlastNameが提供されている場合は無視されます）。</ParamField>
      - <ParamField query="firstName" type="string">顧客の名。</ParamField>
      - <ParamField query="lastName" type="string">顧客の姓。</ParamField>
      - <ParamField query="email" type="string">顧客のメールアドレス。</ParamField>
      - <ParamField query="country" type="string">顧客の国。</ParamField>
      - <ParamField query="addressLine" type="string">住所。</ParamField>
      - <ParamField query="city" type="string">市。</ParamField>
      - <ParamField query="state" type="string">州または県。</ParamField>
      - <ParamField query="zipCode" type="string">郵便番号/ZIPコード。</ParamField>
      - <ParamField query="showDiscounts" type="boolean">trueまたはfalse</ParamField>
    </Accordion>
  </AccordionGroup>
</Step>
<Step title="フォームフィールドを制御する（オプション）">
  <Info>
  特定のフィールドを無効にして、顧客にとって読み取り専用にすることができます。これは、すでに顧客の詳細を持っている場合（例：ログインユーザー）に便利です。
  </Info>
  フィールドを無効にするには、その値を提供し、対応する<code>disable...</code>フラグを<code>true</code>に設定します：
  <CodeGroup>

  ```text Example
  &email=alice@example.com&disableEmail=true
  ```

  </CodeGroup>
  <Tabs>
    <Tab title="フラグ無効化テーブル">
      | フィールド       | 無効化フラグ         | 必須パラメータ |
      |---------------|---------------------|-------------------|
      | フルネーム     | `disableFullName`   | `fullName`        |
      | 名            | `disableFirstName`  | `firstName`       |
      | 姓            | `disableLastName`   | `lastName`        |
      | メール        | `disableEmail`      | `email`           |
      | 国            | `disableCountry`    | `country`         |
      | 住所行        | `disableAddressLine`| `addressLine`     |
      | 市            | `disableCity`       | `city`            |
      | 州            | `disableState`      | `state`           |
      | 郵便番号      | `disableZipCode`    | `zipCode`         |
    </Tab>
  </Tabs>
  <Tip>
  フィールドを無効化することで、誤った変更を防ぎ、データの一貫性を確保します。
  </Tip>
  <Note>
  <code>showDiscounts=false</code>を設定すると、チェックアウトフォームの割引セクションが無効化され、非表示になります。これを使用すると、顧客がチェックアウト中にクーポンやプロモコードを入力するのを防ぐことができます。
  </Note>

</Step>
<Step title="高度なコントロールを追加する（オプション）">
  - <ParamField query="paymentCurrency" type="string">支払い通貨を指定します。デフォルトは請求国の通貨です。</ParamField>
  - <ParamField query="showCurrencySelector" type="boolean" default="true">通貨セレクターを表示または非表示にします。</ParamField>
  - <ParamField query="paymentAmount" type="integer">セント単位の金額（「支払いたい金額」価格設定のみ）。</ParamField>
  - <ParamField query="metadata_*" type="string">カスタムメタデータフィールド（例：<code>metadata_orderId=123</code>）。</ParamField>
</Step>
<Step title="リンクを共有する">
  完成した支払いリンクを顧客に送信します。顧客が訪問すると、すべてのクエリパラメータが収集され、セッションIDと共に保存されます。URLは、セッションパラメータのみを含むように簡略化されます（例：<code>?session=sess_1a2b3c4d</code>）。保存された情報はページのリフレッシュを通じて持続し、チェックアウトプロセス全体でアクセス可能です。
  <Check>
  顧客のチェックアウト体験は、あなたのパラメータに基づいてスムーズでパーソナライズされています。
  </Check>
</Step>
</Steps>

#### 4. 動的支払いリンク
<Tip>
ほとんどのユースケースにはチェックアウトセッションを推奨します。これにより、より柔軟性と制御が得られます。
</Tip>

API呼び出しまたはSDKを介して顧客の詳細で作成されます。以下はその例です：

    ```javascript
export async function POST(request: NextRequest) {
  try {
    const body = await request.json();
    const { formData, cartItems } = paymentRequestSchema.parse(body);

    const response = await fetch(`${process.env.NEXT_PUBLIC_DODO_TEST_API}/payments`, {
      method: "POST",

        "Content-Type": "application/json",
        Authorization: `Bearer ${process.env.DODO_API_KEY}`, // Replace with your API secret key generated from the Dodo Payments Dashboard
      },
      body: JSON.stringify({
        billing: {
          city: formData.city,
          country: formData.country,
          state: formData.state,
          street: formData.addressLine,
          zipcode: parseInt(formData.zipCode),
        },
        customer: {
          email: formData.email,
          name: `${formData.firstName} ${formData.lastName}`,
          phone_number: formData.phoneNumber || undefined,
        },
        payment_link: true,
        product_cart: cartItems.map((id) => ({
          product_id: id,
          quantity: 1,
        })),
        return_url: process.env.NEXT_PUBLIC_RETURN_URL,
      }),
    });

    if (!response.ok) {
      const errorData = await response.json().catch(() => null);
      return NextResponse.json(
        { error: "Payment link creation failed", details: errorData },
        { status: response.status }
      );
    }

    const data = await response.json();
    return NextResponse.json({ paymentLink: data.payment_link });
  } catch (err) {
    console.error("Payment error:", err);
    return NextResponse.json(
      {
        error: err instanceof Error ? err.message : "An unknown error occurred",
      },
      { status: 500 }
    );
  }
}
```

### ウェブフックの実装

支払い通知を受信するためのAPIエンドポイントを設定します。以下はNext.jsを使用した例です：

```javascript
import { Webhook } from "standardwebhooks";
import { headers } from "next/headers";
import { WebhookPayload } from "@/types/api-types";

const webhook = new Webhook(process.env.DODO_WEBHOOK_KEY!); // Replace with your secret key generated from the Dodo Payments Dashboard

export async function POST(request: Request) {
  const headersList = headers();
  const rawBody = await request.text();

  const webhookHeaders = {
    "webhook-id": headersList.get("webhook-id") || "",
    "webhook-signature": headersList.get("webhook-signature") || "",
    "webhook-timestamp": headersList.get("webhook-timestamp") || "",
  };

  await webhook.verify(rawBody, webhookHeaders);
  const payload = JSON.parse(rawBody) as WebhookPayload;
  
  // Process the payload according to your business logic
}
```

私たちのウェブフック実装は、[Standard Webhooks](https://standardwebhooks.com/)仕様に従っています。ウェブフックタイプの定義については、[ウェブフックイベントガイド](/developer-resources/webhooks/intents/webhook-events-guide)を参照してください。

Next.jsとTypeScriptを使用したデモ実装のあるこのプロジェクトを[GitHub](https://github.com/dodopayments/dodo-checkout-demo)で参照できます。

ライブ実装は[こちら](https://atlas.dodopayments.com/)で確認できます。
