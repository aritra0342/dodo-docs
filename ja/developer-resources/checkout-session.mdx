## 最初のチェックアウトセッションを作成する

<Tabs>
<Tab title="Node.js SDK">

```javascript expandable
import DodoPayments from 'dodopayments';

// Initialize the Dodo Payments client
const client = new DodoPayments({
  bearerToken: process.env.DODO_PAYMENTS_API_KEY,
  environment: 'test_mode', // defaults to 'live_mode'
});

async function createCheckoutSession() {
  try {
    const session = await client.checkoutSessions.create({
      // Products to sell - use IDs from your Dodo Payments dashboard
      product_cart: [
        {
          product_id: 'prod_123', // Replace with your actual product ID
          quantity: 1
        }
      ],
      
      // Pre-fill customer information to reduce friction
      customer: {
        email: 'customer@example.com',
        name: 'John Doe',
        phone_number: '+1234567890'
      },
      
      // Billing address for tax calculation and compliance
      billing_address: {
        street: '123 Main St',
        city: 'San Francisco',
        state: 'CA',
        country: 'US', // Required: ISO 3166-1 alpha-2 country code
        zipcode: '94102'
      },
      
      // Where to redirect after successful payment
      return_url: 'https://yoursite.com/checkout/success',
      
      // Custom data for your internal tracking
      metadata: {
        order_id: 'order_123',
        source: 'web_app'
      }
    });

    // Redirect your customer to this URL to complete payment
    console.log('Checkout URL:', session.checkout_url);
    console.log('Session ID:', session.session_id);
    
    return session;
    
  } catch (error) {
    console.error('Failed to create checkout session:', error);
    throw error;
  }
}

// Example usage in an Express.js route
app.post('/create-checkout', async (req, res) => {
  try {
    const session = await createCheckoutSession();
    res.json({ checkout_url: session.checkout_url });
  } catch (error) {
    res.status(500).json({ error: 'Failed to create checkout session' });
  }
});
```

```python expandable
import os
from dodopayments import DodoPayments

            # Billing address for tax calculation and compliance
            billing_address={
                "street": "123 Main St",
                "city": "San Francisco", 
                "state": "CA",
                "country": "US",  # Required: ISO 3166-1 alpha-2 country code
                "zipcode": "94102"

            # Where to redirect after successful payment
            return_url="https://yoursite.com/checkout/success",
            
            # Custom data for your internal tracking

        # Redirect your customer to this URL to complete payment

# Example usage in a Flask route
```

</Step>

```javascript
// Redirect immediately
window.location.href = session.checkout_url;

// Or open in new window
window.open(session.checkout_url, '_blank');
```

<Tip>
**代替統合オプション**: リダイレクトの代わりに、[オーバーレイチェックアウト](/developer-resources/overlay-checkout)（モーダルオーバーレイ）または[インラインチェックアウト](/developer-resources/inline-checkout)（完全に埋め込まれた）を使用して、チェックアウトを直接ページに埋め込むことができます。どちらのオプションも同じチェックアウトセッションURLを使用します。
</Tip>
</Step>

<Step title="リターンの処理">
支払い後、顧客は支払いステータスの追加クエリパラメータを持つあなたの `return_url` にリダイレクトされます。
</Step>

### 必須フィールド

<ParamField body="product_cart" type="array" required>
チェックアウトセッションに含める製品の配列。各製品は、あなたのDodo Paymentsダッシュボードからの有効な `product_id` を持っている必要があります。

<Tip>
**混合チェックアウト**: 一回限りの支払い製品とサブスクリプション製品を同じチェックアウトセッションで組み合わせることができます。これにより、サブスクリプションに対するセットアップ料金、SaaSとのハードウェアバンドルなどの強力なユースケースが可能になります。
</Tip>

<Expandable title="製品カートアイテムのプロパティ">
<ParamField body="product_id" type="string" required>
Dodo Paymentsダッシュボードからの製品の一意の識別子です。

**例:** `"prod_123abc456def"`
</ParamField>

<ParamField body="quantity" type="integer" required>
製品の数量です。

**例:** `1` 単一アイテム用、`3` 複数数量用
</ParamField>

<ParamField body="addons" type="array">
製品に添付するアドオンの配列。製品がサブスクリプションの場合のみ有効です。

<Expandable title="アドオンアイテムのプロパティ">
<ParamField body="addon_id" type="string" required />

<ParamField body="quantity" type="integer" required />
</Expandable>
</ParamField>

<ParamField body="amount" type="integer">
pay_what_you_wantが有効な場合、顧客が支払う金額です。無効な場合、このフィールドは無視されます。

**フォーマット**: 通貨の最小単位で表されます（例: USDの場合はセント）。例えば、$1.00を請求するには、`100` を渡します。
</ParamField>
</Expandable>
</ParamField>

<Tip>
**製品IDを見つける**: Dodo Paymentsダッシュボードの製品→詳細を表示で製品IDを見つけるか、[製品リストAPI](/api-reference/products/get-products)を使用してください。
</Tip>

### オプションフィールド

これらのフィールドを設定して、チェックアウト体験をカスタマイズし、支払いフローにビジネスロジックを追加します。

<Accordion title="顧客情報">

<ParamField body="customer" type="object">

<ParamField body="customer_id" type="string" required>
既存の顧客の一意の識別子です。これを使用して、チェックアウトセッションを既存の顧客に添付します。

<Tab title="新しい顧客">

<Expandable title="新しい顧客のプロパティ">
<ParamField body="email" type="string" required>
顧客のメールアドレス。新しい顧客を作成する際に必要です。
</ParamField>

<ParamField body="name" type="string">
領収書や請求書に表示される顧客のフルネームです。

**例**: `"John Doe"` または `"Jane Smith"`
</ParamField>

<ParamField body="phone_number" type="string">
国際フォーマットの顧客の電話番号。いくつかの支払い方法や詐欺防止のために必要です。

**フォーマット**: 国コードを含めてください。例: `"+1234567890"` は米国の番号用
</ParamField>
</Expandable>
</Tab>
</Tabs>

<ParamField body="billing_address" type="object">
正確な税計算、詐欺防止、規制遵守のための請求先住所情報です。

<Info>
`confirm` が `true` に設定されている場合、すべての請求先住所フィールドが成功したセッション作成のために必須になります。
</Info>

<Expandable title="請求先住所オブジェクトのプロパティ">
<ParamField body="street" type="string">
家番号、通り名、アパート/ユニット番号を含む完全な住所です。

**例**: `"123 Main St, Apt 4B"` または `"456 Oak Avenue"`
</ParamField>

<ParamField body="city" type="string">
市または自治体名です。

**例**: `"San Francisco"`, `"New York"`, `"London"`
</ParamField>

<ParamField body="state" type="string">
州、県、または地域名。フルネームまたは標準の略称を使用します。

**例**: `"California"` または `"CA"`, `"Ontario"` または `"ON"`
</ParamField>

<ParamField body="country" type="string" required>
2文字のISO国コード（ISO 3166-1 alpha-2）。このフィールドは、billing_addressが提供される場合は常に必要です。

**例**: `"US"` (アメリカ合衆国), `"CA"` (カナダ), `"GB"` (イギリス), `"DE"` (ドイツ)

<Card title="国コードリファレンス" icon="globe" href="/api-reference/misc/supported-countries">
  サポートされている国とそのISOコードの完全なリストを表示
</Card>
</ParamField>

<ParamField body="zipcode" type="string">
郵便番号、ZIPコード、または国の要件に基づく同等のものです。

**例**: `"94102"` (米国), `"M5V 3A8"` (カナダ), `"SW1A 1AA"` (英国)
</ParamField>
</Expandable>
</ParamField>

<Accordion title="支払い設定">

<ParamField body="allowed_payment_method_types" type="array">
チェックアウト中に顧客が利用できる支払い方法を制御します。これにより、特定の市場やビジネス要件に最適化できます。

**利用可能なオプション**: `credit`, `debit`, `upi_collect`, `upi_intent`, `apple_pay`, `google_pay`, `amazon_pay`, `klarna`, `affirm`, `afterpay_clearpay`, `sepa`, `ach`, `cashapp`, `multibanco`, `bancontact_card`, `eps`, `ideal`, `przelewy24`, `paypal`

<Warning>
**重要**: 常に `credit` と `debit` をフォールバックオプションとして含めて、好ましい支払い方法が利用できない場合のチェックアウト失敗を防いでください。
</Warning>

**例**: 

```json expandable
["apple_pay", "google_pay", "credit", "debit"]
```

</ParamField>

<ParamField body="billing_currency" type="string">
デフォルトの通貨選択を固定の請求通貨で上書きします。ISO 4217通貨コードを使用します。

**サポートされている通貨**: `USD`, `EUR`, `GBP`, `CAD`, `AUD`, `INR`, など

**例**: `"USD"` 米ドル用、`"EUR"` ユーロ用

<Note>
このフィールドは、アダプティブプライシングが有効な場合にのみ効果があります。アダプティブプライシングが無効な場合、製品のデフォルト通貨が使用されます。
</Note>
</ParamField>

<ParamField body="show_saved_payment_methods" type="boolean" default="false">
リピート顧客のために以前に保存された支払い方法を表示し、チェックアウトのスピードとユーザー体験を向上させます。
</ParamField>

<Accordion title="セッション管理">

<ParamField body="return_url" type="string">
成功した支払いまたはキャンセル後に顧客をリダイレクトするURLです。
</ParamField>

<ParamField body="confirm" type="boolean" default="false">
trueの場合、すべてのセッション詳細を即座に確定します。必要なデータが欠けている場合、APIはエラーをスローします。
</ParamField>

<ParamField body="discount_code" type="string">
チェックアウトセッションに割引コードを適用します。
</ParamField>

<ParamField body="metadata" type="object">
セッションに関する追加情報を保存するためのカスタムキーと値のペアです。
</ParamField>

<ParamField body="force_3ds" type="boolean">
このセッションのマーチャントデフォルトの3DS動作を上書きします。
</ParamField>

<ParamField body="minimal_address" type="boolean" default="false">
最小限の住所収集モードを有効にします。有効にすると、チェックアウトでは次の情報のみを収集します:
- **国**: 税金の決定に常に必要
- **郵便番号**: 売上税、VAT、またはGST計算に必要な地域のみ

これにより、不要なフォームフィールドを排除することでチェックアウトの摩擦が大幅に軽減されます。

<Tip>
チェックアウトの完了を早めるために最小限の住所を有効にしてください。完全な住所収集は、完全な請求詳細が必要なビジネスのために引き続き利用可能です。
</Tip>

<ParamField body="product_description" type="string">

<ParamField body="adaptive_currency_fees_inclusive" type="boolean">
アダプティブ通貨手数料を製品価格に含めるか（true）、または上乗せするか（false）。アダプティブプライシングが無効な場合は無視されます。

### 11. 既存の支払い方法を使用した即時チェックアウト

顧客の保存された支払い方法を使用して、支払い方法の収集をスキップし、即座に処理されるチェックアウトセッションを作成します:

```javascript expandable
const session = await client.checkoutSessions.create({
  product_cart: [
    {
      product_id: 'prod_premium_plan',
      quantity: 1
    }
  ],
  customer: {
    customer_id: 'cus_123'  // Required when using payment_method_id
  },
  payment_method_id: 'pm_abc123',  // Use customer's saved payment method
  confirm: true,  // Required when using payment_method_id
  return_url: 'https://yourapp.com/success'
});
```

<Warning>
`payment_method_id` を使用する場合、`confirm` は `true` に設定され、既存の `customer_id` が提供される必要があります。支払い方法は、支払いの通貨に対して適格性が検証されます。
</Warning>

<Info>
支払い方法は顧客に属し、支払い通貨と互換性がある必要があります。これにより、リピーターの顧客に対してワンクリック購入が可能になります。
</Info>

```javascript expandable
const session = await client.checkoutSessions.create({
```

短縮リンクはSMS、メール、またはソーシャルメディアでの共有に最適です。長いURLよりも覚えやすく、顧客の信頼を高めます。
</Tip>

### 13. 支払い成功ページをスキップして即時リダイレクト

```javascript expandable
    {
      product_id: 'prod_digital_product',
      quantity: 1
    }
  ],

  },

    email: 'customer@example.com',
    name: 'Jane Smith'

```

<Tip>
カスタム成功ページがデフォルトの支払い成功ページよりも優れたユーザーエクスペリエンスを提供する場合は、`redirect_immediately: true` を使用してください。これは特にモバイルアプリや埋め込みチェックアウトフローに役立ちます。
</Tip>

<Note>
`redirect_immediately` が有効になっている場合、顧客は支払い完了後すぐにあなたの `return_url` にリダイレクトされ、デフォルトの成功ページを完全にスキップします。
</Note>

## ダイナミックリンクからチェックアウトセッションへの移行

### 主な違い
以前は、ダイナミックリンクで支払いリンクを作成する際に、顧客の完全な請求先住所を提供する必要がありました。

チェックアウトセッションでは、これがもはや必要ありません。お持ちの情報をそのまま渡すだけで、残りは当社が処理します。例えば:

- 顧客の請求国のみがわかっている場合は、それを提供してください。
- チェックアウトフローは、顧客を支払いページに移動させる前に不足している詳細を自動的に収集します。
- 一方、すでにすべての必要な情報を持っていて、支払いページに直接スキップしたい場合は、完全なデータセットを渡し、リクエストボディに `confirm=true` を含めることができます。

### 移行プロセス

ダイナミックリンクからチェックアウトセッションへの移行は簡単です:

<Steps>
<Step title="統合を更新する">
新しいAPIまたはSDKメソッドを使用するように統合を更新します。
</Step>

<Step title="リクエストペイロードを調整する">
チェックアウトセッション形式に従ってリクエストペイロードを調整します。
</Step>

<Step title="それだけです！">
はい。あなたの側で追加の処理や特別な移行手順は必要ありません。
</Step>
</Steps>
