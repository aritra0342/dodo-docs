# 概要

Dodo Payments 用の <b>Better Auth Adapter</b> は、以下を提供します：

- サインアップ時の自動顧客作成
- 製品スラッグマッピングを使用したチェックアウトセッション（推奨）
- セルフサービス顧客ポータル
- 使用量ベースの請求のためのメーター使用量取り込みおよび報告エンドポイント
- 署名検証付きのリアルタイム Webhook イベント処理
- 完全な TypeScript サポート

<Note>
  この統合を使用するには Dodo Payments アカウントと API キーが必要です。
</Note>

# 前提条件

- Node.js 20+
- Dodo Payments ダッシュボードへのアクセス
- [better-auth](https://www.npmjs.com/package/better-auth) を使用している既存のプロジェクト

```typescript expandable
import { betterAuth } from "better-auth";
import {
  dodopayments,
  checkout,
  portal,
  webhooks,
  usage,
} from "@dodopayments/better-auth";
import DodoPayments from "dodopayments";

export const dodoPayments = new DodoPayments({
  bearerToken: process.env.DODO_PAYMENTS_API_KEY!,
  environment: "test_mode", // or "live_mode" for production
});

export const { auth, endpoints, client } = BetterAuth({
  plugins: [
    dodopayments({
      client: dodoPayments,
      createCustomerOnSignUp: true,
      use: [
        checkout({
          products: [
            {
              productId: "pdt_xxxxxxxxxxxxxxxxxxxxx",
              slug: "premium-plan",
            },
          ],
          successUrl: "/dashboard/success",
          authenticatedUsersOnly: true,
        }),
        portal(),
        webhooks({
          webhookKey: process.env.DODO_PAYMENTS_WEBHOOK_SECRET!,
          onPayload: async (payload) => {
            console.log("Received webhook:", payload.event_type);
          },
        }),
        usage(),
      ],
    }),
  ],
});
```

# 使用例

<Info>
  新しい統合では <code>authClient.dodopayments.checkoutSession</code> を優先してください。従来の <code>checkout</code> メソッドは非推奨で、後方互換性のために維持されています。
</Info>

## チェックアウトセッションの作成（推奨）

<CodeGroup>

```typescript Using a slug
const { data: session, error } = await authClient.dodopayments.checkoutSession({
  // Option A: use a configured slug
  slug: "premium-plan",

// Your internal id for reference
referenceId: "order_123",
});

if (session) {
window.location.href = session.url;
}

```

`

```typescript Using product cart
const { data: session, error } = await authClient.dodopayments.checkoutSession({
  // Option B: pass a product cart/product id directly
  product_cart: [
    {
      product_id: "pdt_xxxxxxxxxxxxxxxxxxxxx",
      quantity: 1,
    }
  ],

  // Your internal id for reference
  referenceId: "order_123",
});

if (session) {
  window.location.href = session.url;
}
```

`

</CodeGroup>

<Info>
  従来の <code>checkout</code> メソッドとは異なり、<code>checkoutSession</code> ではチェックアウトページでユーザーから請求情報が取得されるため、事前に請求情報を提供する必要はありません。必要であれば引数に <code>billing</code> キーを渡して上書きできます。
</Info>

<Info>
  従来の <code>checkout</code> メソッドと同様に、<code>checkoutSession</code> では customer のメールと名前を better-auth セッションから取得しますが、<code>customer</code> オブジェクトにメールと名前を含めて渡せば上書きできます。
</Info>

<Info>
  引数には [Create Checkout Session](https://docs.dodopayments.com/api-reference/checkout-sessions/create) エンドポイントのリクエストボディと同じオプションを渡せます。
</Info>

<Note>
  リターン URL はサーバープラグインで設定された <code>successUrl</code> から取得されます。クライアントのペイロードに <code>return_url</code> を含める必要はありません。
</Note>

## メーター使用量の追跡

サーバーで <code>usage()</code> プラグインを有効にして、メーターイベントをキャプチャし、顧客が使用量ベースの請求を確認できるようにします。

- <code>authClient.dodopayments.usage.ingest</code> は、サインインしたメール確認済みのユーザーのイベントを取り込みます。
- <code>authClient.dodopayments.usage.meters.list</code> は、その顧客のサブスクリプションに関連付けられたメーターの最近の使用量をリストします。

```typescript expandable
// Record a metered event (e.g., an API request)
const { error: ingestError } = await authClient.dodopayments.usage.ingest({
  event_id: crypto.randomUUID(),
  event_name: "api_request",
  metadata: {
    route: "/reports",
    method: "GET",
  },
  // Optional Date; defaults to now if omitted
  timestamp: new Date(),
});

if (ingestError) {
  console.error("Failed to record usage", ingestError);
}

// List recent usage for the current customer
const { data: usage, error: usageError } =
  await authClient.dodopayments.usage.meters.list({
    query: {
      page_size: 20,
      meter_id: "mtr_yourMeterId", // optional
    },
  });

if (usage?.items) {
  usage.items.forEach((event) => {
    console.log(event.event_name, event.timestamp, event.metadata);
  });
}
```

<Tip>
  使用状況を取り込む際、1 時間より古いタイムスタンプまたは 5 分以上未来のタイムスタンプは拒否されます。
</Tip>
<Tip>
  使用状況メーターを一覧表示する際に <code>meter_id</code> を省略すると、顧客のサブスクリプションに関連するすべてのメーターが返されます。
</Tip>

# 設定リファレンス

<AccordionGroup>
  {/* LOCKED_PATTERN_9586fa213a07317c452129bb9c8365ad */}
    - <b>client</b>（必須）：DodoPayments クライアントインスタンス - <b>createCustomerOnSignUp</b>（任意）：ユーザー登録時に顧客を自動作成 - <b>use</b>（必須）：有効化するプラグインの配列（checkout、portal、usage、webhooks）
  </Accordion>
  {/* LOCKED_PATTERN_a313db3f1395a5521aceb89c3251fabd */}
    - <b>products</b>：製品の配列または製品を返す非同期関数 - <b>successUrl</b>：支払い成功後にリダイレクトする URL - <b>authenticatedUsersOnly</b>：ユーザー認証を要求する（デフォルト：false）
  </Accordion>
</AccordionGroup>

# トラブルシューティングとヒント

<AccordionGroup>
  {/* LOCKED_PATTERN_8cb977e6bc7a95fb0f5c0e3c34015b67 */}
    - <b>Invalid API key</b>：<code>.env</code> 内の <code>DODO_PAYMENTS_API_KEY</code> を再確認してください。 - <b>Webhook signature mismatch</b>：Webhook シークレットが Dodo Payments ダッシュボードに設定したものと一致するか確認してください。 - <b>Customer not created</b>：<code>createCustomerOnSignUp</code> が <code>true</code> に設定されていることを確認してください。
  </Accordion>
  {/* LOCKED_PATTERN_328949dc9617c5646d2623ce7f82eb78 */}
    - すべてのシークレットとキーに環境変数を使用してください。 - <code>test_mode</code> でテストしてから <code>live_mode</code> に切り替えてください。 - デバッグと監査のために webhook イベントをログに記録してください。
  </Accordion>
</AccordionGroup>
