## はじめに

支払いイベントが発生したときに、トランザクションメールを自動的に送信し、顧客コミュニケーションを管理します。ループのメールインフラストラクチャを使用して、支払い確認、サブスクリプションの更新、および重要な通知を配信します。

<Info>
この統合には、認証のためにあなたのループAPIキーが必要です。
</Info>

## 始め方

<Steps>
<Step title="Webhookセクションを開く">
  Dodo Paymentsダッシュボードで、<b>Webhooks → + エンドポイントを追加</b>に移動し、統合のドロップダウンを展開します。
  <Frame>
    <img src="/images/integrations/loops.png" alt="エンドポイントの追加と統合のドロップダウン" />
  </Frame>
</Step>

<Step title="ループを選択">
  <b>ループ</b>統合カードを選択します。
</Step>

<Step title="APIキーを入力">
  設定にあなたのループAPIキーを提供します。
</Step>

<Step title="変換を設定">
  ループのAPI用にメールをフォーマットするために変換コードを編集します。
</Step>

<Step title="テストと作成">
  サンプルペイロードでテストし、<b>作成</b>をクリックしてメール送信を有効にします。
</Step>

<Step title="完了！">
  🎉 支払いイベントが自動的にループを介してトランザクションメールをトリガーします。
</Step>
</Steps>

## 変換コードの例

### 支払い確認メール

```javascript payment_confirmation.js icon="js" expandable
function handler(webhook) {
  if (webhook.eventType === "payment.succeeded") {
    const p = webhook.payload.data;
    webhook.url = "https://api.loops.so/v1/events/send";
    webhook.payload = {
      eventName: "payment_confirmation",
      email: p.customer.email,
      properties: {
        customer_name: p.customer.name,
        payment_id: p.payment_id,
        amount: (p.total_amount / 100).toFixed(2),
        currency: p.currency || "USD",
        payment_method: p.payment_method || "unknown",
        payment_date: new Date(webhook.payload.timestamp).toLocaleDateString()
      }
    };
  }
  return webhook;
}
```

### サブスクリプションウェルカムメール

```javascript subscription_welcome.js icon="js" expandable
function handler(webhook) {
  if (webhook.eventType === "subscription.active") {
    const s = webhook.payload.data;
    webhook.url = "https://api.loops.so/v1/events/send";
    webhook.payload = {
      eventName: "subscription_welcome",
      email: s.customer.email,
      properties: {
        customer_name: s.customer.name,
        subscription_id: s.subscription_id,
        product_id: s.product_id,
        amount: (s.recurring_pre_tax_amount / 100).toFixed(2),
        frequency: s.payment_frequency_interval,
        next_billing: s.next_billing_date
      }
    };
  }
  return webhook;
}
```

### 支払い失敗通知

```javascript payment_failure.js icon="js" expandable
function handler(webhook) {
  if (webhook.eventType === "payment.failed") {
    const p = webhook.payload.data;
    webhook.url = "https://api.loops.so/v1/events/send";
    webhook.payload = {
      eventName: "payment_failed",
      email: p.customer.email,
      properties: {
        customer_name: p.customer.name,
        payment_id: p.payment_id,
        amount: (p.total_amount / 100).toFixed(2),
        error_message: p.error_message || "Payment processing failed",
        retry_link: `https://yourdomain.com/retry-payment/${p.payment_id}`
      }
    };
  }
  return webhook;
}
```

## ヒント

- より良いメールテンプレートの整理のために、説明的なイベント名を使用する
- パーソナライズのために関連する顧客プロパティを含める
- 各イベントのためにループダッシュボードでメールテンプレートを設定する
- イベント間で一貫したプロパティ名を使用する
- 本番環境に移行する前にメール配信をテストする

## トラブルシューティング

<AccordionGroup>
<Accordion title="メールが送信されない">
- APIキーが正しくアクティブであることを確認する
- イベント名がループテンプレートと一致していることを確認する
- 受信者のメールアドレスが有効であることを確認する
- ループの送信制限とクォータを確認する
</Accordion>

<Accordion title="変換エラー">
- JSON構造がループAPIフォーマットと一致していることを検証する
- 必要なすべてのフィールドが存在することを確認する
- イベント名が正しくフォーマットされていることを確認する
- APIキーの権限を確認する
</Accordion>
</AccordionGroup>
