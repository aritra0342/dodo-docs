## 新機能 🚀

### 使用量に基づく請求
実際の消費に基づいて顧客を正確に請求します。メーターを定義し、イベントを取り込み、使用量を自動的に料金に変換します。

- **柔軟な集計**: カウント、合計（プロパティごと）、ユニーク、最大、最後
- **イベントフィルタリング**: 関連するイベントのみを含めるためのAND/ORロジックと豊富な比較子
- **製品リンク**: 単位あたりの価格とオプションの無料閾値を持つメーターを製品に添付
- **分析**: トレンド、イベントストリーム、および顧客ごとの消費を追跡

### サブスクリプションの100%割引
補完期間、スポンサーシップ、またはプロモーションキャンペーンをサポートするために、任意の数の請求サイクルに対して100%の割引を適用します。請求書は全額割引を反映し、按分はプラン設定に従います。

### 小さな改善

- Refund Receipt API は現在一般公開されています。ドキュメントを参照してください: [/api-reference/refunds/get-refund-receipt](/api-reference/refunds/get-refund-receipt)。
- Get Payment API のレスポンスと支払いウェブフックに `checkout session ID` を追加し、支払いとそれに対応するチェックアウトセッションとの関連付けを容易にしました。
