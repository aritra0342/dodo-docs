## 新機能 🚀

### 強化された分析ダッシュボード

収益と維持の洞察を包括的に提供する分析ダッシュボードを拡張し、ビジネスパフォーマンスをよりよく理解できるようにしました。

#### 収益内訳分析

詳細な内訳で定期収益指標を追跡します：

- **予測ARR**: トレンド分析を伴う年間定期収益の予測
- **MRR内訳と成長率**: 月次定期収益の分析と月ごとの成長指標  
- **新規MRRチャート**: 新規顧客の収益獲得の視覚的表現
- **拡張MRRチャート**: 既存顧客のアップグレードや拡張からの収益成長を追跡
- **解約MRRチャート**: 顧客のキャンセルやダウングレードから失われた収益を監視
- **ネット新規MRRチャート**: 新規、拡張、解約MRRを組み合わせたネット収益影響

#### 維持分析

顧客の維持と解約パターンを監視します：

- **顧客解約率**: サブスクリプションをキャンセルした顧客の割合
- **収益解約率**: キャンセルやダウングレードから失われた収益の割合
- **解約率トレンド**: 時間の経過に伴う顧客と収益の解約パターンの履歴追跡
- **ユーザー維持マトリックス**: 異なる期間にわたる顧客維持率を示すコホート分析

### レポート2.0

レポートセクションを完全に改訂し、より包括的で詳細な報告機能を提供しました。新しい報告スイートにアクセスするには、**ビジネス > レポート**に移動してください。

利用可能なレポートは次のとおりです：

- **支払いレポート**: すべての処理された支払いの詳細な記録と取引状況の追跡
- **返金レポート**: すべての返金活動の包括的なログと詳細な返金情報
- **支払いレポート**: アカウントに対して行われた支払いの概要
- **製品別売上レポート**: 収益とビジネス成長指標を伴う売上パフォーマンスの洞察
- **支払い失敗レポート**: 失敗した支払い試行の概要と失敗理由
- **顧客レポート**: 顧客ごとに整理された支払いの概要
- **紛争レポート**: 解決状況を伴う支払い紛争の概要
- **アカウント概要レポート**: アカウントの完全な概要と要約

今すぐ[https://mcp.dodopayments.com](https://mcp.dodopayments.com)でホストされたリモートMCPサーバーを使用して、即座にMCP統合を行えます—ローカルのセットアップは不要です。

<Note>
これは、Dodo PaymentsのMCP統合を開始する最も迅速な方法です。サーバーのインストールやメンテナンスは不要です。
</Note>

<Steps>
<Step title="リモートMCPサーバーを開く">
    ブラウザで<a href="https://mcp.dodopayments.com" target="_blank">https://mcp.dodopayments.com</a>にアクセスします。
</Step>
<Step title="MCP JSON構成をコピー">
    サイトで、クライアント用に提供されたMCP JSON構成をコピーします。
</Step>
<Step title="AIアプリケーションを構成">
    構成をアプリケーションのMCP設定に貼り付けます（例：CursorまたはClaude）。
    <Tip>
    Cursorの場合、構成は次のようになります：
    </Tip>

    ```json
    {
      "mcpServers": {
        "dodopayments_api": {
          "command": "npx",
          "args": ["-y", "mcp-remote@latest", "https://mcp.dodopayments.com/sse"]
        }
      }
    }
    ```

</Step>
<Step title="認証と環境設定を完了">
    JSON構成を追加した後、ポップアップが表示され、次のことを求められます：
    - Dodo Payments APIキーを入力
    - 環境を選択（テストまたは本番）
    - 設定オプションの下でMCPクライアントを選択
</Step>
<Step title="ログインと承認">
    - ログインと承認ボタンをクリック
</Step>
<Step title="接続の成功を確認">
    設定が完了すると、成功画面が表示されます。
    <Check>
    Dodo Payments MCPサーバーを使用する準備が整いました！
    </Check>
</Step>
</Steps>
