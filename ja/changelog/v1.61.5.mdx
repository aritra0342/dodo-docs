## 新機能 🚀

1. **EURおよびGBPの価格設定 - 完全展開**  
   € EURおよび£ GBPのネイティブサポートが、全世界のすべてのビジネスに完全に展開され、これらの主要通貨でのシームレスな価格設定と支払い収集が可能になりました。
   
   これにより、次のことが可能になります：
   - €および£で製品の価格を設定
   - これらの通貨でネイティブに支払いを収集
   - 為替手数料なしで€および£で直接決済
   
   <Check>
   結果：英国およびヨーロッパ全体での成功率の向上と受け入れの改善。
   </Check>

2. **支払い方法管理**  
   サブスクリプションおよび顧客向けの支払い方法管理機能が強化されました。既存のサブスクリプションの支払い方法を更新し、顧客に関連付けられたすべての支払い方法を取得できるようになり、期限切れのカード、失敗した支払い、顧客の好みの変更をより簡単に処理できます。
   
   **サブスクリプションの支払い方法を更新：**
   - 中断することなくアクティブなサブスクリプションの支払い方法を更新
   - 支払い方法を更新することで`on_hold`サブスクリプションを自動的に再アクティブ化
   - 既存の保存された支払い方法を使用するか、新しいものを追加
   - サブスクリプションを再アクティブ化する際の残りの未払い分に対する自動請求の作成
   
   **顧客の支払い方法を取得：**
   - 顧客のすべての保存された支払い方法をリスト
   - サブスクリプションを更新する際に支払い方法IDを使用
   - 利用可能な支払いオプションを表示することで顧客体験を向上
   
   詳細はこちら： [支払い方法更新API](/api-reference/subscriptions/update-payment-method) | [支払い方法リストAPI](/api-reference/customers/get-customer-payment-methods)

3. **顧客および返金オブジェクトのメタデータサポート**  
   メタデータが顧客および返金オブジェクトで利用可能になり、カスタム情報を保存し、外部システムと統合する能力が拡張されました。
   - CRM統合およびカスタム分類のために顧客にメタデータを添付
   - より良い追跡と報告のために返金に追加のコンテキストを保存
   - メタデータをサポートする他のDodo Paymentsオブジェクトとの一貫性を維持
   
   詳細はこちら： [メタデータガイド](/api-reference/metadata)

4. **顧客編集のためのチェックアウトセッション機能フラグ**  
   チェックアウトセッションの新しい機能フラグにより、顧客がチェックアウト中に編集できる内容を細かく制御できるようになり、柔軟性とユーザー体験が向上します。
   - `allow_customer_editing_name`: 顧客が名前を編集できるようにする
   - `allow_customer_editing_email`: メールアドレスの変更を許可
   - `allow_customer_editing_street`: 住所の編集を許可
   - `allow_customer_editing_city`: 市区町村フィールドの編集を有効にする
   - `allow_customer_editing_state`: 州/県の編集を許可
   - `allow_customer_editing_zipcode`: 郵便番号の編集を有効にする
   - `allow_customer_editing_country`: 国の選択変更を許可
   
   これらのフラグは、チェックアウト体験を細かく制御し、ビジネス要件に基づいて顧客が変更できるフィールドをカスタマイズすることを可能にします。

5. **チェックアウトライセンスキーおよびファイルインジケーター**  
   チェックアウトページでは、ライセンスキーまたはデジタルファイルが購入されている場合に明確なインジケーターが表示され、透明性と顧客の信頼が向上します。
   - ライセンスキー購入のための視覚的インジケーター
   - 明確なファイルダウンロードインジケーター
   - チェックアウト中の製品タイプの可視性の向上

6. **ログインページのGitHub OAuth**  
   GitHub OAuth統合による認証オプションの強化により、開発者や技術ユーザーにとって馴染みのある安全なログイン方法が提供されます。
   - GitHubアカウントでサインイン
   - 開発者アカウントのための簡素化された認証
   - チームアクセスのための追加のセキュリティレイヤー

   機能リクエストを提出し、公開ロードマップを追跡するためのより良い体験を提供するためにFeaturebaseを統合しました。
   - 機能リクエストを提出し、投票
   - 公開製品ロードマップを表示
   - 機能開発のステータスを追跡
   - コミュニティ主導の優先順位付け
   
   公開ロードマップを訪問： [https://dodopayments.featurebase.app/](https://dodopayments.featurebase.app/)
   
   詳細はこちら： [機能リクエストとロードマップ](/features/feature-request)
