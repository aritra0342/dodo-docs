## 新機能 🚀

1. **`subscription.updated` Webhook**  
   サブスクリプションオブジェクトの更新ごとに発火する新しいWebhookイベントで、サブスクリプションの変更に関するリアルタイム通知を提供します。
   
   このWebhookを使用して:
   - サブスクリプションの詳細が変更されたときに即座に通知を受け取る
   - アプリケーションの状態をDodo Paymentsと同期させる
   - APIをポーリングせずにサブスクリプションの更新に反応する
   
   <Tip>
   `subscription.updated`に登録して、ポーリングの必要を排除し、リアルタイムのサブスクリプション変更とシステムを同期させましょう。
   </Tip>
   
   詳細はこちら: [Subscription Webhooks](/developer-resources/webhooks/intents/subscription)

2. **チェックアウトにおける最小住所サポート**  

   <Frame>
   <img src="/images/changelog/minimal-address.png" alt="チェックアウトにおける最小住所サポート" />
   </Frame>

   新しい最小住所収集モードでチェックアウトの摩擦を軽減します。完全な住所詳細を収集する代わりに、最小住所モードでは次の情報のみを収集します:
   
   - **国**: 税金の決定に常に必要
   - **郵便番号**: 売上税、VAT、またはGST計算に必要な地域でのみ収集
   
   <Check>
   結果: 顧客のためのフォームフィールドが少なく、チェックアウトの完了が大幅に速くなります。
   </Check>
   
   **最小住所を有効にする:**
   
   <Tabs>
   <Tab title="チェックアウトセッション">

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [{ product_id: 'prod_123', quantity: 1 }],
     minimal_address: true
   });
   ```

   </Tab>
   
   <Tab title="静的支払いリンク">
   支払いリンクURLに`minimalAddress=true`を追加します:
   

   ```
   https://checkout.dodopayments.com/buy/prod_123?minimalAddress=true
   ```

   </Tab>
   </Tabs>
   
   <Info>
   デフォルトでは、チェックアウトは完全な住所を収集し続けます。すべてのチェックアウトリンクを最小住所モードに徐々に移行します。今すぐ機能フラグを有効にして、チェックアウトの摩擦を軽減するメリットを享受してください。
   </Info>

3. **読み取り専用APIキー**  
   より安全な統合のために、読み取り専用権限を持つAPIキーを作成します。新しいAPIキーを作成する際に、**書き込みアクセスを有効にする**のチェックを外して、読み取り専用キーを作成します。
   
   読み取り専用キーは:
   - 支払い、サブスクリプション、および顧客データを取得
   - 製品をリストし、詳細を取得
   - レポートおよび分析にアクセス
   
   読み取り専用キーは**できません**:
   - 支払いを作成または変更
   - サブスクリプションを更新
   - 製品設定を変更
   
   <Tip>
   ダッシュボード統合、分析ツール、およびデータを変更せずに表示するだけのシステムには、「書き込みアクセスを有効にする」のチェックを外してください。
   </Tip>

4. **UPI AutoPayおよびRuPayカードサブスクリプション - 完全展開**  
   UPI AutoPayおよびRuPayカードのサブスクリプションがすべてのビジネス向けに利用可能になりました。これにより、インドの顧客向けに定期的な支払いが可能になります:
   
   - **UPI AutoPay**: UPIマンダテによる自動的な定期請求
   - **RuPayカード**: インドの国内カードネットワーク向けのサブスクリプションサポート
   
   <Check>
   結果: RBI準拠のマンダテ処理を伴うインド市場向けのサブスクリプション支払いオプションが拡大しました。
   </Check>
   
   詳細はこちら: [RBI準拠のマンダテを使用したサブスクリプション](/features/subscription#subscriptions-with-rbi-compliant-mandates)

5. **プラン変更プレビューAPI**  
   サブスクリプションプランの変更を確定する前にプレビューするための新しいAPIエンドポイントです。正確な請求額を取得し、変更後のサブスクリプションがどのように見えるかを確認できます - 実際の変更を行うことなく。
   

   ```javascript
   const preview = await client.subscriptions.previewChangePlan('sub_123', {
     product_id: 'prod_pro_plan',
     proration_billing_mode: 'prorated_immediately',
     quantity: 1
   });
   
   console.log('Charge amount:', preview.immediate_charge.summary);
   console.log('New plan details:', preview.new_plan);
   ```

   
   これを使用して:
   - アップグレード/ダウングレードを確認する前に、顧客に正確に支払う金額を示す
   - 価格計算機やプラン比較ツールを構築
   - 価格の透明性を向上させることでサポートチケットを減少させる
   
   詳細はこちら: [プラン変更プレビューAPI](/api-reference/subscriptions/preview-change-plan)

6. **一般的なバグ修正と改善**  
   このリリースには、UIのポリッシュ、パフォーマンスの改善、およびより信頼性の高い、スムーズな体験のための小さなバグの修正が含まれています。
