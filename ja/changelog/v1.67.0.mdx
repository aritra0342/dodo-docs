## 新機能 🚀

1. **Mixed Cart Checkout: One-Time and Subscription Products in a Single Cart**  
   チェックアウトでは、都度払い商品とサブスクリプション商品を単一のカートで組み合わせることをサポートし、強力な新しい請求ユースケースを実現します。
   
   <Frame>
   <img src="/images/changelog/mixed-checkout.png" alt="都度払い商品とサブスクリプション商品を組み合わせた混合チェックアウト" />
   </Frame>
   
   ### これにより可能になること
   
   - **初期設定料＋サブスクリプション**：単一のチェックアウトで一度限りの初期設定料と定期的なサブスクリプションを同時に請求できます
   - **継続的な要素を含むバンドル**：継続的なサービスアクセス（サブスクリプション）と一括販売の商品（都度払い）をセットで提供します
   - **コース＋メンバーシップの組み合わせ**：コース購入と任意のコミュニティメンバーシップサブスクリプションを提供します
   - **永久アクセス＋追加サブスクリプション**：永続的な商品アクセスに任意の定期的なプレミアム機能を組み合わせます
   
   ### 仕組み
   
   都度払い商品とサブスクリプション商品を両方チェックアウトセッションに追加するだけです:
   

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [
       // One-time setup fee
       { product_id: 'prod_setup_fee', quantity: 1 },
       // Monthly subscription
       { product_id: 'prod_monthly_plan', quantity: 1 }
     ],
     customer: {
       email: 'customer@example.com',
       name: 'Jane Doe'
     },
     return_url: 'https://yourapp.com/success'
   });
   ```

   
   <Check>
   結果：都度払いの処理とサブスクリプションの開始を同じチェックアウトフローで行うことで摩擦が減り、コンバージョン率が向上します。
   </Check>
   
   <Info>
   製品タイプを混在させる場合、都度払いは即時に請求され、サブスクリプションは設定された請求サイクル（設定した試用期間を含む）に従って開始されます。
   </Info>
   
   詳細: [Checkout Sessions](/developer-resources/checkout-session)

2. **Unified Customer Portal**  
   
   Dodo Payments上のさまざまなビジネスでのすべての購入とサブスクリプションを確認・管理できる新しい集中型顧客ポータル [customer.dodopayments.com](https://customer.dodopayments.com)。
   
   <Frame>
   <img src="/images/changelog/unified-customer-portal.png" alt="統合顧客ポータル" />
   </Frame>
   
   ### ポータルの機能
   
   - **ビジネス横断の可視化**：Dodo Paymentsを利用するすべての事業者の購入やサブスクリプションを一か所で確認できます
   - **集中型サブスクリプション管理**：異なるマーチャントのサブスクリプションを単一のダッシュボードで管理します
   - **統一された請求履歴**：すべての購入に関する請求書と支払い履歴にアクセスできます
   - **シングルサインオン体験**：メールアドレスで一度ログインすれば、すべてのDodo Paymentsマーチャントの購入にアクセス可能です
   
   ### 利用方法
   
   顧客は [customer.dodopayments.com](https://customer.dodopayments.com) にアクセスしてメールアドレスを入力するだけです。安全なログインリンクが送信され、Dodo Paymentsエコシステム全体の購入にアクセスできます。
   
   <Tip>
   統合顧客ポータルは事業者固有の顧客ポータルを補完します。顧客はブランド体験を重視する場合は個別のマーチャントポータルにアクセスすることも、統合ポータルで統合ビューを利用することもできます。
   </Tip>
   
   <Note>
   統合ポータルは、事業者ごとのポータルと同じメールベースの認証を使用して、一貫性とセキュリティの高い体験を提供します。
   </Note>

3. **Invoice ID in Payment Response**  
   支払いレスポンスには `invoice_id` フィールドが含まれるようになり、支払いと請求書の照合や会計システムとの統合が容易になります。
   

   ```json
   {
     "payment_id": "pay_2IjeQm4hqU6RA4Z4kwDee",
     "status": "succeeded",
     "invoice_id": "inv_2IsUnWGtRKFLxk7xAQeyt",
     "amount": 9900,
     "currency": "USD",
     // ... other fields
   }
   ```

   
   ### 利用例
   
   - **会計統合**：支払いを請求書に直接紐付けて会計ソフトに連携します
   - **カスタマーサポート**：サポート対応時に支払いに紐づく請求書を素早く特定します
   - **照合**：プログラム的に支払いと請求書を照合して財務の突合せを簡素化します
   - **コンプライアンス**：支払いと対応する税務請求書を紐づけた正確な記録を保持します
   
   <Info>
   `invoice_id` フィールドは、利用可能な場合にインド特有の請求書IDを使用して、現地の請求要件に準拠します。
   </Info>
   
   詳細: [Get Payment Detail API](/api-reference/payments/get-payments-1)
