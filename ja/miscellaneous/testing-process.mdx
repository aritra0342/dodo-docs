## テストカードの詳細

Dodo Paymentsは、さまざまな支払いシナリオをシミュレートするために使用できるテストカード番号のセットを提供しています。これらのテストカード番号を使用して、実際の支払いまたは請求を処理することなく、テストモードでシミュレートされた支払いを作成できます。

{/* LOCKED_PATTERN_6d744560e4135463c359b094ae69cd5f */}
{/* LOCKED_PATTERN_717af1b1289e4c0b6d1c95c8c204d71e */}
テスト取引は実際の支払いを処理せず、実アカウントにも影響を与えません。
</Card>

{/* LOCKED_PATTERN_5ba03ec58a56547ac9750ef18da96a7f */}
すべての API エンドポイントはテストモードとライブモードで同じように動作します。
</Card>

{/* LOCKED_PATTERN_1f50b75a756a498817926e42de6e14a9 */}
テスト取引でも、本番と同様に Webhook がトリガーされます。
</Card>
</CardGroup>

<Warning>
テスト認証情報はテストモードでのみ有効です。ライブモードで使用しようとすると支払いが失敗します。
</Warning>

## テストAfterpayの詳細

チェックアウト統合をテストする際は、Afterpayを支払い方法として選択し、支払いボタンをクリックしてください。

### 成功した支払い
- 成功した支払いには、有効なメールアドレスと配送先住所を使用してください。

| ステータス | UPI ID |
| :----- | :----- |
| 成功 | `success@upi` |
| 失敗 | `failure@upi` |

**注意**: Afterpayトランザクションに必要な最低金額は$50です。

- 請求先の国は `IN` に設定する必要があります
- 通貨は `INR` である必要があります
- インド以外のマーチャントの場合: Adaptive Currency を有効にする必要があります

## Apple Payのテスト

Dodo Paymentsのテストアカウントを使用してApple Payをテストできます。

<Info>
Apple Payをテストするには、iPad、iPhone、またはMacが必要です。MacではSafariを使用してください。
</Info>

<Steps>
<Step title="Dodo PaymentsアカウントでApple Payを有効にする">
Dodo Paymentsのアカウント設定でApple Payを有効にします。
</Step>

<Step title="Apple IDにサインアップする">
まだ持っていない場合は、[Apple IDにサインアップ](https://support.apple.com/apple-id)してください。
</Step>

<Step title="Apple Payに少なくとも1枚のカードを追加する">
[Apple Payに少なくとも1枚のカードを追加](https://support.apple.com/HT204506)します。Apple Payでテストする際は、実際のカード情報を使用しますが、カードは請求されません。
</Step>

{/* LOCKED_PATTERN_53d873bd6fa65f248f65f62194471195 */}
Klarna が支払いオプションとして表示されるには、最低取引額が **$50.01** である必要があります。
{/* LOCKED_PATTERN_49aff3a3aee1d0bab41e7eade47939d0 */}

<Step title="カードを選択してテスト購入を完了する">
Apple Payウォレットからカードを選択し、テスト購入を完了します。
</Step>
</Steps>

<Steps>
{/* LOCKED_PATTERN_50be67b06aca0719749c0148b14ededb */}
チェックアウトで Afterpay を支払い方法として選択し、「Pay」をクリックします。
</Step>

{/* LOCKED_PATTERN_4bb7c870b57f0176daa77351314d2d83 */}
有効なメールアドレスと配送先住所を使用すると、支払いが成功します。
</Step>

{/* LOCKED_PATTERN_9b28d8c27c08f6eeb134ce8a1d8220dd */}
失敗をシミュレートするには、リダイレクトページの Afterpay モーダルウィンドウを閉じてください。支払いは `requires_action` から `requires_payment_method` に遷移します。
</Step>
</Steps>

{/* LOCKED_PATTERN_53d873bd6fa65f248f65f62194471195 */}
Afterpay が支払いオプションとして表示されるには、最低取引額が **$50.01** である必要があります。
{/* LOCKED_PATTERN_49aff3a3aee1d0bab41e7eade47939d0 */}

<Steps>
<Step title="Dodo PaymentsアカウントでGoogle Payを有効にする">
Dodo Paymentsのアカウント設定でGoogle Payを有効にします。
</Step>

<Step title="Googleアカウントにサインアップする">
まだ持っていない場合は、[Googleアカウントにサインアップ](https://accounts.google.com/signup)してください。
</Step>

<Step title="Google Payに少なくとも1枚のカードを追加する">
[Google Payに少なくとも1枚のカードを追加](https://payments.google.com/)します。Google Payを使用する際はテストカードを使用できますが、ウォレットに有効なカードが少なくとも1枚必要です。
</Step>

<Steps>
{/* LOCKED_PATTERN_540056f13df545529727751bb5b93f77 */}
Dodo Payments のテスト API キーを使用してください。
</Step>

{/* LOCKED_PATTERN_837aa82e384540b5d8217cef3491dd36 */}
実際のカードを Apple Wallet に追加します。テストモードではカードに課金されません。
</Step>

{/* LOCKED_PATTERN_1d6cb75670bb888842ec7c28026f5dfa */}
Apple デバイスでチェックアウトを開き、Apple Pay のフローを完了してください。
</Step>
</Steps>

<Warning>
Apple Pay は HTTPS が必要です。適切な SSL を設定していないと `localhost` には表示されません。ドメイン認証も完了している必要があります。
</Warning>

### Amazon Pay、Cash App Pay & RevolutPay

テストAPIキーを使用し、標準のチェックアウトフローに従ってください。テスト取引は実際の料金なしでシミュレートされます。

Apple Payのドメイン確認を含む完全なデジタルウォレットのドキュメントについては、[デジタルウォレット](/features/payment-methods/digital-wallets)ページを参照してください。

## ヨーロッパのメソッドのテスト

ヨーロッパの支払い方法（iDEAL、Bancontact、EPS、Multibanco）は、サンドボックスモードでテストできます。

<Steps>
{/* LOCKED_PATTERN_540056f13df545529727751bb5b93f77 */}
Dodo Payments のテスト API キーを使用してください。
</Step>

{/* LOCKED_PATTERN_73a737ada9108be9fcbfe7701d196c25 */}
請求先住所の国を支払い方法に合わせてください:
- iDEAL の場合: `NL`
- Bancontact の場合: `BE`
- EPS の場合: `AT`
- Multibanco の場合: `PT`
</Step>

{/* LOCKED_PATTERN_111b236b56d1ef796758b64d1fe1fcd1 */}
ヨーロッパの決済方法では EUR 通貨が必要です。
</Step>

{/* LOCKED_PATTERN_3efbb33511174ca4755c102c30869543 */}
テスト環境でシミュレートされた銀行認証フローに従ってください。
</Step>
</Steps>

完全なヨーロッパの支払い方法のドキュメントについては、[ヨーロッパ](/features/payment-methods/europe)ページを参照してください。

## テストのベストプラクティス

<AccordionGroup>
{/* LOCKED_PATTERN_065a271b5abfd247b6320fbca34af90c */}
成功した支払いだけでなく、拒否、キャンセル、残高不足などのエッジケースもテストしてください。
</Accordion>

{/* LOCKED_PATTERN_b9eb520012676a2badc82027085ca781 */}
Webhook エンドポイントがすべてのイベントタイプ、特に `payment.succeeded`、`payment.failed`、サブスクリプションイベントを正しく処理していることを確認してください。

ローカルで Webhook をテストするには [Dodo Payments CLI](/developer-resources/sdks/cli#webhook-testing) を使用してください:
- `dodo wh listen` はライブテスト Webhook をローカルサーバーに転送します
- `dodo wh trigger` は 22 種類のイベントすべてに対してモックペイロードを送信します
</Accordion>

{/* LOCKED_PATTERN_7fd9589439eedb7dd6977012ac791713 */}
Apple Pay と Google Pay では、実際の iOS および Android デバイスでテストしてください。シミュレータではウォレットの動作を完全には再現できません。
</Accordion>

{/* LOCKED_PATTERN_ef9afd776f8103114546b31cb4fef641 */}
地域の支払い方法（UPI、iDEAL など）では、請求先住所が一致している必要があります。米国の請求先住所では iDEAL が表示されません。
</Accordion>

{/* LOCKED_PATTERN_52e65b7bb9610a0aa115d3b6ba276057 */}
BNPL（後払い）では最低 $50.01 が必要です。カート合計に応じて表示/非表示が正しく切り替わるかテストしてください。
</Accordion>
</AccordionGroup>
