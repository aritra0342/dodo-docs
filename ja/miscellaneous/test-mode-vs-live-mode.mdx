### 環境依存性

**環境依存:** すべての環境依存機能は、テストモードとライブモードで独立したデータを持ちます。一方の環境で行った変更は、他方には影響しません。例：テストモードで生成されたAPIキーは、ライブモードで生成されたAPIキーとは独立しています。

**環境に依存しない:** すべての環境に依存しない機能は、ライブモードとテストモードの両方で同じデータを持ちます。例：マイアカウントセクションは、テストモードとライブモードで同じデータを持ちます。一方の環境で行った変更は、他方の環境にも影響します。

ライブモードは、本人確認とビジネス確認が成功裏に完了した後にのみ利用可能です。実際の支払いは\`live mode\`でのみ受け付けることができます。

テストモードとライブモードのAPIホスト名は以下の通りです：
- テストモード: [`https://test.dodopayments.com`](https://test.dodopayments.com/)
- ライブモード: [`https://live.dodopayments.com`](https://live.dodopayments.com/)

## 主なハイライト

1. テストモードまたはライブモードで作成されたすべての製品は、それぞれの環境でのみ利用可能です。テストモード環境には、ライブモードにコピーするオプションがあります。これにより、関連するすべての詳細を持つ製品がコピーされ、ライブモードに作成されます。

2. テストモードで作成されたAPIキーとWebhookは、テストモード環境にのみ制限され、ライブモードのデプロイメントには影響しません。その逆も同様です。

3. 検証のために提出されたすべての文書と提出された銀行口座の詳細は、テストモードとライブモードで同じです。いずれかの環境でこの情報が変更されると、他の環境でも変更されます。

## テストカードとUPI ID

テストカード、UPI ID、およびその他のテスト方法に関する詳細情報については、包括的な[テストプロセス](/miscellaneous/testing-process)ガイドを参照してください。

このガイドには以下が含まれます：
- 様々なシナリオのためのテストカード番号
- 支払いをシミュレーションするためのテストUPI ID
- 様々な支払い結果をテストするための手順
- 統合テストのベストプラクティス

<Note>
テスト支払い方法はテストモードでのみ有効です。実際の支払いには絶対に使用しないでください。
</Note>
