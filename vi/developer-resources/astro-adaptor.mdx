## Cài Đặt

<Steps>
<Step title="Install the package">
Chạy lệnh sau trong thư mục gốc dự án của bạn:

```bash
npm install @dodopayments/astro
```

</Step>
<Step title="Set up environment variables">
Tạo một <code>.env</code> file trong thư mục gốc dự án của bạn:

```env expandable
DODO_PAYMENTS_API_KEY=your-api-key
DODO_PAYMENTS_WEBHOOK_KEY=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT="test_mode" or "live_mode"
DODO_PAYMENTS_RETURN_URL=https://yourdomain.com/checkout/success
```

<Warning>
Không bao giờ cam kết tệp <code>.env</code> hoặc bí mật của bạn vào hệ thống kiểm soát phiên bản.
</Warning>
</Step>
</Steps>

## Xử Lý Đường Dẫn Thanh Toán

<Info>
Dodo Payments hỗ trợ ba loại luồng thanh toán để tích hợp thanh toán vào trang web của bạn, trình điều hợp này hỗ trợ tất cả các loại luồng thanh toán.
</Info>

- **Liên Kết Thanh Toán Tĩnh:** URL có thể chia sẻ ngay lập tức để thu thập thanh toán nhanh chóng, không cần mã.
- **Liên Kết Thanh Toán Động:** Tạo liên kết thanh toán theo chương trình với các chi tiết tùy chỉnh bằng cách sử dụng API hoặc SDK.
- **Phiên Thanh Toán:** Tạo trải nghiệm thanh toán an toàn, tùy chỉnh với giỏ hàng sản phẩm đã được cấu hình trước và thông tin khách hàng.

### Tham số truy vấn
<ParamField query="customer_id" type="string" required>
ID khách hàng cho phiên cổng (ví dụ, <code>?customer_id=cus_123</code>).
</ParamField>

Nếu đặt thành <code>true</code>, gửi email cho khách hàng kèm liên kết cổng.
</ParamField>

<Warning>
Trả về 400 nếu thiếu <code>customer_id</code>.
</Warning>

## Xử Lý Đường Dẫn Webhook

- **Phương Thức:** Chỉ hỗ trợ yêu cầu POST. Các phương thức khác trả về 405.
- **Xác Minh Chữ Ký:** Xác minh chữ ký webhook bằng cách sử dụng <code>webhookKey</code>. Trả về 401 nếu xác minh thất bại.
- **Xác Thực Tải:** Được xác thực bằng Zod. Trả về 400 cho các tải không hợp lệ.
- **Xử Lý Lỗi:**
  - 401: Chữ ký không hợp lệ
  - 400: Tải không hợp lệ
  - 500: Lỗi nội bộ trong quá trình xác minh
- **Định Tuyến Sự Kiện:** Gọi trình xử lý sự kiện thích hợp dựa trên loại tải.
