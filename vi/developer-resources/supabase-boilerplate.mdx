## Tổng quan

Một bộ khởi động sẵn sàng cho sản xuất cho các đăng ký sử dụng Next.js 15, React 19, Supabase, Drizzle ORM và Dodo Payments. Nó đi kèm với Google OAuth, thanh toán đăng ký, xử lý webhook, sơ đồ cơ sở dữ liệu và một bảng điều khiển cơ bản.

<Info>
Nếu bạn chỉ cần các trình xử lý tuyến cho ứng dụng hiện có, hãy xem các bộ điều hợp chuyên dụng: <a href="/developer-resources/nextjs-adaptor">Next.js Adaptor</a> và <a href="/developer-resources/express-adaptor">Express Adaptor</a>.
</Info>

## Điều kiện tiên quyết

- Node.js 18+ (hoặc Bun 1.0+)
- Dự án Supabase (URL, Anon key, Service role key, Database URL)
- Tài khoản Dodo Payments (API key, Webhook secret)
- Google Cloud OAuth client (Client ID và Secret)

## Bắt đầu nhanh

<Steps>

```bash
git clone https://github.com/dodopayments/dodo-supabase-subscription-starter.git
cd dodo-supabase-subscription-starter
# choose one
bun install
# or
npm install
# or
```

{/* LOCKED_PATTERN_ea54c61e84e28cd275c0eed6d7634e0f */}

- NEXT_PUBLIC_SUPABASE_URL

- SUPABASE_SERVICE_ROLE_KEY
- DATABASE_URL (chuỗi kết nối)

{/* LOCKED_PATTERN_42bf2f0f0010fd47b77c88325d34313c */}
Đặt URI chuyển hướng là: `https://[your-project-ref].supabase.co/auth/v1/callback` trong Google Cloud, sau đó bật nhà cung cấp Google trong Supabase Auth bằng Client ID và Secret của bạn.
</Step>

{/* LOCKED_PATTERN_f32b65d8a917a558befa771bede82115 */}
Tạo khóa API và bí mật Webhook từ bảng điều khiển Dodo. Đặt môi trường thành `test_mode` trong khi phát triển.
</Step>

{/* LOCKED_PATTERN_a34a8e162789e5a79d108d9eb3c86df5 */}

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role

# Database
DATABASE_URL=postgresql://postgres:[password]@db.[project-ref].supabase.co:5432/postgres

# Dodo Payments
DODO_PAYMENTS_API_KEY=your-dodo-api-key
DODO_WEBHOOK_SECRET=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT=test_mode
```

<Warning>
Không bao giờ đưa bí mật vào mã. Sử dụng biến môi trường trong môi trường triển khai.
</Warning>
</Step>

{/* LOCKED_PATTERN_c35fe33eae6aa68b4bc39bd740632964 */}

```bash

# or

# or
pnpm run db:push
```

<Check>
Các bảng đã tạo: `users`, `subscriptions`, `payments`.
</Check>
</Step>

{/* LOCKED_PATTERN_560d1d81fad22bd0622f0c8dcb3c706d */}

```bash

bunx supabase login
# or

# deploy the edge function
bun run deploy:webhook --project-ref [your-project-ref]
# or
npm run deploy:webhook -- --project-ref [your-project-ref]
# or

```

```bash cURL
curl -X POST \
  'https://[your-project-ref].supabase.co/functions/v1/dodo-webhook' \
  -H 'Content-Type: application/json' \
  -H 'Dodo-Signature: <signature>' \
  -d '{"type":"payment.succeeded","data":{}}'
```

</Step>

## Những gì được bao gồm

- Xác thực qua Supabase (Google OAuth đã được cấu hình)
- Thanh toán đăng ký thông qua Dodo Payments
- Hàm cạnh Supabase cho webhook (`dodo-webhook`)
- Định nghĩa sơ đồ Drizzle ORM và các di trú
- Bảng điều khiển với hóa đơn, trạng thái đăng ký và tính năng gói

<Tip>
Giữ `DODO_PAYMENTS_ENVIRONMENT` là `test_mode` cho đến khi bạn hoàn thành kiểm tra đầu cuối.
</Tip>

## Xác minh và khắc phục sự cố

<AccordionGroup>
{/* LOCKED_PATTERN_6b286e9e2bbacd5c7684f8442203f10e */}
- Đảm bảo `DODO_WEBHOOK_SECRET` khớp giá trị từ bảng điều khiển Dodo
- Xác nhận bạn đã triển khai hàm `dodo-webhook` mới nhất
- Xác minh tên tiêu đề chính xác trong hàm của bạn (Dodo-Signature)
</Accordion>
{/* LOCKED_PATTERN_c270a2f8797d0168d608bb07bd40f056 */}
- Kiểm tra cú pháp `DATABASE_URL` và luồng mạng Supabase
- Chờ khoảng 2–3 phút sau khi tạo dự án trước khi đẩy lần đầu
</Accordion>
{/* LOCKED_PATTERN_276d2f8137bb346c423586f69d6b7cae */}
- URI chuyển hướng phải là `https://[ref].supabase.co/auth/v1/callback`
- Đảm bảo cùng giá trị trong Google Cloud và nhà cung cấp Supabase Auth
</Accordion>
</AccordionGroup>
