## Cài đặt

<Steps>
<Step title="Install the package">
Chạy lệnh sau trong thư mục gốc dự án của bạn:

```bash
bun add @dodopayments/bun
```

</Step>
<Step title="Set up environment variables">
Tạo tệp <code>.env</code> trong thư mục gốc dự án của bạn:

```env expandable
DODO_PAYMENTS_API_KEY=your-api-key
DODO_PAYMENTS_WEBHOOK_KEY=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT="test_mode" or "live_mode"
DODO_PAYMENTS_RETURN_URL=your-return-url
```

<Warning>
Đừng bao giờ commit tệp <code>.env</code> hoặc bí mật vào hệ thống kiểm soát phiên bản.
</Warning>
</Step>
</Steps>

## Ví dụ bộ xử lý tuyến đường

<Info>
Tất cả ví dụ giả định bạn đang sử dụng máy chủ gốc của Bun với <code>Bun.serve()</code>.
</Info>

<Tabs>
<Tab title="Checkout Handler">
  <Info>
  Sử dụng bộ xử lý này để tích hợp thanh toán Dodo Payments vào máy chủ Bun của bạn. Hỗ trợ luồng thanh toán tĩnh (GET), động (POST) và phiên (POST).
  </Info>
  <CodeGroup>

    ```typescript Bun Server Handler expandable
    import { Checkout } from '@dodopayments/bun';

    const staticCheckoutHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "static"
    });

    const sessionCheckoutHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "session"
    });

    const dynamicCheckoutHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "dynamic"
    });

    Bun.serve({
      port: 3000,
      fetch(request) {
        const url = new URL(request.url);
        
        if (url.pathname === "/api/checkout") {
          if (request.method === "GET") {
            return staticCheckoutHandler(request);
          }
          if (request.method === "POST") {
            return sessionCheckoutHandler(request);
            // or return dynamicCheckoutHandler(request);
          }
        }
        
        return new Response("Not Found", { status: 404 });
      },
    });
    ```

  </CodeGroup>

## Bộ xử lý tuyến thanh toán

<Info>
Dodo Payments hỗ trợ ba loại luồng thanh toán để tích hợp thanh toán vào trang web của bạn, bộ chuyển này hỗ trợ tất cả các loại luồng thanh toán.
</Info>

- **Liên kết thanh toán tĩnh:** URL chia sẻ ngay lập tức để thu tiền nhanh chóng, không cần mã.
- **Liên kết thanh toán động:** Tạo liên kết thanh toán theo chương trình với các chi tiết tùy chỉnh bằng API hoặc SDK.
- **Phiên thanh toán:** Tạo trải nghiệm thanh toán được lưu trữ, bảo mật với giỏ hàng và thông tin khách hàng được cấu hình sẵn.

## Bộ xử lý tuyến cổng khách hàng

Bộ xử lý tuyến cổng khách hàng giúp bạn tích hợp liền mạch cổng khách hàng Dodo Payments vào ứng dụng máy chủ Bun.

### Tham số truy vấn
<ParamField query="customer_id" type="string" required>
ID khách hàng cho phiên cổng (ví dụ, <code>?customer_id=cus_123</code>).
</ParamField>
<ParamField query="send_email" type="boolean">
Nếu được đặt thành <code>true</code>, gửi email cho khách hàng kèm liên kết cổng.
</ParamField>

<Warning>
Trả về 400 nếu thiếu <code>customer_id</code>.
</Warning>

## Bộ xử lý tuyến webhook

- **Phương thức:** Chỉ hỗ trợ yêu cầu POST. Các phương thức khác trả về 405.
- **Xác thực chữ ký:** Xác minh chữ ký webhook bằng <code>webhookKey</code>. Trả về 401 nếu xác minh thất bại.
- **Xác thực payload:** Được xác minh bằng Zod. Trả về 400 cho payload không hợp lệ.
- **Xử lý lỗi:**
  - 401: Chữ ký không hợp lệ
  - 400: Payload không hợp lệ
  - 500: Lỗi nội bộ khi xác minh
- **Định tuyến sự kiện:** Gọi bộ xử lý sự kiện phù hợp dựa trên loại payload.

  bearerToken: process.env.DODO_PAYMENTS_API_KEY,
  returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,

const customerPortalHandler = CustomerPortal({
