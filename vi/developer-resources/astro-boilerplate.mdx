## Tổng quan

Mẫu Astro tối giản cung cấp một điểm khởi đầu sẵn sàng để tích hợp Dodo Payments với ứng dụng Astro của bạn. Mẫu này bao gồm các phiên thanh toán, xử lý webhook, cổng khách hàng và các thành phần UI hiện đại để giúp bạn bắt đầu chấp nhận thanh toán nhanh chóng.

<Info>
Boilerplate này sử dụng Astro 5 với TypeScript, Tailwind CSS 4, và adapter `@dodopayments/astro`.
</Info>

### Tính năng

- **Cài đặt nhanh** - Bắt đầu trong chưa đầy 5 phút
- **Tích hợp thanh toán** - Quy trình thanh toán được cấu hình sẵn sử dụng `@dodopayments/astro`
- **Giao diện hiện đại** - Trang giá sạch sẽ, chủ đề tối với Tailwind CSS
- **Xử lý Webhook** - Điểm cuối webhook sẵn sàng sử dụng cho các sự kiện thanh toán
- **Cổng thông tin khách hàng** - Quản lý đăng ký chỉ với một cú nhấp chuột
- **TypeScript** - Được gõ hoàn toàn với các kiểu tối thiểu, tập trung
- **Thanh toán đã được điền sẵn** - Chứng minh việc truyền dữ liệu khách hàng để cải thiện UX

## Điều kiện tiên quyết

Trước khi bắt đầu, hãy đảm bảo bạn đã có:

- **Phiên bản Node.js LTS** (cần thiết cho Astro 5)
- **Tài khoản Dodo Payments** (để truy cập API và Webhook Keys từ bảng điều khiển)

## Bắt đầu nhanh

<Steps>
<Step title="Nhân bản kho lưu trữ">

```bash
git clone https://github.com/dodopayments/dodo-astro-minimal-boilerplate.git
cd dodo-astro-minimal-boilerplate
```

</Step>

<Step title="Cài đặt các phụ thuộc">

```bash
npm install
```

</Step>

<Step title="Lấy thông tin xác thực API">
Đăng ký tại [Dodo Payments](https://dodopayments.com/) và lấy thông tin xác thực của bạn từ bảng điều khiển:

- **API Key:** [Bảng điều khiển → Nhà phát triển → API Keys](https://app.dodopayments.com/developer/api-keys)
- **Webhook Key:** [Bảng điều khiển → Nhà phát triển → Webhooks](https://app.dodopayments.com/developer/webhooks)

<Tip>
Hãy chắc chắn rằng bạn đang ở **Chế độ thử nghiệm** trong khi phát triển!
</Tip>
</Step>

<Step title="Cấu hình Biến môi trường">
Tạo một file `.env` trong thư mục gốc:

```bash
cp .env.example .env
```

Cập nhật các giá trị với thông tin xác thực Dodo Payments của bạn:

```env
DODO_PAYMENTS_API_KEY=your_api_key_here
DODO_PAYMENTS_WEBHOOK_KEY=your_webhook_signing_key_here
DODO_PAYMENTS_RETURN_URL=http://localhost:4321/
DODO_PAYMENTS_ENVIRONMENT=test_mode
```

<Warning>
Không bao giờ cam kết file `.env` của bạn vào kiểm soát phiên bản. Nó đã được bao gồm trong `.gitignore`.
</Warning>
</Step>

<Step title="Thêm Sản phẩm của bạn">
Cập nhật `src/lib/products.ts` với các ID sản phẩm thực tế từ Dodo Payments:

```typescript
export const products: Product[] = [
  {
    product_id: "pdt_001", // Replace with your product ID
    name: "Basic Plan",
    description: "Get access to basic features and support",
    price: 9999, // in cents
    features: [
      "Access to basic features",
      "Email support",
      "1 Team member",
      "Basic analytics",
    ],
  },
  // ... add more products
];
```

</Step>

<Step title="Chạy máy chủ phát triển">

```bash
npm run dev
```

Mở [http://localhost:4321](http://localhost:4321) để xem trang giá của bạn!
</Step>
</Steps>

## Cấu trúc dự án

```text
src/
├── components/
│   ├── Footer.astro           # Reusable footer
│   ├── Header.astro           # Navigation header
│   └── ProductCard.astro      # Product pricing card
├── layouts/
│   └── Layout.astro           # Root layout
├── lib/
│   └── products.ts            # Product definitions
├── pages/
│   ├── index.astro            # Pricing page (home)
│   └── api/
│       ├── checkout.ts        # Checkout session handler
│       ├── customer-portal.ts # Customer portal redirect
│       └── webhook.ts         # Webhook event handler
└── styles/
    └── global.css             # Global styles with Tailwind
```

## Tùy chỉnh

### Cập nhật thông tin sản phẩm

Chỉnh sửa `src/lib/products.ts` để thay đổi:
- ID sản phẩm (từ bảng điều khiển Dodo của bạn)
- Giá cả
- Tính năng
- Mô tả

### Điền trước dữ liệu khách hàng

Trong `src/components/ProductCard.astro`, thay thế các giá trị cứng bằng dữ liệu người dùng thực tế của bạn:

```typescript
customer: {
  name: "John Doe",
  email: "john@example.com",
},
```

### Cập nhật cổng khách hàng

Trong `src/components/Header.astro`, thay thế ID khách hàng cứng bằng ID khách hàng thực tế từ hệ thống xác thực hoặc cơ sở dữ liệu của bạn:

```typescript
const CUSTOMER_ID = "cus_001"; // Replace with actual customer ID
```

Bạn có thể hoàn thành một giao dịch thử nghiệm để lấy ID khách hàng cho việc thử nghiệm.

## Sự kiện Webhook

Boilerplate chứng minh việc xử lý các sự kiện webhook trong `src/pages/api/webhook.ts`:

- `onSubscriptionActive` - Kích hoạt khi một đăng ký trở nên hoạt động
- `onSubscriptionCancelled` - Kích hoạt khi một đăng ký bị hủy

Thêm logic kinh doanh của bạn bên trong các trình xử lý này:

```typescript
onSubscriptionActive: async (payload) => {
  // Grant access to your product
  // Update user database
  // Send welcome email
},
```

Thêm nhiều sự kiện webhook nếu cần.

Đối với phát triển cục bộ, bạn có thể sử dụng các công cụ như [ngrok](https://ngrok.com/) để tạo một đường hầm bảo mật đến máy chủ cục bộ của bạn và sử dụng nó làm URL webhook của bạn.

## Triển khai

Mẫu này sử dụng đầu ra tĩnh với việc kết xuất theo yêu cầu cho các tuyến API. Bạn sẽ cần cài đặt một adapter cho nền tảng triển khai của bạn:

| Nền tảng | Hướng dẫn |
|----------|-------|
| Vercel | [Triển khai lên Vercel](https://docs.astro.build/en/guides/deploy/vercel/) |
| Netlify | [Triển khai lên Netlify](https://docs.astro.build/en/guides/deploy/netlify/) |
| Cloudflare | [Triển khai lên Cloudflare](https://docs.astro.build/en/guides/deploy/cloudflare/) |

Xem [hướng dẫn triển khai của Astro](https://docs.astro.build/en/guides/deploy/) cho tất cả các nền tảng.

### Cập nhật URL Webhook

Sau khi triển khai, hãy cập nhật URL webhook của bạn trong [Bảng điều khiển Dodo Payments](https://app.dodopayments.com/developer/webhooks):

```text
https://your-domain.com/api/webhook
```

Cũng nhớ cập nhật biến môi trường `DODO_PAYMENTS_WEBHOOK_KEY` trong môi trường sản xuất của bạn để khớp với khóa ký webhook cho miền đã triển khai của bạn.

## Khắc phục sự cố

<AccordionGroup>
<Accordion title="Module không tìm thấy hoặc lỗi xây dựng">
Xóa `node_modules` và cài đặt lại các phụ thuộc:

```bash
rm -rf node_modules package-lock.json
npm install
```

</Accordion>

<Accordion title="Chuyển hướng thanh toán không thành công">
**Nguyên nhân phổ biến:**
- ID sản phẩm không hợp lệ - xác minh nó tồn tại trong bảng điều khiển Dodo của bạn
- Khóa API hoặc cài đặt môi trường sai trong `.env`
- Kiểm tra bảng điều khiển trình duyệt và terminal để tìm lỗi
</Accordion>

<Accordion title="Webhooks không nhận sự kiện">
Đối với thử nghiệm cục bộ, sử dụng [ngrok](https://ngrok.com) để phơi bày máy chủ của bạn:

```bash
ngrok http 4321
```

Cập nhật URL webhook trong [bảng điều khiển Dodo](https://app.dodopayments.com/developer/webhooks) thành URL ngrok. Nhớ cập nhật file .env của bạn với khóa xác minh webhook chính xác.
</Accordion>

<Accordion title="Liên kết cổng thông tin khách hàng không hoạt động">
Thay thế `CUSTOMER_ID` cứng trong `src/components/Header.astro` bằng một ID khách hàng thực tế từ bảng điều khiển Dodo của bạn.

Hoặc tích hợp hệ thống xác thực và cơ sở dữ liệu của bạn để lấy ID khách hàng một cách động.

Xem [hướng dẫn triển khai của Astro](https://docs.astro.build/en/guides/deploy/) để biết chi tiết.
