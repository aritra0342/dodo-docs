## Overview

The Model Context Protocol (MCP) is an open standard that enables AI applications to securely connect to external data sources and APIs. The Dodo Payments MCP Server provides AI assistants like Claude, Cursor, and other MCP-compatible clients with structured access to your payment infrastructure.

The Dodo Payments MCP Server uses the **Code Mode** architecture. Instead of exposing hundreds of individual tools for every API endpoint, Code Mode enables AI agents to write and execute TypeScript code against the Dodo Payments SDK in an isolated sandbox environment.

### Key capabilities

- **Payment Operations**: Create, retrieve, and manage payments and refunds
- **Subscription Management**: Handle recurring billing, upgrades, and cancellations
- **Customer Administration**: Manage customer data and portal access
- **Product Catalog**: Create and update products, pricing, and discounts
- **License Management**: Activate, validate, and manage software licenses
- **Usage-Based Billing**: Track and bill for metered usage

## How Code Mode Works

Máy chủ Dodo Payments MCP cung cấp cho tác nhân AI của bạn chính xác hai công cụ:

1. **Docs Search Tool**: Truy vấn tài liệu về API và SDK Dodo Payments để hiểu các thao tác và tham số có thể dùng.
2. **Code Execution Tool**: Viết mã TypeScript sử dụng SDK và thực thi trong môi trường hộp cát an toàn.

Kiến trúc này cho phép các tác nhân thực hiện các thao tác phức tạp nhiều bước trong một lần gọi. Ví dụ, một tác nhân có thể liệt kê tất cả đăng ký đang hoạt động, lọc theo tiêu chí cụ thể và áp dụng giảm giá cho từng cái — tất cả trong một kịch bản duy nhất.

<Tip>
Sử dụng Code Mode, các tác nhân có thể xâu chuỗi nhiều cuộc gọi API, xử lý logic điều kiện và thực hiện các phép tính mà không cần nhiều vòng phản hồi với LLM.
</Tip>

### Thiết lập nhanh

Kết nối với Máy chủ Dodo Payments MCP trong ứng dụng AI của bạn:

<Tabs>
<Tab title="Cursor">

```json
{
  "mcpServers": {
    "dodopayments": {
      "command": "npx",

  }
```

## Dodo Knowledge MCP

Bên cạnh Máy chủ Dodo Payments MCP (dùng để thực thi thao tác API), chúng tôi cung cấp **Dodo Knowledge MCP** — một máy chủ tìm kiếm ngữ nghĩa giúp trợ lý AI truy cập ngay lập tức tài liệu và cơ sở kiến thức Dodo Payments.

<Info>
**Được xây dựng với [ContextMCP.ai](https://contextmcp.ai)**: Dodo Knowledge MCP được cung cấp bởi ContextMCP, cho phép tìm kiếm ngữ nghĩa nhanh chóng trên toàn bộ tài liệu của chúng tôi bằng cách sử dụng nhúng vector.
</Info>

### Dodo Knowledge MCP là gì?

Dodo Knowledge MCP là một máy chủ MCP từ xa cung cấp:

- **Semantic Documentation Search**: Tìm tài liệu phù hợp bằng truy vấn ngôn ngữ tự nhiên.
- **Contextual Answers**: Trợ lý AI nhận được thông tin chính xác, cập nhật về Dodo Payments.
- **Zero Setup**: Không cần khóa API hay cài đặt cục bộ — chỉ cần kết nối và bắt đầu truy vấn.

### Sử dụng cả hai Máy chủ MCP cùng lúc

Để có trải nghiệm phát triển có trợ giúp AI tốt nhất, chúng tôi khuyên bạn sử dụng cả hai máy chủ MCP:

| Máy chủ | Mục đích | Tình huống sử dụng |
|--------|---------|----------|
| **Dodo Knowledge MCP** | Tìm kiếm tài liệu | “Làm sao xử lý webhooks?”, “Các phương thức thanh toán nào được hỗ trợ?” |
| **Dodo Payments MCP** | Thao tác API | Tạo thanh toán, quản lý đăng ký, xử lý hoàn tiền |

```json
{
  "mcpServers": {
    "dodo-knowledge": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://knowledge.dodopayments.com/mcp"]
    },
    "dodopayments": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://mcp.dodopayments.com/sse"]
```

### Gói NPM

Cài đặt và chạy máy chủ MCP cục bộ bằng NPM.

<Tabs>

```bash
# Set your environment variables
export DODO_PAYMENTS_API_KEY="dodo_test_..."
export DODO_PAYMENTS_WEBHOOK_KEY="your_webhook_key"
export DODO_PAYMENTS_ENVIRONMENT="live_mode"

npx -y dodopayments-mcp@latest
```

</Tab>

<Tab title="MCP Client Configuration">

```json
  "mcpServers": {
    "dodopayments_api": {

    }
  }
```

</Tab>

<Tip>
Sử dụng `@latest` để luôn tải phiên bản mới nhất, hoặc cố định vào một phiên bản cụ thể như `@1.0.0` để đảm bảo ổn định.
</Tip>

### Docker

Chạy máy chủ MCP trong môi trường container để triển khai nhất quán.

<CodeGroup>

```bash Docker
# Pull the latest image
docker pull ghcr.io/dodopayments/mcp:latest

# Run the container
docker run -e DODO_PAYMENTS_API_KEY="dodo_test_..." \
  -e DODO_PAYMENTS_WEBHOOK_KEY="your_webhook_key" \
  -e DODO_PAYMENTS_ENVIRONMENT="live_mode" \
  ghcr.io/dodopayments/mcp:latest
```

```yaml Docker Compose
services:
  dodopayments-mcp:
    image: ghcr.io/dodopayments/mcp:latest
    environment:
      - DODO_PAYMENTS_API_KEY=dodo_test_...
      - DODO_PAYMENTS_WEBHOOK_KEY=your_webhook_key
      - DODO_PAYMENTS_ENVIRONMENT=live_mode
    restart: unless-stopped
```

</CodeGroup>

<Note>
Các image Docker có sẵn trên [GitHub Container Registry](https://github.com/orgs/dodopayments/packages/container/package/mcp).
</Note>

## Biến môi trường

Cấu hình hành vi của máy chủ MCP bằng các biến môi trường.

| Biến | Mô tả | Bắt buộc |
|----------|-------------|----------|
| `DODO_PAYMENTS_API_KEY` | Khóa API Dodo Payments của bạn | Có |
| `DODO_PAYMENTS_WEBHOOK_KEY` | Khóa ký webhook của bạn | Không |
| `DODO_PAYMENTS_ENVIRONMENT` | Đặt thành `live_mode` cho môi trường sản xuất | Không |

## Chạy từ xa

Triển khai máy chủ MCP dưới dạng máy chủ HTTP từ xa cho các ứng dụng khách web hoặc quy trình làm việc tác nhân.

```bash
# Start as remote server
npx -y dodopayments-mcp --transport=http

# Specify a custom port
npx -y dodopayments-mcp --transport=http --port=3000
```

### Cấu hình máy chủ từ xa

Sau khi triển khai, các ứng dụng khách có thể kết nối bằng URL máy chủ:

```json
{
  "mcpServers": {
    "dodopayments_api": {
      "url": "http://localhost:3000",
      "headers": {
        "Authorization": "Bearer your_api_key"
      }
    }
  }
}
```

### Header ủy quyền

Máy chủ từ xa chấp nhận xác thực qua các header sau:

| Header | Mô tả |
|--------|-------------|
| `Authorization` | Xác thực bearer token |
| `x-dodo-payments-api-key` | Header khóa API trực tiếp |

Code Mode mang lại bảo mật vốn có bằng việc thực thi mã trong môi trường hộp cát và tiêm khóa API phía máy chủ. Tuân theo các thực hành bổ sung sau để bảo vệ thông tin đăng nhập.

<AccordionGroup>
<Accordion title="API Key Management">

**Sử dụng khóa thử nghiệm cho phát triển**

## Tại sao là Code Mode

Các triển khai MCP truyền thống thường gặp phải "sự nở rộ công cụ", khi mỗi điểm cuối API được phơi bày dưới dạng một công cụ riêng biệt. Code Mode là phương pháp vượt trội vì nhiều lý do:

### LLMs giỏi viết mã hơn gọi công cụ
LLM đã được huấn luyện trên hàng triệu dòng mã thực tế, khiến chúng tự nhiên thành thạo việc viết kịch bản. Ngược lại, việc gọi công cụ thường dựa trên các ví dụ tổng hợp.

<Info>
"Bắt một LLM thực hiện tác vụ bằng cách gọi công cụ giống như bắt Shakespeare tham gia lớp học tiếng Trung kéo dài một tháng rồi yêu cầu ông viết một vở kịch bằng ngôn ngữ đó." — [Cloudflare](https://blog.cloudflare.com/code-mode/)
</Info>

### Loại bỏ tình trạng tiêu tốn ngữ cảnh
Trong cách tiếp cận truyền thống, mỗi định nghĩa công cụ tiêu tốn token trước cả khi cuộc hội thoại bắt đầu. Phơi bày hơn 50 công cụ có thể dễ dàng ngốn 55K–100K+ token. Anthropic nhận thấy định nghĩa công cụ có thể tiêu hết tới 134K token trước tối ưu hóa.

Với Code Mode, chỉ có 2 định nghĩa công cụ được tải (~1K token). Tác nhân tìm kiếm tài liệu cần thiết theo yêu cầu. Công cụ Tìm kiếm của Anthropic giữ lại 95% cửa sổ ngữ cảnh, giảm chi phí từ 77K xuống còn 8.7K token.

### Giảm độ trễ thông qua điều phối lập trình
Việc gọi công cụ truyền thống yêu cầu một vòng suy diễn mô hình cho mỗi thao tác riêng biệt. Nếu một tác vụ cần 20 cuộc gọi API, thì phải chịu 20 vòng lặp. Trong Code Mode, tác nhân viết một kịch bản thực thi tất cả các cuộc gọi và chỉ trả về kết quả cuối cùng.

Anthropic quan sát thấy giảm 37% token và cải thiện độ chính xác (khả năng truy xuất kiến thức tăng từ 25.6% lên 28.5%) khi sử dụng cách tiếp cận lập trình này.

### Bảo mật hơn theo thiết kế
Code Mode cung cấp lợi ích bảo mật vốn có:
- **Không có khóa API trong tham số**: Khóa API được tiêm phía máy chủ và không bao giờ hiển thị trong tham số gửi đến LLM.
- **Hộp cát cô lập**: Mã chạy trong môi trường an toàn không truy cập mạng hay hệ thống tệp máy chủ.
- **SDK được kiểm soát**: Chỉ những phương thức SDK được cấp phép mới có sẵn cho tác nhân.

### Mở rộng tới mọi quy mô API
Khi một API phát triển, hiệu suất MCP truyền thống suy giảm vì phải nạp nhiều công cụ hơn vào ngữ cảnh. Code Mode vẫn giữ nguyên 2 công cụ bất kể diện tích bề mặt API. Cloudflare đã hợp nhất hơn 2,500 điểm cuối API lại chỉ còn 2 công cụ và khoảng 1,000 token ngữ cảnh.

<Info>
Để biết thêm chi tiết về lợi ích của kiến trúc này, hãy xem các blog kỹ thuật từ [Anthropic](https://www.anthropic.com/engineering/advanced-tool-use) và [Cloudflare](https://blog.cloudflare.com/code-mode/), cũng như tài liệu [Programmatic Tool Calling](https://platform.claude.com/docs/en/agents-and-tools/tool-use/programmatic-tool-calling) từ Claude.
</Info>
