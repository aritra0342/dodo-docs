## Détails de la Carte de Test

Dodo Payments fournit un ensemble de numéros de carte de test que vous pouvez utiliser pour simuler divers scénarios de paiement. Vous pouvez utiliser ces numéros de carte de test pour créer des paiements simulés en mode test sans traiter de paiements ou de frais réels.

{/* LOCKED_PATTERN_6d744560e4135463c359b094ae69cd5f */}
{/* LOCKED_PATTERN_717af1b1289e4c0b6d1c95c8c204d71e */}
Les transactions de test ne traitent pas de paiements réels et n'affectent pas de comptes réels.
</Card>

{/* LOCKED_PATTERN_5ba03ec58a56547ac9750ef18da96a7f */}
Toutes les API fonctionnent de manière identique en mode test et production.
</Card>

{/* LOCKED_PATTERN_1f50b75a756a498817926e42de6e14a9 */}
Les webhooks sont déclenchés pour les transactions de test comme en production.
</Card>
</CardGroup>

<Warning>
Les identifiants de test ne fonctionnent qu'en mode test. Les utiliser en mode production entraînera des échecs de paiement.
</Warning>

## Détails Afterpay de Test

Lors de la test de votre intégration de Checkout, sélectionnez Afterpay comme méthode de paiement et cliquez sur le bouton Payer.

### Paiement Réussi
- Utilisez n'importe quelle adresse email et adresse de livraison valide pour des paiements réussis.

| Statut | ID UPI |
| :----- | :------ |
| Succès | `success@upi` |
| Échec | `failure@upi` |

**Remarque** : Le montant minimum requis pour les transactions Afterpay est de 50 $.

- Le pays de facturation doit être défini sur `IN`
- La devise doit être `INR`
- Pour les commerçants non indiens : la devise adaptative doit être activée

## Tester Apple Pay

Vous pouvez tester Apple Pay en utilisant votre compte de test Dodo Payments.

<Info>
Vous aurez besoin d'un iPad, d'un iPhone ou d'un Mac pour tester Apple Pay. Utilisez Safari sur Mac.
</Info>

<Steps>
<Step title="Activer Apple Pay pour votre compte Dodo Payments">
Activez Apple Pay dans les paramètres de votre compte Dodo Payments.
</Step>

<Step title="Inscrivez-vous pour un identifiant Apple">
Si vous n'en avez pas déjà un, [inscrivez-vous pour un identifiant Apple](https://support.apple.com/apple-id).
</Step>

<Step title="Ajoutez au moins une carte à Apple Pay">
[Ajoutez au moins une carte à Apple Pay](https://support.apple.com/HT204506). Lorsque vous testez avec Apple Pay, vous utilisez de vraies informations de carte mais votre carte n'est pas débitée.
</Step>

## Tester les Méthodes Européennes

Les méthodes de paiement européennes (iDEAL, Bancontact, EPS, Multibanco) peuvent être testées en mode sandbox.

<Steps>
{/* LOCKED_PATTERN_540056f13df545529727751bb5b93f77 */}
Utilisez vos clés API de test Dodo Payments.
</Step>

{/* LOCKED_PATTERN_73a737ada9108be9fcbfe7701d196c25 */}
Définissez le pays de l'adresse de facturation pour qu'il corresponde au moyen de paiement :
- `NL` pour iDEAL
- `BE` pour Bancontact
- `AT` pour EPS
- `PT` pour Multibanco
</Step>

{/* LOCKED_PATTERN_111b236b56d1ef796758b64d1fe1fcd1 */}
Les méthodes européennes exigent la monnaie EUR.
</Step>

{/* LOCKED_PATTERN_3efbb33511174ca4755c102c30869543 */}
Suivez le flux d'authentification bancaire simulé dans l'environnement de test.
</Step>
</Steps>

Pour la documentation complète des méthodes de paiement européennes, consultez la page [Europe](/features/payment-methods/europe).

## Meilleures Pratiques de Test

<AccordionGroup>
{/* LOCKED_PATTERN_065a271b5abfd247b6320fbca34af90c */}
Ne testez pas seulement les paiements réussis. Testez les refus, annulations et cas limites comme les fonds insuffisants.
</Accordion>

{/* LOCKED_PATTERN_b9eb520012676a2badc82027085ca781 */}
Assurez-vous que vos points de terminaison webhook traitent correctement tous les types d'événements, en particulier `payment.succeeded`, `payment.failed` et les événements d'abonnement.

Utilisez la [Dodo Payments CLI](/developer-resources/sdks/cli#webhook-testing) pour tester les webhooks localement :
- `dodo wh listen` redirige les webhooks de test en direct vers votre serveur local
- `dodo wh trigger` envoie des charges utiles factices pour les 22 types d'événements
</Accordion>

{/* LOCKED_PATTERN_7fd9589439eedb7dd6977012ac791713 */}
Pour Apple Pay et Google Pay, testez sur de véritables appareils iOS et Android. Les simulateurs ne reproduisent pas entièrement le comportement du portefeuille.
</Accordion>

{/* LOCKED_PATTERN_ef9afd776f8103114546b31cb4fef641 */}
Les méthodes de paiement régionales (UPI, iDEAL, etc.) exigent que les adresses de facturation correspondent. Une adresse de facturation américaine n'affichera pas iDEAL.
</Accordion>

{/* LOCKED_PATTERN_52e65b7bb9610a0aa115d3b6ba276057 */}
Les méthodes BNPL exigent un minimum de $50.01. Vérifiez qu'elles apparaissent ou disparaissent correctement en fonction du total du panier.
</Accordion>
</AccordionGroup>
