## Qu'est-ce qu'une migration d'abonnement ?

Une migration d'abonnement transfère vos abonnés existants de votre processeur de paiement hérité vers Dodo Payments, vous offrant une source unique de vérité pour toutes vos données clients. Sans migration, les clients que vous avez acquis jusqu'à présent restent dans votre système hérité, et seuls les nouveaux clients seraient visibles dans Dodo Payments.

Le processus de migration garantit une expérience de facturation fluide pour vos clients : ils n'auront pas besoin de ressaisir leurs informations de paiement ni de subir d'interruption de leurs abonnements.

<Warning>
Actuellement, nous ne supportons que la migration des **tokens de carte**. Apple Pay, Google Pay et d'autres méthodes de paiement par portefeuille ne peuvent pas être transférés entre les processeurs de paiement en raison des restrictions de tokenisation imposées par ces fournisseurs.
</Warning>

## Aperçu du processus de migration

Le processus de migration d'abonnement se compose de trois parties clés :

1. **Transfert de méthode de paiement** — Transfert sécurisé des données sensibles de carte de votre processeur actuel
2. **Transfert de données d'abonné** — Exportation des détails des clients, des plans d'abonnement et des calendriers de facturation
3. **Importation et validation** — Unification des méthodes de paiement et des données d'abonné dans Dodo Payments

<Note>
Pour commencer une migration d'abonnement, contactez notre [équipe de support](mailto:support@dodopayments.com). Ils vous mettront en relation avec un ingénieur solutions dédié qui vous guidera tout au long du processus.
</Note>

## Exigences

Avant de commencer une migration, assurez-vous de répondre aux exigences suivantes :

<AccordionGroup>
<Accordion title="Vérification de conformité PCI DSS">
Dodo Payments est conforme au niveau 1 PCI DSS. Pour répondre aux obligations de conformité, nous ne pouvons recevoir des données de carte que d'un autre processeur de paiement conforme au niveau 1 PCI DSS.

Votre processeur actuel doit fournir :
- Leur attestation de conformité PCI actuelle (AOC), ou leur inscription dans le registre mondial des fournisseurs de services de Visa
- Confirmation qu'ils supportent les exportations sécurisées de données de carte

Consultez notre [Attestation de conformité (AOC)](https://pub-2a89184e4c954b599464449d23f506bb.r2.dev/dodo-payments-pci-dss-aoc.pdf) à partager avec votre processeur actuel.
</Accordion>

<Accordion title="Intégration de Dodo Payments">
Complétez votre intégration Dodo Payments et assurez-vous que tous les nouveaux clients sont dirigés vers le paiement Dodo Payments avant de commencer la migration. Cela évite les incohérences de données pendant la période de transition.
</Accordion>

<Accordion title="Autorisation d'accès aux données">
Assurez-vous d'avoir l'autorisation nécessaire pour demander une exportation de données de votre processeur actuel. Cela nécessite généralement :
- Vérification du propriétaire du compte
- Autorisation écrite pour l'accès aux données sensibles
- Achèvement des étapes de vérification de sécurité de votre processeur
</Accordion>
</AccordionGroup>

## Guide de migration étape par étape

<Steps>
<Step title="Demander l'exportation de la méthode de paiement de votre processeur actuel">
Contactez l'équipe de support de votre processeur de paiement actuel et soumettez une demande formelle pour un export complet de vos tokens de carte client.

Vous devrez :
- Fournir une vérification de votre identité et de votre autorisation
- Demander qu'ils encryptent l'exportation en utilisant la clé publique PGP de Dodo Payments (fournie ci-dessous)
- Leur demander de transférer le fichier encrypté directement à notre serveur SFTP sécurisé

<Info>
Les transferts sécurisés se font directement de serveur à serveur via SFTP. Votre ingénieur solutions vous fournira les identifiants SFTP à partager avec votre processeur actuel.
</Info>

En moyenne, ce processus prend environ **2 semaines**, bien que le timing varie en fonction des politiques et des temps de réponse de votre processeur actuel.
</Step>

<Step title="Préparer l'exportation des données d'abonné">
Pendant que le transfert de méthode de paiement est en cours, exportez vos données d'abonné de votre système hérité. Votre ingénieur solutions vous fournira un modèle CSV et vous guidera à travers les champs requis.

### Données d'abonné requises

| Champ | Description |
|-------|-------------|
| `customer_id` | Votre identifiant client interne |
| `email` | Adresse e-mail du client |
| `name` | Nom complet du client |
| `country` | Code de pays à deux lettres (ISO 3166-1 alpha-2) |
| `currency` | Code de devise à trois lettres (ISO 4217) |
| `payment_amount` | Montant de l'abonnement dans la plus petite unité de devise |
| `payment_interval` | Fréquence de facturation (mensuelle, annuelle, etc.) |
| `next_billing_date` | Prochaine date de facturation prévue (ISO 8601) |
| `card_token` | Token unique de votre processeur actuel |

<Warning>
Le champ `card_token` est critique : il est utilisé pour faire correspondre les enregistrements d'abonné avec leurs méthodes de paiement lors de l'importation. Assurez-vous qu'il correspond exactement au token dans l'exportation de méthode de paiement.
</Warning>

### Format de données d'exemple

```json
{
  "customers": [
    {
      "customer_id": "cus_abc123",
      "email": "jane.doe@example.com",
      "name": "Jane Doe",
      "country": "US",
      "currency": "USD",
      "payment_amount": 2999,
      "payment_interval": "monthly",
      "next_billing_date": "2025-01-15T00:00:00Z",
      "card_token": "tok_xyz789",
      "metadata": {
        "plan": "pro",
        "signup_source": "website"
      }
    }
  ]
}
```

</Step>

<Step title="Validation et test">
Avant la date d'importation, notre équipe va :

- Vérifier l'intégrité du fichier et l'encryption des données de méthode de paiement
- Valider votre CSV d'abonné pour un format correct
- Effectuer un test d'importation dans notre environnement sandbox
- Identifier et résoudre tout problème de mappage de données

<Check>
Vous recevrez un rapport de validation confirmant que tout est prêt pour l'importation en production.
</Check>
</Step>

<Step title="Importation en production">
À la date d'importation convenue, notre équipe d'ingénierie va :

1. Importer les données de méthode de paiement dans notre coffre-fort de cartes conforme PCI
2. Importer vos données d'abonné
3. Faire correspondre les abonnés à leurs méthodes de paiement en utilisant les tokens de carte
4. Activer tous les abonnements migrés

<Info>
Le temps de traitement typique est de **2-3 jours ouvrables** pour les migrations standard, en fonction du volume d'abonnés.
</Info>
</Step>

<Step title="Vérification post-migration">
Après l'importation, vous recevrez :

- Un rapport de migration complet
- Un CSV mappant vos anciens IDs d'abonnement aux nouveaux IDs de Dodo Payments
- Accès à tous les clients migrés dans votre tableau de bord Dodo Payments

Nous recommandons :
- De vérifier un échantillon d'abonnements migrés dans votre tableau de bord
- De tester quelques renouvellements d'abonnement en mode sandbox
- De mettre à jour vos références de base de données internes avec les nouveaux IDs
- D'effectuer une surveillance parallèle pour le premier cycle de facturation
</Step>
</Steps>

## Données migrables

Les données suivantes peuvent être transférées lors d'une migration d'abonnement :

| Type de données | Supporté | Remarques |
|----------------|----------|----------|
| Numéros de carte de crédit/débit | ✅ | Encrypté via PGP |
| Dates d'expiration de carte | ✅ | |
| Noms des titulaires de carte | ✅ | |
| Adresses de facturation | ✅ | |
| Adresses e-mail des clients | ✅ | |
| Métadonnées des clients | ✅ | Paires clé-valeur personnalisées |
| Détails d'abonnement | ✅ | Plan, montant, intervalle |
| Tokens Apple Pay | ❌ | Ne peuvent pas être transférés |
| Tokens Google Pay | ❌ | Ne peuvent pas être transférés |
| Autres tokens de portefeuille | ❌ | Ne peuvent pas être transférés |
| Historique des paiements | ❌ | Exporter de votre processeur actuel |

<Note>
L'historique des paiements et les enregistrements de transactions ne peuvent pas être migrés. Nous recommandons d'exporter ces données de votre processeur actuel pour vos dossiers avant de compléter la transition.
</Note>

## Attentes de calendrier

| Phase | Durée |
|-------|----------|
| Configuration initiale et planification | 1-2 jours |
| Transfert de méthode de paiement | ~2 semaines |
| Préparation des données d'abonné | 3-5 jours (en parallèle) |
| Validation et test | 2-3 jours |
| Importation en production | 2-3 jours ouvrables |
| Vérification post-migration | 1-2 jours |

<Tip>
La migration totale prend généralement **3-4 semaines** du début à la fin. Commencer la préparation de vos données d'abonné pendant que le transfert de méthode de paiement est en cours aide à minimiser le calendrier global.
</Tip>
