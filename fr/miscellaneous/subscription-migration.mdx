## Qu'est-ce qu'une migration d'abonnement ?

Une migration d'abonnement transfère vos abonnés existants de votre processeur de paiement hérité vers Dodo Payments, vous offrant une source unique de vérité pour toutes vos données clients. Sans migration, les clients que vous avez acquis jusqu'à présent restent dans votre système hérité, et seuls les nouveaux clients seraient visibles dans Dodo Payments.

Le processus de migration garantit une expérience de facturation fluide pour vos clients : ils n'auront pas besoin de ressaisir leurs informations de paiement ni de subir d'interruption de leurs abonnements.

<Warning>
Actuellement, nous ne prenons en charge la migration que des **jetons de carte**. Apple Pay, Google Pay et autres méthodes de paiement wallet ne peuvent pas être transférés entre processeurs de paiement en raison des restrictions de tokenisation imposées par ces fournisseurs.
</Warning>

## Aperçu du processus de migration

Le processus de migration d'abonnement se compose de trois parties clés :

1. **Transfert de méthode de paiement** — Transfert sécurisé des données sensibles de carte de votre processeur actuel
2. **Transfert de données d'abonné** — Exportation des détails des clients, des plans d'abonnement et des calendriers de facturation
3. **Importation et validation** — Unification des méthodes de paiement et des données d'abonné dans Dodo Payments

<Note>
Pour lancer une migration d’abonnement, contactez notre [équipe d’assistance](mailto:support@dodopayments.com). Elle vous mettra en relation avec un ingénieur Solutions dédié qui vous guidera tout au long du processus.
</Note>

## Exigences

Avant de commencer une migration, assurez-vous de répondre aux exigences suivantes :

<AccordionGroup>
<Accordion title="PCI DSS compliance verification">
Dodo Payments est conforme à la norme PCI DSS Niveau 1. Pour satisfaire aux obligations de conformité, nous ne pouvons recevoir les données de carte que d’un autre processeur de paiement conforme à la norme PCI DSS Niveau 1.

Votre processeur actuel doit fournir :
- Leur attestation de conformité PCI actuelle (AOC), ou leur inscription dans le registre mondial des fournisseurs de services de Visa
- Confirmation qu'ils supportent les exportations sécurisées de données de carte

Consultez notre [Attestation de conformité (AOC)](https://pub-2a89184e4c954b599464449d23f506bb.r2.dev/dodo-payments-pci-dss-aoc.pdf) à partager avec votre processeur actuel.
</Accordion>

<Accordion title="Dodo Payments integration">
Terminez votre intégration Dodo Payments et assurez-vous que tous les nouveaux clients sont dirigés vers le processus de paiement Dodo Payments avant de commencer la migration. Cela évite les incohérences de données pendant la période de transition.
</Accordion>

<Accordion title="Data access authorization">
Assurez-vous de disposer de l’autorisation nécessaire pour demander une exportation de données auprès de votre processeur actuel. Cela nécessite généralement :
- Une vérification du propriétaire du compte
- Une autorisation écrite pour l’accès aux données sensibles
- L’accomplissement des étapes de vérification de sécurité de votre processeur
</Accordion>
</AccordionGroup>

## Guide de migration étape par étape

<Steps>
<Step title="Request payment method export from your current processor">
Contactez l’équipe d’assistance de votre processeur de paiement actuel et soumettez une demande formelle d’exportation complète de vos jetons de carte client.

Vous devrez :
- Fournir une vérification de votre identité et de votre autorisation
- Demander qu'ils encryptent l'exportation en utilisant la clé publique PGP de Dodo Payments (fournie ci-dessous)
- Leur demander de transférer le fichier encrypté directement à notre serveur SFTP sécurisé

<Info>
Les transferts sécurisés s’effectuent directement serveur à serveur via SFTP. Votre ingénieur Solutions vous fournira les identifiants SFTP à transmettre à votre processeur actuel.
</Info>

En moyenne, ce processus prend environ **2 semaines**, bien que les délais varient selon les politiques et les temps de réponse de votre processeur actuel.
</Step>

<Step title="Prepare subscriber data export">
Pendant le transfert de la méthode de paiement, exportez les données de vos abonnés depuis votre système historique. Votre ingénieur Solutions fournira un modèle CSV et vous guidera sur les champs requis.

### Données d'abonné requises

| Field | Description |
|-------|-------------|
| `customer_id` | Votre identifiant client interne |
| `email` | Adresse e-mail du client |
| `name` | Nom complet du client |
| `country` | Code pays à deux lettres (ISO 3166-1 alpha-2) |
| `currency` | Code devise à trois lettres (ISO 4217) |
| `payment_amount` | Montant de l’abonnement dans l’unité monétaire la plus petite |
| `payment_interval` | Fréquence de facturation (mensuelle, annuelle, etc.) |
| `next_billing_date` | Date de facturation suivante prévue (ISO 8601) |
| `card_token` | Jeton unique provenant de votre processeur actuel |

<Warning>
Le champ `card_token` est essentiel : il sert à faire correspondre les enregistrements d’abonnés avec leurs méthodes de paiement pendant l’importation. Assurez-vous qu’il correspond exactement au jeton présent dans l’export des méthodes de paiement.
</Warning>

### Format de données d'exemple

```json
{
  "customers": [
    {
      "customer_id": "cus_abc123",
      "email": "jane.doe@example.com",
      "name": "Jane Doe",
      "country": "US",
      "currency": "USD",
      "payment_amount": 2999,
      "payment_interval": "monthly",
      "next_billing_date": "2025-01-15T00:00:00Z",
      "card_token": "tok_xyz789",
      "metadata": {
        "plan": "pro",
        "signup_source": "website"
      }
    }
  ]
}
```

</Step>

<Step title="Validation and testing">
Avant la date d’importation, notre équipe :

- Vérifier l'intégrité du fichier et l'encryption des données de méthode de paiement
- Valider votre CSV d'abonné pour un format correct
- Effectuer un test d'importation dans notre environnement sandbox
- Identifier et résoudre tout problème de mappage de données

<Check>
Vous recevrez un rapport de validation confirmant que tout est prêt pour l’importation en production.
</Check>
</Step>

<Step title="Production import">
À la date d’importation convenue, notre équipe d’ingénierie :

1. Importer les données de méthode de paiement dans notre coffre-fort de cartes conforme PCI
2. Importer vos données d'abonné
3. Faire correspondre les abonnés à leurs méthodes de paiement en utilisant les tokens de carte
4. Activer tous les abonnements migrés

<Info>
Le délai de traitement habituel est de **2 à 3 jours ouvrés** pour les migrations standard, selon le volume d’abonnés.
</Info>
</Step>

<Step title="Post-migration verification">
Une fois l’importation terminée, vous recevrez :

- Un rapport de migration complet
- Un CSV mappant vos anciens IDs d'abonnement aux nouveaux IDs de Dodo Payments
- Accès à tous les clients migrés dans votre tableau de bord Dodo Payments

Nous vous recommandons :
- De vérifier un échantillon d’abonnements migrés dans votre tableau de bord
- De tester quelques renouvellements d’abonnement en mode bac à sable
- De mettre à jour vos références internes en base de données avec les nouveaux identifiants
- De mettre en place une surveillance parallèle pour le premier cycle de facturation
</Step>
</Steps>

## Données migrables

Les données suivantes peuvent être transférées lors d'une migration d'abonnement :

| Type de données | Supporté | Remarques |
|----------------|----------|----------|
| Numéros de carte de crédit/débit | ✅ | Encrypté via PGP |
| Dates d'expiration de carte | ✅ | |
| Noms des titulaires de carte | ✅ | |
| Adresses de facturation | ✅ | |
| Adresses e-mail des clients | ✅ | |
| Métadonnées des clients | ✅ | Paires clé-valeur personnalisées |
| Détails d'abonnement | ✅ | Plan, montant, intervalle |
| Tokens Apple Pay | ❌ | Ne peuvent pas être transférés |
| Tokens Google Pay | ❌ | Ne peuvent pas être transférés |
| Autres tokens de portefeuille | ❌ | Ne peuvent pas être transférés |
| Historique des paiements | ❌ | Exporter de votre processeur actuel |

<Note>
L’historique des paiements et les enregistrements de transactions ne peuvent pas être migrés. Nous vous recommandons d’exporter ces données auprès de votre processeur actuel pour vos archives avant de finaliser la transition.
</Note>

## Attentes de calendrier

| Phase | Durée |
|-------|----------|
| Configuration initiale et planification | 1-2 jours |
| Transfert de méthode de paiement | ~2 semaines |
| Préparation des données d'abonné | 3-5 jours (en parallèle) |
| Validation et test | 2-3 jours |
| Importation en production | 2-3 jours ouvrables |
| Vérification post-migration | 1-2 jours |

<Tip>
La migration complète prend généralement **3 à 4 semaines** du début à la fin. Commencer la préparation des données d’abonnés pendant que le transfert des méthodes de paiement est en cours contribue à réduire la durée totale.
</Tip>
