### Dépendance de l'Environnement

**Dépendant de l'Environnement :** Toutes les fonctionnalités dépendantes de l'environnement auront des données indépendantes en Mode Test et Mode Live. Les modifications apportées dans un environnement n'affecteront pas l'autre. Par exemple, les clés API générées en Mode Test seront indépendantes des clés API générées en Mode Live et vice versa.

**Indépendant de l'Environnement :** Toutes les fonctionnalités indépendantes de l'environnement auront les mêmes données dans le mode live et le mode test. Par exemple, la section Mon Compte aura les mêmes données en mode test et en mode live. Les modifications apportées dans un environnement affecteront l'autre environnement.

Le mode Live ne sera disponible qu'après la réussite de la vérification d'identité et de la vérification de l'entreprise. Vous pouvez accepter des paiements réels uniquement dans \`live mode\`.

Les noms d'hôtes API pour le mode Test et le mode Live sont ci-dessous :
- Mode Test : [`https://test.dodopayments.com`](https://test.dodopayments.com/)
- Mode Live : [`https://live.dodopayments.com`](https://live.dodopayments.com/) 

## Points Clés

1. Tous les produits créés en mode test ou en mode live ne seront disponibles que dans l'environnement respectif. Il existe une option de copier vers le mode live dans l'environnement de mode test. Cela copiera le produit avec tous les détails pertinents et le créera dans le mode live.

2. Les clés API et les Webhooks créés en mode test seront limités à l'environnement de mode test et n'affecteront pas le déploiement en mode live et vice versa.

3. Tous les documents soumis pour vérification et les détails du compte bancaire soumis seront les mêmes en mode test et en mode live. Un changement dans ces informations dans un environnement changera également dans l'autre environnement.

- Meilleures pratiques pour tester votre intégration

<Note>
Les méthodes de paiement de test ne sont valides qu'en mode test. Ne les utilisez jamais pour des paiements réels.
</Note>
