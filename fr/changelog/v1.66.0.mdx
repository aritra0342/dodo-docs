## Nouvelles fonctionnalités 🚀

1. **`subscription.updated` Webhook**  
   Un nouvel événement webhook qui se déclenche à chaque mise à jour d’un objet d’abonnement, vous offrant des notifications en temps réel sur toute modification de vos abonnements.
   
   Utilisez ce webhook pour :
   - Recevoir des notifications instantanées lorsque les détails d’un abonnement changent
   - Maintenir l’état de votre application synchronisé avec Dodo Payments
   - Réagir aux mises à jour d’abonnement sans interroger l’API en continu
   
   <Tip>
   Abonnez-vous à `subscription.updated` pour éliminer le besoin de sondages et garder votre système synchronisé avec les changements d’abonnement en temps réel.
   </Tip>
   
   En savoir plus : [Subscription Webhooks](/developer-resources/webhooks/intents/subscription)

2. **Support d'adresse minimale dans le paiement**  

   <Frame>
   <img src="/images/changelog/minimal-address.png" alt="Minimal Address Support in Checkout" />
   </Frame>

   Réduisez les frictions de paiement avec le nouveau mode de collecte d’adresse minimale. Au lieu de collecter l’intégralité des informations d’adresse, le mode d’adresse minimale ne recueille que :
   
   - **Pays** : toujours requis pour la détermination des taxes
   - **Code postal** : uniquement collecté dans les régions où cela est nécessaire pour le calcul de la taxe de vente, de la TVA ou de la TPS
   
   <Check>
   Résultat : finalisation de paiement nettement plus rapide avec moins de champs pour les clients.
   </Check>
   
   **Activer l’adresse minimale :**
   
   <Tabs>
   {/* LOCKED_PATTERN_0090a508f27e87d00759ddcd3b9fe94c */}

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [{ product_id: 'prod_123', quantity: 1 }],
     minimal_address: true
   });
   ```

   </Tab>
   
   {/* LOCKED_PATTERN_ed987f4815610e04ce90e6f11a8998dc */}
   Ajoutez `minimalAddress=true` à l’URL de votre lien de paiement :
   

   ```
   https://checkout.dodopayments.com/buy/prod_123?minimalAddress=true
   ```

   </Tab>
   </Tabs>
   
   <Info>
   Par défaut, le paiement continue de collecter les adresses complètes. Nous allons progressivement passer tous les liens de paiement en mode adresse minimale. Activez dès maintenant le drapeau de fonctionnalité pour commencer à bénéficier d’une expérience de paiement plus fluide.
   </Info>

3. **Clés API en lecture seule**  
   Créez des clés API avec des autorisations en lecture seule pour des intégrations plus sûres. Lors de la création d’une nouvelle clé API, décochez **Activer l’accès en écriture** pour générer une clé en lecture seule.
   
   Les clés en lecture seule peuvent :
   - Récupérer les données de paiement, d’abonnement et de client
   - Lister les produits et obtenir leurs détails
   - Accéder aux rapports et à l’analytique
   
   Les clés en lecture seule **ne peuvent pas** :
   - Créer ou modifier des paiements
   - Mettre à jour des abonnements
   - Modifier les configurations de produits
   
   <Tip>
   Décochez « Activer l’accès en écriture » pour les intégrations de tableau de bord, les outils d’analytique et tout système qui n’a besoin que de consulter les données sans les modifier.
   </Tip>

4. **Paiements récurrents UPI AutoPay et abonnements par carte RuPay – Déploiement complet**  
   Les paiements récurrents UPI AutoPay et les abonnements par carte RuPay sont désormais disponibles pour toutes les entreprises. Cela permet les paiements récurrents pour les clients indiens avec :
   
   - **UPI AutoPay** : prélèvements récurrents automatisés via mandat UPI
   - **Cartes RuPay** : prise en charge des abonnements pour le réseau de cartes domestique indien
   
   <Check>
   Résultat : options de paiement d’abonnement étendues pour le marché indien avec une gestion des mandats conforme à la RBI.
   </Check>
   
   En savoir plus : [Subscriptions with RBI-Compliant Mandates](/features/subscription#subscriptions-with-rbi-compliant-mandates)

5. **API d'aperçu des changements de plan**  
   Un nouvel endpoint API pour prévisualiser les changements de plan d'abonnement avant de s'y engager. Obtenez le montant exact de la charge et voyez à quoi ressemblera l'abonnement après le changement - sans apporter de modifications réelles.
   

   ```javascript
   const preview = await client.subscriptions.previewChangePlan('sub_123', {
     product_id: 'prod_pro_plan',
     proration_billing_mode: 'prorated_immediately',
     quantity: 1
   });
   
   console.log('Charge amount:', preview.immediate_charge.summary);
   console.log('New plan details:', preview.new_plan);
   ```

   
   Utilisez ceci pour :
   - Montrer aux clients exactement ce qu'ils paieront avant de confirmer les mises à niveau/dégradations
   - Construire des calculateurs de prix et des outils de comparaison de plans
   - Réduire les tickets de support en améliorant la transparence des prix
   
   En savoir plus : [API d'aperçu des changements de plan](/api-reference/subscriptions/preview-change-plan)
