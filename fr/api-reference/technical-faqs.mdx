<Accordion title="Q1 : Pourquoi n'ai-je pas reçu l'identifiant du produit dans le webhook payment.succeeded pour un produit d'abonnement ?">
  R : L'événement webhook payment.succeeded pour les produits d'abonnement n'inclut pas de product_id. Au lieu de cela, il fournit le `subscription_id` associé à l'abonnement, ainsi que d'autres détails pertinents.
</Accordion>

<Accordion title="Q2 : Comment puis-je vérifier que les webhooks que je reçois proviennent réellement de Dodo Payments et non d'acteurs malveillants ?">
  R : Pour vérifier que les webhooks que vous recevez proviennent réellement de Dodo Payments et non d'acteurs malveillants, suivez ces étapes :
    1. **Vérifiez la signature** : Chaque webhook envoyé par Dodo Payments inclut une signature unique dans l'en-tête **`webhook-signature`**. Cette signature est générée à l'aide de la clé secrète de votre webhook et de la charge utile de l'événement.
    2. **Récupérez la clé secrète** : Assurez-vous d'avoir la clé secrète pour votre webhook, qui peut être trouvée dans votre tableau de bord Dodo Payments sous les paramètres des webhooks.
    3. **Recalculez la signature** : Utilisez le même algorithme de signature (HMAC-SHA256) que Dodo Payments pour calculer la signature sur votre serveur. Utilisez la charge utile du webhook et votre clé secrète de webhook.
    4. **Comparez les signatures** : Comparez la signature recalculée avec celle fournie dans l'en-tête **`webhook-signature`**. Si elles correspondent, le webhook est authentique ; sinon, il doit être rejeté.
    5. **Utilisez HTTPS** : Assurez-vous que votre point de terminaison webhook utilise HTTPS pour sécuriser la communication et prévenir toute falsification lors de la transmission des données.
  En mettant en œuvre ces étapes, vous pouvez vérifier en toute confiance l'authenticité des webhooks envoyés par Dodo Payments.
</Accordion>

<Accordion title="Q3 : Pourquoi reçois-je un webhook subscription.renew même pour le paiement du premier mois ?">
  R : Pour les abonnements, `subscription.renew` sera déclenché chaque fois que le montant de l'abonnement est prélevé. Donc oui, c'est un comportement attendu.
</Accordion>

<Accordion title="Q5 : Comment puis-je tester les webhooks localement ?">
  R : Pour tester les webhooks localement, vous pouvez suivre ces étapes :
  
1. **Utilisez un outil de tunneling** : Des outils comme [ngrok](https://ngrok.com/), [LocalTunnel](https://localtunnel.github.io/www/) ou [Cloudflare Tunnel](https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/) peuvent exposer votre serveur local à Internet. Cela fournit une URL publique que Dodo Payments peut utiliser pour envoyer des événements webhook à votre environnement local.
2. **Configurez le point de terminaison webhook** : Créez un point de terminaison webhook dans votre application locale pour gérer les événements webhook entrants. Assurez-vous qu'il écoute les requêtes HTTP POST.
3. **Configurez le webhook dans Dodo Payments** : Allez dans votre tableau de bord Dodo Payments, naviguez vers les paramètres des webhooks et mettez à jour l'URL du webhook avec l'URL publique générée par l'outil de tunneling.
4. **Déclenchez des événements** : Effectuez des actions dans votre environnement pour déclencher des événements webhook. Par exemple, vous pouvez simuler des paiements ou des renouvellements d'abonnement.
5. **Inspectez les requêtes entrantes** : Utilisez les journaux de l'outil de tunneling ou les journaux de votre application pour inspecter les charges utiles des webhooks entrants. Cela vous permet de vérifier que votre point de terminaison reçoit les événements et les traite correctement.
6. **Vérifiez la signature** : Assurez-vous de valider l'en-tête **`webhook-signature`** pour confirmer que les événements webhook sont authentiques et envoyés par Dodo Payments.

En suivant ces étapes, vous pouvez facilement tester et déboguer les webhooks dans votre environnement de développement local.
</Accordion>
