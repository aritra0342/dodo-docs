## Gestion des échecs de paiement

Utilisez le paramètre `on_payment_failure` pour contrôler ce qui se passe lorsque le paiement du changement de forfait échoue :

| Valeur | Comportement |
|-------|-------------|
| `prevent_change` | Maintient l’abonnement sur le forfait actuel jusqu’à ce que le paiement réussisse. Le changement de forfait reste en attente. |
| `apply_change` | Applique immédiatement le changement de forfait, indépendamment du résultat du paiement. C’est le comportement par défaut. |
