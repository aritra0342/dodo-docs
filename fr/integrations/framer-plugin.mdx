## Aperçu

Le plugin Framer Dodo Payments intègre parfaitement le traitement des paiements dans vos sites Web Framer. Créez des expériences de paiement personnalisables avec un support pour les environnements de test et de production.

<Card title="Plugin Officiel" icon="link" href="https://www.framer.com/marketplace/plugins/dodo-payments">
Accédez au plugin officiel Dodo Payments sur le marché Framer.
</Card>

## Guide d'Installation

<Steps>
<Step title="Générer une Clé API">
Visitez le [Tableau de Bord Dodo Payments](https://app.dodopayments.com/developer/api-keys) et créez une nouvelle clé API.

<Tabs>

<Tab title="Mode En Direct">
Pour une utilisation en production avec un traitement de paiement réel.
</Tab>

<Tab title="Mode Test">
Parfait pour le développement et les tests sans frais réels.
</Tab>

</Tabs>

<Check>
Copiez votre clé API en toute sécurité - vous en aurez besoin pour l'étape suivante.
</Check>
</Step>

<Step title="Authentifier le Plugin">
1. Ouvrez le plugin Dodo Payments dans Framer
2. Accédez à la page d'authentification
3. Collez votre clé API
4. Sélectionnez le mode approprié (Test ou En Direct)
5. Cliquez sur **Continuer** pour authentifier

<Warning>
Gardez vos clés API en sécurité et ne les partagez jamais publiquement.
</Warning>
</Step>

<Step title="Gérer les Produits">
Après l'authentification, vous verrez votre liste de produits :

- **Produits existants** : Affichés automatiquement
- **Aucun produit** : Cliquez sur le bouton **Plus (+)** à côté de 'Produits'

<Info>
Vous serez redirigé vers le Tableau de Bord Dodo Payments pour créer des produits avec des prix, des descriptions et des paramètres.
</Info>
</Step>

<Step title="Ajouter des boutons d'achat">
1. Sélectionnez un produit dans votre liste disponible
2. Cliquez sur le bouton `Add Buy Button`
3. Un bouton d'achat avec le texte `Acheter {product_name}` sera ajouté à votre toile

<Warning>
Le bouton Ajouter un Bouton d'Achat sera désactivé si vous n'avez pas les autorisations de projet appropriées.
</Warning>
</Step>
</Steps>

## Personnalisation

<Steps>
<Step title="Sélectionner l'Expérience de Paiement">
    Choisissez votre expérience de paiement préférée :
    - **Superposition (Intégrée)** : Le paiement apparaît comme une superposition dans votre projet Framer, gardant les utilisateurs sur votre site.
    - **Hébergé** : Les utilisateurs sont redirigés vers une page de paiement hébergée par Dodo Payments pour une expérience simplifiée.
</Step>

<Step title="Personnaliser les Options d'Apparence">
    Personnalisez votre expérience de paiement avec les paramètres d'apparence suivants :
    - **Thème de Paiement** : Choisissez entre des thèmes clairs ou sombres pour correspondre à l'esthétique de votre marque.
    - **Étiquette du Bouton** : Personnalisez le texte (par exemple, 'Acheter Maintenant', 'Acheter', 'Commencer').
    - **Styles de Bouton** : Ajustez les couleurs, les polices, les tailles et d'autres propriétés de style.
    - **URL de Retour** : Définissez une page Framer pour que l'utilisateur revienne après le paiement.
</Step>
</Steps>

## Dépannage

<AccordionGroup>
<Accordion title="Problèmes d'Authentification">
**Problème** : Le plugin ne s'authentifie pas

**Solutions** :
- Vérifiez que votre clé API est correcte
- Assurez-vous que vous utilisez le bon mode (Test/En Direct)
- Vérifiez votre connexion Internet
</Accordion>

<Accordion title="Problèmes de Création de Bouton">
**Problème** : Ajouter un Bouton d'Achat est désactivé

**Solutions** :
- Vérifiez que vous avez les autorisations appropriées dans votre projet Framer
- Assurez-vous que vous êtes correctement authentifié
- Essayez de rafraîchir le plugin
</Accordion>

<Accordion title="Problèmes d'Affichage des Produits">
**Problème** : Les produits ne s'affichent pas

**Solutions** :
- Assurez-vous d'avoir créé des produits dans votre tableau de bord Dodo Payments
- Vérifiez que votre clé API a les bonnes autorisations
- Essayez de vous réauthentifier
</Accordion>
</AccordionGroup>
