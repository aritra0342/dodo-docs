---
title: Soumettez votre projet
description: Comment ajouter votre projet open-source au répertoire de la
  communauté Dodo Payments.
icon: plus
---

<Steps>
{/* LOCKED_PATTERN_b8e49197111d4f91963d876b7d3cbd9b */}
- Le projet est open-source avec une licence permissive.
- Il est pertinent pour Dodo Payments (SDK, plugin, intégration, modèle).
- Il dispose d’un README avec l’installation, l’utilisation et l’état de maintenance.
</Step>

{/* LOCKED_PATTERN_29bfb0d9a10f5f1cc614e31786e69c52 */}
Modifiez `community/projects.mdx` et ajoutez un nouveau `<Card>` sous l’onglet le plus pertinent.

```mdx
<Card title="PROJECT_NAME (LANG)" href="REPO_URL" icon="code">
  One-sentence description of your project.
  <br/>By <a href="GITHUB_PROFILE_URL">@github_username</a>
  <br/><br/>`LANGUAGE` `FRAMEWORK` `CATEGORY`
</Card>
```

</Step>

{/* LOCKED_PATTERN_5390269a28150786d344085d66aa0ef8 */}
Si vous préférez, ouvrez une issue à l’aide du modèle « Community Project Submission » avec :
- URL du dépôt, description, catégorie (SDK/Plugin/Boilerplate)
- Langage/cadre principal
- Lien vers un extrait d’installation/d’utilisation
- État de maintenance et dernière mise à jour
</Step>

{/* LOCKED_PATTERN_03e556e84db70a2199e5921d65cd601a */}
Nous examinons la pertinence, la clarté et la sécurité. Les contributions approuvées sont fusionnées et publiées.
</Step>
</Steps>

<Warning>
Nous pouvons supprimer les entrées non maintenues, non sécurisées, trompeuses ou contraires à la politique.
</Warning>
