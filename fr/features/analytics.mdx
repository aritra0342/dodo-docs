Suivez les parcours clients depuis votre vitrine jusqu'à la finalisation de l'achat. Dodo Payments s’intègre à Google Analytics 4, Google Tag Manager et Meta Pixel pour offrir une analyse complète sur toutes vos pages hébergées.

<Info>
Le suivi analytique est activé automatiquement sur votre vitrine, vos pages de paiement et votre portail client dès que vous ajoutez vos identifiants de suivi dans le tableau de bord.
</Info>

<Warning>
Les bloqueurs de publicités dans les navigateurs des clients peuvent empêcher le suivi analytique pour certains visiteurs de votre boutique.
</Warning>

## Premiers pas

<Steps>
<Step title="Open Business Settings">
  Dans votre tableau de bord Dodo Payments, allez dans **Paramètres** et faites défiler jusqu'à la section **Analytics**.
</Step>

<Step title="Enable Your Tracking Platform">
  Activez la plateforme analytique que vous souhaitez utiliser et saisissez votre identifiant de suivi :
  - **Google Analytics** : saisissez votre ID de mesure (format : `G-XXXXXXXXXX`)
  - **Google Tag Manager** : saisissez votre ID de conteneur (format : `GTM-XXXXXXX`)
  - **Meta Pixel** : saisissez votre ID de pixel (numérique, 10 à 20 chiffres)
</Step>

<Step title="Save Your Settings">
  Cliquez sur **Enregistrer** pour appliquer votre configuration de suivi.
</Step>

<Step title="Verify Tracking">
  Rendez-vous sur votre vitrine ou créez un paiement test pour vérifier que les événements sont bien suivis dans votre tableau de bord analytique.
</Step>
</Steps>

## Google Analytics 4

[Google Analytics 4](https://marketingplatform.google.com/about/analytics/) fournit une analyse web complète avec des capacités de suivi e-commerce. Grâce à l’intégration Dodo Payments, vous pouvez suivre le parcours client complet, de la navigation sur votre boutique à la finalisation de l’achat.

### Ajoutez votre ID de mesure Google Analytics

1. Dans Google Analytics, allez dans **Admin** → **Flux de données** → sélectionnez votre flux web
2. Copiez l’**ID de mesure** (commence par `G-`)
3. Dans le tableau de bord Dodo Payments, allez dans **Paramètres** → **Analytics**
4. Activez **Google Analytics** et collez votre ID de mesure
5. Cliquez sur **Enregistrer**

<Tip>
Si vous n’avez pas encore de propriété Google Analytics, suivez le [guide de configuration de Google](https://support.google.com/analytics/answer/9304153) pour en créer une.
</Tip>

### Événements suivis dans Google Analytics

Nous suivons les événements suivants sur votre vitrine et vos pages de paiement hébergées. Les données des événements, y compris les informations produits, les totaux de transaction et la devise, sont envoyées avec chaque événement.

#### Événements sur la vitrine

| Événement | Description |
|-----------|-------------|
| `view_item_list` | Se déclenche au chargement de la vitrine avec les produits |
| `select_item` | Se déclenche lorsqu’un client clique sur un produit pour lancer le paiement |

#### Événements de paiement

| Événement | Description |
|-----------|-------------|
| `view_item` | Se déclenche lorsque la page de paiement se charge |
| `view_cart` | Se déclenche au chargement du paiement et lorsqu’une variante est sélectionnée |
| `apply_discount`* | Se déclenche lorsqu’un code de réduction valide est appliqué avec succès |
| `remove_discount`* | Se déclenche lorsqu’un code de réduction est supprimé |
| `add_payment_info` | Se déclenche lorsque le client soumet le formulaire de paiement |
| `purchase` | Se déclenche lorsque le paiement est réussi |

<Note>
`apply_discount` et `remove_discount` sont des événements personnalisés non inclus dans les [événements e-commerce standard de Google Analytics](https://support.google.com/analytics/answer/9267735).
</Note>

### Utilisation de GA4 avec l’overlay/l’inline checkout

Pour suivre les événements Google Analytics dans un paiement en overlay ou inline intégré à votre propre site, vous devez installer le code de suivi Google Analytics sur la page où le paiement est intégré.

Une fois installé, tous les événements de suivi listés ci-dessus seront envoyés automatiquement à votre compte Google Analytics, sans configuration supplémentaire.

## Google Tag Manager

[Google Tag Manager](https://marketingplatform.google.com/about/tag-manager/) vous permet de gérer les balises marketing et analytiques sans modifier le code. Les balises déployées via GTM seront déclenchées sur toutes les pages hébergées par Dodo Payments.

### Ajoutez votre ID de conteneur GTM

1. Dans Google Tag Manager, copiez votre **ID de conteneur** (commence par `GTM-`)
2. Dans le tableau de bord Dodo Payments, allez dans **Paramètres** → **Analytics**
3. Activez **Google Tag Manager** et collez votre ID de conteneur
4. Cliquez sur **Enregistrer**

<Info>
Lorsque vous utilisez GTM, vous pouvez configurer des suivis supplémentaires (comme les conversions Google Ads) directement dans votre conteneur GTM. L’intégration Dodo Payments injecte à la fois le script GTM et la balise noscript de secours pour une compatibilité maximale.
</Info>

### Utilisation de GTM avec l’overlay/l’inline checkout

Pour les paiements en overlay ou inline intégrés à votre site, installez GTM sur votre propre site. Le paiement héritera de votre configuration GTM et déclenchera les événements depuis le cadre intégré.

## Meta Pixel

[Meta Pixel](https://www.facebook.com/business/tools/meta-pixel/) suit les actions client sur votre boutique pour mesurer l’efficacité de vos campagnes publicitaires Facebook et Instagram. Suivez les conversions du clic publicitaire jusqu’à la finalisation de l’achat.

### Ajoutez votre ID Meta Pixel

1. Dans le [Meta Events Manager](https://www.facebook.com/events_manager2/), créez ou sélectionnez votre pixel
2. Copiez l’**ID de pixel** (numérique, généralement 15 à 16 chiffres)
3. Dans le tableau de bord Dodo Payments, allez dans **Paramètres** → **Analytics**
4. Activez **Meta Pixel** et collez votre ID de pixel
5. Cliquez sur **Enregistrer**

<Tip>
Suivez le [guide d’installation du pixel Meta](https://www.facebook.com/business/help/952192354843755) si vous n’avez pas encore créé de pixel.
</Tip>

### Événements suivis avec Meta Pixel

Nous suivons les événements suivants qui apparaissent dans votre [Meta Events Manager](https://www.facebook.com/events_manager2/). Les données des événements, y compris les identifiants de contenu, les valeurs et la devise, sont envoyées avec chaque événement.

<Note>
Les événements peuvent ne pas apparaître immédiatement dans Meta Events Manager - il peut y avoir un délai de quelques minutes.
</Note>

#### Événements sur la vitrine

| Événement | Description |
|-----------|-------------|
| `PageView` | Se déclenche au chargement de la vitrine |
| `ViewContent` | Se déclenche lorsque la vitrine se charge avec les produits |
| `InitiateCheckout` | Se déclenche lorsqu’un client clique sur un produit pour lancer le paiement |

#### Événements de paiement

| Événement | Description |
|-----------|-------------|
| `PageView` | Se déclenche lorsque la page de paiement se charge |
| `AddToCart` | Se déclenche au chargement du paiement et lorsqu’une variante est sélectionnée |
| `ApplyDiscount`* | Se déclenche lorsqu’un code de réduction valide est appliqué avec succès |
| `RemoveDiscount`* | Se déclenche lorsqu’un code de réduction est supprimé |
| `AddPaymentInfo` | Se déclenche lorsque le client soumet le formulaire de paiement |
| `Purchase` | Se déclenche lorsque le paiement est réussi |

<Note>
`ApplyDiscount` et `RemoveDiscount` sont des événements personnalisés non inclus dans les [événements standard de Meta](https://developers.facebook.com/docs/meta-pixel/reference#standard-events).
</Note>

## Référence complète des événements

Ce tableau montre tous les événements suivis à la fois par Google Analytics 4 et Meta Pixel :

| Déclencheur | Événement GA4 | Événement Meta Pixel | Emplacement |
|------------|---------------|----------------------|-------------|
| Chargement de la page vitrine | `view_item_list` | `ViewContent` | Vitrine |
| Clic sur un produit par un client | `select_item` | `InitiateCheckout` | Vitrine |
| Chargement de la page de paiement | `view_item` | `AddToCart` | Paiement |
| Visualisation du panier/sélection de variante | `view_cart` | - | Paiement |
| Application d’un code de réduction | `apply_discount` | `ApplyDiscount` | Paiement |
| Suppression d’un code de réduction | `remove_discount` | `RemoveDiscount` | Paiement |
| Soumission du formulaire de paiement | `add_payment_info` | `AddPaymentInfo` | Paiement |
| Paiement réussi | `purchase` | `Purchase` | Paiement |

## Couverture par plateforme

Le suivi analytique est disponible sur différentes interfaces Dodo Payments :

| Interface | Google Analytics 4 | Google Tag Manager | Meta Pixel |
|-----------|-------------------|-------------------|------------|
| Vitrine | Oui | Oui | Oui |
| Paiement (hébergé) | Oui | Oui | Oui |
| Paiement (Overlay/Inline)* | Oui | Oui | Oui |
| Portail client | Oui | Oui | Non |

<Info>
*Pour les paiements overlay et inline, installez le code de suivi sur votre propre site pour une attribution complète. Les événements provenant du paiement intégré seront suivis automatiquement.
</Info>

## Données d’événement incluses

Chaque événement de suivi inclut les données e-commerce pertinentes pour vous aider à analyser le comportement client :

### Données produit
- ID du produit
- Nom du produit
- Prix
- Devise
- Quantité

### Données de transaction (événements d’achat)
- ID de transaction/paiement
- Montant total
- Montant de la taxe
- Devise
- Réduction appliquée

### Données client
- ID client (pour les clients répétés)
- Email (haché pour la confidentialité lorsque c’est applicable)

## Bonnes pratiques

<AccordionGroup>
<Accordion title="Test Your Integration">
Après avoir ajouté vos identifiants de suivi, rendez-vous sur votre vitrine et réalisez un paiement test pour vérifier que les événements apparaissent dans vos tableaux de bord analytiques. Utilisez les rapports en temps réel de Google Analytics ou la fonction Événements test de Meta Events Manager.
</Accordion>

<Accordion title="Use UTM Parameters">
Ajoutez des paramètres UTM à vos liens marketing pour suivre les campagnes qui génèrent des conversions. Dodo Payments conserve les paramètres UTM tout au long du parcours de paiement.
</Accordion>

<Accordion title="Set Up Conversion Goals">
Dans Google Analytics, créez des événements de conversion pour `purchase` afin de suivre votre taux de conversion. Dans Meta Ads Manager, utilisez l’événement `Purchase` comme objectif d’optimisation de vos campagnes.
</Accordion>

<Accordion title="Enable Enhanced E-commerce">
Si vous utilisez Google Tag Manager, envisagez de mettre en place des rapports Enhanced E-commerce pour une analyse détaillée des entonnoirs et des performances produits.
</Accordion>
</AccordionGroup>

## Dépannage

<AccordionGroup>
<Accordion title="Events not appearing in Google Analytics">
- Vérifiez que votre ID de mesure est correct (format : `G-XXXXXXXXXX`)
- Consultez les rapports en temps réel de Google Analytics — les événements y apparaissent en premier
- Assurez-vous que l’option est activée dans vos paramètres Dodo Payments
- Attendez quelques minutes car il peut y avoir des délais de traitement
- Testez en fenêtre de navigation privée pour écarter les bloqueurs de publicités
</Accordion>

<Accordion title="Events not appearing in Meta Events Manager">
- Vérifiez que votre ID de pixel est correct (numérique, 10 à 20 chiffres)
- Utilisez l’extension Meta Pixel Helper pour déboguer
- Consultez l’onglet Événements test dans Events Manager
- Les événements peuvent mettre plusieurs minutes à apparaître
- Désactivez les bloqueurs de publicités pendant les tests
</Accordion>

<Accordion title="GTM tags not firing">
- Confirmez que votre ID de conteneur est correct (format : `GTM-XXXXXXX`)
- Utilisez le mode Aperçu de GTM pour déboguer le déclenchement des balises
- Vérifiez que les déclencheurs sont configurés correctement dans votre conteneur GTM
- Assurez-vous que le conteneur est publié (et pas uniquement en brouillon)
</Accordion>

<Accordion title="Overlay/Inline checkout not tracking">
- Installez d’abord le code de suivi sur votre propre site
- Assurez-vous que le suivi se charge avant l’ouverture de l’overlay de paiement
- Consultez la console du navigateur pour détecter les erreurs JavaScript
- Vérifiez que le suivi inter-domaines est configuré si vous utilisez GA4
</Accordion>

<Accordion title="Purchase events showing wrong amounts">
- Les montants sont envoyés dans la plus petite unité monétaire (centimes)
- Vérifiez que votre outil analytique est configuré pour afficher la bonne devise
- Vérifiez si la taxe est incluse ou exclue comme prévu
</Accordion>
</AccordionGroup>
