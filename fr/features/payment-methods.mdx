## Comment cela fonctionne-t-il ?

Les méthodes de paiement sont automatiquement présentées aux clients en fonction de :

- **Localisation du client** — Les méthodes régionales comme iDEAL (Pays-Bas) ou UPI (Inde) n'apparaissent que pour les clients dans ces régions
- **Monnaie de transaction** — Les méthodes de paiement sont filtrées par les monnaies supportées
- **Type de dispositif** — Apple Pay n'apparaît que sur les dispositifs et navigateurs Apple compatibles
- **Type de transaction** — Certaines méthodes ne prennent pas en charge les abonnements ou ont des montants minimums

<Info>
**Activation automatique :** Toutes les méthodes de paiement sont préconfigurées et gérées par Dodo Payments. Aucune configuration supplémentaire n'est nécessaire — elles deviennent disponibles dès que votre compte est vérifié et que le mode production est activé.
</Info>

## Configurer les méthodes de paiement

Par défaut, toutes les méthodes de paiement éligibles apparaissent automatiquement. Utilisez `allowed_payment_method_types` pour restreindre les méthodes affichées :

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://test.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

### Types de méthodes disponibles

| Type | Description |
| :--- | :---------- |
| `credit` | Cartes de crédit (Visa, Mastercard, Amex, etc.) |
| `debit` | Cartes de débit |
| `apple_pay` | Apple Pay |
| `google_pay` | Google Pay |
| `amazon_pay` | Amazon Pay |
| `cashapp` | Cash App Pay |
| `klarna` | Klarna BNPL |
| `afterpay_clearpay` | Afterpay / Clearpay |
| `upi_collect` | UPI Collect (QR code) |
| `ideal` | iDEAL (Pays-Bas) |
| `bancontact_card` | Bancontact (Belgique) |
| `eps` | EPS (Autriche) |
| `multibanco` | Multibanco (Portugal) |
| `revolut_pay` | RevolutPay |
| `billie` | Billie B2B BNPL |

<Warning>
Incluez toujours `credit` et `debit` en tant que solutions de secours. Si toutes les méthodes spécifiées sont indisponibles pour un client, le paiement échouera.
</Warning>

## Bonnes pratiques

<AccordionGroup>
{/* LOCKED_PATTERN_63da960e31e014d866efe70101f28b99 */}
Incluez les méthodes de paiement régionales pour vos marchés cibles. iDEAL représente ~60 % des paiements en ligne aux Pays-Bas ; l'UPI domine en Inde. Omettre ces méthodes nuit considérablement à la conversion.
</Accordion>

{/* LOCKED_PATTERN_78f59c035688fa14218708818c9f0fca */}
Même lorsque vous ciblez des méthodes spécifiques, incluez `credit` et `debit` en tant que solutions de secours. Tous les clients n'ont pas accès à chaque méthode de paiement.
</Accordion>

{/* LOCKED_PATTERN_564640b27644e47b9e4e7faaec6c2d88 */}
Les méthodes BNPL (Klarna, Afterpay) exigent un minimum de 50,01 $. Ne les incluez pas pour des produits de faible valeur — elles n'apparaîtront tout simplement pas, ce qui est acceptable.
</Accordion>

{/* LOCKED_PATTERN_ea5461a635a5291807627a6b40d7922e */}

## Dépannage

<AccordionGroup>
{/* LOCKED_PATTERN_fbf2d38e7725fb97384bef01324b59a1 */}
**Vérifiez ces éléments :**
1. La méthode est-elle incluse dans `allowed_payment_method_types` ?
2. L'emplacement du client correspond-il à la région prise en charge par la méthode ?
3. La devise de la transaction est-elle prise en charge par la méthode ?
4. Le montant de la transaction respecte-t-il le minimum (par exemple, 50,01 $ pour BNPL) ?
5. Pour les abonnements : la méthode prend-elle en charge les paiements récurrents ?

**Solution :** Supprimez temporairement `allowed_payment_method_types` pour voir toutes les méthodes éligibles, puis réduisez l'origine du problème.
</Accordion>

{/* LOCKED_PATTERN_57385d59bd851fb1e8c26266b489efd4 */}
**Causes possibles :**
- `allowed_payment_method_types` n'est pas transmis correctement
- Paramètre mal formaté (doit être un tableau de chaînes)


**Solution :** Enregistrez la charge utile de votre requête API et vérifiez la structure du paramètre.
</Accordion>

{/* LOCKED_PATTERN_6d1a1b1a59cbe2789bbc9434eb1ec153 */}
**Vérifiez :**
- L'adresse de facturation du client correspond-elle à la région de la méthode
- La devise est-elle définie correctement (par exemple, EUR pour iDEAL)
- La devise adaptive est-elle activée si le client est international

**Solution :** Vérifiez le code pays de l'adresse de facturation et la devise dans votre requête API.
</Accordion>

{/* LOCKED_PATTERN_52f3532b80d2def876ef3cc78b55f9f6 */}
Les abonnements UPI et Rupay ont des exigences spécifiques, notamment des délais de traitement de 48 heures et des limites de mandat de la RBI.

**Consultez :** <a href="/features/payment-methods/india">India Payment Methods</a> pour des instructions complètes.
</Accordion>
</AccordionGroup>
