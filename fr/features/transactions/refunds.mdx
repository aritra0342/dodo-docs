## Initier un Remboursement

Vous pouvez initier deux types de remboursements depuis la Page des Détails de Paiement :

- **Remboursement complet** : Rembourse le montant total du paiement au client.
- **Remboursement partiel** : Rembourse uniquement une partie du montant du paiement, comme spécifié par vous.

Pour commencer, cliquez sur le bouton **Initier un Remboursement**. Le remboursement ne sera effectué que si toutes les règles de remboursement sont respectées.

</Frame>

<Tip>
Les remboursements partiels sont utiles lorsque seule une partie de la commande doit être remboursée.
</Tip>

### Détails Requis pour le Remboursement

* **Montant du Remboursement** : Le montant à rembourser. Pour les remboursements partiels, entrez le montant spécifique que vous souhaitez retourner au client. Le système vous empêchera de rembourser plus que le paiement original.

* **Raison du Remboursement** : La raison du remboursement, à enregistrer dans le système.

### Règles de Remboursement

1. Le paiement doit avoir été réussi.

2. Les remboursements (complets ou partiels) doivent être initiés dans les 30 jours suivant la date de la transaction. L'initiation de remboursements ne sera pas autorisée après cela.

3. Les remboursements sont émis sur le mode de paiement original utilisé par le client.

Si une règle est violée, la demande de remboursement sera refusée. Veuillez contacter le support pour obtenir de l'aide si nécessaire.

## Statut du Remboursement

Suivez l'avancement de votre remboursement grâce aux indicateurs de statut suivants :

| Statut    | Description                                                         | Raison possible                                                                                      |
| --------- | ------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| réussi    | Le remboursement a été traité avec succès.                          | —                                                                                                    |
| en attente| Le remboursement est en cours de traitement et attend d'être complété.| —                                                                                                    |
| échoué    | Le remboursement n'a pas pu être traité en raison d'une erreur.     | <ul><li>Paiement contesté</li><li>Solde insuffisant dans le portefeuille</li><li>Problème bancaire</li></ul>     |

## Page des Remboursements

La **Page des Remboursements** fournit une vue détaillée de toutes les transactions de remboursement :&#x20;

* En cliquant sur l'**ID de Paiement Associé**, le commerçant sera dirigé vers la **Page des Détails de Paiement** pour cette transaction.

* Les commerçants peuvent exporter le rapport de remboursement au format CSV pour une plage de dates spécifiée.

## Reçu de Remboursement

Une fois qu'un remboursement est traité avec succès, un **Reçu de Remboursement** est automatiquement envoyé au client.

**Modèle de Reçu de Remboursement :**

<Frame>
  <img src="/images/transactions/refunds/refund-receipt-template.png" width="50%" alt="Modèle de reçu de remboursement montrant les détails du remboursement" />
</Frame>

Le reçu de remboursement comprend :

- **ID de Remboursement** : L'identifiant unique du remboursement.
- **Montant remboursé** : Le montant remboursé au client.
- **Date de remboursement** : La date à laquelle le remboursement a été traité.
- **Méthode de remboursement** : La méthode utilisée pour rembourser le client.
- **Statut du remboursement** : Le statut du remboursement.
- **Raison du remboursement** : La raison du remboursement.
