**Pay What You Want (PWYW)** permet aux clients de décider combien ils paient—au moins le minimum défini par le commerçant. Cela augmente l'engagement, encourage le soutien communautaire et offre une tarification flexible pour les produits numériques.

## Qu'est-ce que Pay What You Want ?

PWYW est un modèle de tarification où les clients fixent leur propre prix dans les limites que vous définissez. Il est idéal pour :

- **Produits et médias numériques** : Modèles, e‑books, cours, plugins
- **Lancements expérimentaux** : Accès anticipé, versions bêta, promotions limitées dans le temps
- **Bundles et extras** : Ajoutez un niveau PWYW pour augmenter la participation

## Avantages clés

- **Tarification dirigée par le client** : Les acheteurs choisissent ce qu'ils paient
- **Plancher de revenus** : Imposer un minimum pour protéger les marges
- **Attentes ancrées** : Prix suggéré optionnel pour guider les choix
- **Paiement sans friction** : Paiement et reçus hébergés gérés pour vous
- **Orienté développeur** : APIs claires pour la création, les remboursements et les webhooks

<Warning>
PWYW est disponible uniquement pour les produits à **paiement unique** (ponctuel).
</Warning>

## Créer Pay What You Want

Créez un produit unique et activez PWYW dans votre tableau de bord Dodo Payments. Les sections ci-dessous correspondent directement au formulaire de création.

### Création de produit

#### Détails du produit

- **Nom du produit** (obligatoire) : Nom affiché lors du paiement et sur les factures
- **Description du produit** (obligatoire) : Déclaration de valeur claire que les clients voient avant de payer
- **Image du produit** (obligatoire) : PNG/JPG/WebP jusqu'à 3 Mo
- **Marque** : Associez le produit à une marque spécifique pour le thème et les e-mails
- **Catégorie fiscale** (obligatoire) : Choisissez la bonne catégorie pour déterminer les règles fiscales

<Tip>
Choisissez la catégorie fiscale la plus précise pour garantir une collecte correcte selon la région.
</Tip>

#### Tarification

- **Type de tarification** : Choisissez <b>Paiement Unique</b>
- **Pay What You Want** (toggle) : Activez pour permettre aux acheteurs de choisir le montant
- **Prix minimum** (obligatoire) : Montant le plus bas qu'un client peut payer
- **Prix suggéré** (optionnel) : Montant recommandé affiché sur la page produit

<Tip>
Utilisez les métadonnées pour stocker des identifiants de votre système (par exemple, orderId) afin de pouvoir rapprocher les événements et les reçus ultérieurement.
</Tip>

## Cas d'utilisation courants

- **Actifs numériques** : Thèmes, modèles, ensembles d'icônes, médias ou plugins
- **Contenu et éducation** : E‑books, cours, ateliers avec tarification pour les soutiens
- **Communauté & dons** : Collectes de fonds, parrainages open-source, soutien aux créateurs
- **Expériences de lancement** : Accès bêta et campagnes limitées dans le temps pour évaluer la demande

## Exemples d'intégration

### Sessions de paiement (paiement unique, PWYW)
Ajoutez un produit PWYW au panier. Le paiement collecte le montant de l'acheteur (respectant votre minimum) — aucun champ supplémentaire n'est requis.

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    { product_id: 'prod_pwyw_demo', quantity: 1 }
  ],
  return_url: 'https://yoursite.com/checkout/success'
});
```

## Meilleures pratiques

- **Choisissez un minimum raisonnable** : Couvrez les coûts tout en facilitant la participation
- **Utilisez un prix suggéré** : Ancrez les attentes pour améliorer la valeur moyenne des commandes
- **Expliquez clairement la valeur** : Dites aux acheteurs ce que leur contribution débloque
- **Testez des campagnes A/B** : Limitez dans le temps les offres et expérimentez avec le message
- **Surveillez la distribution** : Suivez les montants payés pour affiner le minimum et le prix suggéré

<Info>
PWYW est une manière flexible d’attirer plus de clients tout en conservant un plancher de revenu. Commencez simplement, testez rigoureusement et itérez en fonction des conversions, de la répartition des contributions et des taux de remboursement.
</Info>
