Les paiements par carte sont la base des paiements en ligne, acceptés dans le monde entier et faisant confiance aux clients du monde entier. Dodo Payments prend en charge tous les principaux réseaux de cartes avec une protection contre la fraude intégrée et la conformité PCI.

## Configuration

Utilisez ces valeurs dans `allowed_payment_method_types`:

| Type | Description |
| :--- | :---------- |
| `credit` | Toutes les cartes de crédit |
| `debit` | Toutes les cartes de débit |

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: ['credit', 'debit'],
  return_url: 'https://example.com/success'
});
```

<Tip>
Incluez à la fois `credit` et `debit` à moins que vous n'ayez une raison spécifique d'exclure l'un d'eux. Les cartes de débit sont préférées par de nombreux clients et ont souvent des frais moins élevés.
</Tip>

## Authentification 3D Secure

3D Secure (3DS) ajoute un niveau d'authentification qui réduit la fraude et les rétrofacturations en vérifiant l'identité du titulaire de la carte.

### Quand 3DS est-il Déclenché

3DS est automatiquement déclenché lorsque :
- Exigé par le réseau de cartes
- Exigé par les réglementations régionales (par ex., PSD2 en Europe)
- La transaction est signalée comme à haut risque

### Forcer 3DS

Vous pouvez exiger 3DS sur toutes les transactions :

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  force_3ds: true,
  return_url: 'https://example.com/success'
});
```

<Note>
Activer 3DS pour toutes les transactions réduit la fraude mais peut légèrement diminuer le taux de conversion, car certains clients abandonnent lors de l'authentification.
</Note>

Les clients peuvent enregistrer leurs cartes pour des paiements plus rapides lors de futurs achats.

### Activer les Cartes Enregistrées

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  show_saved_payment_methods: true,
  customer: { customer_id: 'cus_existing_123' },
  return_url: 'https://example.com/success'
});
```

### Achats en Un Clic

```javascript
// Get customer's saved payment methods
const methods = await client.customers.getPaymentMethods('cus_123');

// Use saved card for instant checkout
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  customer: { customer_id: 'cus_123' },
  payment_method_id: methods[0].payment_method_id,
  confirm: true,
  return_url: 'https://example.com/success'
});
```

## Bonnes Pratiques

<AccordionGroup>
<Accordion title="Accepter tous les principaux réseaux">
Ne limitez pas les types de cartes sauf si nécessaire. Les clients s'attendent à ce que leur carte préférée fonctionne.
</Accordion>

<Accordion title="Afficher les logos des cartes">
Afficher les logos Visa, Mastercard, Amex lors de votre paiement pour établir la confiance.
</Accordion>

<Accordion title="Gérer les refus avec tact">
Afficher des messages d'erreur clairs. Ne pas exposer de codes d'erreur bruts aux clients.
</Accordion>

<Accordion title="Activer les cartes enregistrées pour les clients réguliers">
Les méthodes de paiement enregistrées augmentent considérablement le taux de conversion pour les achats répétés.
</Accordion>
</AccordionGroup>

## Dépannage

<AccordionGroup>
<Accordion title="Carte refusée">
**Causes :** Fonds insuffisants, carte expirée, CVV incorrect, protection contre la fraude de la banque.

**Solution :** Demander au client de vérifier les détails ou d'essayer une autre carte.
</Accordion>

<Accordion title="Échec de l'authentification 3DS">
**Causes :** Client abandonné, système bancaire indisponible, délai d'attente.

**Solution :** Réessayer ou demander au client de contacter sa banque.
</Accordion>

<Accordion title="Carte non supportée">
**Causes :** Carte régionale non supportée, limitations pour les prépayées.

**Solution :** Le client devrait essayer une autre carte d'un réseau majeur.
</Accordion>
</AccordionGroup>
