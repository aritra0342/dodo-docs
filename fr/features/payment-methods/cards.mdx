Les paiements par carte sont la base des paiements en ligne, acceptés dans le monde entier et faisant confiance aux clients du monde entier. Dodo Payments prend en charge tous les principaux réseaux de cartes avec une protection contre la fraude intégrée et la conformité PCI.

## Configuration

Utilisez ces valeurs dans `allowed_payment_method_types`:

| Type | Description |
| :--- | :---------- |
| `credit` | Toutes les cartes de crédit |
| `debit` | Toutes les cartes de débit |

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: ['credit', 'debit'],
  return_url: 'https://example.com/success'
});
```

<Tip>
Incluez à la fois `credit` et `debit` sauf que vous ayez une raison spécifique d'en exclure un. Les cartes de débit sont préférées par de nombreux clients et ont souvent des frais plus faibles.
</Tip>

## Authentification 3D Secure

3D Secure (3DS) ajoute un niveau d'authentification qui réduit la fraude et les rétrofacturations en vérifiant l'identité du titulaire de la carte.

### Quand 3DS est-il Déclenché

3DS est automatiquement déclenché lorsque :
- Exigé par le réseau de cartes
- Exigé par les réglementations régionales (par ex., PSD2 en Europe)
- La transaction est signalée comme à haut risque

### Forcer 3DS

Vous pouvez exiger 3DS sur toutes les transactions :

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  force_3ds: true,
  return_url: 'https://example.com/success'
});
```

<Note>
Activer 3DS pour toutes les transactions réduit la fraude mais peut légèrement diminuer la conversion car certains clients abandonnent lors de l'authentification.
</Note>

## Méthodes de Paiement Enregistrées

Les clients peuvent enregistrer leurs cartes pour des paiements plus rapides lors de futurs achats.

### Activer les Cartes Enregistrées

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  show_saved_payment_methods: true,
  customer: { customer_id: 'cus_existing_123' },
  return_url: 'https://example.com/success'
});
```

### Achats en Un Clic

```javascript
// Get customer's saved payment methods
const methods = await client.customers.getPaymentMethods('cus_123');

// Use saved card for instant checkout
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  customer: { customer_id: 'cus_123' },
  payment_method_id: methods[0].payment_method_id,
  confirm: true,
  return_url: 'https://example.com/success'
});
```

## Bonnes Pratiques

<AccordionGroup>
<Accordion title="Accept all major networks">
Ne limitez pas les types de cartes sauf si c'est nécessaire. Les clients s'attendent à ce que leur carte préférée fonctionne.
</Accordion>

<Accordion title="Display card logos">
Affichez les logos Visa, Mastercard et Amex lors de votre paiement pour instaurer la confiance.
</Accordion>

<Accordion title="Handle declines gracefully">
Affichez des messages d'erreur clairs. N'exposez pas les codes d'erreur bruts aux clients.
</Accordion>

<Accordion title="Enable saved cards for returning customers">
Les méthodes de paiement enregistrées augmentent considérablement la conversion pour les achats répétés.
</Accordion>
</AccordionGroup>

## Dépannage

<AccordionGroup>
<Accordion title="Card declined">
**Causes :** Fonds insuffisants, carte expirée, CVV incorrect, protection anti-fraude de la banque.

**Solution :** Demandez au client de vérifier les informations ou d'essayer une autre carte.
</Accordion>

<Accordion title="3DS authentication failed">
**Causes :** Client abandonné, système bancaire indisponible, expiration du délai.

**Solution :** Réessayez ou demandez au client de contacter sa banque.
</Accordion>

<Accordion title="Card not supported">
**Causes :** Carte régionale non prise en charge, restrictions prépayées.

**Solution :** Le client devrait essayer une autre carte d'un réseau majeur.
</Accordion>
</AccordionGroup>
