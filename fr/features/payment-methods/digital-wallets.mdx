Les portefeuilles numériques permettent aux clients de payer d'un simple clic en utilisant les méthodes de paiement enregistrées sur leurs appareils. Ils offrent des taux de conversion plus élevés, une sécurité renforcée grâce à la tokenisation et une expérience de paiement simplifiée.

## Portefeuilles pris en charge

| Portefeuille | Disponibilité | Devises | Abonnements | Configuration requise |
| :----- | :----------- | :--------- | :-----------: | :------------- |
| **Apple Pay** | Global (hors Inde) | Toutes | Oui | Vérification de domaine |
| **Google Pay** | Global (hors Inde) | Toutes | Oui | Aucune |
| **Amazon Pay** | Global (hors Inde) | USD | Non | Aucune |
| **Cash App Pay** | Seulement aux États-Unis | USD | Non | Aucune |
| **RevolutPay** | Global | GBP | Non | Aucune |

<Warning>
**Support PayPal suspendu :** PayPal n'est temporairement pas disponible pour de nouvelles transactions.
</Warning>

## Apple Pay

Apple Pay permet aux clients de payer en utilisant les cartes enregistrées dans leur Apple Wallet, avec authentification via Face ID ou Touch ID.

### Exigences
- **Appareils :** iOS 17+, iPadOS 17+, Safari 17+ sur macOS
- **Protocole :** HTTPS requis
- **Domaine :** Doit être vérifié auprès d'Apple

### Vérification de domaine

<Steps>
<Step title="Download the domain association file">
Téléchargez le [fichier d'association de domaine Apple Pay](http://checkout.dodopayments.com/.well-known/apple-developer-merchantid-domain-association).
</Step>

<Step title="Host the file on your domain">
Téléversez-le sur votre serveur à :

```
https://yourdomain.com/.well-known/apple-developer-merchantid-domain-association
```

Le fichier doit être accessible sans redirections et servi avec `Content-Type: application/octet-stream` ou `text/plain`.
</Step>

<Step title="Request activation">
Envoyez un courriel à **support@dodopayments.com** avec :
- Votre domaine de production (par ex., `https://yourdomain.com`)
- Confirmation que le fichier est hébergé

Nous vérifierons et activerons Apple Pay sous 1 à 2 jours ouvrables.
</Step>

<Step title="Test the integration">
Après confirmation :
1. Ouvrez le paiement sur un appareil Apple
2. Vérifiez que le bouton Apple Pay apparaît
3. Effectuez une transaction de test
</Step>
</Steps>

<Warning>
Apple Pay n'apparaîtra dans le paiement qu'une fois la vérification du domaine terminée. Planifiez cela lors du lancement.
</Warning>

### Configuration

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: ['apple_pay', 'credit', 'debit'],
  return_url: 'https://example.com/success'
});
```

## Google Pay

Google Pay permet aux clients de payer en utilisant les cartes enregistrées dans leur compte Google, avec une protection contre la fraude intégrée.

### Exigences
- **Navigateurs :** Chrome (bureau/mobile), Safari, Firefox
- **Appareils :** Appareils Android avec l'application Google Pay
- **Paramètres :** "Autoriser les sites à vérifier si vous avez des méthodes de paiement" activé dans Chrome

### Configuration

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: ['google_pay', 'credit', 'debit'],
  return_url: 'https://example.com/success'
});
```

<Info>
Aucune vérification de domaine n'est nécessaire pour Google Pay — il fonctionne automatiquement une fois votre compte activé.
</Info>

## Amazon Pay

### Limitations

## Tests

### Tests Apple Pay

<Steps>
<Step title="Use test mode">
Activez le mode test dans votre tableau de bord Dodo Payments.
</Step>

<Step title="Add a card to Apple Wallet">
Ajoutez une carte réelle à votre Apple Wallet. En mode test, la carte ne sera pas débitée.
</Step>

<Step title="Complete test purchase">
Ouvrez le paiement sur un appareil Apple et procédez au flux Apple Pay.
</Step>
</Steps>

<Warning>
Apple Pay nécessite HTTPS. Il n'apparaîtra pas sur `localhost` sans une configuration SSL appropriée.
</Warning>

### Tests Google Pay

<Steps>
<Step title="Join the test card group">
[Rejoignez le groupe de cartes de test Google Pay](https://groups.google.com/g/googlepay-test-mode-stub-data) pour que des cartes de test soient ajoutées automatiquement à votre portefeuille.
</Step>

<Step title="Enable test mode">
Utilisez vos clés API de test Dodo Payments.
</Step>

<Step title="Complete test purchase">
Sélectionnez l'une des cartes de test dans Google Pay pour terminer la transaction.
</Step>
</Steps>

### Amazon Pay & Cash App Pay

Utilisez vos clés API de test et suivez le flux de paiement standard. Les transactions de test seront simulées sans frais réels.

## Support SDK

### SDK Web
Tous les portefeuilles numériques sont entièrement pris en charge dans :
- Overlay Checkout
- Inline Checkout
- Intégration API directe

### SDK Mobile

<Warning>
**Limitation React Native :** Apple Pay, Google Pay, Cash App Pay et UPI ne sont **pas encore pris en charge** dans le SDK React Native. Utilisez la superposition de paiement Web pour les applications mobiles qui nécessitent ces méthodes.
</Warning>

Pour un support mobile natif, envisagez :
- D'ouvrir le processus de paiement dans un WebView
- D'utiliser le [Overlay Checkout](/developer-resources/overlay-checkout) dans un contexte web
- D'implémenter séparément les SDK natifs Apple Pay / Google Pay

## Meilleures pratiques

<AccordionGroup>
<Accordion title="Prioritize wallets on mobile">
Les utilisateurs mobiles s'attendent à Apple Pay et Google Pay. Affichez ces options de manière bien visible lors des paiements mobiles.
</Accordion>

<Accordion title="Complete domain verification early">
Apple Pay nécessite une vérification de domaine. N'attendez pas le jour du lancement — commencez le processus pendant le développement.
</Accordion>

<Accordion title="Always include card fallbacks">
Tous les clients ne disposent pas de portefeuilles numériques. Incluez toujours `credit` et `debit` dans vos méthodes autorisées.
</Accordion>

<Accordion title="Test on real devices">
Les simulateurs ne reproduisent pas entièrement le comportement des portefeuilles. Testez sur de vrais appareils iOS et Android avant le lancement.
</Accordion>

<Accordion title="Handle wallet unavailability gracefully">
Si un portefeuille n'est pas disponible (mauvais appareil, pas de cartes enregistrées), assurez-vous que d'autres options de paiement sont visibles.
</Accordion>
</AccordionGroup>

## Dépannage

<AccordionGroup>
<Accordion title="Apple Pay button not appearing">
**Vérifications :**
1. Domaine vérifié avec Dodo Payments ?
2. Utilisez-vous HTTPS ?
3. L'appareil est iOS 17+ / Safari 17+ / iPadOS 17+ ?
4. Le client a des cartes dans Apple Wallet ?
5. `apple_pay` inclus dans `allowed_payment_method_types` ?

**Solution :** Effectuez la vérification du domaine et assurez-vous de l'HTTPS. Testez sur un vrai appareil Apple.
</Accordion>

<Accordion title="Google Pay button not appearing">
**Vérifications :**
1. Utilisez-vous Chrome ou un navigateur compatible ?
2. Le client a-t-il des cartes dans Google Pay ?
3. Le paramètre Chrome « Autoriser les sites à vérifier les moyens de paiement » est-il activé ?
4. `google_pay` inclus dans `allowed_payment_method_types` ?

**Solution :** Vérifiez la compatibilité du navigateur et la configuration Google Pay du client.
</Accordion>

<Accordion title="Wallet payment failed">
**Causes courantes :**
- La carte dans le portefeuille a expiré ou est invalide
- La banque a refusé la transaction
- Problème de connectivité réseau

**Solution :** Le client devrait vérifier sa carte dans son portefeuille ou essayer un autre moyen de paiement.
</Accordion>

<Accordion title="Wrong device type">
Les portefeuilles numériques n'apparaissent que sur des appareils compatibles :
- Apple Pay : uniquement les appareils Apple
- Google Pay : navigateur Chrome ou appareils Android

**Solution :** Il s'agit d'un comportement attendu. Assurez-vous que des alternatives de carte sont disponibles.
</Accordion>
</AccordionGroup>
