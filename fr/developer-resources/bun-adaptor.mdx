## Installation

<Steps>
<Step title="Install the package">
Exécutez la commande suivante à la racine de votre projet :

```bash
bun add @dodopayments/bun
```

</Step>
<Step title="Set up environment variables">
Créez un fichier <code>.env</code> à la racine de votre projet :

```env expandable
DODO_PAYMENTS_API_KEY=your-api-key
DODO_PAYMENTS_WEBHOOK_KEY=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT="test_mode" or "live_mode"
DODO_PAYMENTS_RETURN_URL=your-return-url
```

<Warning>
Ne commitez jamais votre fichier <code>.env</code> ou vos secrets dans un contrôle de version.
</Warning>
</Step>
</Steps>

## Route Handler Examples

<Info>
Tous les exemples supposent que vous utilisez le serveur natif de Bun avec <code>Bun.serve()</code>.
</Info>

<Tabs>
<Tab title="Checkout Handler">
  <Info>
  Utilisez ce gestionnaire pour intégrer le paiement Dodo Payments à votre serveur Bun. Il prend en charge les flux de paiement statiques (GET), dynamiques (POST) et de session (POST).
  </Info>
  <CodeGroup>

    ```typescript Bun Server Handler expandable
    import { Checkout } from '@dodopayments/bun';

    const staticCheckoutHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "static"
    });

    const sessionCheckoutHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "session"
    });

    const dynamicCheckoutHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "dynamic"
    });

    Bun.serve({
      port: 3000,
      fetch(request) {
        const url = new URL(request.url);
        
        if (url.pathname === "/api/checkout") {
          if (request.method === "GET") {
            return staticCheckoutHandler(request);
          }
          if (request.method === "POST") {
            return sessionCheckoutHandler(request);
            // or return dynamicCheckoutHandler(request);
          }
        }
        
        return new Response("Not Found", { status: 404 });
      },
    });
    ```

  </CodeGroup>

## Gestionnaire de route de paiement

<Info>
Dodo Payments prend en charge trois types de flux de paiement pour intégrer les paiements à votre site web, cet adaptateur prend en charge tous les types de flux.
</Info>

- **Liens de paiement statiques :** URL immédiatement partageables pour collecter rapidement un paiement sans coder.
- **Liens de paiement dynamiques :** Générez programmétiquement des liens de paiement avec des détails personnalisés à l'aide de l'API ou des SDK.
- **Sessions de paiement :** Créez des expériences de paiement sécurisées et personnalisables avec des paniers de produits préconfigurés et des informations client.

Le paiement statique renvoie une réponse JSON contenant l'URL de paiement :

```json
{
  "checkout_url": "https://checkout.dodopayments.com/..."
}
```

<Accordion title="Dynamic Checkout (POST)">
- Envoyez les paramètres dans le corps JSON d'une requête POST.
- Prend en charge les paiements ponctuels et récurrents.
- Pour la liste complète des champs pris en charge dans le corps POST, consultez :
  - [Request body for a One Time Payment Product](https://docs.dodopayments.com/api-reference/payments/post-payments)
  - [Request body for a Subscription Product](https://docs.dodopayments.com/api-reference/subscriptions/post-subscriptions)

### Format de réponse
Le paiement dynamique renvoie une réponse JSON contenant l'URL de paiement :

```json
{
  "checkout_url": "https://checkout.dodopayments.com/..."
}
```

</Accordion>
<Accordion title="Checkout Sessions (POST)">
Les sessions de paiement offrent une expérience de paiement hébergée plus sécurisée qui gère l'ensemble du flux de paiement pour les achats ponctuels et les abonnements avec un contrôle complet de personnalisation.

Reportez-vous au [Guide d'intégration des sessions de paiement](https://docs.dodopayments.com/developer-resources/checkout-session) pour plus de détails et une liste complète des champs pris en charge.

### Format de réponse
Les sessions de paiement renvoient une réponse JSON avec l'URL de paiement :

```json
{
  "checkout_url": "https://checkout.dodopayments.com/session/..."
}
```

## Gestionnaire de route du portail client

Le gestionnaire de route du portail client vous permet d'intégrer de manière transparente le portail client Dodo Payments à votre application serveur Bun.

### Query Parameters
<ParamField query="customer_id" type="string" required>
L'ID du client pour la session du portail (par ex. <code>?customer_id=cus_123</code>).
</ParamField>
<ParamField query="send_email" type="boolean">
S'il est défini sur <code>true</code>, envoie un e-mail au client contenant le lien du portail.
</ParamField>

<Warning>
Renvoie 400 si <code>customer_id</code> est manquant.
</Warning>

## Gestionnaire de route webhook

- **Méthode :** Seules les requêtes POST sont prises en charge. Les autres méthodes renvoient 405.
- **Vérification de la signature :** Vérifie la signature du webhook à l'aide de <code>webhookKey</code>. Renvoie 401 si la vérification échoue.
- **Validation de la charge :** Validée avec Zod. Renvoie 400 pour les charges invalides.
- **Gestion des erreurs :**
  - 401 : Signature invalide
  - 400 : Charge invalide
  - 500 : Erreur interne lors de la vérification
- **Routage des événements :** Appelle le gestionnaire d'événements approprié en fonction du type de charge.

import { Checkout, CustomerPortal, Webhooks } from "@dodopayments/bun";

const staticCheckoutHandler = Checkout({
  bearerToken: process.env.DODO_PAYMENTS_API_KEY,
  returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
  environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
  type: "static",
});

const sessionCheckoutHandler = Checkout({
  bearerToken: process.env.DODO_PAYMENTS_API_KEY,
  returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
  environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
  type: "session",
});

const customerPortalHandler = CustomerPortal({
  bearerToken: process.env.DODO_PAYMENTS_API_KEY,
  environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
});

const webhookHandler = Webhooks({
  webhookKey: process.env.DODO_PAYMENTS_WEBHOOK_KEY,
  onPaymentSucceeded: async (payload) => {
    console.log("Payment succeeded:", payload);
    // Your business logic here
  },
  onSubscriptionActive: async (payload) => {
    console.log("Subscription activated:", payload);
    // Your business logic here
  },

  port: 3000,
  fetch(request) {
