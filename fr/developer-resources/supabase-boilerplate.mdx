## Aperçu

Un modèle prêt pour la production pour les abonnements utilisant Next.js 15, React 19, Supabase, Drizzle ORM et Dodo Payments. Il est livré avec Google OAuth, le paiement d'abonnement, la gestion des webhooks, le schéma de base de données et un tableau de bord de base.

<Info>
Si vous avez seulement besoin de gestionnaires de route pour une application existante, consultez les adaptateurs dédiés : <a href="/developer-resources/nextjs-adaptor">Next.js Adaptor</a> et <a href="/developer-resources/express-adaptor">Express Adaptor</a>.
</Info>

## Prérequis

- Node.js 18+ (ou Bun 1.0+)
- Projet Supabase (URL, clé Anon, clé de rôle de service, URL de base de données)
- Compte Dodo Payments (clé API, secret de webhook)
- Client OAuth Google Cloud (ID et secret du client)

## Démarrage rapide

<Steps>
{/* LOCKED_PATTERN_587a3a4c926aec89f48dfc1e7c21054e */}

```bash
git clone https://github.com/dodopayments/dodo-supabase-subscription-starter.git
cd dodo-supabase-subscription-starter
# choose one
bun install
# or
npm install
# or
pnpm install
```

</Step>

{/* LOCKED_PATTERN_ea54c61e84e28cd275c0eed6d7634e0f */}
Créez un projet Supabase et copiez :
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
- SUPABASE_SERVICE_ROLE_KEY
- DATABASE_URL (chaîne de connexion)
</Step>

{/* LOCKED_PATTERN_42bf2f0f0010fd47b77c88325d34313c */}
Définissez l'URI de redirection sur : `https://[your-project-ref].supabase.co/auth/v1/callback` dans Google Cloud, puis activez le fournisseur Google dans Supabase Auth en utilisant votre ID client et votre secret.
</Step>

{/* LOCKED_PATTERN_f32b65d8a917a558befa771bede82115 */}
Générez une clé API et un secret de webhook depuis le tableau de bord Dodo. Définissez l'environnement sur `test_mode` pendant le développement.
</Step>

{/* LOCKED_PATTERN_a34a8e162789e5a79d108d9eb3c86df5 */}

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role

# Database
DATABASE_URL=postgresql://postgres:[password]@db.[project-ref].supabase.co:5432/postgres

# Dodo Payments
DODO_PAYMENTS_API_KEY=your-dodo-api-key
DODO_WEBHOOK_SECRET=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT=test_mode
```

<Warning>
Ne validez jamais de secrets. Utilisez des variables d'environnement dans les environnements de déploiement.
</Warning>
</Step>

{/* LOCKED_PATTERN_c35fe33eae6aa68b4bc39bd740632964 */}

```bash
bun run db:push
# or
npm run db:push
# or
pnpm run db:push
```

<Check>
Tables créées : `users`, `subscriptions`, `payments`.
</Check>
</Step>

{/* LOCKED_PATTERN_560d1d81fad22bd0622f0c8dcb3c706d */}

```bash
# login (one-time)
bunx supabase login
# or
npx supabase login

# deploy the edge function
bun run deploy:webhook --project-ref [your-project-ref]
# or
npm run deploy:webhook -- --project-ref [your-project-ref]
# or
pnpm run deploy:webhook --project-ref [your-project-ref]
```

```bash cURL
curl -X POST \
  'https://[your-project-ref].supabase.co/functions/v1/dodo-webhook' \
  -H 'Content-Type: application/json' \
  -H 'Dodo-Signature: <signature>' \
  -d '{"type":"payment.succeeded","data":{}}'
```

</Step>

{/* LOCKED_PATTERN_dd5db8f3734010c91f70fdf794c4e2a2 */}

## Ce qui est inclus

- Authentification via Supabase (OAuth Google configuré)
- Paiement d'abonnement via Dodo Payments
- Fonction Edge Supabase pour les webhooks (`dodo-webhook`)
- Schéma et migrations Drizzle ORM
- Tableau de bord avec factures, état de l'abonnement et fonctionnalités du plan

<Tip>
Conservez `DODO_PAYMENTS_ENVIRONMENT` comme `test_mode` jusqu'à ce que vous ayez terminé les tests de bout en bout.
</Tip>

## Vérification et dépannage

<AccordionGroup>
{/* LOCKED_PATTERN_6b286e9e2bbacd5c7684f8442203f10e */}
- Assurez-vous que `DODO_WEBHOOK_SECRET` correspond à la valeur du tableau de bord Dodo
- Confirmez que vous avez déployé la dernière fonction `dodo-webhook`
- Vérifiez que le nom de l'en-tête est correct dans votre fonction (Dodo-Signature)
</Accordion>
{/* LOCKED_PATTERN_c270a2f8797d0168d608bb07bd40f056 */}
- Vérifiez la syntaxe de `DATABASE_URL` et l'egress réseau Supabase
- Attendez environ 2 à 3 minutes après la création du projet avant le premier push
</Accordion>
{/* LOCKED_PATTERN_276d2f8137bb346c423586f69d6b7cae */}
- L'URI de redirection doit être `https://[ref].supabase.co/auth/v1/callback`
- Assurez-vous qu'elle est identique dans Google Cloud et le fournisseur Supabase Auth
</Accordion>
</AccordionGroup>
