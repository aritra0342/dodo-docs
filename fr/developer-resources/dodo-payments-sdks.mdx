Dodo Payments fournit des SDK officiels pour plusieurs langages de programmation, chacun conçu avec les meilleures pratiques spécifiques au langage et des fonctionnalités modernes pour une intégration de paiement sans faille.

<Tip>
Utilisez toujours la dernière version du SDK pour accéder aux fonctionnalités et améliorations les plus récentes. Vérifiez régulièrement votre gestionnaire de paquets pour les mises à jour afin de garantir l'accès à toutes les capacités de Dodo Payments.
</Tip>

## Démarrage rapide

Commencez avec n'importe quel SDK en quelques minutes :

<Steps>
{/* LOCKED_PATTERN_776027320500bde6b99bac6bed1cc64d */}
  Utilisez le gestionnaire de paquets de votre langage pour installer le SDK
  
  <Tabs>
  <Tab title="TypeScript/Node.js">

  ```bash
  npm install dodopayments
  ```

  </Tab>
  
  <Tab title="Python">

  ```bash
  pip install dodopayments
  ```

  </Tab>
  
  <Tab title="PHP">

  ```bash
  composer require dodopayments/client
  ```

  </Tab>
  
  <Tab title="Go">

  ```bash
  go get github.com/dodopayments/dodopayments-go
  ```

  </Tab>
  </Tabs>
</Step>

{/* LOCKED_PATTERN_2e38842e99ea3bb8f9763781777f831c */}
  Configurez le client avec votre clé API
  
  <Tabs>
  <Tab title="TypeScript">

  ```typescript
  import DodoPayments from 'dodopayments';
  const client = new DodoPayments({ bearerToken: 'your_api_key' });
  ```

  </Tab>
  
  <Tab title="Python">

  ```python
  from dodopayments import DodoPayments
  client = DodoPayments(bearer_token="your_api_key")
  ```

  </Tab>
  
  <Tab title="PHP">

  ```php
  use Dodopayments\Client;
  $client = new Client(bearerToken: 'your_api_key');
  ```

  </Tab>
  
  <Tab title="Go">

  ```go
  import "github.com/dodopayments/dodopayments-go"
  client := dodopayments.NewClient(option.WithBearerToken("your_api_key"))
  ```

  </Tab>
  </Tabs>
  
  <Warning>
  Stockez toujours vos clés API en toute sécurité en utilisant des variables d'environnement. Ne les validez jamais dans le contrôle de version.
  </Warning>
</Step>

{/* LOCKED_PATTERN_6ddfb9c24c549e202af4c004c17b0c78 */}
  Créez une session de paiement ou un paiement
  
  <Check>
  Vous êtes maintenant prêt à traiter des paiements ! Consultez les pages des SDK individels pour des guides détaillés et des exemples.
  </Check>
</Step>
</Steps>

## Fonctionnalités clés

Tous les SDK partagent ces capacités fondamentales :

- **Sécurité de type** : Typage fort pour la sécurité à la compilation et un meilleur support IDE
- **Gestion des erreurs** : Gestion complète des exceptions avec des messages d'erreur détaillés
- **Authentification** : Authentification simple par clé API avec support des variables d'environnement
- **Support asynchrone** : Modèles modernes async/await lorsque cela est applicable
- **Pagination automatique** : Pagination automatique pour les réponses de liste
- **Facturation basée sur l'utilisation** : Support intégré pour le suivi et l'ingestion des événements d'utilisation
- **Tests** : Support complet de l'environnement sandbox pour le développement et les tests

## SDK React Native

Pour les applications mobiles, nous fournissons un SDK React Native dédié :

{/* LOCKED_PATTERN_7cd688e2c7768dfaddbe428ff19fae7c */}
  Créez des expériences de paiement sécurisées pour les applications iOS et Android avec des composants d'interface natifs et une collecte de données de paiement simplifiée
</Card>

**Fonctionnalités :**
- Composants UI natifs pour Android et iOS
- Sécurité simplifiée pour la collecte de données de paiement sensibles
- Support pour plusieurs méthodes de paiement
- Intégration transparente avec les applications React Native

<Info>
Apple Pay, Google Pay, Cash App et UPI ne sont actuellement pas pris en charge dans le SDK React Native. La prise en charge de ces méthodes de paiement est prévue pour de futures versions.
</Info>

Nous accueillons les contributions à tous nos SDK ! Chaque dépôt dispose d'un fichier `CONTRIBUTING.md` contenant des directives pour :

- Signaler des bogues
- Demander des fonctionnalités
- Soumettre des demandes de tirage

- Style de code et conventions
