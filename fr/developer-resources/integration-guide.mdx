## Prérequis

Pour intégrer l'API Dodo Payments, vous aurez besoin de :
- Un compte marchand Dodo Payments
- Identifiants API (clé API et clé secrète de webhook) depuis le tableau de bord

## Configuration du tableau de bord

1. Accédez au [Tableau de bord Dodo Payments](https://app.dodopayments.com/)

2. Créez un produit (paiement unique ou abonnement)

4. Générez votre clé API :
   - Allez dans Développeur > API
   - [Guide détaillé](/api-reference/introduction#api-key-generation)
   - Copiez la clé API dans l'environnement nommé DODO_PAYMENTS_API_KEY

5. Configurez les webhooks :
   - Allez dans Développeur > Webhooks
   - Créez une URL de webhook pour les notifications de paiement
   - Copiez la clé secrète de webhook dans l'environnement

## Intégration

### Liens de paiement

Choisissez le chemin d'intégration qui correspond à votre cas d'utilisation :
- **Sessions de paiement (recommandé)** : Meilleur pour la plupart des intégrations. Créez une session sur votre serveur et redirigez les clients vers un paiement sécurisé hébergé.
- **Overlay Checkout (intégré)** : Utilisez-le lorsque vous avez besoin d'une expérience en page qui intègre le paiement hébergé sur votre site.
- **Liens de paiement statiques** : URL partageables instantanément sans code pour une collecte rapide des paiements.
- **Liens de paiement dynamiques** : Liens créés par programmation. Cependant, les sessions de paiement sont recommandées et offrent plus de flexibilité.

#### 1. Sessions de paiement

Utilisez les sessions de paiement pour créer une expérience de paiement sécurisée et hébergée pour les paiements uniques ou les abonnements. Vous créez une session sur votre serveur, puis redirigez le client vers le `checkout_url` retourné.

<Info>
Les sessions de paiement sont valides pendant 24 heures par défaut. Si vous passez <code>confirm=true</code>, les sessions sont valides pendant 15 minutes et tous les champs requis doivent être fournis.
</Info>

<Steps>
<Step title="Créer une session de paiement">
  Choisissez votre SDK préféré ou appelez l'API REST.

  <Tabs>
  <Tab title="Node.js SDK">

  ```javascript
  import DodoPayments from 'dodopayments';

  const client = new DodoPayments({
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    environment: 'test_mode', // defaults to 'live_mode'
  });

  const session = await client.checkoutSessions.create({
    product_cart: [{ product_id: 'prod_123', quantity: 1 }],
    customer: { email: 'customer@example.com', name: 'John Doe' },
    return_url: 'https://yourapp.com/checkout/success',
  });
  ```

  </Tab>

  <Tab title="Python SDK">

      customer={"email": "customer@example.com", "name": "John Doe"},
      return_url="https://yourapp.com/checkout/success",

<Step title="Rediriger le client vers le paiement">
  Après la création de la session, redirigez vers le `checkout_url` pour commencer le flux hébergé.

#### 3. Liens de paiement statiques

Les liens de paiement statiques vous permettent d'accepter rapidement des paiements en partageant une URL simple. Vous pouvez personnaliser l'expérience de paiement en passant des paramètres de requête pour pré-remplir les détails du client, contrôler les champs du formulaire et ajouter des métadonnées personnalisées.

<Steps>
<Step title="Construire votre lien de paiement">
  Commencez par l'URL de base et ajoutez votre ID de produit :

  ```text
  https://checkout.dodopayments.com/buy/{productid}
  ```

</Step>
<Step title="Ajouter des paramètres essentiels">
  Incluez les paramètres de requête essentiels :
  - <ParamField query="quantity" type="integer" default="1">Nombre d'articles à acheter.</ParamField>
  - <ParamField query="redirect_url" type="string" required>URL vers laquelle rediriger après la finalisation du paiement.</ParamField>
  <Note>
  L'URL de redirection inclura les détails du paiement en tant que paramètres de requête, par exemple :<br/>
  <code>https://example.com/?payment_id=pay_ts2ySpzg07phGeBZqePbH&status=succeeded</code>
  </Note>
</Step>
<Step title="Pré-remplir les informations du client (optionnel)">
  Ajoutez des champs client ou de facturation en tant que paramètres de requête pour simplifier le paiement.
  <AccordionGroup>
    <Accordion title="Champs client pris en charge">
      - <ParamField query="fullName" type="string">Nom complet du client (ignoré si firstName ou lastName est fourni).</ParamField>
      - <ParamField query="firstName" type="string">Prénom du client.</ParamField>
      - <ParamField query="lastName" type="string">Nom de famille du client.</ParamField>
      - <ParamField query="email" type="string">Adresse e-mail du client.</ParamField>
      - <ParamField query="country" type="string">Pays du client.</ParamField>
      - <ParamField query="addressLine" type="string">Adresse de rue.</ParamField>
      - <ParamField query="city" type="string">Ville.</ParamField>
      - <ParamField query="state" type="string">État ou province.</ParamField>
      - <ParamField query="zipCode" type="string">Code postal.</ParamField>
      - <ParamField query="showDiscounts" type="boolean">true ou false</ParamField>
    </Accordion>
  </AccordionGroup>
</Step>
<Step title="Contrôler les champs du formulaire (optionnel)">
  <Info>
  Vous pouvez désactiver des champs spécifiques pour les rendre en lecture seule pour le client. Cela est utile lorsque vous avez déjà les détails du client (par exemple, les utilisateurs connectés).
  </Info>
  Pour désactiver un champ, fournissez sa valeur et définissez le drapeau <code>disable...</code> correspondant sur <code>true</code> :
  <CodeGroup>

  ```text Example
  &email=alice@example.com&disableEmail=true
  ```

  </CodeGroup>
  <Tabs>
    <Tab title="Tableau des drapeaux de désactivation">
      | Champ         | Drapeau de désactivation | Paramètre requis |
      |---------------|-------------------------|-------------------|
      | Nom complet   | `disableFullName`   | `fullName`        |
      | Prénom        | `disableFirstName`  | `firstName`       |
      | Nom de famille | `disableLastName`   | `lastName`        |
      | E-mail        | `disableEmail`      | `email`           |
      | Pays          | `disableCountry`    | `country`         |
      | Ligne d'adresse | `disableAddressLine`| `addressLine`     |
      | Ville         | `disableCity`       | `city`            |
      | État         | `disableState`      | `state`           |
      | Code postal   | `disableZipCode`    | `zipCode`         |
    </Tab>
  </Tabs>
  <Tip>
  Désactiver des champs aide à prévenir les modifications accidentelles et garantit la cohérence des données.
  </Tip>
  <Note>
  Définir <code>showDiscounts=false</code> désactivera et masquera la section des réductions dans le formulaire de paiement. Utilisez cela si vous souhaitez empêcher les clients d'entrer des codes de coupon ou de promotion lors du paiement.
  </Note>

</Step>
<Step title="Ajouter des contrôles avancés (optionnel)">
  - <ParamField query="paymentCurrency" type="string">Spécifie la devise de paiement. Par défaut, c'est la devise du pays de facturation.</ParamField>
  - <ParamField query="showCurrencySelector" type="boolean" default="true">Afficher ou masquer le sélecteur de devise.</ParamField>
  - <ParamField query="paymentAmount" type="integer">Montant en cents (pour le prix Pay What You Want uniquement).</ParamField>
  - <ParamField query="metadata_*" type="string">Champs de métadonnées personnalisées (par exemple, <code>metadata_orderId=123</code>).</ParamField>
</Step>
<Step title="Partager le lien">
  Envoyez le lien de paiement complété à votre client. Lorsqu'il le visite, tous les paramètres de requête sont collectés et stockés avec un ID de session. L'URL est ensuite simplifiée pour inclure uniquement le paramètre de session (par exemple, <code>?session=sess_1a2b3c4d</code>). Les informations stockées persistent à travers les actualisations de page et sont accessibles tout au long du processus de paiement.
  <Check>
  L'expérience de paiement du client est maintenant rationalisée et personnalisée en fonction de vos paramètres.
  </Check>
</Step>
</Steps>

    );

### Mise en œuvre des webhooks

Configurez un point de terminaison API pour recevoir des notifications de paiement. Voici un exemple utilisant Next.js :

```javascript
import { Webhook } from "standardwebhooks";
import { headers } from "next/headers";
import { WebhookPayload } from "@/types/api-types";

const webhook = new Webhook(process.env.DODO_WEBHOOK_KEY!); // Replace with your secret key generated from the Dodo Payments Dashboard

export async function POST(request: Request) {
  const headersList = headers();
  const rawBody = await request.text();

  const webhookHeaders = {
    "webhook-id": headersList.get("webhook-id") || "",
    "webhook-signature": headersList.get("webhook-signature") || "",
    "webhook-timestamp": headersList.get("webhook-timestamp") || "",
  };

  await webhook.verify(rawBody, webhookHeaders);
  const payload = JSON.parse(rawBody) as WebhookPayload;
  
  // Process the payload according to your business logic
}
```

Notre mise en œuvre des webhooks suit la spécification des [Webhooks Standards](https://standardwebhooks.com/). Pour les définitions de types de webhook, consultez notre [Guide des événements de webhook](/developer-resources/webhooks/intents/webhook-events-guide).

Vous pouvez vous référer à ce projet avec une mise en œuvre de démonstration sur [GitHub](https://github.com/dodopayments/dodo-checkout-demo) utilisant Next.js et TypeScript.

Vous pouvez consulter la mise en œuvre en direct [ici](https://atlas.dodopayments.com/).
