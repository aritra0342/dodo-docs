## Aperçu

Sentra ajoute de l'intelligence et de l'exécution au-dessus des piles de base de Dodo Payments. C'est un agent IA qui intègre, analyse et agit sur la facturation et les paiements. Sentra connecte les SDK, les API et les adaptateurs à travers votre pile (Auth, DB, CRM, etc.), met en lumière des informations sur les revenus comme les moteurs de croissance, et exécute des actions clients telles que des remboursements, des mises à niveau et des crédits.

<Info>
Sentra est actuellement en Beta. Les fonctionnalités, les API et les intégrations peuvent changer rapidement. Nous accueillons vos retours et rapports de bogues pendant que nous itérons !
</Info>

## Configuration

Une fois installé, vous devez configurer Sentra avec votre clé API Dodo Payments et les paramètres d'environnement.

<Steps>
<Step title="Obtenez votre clé API">
1. Connectez-vous à votre [tableau de bord Dodo Payments](https://dashboard.dodopayments.com).
2. Accédez à **Paramètres** › **Clés API**.
3. Copiez votre clé API.
</Step>

<Step title="Configurer Sentra">
1. Ouvrez votre IDE (VS Code, Cursor ou Windsurf).
2. Ouvrez la Palette de Commandes (`Cmd+Shift+P` sur macOS / `Ctrl+Shift+P` sur Windows/Linux).
3. Tapez "Sentra: Configurer" et sélectionnez-le.
4. Entrez votre clé API Dodo Payments lorsque vous y êtes invité.

<Frame>
  <img
    src="/images/sentra/setup.png"
    alt="Écran de configuration de Sentra avec saisie de clé API et sélection d'environnement"
    width="50%"
  />
</Frame>
</Step>

<Step title="Sélectionner le Mode d'Environnement">
Choisissez votre mode d'environnement :
- **Mode Test** : Pour le développement et les tests
- **Mode Live** : Pour une utilisation en production

<Warning>
Commencez par le mode Test pour valider votre configuration avant de passer au mode Live pour les données de production.
</Warning>
</Step>

<Step title="Définir l'Effort de Raisonnement du Modèle">
Sélectionnez votre niveau d'effort de raisonnement de modèle préféré :
- **Faible** : Réponses plus rapides, adapté aux tâches simples
- **Moyen** : Performance équilibrée (recommandé)
- **Élevé** : Analyse plus approfondie, adapté aux intégrations complexes
</Step>

<Step title="Compléter la Configuration">
Cliquez sur **C'est parti !** pour compléter la configuration et commencer à utiliser Sentra.

<Check>
Vos clés API sont stockées en toute sécurité dans le stockage secret de votre IDE et ne sont jamais exposées dans votre code.
</Check>
</Step>
</Steps>

## Comment ça fonctionne

Sentra suit un processus simple en quatre étapes, de l'invite à la production :

<Steps>
<Step title="Invite">
Tapez ce que vous souhaitez construire. Par exemple : "Ajoutez une facturation basée sur l'utilisation avec un essai gratuit pour mon plan Pro."

<Frame>
  <img
    src="/images/sentra/welcome.png"
    alt="Interface de Sentra montrant l'entrée de l'invite avec une tâche d'exemple"
    width="50%"
  />
</Frame>
</Step>

<Step title="Plan">
Sentra génère les modifications (hooks SDK, objets, config) et présente un plan pour votre révision.
</Step>

<Step title="Approuver">
Examinez le plan généré et confirmez les modifications que vous souhaitez que Sentra mette en œuvre.
</Step>

<Step title="Exécuter">
Sentra applique les modifications, les teste et confirme le succès.
</Step>
</Steps>

## Utiliser Sentra

### Modes

Sentra propose trois modes d'interaction accessibles depuis l'interface :

- **Demander** : Posez des questions sur Dodo Payments, les modèles de facturation, l'utilisation de l'API et les meilleures pratiques
- **Construire** : Intégrez la facturation et l'interface utilisateur dans votre application avec une assistance guidée
- **Planifier** : Concevez des architectures de facturation évolutives avec des conseils architecturaux

### Tâches d'Exemple

Voici quelques tâches courantes que vous pouvez accomplir avec Sentra :

- **Intégrer la Page de Tarification** : Configurez une page de tarification complète avec des listes de produits et un processus de paiement
- **Implémenter un Webhook** : Configurez des gestionnaires de webhook pour les événements de paiement
- **Ajouter un Flux d'Abonnement** : Créez une gestion d'abonnement avec des capacités de mise à niveau/déclassement
- **Configurer la Facturation Basée sur l'Utilisation** : Configurez des compteurs et l'ingestion d'événements pour le suivi de l'utilisation
- **Créer un Portail Client** : Construisez un portail d'auto-service pour la gestion de la facturation

### Paramètres d'Auto-Approvisionnement

Vous pouvez configurer des paramètres d'auto-approvisionnement pour rationaliser votre flux de travail :

- **Aucun** : Examinez et approuvez toutes les modifications manuellement (recommandé pour la production)
- **Faible Risque** : Auto-approuvez les modifications simples et à faible risque
- **Toutes** : Auto-approuvez toutes les modifications (à utiliser avec prudence)

## Modèles de Facturation Pris en Charge

Sentra prend en charge tous les modèles de facturation de Dodo Payments :

- **Paiements Uniques** : Transactions d'achat uniques
- **Abonnements** : Facturation récurrente avec plusieurs intervalles
- **Facturation Basée sur l'Utilisation** : Facturation mesurée basée sur la consommation
- **Facturation Basée sur les Résultats** : Modèles de tarification au résultat

## Pour Qui Est-ce ?

Sentra fait le lien entre l'ingénierie et les affaires, afin que chaque équipe puisse expédier la monétisation sans friction :

- **Développeurs** : Expédiez la monétisation plus rapidement dans votre IDE
- **Fondateurs & PMs** : Prototypage de modèles de facturation sans travail d'infrastructure approfondi
- **RevOps & Finance** : Utilisez des actions encadrées sans avoir besoin d'ingénieurs
- **Support & Succès** : Résolvez les cas de facturation en toute sécurité, avec des approbations là où c'est nécessaire

## Dépannage

<AccordionGroup>
<Accordion title="L'extension n'apparaît pas après l'installation">
Si Sentra n'apparaît pas dans votre IDE après l'installation, essayez ce qui suit :

1. **Redémarrez votre éditeur** : Fermez et rouvrez complètement VS Code, Cursor ou Windsurf
2. **Vérifiez l'installation** : Vérifiez que l'extension est installée en allant dans Extensions et en recherchant "Sentra"
3. **Recharger la fenêtre** : Utilisez la Palette de Commandes (`Cmd+Shift+P` / `Ctrl+Shift+P`) et exécutez "Développeur : Recharger la fenêtre"

<Check>
Le redémarrage de l'éditeur résout le problème dans la plupart des cas.
</Check>
</Accordion>

<Accordion title="Clé API invalide ou erreurs d'authentification">
- Vérifiez que votre clé API est correcte et active dans le tableau de bord Dodo Payments
- Assurez-vous d'utiliser le bon mode d'environnement (Test vs Live) qui correspond à votre clé API
- Vérifiez que votre clé API a les autorisations nécessaires
- Essayez de régénérer votre clé API si le problème persiste
</Accordion>

<Accordion title="Configuration non enregistrée">
- Assurez-vous d'avoir complété tous les champs requis dans l'écran de configuration
- Vérifiez que votre IDE a la permission d'accéder au stockage secret
- Essayez de reconfigurer Sentra via la Palette de Commandes
</Accordion>

<Accordion title="Échec de la génération du plan">
- Vérifiez que votre clé API est valide et a accès aux ressources requises
- Vérifiez votre connexion Internet
- Assurez-vous d'utiliser un modèle de facturation pris en charge
- Essayez de reformuler votre invite avec des détails plus spécifiques
</Accordion>

<Accordion title="Erreurs d'exécution">
- Examinez attentivement le plan généré avant approbation
- Vérifiez que la structure de votre projet correspond aux attentes de Sentra
- Vérifiez que toutes les dépendances sont installées
- Consultez le panneau de sortie de l'IDE pour des messages d'erreur détaillés
</Accordion>
</AccordionGroup>

<Warning>
Si vous continuez à rencontrer des problèmes après avoir redémarré votre éditeur, contactez le support avec des détails sur votre version d'IDE et les messages d'erreur spécifiques que vous voyez.
</Warning>
