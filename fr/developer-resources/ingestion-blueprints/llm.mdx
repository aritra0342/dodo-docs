## Démarrage rapide

Commencez avec le suivi automatique des tokens LLM en seulement 2 minutes :

<Steps>
{/* LOCKED_PATTERN_776027320500bde6b99bac6bed1cc64d */}
Installez les Blueprints d'ingestion de Dodo Payments :

```bash
npm install @dodopayments/ingestion-blueprints
```

</Step>

{/* LOCKED_PATTERN_e608972569617212f31c0b7d72677865 */}
Vous aurez besoin de deux clés API :

- **Clé API Dodo Payments** : Obtenez-la depuis le [Tableau de bord Dodo Payments](https://app.dodopayments.com/developer/api-keys)
- **Clé API du fournisseur LLM** : Depuis AI SDK, OpenAI, Anthropic, Groq, etc.

<Tip>
Stockez vos clés API de manière sécurisée dans des variables d'environnement. Ne les engagez jamais dans le contrôle de version.
</Tip>
</Step>

{/* LOCKED_PATTERN_3c26a22b684f0fb6b635e5bf1e61dd55 */}
Avant de suivre l'utilisation, créez un compteur dans votre tableau de bord Dodo Payments :

1. **Connectez-vous** à [Dodo Payments Dashboard](https://app.dodopayments.com/)
2. **Accédez à** Products → Meters
3. **Cliquez** sur "Create Meter"
4. **Configurez votre compteur** :
   - **Nom du compteur** : Choisissez un nom descriptif (par exemple, "LLM Token Usage")
   - **Nom de l'événement** : Définissez un identifiant d'événement unique (par exemple, `llm.chat_completion`)
   - **Type d’agrégation** : Sélectionnez `sum` pour additionner les comptes de jetons
   - **Propriété à suivre** : Choisissez ce que vous souhaitez suivre :
     - `inputTokens` - Suivre les jetons d'entrée/prompt
     - `outputTokens` - Suivre les jetons de sortie/completion (inclut les jetons de raisonnement lorsque cela s'applique)
     - `totalTokens` - Suivre le total des jetons d'entrée + de sortie

<Info>
Le **Nom de l'événement** que vous définissez ici doit correspondre exactement à celui que vous passez au SDK (sensible à la casse).
</Info>

Pour des instructions détaillées, consultez le [Guide de facturation à l'usage](/developer-resources/usage-based-billing-guide).
</Step>

## Configuration

### Configuration du tracker

Créez un tracker une fois au démarrage de l'application avec ces paramètres requis :

<ParamField path="apiKey" type="string" required>
Votre clé API Dodo Payments. Obtenez-la depuis la [page des clés API](https://app.dodopayments.com/developer/api-keys).

```javascript
apiKey: process.env.DODO_PAYMENTS_API_KEY
```

</ParamField>

<ParamField path="environment" type="string" required>
Le mode d'environnement du traceur.

- `test_mode` - À utiliser pour le développement et les tests
- `live_mode` - À utiliser en production

```javascript
environment: 'test_mode' // or 'live_mode'
```

<Warning>
Utilisez toujours `test_mode` pendant le développement pour éviter d'impacter les métriques de production.
</Warning>
</ParamField>

<ParamField path="eventName" type="string" required>
Le nom de l'événement qui déclenche votre compteur. Il doit correspondre exactement à ce que vous avez configuré dans votre compteur Dodo Payments (sensible à la casse).

```javascript
eventName: 'llm.chat_completion'
```

<Info>
Ce nom d'événement relie votre utilisation suivie au compteur approprié pour les calculs de facturation.
</Info>
</ParamField>

### Configuration de l'enveloppeur

Lors de l'enveloppement de votre client LLM, fournissez ces paramètres :

<ParamField path="client" type="object" required>
Votre instance de client LLM (OpenAI, Anthropic, Groq, etc.).

```javascript
client: openai
```

</ParamField>

<ParamField path="customerId" type="string" required>
L'identifiant client unique pour la facturation. Il doit correspondre à votre ID client dans Dodo Payments.

```javascript
customerId: 'customer_123'
```

<Tip>
Utilisez l'ID utilisateur ou l'ID client de votre application pour garantir une facturation précise par client.
</Tip>
</ParamField>

<ParamField path="metadata" type="object">
Données supplémentaires optionnelles à joindre à l'événement de suivi. Utile pour le filtrage et l'analyse.

```javascript
metadata: {
  feature: 'chat',
  userTier: 'premium',
  sessionId: 'session_123',
  modelVersion: 'gpt-4'
}
```

</ParamField>

### Exemple de configuration complète

<CodeGroup>

```javascript Full Configuration
import { createLLMTracker } from "@dodopayments/ingestion-blueprints";
import { generateText } from "ai";
import { google } from "@ai-sdk/google";
import "dotenv/config";

async function aiSdkExample() {
  console.log("🤖 AI SDK Simple Usage Example\n");

  try {
    // 1. Create tracker
    const llmTracker = createLLMTracker({
      apiKey: process.env.DODO_PAYMENTS_API_KEY!,
      environment: "test_mode",
      eventName: "your_meter_event_name",
    });

    // 2. Wrap the ai-sdk methods
    const client = llmTracker.wrap({
      client: { generateText },
      customerId: "customer_123",
      metadata: {
        provider: "ai-sdk",
      },
    });

    // 3. Use the wrapped function
    const response = await client.generateText({
      model: google("gemini-2.5-flash"),
      prompt: "Hello, I am a cool guy! Tell me a fun fact.",
      maxOutputTokens: 500,
    });

    console.log(response);
    console.log(response.usage);
    console.log("✅ Automatically tracked for customer\n");
  } catch (error) {
    console.error(error);
  }
}

aiSdkExample().catch(console.error);
```

### Intégration API Express.js

Exemple complet d'intégration du suivi LLM dans une API Express.js :

<CodeGroup>

```javascript Express.js Server
import express from 'express';
import { createLLMTracker } from '@dodopayments/ingestion-blueprints';
import OpenAI from 'openai';

const app = express();
app.use(express.json());

// Initialize OpenAI client
const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

// Create tracker once at startup
const tracker = createLLMTracker({
  apiKey: process.env.DODO_PAYMENTS_API_KEY,
  environment: process.env.NODE_ENV === 'production' ? 'live_mode' : 'test_mode',
  eventName: 'api.chat_completion'
});

// Chat endpoint with automatic tracking
app.post('/api/chat', async (req, res) => {
  try {
    const { message, userId } = req.body;
    
    // Validate input
    if (!message || !userId) {
      return res.status(400).json({ error: 'Missing message or userId' });
    }
    
    // Wrap client for this specific user
    const trackedClient = tracker.wrap({
      client: openai,
      customerId: userId,
      metadata: { 
        endpoint: '/api/chat',
        timestamp: new Date().toISOString()
      }
    });
    
    // Make LLM request - automatically tracked
    const response = await trackedClient.chat.completions.create({
      model: 'gpt-4',
      messages: [{ role: 'user', content: message }],
      temperature: 0.7
    });
    
    const completion = response.choices[0].message.content;
    
    res.json({ 
      message: completion,
      usage: response.usage
    });
  } catch (error) {
    console.error('Chat error:', error);
    res.status(500).json({ error: 'Internal server error' });
  }
});

app.listen(3000, () => {
  console.log('Server running on port 3000');
});
```

</CodeGroup>

## Ce qui est suivi

Chaque appel API LLM envoie automatiquement un événement d'utilisation à Dodo Payments avec la structure suivante :

<CodeGroup>

```json Event Structure
{
  "event_id": "llm_1673123456_abc123",
  "customer_id": "customer_123",
  "event_name": "llm.chat_completion",
  "timestamp": "2024-01-08T10:30:00Z",
  "metadata": {
    "inputTokens": 10,
    "outputTokens": 25,
    "totalTokens": 35,
    "model": "gpt-4",
  }
}
```

</CodeGroup>

### Champs d'événement

<ParamField path="event_id" type="string">
Identifiant unique pour cet événement spécifique. Généré automatiquement par le SDK.

Format : `llm_[timestamp]_[random]`
</ParamField>

<ParamField path="customer_id" type="string">
L'ID client que vous avez fourni lors de l'encapsulation du client. Utilisé pour la facturation.
</ParamField>

<ParamField path="event_name" type="string">
Le nom de l'événement qui déclenche votre compteur. Correspond à votre configuration de suivi.
</ParamField>

<ParamField path="timestamp" type="string">
Horodatage ISO 8601 de l'événement.
</ParamField>

<ParamField path="metadata" type="object">
Utilisation des jetons et données supplémentaires de suivi :

- `inputTokens` - Nombre de jetons d'entrée/prompt utilisés
- `outputTokens` - Nombre de jetons de sortie/completion utilisés (inclut les jetons de raisonnement lorsque cela s'applique)
- `totalTokens` - Nombre total de jetons (entrée + sortie)
- `model` - Le modèle LLM utilisé (par exemple, "gpt-4")
- `provider` - Le fournisseur LLM (si inclus dans les métadonnées de l'enveloppe)
- Toutes métadonnées personnalisées que vous avez fournies lors de l'encapsulation du client

<Note>
**Jetons de raisonnement :** Pour les modèles avec capacités de raisonnement, `outputTokens` inclut automatiquement à la fois les jetons de complétion et les jetons de raisonnement.
</Note>
</ParamField>

<Info>
Votre compteur Dodo Payments utilise les champs `metadata` (en particulier `inputTokens`, `outputTokens` ou `totalTokens`) pour calculer l'utilisation et la facturation.
</Info>
