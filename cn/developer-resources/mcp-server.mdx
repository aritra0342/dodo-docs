## Overview

The Model Context Protocol (MCP) is an open standard that enables AI applications to securely connect to external data sources and APIs. The Dodo Payments MCP Server provides AI assistants like Claude, Cursor, and other MCP-compatible clients with structured access to your payment infrastructure.

Dodo Payments MCP Server 使用 **代码模式** 架构。与其为每个 API 端点暴露数百个单独的工具，代码模式使 AI 代理能够在隔离的沙箱环境中编写并执行针对 Dodo Payments SDK 的 TypeScript 代码。

### Key capabilities

- **Payment Operations**: Create, retrieve, and manage payments and refunds
- **Subscription Management**: Handle recurring billing, upgrades, and cancellations
- **Customer Administration**: Manage customer data and portal access
- **Product Catalog**: Create and update products, pricing, and discounts
- **License Management**: Activate, validate, and manage software licenses
- **Usage-Based Billing**: Track and bill for metered usage

## How Code Mode Works

Dodo Payments MCP Server 为您的 AI 代理提供了恰好两个工具：

1. **文档搜索工具**：查询有关 Dodo Payments API 与 SDK 的文档，以了解可用操作和参数。
2. **代码执行工具**：编写针对 SDK 的 TypeScript 代码，并在安全的沙箱环境中执行。

这种架构允许代理在一次调用中执行复杂的多步骤操作。例如，代理可以列出所有活动订阅，根据特定标准筛选它们，然后在一个脚本中对每个订阅应用折扣。

<Tip>
使用代码模式，代理可以串联多个 API 调用、处理条件逻辑并执行计算，而无需对 LLM 进行多次往返。
</Tip>

### 快速设置

在您的 AI 客户端中连接到 Dodo Payments MCP Server：

<Tabs>
<Tab title="Cursor">
添加到 `~/.cursor/mcp.json`：

```json
{
  "mcpServers": {
    "dodopayments": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://mcp.dodopayments.com/sse"]
    }
  }
}
```

</Tab>

<Tab title="Claude Desktop">
添加到您的 Claude Desktop 配置中：

```json
{
  "mcpServers": {
    "dodopayments": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://mcp.dodopayments.com/sse"]
    }
  }
}
```

配置文件位置：
- **macOS**：`~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**：`%APPDATA%\Claude\claude_desktop_config.json`
- **Linux**：`~/.config/Claude/claude_desktop_config.json`
</Tab>

<Tab title="Windsurf">
添加到 `~/.codeium/windsurf/mcp_config.json`：

```json
{
  "mcpServers": {
    "dodopayments": {
      "command": "npx",

    }
  }
}
```

</Tab>

```bash
claude mcp add dodopayments -- npx -y mcp-remote@latest https://mcp.dodopayments.com/sse
```

## Dodo Knowledge MCP

除了用于执行 API 操作的 Dodo Payments MCP Server，我们还提供 **Dodo Knowledge MCP**——一个语义搜索服务器，可让 AI 助手即时访问 Dodo Payments 文档和知识库。

<Info>
**基于 [ContextMCP.ai](https://contextmcp.ai) 构建**：Dodo Knowledge MCP 由 ContextMCP 提供支持，利用向量嵌入在我们的文档中实现快速语义搜索。
</Info>

### 什么是 Dodo Knowledge MCP？

Dodo Knowledge MCP 是一个远程 MCP 服务器，提供：

- **语义文档搜索**：使用自然语言查询查找相关文档。

- **零设置**：无需 API 密钥或本地安装——只需连接并开始查询。

### 同时使用两个 MCP 服务器

为了获得最佳的 AI 辅助开发体验，我们建议同时使用两个 MCP 服务器：

| 服务器 | 目的 | 用例 |
|--------|---------|----------|
| **Dodo Knowledge MCP** | 文档搜索 | “我如何处理 webhook？”，“支持哪些支付方式？” |
| **Dodo Payments MCP** | API 操作 | 创建支付、管理订阅、处理退款 |

```json
{
  "mcpServers": {
    "dodo-knowledge": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://knowledge.dodopayments.com/mcp"]
    },
    "dodopayments": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://mcp.dodopayments.com/sse"]
    }
  }
}
```

<Tip>
配置好两个服务器后，您的 AI 助手可以先搜索文档以了解如何实现某个功能，然后在一次对话中执行实际的 API 调用。
</Tip>

### Knowledge MCP 故障排除

如果遇到连接问题：

1. **清除 MCP 身份验证缓存**：`rm -rf ~/.mcp-auth`
2. **重新启动客户端应用**
3. **检查客户端日志** 以获取错误信息
4. **验证 Node.js 版本**：需要 Node.js 18+

### Docker

在容器化环境中运行 MCP 服务器，以实现一致的部署体验。

<CodeGroup>

```bash Docker
# Pull the latest image
docker pull ghcr.io/dodopayments/mcp:latest

# Run the container
docker run -e DODO_PAYMENTS_API_KEY="dodo_test_..." \
  -e DODO_PAYMENTS_WEBHOOK_KEY="your_webhook_key" \
  -e DODO_PAYMENTS_ENVIRONMENT="live_mode" \
  ghcr.io/dodopayments/mcp:latest
```

```yaml Docker Compose
services:
  dodopayments-mcp:
    image: ghcr.io/dodopayments/mcp:latest
    environment:
      - DODO_PAYMENTS_API_KEY=dodo_test_...
      - DODO_PAYMENTS_WEBHOOK_KEY=your_webhook_key
      - DODO_PAYMENTS_ENVIRONMENT=live_mode
    restart: unless-stopped
```

</CodeGroup>

<Note>
Docker 镜像可在 [GitHub Container Registry](https://github.com/orgs/dodopayments/packages/container/package/mcp) 上获取。
</Note>

## 环境变量

使用环境变量配置 MCP 服务器行为。

| 变量 | 描述 | 是否必需 |
|----------|-------------|----------|
| `DODO_PAYMENTS_API_KEY` | 您的 Dodo Payments API 密钥 | 是 |
| `DODO_PAYMENTS_WEBHOOK_KEY` | 您的 webhook 签名密钥 | 否 |
| `DODO_PAYMENTS_ENVIRONMENT` | 在生产中设置为 `live_mode` | 否 |

## 远程运行

将 MCP 服务器部署为远程 HTTP 服务器，以供基于 Web 的客户端或 agentic 工作流使用。

```bash
# Start as remote server
npx -y dodopayments-mcp --transport=http

# Specify a custom port
npx -y dodopayments-mcp --transport=http --port=3000
```

### 远程服务器配置

部署后，客户端可以使用服务器 URL 连接：

```json
{
  "mcpServers": {
    "dodopayments_api": {
      "url": "http://localhost:3000",
      "headers": {
        "Authorization": "Bearer your_api_key"
      }
    }
  }
}
```

### 授权头部

远程服务器接受以下头部中的身份验证：

| 头部 | 描述 |
|--------|-------------|
| `Authorization` | Bearer 令牌认证 |
| `x-dodo-payments-api-key` | 直接 API 密钥头 |

## 安全最佳实践

代码模式通过在沙箱环境中执行代码并在服务器端注入 API 密钥来提供固有安全性。请遵循以下额外最佳实践以保护凭据。

<AccordionGroup>
<Accordion title="API Key Management">
**切勿将凭据提交到版本控制系统**

将 API 密钥存储在环境变量或安全的密钥管理系统中。

```bash
# Use environment variables
export DODO_PAYMENTS_API_KEY="dodo_test_..."

# Use a .env file (add to .gitignore)
echo "DODO_PAYMENTS_API_KEY=dodo_test_..." > .env
```

**定期轮换密钥**

定期生成新的 API 密钥，并通过 Dodo Payments 仪表板撤销旧密钥。

**开发过程中使用测试密钥**

开发时始终使用测试模式 API 密钥，以免影响生产数据。
</Accordion>

<Accordion title="Access Control">
**为远程服务器实施身份验证**

部署远程服务器时，请始终通过 `Authorization` 头或 `x-dodo-payments-api-key` 头强制身份验证。

**监控 API 使用情况**

通过 Dodo Payments 仪表板跟踪 MCP 服务器活动，并为异常模式设置警报。
</Accordion>

<Accordion title="Network Security">
**为远程服务器使用 HTTPS**

始终将远程 MCP 服务器部署在 HTTPS 端点后面。

**实施速率限制**

通过在 MCP 服务器和 API 级别实施速率限制来防止滥用。

**限制网络访问**

配置防火墙规则限制可以连接到 MCP 服务器的客户端。
</Accordion>
</AccordionGroup>

## 故障排除

<AccordionGroup>
<Accordion title="Connection issues">
**验证您的 API 密钥**

确保您的 API 密钥设置正确并具有必要的权限。

```bash
# Test your API key
curl -H "Authorization: Bearer dodo_test_..." \
  https://test.dodopayments.com/payments
```

**检查网络连接**

验证您能否访问 Dodo Payments API 端点。

**查看客户端日志**

启用 MCP 客户端的详细日志以诊断连接问题。
</Accordion>

<Accordion title="Authentication errors">
**确认 API 密钥环境**

确保您在测试端点使用测试密钥，在生产端点使用生产密钥。

**检查环境变量**

验证 `DODO_PAYMENTS_ENVIRONMENT` 是否设置正确（生产环境为 `live_mode`）。

**重新生成凭据**

如果问题仍然存在，请通过仪表板生成新的 API 密钥。
</Accordion>

<Accordion title="Tool execution failures">
**验证输入参数**

确保 AI 助手为每个工具提供的参数格式正确。

**查看错误消息**

检查 API 返回的错误响应，以获取具体的故障指导。

**直接测试 API**

通过 curl 或 Postman 直接调用 Dodo Payments API，确认操作是否可行。
</Accordion>
</AccordionGroup>

## 为什么选择代码模式

传统的 MCP 实现常常陷入“工具泛滥”时，每个 API 端点都作为单独工具暴露。代码模式在多个方面是更优的方式：

### 与调用工具相比，LLM 更擅长写代码
LLM 已在数百万行真实世界代码上训练，使其天生擅长编写脚本。而调用工具通常基于合成示例。

<Info>
“让 LLM 通过调用工具执行任务，就像让莎士比亚参加一个月的汉语课程，然后要求他用汉语写一部剧。” — [Cloudflare](https://blog.cloudflare.com/code-mode/)
</Info>

### 消除上下文窗口膨胀
在传统方法中，每个工具定义在对话开始前就消耗 tokens。暴露 50+ 个工具可以轻松消耗 55K–100K+ tokens。Anthropic 发现工具定义在优化前可能消耗多达 134K tokens。

使用代码模式，仅加载 2 个工具（约 1K tokens）。代理按需搜索所需文档。Anthropic 的工具搜索工具保留了 95% 的上下文窗口，将开销从 77K 降低到 8.7K tokens。

### 通过编程编排减少延迟
传统工具调用要求每次操作都进行完整模型推理往返。如果任务需要 20 次 API 调用，那就要 20 次往返。在代码模式中，代理编写一个脚本执行所有调用，并只返回最终结果。

Anthropic 观察到这种编程方法令 token 数量减少 37%，准确率提升（知识检索从 25.6% 提高至 28.5%）。

### 设计上更安全
代码模式提供固有的安全优势：
- **参数中无 API 密钥**：API 密钥在服务器端注入，从不在发送给 LLM 的工具参数中暴露。
- **隔离的沙箱**：代码在安全环境中运行，无法访问网络或主机文件系统。
- **受控的 SDK**：只有授权的 SDK 方法对代理可用。

### 可扩展到任何 API 规模
随着 API 的增长，传统 MCP 性能会下降，因为必须在上下文中加载更多工具。无论 API 表面多大，代码模式始终保持 2 个工具。Cloudflare 成功将 2,500 多个 API 端点压缩到仅 2 个工具和约 1,000 tokens 的上下文中。

<Info>
有关此架构优势的更多详情，请参阅 [Anthropic](https://www.anthropic.com/engineering/advanced-tool-use) 及 [Cloudflare](https://blog.cloudflare.com/code-mode/) 的工程博客，以及 Claude 的 [编程式工具调用](https://platform.claude.com/docs/en/agents-and-tools/tool-use/programmatic-tool-calling) 文档。
</Info>
