## 概述

一个适用于生产的订阅模板，使用 Next.js 15、React 19、Supabase、Drizzle ORM 和 Dodo Payments。它配备了 Google OAuth、订阅结账、Webhook 处理、数据库架构和基本仪表板。

<Info>
如果您只需要现有应用的路由处理程序，请参阅专用适配器：<a href="/developer-resources/nextjs-adaptor">Next.js Adaptor</a> 和 <a href="/developer-resources/express-adaptor">Express Adaptor</a>。
</Info>

## 先决条件

- Node.js 18+（或 Bun 1.0+）
- Supabase 项目（URL、匿名密钥、服务角色密钥、数据库 URL）
- Dodo Payments 账户（API 密钥、Webhook 密钥）
- Google Cloud OAuth 客户端（客户端 ID 和密钥）

## 快速开始

<Steps>
{/* LOCKED_PATTERN_587a3a4c926aec89f48dfc1e7c21054e */}

```bash
git clone https://github.com/dodopayments/dodo-supabase-subscription-starter.git
cd dodo-supabase-subscription-starter
# choose one
bun install
# or
npm install
# or
pnpm install
```

</Step>

{/* LOCKED_PATTERN_ea54c61e84e28cd275c0eed6d7634e0f */}
创建 Supabase 项目并复制：
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
- SUPABASE_SERVICE_ROLE_KEY
- DATABASE_URL（连接字符串）
</Step>

{/* LOCKED_PATTERN_42bf2f0f0010fd47b77c88325d34313c */}
在 Google Cloud 中将重定向 URI 设置为：`https://[your-project-ref].supabase.co/auth/v1/callback`，然后使用您的客户端 ID 和密钥在 Supabase Auth 中启用 Google 提供者。
</Step>

{/* LOCKED_PATTERN_f32b65d8a917a558befa771bede82115 */}
从 Dodo 仪表板生成 API 密钥和 Webhook 密钥。在开发期间将环境设置为 `test_mode`。
</Step>

{/* LOCKED_PATTERN_a34a8e162789e5a79d108d9eb3c86df5 */}

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role

# Database
DATABASE_URL=postgresql://postgres:[password]@db.[project-ref].supabase.co:5432/postgres

# Dodo Payments
DODO_PAYMENTS_API_KEY=your-dodo-api-key
DODO_WEBHOOK_SECRET=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT=test_mode
```

<Warning>
切勿提交秘密。部署环境中请使用环境变量。
</Warning>
</Step>

{/* LOCKED_PATTERN_c35fe33eae6aa68b4bc39bd740632964 */}

```bash
bun run db:push
# or
npm run db:push
# or
pnpm run db:push
```

<Check>
已创建的表：`users`、`subscriptions`、`payments`。
</Check>
</Step>

{/* LOCKED_PATTERN_560d1d81fad22bd0622f0c8dcb3c706d */}

```bash
# login (one-time)
bunx supabase login
# or
npx supabase login

# deploy the edge function
bun run deploy:webhook --project-ref [your-project-ref]
# or
npm run deploy:webhook -- --project-ref [your-project-ref]
# or
pnpm run deploy:webhook --project-ref [your-project-ref]
```

```bash cURL
curl -X POST \
  'https://[your-project-ref].supabase.co/functions/v1/dodo-webhook' \
  -H 'Content-Type: application/json' \
  -H 'Dodo-Signature: <signature>' \
  -d '{"type":"payment.succeeded","data":{}}'
```

</Step>

{/* LOCKED_PATTERN_dd5db8f3734010c91f70fdf794c4e2a2 */}
将端点 URL 设置为：

```text
https://[your-project-ref].supabase.co/functions/v1/dodo-webhook
```

选择支付和订阅事件。
</Step>

{/* LOCKED_PATTERN_43dfdbde6ac180daa9987aee2baf2da7 */}
在 Dodo 仪表板 → Products → Create Product。可选地添加元数据：

```json
{
  "features": ["Feature 1", "Feature 2", "Feature 3"]
}
```

定价 UI 会读取此 `features` 数组并动态渲染。
</Step>

{/* LOCKED_PATTERN_e437399605995e5a4845b4d749b71d01 */}

```bash
bun run dev
# or
npm run dev
# or
pnpm run dev
```

打开 http://localhost:3000。
</Step>
</Steps>

## 包含内容

- 通过 Supabase 进行身份验证（已配置 Google OAuth）
- 通过 Dodo Payments 进行订阅结账
- Supabase Edge 函数用于 Webhook（`dodo-webhook`）
- Drizzle ORM 模式和迁移
- 包含发票、订阅状态以及套餐功能的仪表板

<Tip>
在完成端到端测试之前，保持 `DODO_PAYMENTS_ENVIRONMENT` 为 `test_mode`。
</Tip>

## 验证和故障排除

<AccordionGroup>
{/* LOCKED_PATTERN_6b286e9e2bbacd5c7684f8442203f10e */}
- 确保 `DODO_WEBHOOK_SECRET` 与 Dodo 仪表板中的值匹配
- 确认您已部署最新的 `dodo-webhook` 函数
- 验证您的函数中的头名称是否正确（Dodo-Signature）
</Accordion>
{/* LOCKED_PATTERN_c270a2f8797d0168d608bb07bd40f056 */}
- 检查 `DATABASE_URL` 语法及 Supabase 网络出口
- 在项目创建后等待约 2–3 分钟再进行首次推送
</Accordion>
{/* LOCKED_PATTERN_276d2f8137bb346c423586f69d6b7cae */}
- 重定向 URI 必须为 `https://[ref].supabase.co/auth/v1/callback`
- 确保 Google Cloud 和 Supabase Auth 提供者中也一致
</Accordion>
</AccordionGroup>
