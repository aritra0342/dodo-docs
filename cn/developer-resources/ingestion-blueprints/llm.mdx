## 快速开始

仅需 2 分钟即可开始使用自动 LLM 令牌跟踪：

<Steps>
{/* LOCKED_PATTERN_776027320500bde6b99bac6bed1cc64d */}
安装Dodo Payments摄取蓝图：

```bash
npm install @dodopayments/ingestion-blueprints
```

</Step>

{/* LOCKED_PATTERN_e608972569617212f31c0b7d72677865 */}
您需要两个API密钥：

- **Dodo Payments API 密钥**：从 [Dodo Payments Dashboard](https://app.dodopayments.com/developer/api-keys) 获取
- **LLM 提供者 API 密钥**：来自 AI SDK、OpenAI、Anthropic、Groq 等。

<Tip>
将API密钥安全地存储在环境变量中，切勿提交到版本控制。
</Tip>
</Step>

{/* LOCKED_PATTERN_3c26a22b684f0fb6b635e5bf1e61dd55 */}
在跟踪使用情况之前，请在Dodo Payments仪表板中创建一个仪表：

1. **登录** 到 [Dodo Payments Dashboard](https://app.dodopayments.com/)
2. **转到** 产品 → 仪表
3. **点击** “Create Meter”
4. **配置您的仪表**：
   - **仪表名称**：选择一个描述性名称（例如“LLM Token Usage”）
   - **事件名称**：设置唯一事件标识符（例如 `llm.chat_completion`）
   - **聚合类型**：选择 `sum` 来累加令牌计数
   - **Over 属性**：选择要跟踪的内容：
     - `inputTokens` - 跟踪输入/提示令牌
     - `outputTokens` - 跟踪输出/完成令牌（在适用时包括推理令牌）
     - `totalTokens` - 跟踪输入 + 输出的总令牌

<Info>
您在此处设置的 **事件名称** 必须与您传递给SDK的名称完全匹配（区分大小写）。
</Info>

有关详细说明，请参阅[Usage-Based Billing Guide](/developer-resources/usage-based-billing-guide)。
</Step>

## 配置

### 跟踪器配置

在应用程序启动时创建一个跟踪器，并提供以下必需参数：

<ParamField path="apiKey" type="string" required>
您的Dodo Payments API密钥。从[API Keys page](https://app.dodopayments.com/developer/api-keys)获取。
</ParamField>

```javascript
apiKey: process.env.DODO_PAYMENTS_API_KEY
```

</ParamField>

<ParamField path="environment" type="string" required>
跟踪器的环境模式。

- `test_mode` - 用于开发和测试
- `live_mode` - 用于生产

```javascript
environment: 'test_mode' // or 'live_mode'
```

<Warning>
开发过程中始终使用 `test_mode`，以避免影响生产指标。
</Warning>
</ParamField>

<ParamField path="eventName" type="string" required>
触发仪表的事件名称。必须与您在Dodo Payments仪表中配置的名称完全一致（区分大小写）。

```javascript
eventName: 'llm.chat_completion'
```

<Info>
此事件名称将您跟踪的使用情况与正确的仪表关联以进行计费计算。
</Info>
</ParamField>

### 包装器配置

在包装您的 LLM 客户端时，提供以下参数：

<ParamField path="client" type="object" required>
您的LLM客户端实例（OpenAI、Anthropic、Groq等）。

```javascript
client: openai
```

</ParamField>

<ParamField path="customerId" type="string" required>
用于计费的唯一客户标识符。应与您在Dodo Payments中的客户ID匹配。

```javascript
customerId: 'customer_123'
```

<Tip>
使用您的应用用户ID或客户ID，以确保每位客户的计费准确。
</Tip>
</ParamField>

<ParamField path="metadata" type="object">
附加数据是可选的，用于附加到跟踪事件。对于筛选和分析非常有用。


```javascript
metadata: {
  feature: 'chat',
  userTier: 'premium',
  sessionId: 'session_123',
  modelVersion: 'gpt-4'
}
```

</ParamField>

### 完整配置示例

<CodeGroup>

```javascript Full Configuration
import { createLLMTracker } from "@dodopayments/ingestion-blueprints";
import { generateText } from "ai";
import { google } from "@ai-sdk/google";
import "dotenv/config";

async function aiSdkExample() {
  console.log("🤖 AI SDK Simple Usage Example\n");

  try {
    // 1. Create tracker
    const llmTracker = createLLMTracker({
      apiKey: process.env.DODO_PAYMENTS_API_KEY!,
      environment: "test_mode",
      eventName: "your_meter_event_name",
    });

    // 2. Wrap the ai-sdk methods
    const client = llmTracker.wrap({
      client: { generateText },
      customerId: "customer_123",
      metadata: {
        provider: "ai-sdk",
      },
    });

    // 3. Use the wrapped function
    const response = await client.generateText({
      model: google("gemini-2.5-flash"),
      prompt: "Hello, I am a cool guy! Tell me a fun fact.",
      maxOutputTokens: 500,
    });

    console.log(response);
    console.log(response.usage);
    console.log("✅ Automatically tracked for customer\n");
  } catch (error) {
    console.error(error);
  }
}

aiSdkExample().catch(console.error);
```

</CodeGroup>

<Info>
**自动跟踪：** SDK会在后台自动跟踪令牌使用情况，而不会修改响应。您的代码保持简洁，与使用原始提供商SDK完全一致。
</Info>

### Express.js API 集成

将 LLM 跟踪集成到 Express.js API 的完整示例：

<CodeGroup>

```javascript Express.js Server
import express from 'express';
import { createLLMTracker } from '@dodopayments/ingestion-blueprints';
import OpenAI from 'openai';

const app = express();
app.use(express.json());

// Initialize OpenAI client
const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

// Create tracker once at startup
const tracker = createLLMTracker({
  apiKey: process.env.DODO_PAYMENTS_API_KEY,
  environment: process.env.NODE_ENV === 'production' ? 'live_mode' : 'test_mode',
  eventName: 'api.chat_completion'
});

// Chat endpoint with automatic tracking
app.post('/api/chat', async (req, res) => {
  try {
    const { message, userId } = req.body;
    
    // Validate input
    if (!message || !userId) {
      return res.status(400).json({ error: 'Missing message or userId' });
    }
    
    // Wrap client for this specific user
    const trackedClient = tracker.wrap({
      client: openai,
      customerId: userId,
      metadata: { 
        endpoint: '/api/chat',
        timestamp: new Date().toISOString()
      }
    });
    
    // Make LLM request - automatically tracked
    const response = await trackedClient.chat.completions.create({
      model: 'gpt-4',
      messages: [{ role: 'user', content: message }],
      temperature: 0.7
    });
    
    const completion = response.choices[0].message.content;
    
    res.json({ 
      message: completion,
      usage: response.usage
    });
  } catch (error) {
    console.error('Chat error:', error);
    res.status(500).json({ error: 'Internal server error' });
  }
});

app.listen(3000, () => {
  console.log('Server running on port 3000');
```

## 跟踪内容

每个 LLM API 调用会自动向 Dodo Payments 发送一个使用事件，结构如下：

<CodeGroup>

```json Event Structure
{
  "event_id": "llm_1673123456_abc123",
  "customer_id": "customer_123",
  "event_name": "llm.chat_completion",
  "timestamp": "2024-01-08T10:30:00Z",
  "metadata": {
    "inputTokens": 10,
    "outputTokens": 25,
    "totalTokens": 35,
    "model": "gpt-4",
  }
}
```

</CodeGroup>

### 事件字段

<ParamField path="event_id" type="string">
此特定事件的唯一标识符。由SDK自动生成。


格式：`llm_[timestamp]_[random]`
</ParamField>

<ParamField path="customer_id" type="string">
您在包装客户端时提供的客户ID。用于计费。
</ParamField>

<ParamField path="event_name" type="string">
触发仪表的事件名称。与跟踪器配置匹配。
</ParamField>

<ParamField path="timestamp" type="string">
事件发生的ISO 8601时间戳。
</ParamField>

<ParamField path="metadata" type="object">
令牌使用和附加跟踪数据：

- `inputTokens` - 使用的输入/提示令牌数量
- `outputTokens` - 使用的输出/完成令牌数量（在适用时包括推理令牌）
- `totalTokens` - 总令牌数（输入 + 输出）
- `model` - 使用的LLM模型（例如“gpt-4”）
- `provider` - LLM提供商（如果包装元数据中包含）
- 您在包装客户端时提供的任何自定义元数据

- `inputTokens` - 使用的输入/提示令牌数量
- `outputTokens` - 使用的输出/完成令牌数量（在适用时包括推理令牌）
- `totalTokens` - 总令牌（输入 + 输出）
- `model` - 使用的 LLM 模型（例如，"gpt-4"）
- `provider` - LLM 提供者（如果包含在包装元数据中）
- 您在包装客户端时提供的任何自定义元数据

<Note>
**推理令牌：** 对于具有推理能力的模型，`outputTokens` 会自动同时包含完成令牌和推理令牌。
</Note>
</ParamField>

</Info>
