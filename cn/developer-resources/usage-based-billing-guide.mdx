## 创建计量器

计量器定义了如何聚合和测量您的使用事件以进行计费。

在创建计量器之前，规划您的使用跟踪策略：
- 确定您想要跟踪的使用事件
- 确定事件应如何聚合（计数、求和等）
- 定义特定用例的任何过滤要求

## 在产品中链接计量器

创建计量器后，您需要将其链接到产品，以启用基于使用的计费。此过程将计量器的使用数据与客户计费的定价规则连接起来。

将计量器链接到产品建立了使用跟踪与计费之间的连接：
- 产品定义定价规则和计费行为
- 计量器提供计费计算所需的使用数据
- 可以将多个计量器链接到单个产品，以实现复杂的计费场景

### 产品配置过程

通过正确配置产品设置，将您的使用数据转化为可计费费用：

<Steps>
{/* LOCKED_PATTERN_c6c0a713c4c40a6d7afb5356f839b072 */}
转到产品创建或编辑页面，并选择 **Usage-Based** 作为产品类型。


</Step>

{/* LOCKED_PATTERN_96b3ccc2ef72fc5c11840c638f374759 */}
点击 **Associated Meter** 打开侧边的计量器选择面板。


该面板可让您配置哪些计量器将跟踪该产品的用量。
</Step>

{/* LOCKED_PATTERN_80a284cd85458e962fc3cec7c255149a */}
在计量器选择面板中：


1. 单击 **添加计量器** 查看可用计量器
2. 从下拉列表中选择您创建的计量器
3. 选定的计量器将出现在您的产品配置中

</Step>

{/* LOCKED_PATTERN_6b0ad6e340264c8baa56736989bc3e50 */}
为每个由计量器跟踪的用量单位设置价格。


{/* LOCKED_PATTERN_213cd3b101c5416e3cd966a3d8652d15 */}
确定每个计量器测量的单位应收取的费用。


**示例**：将 `$0.50` 每单位计费意味着：
- 消耗 1,000 单位 = 1,000 × $0.50 = 收费 500.00
- 消耗 500 单位 = 500 × $0.50 = 收费 250.00
- 消耗 100 单位 = 100 × $0.50 = 收费 50.00
</ParamField>

</Step>

{/* LOCKED_PATTERN_c1485ee2c65c25ecef83c7e0c270509b */}
配置计费前的免费用量配额。


{/* LOCKED_PATTERN_a57cc2829e5cf6a714d2d75f1b5361ce */}
客户在开始付费用量计算之前可免费使用的单位数。


**运作方式**：
- 免费门槛：100 单位
- 单位价格：$0.50
- 客户用量：250 单位
- **计算**：(250 - 100) × $0.50 = **$75.00** 收费
</ParamField>

<Info>
免费门槛适用于免费增值模式、试用期或为客户提供计划中包含的基础配额。
</Info>

<Check>
免费门槛适用于每个计费周期，为客户提供按月或根据您的计费安排刷新的一次性配额。
</Check>
</Step>

{/* LOCKED_PATTERN_08395c29a4a5c780a5cf3de53acf78b9 */}
查看计量器与定价配置，然后点击 **Save Changes** 完成设置。


<Check>
您的产品现已配置为基于用量计费，并将根据实际消耗自动向客户收费。
</Check>

**接下来会发生什么**：
- 将发送到计量器的用量事件会被跟踪并汇总
- 计费计算将自动应用您的定价规则
- 客户将在每个计费周期根据实际消耗计费
</Step>
</Steps>

<Note>
请记住，每个产品最多可以添加 10 个计量器，从而在多个维度（如 API 调用、存储、计算时间和自定义指标）上实现复杂的用量跟踪。
</Note>

## 发送使用事件

一旦您的计量器配置完成，您可以开始从应用程序发送使用事件以跟踪客户使用情况。

### 事件结构

每个使用事件必须包含以下必填字段：

<ParamField body="event_id" type="string" required>
此特定事件的唯一标识符。必须在所有事件中唯一。
</ParamField>

<ParamField body="customer_id" type="string" required>
应将此用量归属到的 Dodo Payments 客户 ID。
</ParamField>

<ParamField body="event_name" type="string" required>
与您的计量器配置匹配的事件名称。事件名称会触发相应计量器。
</ParamField>

<ParamField body="timestamp" type="string">
事件发生的 ISO 8601 时间戳。如果未提供，则默认为当前时间。
</ParamField>

<ParamField body="metadata" type="object">
用于过滤和汇总的附加属性。请包含任何在计量器的“Over Property”或过滤条件中引用的值。
</ParamField>

### 使用事件 API 示例

使用事件 API 将使用事件发送到您配置的计量器：

<CodeGroup>

### 常见问题

大多数基于使用的计费问题都属于以下类别：
- 事件交付和处理问题
- 计量器配置问题
- 数据类型和格式错误
- 客户 ID 和身份验证问题

### 调试步骤

1. 验证事件交付在事件分析选项卡中

3. 验证客户 ID 和 API 身份验证
