## 概述

一个适用于将 Dodo Payments 集成到 Expo/React Native 应用程序的生产就绪模板。该模板包括结账会话创建、支付处理和适当的 TypeScript 配置，帮助您快速开始在移动应用中接受支付。

<Info>
该样板使用 Expo SDK 和 TypeScript，并包含用于结账会话和支付处理的示例实现。
</Info>

## 先决条件

- Node.js 18+
- Expo CLI 或 EAS CLI
- Dodo Payments 账户（来自仪表板的 API 密钥）
- iOS 模拟器或 Android 模拟器（或物理设备）

## 快速开始

<Steps>
{/* LOCKED_PATTERN_3810046f56651f3ba0e1684e009cd2e3 */}
克隆样板仓库并进入项目目录：

```bash
git clone https://github.com/dodopayments/expo-boilerplate.git
cd expo-boilerplate
```

</Step>

{/* LOCKED_PATTERN_985ffb13172d607786ae5265635e02cf */}
在测试模式下从 Dodo Payments 仪表板获取凭证：

- **API 密钥**：在您的 [仪表板](https://app.dodopayments.com) 中导航到 **开发者 → API 密钥**

<Tip>
从测试模式开始，安全地测试集成而无需处理真实支付。
</Tip>
</Step>

{/* LOCKED_PATTERN_f8f1c4a54590eca96bda1740ea80800b */}
在根目录下创建 `.env` 文件：

```bash
touch .env
```

添加您的 Dodo Payments 凭据：

```bash .env
EXPO_PUBLIC_DODO_API_KEY=your_test_api_key_here
```

<Warning>
切勿将 `.env` 文件提交到版本控制。它已包含在 `.gitignore` 中。
</Warning>
</Step>

{/* LOCKED_PATTERN_5e9c44b19c1da0e28673e0b0f87e9162 */}
安装所需的 npm 包：

```bash
npm install
```

</Step>

{/* LOCKED_PATTERN_e0b1f8ca8b418cb641bfae6fc3ff28bc */}
启动 Expo 开发服务器：

```bash
npx expo start
```

使用 Expo Go 应用扫描二维码，或按 `i` 以在 iOS 模拟器上运行，或 `a` 以在 Android 模拟器上运行。

<Check>
你已经可以开始构建了！该样板包含示例屏幕和组件，帮助你理解集成过程。
</Check>
</Step>
</Steps>

## 项目结构

```
expo-boilerplate/
├── app/
│   ├── (tabs)/
│   │   └── index.tsx          # Home screen with examples
│   └── checkout.tsx           # Checkout screen
├── components/
│   └── PaymentButton.tsx      # Payment button component
├── lib/
│   └── dodo.ts                # Dodo Payments client setup
├── .env.example               # Environment variables template
└── package.json
```

## 为生产构建

<Steps>
{/* LOCKED_PATTERN_8da0a64f26eca1e80276896a6999817f */}

```bash
npm install -g eas-cli
```

</Step>

{/* LOCKED_PATTERN_68d4159ac59cdefe3384c2094bbecbd9 */}

```bash
eas build:configure
```

</Step>

{/* LOCKED_PATTERN_46d8869736258162472d6c37d4e5967b */}
对于 iOS：

```bash
eas build --platform ios
```

对于 Android：

```bash
eas build --platform android
```

</Step>
</Steps>
