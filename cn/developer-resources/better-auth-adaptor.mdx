# 概述

<b>Better Auth Adapter</b> 为 Dodo Payments 提供：

- 注册时自动创建客户
- 结账会话（首选）与产品 slug 映射
- 自助客户门户
- 用于基于使用量计费的计量使用摄取和报告端点
- 实时 webhook 事件处理和签名验证
- 完全支持 TypeScript

<Note>
  要使用此集成，您需要一个 Dodo Payments 帐户和 API 密钥。
</Note>

# 先决条件

- Node.js 20+
- 访问您的 Dodo Payments 仪表板
- 使用 [better-auth](https://www.npmjs.com/package/better-auth) 的现有项目

# 设置

<Steps>
<Step title="Configure environment variables">
将这些添加到您的 <code>.env</code> 文件：

```env expandable
DODO_PAYMENTS_API_KEY=your_api_key_here
DODO_PAYMENTS_WEBHOOK_SECRET=your_webhook_secret_here
BETTER_AUTH_URL=http://localhost:3000
BETTER_AUTH_SECRET=your_better_auth_secret_here
```

<Warning>
切勿将 API 密钥或密钥提交到版本控制。
</Warning>
</Step>

<Step title="Set up server-side integration">
创建或更新 <code>src/lib/auth.ts</code>：

```typescript expandable
import { betterAuth } from "better-auth";
import {
  dodopayments,
  checkout,
  portal,
  webhooks,
  usage,
} from "@dodopayments/better-auth";
import DodoPayments from "dodopayments";

export const dodoPayments = new DodoPayments({
  bearerToken: process.env.DODO_PAYMENTS_API_KEY!,
  environment: "test_mode", // or "live_mode" for production
});

export const { auth, endpoints, client } = BetterAuth({
  plugins: [
    dodopayments({
      client: dodoPayments,
      createCustomerOnSignUp: true,
      use: [
        checkout({
          products: [
            {
              productId: "pdt_xxxxxxxxxxxxxxxxxxxxx",
              slug: "premium-plan",
            },
          ],
          successUrl: "/dashboard/success",
          authenticatedUsersOnly: true,
        }),
        portal(),
        webhooks({
          webhookKey: process.env.DODO_PAYMENTS_WEBHOOK_SECRET!,
          onPayload: async (payload) => {
            console.log("Received webhook:", payload.event_type);
          },
        }),
        usage(),
      ],
    }),
  ],
});
```

<Tip>
在生产环境中将 <code>environment</code> 设置为 <code>live_mode</code>。
</Tip>
</Step>

<Step title="Set up client-side integration">
创建或更新 <code>src/lib/auth-client.ts</code>：

```typescript expandable
import { createAuthClient } from "better-auth/react";
import { dodopaymentsClient } from "@dodopayments/better-auth";

export const authClient = createAuthClient({
  baseURL: process.env.BETTER_AUTH_URL || "http://localhost:3000",
  plugins: [dodopaymentsClient()],
});
```

</Step>
</Steps>

## 跟踪计量使用

在服务器上启用 <code>usage()</code> 插件以捕获计量事件，并让客户检查他们的基于使用量的计费。

- <code>authClient.dodopayments.usage.ingest</code> 为已登录、电子邮件已验证的用户摄取事件。
- <code>authClient.dodopayments.usage.meters.list</code> 列出与该客户订阅相关的最近使用情况。

```typescript expandable
// Record a metered event (e.g., an API request)
const { error: ingestError } = await authClient.dodopayments.usage.ingest({
  event_id: crypto.randomUUID(),
  event_name: "api_request",
  metadata: {
    route: "/reports",
    method: "GET",
  },
  // Optional Date; defaults to now if omitted
  timestamp: new Date(),
});

if (ingestError) {
  console.error("Failed to record usage", ingestError);
}

// List recent usage for the current customer
const { data: usage, error: usageError } =
  await authClient.dodopayments.usage.meters.list({
    query: {
      page_size: 20,
      meter_id: "mtr_yourMeterId", // optional
    },
  });

if (usage?.items) {
  usage.items.forEach((event) => {
    console.log(event.event_name, event.timestamp, event.metadata);
  });
}
```

<Tip>
  在摄取用量数据时，早于一小时前或超过五分钟以后的时间戳会被拒绝。
</Tip>
<Tip>
  如果在列出使用量计量器时省略 <code>meter_id</code>，则会返回与客户订阅相关的所有计量器。
</Tip>

# Webhooks

<Info>
  webhooks 插件会处理来自 Dodo Payments 的实时支付事件，并进行安全的签名验证。
  默认端点是 <code>/api/auth/dodopayments/webhooks</code>。
</Info>

<Steps>
<Step title="Generate and set webhook secret">
在 Dodo Payments 仪表板为您的端点 URL（例如 <code>https://&lt;your-domain&gt;/api/auth/dodopayments/webhooks</code>）生成一个 webhook 密钥，并将其设置到您的 <code>.env</code> 文件中：

```env
DODO_PAYMENTS_WEBHOOK_SECRET=your_webhook_secret_here
```

</Step>

<Step title="Handle webhook events">
示例处理程序：

```typescript expandable
webhooks({
  webhookKey: process.env.DODO_PAYMENTS_WEBHOOK_SECRET!,
  onPayload: async (payload) => {
    console.log("Received webhook:", payload.event_type);
  },
});
```

</Step>
</Steps>

# 配置参考

<AccordionGroup>
  <Accordion title="Plugin Options">
    - <b>client</b> (required): DodoPayments 客户端实例 -{" "}
    <b>createCustomerOnSignUp</b> (optional): 在用户注册时自动创建客户 -{" "}
    <b>use</b> (required): 要启用的插件数组（checkout、portal、usage、webhooks）
  </Accordion>

    - <b>products</b>: 产品数组或返回产品的异步函数 -{" "}
    <b>successUrl</b>: 成功支付后重定向的 URL -{" "}

  </Accordion>
</AccordionGroup>

# 故障排除与提示

<AccordionGroup>
  <Accordion title="Common Issues">
    - <b>无效的 API 密钥</b>：请仔细检查 <code>.env</code> 中的 <code>DODO_PAYMENTS_API_KEY</code>。 -{" "}
    <b>Webhook 签名不匹配</b>：确保 webhook 密钥与 Dodo Payments 仪表板中设置的一致。 -{" "}
    <b>客户未创建</b>：确认 <code>createCustomerOnSignUp</code> 已设置为 <code>true</code>。
  </Accordion>
  <Accordion title="Best Practices">
    - 对所有密钥和凭证使用环境变量。 -{" "}
    在切换到 <code>live_mode</code> 之前先在 <code>test_mode</code> 中测试。 -{" "}
    记录 webhook 事件以便调试和审计。
  </Accordion>
</AccordionGroup>
