<Note>
本指南仅涵盖涉及卡片、Apple Pay 和 Google Pay 的一次性付款。
</Note>

## API 端点

| 环境 | 基本 URL | 确认付款端点 |
| --- | --- | --- |
| **测试模式** | `https://test.dodopayments.com` | `https://api.custom.hs.dodopayments.com/payments/<payment_id>/confirm` |
| **实时模式** | `https://live.dodopayments.com` | `https://api.hs.dodopayments.com/payments/<payment_id>/confirm` |

## 分步卡片支付流程

### 第 1 步：获取秘密密钥

首先使用以下 API 创建付款：

- **Payments**： [创建付款 API](/api-reference/payments/post-payments)

此 API 会返回 **`secret_key`**，后续步骤需要使用该值。

### 第 2 步：确认付款

通过调用 `payments confirm` 端点来确认付款：

- **测试模式**：`POST https://api.custom.hs.dodopayments.com/payments/<payment_id>/confirm`
- **实时模式**：`POST https://api.hs.dodopayments.com/payments/<payment_id>/confirm`

在请求正文中包含以下详细信息：

- `client_secret`：第 1 步中的秘密密钥
- `payment_method`：设置为 `card`
- `payment_method_data`：卡片和账单详细信息
- `browser_info`：特定于浏览器的数据（用户代理、屏幕细节等）
- `payment_type`：明确设置为 `normal`（用于一次性付款）

所需头部：

- 从 DodoPayments 支持获取您的可发布密钥
- `api-key`：您的可发布密钥（向 Dodo Payments 团队获取）
- `content-type`：`application/json`

**示例请求正文：**

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "card",
  "payment_method_data": {
    "card": {
      "card_number": "<card_number>",
      "card_exp_month": "<exp_month>",
      "card_exp_year": "<exp_year>",
      "card_holder_name": "<cardholder_name>",
      "card_cvc": "<cvc>",
      "card_network": "<network>"
    },
    "billing": {
      "address": {
        "first_name": "<first_name>",
        "last_name": "<last_name>"
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
    "time_zone": "<time_zone>",
    "java_enabled": true,
    "java_script_enabled": true,
    "ip_address": "<ip_address_of_client>",
    "device_model": "<device_model>",
    "os_type": "<os_type>",
    "os_version": "<version>"
  },
  "payment_type": "normal"
}
```

### 处理响应

- 如果响应 **`status`** 是 **`requires_customer_action`**，请将客户重定向至 **`next_action`** 字段中提供的 URL 以进行进一步验证。

<Warning>
处理原始卡片数据时，请确保遵守 PCI 合规指南。
</Warning>

## Apple Pay 集成

要使用 Apple Pay，请将 `payment_method` 设置为 `wallet`，并在 `payment_method_data` 中包含 Apple Pay 数据。

### 请求正文结构

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "wallet",
  "payment_method_data": {
    "wallet": {
      "apple_pay": {
        "payment_data": "<decrypted_payment_data>",
        "payment_method": {
          "display_name": "Visa 1234",
          "network": "Visa",
          "type": "debit"
        },
        "transaction_identifier": "UNIQUE_TRANSACTION_ID_FROM_APPLE"
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
    "time_zone": "<time_zone>",
    "java_enabled": true,
    "java_script_enabled": true,
    "ip_address": "<ip_address_of_client>",
    "device_model": "<device_model>",
    "os_type": "<os_type>",
    "os_version": "<version>"
  },
  "payment_type": "normal"
}
```

### Apple Pay 字段参考

| 字段 | 类型 | 是否必需 | 描述 |
| --- | --- | --- | --- |
| `payment_data` | string | 是 | 此枚举用于表示 Apple Pay 支付数据，可以是加密或已解密的数据。 |
| `payment_method.display_name` | string | 是 | 显示在 Apple Pay 按钮上的名称（例如 “Visa 1234”） |
| `payment_method.network` | string | 是 | 卡网络（例如 “Visa”、“Mastercard”、“Amex”） |
| `payment_method.type` | string | 是 | 支付方式类型（例如 “debit”、“credit”） |
| `payment_method.card_exp_month` | string | 否 | 卡片到期月份（例如 “12”） |
| `payment_method.card_exp_year` | string | 否 | 卡片到期年份（例如 “25”） |
| `transaction_identifier` | string | 是 | 来自 Apple Pay 的唯一交易标识符 |

## Google Pay 集成

要使用 Google Pay，请将 `payment_method` 设置为 `wallet`，并在 `payment_method_data` 中包含 Google Pay 数据。

### 请求正文结构

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "wallet",
  "payment_method_data": {
    "wallet": {
      "google_pay": {
        "type": "CARD",
        "description": "Visa ending in 1234",
        "info": {
          "card_network": "VISA",
          "card_details": "1234"
        },
        "tokenization_data": {
          "type": "PAYMENT_GATEWAY",
          "token": "<encrypted_payment_token>"
        }
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
    "time_zone": "<time_zone>",
    "java_enabled": true,
    "java_script_enabled": true,
    "ip_address": "<ip_address_of_client>",
    "device_model": "<device_model>",
    "os_type": "<os_type>",
    "os_version": "<version>"
  },
  "payment_type": "normal"
}
```

### Google Pay 字段参考

| 字段 | 类型 | 是否必需 | 描述 |
| --- | --- | --- | --- |
| `type` | string | 是 | 支付方式类型（通常为 “CARD”） |
| `description` | string | 是 | 面向用户的支付方式描述 |
| `info.card_network` | string | 是 | 卡网络（例如 “VISA”、“MASTERCARD”、“AMEX”） |
| `info.card_details` | string | 是 | 卡的后 4 位数字 |
| `info.assurance_details` | object | 否 | 验证保障详细信息 |
| `tokenization_data.type` | string | 是 | 令牌化类型（例如 “PAYMENT_GATEWAY”） |
| `tokenization_data.token` | string | 是 | 来自 Google Pay 的加密支付令牌 |
