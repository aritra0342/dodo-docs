## API 密钥管理和身份验证

<Steps>
<Step title="访问 API 密钥">
在您的仪表板中导航到 **开发者 → API 密钥** 以管理您的密钥。
</Step>

<Step title="生成新密钥">
选择 **添加 API 密钥**，提供描述性名称，并配置写入权限：

- **启用写入权限**（已勾选）：对所有 API 操作具有完全的读写权限
- **启用写入权限**（未勾选）：只读访问 - 只能获取数据（支付、订阅、客户、产品）。无法创建或修改资源。

<Tip>
对于仪表板集成、分析工具和任何只需查看数据而不进行更改的系统，请取消勾选 "启用写入权限"。
</Tip>
</Step>

<Step title="安全存储您的密钥">
立即复制生成的密钥，并确保安全存储。
</Step>

<Step title="验证您的 API 请求">
使用您的 API 密钥来验证所有请求。应用以下授权格式：

```bash 
Authorization: Bearer YOUR_API_KEY
```

<Warning>
切勿在客户端代码或公共代码库中暴露您的秘密 API 密钥。
</Warning>
</Step>
</Steps>

## 响应格式

<CodeGroup>

```json Success Response
{
  "id": "pay_1234567890",
  "status": "completed",
  "amount": 2999,
  "currency": "USD",
  "created_at": "2024-01-15T10:30:00Z"
}
```

```json Error Response
{
  "code": "INVALID_REQUEST",
  "message": "The request contains invalid parameters"
}
```

</CodeGroup>

## 速率限制

- **标准**: 每个业务每分钟 1000 次请求
- **突发**: 每秒最多 100 次请求

<Note>
监控 `X-RateLimit-Remaining` 头部以跟踪使用情况。
</Note>
