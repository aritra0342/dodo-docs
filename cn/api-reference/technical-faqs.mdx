<Accordion title="Q1: 为什么我在支付成功的 webhook 中没有收到订阅产品的 product id？">
  A: 订阅产品的 payment.succeeded webhook 事件不包括 product_id。相反，它提供与订阅相关的 `subscription_id` 以及其他相关细节。
</Accordion>

<Accordion title="Q2: 我如何验证收到的 webhook 确实来自 Dodo Payments，而不是来自任何恶意行为者？">
  A: 要验证您收到的 webhook 确实来自 Dodo Payments 而不是恶意行为者，请按照以下步骤操作：
    1. **检查签名**：Dodo Payments 发送的每个 webhook 都在 **`webhook-signature`** 头中包含一个唯一的签名。此签名是使用您的 webhook 秘钥和事件的有效负载生成的。
    2. **检索秘钥**：确保您拥有 webhook 的秘钥，可以在 Dodo Payments 仪表板的 Webhooks 设置中找到。
    3. **重新计算签名**：在您的服务器上使用与 Dodo Payments 相同的签名算法（HMAC-SHA256）计算签名。使用 webhook 的有效负载和您的 webhook 秘钥。
    4. **比较签名**：将重新计算的签名与 **`webhook-signature`** 头中提供的签名进行比较。如果它们匹配，则 webhook 是有效的；如果不匹配，则应拒绝。
    5. **使用 HTTPS**：确保您的 webhook 端点使用 HTTPS 来保护通信并防止数据传输过程中的篡改。
  通过实施这些步骤，您可以自信地验证 Dodo Payments 发送的 webhook 的真实性。
</Accordion>

<Accordion title="Q3: 为什么我在第一个月的付款中也收到了 subscription.renew webhook？">
  A: 对于订阅，`subscription.renew` 每当扣除订阅金额时都会被触发。因此，是的，这是预期的行为。
</Accordion>

<Accordion title="Q5: 我如何在本地测试 webhooks？">
  A: 要在本地测试 webhooks，您可以按照以下步骤操作：
  
1. **使用隧道工具**：像 [ngrok](https://ngrok.com/)、[LocalTunnel](https://localtunnel.github.io/www/) 或 [Cloudflare Tunnel](https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/) 这样的工具可以将您的本地服务器暴露到互联网。这提供了一个公共 URL，Dodo Payments 可以使用该 URL 将 webhook 事件发送到您的本地环境。
2. **设置 webhook 端点**：在您的本地应用程序中创建一个 webhook 端点，以处理传入的 webhook 事件。确保它监听 HTTP POST 请求。
3. **在 Dodo Payments 中配置 webhook**：转到您的 Dodo Payments 仪表板，导航到 Webhooks 设置，并将 webhook URL 更新为隧道工具生成的公共 URL。
4. **触发事件**：在您的环境中执行操作以触发 webhook 事件。例如，您可以模拟付款或订阅续订。
5. **检查传入请求**：使用隧道工具的日志或您应用程序的日志检查传入的 webhook 有效负载。这使您能够验证您的端点是否接收事件并正确处理它们。
6. **验证签名**：确保您验证 **`webhook-signature`** 头，以确认 webhook 事件是有效的并且是由 Dodo Payments 发送的。

通过遵循这些步骤，您可以轻松地在本地开发环境中测试和调试 webhooks。
</Accordion>
