## 付款失败处理

使用 `on_payment_failure` 参数来控制计划变更付款失败时的处理方式：

| 值 | 行为 |
|-------|----------|
| `prevent_change` | 在付款成功之前保持订阅在当前计划上。计划变更保持待定状态。 |
| `apply_change` | 无论付款结果如何立即应用计划变更。这是默认行为。 |

 {/* LOCKED_PATTERN_6fa96040307d68e9fa44436559d63ee8 */}
如果未指定 `on_payment_failure`，则行为默认取决于仪表板中配置的业务级设置。
