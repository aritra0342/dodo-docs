## 新功能 🚀

1. **EUR & GBP Pricing - Full Rollout**  
   现在对 € EUR 和 £ GBP 的原生支持已全面部署到全球所有企业，实现这些主要货币的无缝定价和收款。
   
   您现在可以：
   - 使用 € 和 £ 为产品定价
   - 原生以这些货币收款
   - 直接以 € 和 £ 结算，无需外汇费用
   
   <Check>
   结果：在英国和欧洲实现更高的成功率和更好的接受度。
   </Check>

2. **Payment Method Management**  
   增强了针对订阅和客户的付款方式管理功能。您现在可以更新现有订阅的付款方式，并检索与某个客户关联的所有付款方式，从而更轻松地处理过期卡、付款失败和客户偏好更改。
   
   **为订阅更新付款方式：**
   - 在不中断的情况下更新活跃订阅的付款方式
   - 通过更新付款方式自动重新激活 `on_hold` 订阅
   - 使用现有保存的付款方式或添加新的付款方式
   - 重新激活订阅时自动创建剩余费用的扣款
   
   **获取客户付款方式：**
   - 列出客户所有保存的付款方式
   - 更新订阅时使用付款方式 ID
   - 显示可用付款选项，提升客户体验
   
   了解更多：[Update Payment Method API](/api-reference/subscriptions/update-payment-method) | [List Payment Methods API](/api-reference/customers/get-customer-payment-methods)

3. **客户和退款对象中的元数据支持**  
   现在客户和退款对象中可用元数据，扩展了您存储自定义信息和与外部系统集成的能力。
   - 将元数据附加到客户以进行 CRM 集成和自定义分类
   - 存储退款的额外上下文，以便更好地跟踪和报告
   - 与其他支持元数据的 Dodo Payments 对象保持一致
   
   了解更多：[元数据指南](/api-reference/metadata)

4. **Checkout Session Feature Flags for Customer Editing**  
   结账会话中的新功能标志可让您对客户在结账过程中可编辑的内容进行细粒度控制，从而提升灵活性和用户体验。
   - `allow_customer_editing_name`：允许客户编辑姓名
   - `allow_customer_editing_email`：允许修改电子邮件地址
   - `allow_customer_editing_street`：允许编辑街道地址
   - `allow_customer_editing_city`：启用城市字段编辑
   - `allow_customer_editing_state`：允许编辑州/省份
   - `allow_customer_editing_zipcode`：启用邮政编码编辑
   - `allow_customer_editing_country`：允许更改国家选择
   
   这些标志为结账体验提供了细致的控制，允许您根据业务需求定制客户可以修改的字段。

5. **结账许可证密钥和文件指示器**  
   结账页面现在在购买许可证密钥或数字文件时显示清晰的指示器，提高了透明度和客户信心。
   - 许可证密钥购买的视觉指示器
   - 清晰的文件下载指示器
   - 在结账过程中增强产品类型的可见性

6. **GitHub OAuth 登录页面**  
   通过 GitHub OAuth 集成增强了身份验证选项，为开发人员和技术用户提供了一种熟悉且安全的登录方式。
   - 使用 GitHub 账户登录
   - 为开发者账户简化身份验证
   - 为团队访问提供额外的安全层
