## 新功能 🚀

1. **混合购物车结账：单次和订阅产品同一购物车**  
   现在结账支持在单个购物车中组合一次性支付产品和订阅产品，解锁强大的新计费场景。
   
   <Frame>
   <img src="/images/changelog/mixed-checkout.png" alt="Mixed checkout with one-time and subscription products" />
   </Frame>
   
   ### 这意味着什么
   
   - **设置费 + 订阅**：在一次结账中收取一次性设置费和经常性订阅
   - **包含经常性组件的捆绑**：销售一个一次性产品，捆绑持续的服务访问（订阅）
   - **课程 + 会员组合**：提供课程购买，并附加可选的社区会员订阅
   - **终身访问 + 附加订阅**：将终生产品访问与可选的经常性高级功能结合起来
   
   ### 工作原理
   
   只需在结账会话中添加一次性产品和订阅产品即可：
   

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [
       // One-time setup fee
       { product_id: 'prod_setup_fee', quantity: 1 },
       // Monthly subscription
       { product_id: 'prod_monthly_plan', quantity: 1 }
     ],
     customer: {
       email: 'customer@example.com',
       name: 'Jane Doe'
     },
     return_url: 'https://yourapp.com/success'
   });
   ```

   
   <Check>
   结果：单一结账流程既处理一次性付款，又启动订阅 —— 减少摩擦并提升转化率。
   </Check>
   
   <Info>
   在混合产品类型时，一次性付款会立即扣款，而订阅则按照其配置的计费周期（包括您设置的任何试用期）进行。
   </Info>
   
   了解更多：[Checkout Sessions](/developer-resources/checkout-session)

2. **统一客户门户**  
   
   一个新的集中式客户门户，位于 [customer.dodopayments.com](https://customer.dodopayments.com)，客户可以在此查看并管理在 Dodo Payments 不同企业的所有购买和订阅。
   
   <Frame>
   <img src="/images/changelog/unified-customer-portal.png" alt="Unified Customer Portal" />
   </Frame>
   
   ### 门户功能
   
   - **跨企业可见性**：在一个地方查看所有使用 Dodo Payments 的企业的购买和订阅
   - **集中订阅管理**：从单一仪表板管理不同商家的订阅
   - **统一账单历史**：访问所有购买的发票和付款历史
   - **单点登录体验**：使用电子邮件登录一次，即可访问所有 Dodo Payments 商家的购买记录
   
   ### 工作原理
   
   客户只需访问 [customer.dodopayments.com](https://customer.dodopayments.com) 并输入他们的电子邮件地址。他们会收到一个安全登录链接，从而可以访问 Dodo Payments 生态系统中所有的购买记录。
   
   <Tip>
   统一客户门户补充了面向具体企业的客户门户。客户仍可访问各个商家门户以获得品牌体验，或使用统一门户查看整合视图。
   </Tip>
   
   <Note>
   统一门户使用与具体企业门户相同的基于电子邮件的认证，确保体验一致且安全。
   </Note>

3. **支付响应中的发票 ID**  
   支付响应现在包含 `invoice_id` 字段，使支付与发票对账和会计系统集成更方便。
   

   ```json
   {
     "payment_id": "pay_2IjeQm4hqU6RA4Z4kwDee",
     "status": "succeeded",
     "invoice_id": "inv_2IsUnWGtRKFLxk7xAQeyt",
     "amount": 9900,
     "currency": "USD",
     // ... other fields
   }
   ```

   
   ### 使用场景
   
   - **会计集成**：在会计软件中直接将支付与发票关联
   - **客户支持**：在处理支持请求时快速找到与支付对应的发票
   - **对账**：通过编程方式将支付匹配到发票，简化财务对账
   - **合规**：保持准确记录，将支付与其对应的税务发票关联
   
   <Info>
   `invoice_id` 字段在可用时使用印度特定的发票 ID，确保符合当地开票要求。
   </Info>
   
   了解更多：[Get Payment Detail API](/api-reference/payments/get-payments-1)
