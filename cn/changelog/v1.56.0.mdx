## 新功能 🚀

1. **全新入职体验**  
   入职流程现在更加简化，让您可以更快开始并接受付款，而无需不必要的延迟。
   - 更快的账户激活
   - 简化的验证流程

2. **双因素认证 (2FA)**  
   通过双因素认证增强账户安全性。为您的商户账户和团队成员提供超越密码的额外安全层。
   - 保护您的账户免受未授权访问
   - 支持身份验证器应用
   - 对于敏感操作是必需的

3. **叠加结账中的会话链接**  
   结账会话现在与叠加结账无缝集成，提供更流畅的嵌入式支付体验。创建结账会话并在叠加窗口中显示，而无需将客户重定向离开您的应用程序。

4. **C# SDK**  
   官方的 Dodo Payments C# SDK 现已推出，为您的 C# 和 .NET 应用程序集成支付功能提供原生 .NET 支持。
   - 完全的类型安全与 C# 接口
   - 所有 API 操作支持 Async/await
   - 可在 NuGet 上轻松安装

   了解更多： [Dodo Payments SDKs](/developer-resources/dodo-payments-sdks)

5. **所有 SDK 中的 Webhook 验证**  
   所有 SDK 现在都包含内置的 Webhook 验证功能，以安全地验证传入的 Webhook 事件。由行业标准的 [webhooks library](https://github.com/standard-webhooks) 提供支持，这些功能使得验证和处理 Webhook 负载变得简单。
   
   提供两种方法：
   - `unwrap()`：使用您的 Webhook 密钥验证 Webhook 签名以供生产使用
   - `unsafe_unwrap()`：在本地测试和开发中解析 Webhook 负载而不进行验证
   
   要使用 Webhook 验证，请在初始化 Dodo Payments 客户端时提供您的 `DODO_PAYMENTS_WEBHOOK_KEY`。这两个功能会自动解析 Webhook 负载并提取事件类型，使 Webhook 处理更加安全和方便。

## 改进与更新 ✨

- **在订阅账单详情中更新客户名称**  
  现在可以直接在订阅账单详情中修改客户名称，更容易纠正信息或更新记录。

- **获取结账会话 API**  
  以编程方式检索结账会话详情，以检查状态、验证会话或与您的分析工作流集成。

- **支持基于使用情况的订阅计划变更**  
  现在完全支持基于使用情况的计费模型的订阅计划变更，使得在保留使用情况跟踪的同时实现无缝升级和降级。

- **订阅响应中的税号**  
  现在在订阅 API 响应中包含税务识别号码，改善合规报告和发票生成。

- **在 API 中强制 3DS 标志**  
  在结账会话、支付和订阅 API 中增强 3D Secure 控制，现可使用强制 3DS 标志以满足更严格的身份验证要求。

- **按货币过滤余额分类账**  
  仪表板现在支持按货币过滤余额分类账条目，简化多货币操作的对账。

- **将支付阈值降低至 \$50**  
  商户现在可以将支付阈值设置为低至 \$50，为小型企业提供更多灵活性并更快地获取资金。

- **小幅改进和错误修复**  
  平台各处的常规增强和稳定性改进。
