## 新功能 🚀

### 客户钱包余额
  钱包余额现在会**自动应用于经常性订阅发票**，让您轻松：

  - 通过促销资金奖励忠实客户
  - 以钱包余额形式发放服务补偿
  - 在不干扰计费周期的情况下处理手动调整

  <Tip>
  使用客户钱包余额进行：
  - 促销激励
  - 服务中断赔偿
  - 预付账户余额
  </Tip>

  您可以**使用 API 为客户钱包添加或扣除资金**，实现客户支持、留存或忠诚度计划的自动化工作流。

  **示例用例：**
  - 在成功推荐后为客户的钱包充值
  - 作为自定义升级路径的一部分扣除钱包余额

  探索新的及更新的客户钱包 API：  
  - [获取客户钱包](/api-reference/customers/get-customer-wallets)  
  - [列出客户钱包分类账条目](/api-reference/customers/get-customer-wallets-ledger-entries)  
  - [创建用于增加和扣除资金的客户钱包分类账条目](/api-reference/customers/post-customer-wallets-ledger-entries)

   <Info>
  在结账过程中应用钱包余额的功能将在下一个版本中提供，允许客户在购买时直接兑换其钱包资金。
  </Info>

## 改进与更新 ✨

- **获取退款API中的客户详情**  
  退款检索端点现在返回相关的客户信息，简化了对账和支持工作流。  
  了解更多： [获取退款API](/api-reference/refunds/get-refunds-1)

- **计量器限制增加到50**  
  现在您可以为每个账户创建最多50个计量器，使得在您的产品和功能中进行更细致的使用跟踪成为可能。  
  了解更多： [获取计量器API](/api-reference/meters/get-meters)

- **激活许可证密钥API中的产品和客户上下文**  
  许可证激活响应现在包括详细的产品和客户上下文，改善了权限检查和下游集成。  
  了解更多： [激活许可证密钥API](/api-reference/licenses/activate-license)

- **订阅可以降级到\$0**  
  现在您可以将订阅降级到免费或$0计划，使得支持试用、社区层级和赞助变得更加容易。  
  管理订阅： [更新订阅API](/api-reference/subscriptions/patch-subscriptions)
