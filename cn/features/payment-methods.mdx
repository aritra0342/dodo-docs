## 工作原理

支付方式会根据以下内容自动向客户展示：

- **客户位置** — iDEAL（荷兰）或 UPI（印度）等区域性支付方式仅在客户位于这些地区时出现
- **交易货币** — 支付方式按支持的货币进行过滤
- **设备类型** — Apple Pay 仅在兼容的 Apple 设备和浏览器上显示
- **交易类型** — 一些支付方式不支持订阅或有最低金额要求

<Info>
**自动激活：**所有支付方式由 Dodo Payments 预配置并管理。无需额外设置——一旦账户通过验证并启用实时模式，即会可用。
</Info>

## 配置支付方式

默认情况下，会自动显示所有符合条件的支付方式。使用 `allowed_payment_method_types` 来限制显示的方式：

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://test.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

</Warning>

## 最佳实践

<AccordionGroup>
<Accordion title="Match methods to your market">
为目标市场包含地区支付方式。iDEAL 占荷兰在线支付约 60%；UPI 在印度占主导。忽略这些方式会显著影响转化率。
</Accordion>

<Accordion title="Always provide card fallbacks">
即使针对特定方式，也要包括 `credit` 和 `debit` 作为回退。并不是所有客户都能使用每种支付方式。
</Accordion>

<Accordion title="Consider transaction minimums">
BNPL 方式（Klarna、Afterpay）要求最低 50.01 美元。不要为低价商品包含它们——它们根本不会出现，这没问题。
</Accordion>

<Accordion title="Test across regions and devices">
上线前使用测试模式验证不同客户位置、币种和设备类型的支付方式可用性。
</Accordion>
</AccordionGroup>

## 故障排除

<AccordionGroup>
<Accordion title="Payment method not appearing">
**检查这些因素：**
1. 该方式是否包含在 `allowed_payment_method_types` 中？
2. 客户位置是否与方式支持的区域匹配？
3. 交易币种是否受该方式支持？
4. 交易金额是否达到最低要求（例如 BNPL 为 50.01 美元）？
5. 对于订阅：该方式是否支持定期付款？

**解决方案：**暂时移除 `allowed_payment_method_types`，以查看所有符合条件的方式，然后缩小问题范围。
</Accordion>

<Accordion title="Unexpected methods appearing">
**可能原因：**
- `allowed_payment_method_types` 未正确传递
- 参数格式不正确（必须是字符串数组）

**解决方案：**记录你的 API 请求有效负载并验证参数结构。
</Accordion>

<Accordion title="Regional method not available">
**检查：**
- 客户账单地址是否与方式区域匹配
- 币种设置是否正确（例如 iDEAL 使用 EUR）
- 如果客户为国际用户，是否启用了自适应币种

**解决方案：**验证 API 请求中的账单地址国家代码和币种。
</Accordion>

<Accordion title="Indian payment method issues">
UPI 和 Rupay 订阅有独特要求，包括 48 小时处理延迟和 RBI 授权限制。

**详见：** <a href="/features/payment-methods/india">India Payment Methods</a> 以获取完整指南。
</Accordion>
</AccordionGroup>
