## 工作原理

支付方式会根据以下内容自动向客户展示：

- **客户位置** — iDEAL（荷兰）或 UPI（印度）等区域性支付方式仅在客户位于这些地区时出现
- **交易货币** — 支付方式按支持的货币进行过滤
- **设备类型** — Apple Pay 仅在兼容的 Apple 设备和浏览器上显示
- **交易类型** — 一些支付方式不支持订阅或有最低金额要求

<Info>
**自动激活：** 所有支付方式都是由 Dodo Payments 预配置和管理的。无需额外设置——一旦您的账号通过验证并启用实时模式，它们就会可用。
</Info>

## 配置支付方式

默认情况下，所有符合条件的支付方式会自动显示。使用 `allowed_payment_method_types` 来限制显示的方式：

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://api.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

### 可用的方式类型

| 类型 | 描述 |
| :--- | :---------- |
| `credit` | 信用卡（Visa、Mastercard、Amex 等） |
| `debit` | 借记卡 |
| `apple_pay` | Apple Pay |
| `google_pay` | Google Pay |
| `amazon_pay` | Amazon Pay |
| `cashapp` | Cash App Pay |
| `klarna` | Klarna BNPL |
| `afterpay_clearpay` | Afterpay / Clearpay |
| `upi_collect` | UPI 收款（二维码） |
| `ideal` | iDEAL（荷兰） |
| `bancontact_card` | Bancontact（比利时） |
| `eps` | EPS（奥地利） |
| `multibanco` | Multibanco（葡萄牙） |
| `revolut_pay` | RevolutPay |
| `billie` | Billie B2B BNPL |

<Warning>
始终包括 `credit` 和 `debit` 作为后备选项。如果客户没有所有指定的支付方式，结账将失败。
</Warning>

## 最佳实践

<AccordionGroup>
<Accordion title="将方法与您的市场匹配">
为您的目标市场包括区域支付方式。iDEAL 捕获荷兰约 60% 的在线支付；UPI 在印度占主导地位。省略这些方式会显著损害转化率。
</Accordion>

<Accordion title="始终提供卡片后备选项">
即使针对特定方法，也要包括 `credit` 和 `debit` 作为后备选项。并非所有客户都能访问每种支付方式。
</Accordion>

<Accordion title="考虑交易最低金额">
BNPL 方法（Klarna，Afterpay）要求最低为 $50.01。对于低价值产品，请不要包括它们——它们将根本不出现，这没问题。
</Accordion>

<Accordion title="跨地区和设备进行测试">
在上线前，请使用测试模式验证不同客户位置、货币和设备类型的支付方式可用性。
</Accordion>
</AccordionGroup>

## 故障排除

<AccordionGroup>
<Accordion title="支付方式未出现">
**检查这些因素：**
1. 方法是否包含在 `allowed_payment_method_types` 中？
2. 客户位置与方法的支持区域匹配？
3. 方法支持的货币是否在交易货币中？
4. 交易金额是否达最低要求（例如，BNPL 的 $50.01）？
5. 对于订阅：方法是否支持定期支付？

**解决方案：** 暂时移除 `allowed_payment_method_types` 以查看所有合格的方法，然后缩小问题。 
</Accordion>

<Accordion title="意外出现的方法">
**可能的原因：**
- `allowed_payment_method_types` 传递不正确
- 参数格式不正确（必须为字符串数组）

**解决方案：** 记录您的 API 请求有效负载并验证参数结构。
</Accordion>

<Accordion title="区域方法不可用">
**检查：**
- 客户的账单地址是否匹配方法的区域
- 货币设定是否正确（例如，iDEAL 的欧元）
- 如果客户为国际客户，是否启用自适应货币

**解决方案：** 在您的 API 请求中验证账单地址国家代码和货币。
</Accordion>

<Accordion title="印度支付方式问题">
UPI 和 Rupay 订阅有独特要求，包括 48 小时处理延迟和 RBI 授权限制。

**见：** <a href="/features/payment-methods/india">印度支付方式</a> 以获取完整指导。
</Accordion>
</AccordionGroup>
