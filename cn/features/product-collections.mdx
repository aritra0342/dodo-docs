## 关键亮点

- **基于集合的结构**：将相关产品（计划、层级、定价选项）归为一个集合，以便于组织和管理。
- **一个集合，多个产品**：包括多个产品，如入门版、专业版、终身版等，每个产品都有自己的定价模型。
- **动态结账体验**：在一个结账视图中显示集合中的所有产品，让客户选择其首选计划。
- **商家级控制**：在每个集合内启用、禁用和重新排序产品。第一个产品在结账时会自动预选。
- **生命周期意识**：允许客户通过客户门户在同一集合内升级或降级产品。

## 集合结账

集合实现了统一的结账体验，客户可以在一个地方查看和选择所有可用产品。

### 结账类型

| 类型 | 描述 | 用例 |
|------|-------------|----------|
| **集合结账** | 显示集合内所有有效产品 | 订阅计划选择、分级定价 |
| **单一产品结账** | 仅显示一个特定产品 | 直接购买、促销链接 |

### 集合结账体验

使用集合结账时：

1. **显示所有有效产品**：客户可以看到集合中每个启用的产品
2. **第一个产品预选**：集合中的第一个产品会自动被选中
3. **显示产品详情**：每个产品显示其名称、描述和定价
4. **单一选择**：客户选择一款产品进行购买
5. **标准流程继续**：选择后，结账流程将按照所选产品的定价和计费设置进行

### API 集成

为集合创建一个结账会话：

```typescript
const session = await client.checkoutSessions.create({
  product_collection_id: 'pdc_abc123',
  product_cart: [], // Required: pass an empty array for collection checkout
  return_url: 'https://yoursite.com/return'
});

// Redirect customer to the checkout
window.location.href = session.url;
```

<Warning>
当使用 `product_collection_id` 时，无法在会话创建阶段应用折扣码。若启用，客户仍可在结账时输入折扣码。
</Warning>

## 客户门户集成

客户可以直接通过客户门户在同一集合内升级或降级产品。

<Tip>
**已经有订阅类产品？** 只需将其添加到产品集合，即可在客户门户中启用升级/降级流程。无需重新创建产品。
</Tip>

### Plan Management Actions

| Action | Description | Merchant Control |
|--------|-------------|------------------|
| **View Current Plan** | 显示当前产品名称、价格和续订日期 | 始终可用 |
| **Upgrade Plan** | 切换到同一集合中的更高等级产品 | 可配置（默认：允许） |
| **Downgrade Plan** | 切换到同一集合中的较低等级产品 | 可配置（默认：允许） |
| **Cancel** | 完全取消订阅 | 始终可用 |

### Upgrade/Downgrade Rules

- 升级和降级仅在**同一集合内**的产品之间可用
- 按照你的订阅设置应用按比例收费
- 每次升级、降级或取消都会向企业发送邮件通知

在你仪表盘的“订阅设置”中配置整个企业的订阅和计划更改方式。

### Available Settings

| Setting | Description | Default |
|---------|-------------|---------|
| **Allow Multiple Subscriptions** | 客户可以同时持有多个有效订阅 | 已启用 |
| **Allow Subscription Updates** | 客户可随时通过客户门户升级或降级现有订阅 | 已禁用 |

<Info>
客户门户中的计划更改默认处于禁用状态。在“订阅设置”中启用「允许订阅更新」，即可让客户在同一集合内升降级产品。
</Info>

## Managing Collections

产品集合只能通过 Dodo Payments 仪表盘创建和管理。

<Info>
只能通过仪表盘进行创建、更新和删除集合。API 仅提供只读访问以便集成时检索集合数据。
</Info>

### Dashboard Operations

- **Create**：设置带有产品和分组的新集合
- **Update**：修改名称、描述、图片和产品组织方式
- **Reorder**：拖拽更改产品的显示顺序
- **Enable/Disable products**：控制哪些产品在结账中显示
- **Delete**：删除集合（产品保留但解除关联）

## Best Practices

- **Group logically**：按计费周期（月/年）或功能等级（starter/pro/enterprise）组织产品
- **Order strategically**：将最受欢迎或推荐的方案放在第一位，因为它会在结账时被默认选中
- **Use clear naming**：产品名称应清晰传达价值差异
- **Enable both directions**：同时允许升级和降级，为客户提供灵活性
- **Consider proration**：选择与业务模式一致的按比例收费模式
- **Test thoroughly**：在上线前于测试模式下验证结账与计划更改流程
