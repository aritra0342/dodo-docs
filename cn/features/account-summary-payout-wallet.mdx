## 介绍

**余额**部分为商家提供了其所有货币财务活动的全面概述。它包括每种交易类型 - 支付、退款、费用、税款和支出 - 确保对收入和扣除的完全可见性。

## 钱包结构

每个商业账户根据支持的支付货币维护**四个本地钱包**：

| 钱包 | 货币代码 |
| --- | --- |
| 美元钱包 | USD |
| 英镑钱包 | GBP |
| 欧元钱包 | EUR |
| 印度卢比钱包 | INR |

### 钱包逻辑

- 商家的销售款项根据**客户的支付货币**和**地理位置****记入相应的钱包**。
- 该系统优化了**支付处理器效率**、**降低交易费用**和**减少货币转换费用**。

## 余额账本

每个钱包都有一个**独立的账本**，记录该特定钱包货币的所有相关财务活动 - 支付、退款、费用和支出。

**综合账本**结合所有钱包数据，提供商家整个财务历史的统一视图。

### 过滤器和导航

- **货币过滤器：** 允许商家按特定钱包（美元、印度卢比、英镑、欧元）查看交易。
- **所有交易视图：** 提供所有钱包的汇总视图，具有统一的排序、过滤和搜索功能。

### 交易类型

以下交易类型记录在余额账本中：

| 交易类型 | 符号 | 描述 |
| --- | --- | --- |
| **支付** | 贷记 (+) | 成功的客户支付 |
| **退款** | 借记 (-) | 退还给客户的金额 |
| **交易费用** | 借记 (-) | 应用于支付的处理费用 |
| **退款费用** | 借记 (-) | 处理退款时扣除的费用 |
| **销售税** | 借记 (-) | 对支付收取的税额 |
| **销售税退款** | 贷记 (+) | 退款时退还的销售税部分 |
| **支出** | 借记 (-) | 转入商家银行账户的金额 |
| **支出费用** | 借记 (-) | 从支出中扣除的交易费用 |

#### 其他字段

- **交易金额：** 与交易相关的总值。
- **交易时间戳：** 交易完成的日期和时间。
- **交易ID：** 用于跟踪和支持目的的唯一标识符。

## 总余额

**总余额**部分显示所有钱包余额的**综合指示金额** - **美元、英镑和欧元** - 转换为美元后的金额。

<Note>
**指示性显示：** 显示的金额仅供**参考**。最终支付金额可能会根据转账时的实际汇率有所不同。
</Note>

## 结论

**余额**旨在提供在管理多币种资金方面的**透明度、准确性和灵活性**。

通过维护独立的本地钱包、提供基于货币的过滤器以及启用指示性综合视图，Dodo Payments确保商家始终清楚了解其收入和支付准备情况。
