## 什么是许可证密钥？

许可证密钥是授权访问您产品的唯一令牌。它们非常适合：

- **软件许可**：桌面应用程序、插件和命令行工具
- **每用户控制**：限制每个用户或设备的激活次数
- **数字商品**：限制下载、更新或高级功能

## 主要好处

- **访问控制**：定义到期和激活限制
- **操作可见性**：跟踪使用情况、设备和历史记录
- **安全性和支持**：快速禁用、重新启用和审计

## 创建许可证密钥

<Steps>
<Step title="创建许可证密钥">
    前往您的 Dodo Payments 仪表板并打开许可证密钥部分。选择创建新许可证密钥的选项。
</Step>

<Step title="配置许可证密钥设置">
    通过指定以下字段设置您的许可证密钥的发行和使用方式：
    
    - **许可证密钥到期日期**：定义密钥的有效期。选择特定的持续时间（例如，30天或1年），或选择“无到期”以获得永久密钥。
    - **激活限制**：设置每个密钥的最大并发激活次数。对于单用户（1）或团队许可证（5）等场景使用特定数字，或选择“无限制”以取消激活上限。
    - **激活说明**：为您的客户提供清晰的激活步骤，这些步骤也将与许可证密钥一起通过电子邮件发送。示例：“在设置 → 许可证中粘贴密钥”或“运行: `mycli activate <key>`”。
</Step>

<Step title="审核并保存">
    检查所有输入的详细信息。准备好后，保存许可证密钥配置以完成密钥创建。
    <Tip>
        使用现实的说明并包含支持链接，以减少激活请求。
    </Tip>
</Step>
</Steps>

## 管理与监控

**许可证密钥部分**提供所有许可证密钥活动的全面视图，使您能够有效管理和监控许可证。

### 关键信息

查看每个许可证密钥的详细信息：

- **密钥详细信息**：许可证密钥值、相关产品名称、客户详细信息（customer_id）和购买日期
- **使用活动**：激活次数、激活日期和时间，以及状态（激活、过期、禁用）
- **到期和限制**：密钥到期日期、剩余激活次数和当前激活实例

### 可用操作

您可以对许可证密钥执行以下操作：

- **禁用许可证密钥**：立即禁用密钥以防止进一步使用
- **启用密钥**：重新启用先前禁用的许可证密钥
- **查看激活实例**：查看特定许可证密钥的所有相关激活实例

## 集成示例

### 激活许可证

<Note>
这些端点不需要 API 密钥，可以直接从您的客户端应用程序调用。
</Note>

<CodeGroup>

```typescript TypeScript/Node.js
import DodoPayments from 'dodopayments';

// No API key needed for public license endpoints
const client = new DodoPayments();

const response = await client.licenses.activate({ 
  license_key: 'license_key', 
  name: 'Device Name' 
});

console.log(response.id);
```

```bash cURL
curl -X POST https://test.dodopayments.com/licenses/activate \
  -H "Content-Type: application/json" \
  -d '{
    "license_key": "your_license_key",
    "name": "Device Name"
  }'
```

</CodeGroup>

### 验证许可证

<CodeGroup>

```typescript TypeScript/Node.js
import DodoPayments from 'dodopayments';

// No API key needed for public license endpoints
const client = new DodoPayments();

const response = await client.licenses.validate({ 
  license_key: '2b1f8e2d-c41e-4e8f-b2d3-d9fd61c38f43' 
});

console.log(response.valid);
```

```bash cURL
curl -X POST https://test.dodopayments.com/licenses/validate \
  -H "Content-Type: application/json" \
  -d '{
    "license_key": "2b1f8e2d-c41e-4e8f-b2d3-d9fd61c38f43"
  }'
```

</CodeGroup>

## 最佳实践

- **保持限制清晰**：为到期和激活选择合理的默认值
- **指导用户**：提供精确的激活说明和自助文档
- **自动化检查**：在授予访问权限之前进行服务器端密钥验证
- **监控事件**：使用 Webhook 检测滥用并自动化撤销
