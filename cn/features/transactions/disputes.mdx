## 介绍

争议部分提供了所有提出的争议的详细视图，以及这些争议的当前状态。您需要在您的端管理争议解决，但仪表板将提供状态、争议原因、争议 ID 的可见性，以方便访问信息。

## 查看和理解争议

<Steps>
<Step title="打开交易页面">
    在您的仪表板中转到 <strong>交易</strong> 页面并选择 <strong>争议</strong> 选项卡。这将显示与您的账户相关的所有争议的列表。
</Step>

<Step title="查看争议详情">
    点击任何争议条目旁边的 <strong>眼睛图标</strong> 以打开详细视图。在这里，您可以查看争议原因、状态、争议 ID 和其他相关信息。
</Step>

<Step title="访问相关支付信息">
    在争议详情中，找到 <strong>相关支付</strong> 字段。点击这里的 <strong>眼睛图标</strong> 以查看与争议相关的完整支付详情。
</Step>
</Steps>

## 回应争议

当您的交易被提出争议时，请按照以下步骤有效回应：

1. **检查您的仪表板**
   - 导航到交易 → 争议部分
   - 查找新的争议条目

2. **查看电子邮件通知**
   - 检查您注册的电子邮件以获取争议创建通知

3. **选择您的回应**
   - 您可以接受或反驳争议
   
   <Frame>
       <img src="/images/disputes/1.png" alt="争议创建" />
   </Frame>

4. **如果接受**
   - 不需要进一步操作
   - 您将失去争议

5. **如果反驳**
   - 准备提交证据
   - 遵循以下最佳实践
   
   <Frame>
       <img src="/images/disputes/2.png" alt="争议创建" />
   </Frame>
   
   <Frame>

   </Frame>

<Info>
您在争议创建后有 4 天的时间进行回应
</Info>

## 争议回应最佳实践

<Steps>
<Step title="理解争议原因">
仔细审查争议原因。常见原因包括：

<CardGroup cols={3}>
<Card title="欺诈交易">
客户声称他们没有授权该交易
</Card>
<Card title="未收到产品/服务">
客户声称他们在付款后没有收到产品/服务
</Card>
<Card title="与描述不符">
客户声称产品/服务与广告不符
</Card>
</CardGroup>

<Info>
您的回应必须直接针对具体原因。避免提交通用或无关的证据。
</Info>
</Step>

<Step title="收集正确的证据">
专注于客户身份、访问和使用证据。

<Tabs>
<Tab title="欺诈交易">
<Card title="所需证据">
- **购买发票**
- **购买时的 IP 地址、设备和位置** 详细信息
- **登录时间戳** 显示访问您的平台或服务
- **浏览器/设备指纹** 或账户活动日志
- 来自同一客户或账户的先前成功交易
- 购买时接受的服务条款和退款政策
</Card>
</Tab>
<Tab title="未收到产品/服务">
<Card title="所需证据">
- **购买发票**
- 显示 **成功登录** 的截图或日志
- 带时间戳的活动（下载、查看的内容、开始的会话）
- 确认已授予访问权限的电子邮件或支持消息
- 客户完成的欢迎电子邮件或入职步骤
- 显示产品/服务按承诺交付的内部日志
</Card>
</Tab>
<Tab title="与描述不符">
<Card title="所需证据">
- **购买发票**
- 购买时您网站上的产品/服务描述
- 承诺与交付的比较
- 访问的文档或截图
- 任何客户沟通确认收货或使用
- 向客户提供的退款政策和支持选项
</Card>
</Tab>
</Tabs>
</Step>

<Step title="撰写清晰的描述">
<Card title="关注点">
- 客户购买了什么
- 他们何时以及如何访问它
- 为什么争议声明不准确
</Card>

<Info>
保持您的语言客观和专业。避免情绪化或指责性的陈述。
</Info>

<Card title="示例回应">
"我们提供基于文本的数字图像生成服务，使用 AI。客户于 3 月 5 日购买了月度订阅，并在 3 月 5 日至 10 日期间多次访问该产品，并在此期间生成了 4 张图像。请参见附加的访问日志和显示我们无退款政策的接受条款。"
</Card>
</Step>

<Step title="格式化并提交证据">
<Card title="提交指南">
- 仅使用 **PDF、JPG 或 PNG** 格式
- 清楚地 **标记每个文件**（例如，`Login_Log_March_5.pdf`）
- 仅提交 **相关** 和 **有序** 的文件
- 每个文件保持在 5 MB 以下
- **不要包含链接或 URL**
</Card>
</Step>
</Steps>

<AccordionGroup>
<Accordion title="常见错误">
<Card title="避免这些错误">
- 忽视实际争议原因
- 使用模糊或情绪化的陈述
- 上传无关或无结构的文件
- 提供链接而不是文件
</Card>
</Accordion>
<Accordion title="额外提示">
<Card title="增强您的案例">
- 包括 **带时区的时间戳** 以证明访问时间线
- 突出任何 **双因素认证** 或验证步骤
- 如果符合您的政策，分享 **拒绝退款的沟通**
- 保持您的政策清晰可见，并易于证明已被接受
</Card>
</Accordion>
</AccordionGroup>
