事件是基于使用的计费的基础。当可计费操作发生时发送事件，计量器将其汇总为费用。

完整的 API 文档，包括示例和响应代码。
</Card>

## 事件结构

<Accordion title="必填字段">
<ParamField body="event_id" type="string" required>
唯一标识符。使用 UUID 或组合客户 ID + 时间戳 + 操作。
</ParamField>

<ParamField body="customer_id" type="string" required>
Dodo Payments 客户 ID。必须是有效的现有客户。
</ParamField>

<ParamField body="event_name" type="string" required>
事件类型，匹配您的计量器的事件名称（区分大小写）。示例：`api.call`，`image.generated`
</ParamField>
</Accordion>

<Accordion title="可选字段">

<ParamField body="timestamp" type="string">
ISO 8601 时间戳。如果省略，则默认为服务器时间。对于延迟/批处理事件，包含以确保准确计费。
</ParamField>

<ParamField body="metadata" type="object">
用于聚合和过滤的附加属性：
- 数值：`bytes`，`tokens`，`duration_ms`
- 过滤器：`endpoint`，`method`，`quality`

```javascript
metadata: {
  endpoint: "/v1/orders",
  method: "POST",
  tokens: 1024
}
```

</ParamField>
</Accordion>

## 发送事件

<CodeGroup>

```javascript Single Event
await fetch('https://test.dodopayments.com/events/ingest', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.DODO_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    events: [{
      event_id: "api_call_1234",
      customer_id: "cus_abc123",
      event_name: "api.call",
      metadata: { endpoint: "/v1/orders" }
    }]
  })
});
```

```javascript Batch Events
await fetch('https://test.dodopayments.com/events/ingest', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.DODO_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    events: [
      { event_id: "call_1", customer_id: "cus_abc", event_name: "api.call" },
      { event_id: "call_2", customer_id: "cus_abc", event_name: "api.call" },
      { event_id: "call_3", customer_id: "cus_abc", event_name: "api.call" }
    ]
  })
});
```

```python Python
import requests

requests.post(
    'https://test.dodopayments.com/events/ingest',
    headers={'Authorization': f'Bearer {api_key}', 'Content-Type': 'application/json'},
    json={'events': [{'event_id': 'call_1', 'customer_id': 'cus_abc', 'event_name': 'api.call'}]}
)
```

```curl cURL
curl -X POST 'https://test.dodopayments.com/events/ingest' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{"events": [{"event_id": "call_1", "customer_id": "cus_abc", "event_name": "api.call"}]}'
```

</CodeGroup>

<Tip>
每个请求批量处理最多 100 个事件以获得更好的性能。
</Tip>

## 最佳实践

<AccordionGroup>
<Accordion title="使用唯一事件 ID">
使用确定性 ID 以防止重复：`${customerId}_${action}_${timestamp}`
</Accordion>

<Accordion title="实现重试">
在 5xx 错误时进行重试，使用指数退避。不要在 4xx 错误时重试。
</Accordion>

<Accordion title="包含时间戳">
实时事件可省略。对于延迟/批处理事件，包含以确保准确性。
</Accordion>

<Accordion title="监控交付">
跟踪成功率并将失败的事件排队以进行重试。
</Accordion>
</AccordionGroup>

## 故障排除

<AccordionGroup>
<Accordion title="事件未出现">
- 事件名称必须与计量器完全匹配（区分大小写）
- 客户 ID 必须存在
- 检查计量器过滤器是否未排除事件
- 验证时间戳是否是最近的
</Accordion>

<Accordion title="身份验证错误 (401)">
验证 API 密钥是否正确，并使用格式：`Bearer YOUR_API_KEY`
</Accordion>

<Accordion title="验证错误 (400)">
确保所有必填字段均已填写：`event_id`，`customer_id`，`event_name`
</Accordion>

<Accordion title="元数据未聚合">
- 元数据键必须与计量器的 "Over Property" 完全匹配
- 使用数字，而不是字符串：`tokens: 150` 而不是 `tokens: "150"`
</Accordion>
</AccordionGroup>
