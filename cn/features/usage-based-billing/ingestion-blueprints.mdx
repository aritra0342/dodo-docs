数据摄取蓝图是用于跟踪常见场景中使用情况的预构建模式。与其从头开始确定事件结构和计量配置，不如从经过验证的蓝图开始，并根据您的需求进行自定义。

## 什么是蓝图？

每个蓝图包括：
- **事件结构建议**：最佳事件命名和元数据设计
- **计量配置**：建议的聚合类型和过滤器
- **示例代码**：多种语言的工作实现示例
- **最佳实践**：关于准确性、性能和成本优化的提示

<Tip>
蓝图可以节省数小时的设置时间，并帮助您避免常见的陷阱。从蓝图开始，然后根据您的具体要求进行调整。
</Tip>

## 如何使用蓝图

<Steps>
<Step title="选择您的蓝图">
选择最符合您计费场景的蓝图。每个蓝图页面都包含详细的实施指南。
</Step>

<Step title="审查事件结构">
了解推荐的事件命名和元数据设计。复制适合您用例的事件结构。
</Step>

<Step title="创建计量器">
按照蓝图的计量配置说明进行操作。使用建议的聚合类型和过滤器。
</Step>

<Step title="实现事件发送">
复制适合您语言/框架的示例代码。用您的实际 API 密钥和客户 ID 替换占位符值。
</Step>

<Step title="测试和验证">
发送测试事件并验证它们是否在您的计量仪表板中正确聚合。
</Step>

<Step title="根据需要自定义">
根据您的具体要求调整蓝图——添加自定义元数据、调整过滤器或修改定价。
</Step>
</Steps>

## 蓝图比较

不确定哪个蓝图适合您的需求？以下是快速比较：

| 用例 | 蓝图 | 聚合 | 常见元数据 |
|----------|-----------|-------------|-----------------|
| AI 聊天应用 | LLM | 总和（令牌） | model, input_tokens, output_tokens |
| REST API | API 网关 | 计数 | endpoint, method, status_code |
| 文件存储 | 对象存储 | 总和（字节） | file_size, file_type, operation |
| 视频平台 | 流 | 总和（带宽） | duration, quality, bitrate |
| 无服务器 | 时间范围 | 总和（持续时间） | function_name, execution_time_ms |

## 自定义实现

没有看到适合您确切用例的蓝图？没问题：

1. **从最接近的蓝图开始**：选择与您的需求最相似的蓝图
2. **调整事件结构**：修改元数据字段以匹配您的要求
3. **调整计量配置**：根据需要更改聚合类型或过滤器
4. **彻底测试**：使用示例使用情况验证计费计算

<Info>
所有蓝图都是起点。随意混合不同蓝图的概念或创建完全自定义的实现。
</Info>
