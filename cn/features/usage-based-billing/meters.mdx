<Info>
计量器将原始事件转换为可计费数量。它们会过滤事件并应用聚合函数（Count、Sum、Max、Last）来计算每个客户的使用量。
</Info>

## 创建计量器

<Steps>
{/* LOCKED_PATTERN_148f3cd6016b0fbbcdca7a493126f782 */}
{/* LOCKED_PATTERN_29b9e112369bf16db8b48662b2a38ae8 */}
描述性名称（例如“API Requests”、“Token Usage”）
</ParamField>

{/* LOCKED_PATTERN_9e0b59479445e49da3fde72c89a38300 */}
需要匹配的精确事件名称（区分大小写）。例如：`api.call`、`image.generated`
</ParamField>
</Step>

{/* LOCKED_PATTERN_2971978ae085c72cb2eed501c36fa0a8 */}
{/* LOCKED_PATTERN_97313bb6bf41a89f8d2567ad673466f3 */}
选择事件的聚合方式：

- **Count**：事件总数（API 调用、上传）
- **Sum**：将数值相加（tokens、bytes）
- **Max**：期间内的最高值（峰值用户）
- **Last**：最近的值
</ParamField>

{/* LOCKED_PATTERN_98022025ff5f3e10a05d61d5fcf69601 */}
用于聚合的元数据键（除 Count 外所有类型均需要）。示例：`tokens`、`bytes`、`duration_ms`
</ParamField>

{/* LOCKED_PATTERN_32802d72c2232f03c58c640afba7643d */}
发票的单位标签。示例：`calls`、`tokens`、`GB`、`hours`
</ParamField>
</Step>

{/* LOCKED_PATTERN_709e3e0ef72c95833fd8d928d7a0591a */}
<Frame>
<img src="/images/usage-based/UBB-3.png" alt="事件过滤" />
</Frame>

添加条件以过滤计入的事件：
- **与逻辑**：所有条件必须匹配
- **或逻辑**：任何条件都可以匹配

**比较器**：等于、不等于、大于、小于、包含

启用过滤，选择逻辑，添加具有属性键、比较符和值的条件。
</Step>

{/* LOCKED_PATTERN_314e5b01027621505255760af9937f80 */}
审查配置并点击 **Create Meter**。
</Step>
</Steps>

## 故障排除

<AccordionGroup>
{/* LOCKED_PATTERN_fc1788ddf0982a3a2fe9f6e81812c541 */}
- 事件名称必须完全匹配（区分大小写）
- 检查计量器过滤器是否未排除事件
- 验证客户 ID 是否存在
- 暂时禁用过滤器进行测试
</Accordion>

{/* LOCKED_PATTERN_ddafb5b33ecc6ac484aee2e1a2c4cd7d */}
- 验证 Over Property 是否与元数据键完全匹配
- 使用数字而不是字符串：`tokens: 150` 而非 `"150"`
- 在所有事件中包含必需的属性
</Accordion>

{/* LOCKED_PATTERN_7673f2ad014dbdbdfd53c822f111f71d */}
- 完全匹配大小写
- 针对数据类型使用正确的运算符
- 确保事件包含所筛选的属性
</Accordion>

{/* LOCKED_PATTERN_9641c2e1836928ca6e48f68372e5b21d */}
- 检查 Events 选项卡以统计实际接收的事件
- 验证聚合类型（Count 与 Sum）
- 确保 Sum/Max 所用数值为数字
</Accordion>
</AccordionGroup>
