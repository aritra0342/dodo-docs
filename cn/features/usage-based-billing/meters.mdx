<Info>
计量器将原始事件转换为可计费数量。它们过滤事件并应用聚合函数（计数、总和、最大值、最后值）来计算每个客户的使用情况。
</Info>

## 创建计量器

<Steps>
<Step title="基本信息">
<ParamField path="计量器名称" type="string" required>
描述性名称（例如，“API 请求”，“令牌使用”）
</ParamField>

<ParamField path="事件名称" type="string" required>
精确的事件名称以匹配（区分大小写）。示例：`api.call`，`image.generated`
</ParamField>
</Step>

<Step title="聚合">
<ParamField path="聚合类型" type="string" required>
选择事件的聚合方式：

- **计数**：事件的总数（API 调用、上传）
- **总和**：添加数值（令牌、字节）
- **最大值**：期间内的最高值（峰值用户）
- **最后值**：最近的值
</ParamField>

<ParamField path="聚合属性" type="string">
要聚合的元数据键（对于所有类型，除了计数，都是必需的）。示例：`tokens`，`bytes`，`duration_ms`
</ParamField>

<ParamField path="计量单位" type="string" required>
发票的单位标签。示例：`calls`，`tokens`，`GB`，`hours`
</ParamField>
</Step>

<Step title="过滤（可选）">
<Frame>
<img src="/images/usage-based/UBB-3.png" alt="事件过滤" />
</Frame>

添加条件以过滤计入的事件：
- **与逻辑**：所有条件必须匹配
- **或逻辑**：任何条件都可以匹配

**比较器**：等于、不等于、大于、小于、包含

启用过滤，选择逻辑，添加条件，包括属性键、比较器和值。
</Step>

<Step title="创建">
检查配置并单击 **创建计量器**。
</Step>
</Steps>

## 故障排除

<AccordionGroup>
<Accordion title="事件未出现">
- 事件名称必须完全匹配（区分大小写）
- 检查计量器过滤器是否未排除事件
- 验证客户 ID 是否存在
- 暂时禁用过滤器进行测试
</Accordion>

<Accordion title="聚合未生效">
- 验证聚合属性是否与元数据键完全匹配
- 使用数字，而不是字符串：`tokens: 150` 而不是 `"150"`
- 在所有事件中包含必需的属性
</Accordion>

<Accordion title="过滤器不起作用">
- 完全匹配大小写
- 对于数据类型使用正确的运算符
- 确保事件包含过滤的属性
</Accordion>

<Accordion title="错误的使用总数">
- 检查事件选项卡以计算实际接收到的事件
- 验证聚合类型（计数与总和）
- 确保值为数字以进行总和/最大值
</Accordion>
</AccordionGroup>
