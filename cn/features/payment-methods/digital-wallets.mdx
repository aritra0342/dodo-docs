数字钱包让客户可以使用设备上保存的支付方式轻触一次即可付款。它们提供更高的转化率，通过令牌化实现更强的安全性，并简化了结账体验。

## 支持的钱包

| 钱包 | 可用性 | 货币 | 订阅 | 需要设置 |
| :----- | :----------- | :--------- | :-----------: | :------------- |
| **Apple Pay** | 全球（印度除外） | 全部 | 是 | 域验证 |
| **Google Pay** | 全球（印度除外） | 全部 | 是 | 无 |
| **Amazon Pay** | 全球（印度除外） | 美元 | 否 | 无 |
| **Cash App Pay** | 仅限美国 | 美元 | 否 | 无 |
| **RevolutPay** | 全球 | 英镑 | 否 | 无 |

<Warning>
**PayPal 支持暂停：** PayPal 暂时无法处理新交易。
</Warning>

## Apple Pay

Apple Pay 允许客户使用保存在 Apple Wallet 中的卡片，通过 Face ID 或 Touch ID 进行身份验证后完成付款。

### 要求
- **设备：** iOS 17+、iPadOS 17+、macOS 上的 Safari 17+
- **协议：** 需要 HTTPS
- **域：** 必须通过 Apple 验证

### 域验证

<Steps>
<Step title="Download the domain association file">
下载 [Apple Pay 域关联文件](http://checkout.dodopayments.com/.well-known/apple-developer-merchantid-domain-association)。
</Step>

<Step title="Host the file on your domain">
上传至您的服务器：

```
https://yourdomain.com/.well-known/apple-developer-merchantid-domain-association
```

该文件必须可以在没有重定向的情况下访问，并使用 `Content-Type: application/octet-stream` 或 `text/plain` 提供。
</Step>

<Step title="Request activation">
发送邮件至 **support@dodopayments.com**，内容包括：
- 您的生产域（例如 `https://yourdomain.com`）
- 确认文件已托管


我们将在 1-2 个工作日内完成验证并启用 Apple Pay。
</Step>

<Step title="Test the integration">
确认后：
1. 在 Apple 设备上打开结账页面
2. 确认 Apple Pay 按钮出现
3. 完成一次测试交易
</Step>
</Steps>

<Warning>
在完成域验证之前，Apple Pay 不会出现在结账中。发布时请预留足够时间。
</Warning>

### 配置

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: ['apple_pay', 'credit', 'debit'],
  return_url: 'https://example.com/success'
});
```

## Google Pay

Google Pay 允许客户使用保存在 Google 账户中的卡片支付，同时提供内置的欺诈防护。

### 要求
- **浏览器：** Chrome（桌面/移动）、Safari、Firefox
- **设备：** 安装 Google Pay 应用的 Android 设备
- **设置：** Chrome 中启用“允许站点检查是否有支付方式”

### 配置

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: ['google_pay', 'credit', 'debit'],
  return_url: 'https://example.com/success'
});
```

<Info>
Google Pay 无需域验证——一旦您的账户上线即自动生效。
</Info>

## Amazon Pay

Amazon Pay 允许客户使用其 Amazon 账户凭据和保存的支付方式，提供熟悉的结账体验。

### 限制
- **货币：** 仅限美元
- **订阅：** 不支持
- **区域：** 印度不可用

### 配置

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: ['amazon_pay', 'credit', 'debit'],
  return_url: 'https://example.com/success'
});
```

## Cash App Pay

Cash App Pay 使得来自 Cash App 的支付成为可能，Cash App 在美国消费者中因点对点付款而广受欢迎。

### 限制
- **区域：** 仅限美国
- **货币：** 仅限美元
- **订阅：** 不支持

### 配置

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: ['cashapp', 'credit', 'debit'],
  return_url: 'https://example.com/success'
});
```

## RevolutPay

RevolutPay 让全球 Revolut 用户利用现有 Revolut 账户余额或绑定的卡片进行快速、无缝的支付。

### 限制
- **货币：** 仅限英镑
- **订阅：** 不支持

### 配置

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: ['revolut_pay', 'credit', 'debit'],
  return_url: 'https://example.com/success'
});
```

## 测试

### Apple Pay 测试

<Steps>
<Step title="Use test mode">
在 Dodo Payments 仪表板中启用测试模式。
</Step>

<Step title="Add a card to Apple Wallet">
向 Apple Wallet 添加一张真实卡片。在测试模式下，该卡不会被扣款。
</Step>

<Step title="Complete test purchase">
在 Apple 设备上打开结账页面并完成 Apple Pay 流程。
</Step>
</Steps>

<Warning>
Apple Pay 需要 HTTPS。若未正确设置 SSL，则不会在 `localhost` 上显示。
</Warning>

### Google Pay 测试

<Steps>
<Step title="Join the test card group">
[加入 Google Pay 测试卡组](https://groups.google.com/g/googlepay-test-mode-stub-data)，可让测试卡自动添加到您的钱包。
</Step>

<Step title="Enable test mode">
使用您的 Dodo Payments 测试 API 密钥。
</Step>

<Step title="Complete test purchase">
在 Google Pay 中选择其中一张测试卡完成交易。
</Step>
</Steps>

### Amazon Pay 和 Cash App Pay

使用您的测试 API 密钥并按照标准结账流程完成操作。测试交易将模拟执行，不会产生实际扣款。

## SDK 支持

### Web SDK
所有数字钱包在以下模式中均完全支持：
- Overlay Checkout
- Inline Checkout
- 直接 API 集成

### 移动 SDK
<Warning>
**React Native 限制：** Apple Pay、Google Pay、Cash App Pay 和 UPI **目前在 React Native SDK 中不支持**。需要这些方式的移动应用请使用基于 Web 的结账覆盖层。
</Warning>

对于原生移动支持，可考虑：
- 在 WebView 中打开结账页面
- 在 Web 场景中使用 [Overlay Checkout](/developer-resources/overlay-checkout)
- 单独实现平台原生的 Apple Pay / Google Pay SDK

## 最佳实践

<AccordionGroup>
<Accordion title="Prioritize wallets on mobile">
移动用户期望看到 Apple Pay 和 Google Pay。在移动结账页面上显著显示这些选项。
</Accordion>

<Accordion title="Complete domain verification early">
Apple Pay 需要域验证。不要等到上线当天——在开发阶段就开始处理。
</Accordion>

<Accordion title="Always include card fallbacks">
并非所有客户都拥有数字钱包。始终在允许的方法中包含 `credit` 和 `debit`。
</Accordion>

<Accordion title="Test on real devices">
模拟器无法完全复制钱包行为。上线前务必在实际的 iOS 和 Android 设备上进行测试。
</Accordion>

<Accordion title="Handle wallet unavailability gracefully">
如果钱包不可用（设备不符、未保存卡片），请确保其它支付选项可见。
</Accordion>
</AccordionGroup>

## 故障排查

<AccordionGroup>
<Accordion title="Apple Pay button not appearing">
**检查：**
1. 是否已在 Dodo Payments 完成域验证？
2. 是否使用 HTTPS？
3. 设备是否为 iOS 17+ / Safari 17+ / iPadOS 17+？
4. 客户是否在 Apple Wallet 中有卡片？
5. `apple_pay` 是否包含在 `allowed_payment_method_types` 中？

**解决方案：** 完成域验证并确保使用 HTTPS。在真实 Apple 设备上测试。
</Accordion>

<Accordion title="Google Pay button not appearing">
**检查：**
1. 是否使用 Chrome 或兼容浏览器？
2. 客户是否在 Google Pay 中存有卡片？
3. Chrome 设置“允许站点检查支付方式”是否启用？
4. `google_pay` 是否包含在 `allowed_payment_method_types` 中？

**解决方案：** 验证浏览器兼容性及客户的 Google Pay 设置。
</Accordion>

<Accordion title="Wallet payment failed">
**常见原因：**
- 钱包中的卡片过期或无效
- 银行拒绝交易
- 网络连接问题

**解决方案：** 客户应验证钱包中的卡片或尝试其他支付方式。
</Accordion>

<Accordion title="Wrong device type">
数字钱包只在兼容设备上出现：
- Apple Pay：仅限 Apple 设备
- Google Pay：Chrome 浏览器或 Android 设备

**解决方案：** 这是预期行为。确保卡片回退选项可用。
</Accordion>
</AccordionGroup>
