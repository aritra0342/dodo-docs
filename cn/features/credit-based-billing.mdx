## 什么是基于积分的计费？

基于积分的计费为您的产品提供了一个灵活的系统，用于向客户发放积分权益。与按使用收费或通过功能开关限制访问不同，您会分配一批积分供客户在使用服务时消耗。

积分特别适用于：

- **AI and LLM platforms**：按方案等级授予令牌或生成积分
- **API services**：分配 API 调用积分，并设置超额定价
- **Infrastructure platforms**：发放计算小时或存储积分
- **Communication services**：按订阅提供消息或通话分钟积分
- **SaaS with consumption tiers**：将包含的使用量打包到积分池中

## 核心概念

### 积分类型

创建积分时，可在两种类型之间选择：

<Tabs>
<Tab title="Custom Unit">
以您自己的单位定义积分——令牌、API 调用、计算小时或对您的产品有意义的任何指标。自定义单位使用您设置的精度（0 到 3 位小数）。

**最适合**：API 调用、AI 令牌、计算小时、存储单位、消息
</Tab>

<Tab title="Fiat Credits">
积分代表实际货币价值（例如 USD、EUR）。客户获得的货币积分余额会在他们按照您定义的定价使用服务时逐渐消耗。

**最适合**：预付余额、促销积分、服务补偿
</Tab>
</Tabs>

### 积分生命周期

积分从发行到消费遵循明确的生命周期：

<Steps>
<Step title="Credits Issued">
当客户购买附带积分权益的产品（订阅或一次性）时，会授予积分。对于订阅类产品，积分会在每个结算周期重新发放。
</Step>

<Step title="Credits Consumed">
客户使用服务时，积分会被扣除。对于按使用计费的产品，计量器会根据实时事件自动扣除积分。您也可以通过仪表板或 API 手动扣除积分。
</Step>

<Step title="Credits Expire or Roll Over">
在结算周期结束（或配置的过期期限后），未使用的积分会根据设置过期或结转到下一个周期。
</Step>

<Step title="Overage Handling">
如果积分在周期中耗尽，可允许超额使用（余额用尽后继续使用），并选择如何处理超额——免除、计费或将差额结转。
</Step>
</Steps>

### 授予来源

积分可以从多个来源授予：

| 来源 | 描述 |
|--------|-------------|
| **Subscription** | 随订阅购买发放积分，并在每个计费周期重新发放 |
| **One-Time** | 随一次性支付产品发放积分 |
| **API** | 通过 API 或仪表板手动授予积分 |
| **Rollover** | 从上一个计费周期结转的积分 |

## 创建积分

在仪表板的 **Products → Credits** 部分创建积分权益。每个积分定义单位、精度、过期规则和生命周期行为。

<Frame caption="The Credits tab under Products shows all your credit entitlements.">
<img src="/images/CBB/Desktop - Entitlements  - Credits.jpg" alt="积分列表页面显示已创建的积分权益" />
</Frame>

<Steps>
<Step title="Navigate to Credits">
转到仪表板中的 **Products**，选择 **Credits** 选项卡。点击 **Create Credit** 开始。
</Step>

<Step title="Configure Basic Information">
输入 **积分名称**——这是您对该积分的内部标识。

<Frame caption="The credit creation form with all configuration sections.">
<img src="/images/CBB/Desktop - Create Credit.jpg" alt="积分创建表单显示基本信息、通用设置和订阅设置" />
</Frame>
</Step>

<Step title="Set General Settings">
配置积分类型和展示属性：

<ParamField path="Credit Type" type="string" required>
选择 **自定义单位** 或 **法币积分**。

- **自定义单位** - 定义您自己的指标（令牌、API 调用、计算小时）。需填写 **单位名称**（例如 “平台令牌”）和 **精度** 设置。
- **法币积分** - 积分代表真实货币价值。需选择 **单位货币**（USD、EUR、GBP、INR 等）。
</ParamField>

<ParamField path="Unit Name" type="string">
仅适用于自定义单位积分。客户看到的积分标签（例如 “AI 令牌”、“API 调用”）。在结账和客户门户中显示。
</ParamField>

<ParamField path="Precision" type="number">
仅适用于自定义单位积分。允许的小数位数：

- `0` - 整数（最适合可数的条目，如 API 调用）
- `1` - 一位小数（0.0）
- `2` - 两位小数（0.00） - **默认**
- `3` - 三位小数（0.000）

<Warning>
精度在创建积分后不可更改。
</Warning>
</ParamField>

<ParamField path="Credit Expiry" type="string">
积分在发行后有效的时间：

- **7 天**、**30 天**（默认）、**60 天**、**90 天**、**自定义** 或 **永不过期**

选择 **自定义** 可指定天数（最少 1 天）。
</ParamField>
</Step>

<Step title="Configure Subscription Settings (Optional)">
以下设置控制定期订阅中的积分行为：

<ParamField path="Rollover" type="boolean">
允许未使用的积分结转到下一个计费周期。启用时配置：

- **最大结转百分比**（0–100%）- 限制结转金额
- **结转有效期** - 结转积分保持有效的时长（例如 1 个月）
- **最大结转次数** - 达到此次数后积分将被没收
</ParamField>

**当积分用尽或订阅失效时：**

<ParamField path="Allow Overage" type="boolean">
允许客户在积分余额为零后继续使用服务。启用时配置：

- **超额限制** - 客户可在余额之外消耗的最大积分
- **每单位价格** - 启用超额时每个额外积分的费用（含货币选择器）
</ParamField>

<ParamField path="Overage Behavior" type="string" required>
控制在结算周期结束时如何处理超额：

- **在重置时免除超额**（默认）- 超出积分限额的使用会被记录但不收费。余额在每个周期重置。
- **在计费时收取超额** - 超出积分限额的使用在下一张发票收取费用，然后余额重置。
- **结转差额** - 超出积分限额的使用以负余额形式结转到下一个周期。
- **结转差额（自动补偿）** - 差额结转，并在下一个周期用新积分自动偿还。
</ParamField>
</Step>

<Step title="Create Credit">
点击 **Create Credit** 保存。该积分即可附加到任何产品。

<Check>
您的积分权益已准备就绪。将其附加到产品以开始向客户发行积分。
</Check>
</Step>
</Steps>

<Tip>
从简单设置开始——无结转、无超额——并随着了解客户如何使用积分逐步增加复杂性。大多数设置可以随时更新而不影响现有的授予。请注意，**精度在积分创建后无法更改**。
</Tip>

## 将积分附加到产品

在产品创建或编辑流程中，积分作为**权益**附加到产品中。每个产品最多可附加**3 个积分**。积分适用于所有三种定价类型。

### 订阅产品

对于订阅类产品，积分会在**每个计费周期**发放，并可配置按比例、试用积分和周期特定设置。

<Steps>
<Step title="Create or Edit a Subscription Product">
转到 **Products → Create Product** 或编辑现有产品。选择 **Subscription** 作为定价类型并配置您的周期性价格。
</Step>

<Step title="Open Entitlements Section">
展开 **Entitlements** 部分，并单击 **Credits** 旁边的 **Attach** 按钮。

<Frame caption="The Entitlements section in the product form with Credits, License Key, and Digital Product Delivery options.">
<img src="/images/CBB/Desktop - Attach Credit - Subscription.jpg" alt="产品权益部分显示积分附加按钮" />
</Frame>
</Step>

<Step title="Select Credits to Attach">
打开 **Add Credits** 面板。您可以从下拉列表中选择现有积分，或单击 **Create new credit** 现场定义一个。

<Frame caption="The Add Credits panel lets you select existing credits or create new ones.">
<img src="/images/CBB/Desktop - Attach Credit - Subscription-2.jpg" alt="Add Credits 面板显示积分选择下拉框" />
</Frame>

<Info>
每个产品最多可附加 3 个积分。每个积分都可以有自己的配置。
</Info>
</Step>

<Step title="Configure Credit Settings">
为每个附加的积分配置：

<ParamField path="Credits issued per billing cycle" type="number" required>
客户在每个计费周期获得的积分数量。
</ParamField>

<ParamField path="Low Balance Threshold" type="number">
当积分低于此数量时通知客户。可在客户用尽之前提醒他们。
</ParamField>

<ParamField path="Credits During Free Trial" type="number">
为试用期设置不同的积分数量。启用 **试用期结束后使试用积分过期** 可在试用转为付费订阅时收回未使用的试用积分。
</ParamField>

<ParamField path="Allow Proration" type="boolean">
当客户升级或降级订阅方案时，对剩余积分执行按比例处理。
</ParamField>

<ParamField path="Import Default Credit Settings" type="boolean">
使用积分权益默认的结转、超额和过期设置。关闭此项可针对该产品自定义设置。
</ParamField>

<Frame caption="Credit configuration showing per-cycle amount, trial credits, proration, and custom settings.">
<img src="/images/CBB/Desktop - Attach Credit - Subscription-4.jpg" alt="积分配置表单，包含计费周期、试用和按比例设置" />
</Frame>
</Step>

<Step title="Review and Add">
审核附加的积分，显示名称、数量和过期情况。点击 **Add to Subscription** 以确认。

<Frame caption="Review attached credits before adding them to the subscription.">
<img src="/images/CBB/Desktop - Attach Credit - Subscription-5.jpg" alt="Add Credits 面板显示所选积分及其详情" />
</Frame>
</Step>
</Steps>

<Steps>

### 按使用计费产品

对于按使用计费的产品，积分会**链接到计量器**，并根据实时消耗事件自动扣除。

<Steps>
<Step title="Create a Usage-Based Product">
选择 **Usage Based Billing** 作为定价类型。配置基础价格和计费频率。

<Frame caption="Usage Based Billing pricing type with meter configuration.">
<img src="/images/CBB/Desktop - Attach Credit - UBB.jpg" alt="按使用计费的定价配置" />

在 **选择计量器** 区域点击 **+** 按钮以添加计量器。一个订阅最多可有 **3 个计量器**。

切换 **在积分中计费使用量** 以将积分附加到计量器。从下拉中选择积分权益。

<ParamField path="Free Threshold" type="number" required>
在开始扣除积分之前的免费单位数。

<ParamField path="Bill usage in Credits" type="boolean">
启用后，计量器使用量将从客户的积分余额中扣除，而不是按单位收费。

<ParamField path="Meter units per credit" type="number" required>
每 1 积分所需的使用单位数。例如，如果设置为 `1000`，则 1,000 次 API 调用消耗 1 积分。

<Frame caption="Credit attached to a meter with per-unit conversion rate.">

<Step title="Configure Credit Issuance">
设置发行的积分数量，并可选地为该产品自定义积分设置。

<Frame caption="Configure how many credits to issue and whether to use default settings.">
<img src="/images/CBB/Desktop - Attach Credit - UBB-6.jpg" alt="按使用计费产品的积分配置" />

<Step title="Verify Attachment">
配置完成后，计量器会显示附加的积分名称、单位价格和免费阈值。

</Info>

## 积分设置

### 结转

结转允许未使用的积分携带到下一个计费周期，而不是过期。

| 设置 | 描述 |
|---------|-------------|
| **启用结转** | 开启以允许未使用积分结转 |
| **最大结转百分比** | 限制结转额度（0–100%）。例如 50% 时只有一半的未使用积分结转 |
| **结转有效期** | 结转积分的有效期（天、周、月、年） |
| **最大结转次数** | 允许连续结转的次数。超过后剩余积分被没收 |

**示例**：客户在周期末拥有 200 个未使用积分。75% 结转时，150 个积分结转，50 个被没收。

### 超额

超额控制当客户的积分余额在周期中降至零时会发生什么。

| 设置 | 描述 |
|---------|-------------|
| **允许超额** | 开启以让客户在积分余额用尽后继续使用服务 |
| **超额限制** | 客户可在余额之外消耗的最大积分 |
| **每单位价格** | 超额消耗时每个额外积分的费用（含货币） |
| **超额行为** | 控制结算周期结束时如何处理超额（见下文） |

**超额行为选项：**

| 行为 | 描述 |
|----------|-------------|
| **在重置时免除超额** | 超出积分限额的使用被记录但不收费。余额在每个周期重置 |
| **在计费时收取超额** | 超出积分限额的使用在下一张发票收取费用，然后余额重置 |
| **结转差额** | 超额以负余额形式结转到下一个周期 |
| **结转差额（自动补偿）** | 差额结转，并在下一个周期用新积分自动偿还 |
<Info>
当超额被禁用时，客户在积分余额为零后无法继续使用服务。根据您的计费模型选择超额行为——**在重置时免除** 是默认且最简单的选项。
</Info>

### 过期

| 设置 | 描述 |
|---------|-------------|
| **积分过期** | 积分在发行后多长时间过期（7、30、60、90、自定义天数或永不过期） |
| **试用积分在试用后过期** | 试用专属积分在试用期结束后是否过期 |

<Info>
过期积分会创建一条 `CreditExpired` 分类账条目。如果启用结转，先应用结转百分比，然后剩余部分才会过期。
</Info>

## 使用积分的计费

当积分链接到使用计量器时，系统会创建一个强大的基于消耗的计费模型。客户会获得积分分配，使用事件会自动从余额中扣除。

<Frame caption="The Usage Billing dashboard shows meter events with units consumed, credits consumed, and customer details.">
<img src="/images/CBB/Desktop - Usage Billing.jpg" alt="使用计费仪表板显示事件表及积分消耗" />
</Frame>

### 基于计量器的积分扣除如何运作

1. **您的应用发送使用事件** - 每个事件包括客户 ID、事件名称和元数据
2. **计量器汇总事件** - 使用计数、求和、最大值、最近值或唯一计数聚合
3. **积分自动扣除** - 后台工作线程每分钟处理事件，使用您配置的速率将计量器单位转换为积分，并按 FIFO 顺序（最旧的授予优先）从客户余额中扣除
4. **跟踪超额** - 如果积分余额归零且启用超额，系统会记录超额使用以便在周期末计费

### 计量器面板

使用计费仪表板包括一个列出所有定义计量器及其聚合类型的 **计量器** 面板：

| 聚合 | 描述 | 示例 |
|-------------|-------------|---------|
| **Count** | 事件总数 | API 调用 |
| **Sum** | 某值字段的总和 | 传输总字节数 |
| **Max** | 记录的最大值 | 峰值并发用户数 |
| **Last** | 最近的值 | 当前存储使用量 |
| **Unique Count** | 不同值的数量 | 活跃的唯一用户数 |

## 客户体验

### 结账

当客户购买附带积分的产品时，结账页面会将包含的积分作为产品的一部分展示。

<Frame caption="Checkout shows included credits with the product, making the value proposition clear.">
<img src="/images/CBB/Checkout.png" alt="结账页面显示包含 API 调用积分的产品" />
</Frame>

积分会在产品描述下方的 **包含** 部分显示，展示积分数量和类型（例如 “$1000 API 调用”）。

### 客户门户

客户可在客户门户的 **Credits** 部分查看并管理其积分余额。

<Frame caption="The Customer Portal shows available balance and full transaction history.">
<img src="/images/CBB/Customer Portal.jpg" alt="客户门户积分视图，显示余额与交易记录" />
</Frame>

门户显示：

- **可用余额** - 当前积分余额以突出方式显示
- **积分选项卡** - 在不同积分类型之间切换（例如 “OpenAI 积分”、“使用令牌”）
- **近期交易** - 包含日期、交易 ID、类型、数量和流水余额的完整历史记录

交易类型对客户可见如下：

| 类型 | 描述 | 数量 |

### 积分账本

每次积分操作都会记录在积分账本中，提供完整的审计路径：

| 交易类型 | 描述 |
|-----------------|-------------|
| **积分增加** | 授予的积分（订阅、一次性或 API） |
| **积分扣除** | 通过使用或手动扣除消耗的积分 |
| **积分过期** | 未结转而过期的积分 |

每条账本记录包含交易前后余额、前后超额、描述以及源引用（支付、订阅等）。

## Webhooks

基于积分的计费会在每次积分生命周期变化时触发 webhook 事件。利用这些事件可让应用与积分余额保持同步、触发通知或构建自定义计费流程。

| 事件 | 描述 |
|-------|-------------|
| `credit.added` | 向客户授予积分 |
| `credit.deducted` | 通过使用或手动扣除消耗积分 |
| `credit.expired` | 未使用积分过期 |
| `credit.rolled_over` | 积分结转到新授予 |
| `credit.rollover_forfeited` | 在最大结转次数时积分被没收 |
| `credit.overage_charged` | 发生超额计费 |
| `credit.manual_adjustment` | 进行手动积分/扣除调整 |
| `credit.balance_low` | 余额低于配置阈值 |

所有账本事件（从 `credit.added` 到 `credit.manual_adjustment`）都会包含完整的 `CreditLedgerEntry` 有效载荷，包含前后余额、前后超额以及源引用。`credit.balance_low` 事件包含阈值配置和当前余额。

## 最佳实践

- **从简单开始**：先使用单一积分类型且不启用结转，再根据客户反馈和使用模式逐步增加复杂性。
- **设定明确预期**：在产品页面和客户门户中突出显示积分分配、剩余余额和超额定价。
- **使用有意义的单位**：根据积分所代表的内容命名（如 “API 调用”、“AI 令牌”），而不是使用通用术语，这有助于客户理解价值。
- **合理配置过期**：短期过期（7 天）会带来紧迫感，但可能让客户感到沮丧。对于大多数 SaaS 产品，较长的窗口（30–90 天）更为友好。
- **监控低余额**：设置低余额阈值，在客户用尽前提醒，减少意外的超额费用。
- **在测试模式下验证**：创建积分，附加到测试产品，模拟购买 → 使用 → 扣除 → 过期的完整流程再上线。

<Info>
基于积分的计费与所有其他 Dodo Payments 功能无缝配合——支持试用的订阅、带按比例的方案变更以及客户门户。从基础设置开始，随着计费模型演进逐步扩展。
</Info>
