## 概述

Dodo Payments Framer 插件无缝集成支付处理到您的 Framer 网站中。创建可自定义的结账体验，支持测试和生产环境。

</Card>

## 设置指南

<Steps>
<Step title="生成 API 密钥">
访问 [Dodo Payments Dashboard](https://app.dodopayments.com/developer/api-keys) 并创建一个新的 API 密钥。

<Tabs>

<Tab title="实时模式">
用于真实支付处理的生产使用。
</Tab>

<Tab title="测试模式">
适合开发和测试，无真实费用。
</Tab>

</Tabs>

<Check>
安全地复制您的 API 密钥 - 您将在下一步中需要它。
</Check>
</Step>

<Step title="验证插件">
1. 在 Framer 中打开 Dodo Payments 插件
2. 导航到身份验证页面
3. 粘贴您的 API 密钥
4. 选择适当的模式（测试或实时）
5. 点击 **继续** 进行身份验证

<Warning>
保持您的 API 密钥安全，切勿公开分享。
</Warning>
</Step>

<Step title="管理产品">
身份验证后，您将看到您的产品列表：

- **现有产品**：自动显示
- **没有产品**：点击“产品”旁边的 **加号 (+)** 按钮

<Info>
您将被重定向到 Dodo Payments Dashboard，以创建带有定价、描述和设置的产品。
</Info>
</Step>

<Step title="添加购买按钮">
1. 从可用列表中选择一个产品
2. 点击 `Add Buy Button` 按钮
3. 一个带有文本 `购买 {product_name}` 的购买按钮将被添加到您的画布上

<Warning>
如果您没有正确的项目权限，添加购买按钮将被禁用。
</Warning>
</Step>
</Steps>

## 自定义

<Steps>
<Step title="选择结账体验">
    选择您首选的结账体验：
    - **覆盖（嵌入式）**：结账作为覆盖出现在您的 Framer 项目中，保持用户在您的网站上。
    - **托管**：用户被重定向到 Dodo Payments 托管的结账页面，以获得简化的体验。
</Step>

<Step title="自定义外观选项">
    使用以下外观设置自定义您的结账体验：
    - **结账主题**：选择浅色或深色主题以匹配您的品牌美学。
    - **按钮标签**：自定义文本（例如，'立即购买'，'购买'，'开始'）。
    - **按钮样式**：调整颜色、字体、大小和其他样式属性。
    - **返回 URL**：设置一个 Framer 页面，以便用户在结账后返回。
</Step>
</Steps>

## 故障排除

<AccordionGroup>
<Accordion title="身份验证问题">
**问题**：插件无法验证

**解决方案**：
- 验证您的 API 密钥是否正确
- 确保您使用的是正确的模式（测试/实时）
- 检查您的互联网连接
</Accordion>

<Accordion title="按钮创建问题">
**问题**：添加购买按钮被禁用

**解决方案**：
- 验证您在 Framer 项目中是否具有正确的权限
- 确保您已正确验证
- 尝试刷新插件
</Accordion>

<Accordion title="产品显示问题">
**问题**：产品未显示

**解决方案**：
- 确保您在 Dodo Payments Dashboard 中创建了产品
- 检查您的 API 密钥是否具有正确的权限
- 尝试重新验证
</Accordion>
</AccordionGroup>
