## 介绍

Dodo Payments WooCommerce 插件为您的商店提供无缝集成，使您能够通过多种方式接受支付，包括 UPI、卡片、网银和数字钱包。本综合指南将引导您完成安装和配置过程。

## 分步安装指南

1. 下载最新的 WooCommerce 支付插件的 zip 文件。

    <Card title="下载插件" icon="download" href="https://github.com/dodopayments/dodo-woocommerce/releases">
    获取我们 WooCommerce 插件的最新版本
    </Card>

    或者，您可以直接从 WordPress 插件目录安装：
    
    <Card title="WordPress 插件目录" icon="wordpress" href="https://wordpress.org/plugins/dodo-payments-for-woocommerce/">
    从 WordPress.org 安装 Dodo Payments for WooCommerce
    </Card>

2. 以管理员身份登录到您的 WordPress 网站（我们假设您已经设置了 WooCommerce）
3. 导航到 **插件 → 添加新插件**
    
    <Frame>
        <img src="/images/woocommerce/1.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
4. 点击 **上传插件** 按钮，将出现文件选择对话框

    <Frame>
        <img src="/images/woocommerce/2.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
    <Frame>
        <img src="/images/woocommerce/3.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
5. 点击 **浏览…** 并选择您下载的 zip 文件
6. 点击 **立即安装** 按钮以开始安装过程
    
    <Frame>
        <img src="/images/woocommerce/4.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
7. 安装完成后，点击 **激活插件** 或从 **已安装插件** 部分激活它
    
    <Frame>
        <img src="/images/woocommerce/5.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
8. 插件现在已安装，但仍需配置。继续查看下面的设置指南。

## 配置指南

### 设置 API 密钥和 Webhook

1. 导航到 **WooCommerce → 设置 → 支付** 或点击 WooCommerce 菜单项下左侧边栏中的 **支付** 按钮
    
    <Frame>
        <img src="/images/woocommerce/6.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
    <Frame>
        <img src="/images/woocommerce/7.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
2. 如果尚未启用 **Dodo Payments** 支付提供商，请启用它，然后点击 **管理** 以配置插件
3. 您将看到各种配置选项，每个选项都有有用的说明文本。首先设置您的实时 API 密钥
    
    <Frame>
        <img src="/images/woocommerce/8.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
4. 登录到您的 Dodo Payments 仪表板，进入 **实时模式**，然后导航到 **Dodo Payments (实时模式) > 开发者 > API 密钥**，或访问 [此直接链接](https://app.dodopayments.com/developer/api-keys) 并点击 **添加 API 密钥**
    
    <Frame>
        <img src="/images/woocommerce/9.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
5. 给您的 API 密钥一个描述性名称，然后点击 **创建**
    
    <Frame>
        <img src="/images/woocommerce/10.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
6. 复制生成的 API 密钥并将其粘贴到 WooCommerce 插件设置中的 **实时 API 密钥** 字段中
    
    <Frame>
        <img src="/images/woocommerce/11.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
    <Frame>
        <img src="/images/woocommerce/12.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
7. 接下来，设置 Webhook 签名密钥，以启用 Dodo Payments 和 WooCommerce 之间的支付状态同步
8. 滚动到插件设置页面的底部，复制显示的 webhook URL
    
    <Frame>
        <img src="/images/woocommerce/13.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
9. 返回到您的 Dodo Payments 仪表板，导航到 **Dodo Payments (实时模式) > 开发者 > Webhooks**，然后点击 **添加 Webhook**
    
    <Frame>
        <img src="/images/woocommerce/14.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
10. 将您在第 8 步中复制的 URL 粘贴到对话框中，然后点击 **添加 Webhook**
    
    <Frame>
        <img src="/images/woocommerce/15.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
11. 创建 webhook 后，点击红色 **签名密钥** 旁边的眼睛图标以显示并复制它
    
    <Frame>
        <img src="/images/woocommerce/16.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
12. 将签名密钥粘贴到插件设置中的 **实时 Webhook 签名密钥** 字段中，并保存更改
    
    <Frame>
        <img src="/images/woocommerce/17.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>
    
13. 检查其余设置，例如 **全球税类** 和 **所有价格均含税**，因为这些选项决定了产品如何从 WooCommerce 同步到 Dodo Payments。请注意，**测试 API 密钥** 和 **测试 Webhook 签名密钥** 仅在您计划使用测试模式时需要
    
    <Frame>
        <img src="/images/woocommerce/18.webp" alt="Dodo Payments for WooCommerce 插件" />
    </Frame>

## 主要特点

- **无缝结账体验**：将客户重定向到安全、优化的结账页面
- **多种支付方式**：支持各种本地和国际支付选项
- **实时状态更新**：通过 Webhook 实现即时订单状态同步
- **详细交易记录**：在您的 WooCommerce 仪表板中提供全面的支付历史
- **多货币支持**：接受多种货币的支付

## 故障排除

### 常见问题

1. **支付处理失败**
   - 验证 API 密钥是否正确配置
   - 确保 webhook URL 正确设置
   - 检查您的 SSL 证书是否有效

2. **Webhook 同步问题**
   - 确认 webhook 签名密钥是否匹配
   - 检查服务器日志以查找潜在错误
   - 验证您的服务器是否可以接收传入的 webhook 请求
