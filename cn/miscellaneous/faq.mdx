## 开始使用与账户设置

<Accordion title="Q1: 如何登录 Dodo Payments 仪表板？">
  A: 访问我们的仪表板，输入您在欢迎邮件中收到的登录邮箱和密码。如果您遇到任何问题，请联系支持团队。
</Accordion>

<Accordion title="Q2: 我需要哪些文件来完成验证过程？">
  A: 验证所需的文件包括有效的政府签发的身份证明（用于身份验证）和商业注册文件（用于商业验证）。有关详细信息，请查看 [验证过程页面](/miscellaneous/verification-process)。
</Accordion>

<Accordion title="Q3: 如果我的验证被拒绝，会发生什么？">
  A: 如果您的验证被拒绝，您将收到一封电子邮件，说明拒绝的原因。您可以联系支持团队以解决任何问题，纠正文件并重新申请。如果需要，验证团队将启用重新提交选项。
</Accordion>

<Accordion title="Q4: 验证过程通常需要多长时间？">
  A: 验证过程通常需要 **24-72 个工作小时**。但是，如果需要额外的文件，过程可能会更长。
</Accordion>

<Accordion title="Q5: 我目前没有网站。可以提交我公司的 LinkedIn 页面吗？">
  A: 我们需要您的网站以了解您销售的产品类型并验证您的业务以获得实时访问。如果您没有网站或正在构建一个，请通过 support@dodopayments.com 联系我们。
</Accordion>

<Accordion title="Q6: 我是个人，没有注册的公司。仍然可以使用 Dodo Payments 吗？">
  A: 是的，我们也支持未注册的业务。您不需要注册的公司即可使用 Dodo Payments。您可以作为个人入驻并开始接收国际支付，而无需任何麻烦。作为未注册的业务，您仍需提交身份验证表和商业验证表以便我们为您入驻。
</Accordion>

<Accordion title="Q7: 我想将我的其他业务与 Dodo Payments 集成。应该创建一个新账户吗？">
  A: 不，您可以通过一个账户管理多个业务。要添加新业务，请单击仪表板左下角的 + 图标，然后选择添加业务选项。
您可以参考 /miscellaneous/accounts#managing-multiple-businesses 获取更多详细信息。
</Accordion>

<Accordion title="Q8: 在将 Dodo Payments 集成到我的系统之前，我应该等待验证完成吗？">
  A: 不，您无需等待。您可以开始使用测试模式进行集成，一旦验证成功，您就可以开始在生产环境中进行实时支付。了解有关测试模式和实时模式的更多信息，请访问
/miscellaneous/test-mode-vs-live-mode。
</Accordion>

<Accordion title="Q9: 如何获得实时模式访问权限？">
  A: 确保您已在设置选项卡中提交身份和商业验证文件。如果您已经提交，并且超过 72 小时，请检查同一页面上挂起的原因并联系支持团队。
</Accordion>

<Accordion title="Q10: Dodo Payments 有联盟计划吗？">
  A: 是的，我们确实提供联盟计划！

  <Card title="了解更多关于我们的联盟计划" icon="users" href="/features/affiliates">
    了解如何加入我们的联盟计划、佣金率以及如何开始赚钱。
  </Card>

</Accordion>

<Accordion title="Q11: 我如何删除我创建的产品/客户？">
  A: 目前，您无法删除产品或客户，因为这些记录需要保留以便将来的合规性。我们正在开发一个归档产品功能，允许您从活动产品部分中删除产品。
</Accordion>

<Accordion title="Q12: 我如何为支付页面设置自定义 UI？">
  A: 您可以在支付页面上添加您的公司徽标和公司名称。要上传它们，请访问仪表板上的业务资料页面。
</Accordion>

<Accordion title="Q13: 我可以更新结账页面上的品牌吗？">
  A: 是的。您可以：
  - 转到仪表板 → 设置 → 商业设置
  - 更新您的徽标、品牌名称和支持电子邮件。
  - 更改将反映在静态和动态链接上。
</Accordion>

<Accordion title="Q14: 我可以在 iframe 中嵌入 Dodo Checkout 吗？">
  A: 不。托管的结账流程不支持在 iframe 中嵌入。

  相反：
  - 将用户重定向到 Dodo 结账 URL
  - 或使用我们的覆盖结账功能。[覆盖结账文档](/developer-resources/overlay-checkout)
</Accordion>

<Accordion title="Q15: 我可以在不向 Dodo Payments 添加产品的情况下创建一次性托管结账支付吗？">
  A: 不。产品在 Dodo Payments 系统中是强制性的。
</Accordion>

<Accordion title="Q16: 你们支持电子商务网站吗？">
  A: 不，我们不支持电子商务网站。
</Accordion>

<Accordion title="Q17: 我可以在 Dodo Payments 中切换品牌吗？">
  A: 是的，您可以在产品之间切换品牌。您可以在仪表板中创建或编辑产品时选择所需的品牌。
</Accordion>

<Accordion title="Q18: 我可以删除品牌或更新 Dodo Payments 中的品牌 URL 吗？">
  A: 不，Dodo Payments 目前不支持：
  - **删除品牌**：如果品牌未在使用中，您可以简单地忽略它 — 这不会影响您的账户或操作。
  - **更改或更新品牌 URL**：一旦创建，品牌 URL 不能修改。即使 URL 不正确，也不会影响您的设置或功能。
</Accordion>

<Accordion title="Q19: 我可以将 Dodo Payments 与 Shopify 集成以销售实物产品吗？">
  A: 不，Dodo Payments 目前不支持实物商品，也不提供 Shopify 插件。
</Accordion>

<Accordion title="Q20: 我的 INR 产品在结账时显示为 USD。我该如何解决这个问题？">
  A: 如果自适应货币被禁用，结账将显示所选产品的价格，支付方式将根据结账时选择的账单国家显示。
</Accordion>

## 验证、税务与合规

<Accordion title="Q21: 为什么我的验证需要这么长时间？">
  A: Dodo Payments 通常需要 24-72 个工作小时来处理验证。然而，由于以下原因可能会出现延迟：
  - 申请量大
  - 周末和公共假期
  - 提交不完整或不正确
  - 风险或合规的人工审核

  <Note>
  如果超过 72 小时，请检查您的电子邮件或 Dodo Payments 仪表板以获取任何所需的操作。您也可以在 Discord 上向团队发送 DM，使用您注册的电子邮件进行跟进。
  </Note>
</Accordion>

<Accordion title="Q22: 我提交了 W-8BEN 表格，但签名部分无法正常工作。我该怎么办？">
  A: 使用合适的 PDF 编辑器，例如：
  - Adobe Acrobat
  - Mac 预览（适用于大多数情况）
  - Smallpdf 或 PDFescape（在线编辑器）

  如果签名上有叉号，请确保：
  - 在应用签名后保存文件。
  - 工具正在压平签名层（某些编辑器不支持）。

  <Tip>
  避免在浏览器预览器中签名——它们并不总是正确嵌入数字签名。
  </Tip>
</Accordion>

<Accordion title="Q23: 我选择了错误的注册类型（企业与个人）。我该如何修复？">
  A: 如果您选择了错误的实体类型（例如，选择了“组织”，但您是个人或拥有 GST 的个体户），您的验证可能会被搁置，需要您更改或更新文档。 

  修复步骤：
  第一步：通过 Intercom 或 Discord 通知支持团队，他们将帮助您启用 Payout Compliance 表格的重新提交。
  第二步：选择正确的注册类型，并重新提交所需的文件。

</Accordion>

<Accordion title="Q24: 我的验证因实体不匹配而被拒绝。我该怎么办？">
  A: 如果您的验证因实体不匹配而被拒绝（例如，您选择了“组织”，但您是个人或拥有 GST 的个体户），您需要更正注册类型并重新提交文件。

  修复步骤：
  第一步：检查您的仪表板或电子邮件以获取拒绝的具体原因。
  第二步：通过 Intercom 或支持电子邮件（support@dodopayments.com）联系支持，申请重新提交 Payout Compliance 表格的访问权限。
  第三步：选择正确的注册类型（个人与组织），并重新提交所有所需的文件。
  第四步：等待合规团队审核您的重新提交。

  <Note>
  重新提交选项将在您联系支持后由我们的验证团队启用。确保这次选择正确的实体类型，以避免进一步的延迟。
  </Note>
</Accordion>

<Accordion title="Q25: 我达到了 $100 的收入。我的账户会更快验证吗？">
  A: 是的。一旦您的收入超过支付阈值（$100），Dodo Payments 会优先处理您的验证。

  <Note>
  这并不保证即时批准——确保您的商业模式和文件符合要求。
  </Note>

  如果您已经超过 $100 但仍然看到延迟，请联系支持团队。
</Accordion>

<Accordion title="Q26: 我可以使用不同的名字（比如我父母的名字）注册并获得验证吗？">
  A: 不可以。注册的人必须与企业的法定所有者及所有提交的文件相匹配。

  如果您的父母是实际的企业所有者：
  - 他们应该使用自己的电子邮件和手机号码注册。
  - 所有文件都应以他们的名字为准。

  <Note>
  使用他人的名字进行验证，而您管理企业，可能会导致合规拒绝。
  </Note>
</Accordion>

<Accordion title="Q27: 我如何检查验证拒绝的原因？">
  A: 您可以在以下位置找到原因：
  - Dodo Payments 仪表板的验证部分。
  - 电子邮件通知（如果启用）

  <Tip>
  如果没有可见，请联系 compliance@dodopayments.com 或将您的注册电子邮件 ID 发送给团队成员。
  </Tip>
</Accordion>

<Accordion title="Q28: 如果我之前的账户被拒绝，我可以创建一个新的商业账户吗？">
  A: 可以——但仅在以下情况下：
  - 您已解决导致拒绝的问题。
  - 您没有试图绕过风险决策（例如，因欺诈问题而被取消）。

  <Note>
  如果在未解决根本问题的情况下创建多个账户，新的账户也可能会被拒绝。
  </Note>
</Accordion>

<Accordion title="Q29: 我被告知我的账户之前被取消。这是什么意思？">
  A: 如果您看到一条消息，说明您的账户之前被风险团队取消：
  - 这意味着 Dodo Payments 发现您的活动或业务存在重大合规或风险问题。
  - 使用相同身份或业务详细信息尝试创建新账户可能会再次被拒绝。

  <Note>
  如果您认为这是一个错误，请发送电子邮件至 compliance@dodopayments.com 并请求审核——包括清晰的证据以支持您的案例。
  </Note>
</Accordion>

<Accordion title="Q30: 我如何更快地清除我的支付验证？">
  A: 为了避免延迟：
  - 提交清晰、格式正确的文件
  - 避免在 Discord 中重复消息/标签——这会减慢队列
  - 提及您是否已超过支付阈值或有紧急时间表
  - 如果被要求，请 DM 您的注册电子邮件 ID

  <Tip>
  大多数支付审核在 72 个工作小时内完成，除非您的提交存在问题。
  </Tip>
</Accordion>

<Accordion title="Q31: 如果我选择了错误的表格（W-8BEN 与 W-8BEN-E）怎么办？">
  A: 这些表格与您的实体类型相关：
  - W-8BEN 适用于个人
  - W-8BEN-E 适用于组织

  如果您提交了错误的表格：
  - 联系支持以重置您的表格。
  - 填写并上传正确的表格类型。
  - 确保 PAN 在正确的字段中（个人的第 6a 行）。

  <Tip>
  如果感到困惑，请参考 IRS 官方表格指南。
  </Tip>
</Accordion>

<Accordion title="Q32: 我有一个带有商号的 GST。这算作一个组织吗？">
  A: 不算。如果您以个人（个体户）身份注册了 GST，您仍然被视为个人。

  ✅ 使用 W-8BEN
  ✅ 在注册时选择“个人”
  ❌ 除非您是注册的法人实体，否则请勿填写 W-8BEN-E 或选择“组织”
</Accordion>

<Accordion title="Q33: 我的账户被取消。我可以直接创建一个新账户吗？">
  A: 创建新账户以绕过之前的取消是不允许的。如果您尝试这样做：
  - 新账户也可能被标记或拒绝
  - 如果基础业务或个人相同，风险决策会在注册时延续

  <Tip>
  如果您认为自己被错误地取消，请发送电子邮件至 compliance@dodopayments.com，说明您的情况。
  </Tip>
</Accordion>

<Accordion title="Q34: Dodo Payments 如何处理印度 SaaS 商家的 GST？">
  A: Dodo Payments 会根据您客户的位置自动计算和应用 **GST**。 
</Accordion>

<Accordion title="Q35: Dodo Payments 处理国际销售吗？">
  A: 是的，Dodo Payments 支持国际销售并管理所有相关合规性。
</Accordion>

<Accordion title="Q36: 地址数据（如城市、州、邮政编码字段）是强制性的吗？我可以不向客户请求这些值吗？">
  A: 是的，它们是强制性的。由于我们是记录商并生成发票、报税并确保所有购买合规，我们需要客户的账单地址。如果您已经收集了客户的详细信息，可以实施我们的基于 API 的结账流程，以确保客户体验更顺畅，但我们仍然需要这些详细信息用于每次购买。
</Accordion>

<Accordion title="Q37: 如果我在美国销售，我需要在美国报税吗？">
  A: 不直接。Dodo Payments 作为记录商，处理客户端的税务，并在适用时进行汇款。您仍然需要：
  - 在您的本国（例如印度）报税
  - 提交 W-8BEN(-E) 以避免默认的 30% 美国预扣税

</Accordion>

<Accordion title="Q38: 我不知道是填写 W-8BEN 还是 W-8BEN-E——帮帮我？">
  A: 使用以下逻辑来确定您需要哪个表格：

  | 您是... | 提交的表格 | 商户类型 |
  |------------|----------------|---------------|
  | 个体户、自由职业者、个人 | W-8BEN | 个人 |
  | 私人有限公司、有限责任合伙、注册组织 | W-8BEN-E | 组织 |

  仍然不确定？考虑以下指标：
  - **如果您的银行账户在您个人名下**，您可能是个人 → 使用 W-8BEN
  - **如果您的银行账户在公司名下**，您是组织 → 使用 W-8BEN-E
  - **如果您有 GST 但使用个人银行账户运营**，您仍然被视为个人 → 使用 W-8BEN

  <Tip>
  如果您是拥有 GST 的个体户，您仍然是个人。只有注册的法人实体（私人有限公司、有限责任合伙等）应使用 W-8BEN-E。
  </Tip>
</Accordion>

<Accordion title="Q39: 我需要 PAN 或 GST 才能接收付款吗？">
  A: 对于个人，PAN 是强制性要求（用于 KYC 和税务目的）。对于注册企业，可以提交商业 PAN 或 GSTIN。除非您超过注册阈值或希望获得进项税抵免，否则 GST 是可选的。
</Accordion>

<Accordion title="Q40: 我有 Udyam/MSME 证书。我应该提交 W-8BEN 还是 W-8BEN-E？">
  A: 拥有 Udyam/MSME 证书并不自动使您成为组织。您提交的表格取决于您的银行账户类型：

  - **如果您使用个人银行账户**（即使有 Udyam/MSME 证书）：您被视为 **个人** → 提交 **W-8BEN**
  - **如果您使用公司银行账户**（注册实体）：您是 **组织** → 提交 **W-8BEN-E**

  <Note>
  关键因素是您的银行账户是以个人名义还是公司名义开设，而不是您是否拥有商业注册证书。
  </Note>
</Accordion>

<Accordion title="Q41: 如果 Dodo 处理税务，为什么我还需要提交 W-8BEN(-E)？">
  A: Dodo 处理客户的销售税。W-8BEN(-E) 是为了支付给您的款项的预扣税。

  <Note>
  正确提交 W-8BEN 可以帮助您避免根据印度-美国税收条约的双重征税。
  </Note>
</Accordion>

## 付款与交易

<Accordion title="Q42: 我可以通过 Dodo Payments 接受哪些类型的付款？">
  A: Dodo Payments 支持多种支付方式，包括 **信用卡**、**借记卡** 和 **数字钱包**。我们还支持一次性付款和数字产品的定期订阅付款。
</Accordion>

<Accordion title="Q43: 为什么第一次付款标记为成功且金额为 $0？">
  A: 如果产品有试用期，第一次付款会自动成为 $0 的授权付款。这是预期的行为，确认授权已成功创建。
</Accordion>

<Accordion title="Q44: 试用期结束后，客户需要手动再次支付（产品价格（例如 $25））吗？">
  A: 不需要。一旦试用结束，系统会自动为客户生成产品价格（例如 $25）。他们不需要手动支付。
</Accordion>

<Accordion title="Q45: 如果我的应用与 Dodo Payments 仪表板上显示的试用期不同，我该怎么办？">
  A: 这通常表示您这边存在设置或实施问题。首先在测试模式下测试试用 + 订阅流程，确认其正常工作，然后切换到实时模式。
</Accordion>

<Accordion title="Q46: 我需要在上线之前测试试用功能吗？">
  A: 是的。建议在测试模式下完全实施和测试该功能，以确保在切换到实时模式之前一切正常。
</Accordion>

<Accordion title="Q47: 我如何跟踪付款状态？">
  A: 您可以从仪表板的 **付款** 部分跟踪付款状态。每笔交易都有状态指示：**进行中**、**失败** 或 **成功**。
</Accordion>

<Accordion title="Q48: 如果付款失败，我该怎么办？">
  A: 如果付款失败，请检查 **付款详情页面** 以获取更多信息。常见原因包括付款详情不正确、资金不足或网络问题。您可以将付款链接重新发送给客户以重试付款。
</Accordion>

<Accordion title="Q49: 商户在用户付款失败时会收到通知吗？">
  A: 是的。Dodo Payments 会通过多种渠道在实时模式下通知商户用户的付款失败：
  - 电子邮件通知
  - Webhook 事件
  - 移动通知
  - 仪表板通知
这确保商户及时了解失败的交易。
</Accordion>

<Accordion title="Q50: 如果付款失败，订阅会发生什么？">
  A: 订阅不会立即取消。相反，它会被移至搁置状态，直到成功付款。

  如果您对卡片有问题，可以通过：
  - 点击仪表板中的搁置订阅并使用 **更新付款方式** 按钮
  - 或使用 API 端点：https://docs.dodopayments.com/api-reference/subscriptions/update-payment-method
</Accordion>

<Accordion title="Q51: 如果我自己重试付款，我需要失败的付款 ID 吗？">
  A: 不需要。您只需订阅 ID 即可发起新的付款尝试。
</Accordion>

<Accordion title="Q52: 如果之前的周期成功，为什么订阅付款会因 INCORRECT_NUMBER 失败？">
  A: 如果客户的卡在上一个计费周期后被取消、替换、停用或被其银行或支付提供商标记，订阅付款可能会因 INCORRECT_NUMBER 失败。Dodo Payments 只是显示支付提供商返回的错误信息。
</Accordion>

<Accordion title="Q53: 即使我的企业有无退款政策，退款申请是否适用？">
  A: 是的。即使您在条款或网站中声明“无退款”，退款申请在很大程度上受持卡人银行和卡网络规则（例如，Visa、Mastercard）的管辖。“无退款”政策并不覆盖卡网络的争议解决流程。如果客户提出争议，您的企业仍需接受退款调查。这意味着您必须提供证据（交付证明、使用日志、条款接受等）来对抗或管理争议。如果客户的银行决定支持持卡人，则无论您的政策如何，退款费用和交易撤销将适用。我们建议在出现退款申请时进行清晰沟通、跟踪客户使用情况并提供有力证据。
</Accordion>

<Accordion title="Q54: 为什么 payment.status 有时为 null？">
  A: payment.status 可能在短时间内为 null：
  - 在创建后但在处理完成之前
  - 在订阅计费失败或重试期间

  最终，它会变为：成功、失败或取消。

  <Note>
  使用 webhook 事件，如 payment.succeeded 或 payment.failed 进行更可靠的状态跟踪。
  </Note>
</Accordion>

<Accordion title="Q55: 为什么 product_cart 在结账时为 null？">
  A:
  - 对于订阅付款，product_cart 为空，因为没有一次性商品被购买
  - product_cart 仅在一次性付款时填充
</Accordion>

<Accordion title="Q56: 为什么我在测试期间我的真实卡被阻止？">
  A: 您可能在实时模式下进行测试，而不是测试模式。

  实时付款：
  - 使用真实货币
  - 通过真实银行进行
  - 如果交易量低或不一致，可能会被标记为可疑

  ✅ 始终使用：
  - 仪表板中的测试模式
  - 文档中提供的测试卡号
</Accordion>

<Accordion title="Q57: 我的客户完成了付款，但我没有收到 webhook？">
  A: 检查清单：
  - 确认在仪表板 > 开发者 -> Webhooks 中配置了 webhook
  - 检查 webhook 是否因非 2xx 响应而静默失败
  - 检查日志：是否返回了 429 或超时？
</Accordion>

<Accordion title="Q58: 为什么我的仪表板显示成功，但我的应用没有更新？">
  A: 这可能是因为您仅依赖重定向 URL 查询参数，而没有处理：
  - Webhooks
  - API 确认调用

  ✅ 修复：
  - 始终监听 payment.succeeded webhook
  - 或在重定向后查询 API 以确认交易
</Accordion>

<Accordion title="Q59: 我没有看到 UPI 作为支付选项——为什么？">
  A: 这可能有几个原因：
  - 账单国家未设置为 IN
  - UPI 未为您的企业启用

  如果您是非印度商户，UPI 将不会出现在您的客户面前。
</Accordion>

<Accordion title="Q60: Dodo 支持 UPI 订阅吗？">
  A: 是的，UPI 订阅在符合 RBI 的授权下得到支持。UPI 订阅在 RBI（印度储备银行）法规下运营，具有特定要求：

  - **授权限制**：
    - 收费低于 ₹15,000：创建 ₹15,000 INR 的按需授权
    - 收费 ₹15,000 或以上：创建确切订阅金额的订阅授权
  - **48 小时处理延迟**：收费在预定日期发起，但实际扣款发生在发起后 48 小时（可能延长至 2-3 小时）
  - **授权取消**：客户可以在 48 小时窗口内通过其银行应用取消授权

  <Note>
  UPI 订阅具有与印度卡订阅相同的 RBI 合规授权要求。有关详细信息，请参见 [支付方式文档](/features/payment-methods#upi-and-rupay-india)。
  </Note>
</Accordion>

<Accordion title="Q61: 我如何在静态支付链接上启用 Google Pay / Apple Pay？">
  A: Google Pay 和 Apple Pay 会在实时模式下自动出现在静态支付链接上。您无需采取任何额外步骤来启用它们。
</Accordion>

<Accordion title="Q62: 我如何在 Dodo Payments 中启用 3DS？">
  A: 您可以直接在 Dodo Payments 仪表板的业务设置中启用 3DS。只需转到业务设置 → 支付设置并打开 3DS 选项。
</Accordion>

<Accordion title="Q63: 我如何在不通过 API 传递账单详细信息的情况下收集账单信息？">
  A: 如果您希望用户自己输入账单信息：
  - 使用静态链接，自动收集账单
  - 对于动态流程，您必须手动收集信息并通过 API 传递

  目前在动态链接创建期间没有内置 UI 用于收集账单信息。
</Accordion>

<Accordion title="Q64: 静态和动态支付链接有什么区别？">
  A: **静态支付链接** 是无代码选项，旨在快速使用。它们：
  - 从仪表板配置
  - 自动收集账单信息
  - 不能根据客户/会话进行自定义

  **动态支付链接** 通过 API 创建。它们：
  - 需要您明确传递账单信息
  - 可以实时自定义（价格、用户 ID、重定向 URL 等）

  <Tip>
  如果您不想编写代码，请选择静态链接。如果您需要灵活性或想要构建自定义工作流程，请使用动态链接。
  </Tip>
</Accordion>

<Accordion title="Q65: 当有人开始订阅时，我会收到两个 webhook 事件吗？">
  A: 在第一次成功的订阅付款时，如果没有试用期，您通常会收到这三个 webhook：
  - `subscription.created` webhook
  - `payment.succeeded` webhook
  - `subscription.renewal` webhook
</Accordion>

<Accordion title="Q66: subscription.cancelled_at 是什么意思？这是计划取消吗？">
  A: Dodo 支持立即和计划取消。

  当取消请求被发出时：
  - **立即取消**：subscription.cancelled_at 时间戳会立即设置，订阅在那一刻结束
  - **计划取消**：在当前计费周期结束时使用 `cancel_at_next_billing_date` 参数在订阅补丁 API 中进行取消

</Accordion>

<Accordion title="Q67: 订阅何时收费？">
  A: 如果产品没有试用期，Dodo Payments 会在订阅创建时立即向客户收费。

  <Note>
  对于印度卡和 UPI 订阅，定期收费遵循 RBI 规定，具有 48 小时处理延迟。收费在预定日期发起，但实际从客户账户扣款发生在发起后 48 小时。这个 48 小时窗口可能会根据银行 API 响应延长至 2-3 小时。
  </Note>
</Accordion>

<Accordion title="Q68: 为什么更改计划 API 返回空响应？">
  A: 更改计划 API 仅返回 200 状态码而没有响应体，因此即使计划更改成功，响应看起来也会是空的。
</Accordion>

<Accordion title="Q69: 为什么在从月度计划切换到年度计划时没有提示新的付款？">
  A: 在更改计划时，Dodo Payments 会自动向保存的卡收费。如果收费失败，订阅会被移至“搁置”。因此不会触发手动付款流程。
</Accordion>

<Accordion title="Q70: 为什么我的产品价格（₹100）在结账时显示更高的金额（₹104）？">
  A: 当自适应货币启用时，会发生这种情况。该功能会自动增加额外的 4% 收费。
</Accordion>

<Accordion title="Q71: 为什么结账忽略现有的活跃订阅并创建一个新的？">
  A: 结账始终会创建一个新的订阅。它不会检查客户是否已经有活跃的订阅。您的应用必须阻止现有订阅者开始结账会话，以避免重复收费。
</Accordion>

<Accordion title="Q72: 为什么在订阅升级/降级时我看到 TRANSACTION_NOT_ALLOWED 错误？">
  A: 此错误来自发卡行，而不是 Dodo Payments。银行因该卡的限制而阻止交易。

  常见原因：
  - 卡不允许进行在线、国际或定期付款
  - 商户类别（MCC）被阻止
  - 银行的欺诈规则阻止了收费
  - 预付卡或受限卡可能不支持这些付款
</Accordion>

<Accordion title="Q73: 是否有办法在不进行真实付款的情况下测试实时模式？">
  A: 是的。您可以通过创建 100% 折扣码来测试实时模式结账流程。这使您能够完成完整的付款和重定向流程，而无需支付费用。
</Accordion>

<Accordion title="Q74: 什么是 $0 付款方式？">
  A: 当应付总额变为零（例如，通过应用 100% 折扣）时，会发生 $0 付款。在这种情况下，不会处理实际付款，但结账和订阅流程仍然正常工作。
</Accordion>

<Accordion title="Q75: 我的账户似乎被暂停，但合规性说一切正常。我该怎么办？">
  A: 在大多数情况下，这发生是因为您不小心切换到了旧的或不活跃的商业账户。只需在 Dodo Payments 仪表板中切换回您的活跃商业账户。如果切换后问题仍然存在，请联系 Dodo Payments 支持以获取帮助。
</Accordion>

## 支付与银行集成

<Accordion title="Q76: Dodo Payments 的支付是如何工作的？">
  A: 支付是将您收集的销售收入转移到您链接的银行账户。Dodo Payments 支持多种支付周期：

  - **双月（默认）**：
    - 计费周期 1：1 日至 15 日 → 当月 18 日支付
    - 计费周期 2：16 日至月底 → 下月 4 日支付
  - **每周**：适用于交易量较大的企业（联系支持请求）
  - **每月**：
    - 计费周期：1 日至月底 → 下月 11 日支付

  仅在所有验证成功且达到支付阈值时才会发起支付。通常在支付发起后 1-2 个工作日内，资金会出现在您的银行账户中。

  <Note>
  如果支付处理日期恰逢周末或银行假期，可能会出现延迟。所有企业的默认支付周期为双月。
  </Note>
</Accordion>

<Accordion title="Q77: 支付处理后，费用和税收后的最低阈值金额是多少？">
  A: 最低支付阈值为 **$50 USD**（或 ₹1,000 INR），在扣除平台费用和税收后。您可以直接从仪表板自定义您的 USD 钱包阈值为任何金额 **≥ $50**。如果您的余额未达到阈值，它将转入下一个支付周期。

  您可以在仪表板的 [支付部分](https://app.dodopayments.com/business/payouts) 查看确切的钱包金额并管理您的支付阈值。

  <Note>
  灵活阈值功能目前适用于 USD/EUR/GBP 钱包。INR 钱包阈值保持固定为 ₹1,000。
  </Note>
</Accordion>

<Accordion title="Q78: 我如何链接我的银行账户以接收支付？">
  A: 要链接您的银行账户，请导航到仪表板的 **支付设置** 部分，然后单击 **添加银行账户**。您需要输入银行详细信息并验证账户。
</Accordion>

<Accordion title="Q79: 我可以跟踪我的支付状态吗？">
  A: 是的，您可以从仪表板的 **支付** 部分跟踪您的支付状态。
</Accordion>

<Accordion title="Q80: 支付费用是什么，费用是多少？">
  A: 支付费用是指将我们从客户那里收集到的钱转移到您的银行账户的交易成本。我们不断改进以通过与您所在国家的本地合作伙伴合作来降低支付费用。有关详细信息，请参阅 [定价](https://dodopayments.com/pricing)。
</Accordion>

<Accordion title="Q81: 我收到了“支付处理”电子邮件——资金何时到达？">
  A: 支付通常需要 1-2 个工作日才能在您的银行账户中反映。

  <Note>
  转账不会在周末或银行假期结算。如果您的支付在周五处理，请期待在周一或周二到账。
  </Note>
</Accordion>

<Accordion title="Q82: 我如何将 Payoneer 添加为支付账户？">
  A: 只需在仪表板下的银行信息表单中输入您的 Payoneer 银行详细信息。尚未提供单独的 Payoneer 集成。
</Accordion>

<Accordion title="Q83: 我可以将支付接收至 Payoneer 账户吗？">
  A: 可以。您可以通过以下方式通过 Payoneer 接收支付：
  - 在 Dodo 仪表板的银行信息表单中输入您的 Payoneer 账户详细信息
  - 确保您的账户已获得批准并支持 USD 或您要提取的货币

  目前没有专用的 Payoneer 集成——它被视为常规银行账户。
</Accordion>

<Accordion title="Q84: 我的银行信息和合规表格仍在审核中——我该怎么办？">
  A: 请耐心等待。我们的合规团队通常在 1-2 个工作日内审核表格。如果紧急：
  - 发送电子邮件至 support@dodopayments.com
</Accordion>

<Accordion title="Q85: 如果我的支付日恰逢周五或周末，会发生什么？">
  A: 如果您的支付日恰逢周五，通常会在同一天处理。然而：
  - 如果有银行假期或周末，可能会延迟到下一个工作日（通常是周一）。
  - 确切的时间可能取决于您银行的处理规则。

  <Note>
  如果有疑问，请检查您的仪表板或使用您的注册电子邮件询问支持。
  </Note>
</Accordion>

<Accordion title="Q86: 如果我有紧急发布或广告活动计划，如何加快支付合规？">
  A: 如果您已提交所有内容并在等待支付合规：
  - 在支持聊天中清楚地提及您的紧急情况（例如，实时发布、付费活动）。
  - 包括您的业务 ID 和电子邮件。
  - 避免垃圾邮件或标记多个团队成员——这可能会减慢队列。
</Accordion>

<Accordion title="Q87: 我提交了错误的银行账户详细信息。我该如何更新？">
  A: 如果您输入了错误的银行详细信息：
  - 通过 Intercom 或 Discord 或邮件联系支持。
  - 如果通过 Discord 联系，请 DM 您的电子邮件 ID 并请求手动重置银行验证部分。
  - 等待合规团队重新打开该部分，以便您重新提交。

  <Note>
  在提交后，可能不允许直接在仪表板中编辑银行详细信息。
  </Note>
</Accordion>

<Accordion title="Q88: 如果我有注册公司，是否可以使用我的个人银行账户？">
  A: 不可以。如果您注册为公司或法人实体，必须使用该实体的银行账户。
</Accordion>

<Accordion title="Q89: 我的国家不使用 SWIFT 代码。这将如何影响我的验证？">
  A: 如果您的国家不发放 SWIFT 代码，这不会阻止您的验证，但需要我们的合规团队进行人工审核。

  应该怎么做：
  - 联系支持并解释您的国家不发放 SWIFT 代码
  - 提交官方银行文件以确认：
    - 您的账户号码
    - 账户持有人姓名
    - 您的国家不发放 SWIFT 代码
  - 在某些情况下，可能会根据您的国家使用通用占位符 SWIFT

  <Note>
  此案例将经过人工合规审核。为了更快处理，请发送电子邮件至 compliance@dodopayments.com 进行升级。
  </Note>
</Accordion>

<Accordion title="Q90: 如果我的国家不使用 SWIFT 代码，我可以提供什么替代文件？">
  A: 如果您的国家不使用 SWIFT 代码，请提供以下替代文件：

  - 显示您账户详细信息的官方银行对账单或证书
  - 银行信函确认您的账户号码、持有人姓名，以及不发放 SWIFT 代码
  - 任何政府签发的文件，确认您银行的路由信息格式

  <Tip>
  在提交之前，请联系 support@dodopayments.com，以确保您提供适合您国家的正确文件格式。
  </Tip>
</Accordion>

<Accordion title="Q91: 我可以将支付阈值设置为低于 $50 吗？">
  A: 不可以，最低支付阈值不能设置低于 $50。 
  您可以将支付阈值提高到 $50 以上，但不支持设置为低于 $50。
</Accordion>

<Accordion title="Q93: 我可以自定义发票吗？">

</Accordion>

<Accordion title="Q94: 我如何访问或下载发票？">
  A: 您可以从仪表板的 **发票** 部分查看或下载发票。每笔付款都有一个相关的发票，可以下载为 **PDF**。

<Accordion title="Q95: 我如何查看我的交易和支付报告？">
  A: 您可以从仪表板的 **报告** 部分生成详细报告。这些报告包括交易、退款和支付的数据，您可以按日期过滤并以 CSV 格式导出。
</Accordion>

## 退款与争议

<Accordion title="Q96: 我如何发起退款？">
  A: 要发起退款，请转到您要退款的交易的 **付款详情页面**，然后单击 **发起退款** 按钮。您可以发起 **全额** 或 **部分** 退款，前提是付款符合退款资格规则。
</Accordion>

<Accordion title="Q97: 客户收到退款需要多长时间？">
  A: 一旦发起退款，客户通常会在 **3-5 个工作日** 内收到退款金额，具体取决于付款方式和他们的银行。

## Webhook 与集成

<Accordion title="Q100: 如何计算按比例分配——是按产品价格还是总金额？">
  A: 按比例分配是基于 **支付的总金额** 计算的，而不仅仅是基础产品价格。 
  这意味着如果应用了折扣，按比例分配是基于 **折扣后的最终金额** 计算的。
</Accordion>

<Accordion title="Q101: 为什么我们在第一次订阅时使用 `subscription.renewed` webhook？">
  A: 在第一次付款成功后，订阅进入其第一个计费周期，并设置下一个计费日期。 
  `subscription.renewed` webhook 被发送以指示计费周期现在处于活动状态，并且下一个计费周期已安排。 
  此 webhook 用于每个计费周期——包括第一个——以便每次都能一致地处理计费逻辑。
</Accordion>

<Accordion title="Q102: 我可以在 Python 中手动验证 Dodo Payments webhook 吗？">
  A: 是的。手动验证是可能的，但必须严格遵循正确的签名步骤。为了更简单和更可靠的方法，Dodo Payments 建议使用 standardwebhooks 包：
  

  ```python
  from standardwebhooks.webhooks import Webhook
  
  wh = Webhook(webhook_secret_key)
  payload = request.get_data(as_text=True)
  headers = request.headers
  
  if not wh.verify(payload, headers):
      return jsonify({'error': 'Invalid webhook signature'}), 401
 
  Then parse the payload as needed.
  ```

  </Accordion>

<Accordion title="Q104: 我的 webhook 没有接收到 POST 请求，只有 GET。为什么？">

要接收来自 Dodo Payments 的结构化 POST 有效负载：
- 在仪表板 → 开发者 → Webhooks 下设置 webhook
- 确保 URL 接受 POST 请求并验证签名

<Note>
您可以同时拥有 webhook 和重定向 URL——它们的用途不同。
</Note>
</Accordion>

<Accordion title="Q107: webhook 和重定向 URL 有什么区别？">

<Accordion title="Q109: 我如何验证我收到的 webhook 确实来自 Dodo Payments，而不是来自任何恶意行为者？">
A: 要验证您收到的 webhook 确实来自 Dodo Payments，而不是来自恶意行为者，请遵循以下步骤：
1. **检查签名**：Dodo Payments 发送的每个 webhook 都在 **`webhook-signature`** 头中包含唯一签名。此签名是使用您的 webhook 秘密密钥和事件的有效负载生成的。
2. **检索秘密密钥**：确保您拥有 webhook 的秘密密钥，该密钥可以在 Dodo Payments 仪表板的 Webhooks 设置中找到。
3. **重新计算签名**：使用与 Dodo Payments 相同的签名算法（HMAC-SHA256）在您的服务器上计算签名。使用 webhook 的有效负载和您的 webhook 秘密密钥。
4. **比较签名**：将重新计算的签名与 **`webhook-signature`** 头中提供的签名进行比较。如果它们匹配，则 webhook 是真实的；如果不匹配，则应拒绝。
5. **使用 HTTPS**：确保您的 webhook 端点使用 HTTPS 来保护通信并防止数据传输过程中的篡改。
通过实施这些步骤，您可以自信地验证 Dodo Payments 发送的 webhook 的真实性。

<Tip>我们建议使用 StandardWebhooks 提供的包来验证签名，而不是自己构建。</Tip>
</Accordion>

<Accordion title="Q110: 为什么即使是第一个月的付款，我也会收到 subscription.renew webhook？">
A: 对于订阅，`subscription.renew` 会在每次扣除订阅金额时触发。因此，是的，这是预期的行为。
</Accordion>

<Accordion title="Q111: 我如何在本地测试 webhooks？">
A: 要在本地测试 webhooks，您可以按照以下步骤操作：

1. **使用隧道工具**：像 [ngrok](https://ngrok.com/)、[LocalTunnel](https://localtunnel.github.io/www/) 或 [Cloudflare Tunnel](https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/) 这样的工具可以将您的本地服务器暴露到互联网。这提供了一个公共 URL，Dodo Payments 可以用来将 webhook 事件发送到您的本地环境。
2. **设置 webhook 端点**：在您的本地应用中创建一个 webhook 端点以处理传入的 webhook 事件。确保它监听 HTTP POST 请求。
3. **在 Dodo Payments 中配置 webhook**：转到您的 Dodo Payments 仪表板，导航到 Webhooks 设置，并将 webhook URL 更新为隧道工具生成的公共 URL。
