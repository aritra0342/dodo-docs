## 测试卡详细信息

Dodo Payments 提供了一组测试卡号，您可以使用这些卡号来模拟各种支付场景。您可以使用这些测试卡号在测试模式下创建模拟支付，而无需处理实际的支付或费用。

{/* LOCKED_PATTERN_6d744560e4135463c359b094ae69cd5f */}
{/* LOCKED_PATTERN_717af1b1289e4c0b6d1c95c8c204d71e */}
测试交易不会处理实际支付，也不会影响真实账户。
</Card>

{/* LOCKED_PATTERN_5ba03ec58a56547ac9750ef18da96a7f */}
所有 API 端点在测试和上线模式下表现一致。
</Card>

{/* LOCKED_PATTERN_1f50b75a756a498817926e42de6e14a9 */}
测试交易会触发 Webhook，就像生产环境一样。
</Card>
</CardGroup>

<Warning>
测试凭证仅在测试模式下有效。尝试在上线模式中使用会导致支付失败。
</Warning>

## 测试 Afterpay 详细信息

在测试您的结账集成时，选择 Afterpay 作为支付方式并点击支付按钮。

### 成功支付
- 使用任何有效的电子邮件地址和送货地址进行成功支付。

| Status | UPI ID |
| :----- | :----- |
| 成功 | `success@upi` |
| 失败 | `failure@upi` |

**注意**: Afterpay 交易的最低金额为 $50。

- 账单国家必须设置为 `IN`
- 货币必须是 `INR`
- 对于非印度商家：必须启用自适应货币

## 测试 Apple Pay

您可以使用您的 Dodo Payments 测试账户来测试 Apple Pay。

<Info>
您需要一台 iPad、iPhone 或 Mac 来测试 Apple Pay。请在 Mac 上使用 Safari。
</Info>

<Steps>
<Step title="为您的 Dodo Payments 账户启用 Apple Pay">
在您的 Dodo Payments 账户设置中启用 Apple Pay。
</Step>

<Step title="注册 Apple ID">
如果您还没有 Apple ID，请[注册一个 Apple ID](https://support.apple.com/apple-id)。
</Step>

<Step title="向 Apple Pay 添加至少一张卡片">
[向 Apple Pay 添加至少一张卡片](https://support.apple.com/HT204506)。在使用 Apple Pay 测试时，您使用真实的卡片信息，但不会扣款。
</Step>

<Step title="选择一张卡并完成测试购买">

### 亚马逊支付、Cash App 支付和 RevolutPay

使用您的测试 API 密钥并按照标准结账流程进行操作。测试交易是在没有实际收费的情况下模拟的。

有关完整数字钱包文档，包括 Apple Pay 的域名验证，请参见 [数字钱包](/features/payment-methods/digital-wallets) 页面。

## 测试欧洲支付方法

欧洲支付方法（iDEAL、Bancontact、EPS、Multibanco）可以在沙盒模式中进行测试。

<Steps>
{/* LOCKED_PATTERN_540056f13df545529727751bb5b93f77 */}
使用你的 Dodo Payments 测试 API 密钥。
</Step>

{/* LOCKED_PATTERN_73a737ada9108be9fcbfe7701d196c25 */}
将账单地址国家设置为与支付方式匹配：
- `NL` 适用于 iDEAL
- `BE` 适用于 Bancontact
- `AT` 适用于 EPS
- `PT` 适用于 Multibanco
</Step>

{/* LOCKED_PATTERN_111b236b56d1ef796758b64d1fe1fcd1 */}
欧洲支付方式需要使用 EUR 货币。
</Step>

{/* LOCKED_PATTERN_3efbb33511174ca4755c102c30869543 */}
在测试环境中遵循模拟银行认证流程。
</Step>
</Steps>

有关完整欧洲支付方法的文档，请参见 [欧洲](/features/payment-methods/europe) 页面。

## 测试最佳实践

<AccordionGroup>
{/* LOCKED_PATTERN_065a271b5abfd247b6320fbca34af90c */}
不仅要测试成功支付，还要测试拒付、取消以及资金不足等边缘情况。
</Accordion>

{/* LOCKED_PATTERN_b9eb520012676a2badc82027085ca781 */}
确保你的 webhook 端点正确处理所有事件类型，尤其是 `payment.succeeded`、`payment.failed` 以及订阅事件。

使用 [Dodo Payments CLI](/developer-resources/sdks/cli#webhook-testing) 在本地测试 webhooks：
- `dodo wh listen` 将实时测试 webhooks 转发到你的本地服务器
- `dodo wh trigger` 发送所有 22 种事件类型的模拟负载
</Accordion>

{/* LOCKED_PATTERN_7fd9589439eedb7dd6977012ac791713 */}
对于 Apple Pay 和 Google Pay，请在实际的 iOS 和 Android 设备上测试。模拟器无法完全复现钱包行为。
</Accordion>

{/* LOCKED_PATTERN_ef9afd776f8103114546b31cb4fef641 */}
区域性支付方式（如 UPI、iDEAL 等）需要匹配的账单地址。美国账单地址不会显示 iDEAL。
</Accordion>

{/* LOCKED_PATTERN_52e65b7bb9610a0aa115d3b6ba276057 */}
BNPL 支付方式要求最低 $50.01。测试它们是否会根据购物车总额正确显示或隐藏。
</Accordion>
</AccordionGroup>
