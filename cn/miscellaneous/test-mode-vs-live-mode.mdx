### 环境依赖性

**依赖环境：** 所有依赖环境的功能在测试模式和实时模式中将具有独立的数据。在一个环境中所做的任何更改不会影响另一个环境。例如，在测试模式中生成的 API 密钥将独立于在实时模式中生成的 API 密钥，反之亦然。

**独立于环境：** 所有独立于环境的功能在实时模式和测试模式中将具有相同的数据。例如，我的账户部分在测试模式和实时模式中将具有相同的数据。在一个环境中所做的更改将影响另一个环境。

仅在成功完成身份验证和业务验证后，实时模式才会可用。您只能在 \`live mode\` 中接受真实付款。

测试模式和实时模式的 API 主机名如下：
- 测试模式: [`https://test.dodopayments.com`](https://test.dodopayments.com/)
- 实时模式: [`https://live.dodopayments.com`](https://live.dodopayments.com/) 

## 关键亮点

1. 在测试模式或实时模式中创建的所有产品仅在各自的环境中可用。在测试模式环境中有一个复制到实时模式的选项。这将复制产品及所有相关细节，并在实时模式中创建相同的产品。

2. 在测试模式中创建的 API 密钥和 Webhook 仅限于测试模式环境，不会影响实时模式的部署，反之亦然。

3. 提交的所有验证文件和提交的银行账户详细信息在测试模式和实时模式中将是相同的。在任何环境中更改此信息也会在另一个环境中更改。
