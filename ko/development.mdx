<Info>
  **전제 조건**: 진행하기 전에 Node.js(버전 19 이상)를 설치해 주세요.
</Info>

다음 단계를 따라 운영 체제에 Mintlify를 설치하고 실행하세요:

**1단계**: Mintlify 설치:

<CodeGroup>

  ```bash npm
  npm i -g mintlify
  ```

```bash yarn
yarn global add mintlify
```

</CodeGroup>

**2단계**: docs 디렉토리(`mint.json` 파일이 위치한 곳)로 이동하여 다음 명령어를 실행하세요:

```bash
mintlify dev
```

문서의 로컬 미리보기를 `http://localhost:3000`에서 확인할 수 있습니다.

### 사용자 정의 포트

기본적으로 Mintlify는 포트 3000을 사용합니다. `--port` 플래그를 사용하여 Mintlify가 실행되는 포트를 사용자 정의할 수 있습니다. 예를 들어, Mintlify를 포트 3333에서 실행하려면 다음 명령어를 사용하세요:

```bash
mintlify dev --port 3333
```

이미 사용 중인 포트에서 Mintlify를 실행하려고 하면, 다음 사용 가능한 포트를 사용합니다:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Mintlify 버전

각 CLI 릴리스는 특정 Mintlify 버전과 연결되어 있음을 유의하세요. 로컬 웹사이트가 프로덕션 버전과 일치하지 않으면 CLI를 업데이트하세요:

<CodeGroup>

```bash npm
npm i -g mintlify@latest
```

```bash yarn
yarn global upgrade mintlify
```

</CodeGroup>

## 링크 검증

CLI는 문서에서 작성된 참조 링크를 검증하는 데 도움을 줄 수 있습니다. 끊어진 링크를 식별하려면 다음 명령어를 사용하세요:

```bash
mintlify broken-links
```

## 문제 해결

<AccordionGroup>
  <Accordion title='오류: "sharp" 모듈을 darwin-arm64 런타임을 사용하여 로드할 수 없습니다'>

    이는 구버전의 node 때문일 수 있습니다. 다음을 시도해 보세요:
    1. 현재 설치된 mintlify 버전을 제거합니다: `npm remove -g mintlify`
    2. Node v19 이상으로 업그레이드합니다.
    3. mintlify를 다시 설치합니다: `npm install -g mintlify`
  </Accordion>

  <Accordion title="문제: 알 수 없는 오류 발생">
  
    해결 방법: 장치의 루트로 이동하여 \~/.mintlify 폴더를 삭제합니다. 그 후, `mintlify dev`를 다시 실행합니다.
  </Accordion>
</AccordionGroup>
