## 새로운 기능 🚀

1. **혼합 장바구니 체크아웃: 일회성 및 구독 제품을 단일 장바구니에**  
   체크아웃은 이제 일회성 결제 제품과 구독 제품을 단일 장바구니에 결합하는 것을 지원하여 강력한 새로운 청구 사용 사례를 열어줍니다.
   
   <Frame>
   <img src="/images/changelog/mixed-checkout.png" alt="일회성 및 구독 제품이 혼합된 체크아웃" />
   </Frame>
   
   ### 이 기능이 가능하게 하는 것
   
   - **설치비 + 구독**: 단일 체크아웃에서 반복 구독과 함께 일회성 설치비를 청구
   - **반복 구성 요소가 포함된 번들**: 지속적인 서비스 접근(구독)과 함께 번들로 제공되는 제품(일회성) 판매
   - **강의 + 멤버십 조합**: 선택적 커뮤니티 멤버십 구독과 함께 강의 구매 제공
   - **평생 + 추가 구독**: 선택적 반복 프리미엄 기능과 함께 평생 제품 접근 결합
   
   ### 작동 방식
   
   일회성 및 구독 제품을 체크아웃 세션에 간단히 추가하세요:
   

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [
       // One-time setup fee
       { product_id: 'prod_setup_fee', quantity: 1 },
       // Monthly subscription
       { product_id: 'prod_monthly_plan', quantity: 1 }
     ],
     customer: {
       email: 'customer@example.com',
       name: 'Jane Doe'
     },
     return_url: 'https://yourapp.com/success'
   });
   ```

   
   <Check>
   결과: 일회성 결제를 처리하고 구독을 시작하는 단일 체크아웃 흐름 — 마찰을 줄이고 전환율을 높입니다.
   </Check>
   
   <Info>
   제품 유형을 혼합할 때, 일회성 결제는 즉시 청구되며 구독은 설정한 청구 주기를 따릅니다(설정한 시험 기간 포함).
   </Info>
   
   자세히 알아보기: [체크아웃 세션](/developer-resources/checkout-session)

2. **통합 고객 포털**  
   
   고객이 Dodo Payments의 다양한 비즈니스에서 모든 구매 및 구독을 보고 관리할 수 있는 새로운 중앙 집중식 고객 포털이 [customer.dodopayments.com](https://customer.dodopayments.com)에 있습니다.
   
   <Frame>
   <img src="/images/changelog/unified-customer-portal.png" alt="통합 고객 포털" />
   </Frame>
   
   ### 포털 기능
   
   - **비즈니스 간 가시성**: Dodo Payments를 사용하는 모든 비즈니스의 모든 구매 및 구독을 한 곳에서 볼 수 있습니다.
   - **중앙 집중식 구독 관리**: 단일 대시보드에서 다양한 상인의 구독을 관리합니다.
   - **통합 청구 내역**: 모든 구매의 송장 및 결제 내역에 접근합니다.
   - **단일 로그인 경험**: 이메일로 한 번 로그인하여 모든 Dodo Payments 상인의 구매에 접근합니다.
   
   ### 작동 방식
   
   고객은 [customer.dodopayments.com](https://customer.dodopayments.com)을 방문하고 이메일 주소를 입력하기만 하면 됩니다. 그러면 Dodo Payments 생태계 전반에 걸쳐 모든 구매에 접근할 수 있는 안전한 로그인 링크를 받게 됩니다.
   
   <Tip>
   통합 고객 포털은 비즈니스별 고객 포털을 보완합니다. 고객은 여전히 브랜드 경험을 위해 개별 상인 포털에 접근하거나 통합 포털을 사용하여 통합된 보기를 사용할 수 있습니다.
   </Tip>
   
   <Note>
   통합 포털은 비즈니스별 포털과 동일한 이메일 기반 인증을 사용하여 일관되고 안전한 경험을 보장합니다.
   </Note>

3. **결제 응답의 송장 ID**  
   결제 응답에는 이제 `invoice_id` 필드가 포함되어 있어 결제를 송장과 조정하고 회계 시스템과 통합하기가 더 쉬워졌습니다.
   

   ```json
   {
     "payment_id": "pay_2IjeQm4hqU6RA4Z4kwDee",
     "status": "succeeded",
     "invoice_id": "inv_2IsUnWGtRKFLxk7xAQeyt",
     "amount": 9900,
     "currency": "USD",
     // ... other fields
   }
   ```

   
   ### 사용 사례
   
   - **회계 통합**: 회계 소프트웨어에서 송장에 직접 결제를 연결합니다.
   - **고객 지원**: 지원 요청을 처리할 때 결제와 관련된 송장을 빠르게 찾습니다.
   - **조정**: 결제를 송장과 프로그래밍적으로 일치시켜 재무 조정을 간소화합니다.
   - **준수**: 결제를 해당 세금 송장에 연결하여 정확한 기록을 유지합니다.
   
   <Info>
   `invoice_id` 필드는 사용 가능한 경우 인도 특정 송장 ID를 사용하여 현지 송장 요구 사항을 준수합니다.
   </Info>
   
   자세히 알아보기: [결제 세부 정보 API](/api-reference/payments/get-payments-1)
