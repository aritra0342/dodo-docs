## 새로운 기능 🚀

1. **혼합 카트 결제: 일회성 및 구독 상품을 한 카트에**  
   Checkout은 이제 일회성 결제 상품과 구독 상품을 한 카트에서 결합할 수 있어 강력한 신규 청구 사용 사례를 실현합니다.
   
   <Frame>
   <img src="/images/changelog/mixed-checkout.png" alt="Mixed checkout with one-time and subscription products" />
   </Frame>
   
   ### 가능해지는 것
   
   - **설정 비용 + 구독**: 하나의 결제에서 반복 구독과 함께 일회성 설정 비용을 청구합니다
   - **반복 구성 요소를 포함한 번들**: 일회성 상품을 지속적인 서비스 접근권(구독)과 함께 판매합니다
   - **코스 + 멤버십 조합**: 커뮤니티 멤버십 구독을 선택적으로 포함한 코스 구매를 제공합니다
   - **평생 이용권 + 추가 구독**: 평생 상품 이용권과 선택적 반복 프리미엄 기능을 결합합니다
   
   ### 작동 방식
   
   단순히 일회성 상품과 구독 상품을 모두 결제 세션에 추가하세요:
   

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [
       // One-time setup fee
       { product_id: 'prod_setup_fee', quantity: 1 },
       // Monthly subscription
       { product_id: 'prod_monthly_plan', quantity: 1 }
     ],
     customer: {
       email: 'customer@example.com',
       name: 'Jane Doe'
     },
     return_url: 'https://yourapp.com/success'
   });
   ```

   
   <Check>
   결과: 일회성 결제를 처리하고 구독을 시작하는 단일 결제 흐름 — 마찰을 줄이고 전환율을 높입니다.
   </Check>
   
   <Info>
   상품 유형을 혼합할 때 일회성 결제는 즉시 청구되며, 구독은 구성된 청구 주기(설정한 평가판 포함)를 따릅니다.
   </Info>
   
   자세히 알아보기: [Checkout Sessions](/developer-resources/checkout-session)

2. **통합 고객 포털**  
   
   고객이 Dodo Payments를 사용하는 여러 사업자의 모든 구매 및 구독을 확인하고 관리할 수 있는 새로운 중앙 집중식 고객 포털이 [customer.dodopayments.com](https://customer.dodopayments.com)에 마련되었습니다.
   
   <Frame>
   <img src="/images/changelog/unified-customer-portal.png" alt="Unified Customer Portal" />
   </Frame>
   
   ### 포털 기능
   
   - **사업자 간 가시성**: Dodo Payments를 이용하는 모든 사업자의 구매 및 구독을 한 곳에서 확인
   - **중앙 집중형 구독 관리**: 여러 사업자에 걸친 구독을 단일 대시보드에서 관리
   - **통합 청구 내역**: 모든 구매에 대한 송장과 결제 기록에 접근
   - **싱글 사인온 경험**: 이메일로 한 번 로그인해 모든 Dodo Payments 사업자의 구매에 접근
   
   ### 작동 방식
   
   고객은 [customer.dodopayments.com](https://customer.dodopayments.com)에 접속해 이메일 주소를 입력하기만 하면 됩니다. 그러면 Dodo Payments 생태계 전반의 모든 구매에 접근할 수 있는 보안 로그인 링크를 받습니다.
   
   <Tip>
   통합 고객 포털은 사업자별 고객 포털을 보완합니다. 고객은 여전히 개별 사업자의 포털에 접속해 브랜드화된 경험을 유지하거나 통합 포털을 사용해 통합된 뷰를 이용할 수 있습니다.
   </Tip>
   
   <Note>
   통합 포털은 사업자별 포털과 동일한 이메일 기반 인증을 사용하여 일관되고 안전한 경험을 보장합니다.
   </Note>

3. **결제 응답의 인보이스 ID**  
   결제 응답에 이제 `invoice_id` 필드가 포함되어 결제와 인보이스를 쉽게 대조하고 회계 시스템에 통합할 수 있습니다.
   

   ```json
   {
     "payment_id": "pay_2IjeQm4hqU6RA4Z4kwDee",
     "status": "succeeded",
     "invoice_id": "inv_2IsUnWGtRKFLxk7xAQeyt",
     "amount": 9900,
     "currency": "USD",
     // ... other fields
   }
   ```

   
   ### 활용 사례
   
   - **회계 통합**: 결제를 회계 소프트웨어의 인보이스에 직접 연결
   - **고객 지원**: 지원 요청 처리 시 결제와 연결된 인보이스를 빠르게 확인
   - **조정**: 결제를 인보이스와 프로그램 방식으로 매칭하여 재무 조정 간소화
   - **컴플라이언스**: 결제와 해당 세금 인보이스를 연결하여 정확한 기록을 유지
   
   <Info>
   `invoice_id` 필드는 인도에서 요구되는 인보이스 ID를 사용할 수 있을 때 적용하여 현지 청구 요건을 충족합니다.
   </Info>
   
   자세히 알아보기: [Get Payment Detail API](/api-reference/payments/get-payments-1)
