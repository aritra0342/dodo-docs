## 새로운 기능 🚀

1. **`subscription.updated` 웹훅**  
   모든 구독 객체 업데이트 시 발생하는 새로운 웹훅 이벤트로, 구독에 대한 변경 사항에 대한 실시간 알림을 제공합니다.
   
   이 웹훅을 사용하여:
   - 구독 세부정보가 변경될 때 즉시 알림을 받습니다.
   - Dodo Payments와 애플리케이션 상태를 동기화합니다.
   - API를 폴링하지 않고 구독 업데이트에 반응합니다.
   
   <Tip>
   `subscription.updated`에 구독하여 폴링의 필요성을 없애고 시스템을 실시간 구독 변경 사항과 동기화하세요.
   </Tip>
   
   자세히 알아보기: [구독 웹훅](/developer-resources/webhooks/intents/subscription)

2. **체크아웃에서 최소 주소 지원**  

   <Frame>
   <img src="/images/changelog/minimal-address.png" alt="체크아웃에서 최소 주소 지원" />
   </Frame>

   새로운 최소 주소 수집 모드로 체크아웃 마찰을 줄입니다. 전체 주소 세부정보를 수집하는 대신 최소 주소 모드는 다음만 수집합니다:
   
   - **국가**: 세금 결정에 항상 필요
   - **우편번호/우편 코드**: 판매세, VAT 또는 GST 계산에 필요한 지역에서만 수집
   
   <Check>
   결과: 고객을 위한 더 적은 양식 필드로 체크아웃 완료가 훨씬 빨라집니다.
   </Check>
   
   **최소 주소 활성화:**
   
   <Tabs>
   <Tab title="체크아웃 세션">

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [{ product_id: 'prod_123', quantity: 1 }],
     minimal_address: true
   });
   ```

   </Tab>
   
   <Tab title="정적 결제 링크">
   결제 링크 URL에 `minimalAddress=true`를 추가하세요:
   

   ```
   https://checkout.dodopayments.com/buy/prod_123?minimalAddress=true
   ```

   </Tab>
   </Tabs>
   
   <Info>
   기본적으로 체크아웃은 전체 주소를 계속 수집합니다. 모든 체크아웃 링크를 최소 주소 모드로 점진적으로 전환할 것입니다. 마찰 감소의 혜택을 받기 위해 지금 기능 플래그를 활성화하세요.
   </Info>

3. **읽기 전용 API 키**  
   더 안전한 통합을 위해 읽기 전용 권한으로 API 키를 생성합니다. 새 API 키를 생성할 때 **쓰기 액세스 활성화**를 선택 해제하여 읽기 전용 키를 생성하세요.
   
   읽기 전용 키는:
   - 결제, 구독 및 고객 데이터를 가져올 수 있습니다.
   - 제품 목록을 나열하고 세부정보를 검색할 수 있습니다.
   - 보고서 및 분석에 접근할 수 있습니다.
   
   읽기 전용 키는 **할 수 없습니다**:
   - 결제를 생성하거나 수정할 수 없습니다.
   - 구독을 업데이트할 수 없습니다.
   - 제품 구성을 변경할 수 없습니다.
   
   <Tip>
   대시보드 통합, 분석 도구 및 데이터를 변경하지 않고 보기만 필요한 시스템의 경우 "쓰기 액세스 활성화"를 선택 해제하세요.
   </Tip>

4. **UPI 자동 결제 및 RuPay 카드 구독 - 전체 롤아웃**  
   UPI 자동 결제 및 RuPay 카드 구독이 이제 모든 비즈니스에서 사용할 수 있습니다. 이는 인도 고객을 위한 반복 결제를 가능하게 합니다:
   
   - **UPI 자동 결제**: UPI 위임을 통한 자동 반복 청구
   - **RuPay 카드**: 인도의 국내 카드 네트워크에 대한 구독 지원
   
   <Check>
   결과: RBI 준수 위임 처리를 통해 인도 시장의 구독 결제 옵션이 확장되었습니다.
   </Check>
   
   자세히 알아보기: [RBI 준수 위임이 있는 구독](/features/subscription#subscriptions-with-rbi-compliant-mandates)

5. **플랜 변경 미리보기 API**  
   구독 플랜 변경을 커밋하기 전에 미리 볼 수 있는 새로운 API 엔드포인트입니다. 실제 수정 없이 변경 후 구독이 어떻게 보일지와 정확한 청구 금액을 확인하세요.
   

   ```javascript
   const preview = await client.subscriptions.previewChangePlan('sub_123', {
     product_id: 'prod_pro_plan',
     proration_billing_mode: 'prorated_immediately',
     quantity: 1
   });
   
   console.log('Charge amount:', preview.immediate_charge.summary);
   console.log('New plan details:', preview.new_plan);
   ```

   
   이를 사용하여:
   - 고객에게 업그레이드/다운그레이드를 확인하기 전에 정확히 얼마를 지불할지 보여줍니다.
   - 가격 계산기 및 플랜 비교 도구를 만듭니다.
   - 가격 투명성을 개선하여 지원 티켓을 줄입니다.
   
   자세히 알아보기: [플랜 변경 미리보기 API](/api-reference/subscriptions/preview-change-plan)
