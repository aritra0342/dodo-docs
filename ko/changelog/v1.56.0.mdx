## 새로운 기능 🚀

1. **새로운 온보딩 경험**  
   온보딩이 간소화되어 더 빠르게 시작하고 불필요한 지연 없이 결제를 수락할 수 있습니다.
   - 더 빠른 계정 활성화
   - 간소화된 검증 프로세스

2. **이중 인증 (2FA)**  
   이중 인증 지원으로 계정 보안을 강화합니다. 비밀번호 외에 추가 보안 계층으로 상인 계정과 팀원을 보호하세요.
   - 무단 접근으로부터 계정 보호
   - 인증 앱 지원
   - 민감한 작업에 필요

3. **오버레이 체크아웃의 세션 링크**  
   체크아웃 세션이 이제 오버레이 체크아웃과 원활하게 통합되어 더 매끄러운 임베디드 결제 경험을 제공합니다. 체크아웃 세션을 생성하고 고객을 애플리케이션에서 벗어나지 않고 오버레이에 표시하세요.

4. **C# SDK**  
   공식 Dodo Payments C# SDK가 이제 제공되어 C# 및 .NET 애플리케이션에 결제 기능을 통합할 수 있는 네이티브 .NET 지원을 제공합니다.
   - C# 인터페이스로 전체 타입 안전성
   - 모든 API 작업에 대한 Async/await 지원
   - NuGet에서 쉽게 설치 가능

   자세히 알아보기: [Dodo Payments SDKs](/developer-resources/dodo-payments-sdks)

5. **모든 SDK의 웹훅 검증**  
   모든 SDK는 이제 내장된 웹훅 검증 기능을 포함하여 들어오는 웹훅 이벤트를 안전하게 검증합니다. 업계 표준 [webhooks library](https://github.com/standard-webhooks)의 지원을 받아, 이러한 함수들은 웹훅 페이로드를 검증하고 처리하는 작업을 단순화합니다.
   
   두 가지 방법이 제공됩니다:
   - `unwrap()`: 제작 환경을 위한 웹훅 비밀 키로 웹훅 서명을 검증합니다
   - `unsafe_unwrap()`: 로컬 테스트 및 개발을 위해 검증 없이 웹훅 페이로드를 파싱합니다
   
   웹훅 검증을 사용하려면 Dodo Payments 클라이언트를 초기화할 때 `DODO_PAYMENTS_WEBHOOK_KEY`를 제공하세요. 두 함수는 자동으로 웹훅 페이로드를 파싱하고 이벤트 유형을 추출하므로 웹훅 처리가 보다 안전하고 편리해집니다.

## 개선 사항 및 업데이트 ✨

- **구독 청구 세부정보에서 고객 이름 업데이트**  
  고객 이름을 이제 구독 청구 세부정보에서 직접 수정할 수 있어 정보를 수정하거나 기록을 업데이트하기가 더 쉬워졌습니다.

- **체크아웃 세션 API**  
  체크아웃 세션 세부정보를 프로그래밍 방식으로 검색하여 상태를 확인하고 세션을 검증하거나 분석 워크플로우와 통합할 수 있습니다.

- **사용 기반 구독에 대한 계획 변경 지원**  
  사용 기반 청구 모델에 대해 구독 계획 변경이 이제 완전히 지원되어 사용 추적을 유지하면서 원활한 업그레이드 및 다운그레이드가 가능합니다.

- **구독 응답에서 세금 ID**  
  세금 식별 번호가 이제 구독 API 응답에 포함되어 준수 보고 및 송장 생성을 개선합니다.

- **API에서 3DS 플래그 강제 적용**  
  체크아웃 세션, 결제 및 구독 API에서 더 강력한 인증 요구 사항을 위한 강제 3DS 플래그가 이제 제공됩니다.

- **통화별 잔액 원장 필터링**  
  대시보드에서 이제 통화별로 잔액 원장 항목을 필터링할 수 있어 다중 통화 작업의 조정이 간소화됩니다.

- **지급 한도 \$50로 축소**  
  상인은 이제 지급 한도를 \$50로 설정할 수 있어 소규모 비즈니스에 더 많은 유연성을 제공하고 자금에 더 빠르게 접근할 수 있습니다.

- **경미한 개선 및 버그 수정**  
  플랫폼 전반에 걸쳐 일반적인 개선 및 안정성 향상이 이루어졌습니다.
