## 새로운 기능 🚀

### 향상된 분석 대시보드

우리는 비즈니스 성과를 더 잘 이해할 수 있도록 포괄적인 수익 및 유지 관리 통찰력을 제공하는 분석 대시보드를 확장했습니다.

#### 수익 분해 분석

상세한 분해를 통해 반복 수익 지표를 추적하세요:

- **예상 ARR**: 추세 분석이 포함된 연간 반복 수익 예측
- **MRR 분해 및 성장률**: 월별 반복 수익 분석 및 월간 성장 지표  
- **신규 MRR 차트**: 신규 고객 수익 획득의 시각적 표현
- **확장 MRR 차트**: 기존 고객의 업그레이드 및 확장에서 발생하는 수익 성장 추적
- **이탈 MRR 차트**: 고객 취소 및 다운그레이드로 인한 손실 수익 모니터링
- **순 신규 MRR 차트**: 신규, 확장 및 이탈 MRR을 결합한 순 수익 영향

#### 유지 관리 분석

고객 유지 및 이탈 패턴을 모니터링하세요:

- **고객 이탈률**: 구독을 취소한 고객의 비율
- **수익 이탈률**: 취소 및 다운그레이드로 인한 손실 수익의 비율
- **이탈률 추세**: 시간에 따른 고객 및 수익 이탈 패턴의 역사적 추적
- **사용자 유지 매트릭스**: 다양한 기간에 걸친 고객 유지율을 보여주는 코호트 분석

### 보고서 2.0

우리는 보고서 섹션을 완전히 개편하여 더 포괄적이고 상세한 보고 기능을 추가했습니다. **비즈니스 > 보고서**로 이동하여 새로운 보고서 모음을 이용하세요.

사용 가능한 보고서는 다음과 같습니다:

- **결제 보고서**: 거래 상태 추적이 포함된 모든 처리된 결제의 상세 기록
- **환불 보고서**: 상세한 환불 정보가 포함된 모든 환불 활동의 포괄적인 로그
- **지급 보고서**: 귀하의 계좌로 이루어진 지급의 요약
- **제품별 판매 보고서**: 수익 및 비즈니스 성장 지표가 포함된 판매 성과 통찰력
- **결제 실패 보고서**: 실패한 결제 시도의 요약 및 실패 사유
- **고객 보고서**: 고객별로 정리된 결제 요약
- **분쟁 보고서**: 해결 상태가 포함된 결제 분쟁 요약
- **계정 요약 보고서**: 전체 계정 개요 및 요약

<Note>
Dodo Payments의 MCP 통합을 시작하는 가장 빠른 방법입니다. 서버 설치나 유지 관리가 필요 없습니다.
</Note>

<Steps>
<Step title="원격 MCP 서버 열기">
    브라우저에서 <a href="https://mcp.dodopayments.com" target="_blank">https://mcp.dodopayments.com</a>를 방문하세요.
</Step>
<Step title="MCP JSON 구성 복사하기">
    사이트에서 귀하의 클라이언트를 위해 제공된 MCP JSON 구성을 복사하세요.
</Step>
<Step title="AI 애플리케이션 구성하기">
    애플리케이션의 MCP 설정에 구성을 붙여넣으세요 (예: Cursor 또는 Claude).
    <Tip>
    Cursor의 경우, 귀하의 구성은 다음과 같아야 합니다:
    </Tip>

    ```json
    {
      "mcpServers": {
        "dodopayments_api": {
          "command": "npx",
          "args": ["-y", "mcp-remote@latest", "https://mcp.dodopayments.com/sse"]
        }
      }
    }
    ```

</Step>
<Step title="인증 및 환경 설정 완료하기">
    JSON 구성을 추가한 후, 팝업이 나타나며:
    - Dodo Payments API 키 입력
    - 환경 선택 (테스트 또는 라이브)
    - 구성 옵션에서 MCP 클라이언트 선택
</Step>
<Step title="로그인 및 승인">
    - 로그인 및 승인 버튼 클릭
</Step>
<Step title="성공적인 연결 확인하기">
    설정이 완료되면 성공 화면이 표시됩니다.
    <Check>
    이제 Dodo Payments MCP 서버를 사용할 준비가 되었습니다!
    </Check>
</Step>
</Steps>
