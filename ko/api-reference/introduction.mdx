## API 키 관리 및 인증

<Steps>
<Step title="API 키 접근">
대시보드에서 **개발자 → API 키**로 이동하여 키를 관리하세요.
</Step>

<Step title="새 키 생성">
**API 키 추가**를 선택하고 설명적인 이름을 제공한 후 쓰기 권한을 구성하세요:

- **쓰기 권한 활성화** (체크됨): 모든 API 작업에 대한 전체 읽기 및 쓰기 권한
- **쓰기 권한 활성화** (체크 해제됨): 읽기 전용 액세스 - 데이터(결제, 구독, 고객, 제품)만 가져올 수 있습니다. 리소스를 생성하거나 수정할 수 없습니다.

<Tip>
대시보드 통합, 분석 도구 및 변경 없이 데이터 보기만 필요한 시스템의 경우 "쓰기 권한 활성화"의 체크를 해제하세요.
</Tip>
</Step>

<Step title="키를 안전하게 저장">
생성된 키를 즉시 복사하고 안전하게 저장하세요.
</Step>

<Step title="API 요청 인증">
API 키를 사용하여 모든 요청을 인증하세요. 다음 인증 형식을 적용하세요:

```bash 
Authorization: Bearer YOUR_API_KEY
```

<Warning>
클라이언트 측 코드나 공개 리포지토리에서 비밀 API 키를 노출하지 마세요.
</Warning>
</Step>
</Steps>

## 응답 형식

<CodeGroup>

```json Success Response
{
  "id": "pay_1234567890",
  "status": "completed",
  "amount": 2999,
  "currency": "USD",
  "created_at": "2024-01-15T10:30:00Z"
}
```

```json Error Response
{
  "code": "INVALID_REQUEST",
  "message": "The request contains invalid parameters"
}
```

</CodeGroup>

## 비율 제한

- **표준**: 비즈니스당 분당 1000 요청
- **버스트**: 초당 최대 100 요청

<Note>
사용량을 추적하기 위해 `X-RateLimit-Remaining` 헤더를 모니터링하세요.
</Note>
