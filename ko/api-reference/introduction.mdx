## API 키 관리 및 인증

<Steps>
{/* LOCKED_PATTERN_5683f8e476818682d5c5b829d3d48046 */}
대시보드에서 **Developer → API Keys**로 이동하여 키를 관리하세요.
</Step>

{/* LOCKED_PATTERN_37bf6117c798d1587b84be4347b7e5d8 */}
**Add API Key**를 선택하고 설명이 포함된 이름을 지정한 후 쓰기 접근 권한을 구성하세요:

- **쓰기 권한 활성화** (체크됨): 모든 API 작업에 대한 전체 읽기 및 쓰기 권한
- **쓰기 권한 활성화** (체크 해제됨): 읽기 전용 액세스 - 데이터(결제, 구독, 고객, 제품)만 가져올 수 있습니다. 리소스를 생성하거나 수정할 수 없습니다.

<Tip>
“Enable write access”의 선택을 해제하여 대시보드 통합, 분석 도구 및 변경 없이 데이터 조회만 필요한 시스템에서는 쓰기 권한을 제한하세요.
</Tip>
</Step>

{/* LOCKED_PATTERN_ec093b44892cffcb052c9864b71296b7 */}
생성된 키를 즉시 복사하고 안전하게 저장하세요.
</Step>

{/* LOCKED_PATTERN_c2cecd39269efebd7d9e1bb8b64c1db2 */}
모든 요청에 API 키를 사용하여 인증하세요. 다음 인증 형식을 적용하세요:

```bash 
Authorization: Bearer YOUR_API_KEY
```

<Warning>
비밀 API 키를 클라이언트 측 코드나 공개 저장소에 노출하지 마세요.

## 비율 제한

당사의 API는 돌발 보호 기능이 적용된 이중 창 대역폭 제한 시스템을 사용합니다. 제한은 인증 방법과 비즈니스 등급에 따라 적용됩니다.

### 기본 제한 (Tier 0)

| 창 | 제한 |
|--------|-------|
| 초당 (버스트) | 40 요청 |
| 분당 (지속) | 240 요청 |

### 비즈니스 계층

API 사용량이 많은 비즈니스를 위해 더 높은 비율 제한이 제공됩니다:

| 계층 | 버스트 (초당) | 지속 (분당) |
|------|-------------------|------------------------|
| 계층 0 (기본) | 40 | 240 |
| 계층 1 | 100 | 1,000 |
| 계층 2 | 500 | 5,000 |

<Tip>
상위 비율 제한 계층으로 업그레이드하려면 지원팀에 문의하세요.
</Tip>

### 인증되지 않은 요청

유효한 인증 헤더가 없는 요청은 IP 주소별로 비율 제한이 적용됩니다:

| 창 | 제한 |
|--------|-------|
| 초당 (버스트) | 20 요청 |
| 분당 (지속) | 100 요청 |

### 비율 제한 헤더

다음 응답 헤더를 통해 사용량을 모니터링하세요:

- `X-RateLimit-Limit` - 허용된 최대 요청 수
- `X-RateLimit-Remaining` - 현재 창에서 남은 요청 수
- `X-RateLimit-Reset` - 비율 제한이 재설정되는 시간

<Note>
비율 제한을 초과하면 API는 `429 Too Many Requests` 응답을 반환합니다. 재시도 로직에 지수 백오프를 구현하세요.
</Note>
