### 환경 의존성

**환경 의존:** 모든 환경 의존 기능은 테스트 모드와 라이브 모드에서 독립적인 데이터를 가집니다. 한 환경에서 변경된 사항은 다른 환경에 영향을 미치지 않습니다. 예를 들어, 테스트 모드에서 생성된 API 키는 라이브 모드에서 생성된 API 키와 독립적입니다.

**환경 독립:** 모든 환경 독립 기능은 라이브 모드와 테스트 모드에서 동일한 데이터를 가집니다. 예를 들어, 내 계정 섹션은 테스트 모드와 라이브 모드에서 동일한 데이터를 가집니다. 한 환경에서 변경된 사항은 다른 환경에 영향을 미칩니다.

라이브 모드는 신원 확인 및 비즈니스 확인이 성공적으로 완료된 후에만 사용할 수 있습니다. 실제 결제를 수락할 수 있는 것은 \`live mode\`입니다.

테스트 모드와 라이브 모드의 API 호스트 이름은 아래와 같습니다:
- 테스트 모드: [`https://test.dodopayments.com`](https://test.dodopayments.com/)
- 라이브 모드: [`https://live.dodopayments.com`](https://live.dodopayments.com/)

## 주요 하이라이트

1. 테스트 모드 또는 라이브 모드에서 생성된 모든 제품은 해당 환경에서만 사용할 수 있습니다. 테스트 모드 환경에서 라이브 모드로 복사하는 옵션이 있습니다. 이는 모든 관련 세부정보와 함께 제품을 복사하여 라이브 모드에서 동일하게 생성합니다.

2. 테스트 모드에서 생성된 API 키와 웹훅은 테스트 모드 환경에만 제한되며 라이브 모드 배포에 영향을 미치지 않습니다. 그 반대도 마찬가지입니다.

3. 검증을 위해 제출된 모든 문서와 제출된 은행 계좌 정보는 테스트 모드와 라이브 모드에서 동일합니다. 어느 환경에서든 이 정보의 변경은 다른 환경에도 영향을 미칩니다.

## 테스트 카드 및 UPI ID

테스트 카드, UPI ID 및 기타 테스트 방법에 대한 자세한 정보는 포괄적인 [테스트 프로세스](/miscellaneous/testing-process) 가이드를 참조하세요.

이 가이드에는 다음이 포함됩니다:
- 다양한 시나리오에 대한 테스트 카드 번호
- 결제를 시뮬레이션하기 위한 테스트 UPI ID
- 다양한 결제 결과를 테스트하기 위한 지침
- 통합 테스트를 위한 모범 사례

<Note>
테스트 결제 방법은 테스트 모드에서만 유효합니다. 실제 결제에 사용하지 마세요.
</Note>
