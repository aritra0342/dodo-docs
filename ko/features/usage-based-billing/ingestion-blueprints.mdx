수집 청사진은 일반적인 시나리오에서 사용량을 추적하기 위한 미리 구축된 패턴입니다. 이벤트 구조와 미터 구성을 처음부터 알아내는 대신, 검증된 청사진으로 시작하여 필요에 맞게 사용자 정의하세요.

## 청사진이란 무엇인가요?

각 청사진에는 다음이 포함됩니다:
- **이벤트 구조 권장 사항**: 최적의 이벤트 명명 및 메타데이터 설계
- **미터 구성**: 제안된 집계 유형 및 필터
- **샘플 코드**: 여러 언어로 된 작동 구현 예제
- **모범 사례**: 정확성, 성능 및 비용 최적화를 위한 팁

<Tip>
청사진은 설정에 드는 시간을 절약하고 일반적인 실수를 피하는 데 도움이 됩니다. 청사진으로 시작한 다음 특정 요구 사항에 맞게 조정하세요.
</Tip>

## 사용 가능한 청사진

귀하의 사용 사례에 맞는 청사진을 선택하세요:

<CardGroup cols={1}>
<Card title="LLM Blueprint" icon="brain-circuit" href="/developer-resources/ingestion-blueprints/llm">
OpenAI, Anthropic, Groq, Gemini, Vercel AI SDK 등에서 AI 토큰 사용량을 추적하세요.

**완벽한 용도**: AI 챗봇, 코드 생성기, 콘텐츠 생성 도구, AI 기반 앱.

**추적 항목**: 입력 토큰, 출력 토큰, 모델 사용량, 요청 수.
</Card>

<Card title="API Gateway Blueprint" icon="cloud" href="/developer-resources/ingestion-blueprints/api-gateway">
게이트웨이 수준에서 엔드포인트 필터링 및 속도 제한 지원으로 API 요청을 측정하세요.

**완벽한 용도**: REST API, GraphQL 서비스, 웹훅 제공자, API 우선 플랫폼.

**추적 항목**: 요청 수, 엔드포인트 사용량, HTTP 메서드, 응답 시간.
</Card>

<Card title="Object Storage Blueprint" icon="box-archive" href="/developer-resources/ingestion-blueprints/object-storage">
클라우드 스토리지 서비스의 파일 업로드, 저장 용량, 대역폭을 추적하세요.

**완벽한 용도**: 파일 호스팅, 미디어 플랫폼, 백업 서비스, 문서 관리.

**추적 항목**: 저장된 바이트, 파일 업로드, 다운로드, 스토리지 스냅샷.
</Card>

<Card title="Stream Blueprint" icon="tower-broadcast" href="/developer-resources/ingestion-blueprints/stream">
비디오, 오디오, 실시간 데이터 전송을 위한 스트리밍 대역폭을 측정하세요.

**완벽한 용도**: 비디오 스트리밍, 오디오 플랫폼, 라이브 방송, 실시간 데이터 피드.

**추적 항목**: 소비된 대역폭, 스트림 지속 시간, 품질 수준, 동시 스트림 수.
</Card>

<Card title="Time Range Blueprint" icon="clock" href="/developer-resources/ingestion-blueprints/time-range">
컴퓨트 리소스, 서버리스 함수, 장시간 실행 작업에 대해 경과 시간 기준으로 청구하세요.

**완벽한 용도**: 서버리스 플랫폼, 컨테이너 호스팅, 컴퓨팅 인스턴스, 배치 처리.

**추적 항목**: 실행 시간, 컴퓨팅 사용 시간, 함수 호출, 작업 기간.
</Card>
</CardGroup>

## 청사진 사용 방법

<Steps>
<Step title="Choose your blueprint">
청구 시나리오와 가장 잘 맞는 청사진을 선택하세요. 각 청사진 페이지에는 자세한 구현 가이드가 포함되어 있습니다.
</Step>

<Step title="Review the event structure">
권장되는 이벤트 명명 규칙과 메타데이터 설계를 이해하세요. 사용 사례에 맞게 이벤트 구조를 복사하세요.
</Step>

<Step title="Create the meter">
청사진의 미터 구성 지침을 따르세요. 권장되는 집계 유형과 필터를 사용하세요.
</Step>

<Step title="Implement event sending">
언어/프레임워크에 맞는 샘플 코드를 복사하세요. 자리 표시자 값을 실제 API 키와 고객 ID로 교체하세요.
</Step>

<Step title="Test and verify">
테스트 이벤트를 전송하고 올바른 집계로 미터 대시보드에 표시되는지 확인하세요.
</Step>

<Step title="Customize as needed">
청사진을 특정 요구 사항에 맞게 조정하세요. 사용자 정의 메타데이터를 추가하거나 필터를 조정하거나 가격을 수정하세요.
</Step>
</Steps>

## 청사진 비교

어떤 청사진이 귀하의 요구에 맞는지 확실하지 않으신가요? 간단한 비교입니다:

| 사용 사례 | 청사진 | 집계 | 일반 메타데이터 |
|----------|-----------|-------------|-----------------|
| AI 채팅 앱 | LLM | 합계 (토큰) | 모델, 입력_토큰, 출력_토큰 |
| REST API | API 게이트웨이 | 수 | 엔드포인트, 메서드, 상태_코드 |
| 파일 저장소 | 객체 저장소 | 합계 (바이트) | 파일_크기, 파일_유형, 작업 |
| 비디오 플랫폼 | 스트림 | 합계 (대역폭) | 지속 시간, 품질, 비트 전송률 |
| 서버리스 | 시간 범위 | 합계 (지속 시간) | 함수_이름, 실행_시간_ms |

## 사용자 정의 구현

정확한 사용 사례에 대한 청사진이 없으신가요? 문제 없습니다:

1. **가장 가까운 청사진으로 시작**: 귀하의 필요에 가장 유사한 것을 선택하세요.
2. **이벤트 구조 조정**: 메타데이터 필드를 귀하의 요구 사항에 맞게 수정하세요.
3. **미터 구성 조정**: 필요에 따라 집계 유형이나 필터를 변경하세요.
4. **철저히 테스트**: 샘플 사용량으로 청구 계산을 검증하세요.

<Info>
모든 청사진은 출발점입니다. 서로 다른 청사진의 개념을 혼합하거나 완전히 맞춤형 구현을 만들어도 좋습니다.
</Info>
