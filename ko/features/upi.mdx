## 제품 생성

Dodo Payments의 상인은 제품을 생성할 때 두 가지 옵션이 있습니다:

1. **INR 기반 제품**
    - 주로 인도 고객에게 판매하는 경우 제품 기본 가격을 INR로 설정할 수 있습니다.
    - 체크아웃 시 최종 금액은 INR로 직접 청구되며, 통화 변환이 필요하지 않습니다.
2. **USD 기반 제품**
    - 글로벌 고객을 대상으로 하는 경우 제품 기본 가격을 USD로 설정할 수 있습니다.
    - 상인이 인도에 있고 고객의 청구 국가도 인도인 경우, 시스템은 USD 가격을 **실시간으로** INR로 변환하여 고객에게 INR로 표시합니다.

## UPI 체크아웃 흐름

1. **고객이 UPI 선택**
    - 체크아웃 페이지에서 고객은 결제 수단으로 **UPI**를 선택합니다.
2. **UPI 세부정보**
    - 고객은 체크아웃 페이지에 **UPI ID**를 입력하거나 **UPI QR** 코드를 스캔하도록 선택합니다.
3. **결제 승인**
    - 고객은 선택한 UPI 앱(예: Google Pay, PhonePe, Paytm, BHIM)에서 거래를 승인합니다.
    - 고객은 앱 내에서 요청을 **승인**하거나 **거부**합니다.
4. **거래 확인**
    - 승인되면 결제가 실시간으로 처리됩니다.
    - 고객은 확인 세부정보가 포함된 성공 화면으로 리디렉션되거나 표시됩니다.
5. **주문 최종화**
    - DodoPayments는 거래 상태를 확인하는 **콜백/웹훅**을 귀하의 시스템에 전송합니다.

## 결제 처리

- **로컬 INR 처리**:
    - 모든 INR 거래(USD에서 INR로 변환된 거래 포함)는 대시보드와 송장에 INR로 표시됩니다.
- **직접 은행 세부정보 없음**:
    - UPI는 은행 간 결제 시스템이므로 고객은 귀하의 사이트에서 민감한 은행 정보를 공유할 필요가 없습니다. 고객은 단순히 UPI 앱을 통해 결제를 진행합니다.

## UPI 결제 테스트

원활한 출시를 보장하기 위해 상인은 **Dodo Payments 테스트 모드**에서 UPI 거래를 테스트할 수 있습니다:

1. **테스트 모드 선택**
    - Dodo Payments 대시보드에서 왼쪽 하단 모서리에 있는 테스트 모드 버튼을 활성화합니다.
2. **테스트 제품의 결제 링크 복사**
    - 테스트 제품을 생성하고 해당 제품의 공유 버튼을 클릭하여 결제 링크를 복사합니다.
    - 새 브라우저에 링크를 붙여넣습니다.
    - 청구 국가를 인도로 선택하여 UPI 결제 옵션을 확인합니다.
3. **UPI 테스트**
    - 체크아웃 페이지는 테스트 결제 방법 섹션에서 테스트 UPI ID를 제공하며 UPI QR 코드를 스캔할 수 있는 옵션도 제공합니다.
    - 테스트 환경에서 결제를 승인하거나 거부하면 실제 운영 환경에서의 흐름을 시뮬레이션합니다.
    - UPI 테스트 결제는 거래가 처리되는 데 보통 1-2분이 소요됩니다. 이 기간 동안 인내해 주시기 바랍니다.
4. **콜백 검증**
    - 귀하의 시스템이 **웹훅 알림**(성공/실패)을 제대로 수신하고 처리하는지 확인합니다.
    - INR 금액(및 해당되는 경우 USD 가격)이 올바르게 표시되는지 확인합니다.

## 대시보드 및 송장

**거래 및 원장**

- 각 거래 페이지는 INR(UPI)의 **최종 청구 금액**을 표시합니다.

**송장**

- 고객이 INR로 UPI를 통해 구매를 완료할 때마다 송장은 금액과 해당 세금을 INR로만 표시합니다.

Dodo Payments와 **UPI**를 통합함으로써 인도 상인은 INR로 가격이 책정된 제품이든 USD에서 변환된 제품이든 관계없이 지역 고객에게 매우 인기 있고 편리한 결제 옵션을 제공할 수 있습니다.
