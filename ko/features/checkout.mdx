## 적응형 통화

적응형 통화는 고객이 선호하는 지역 통화로 결제할 수 있도록 하여 신뢰와 전환율을 향상시킵니다.

### 작동 방식

1. **활성화**: 비즈니스 설정에서 적응형 통화를 활성화합니다.
2. **선택**: 고객은 체크아웃에서 직접 통화를 전환할 수 있습니다.
3. **변환**: 가격은 실시간 FX 환율을 사용하여 동적으로 변환됩니다.
4. **표시**: 최종 지불 금액이 결제 전에 투명하게 표시됩니다.

## 자동 세금 계산

세금은 고객의 청구 위치에 따라 자동으로 계산되어 GST, VAT 및 판매세 요구 사항을 준수합니다. 수동 설정이 필요하지 않습니다.

### 세금 계산 작동 방식

<Steps>
<Step title="위치 감지">
  세금 규칙은 고객의 국가(해당되는 경우 지역)에 따라 적용됩니다.
</Step>

<Step title="동적 업데이트">
  세금 금액은 다음과 같은 경우 자동으로 업데이트됩니다:
  - 국가 변경
  - 주소 업데이트
</Step>

<Step title="투명한 표시">
  최종 세금 내역이 결제 전에 명확하게 표시됩니다.
</Step>
</Steps>

<Tip>
세금 계산은 완전히 자동화되어 있습니다. 표준 디지털 상품 및 SaaS 제품에 대해 수동 구성이 필요하지 않습니다.
</Tip>

## 사업자 세금 ID 지원

등록된 사업자의 경우 체크아웃에서 고객이 사업자 세금 ID(예: VAT/GST 번호)를 입력할 수 있습니다.

### 세금 ID가 입력되면 발생하는 일

- 세금 자격이 실시간으로 검증됩니다.
- 적용 가능한 세금 면제 또는 역세금 규칙이 적용됩니다.
- 체크아웃에서 세금 금액이 즉시 업데이트됩니다.

## 할인 코드

고객은 대시보드에서 생성한 할인 또는 프로모션 코드를 체크아웃 페이지에서 직접 적용할 수 있습니다.

### 체크아웃 경험

1. 고객이 할인 코드를 입력합니다.
2. 할인 코드가 즉시 검증됩니다.
3. 업데이트된 가격과 절감액이 명확하게 표시됩니다.

<Frame>
  <img src="/images/checkout/c4.png" alt="체크아웃에서의 할인 코드 입력" style={{ width: '70%' }}/>
</Frame>

### API 통합

할인 코드를 미리 적용하거나 할인 입력 필드를 활성화합니다:

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    { product_id: 'prod_abc', quantity: 1 }
  ],
  discount_code: 'WELCOME20', // Pre-apply a code
  feature_flags: {
    allow_discount_code: true // Show discount input field
  },
  return_url: 'https://yoursite.com/return'
});
```

## 스마트 주소 수집

체크아웃은 빠른 완료를 위한 유연한 주소 입력을 지원합니다.

### 사용 가능한 옵션

| 옵션 | 설명 |
|--------|-------------|
| **구글 주소 자동 완성** | 자동 완성으로 빠른 선택 |
| **수동 입력** | 전체 주소에 대한 완전한 제어 |
| **국가 선택** | 세금 및 준수 논리를 유도합니다 |

<Tip>
주소 수집은 속도, 정확성 및 글로벌 범위를 균형 있게 조정하여 전환율을 극대화하면서 준수를 보장합니다.
</Tip>

## 개인정보 보호정책 및 약관 수락

법적 및 준수 투명성을 보장하기 위해:

- [개인정보 보호정책](https://dodopayments.com/privacy-policy) 및 [구매자 약관](https://dodopayments.com/buyer-terms) 링크가 체크아웃에서 명확하게 표시됩니다.
- 고객은 결제를 완료하기 전에 이를 명시적으로 인정합니다.

<Info>
이는 GDPR 준수를 포함한 글로벌 소비자 보호 및 데이터 개인정보 보호 요구 사항을 충족하는 데 도움이 됩니다.
</Info>

## 컬렉션 체크아웃

제품 컬렉션은 고객이 단일 체크아웃에서 여러 관련 제품(예: 스타터, 프로, 엔터프라이즈 플랜)을 보고 선택할 수 있는 통합 체크아웃 경험을 제공합니다.

### 작동 원리

1. **모든 제품 표시**: 고객은 컬렉션의 모든 활성 제품을 봅니다
2. **첫 번째 제품 자동 선택**: 컬렉션의 첫 번째 제품이 자동으로 선택됩니다
3. **옵션 비교**: 고객은 선택하기 전에 가격 및 기능을 비교할 수 있습니다
4. **단일 선택**: 제품을 선택한 후, 체크아웃은 표준 결제 흐름으로 진행됩니다

### 컬렉션 체크아웃 생성하기

```typescript
const session = await client.checkoutSessions.create({
  product_collection_id: 'pdc_abc123',
  product_cart: [], // Required: pass an empty array for collection checkout
  return_url: 'https://yoursite.com/return'
});
```

<Warning>
`product_collection_id`를 사용할 때는, 빈 `product_cart` 배열을 전달해야 합니다. 할인 코드는 세션 생성 시 미리 적용할 수 없습니다.
</Warning>

## 체크아웃 세션 구성

체크아웃 행동을 Checkout Sessions API를 사용하여 제어합니다:

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    { product_id: 'prod_abc', quantity: 1 }
  ],
  customer: {
    email: 'customer@example.com',
    name: 'Jane Doe'
  },
  billing_currency: 'EUR', // Set specific currency
  discount_code: 'PROMO10',
  feature_flags: {
    allow_discount_code: true
  },
  return_url: 'https://yoursite.com/return',
  metadata: {
    order_ref: 'ORD-12345'
  }
});
```
