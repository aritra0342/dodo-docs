Dodo Payments는 전 세계 고객에게 서비스를 제공하기 위해 다양한 결제 방법을 지원합니다. 나열된 모든 결제 방법은 상인 계정이 확인되고 라이브 모드가 활성화되면 자동으로 사용 가능해집니다.

<Info>
**자동 활성화:** 귀하의 추가 구성은 필요하지 않습니다. 모든 결제 방법은 Dodo Payments에 의해 사전 구성되고 관리됩니다. 아래 설명된 API 구성을 사용하여 체크아웃에서 표시되는 방법을 선택적으로 제한할 수 있습니다.
</Info>

## 지원되는 결제 방법

다음 표는 사용 가능한 모든 결제 방법, 지역별 가용성 및 주요 제한 사항에 대한 빠른 참조를 제공합니다:

| 결제 방법               | 지원 국가           | 지원 통화           | 구독 지원           |
| :----------------------- | :------------------- | :------------------- | :------------------- |
| **신용 카드**           | 전 세계             | 모든 통화           | 예                  |
| **직불 카드**           | 전 세계             | 모든 통화           | 예                  |
| **Apple Pay**            | 전 세계 (인도 제외) | 모든 통화           | 예                  |
| **Google Pay**           | 전 세계 (인도 제외) | 모든 통화           | 예                  |
| **Amazon Pay**           | 전 세계 (인도 제외) | USD                  | 아니오               |
| **Cash App Pay**         | 미국                | USD                  | 아니오               |
| **Afterpay (BNPL)**      | 미국, 영국          | USD, GBP             | 아니오               |
| **Klarna (BNPL)**        | 미국, 유럽*         | USD, EUR, GBP, DKK, NOK, SEK, CZK, RON, PLN, CHF | 아니오               |
| **PayPal**               | 전 세계             | EUR, GBP             | 아니오               |
| **UPI**                  | 인도                | INR                  | 예*                 |
| **Rupay (신용/직불)**    | 인도                | INR                  | 예*                 |
| **Bancontact**           | 벨기에              | EUR                  | 아니오               |
| **EPS**                  | 오스트리아          | EUR                  | 아니오               |
| **iDEAL**                | 네덜란드            | EUR                  | 아니오               |
| **Multibanco**           | 포르투갈            | EUR                  | 아니오               |

<Warning>
**PayPal 지원 일시 중단:** PayPal 결제 방법 지원이 일시적으로 중단되었습니다. 현재 PayPal은 새로운 거래에 사용할 수 없습니다. PayPal 지원이 복원되면 이 페이지를 업데이트하겠습니다.
</Warning>

<Note>
각 거래는 <strong>각 통화에 대한 최소 금액</strong>을 충족해야 합니다. USD의 경우 최소 금액은 <strong>0.50 USD</strong>입니다. 다른 통화의 최소 금액은 지역에 따라 다릅니다.
[지원되는 통화 및 최소 금액 보기](/features/adaptive-currency#supported-currencies)
</Note>

<Note>
Klarna와 Afterpay는 결제 시 자격을 갖추기 위해 최소 거래 금액 <strong>50.01 USD</strong> (또는 지원되는 통화의 동등한 금액)를 요구합니다. 이 기준 이하의 거래는 이러한 BNPL 결제 옵션에 대해 자격이 없습니다. 

Klarna의 지원 국가에 대한 자세한 내용은 아래의 <a href="#buy-now-pay-later-bnpl">BNPL 섹션</a>을 참조하십시오.
</Note>

<Note>
<strong>인도 결제 방법:</strong> UPI 및 Rupay 구독은 RBI 준수 의무와 함께 지원됩니다. 의무 한도 및 처리 지연에 대한 중요한 세부정보는 <a href="/features/subscription#subscriptions-with-rbi-compliant-mandates">RBI 준수 의무가 있는 구독</a> 섹션을 참조하십시오.
</Note>

  - **Cartes Bancaires** – 프랑스 전역에서 널리 사용됨
  - **Diners Club** – 독점 혜택이 있는 프리미엄 국제 카드
  - **한국 로컬 카드** – 한국 국내 카드 네트워크 지원
</Accordion>

<Accordion title="디지털 지갑">

  - **Apple Pay** – iOS, macOS 및 watchOS에서 쉽게 결제
  - **Google Pay** – Android, 웹 및 매장에서 안전한 결제
  - **PayPal** – EUR 및 GBP로 글로벌 결제 (구독에는 사용 불가). <strong>참고: PayPal 지원은 현재 중단되었습니다.</strong>
  - **Cash App Pay** – 미국 고객을 위한 모바일 결제 (구독에는 사용 불가)
  - **Amazon Pay** – 간편한 결제를 위한 Amazon 자격 증명 사용 (구독에는 사용 불가)
</Accordion>

<Accordion title="지금 구매하고 나중에 결제하기 (BNPL)">
  유연한 할부 옵션을 제공하여 전환율을 높입니다:

  - **Klarna** – Klarna의 옵션으로 유연한 결제를 가능하게 합니다: 4회 할부로 결제, 30일 후 결제 또는 구매 자금 조달. Klarna는 미국 및 오스트리아, 벨기에, 체코, 덴마크, 핀란드, 프랑스, 독일, 그리스, 아일랜드, 이탈리아, 네덜란드, 노르웨이, 폴란드, 포르투갈, 루마니아, 스페인, 스웨덴, 스위스, 영국을 포함한 19개 유럽 국가에서 사용할 수 있습니다. 지원되는 통화에는 USD, EUR, GBP, DKK, NOK, SEK, CZK, RON, PLN 및 CHF가 포함됩니다.

</Accordion>

## 결제 방법 구성

모든 결제 방법은 고객의 위치와 통화에 따라 자동으로 사용 가능하지만, 결제 시 어떤 결제 옵션이 표시될지를 선택적으로 제한할 수 있습니다. `allowed_payment_method_types` 매개변수를 사용하십시오.

### 결제 방법을 구성해야 하는 시점

다음과 같은 이유로 허용된 결제 방법을 구성할 수 있습니다:

- 비즈니스 요구 사항에 따라 고객에게 표시되는 결제 옵션을 제어하기 위해
- 특정 제품 유형 또는 거래 흐름에 대한 결제 방법을 제한하기 위해
- 다양한 결제 경험 간의 일관성을 유지하기 위해
- 목표 고객을 위한 체크아웃 경험을 최적화하기 위해

### 구현

<Steps>
<Step title="API 요청에 매개변수 추가">
결제 또는 구독 생성 요청에 `allowed_payment_method_types` 필드를 포함하십시오:

```json
{
  "allowed_payment_method_types": [
    "credit",
    "debit",
    "apple_pay"
  ]
}
```

</Step>

<Step title="구성 테스트">
지정된 결제 방법만 체크아웃에 표시되는지 확인하십시오.

<Tip>
구성된 대로 작동하는지 확인하기 위해 다양한 고객 위치와 통화로 결제 방법 가용성을 테스트하십시오.
</Tip>
</Step>
</Steps>

### 모범 사례

<AccordionGroup>
<Accordion title="기본 동작">
`allowed_payment_method_types`가 지정되지 않은 경우, 모든 적격 결제 방법은 고객의 위치, 통화 및 거래 유형에 따라 사용 가능합니다. 결제 방법을 제한해야 할 때만 이 매개변수를 사용하십시오.
</Accordion>

<Accordion title="지역 고려 사항">
결제 방법을 선택할 때 목표 시장을 고려하십시오. 지역별 옵션(예: 인도의 UPI 또는 네덜란드의 iDEAL)을 포함하면 해당 시장에서 전환율을 크게 향상시킬 수 있습니다.
</Accordion>

<Accordion title="모니터링 및 최적화">
고객이 가장 자주 사용하는 결제 방법을 추적하고 실제 사용 패턴 및 전환 데이터를 기반으로 허용된 방법을 조정하십시오.
</Accordion>
</AccordionGroup>

### 중요한 고려 사항

결제 방법의 가용성은 여러 요인에 따라 달라집니다:
- **고객 위치**: 일부 결제 방법은 지역별입니다.
- **상인 설정**: 귀하의 계정 구성 및 기능
- **통화**: 지원되는 결제 방법은 통화에 따라 다릅니다.
- **거래 유형**: 일부 방법은 구독을 지원하지 않거나 최소 금액이 있습니다.

<Note>
`allowed_payment_method_types`에 결제 방법을 추가한다고 해서 반드시 표시되는 것은 아닙니다. 이러한 요인에 따라 자격이 있어야 합니다.
</Note>

## 문제 해결

<AccordionGroup>
<Accordion title="체크아웃에서 결제 방법이 나타나지 않음">
**가능한 원인:**
- 결제 방법이 `allowed_payment_method_types`에 포함되지 않음
- 결제 방법이 고객의 위치에서 지원되지 않음
- 결제 방법이 거래 통화를 지원하지 않음
- 구독의 경우: 결제 방법이 반복 결제를 지원하지 않음
- 거래 금액이 최소 기준 이하임 (예: BNPL 방법)

**해결책:** 모든 자격 요건이 충족되는지 확인하고 API 요청 구성을 확인하십시오.
</Accordion>

<Accordion title="예상치 못한 결제 방법이 나타남">
**가능한 원인:**
- `allowed_payment_method_types` 매개변수가 올바르게 전달되지 않음
- 상인 대시보드에 충돌하는 구성이 있을 수 있음

**해결책:** API 요청 페이로드를 검토하고 매개변수가 문자열 배열로 올바르게 형식화되었는지 확인하십시오.
</Accordion>

<Accordion title="지역 결제 방법이 사용 불가">
**가능한 원인:**
- 고객의 위치가 결제 방법의 지원 지역과 일치하지 않음
- 통화가 지역 결제 방법에서 지원되지 않음
- 귀하의 상인 계정이 해당 특정 결제 방법에 대해 활성화되지 않았을 수 있음

**해결책:** 지역 및 통화 호환성을 확인하고, 해당 방법이 사용 가능해야 한다고 생각되면 지원팀에 문의하십시오.
</Accordion>

<Accordion title="인도 결제 방법 구독 문제">
**업그레이드/다운그레이드 후 구독 요금 청구 실패:**
- 새로운 청구 금액이 기존 의무 한도(15,000 루피 기준)를 초과할 수 있음
- 고객이 새로운 의무를 설정하기 위해 결제 방법을 업데이트해야 할 수 있음

**해결책:** 고객에게 결제 방법을 업데이트하도록 요청하거나 의무 한도 내에서 구독 금액을 조정하도록 하십시오.

**청구 시작 후 구독 보류:**
- 고객이 48시간 처리 기간 동안 의무를 취소했을 수 있음
- 고객이 고액 청구(≥15,000 루피)를 승인하지 못했을 수 있음

**해결책:** 의무 취소 및 결제 실패에 대한 웹훅 이벤트를 모니터링하십시오. 재시도 로직을 구현하거나 고객에게 결제 방법을 재설정하도록 요청하십시오.

**결제 공제 지연:**
- 정상 동작: 인도 카드 및 UPI 구독은 청구 시작과 실제 공제 사이에 48시간 지연이 있습니다.
- 이 지연은 은행 API에 따라 2-3시간 더 연장될 수 있습니다.

**해결책:** 이는 예상되는 동작입니다. 결제 확인이 웹훅을 통해 수신될 때까지 혜택이나 크레딧을 활성화하지 마십시오.
</Accordion>

<Accordion title="구독을 위한 결제 방법 업데이트">
**결제 방법을 업데이트해야 하는 시점:**
- 고객의 카드가 만료되거나 교체됨
- 고객이 다른 결제 방법을 사용하고 싶어함
- 결제가 실패하여 구독이 `on_hold` 상태에 있음
- 어떤 이유로든 결제 방법을 변경해야 함

**활성 구독의 경우:**
활성 구독의 결제 방법을 업데이트하면 향후 갱신을 위한 결제 방법이 업데이트됩니다. 즉각적인 청구가 생성되지 않습니다.

**보류 중인 구독의 경우:**
구독이 `on_hold` 상태에 있을 때(갱신 실패 또는 요금 변경 청구 실패로 인해), 결제 방법을 업데이트하면:
- 남은 요금에 대한 청구가 자동으로 생성됩니다.
- 청구에 대한 송장이 생성됩니다.
- 새로운 결제 방법을 사용하여 결제가 처리됩니다.
- 성공적인 결제 후 구독이 `active` 상태로 재활성화됩니다.

<CodeGroup>

```javascript Node.js
// Update payment method for active subscription
await client.subscriptions.updatePaymentMethod('sub_123', {
  type: 'new',
  return_url: 'https://example.com/return'
});

// Or use existing payment method
await client.subscriptions.updatePaymentMethod('sub_123', {
  type: 'existing',
  payment_method_id: 'pm_abc123'
});

// For on_hold subscription - automatically creates charge
const response = await client.subscriptions.updatePaymentMethod('sub_123', {

  return_url: 'https://example.com/return'
});

if (response.payment_id) {
  // Charge created for remaining dues
  // Redirect customer to response.payment_link
  // Monitor webhooks: payment.succeeded → subscription.active
}
```
