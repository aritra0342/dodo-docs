## 작동 방식

결제 수단은 다음에 따라 고객에게 자동으로 보여집니다:

- **고객 위치** — iDEAL(네덜란드) 또는 UPI(인도)와 같은 지역 방법은 해당 지역의 고객에게만 나타납니다.
- **거래 통화** — 지원되는 통화에 따라 결제 수단이 필터링됩니다.
- **장치 유형** — Apple Pay는 호환 가능한 Apple 장치와 브라우저에서만 나타납니다.
- **거래 유형** — 일부 방법은 구독을 지원하지 않거나 최소 금액이 있습니다.

<Info>
**자동 활성화:** 모든 결제 수단은 Dodo Payments에서 사전 구성 및 관리됩니다. 추가 설정 없이 계정이 인증되고 실시간 모드가 활성화되면 이용할 수 있습니다.
</Info>

## 결제 수단 구성

기본적으로 모든 적격 결제 수단이 자동으로 표시됩니다. 표시할 수단을 제한하려면 `allowed_payment_method_types`을 사용하세요:

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://test.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

### 사용 가능한 방법 유형

| 유형 | 설명 |
| :--- | :---------- |
| `credit` | 신용카드(Visa, Mastercard, Amex 등) |
| `debit` | 직불카드 |
| `apple_pay` | Apple Pay |
| `google_pay` | Google Pay |
| `amazon_pay` | Amazon Pay |
| `cashapp` | Cash App Pay |
| `klarna` | Klarna BNPL |
| `afterpay_clearpay` | Afterpay / Clearpay |
| `upi_collect` | UPI Collect(QR 코드) |
| `ideal` | iDEAL(네덜란드) |
| `bancontact_card` | Bancontact(벨기에) |
| `eps` | EPS(오스트리아) |
| `multibanco` | Multibanco(포르투갈) |
| `revolut_pay` | RevolutPay |
| `billie` | Billie B2B BNPL |

<Warning>
항상 `credit` 및 `debit`을 대체 수단으로 포함하세요. 지정한 모든 결제 수단이 고객에게 제공되지 않으면 체크아웃이 실패합니다.
</Warning>

## 문제 해결

<AccordionGroup>
{/* LOCKED_PATTERN_fbf2d38e7725fb97384bef01324b59a1 */}
**다음 요소를 확인하세요:**
1. `allowed_payment_method_types`에 수단이 포함되어 있습니까?
2. 고객 위치가 수단의 지원 지역과 일치합니까?
3. 거래 통화가 해당 수단에서 지원됩니까?
4. 거래 금액이 최소 요구치(예: BNPL의 경우 $50.01)를 충족합니까?
5. 구독의 경우: 해당 수단이 정기 결제를 지원합니까?

**해결 방법:** `allowed_payment_method_types`을 일시적으로 제거해 모든 적격 수단을 확인한 다음 문제를 좁히세요.
</Accordion>

{/* LOCKED_PATTERN_57385d59bd851fb1e8c26266b489efd4 */}
**가능한 원인:**
- `allowed_payment_method_types`이 올바르게 전달되지 않음
- 매개변수 형식이 잘못됨(문자열 배열이어야 함)

**해결 방법:** API 요청 페이로드를 기록하고 매개변수 구조를 확인하세요.
</Accordion>

{/* LOCKED_PATTERN_6d1a1b1a59cbe2789bbc9434eb1ec153 */}
**확인 사항:**
- 고객 청구 주소가 수단의 지역과 일치하는지
- 통화가 올바르게 설정되었는지(예: iDEAL은 EUR)
- 고객이 국제 고객이면 Adaptive Currency가 활성화되어 있는지

**해결 방법:** API 요청에서 청구 국가 코드와 통화를 확인하세요.
</Accordion>

{/* LOCKED_PATTERN_52f3532b80d2def876ef3cc78b55f9f6 */}
UPI 및 Rupay 구독에는 48시간 처리 지연과 RBI 청구 권한 한도를 포함한 고유한 요구 사항이 있습니다.

**참고:** <a href="/features/payment-methods/india">India Payment Methods</a>에서 전체 안내를 확인하세요.
</Accordion>
</AccordionGroup>
