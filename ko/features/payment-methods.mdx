Dodo Payments는 고객에게 글로벌 서비스를 제공하기 위해 다양한 결제 수단을 지원합니다. 추가 계약, 통합 또는 상인 계정이 필요 없이 새로운 결제 수단을 즉시 활성화하십시오.

## 작동 방식

결제 수단은 다음에 따라 고객에게 자동으로 보여집니다:

- **고객 위치** — iDEAL(네덜란드) 또는 UPI(인도)와 같은 지역 방법은 해당 지역의 고객에게만 나타납니다.
- **거래 통화** — 지원되는 통화에 따라 결제 수단이 필터링됩니다.
- **장치 유형** — Apple Pay는 호환 가능한 Apple 장치와 브라우저에서만 나타납니다.
- **거래 유형** — 일부 방법은 구독을 지원하지 않거나 최소 금액이 있습니다.

<Info>
**자동 활성화:** 모든 결제 수단은 사전 구성되어 있으며 Dodo Payments에 의해 관리됩니다. 추가 설정이 필요하지 않으며, 계정이 검증되고 라이브 모드가 활성화되면 사용할 수 있게 됩니다.
</Info>

## 결제 수단 구성

기본적으로 모든 적격 결제 수단이 자동으로 표시됩니다. 표시되는 방법을 제한하려면 `allowed_payment_method_types`를 사용하십시오:

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://test.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

### 사용 가능한 방법 유형

| 유형                          | 설명                       |
| :----------------------------| :---------------------------|
| `credit` | 신용 카드 (Visa, Mastercard, Amex 등) |
| `debit` | 직불 카드                  |
| `apple_pay` | Apple Pay                  |
| `google_pay` | Google Pay                 |
| `amazon_pay` | Amazon Pay                 |
| `cashapp` | Cash App Pay               |
| `klarna` | Klarna BNPL                |
| `afterpay_clearpay` | Afterpay / Clearpay        |
| `upi_collect` | UPI Collect (QR 코드)     |
| `ideal` | iDEAL (네덜란드)          |
| `bancontact_card` | Bancontact (벨기에)       |
| `eps` | EPS (오스트리아)         |
| `multibanco` | Multibanco (포르투갈)     |
| `revolut_pay` | RevolutPay                |
| `billie` | Billie B2B BNPL           |

<Warning>
항상 `credit` 및 `debit`를 대체 옵션으로 포함하십시오. 지정된 결제 수단이 고객에게 모두 사용할 수 없으면 체크아웃이 실패합니다.
</Warning>

## 모범 사례

<AccordionGroup>
<Accordion title="시장에 맞는 방법 포함">
목표 시장에 대한 지역 결제 수단을 포함하십시오. iDEAL은 네덜란드의 ~60% 온라인 결제를 차지하며; UPI는 인도에서 지배적입니다. 이러한 방법을 생략하면 전환율에 큰 타격을 줄 수 있습니다.
</Accordion>

<Accordion title="항상 카드 대체 수단 제공">
특정 방법을 목표로 하는 경우에도 `credit` 및 `debit`를 대체 옵션으로 포함하십시오. 모든 고객이 모든 결제 방법을 이용할 수 있는 것은 아닙니다.
</Accordion>

<Accordion title="거래 최소 금액 고려">
BNPL 방법(Klarna, Afterpay)은 $50.01의 최소 금액이 필요합니다. 저가 제품에 대해 포함하지 마십시오 — 이들은 단순히 나타나지 않을 것이며, 괜찮습니다.
</Accordion>

<Accordion title="지역 및 장치별 테스트">
테스트 모드를 사용하여 다양한 고객 위치, 통화 및 장치 유형에 대한 결제 방법 가용성을 확인하십시오. 라이브 전입니다.
</Accordion>
</AccordionGroup>

## 문제 해결

<AccordionGroup>
<Accordion title="결제 방법이 나타나지 않음">
**다음 요소를 확인하십시오:**
1. 방법이 `allowed_payment_method_types`에 포함되어 있습니까?
2. 고객 위치가 방법의 지원 지역과 일치합니까?
3. 거래 통화가 방법에 의해 지원됩니까?
4. 거래 금액이 최소 금액을 충족합니까? (예: BNPL의 경우 $50.01)
5. 구독의 경우: 방법이 반복 결제를 지원합니까?

**해결책:** 모든 적격 방법을 보려면 `allowed_payment_method_types`를 일시적으로 제거한 다음 문제를 좁히십시오.
</Accordion>

<Accordion title="예상치 못한 방법이 나타남">
**가능한 원인:**
- `allowed_payment_method_types`가 올바르게 전달되지 않음
- 매개변수 형식이 올바르지 않음 (문자열 배열이어야 함)

**해결책:** API 요청 페이로드를 기록하고 매개변수 구조를 확인하십시오.
</Accordion>

<Accordion title="지역 방법이 사용 가능하지 않음">
**확인:**
- 고객 청구 주소가 방법의 지역과 일치합니까?
- 통화가 올바르게 설정되었습니까? (예: iDEAL의 경우 EUR)
- 고객이 국제적일 경우 적응 통화가 활성화되어 있습니까?

**해결책:** API 요청에서 청구 주소 국가 코드와 통화를 확인하십시오.
</Accordion>

<Accordion title="인도 결제 방법 문제">
UPI 및 Rupay 구독은 48시간 처리 지연 및 RBI 의무 제한 등 고유한 요구 사항이 있습니다.

**자세한 내용은:** <a href="/features/payment-methods/india">인도 결제 수단</a>을 참조하여 전체 가이드를 확인하십시오.
</Accordion>
</AccordionGroup>
