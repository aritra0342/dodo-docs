## 애드온이란 무엇인가요?

애드온은 고객이 기본 구독과 함께 구매할 수 있는 보조 상품입니다. 다음과 같은 경우에 적합합니다:

- **좌석 기반 청구**: 추가 팀원, 사용자 라이선스 또는 동시 사용자 수
- **기능 업그레이드**: 프리미엄 기능, 고급 분석 또는 우선 지원
- **사용량 확장**: 추가 저장공간, API 호출 또는 대역폭 허용량
- **서비스 애드온**: 전문 서비스, 교육 또는 상담 시간

## 주요 이점

- **유연한 가격 모델**: 선택형 애드온이 포함된 기본 요금제를 제공하여 정교한 가격 구조를 설계하세요. 고객의 니즈 변화에 따라 확장할 수 있는 업그레이드를 통해 다양한 고객군을 타깃팅할 수 있습니다.
- **수익 최적화**: 관련성 높은 애드온을 제안하여 사용자당 평균 수익(ARPU)을 향상시키세요. 시간이 지남에 따라 고객이 기능을 추가하면서 자연스러운 업셀 기회를 만들 수 있습니다.
- **간편한 관리**: 모든 가격 구성 요소를 하나의 대시보드에서 관리하세요. 애드온은 체크아웃 세션과 구독 관리에 자동으로 포함됩니다.
- **고객 선택권**: 고객이 필요한 애드온만 선택하여 구독을 맞춤화할 수 있도록 하여 만족도를 높이고 이탈률을 줄이세요.

## 애드온 만들기

애드온은 Dodo Payments 대시보드에서 별도의 상품으로 생성한 다음 주요 구독 상품에 연결됩니다. 이러한 분리는 다음과 같은 장점을 제공합니다:

- 여러 구독 상품에서 애드온을 재사용할 수 있음
- 가격을 독립적으로 관리할 수 있음
- 애드온 성과를 별도로 추적할 수 있음
- 기본 구독에 영향을 주지 않고 애드온을 업데이트할 수 있음

### 애드온 구성

애드온을 생성할 때 다음을 구성할 수 있습니다:

- **가격**: 애드온에 대한 일회성 또는 반복 가격 설정
- **청구 주기**: 구독의 청구 주기와 일치해야 함
- **수량 제한**: 고객당 최소 및 최대 수량 설정
- **사용 가능 구독**: 애드온을 사용할 수 있는 구독 상품 제어
- **세금 설정**: 적절한 세금 범주 구성

## 통합 예시

### 애드온이 포함된 체크아웃 세션
체크아웃 세션을 생성할 때 맞춤 수량의 애드온을 포함할 수 있습니다:

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    {
      product_id: 'your_subscription_id',
      quantity: 1,
      addons: [
        {
          addon_id: 'your_addon_id',
          quantity: 3 // 3 additional seats
        }
      ]
    }
  ],
  // ... other checkout options
});
```

### 애드온이 포함된 플랜 변경
기존 구독을 수정하여 애드온을 추가, 제거 또는 업데이트하세요:

```typescript
// Add add-ons to existing subscription
await client.subscriptions.changePlan('sub_123', {
  product_id: 'prod_new',
  quantity: 1,
  proration_billing_mode: 'difference_immediately',
  addons: [
    { addon_id: 'addon_123', quantity: 2 }
  ]
});

// Remove all existing add-ons
await client.subscriptions.changePlan('sub_123', {
  product_id: 'prod_new',
  quantity: 1,
  proration_billing_mode: 'difference_immediately',
  addons: [] // Empty array removes all existing add-ons
});
```

### 동적 가격 책정
애드온 선택을 기반으로 총 비용을 동적으로 계산하세요:

```typescript
function calculateTotalCost(basePrice: number, addons: AddonSelection[]) {
  const addonTotal = addons.reduce((sum, addon) => 
    sum + (addon.price * addon.quantity), 0
  );
  return basePrice + addonTotal;
}
```

## 모범 사례

- **간단하게 시작하세요**: 2~3개의 핵심 애드온으로 시작하고 고객 피드백 및 사용량을 토대로 옵션을 확장하세요.
- **가격 명확성 유지**: 애드온 가격과 가치를 명확히 전달하여 고객이 추가 비용에 대해 무엇을 얻는지 이해할 수 있도록 하세요.
- **철저히 테스트하세요**: 애드온 조합을 검증하여 가격 계산이 정확하게 유지되고 체크아웃 흐름이 원활하게 작동하는지 확인하세요.

### 설계 고려사항

- **명확한 가치 제안**: 각 애드온은 고객이 쉽게 이해할 수 있는 명확한 이점을 가져야 합니다.
- **논리적 그룹화**: 체크아웃 흐름에서 관련 애드온을 함께 그룹화하세요.
- **유연한 수량**: 고객이 필요에 따라 애드온 수량을 조정할 수 있도록 하세요.
- **투명한 가격**: 체크아웃 과정 전반에 걸쳐 총 비용을 명확히 표시하세요.

<Info>
애드온은 고객과 함께 확장되는 유연하고 확장 가능한 가격 모델을 만드는 강력한 방법입니다. 간단한 사용 사례로 시작하여 비즈니스와 고객에 가장 적합한 것을 알게 되면 확장하세요.
</Info>
