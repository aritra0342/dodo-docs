## What are Customer Wallets?

고객 지갑은 사용자에게 실제 자금을 보유하는 금전 잔액 계좌입니다. 고객 계정을 생성하면 자동으로 하나씩 부여됩니다. 이러한 지갑을 사용하여 다음과 같은 작업을 수행할 수 있습니다:

- **미래 구독 결제를 위한 선불 자금 저장**
- **카드 환불 대신 지갑 잔액으로 환불 처리**
- **환영 보너스나 충성도 보상 같은 프로모션 잔액 발행**
- **청구 시 지갑 자금을 자동으로 청구서에 적용**
- **자세한 원장 기록으로 금전 거래 추적**

<Info>
고객 계정을 생성하면 매번 지갑이 잔액 0으로 자동 생성됩니다. API를 통해 즉시 자금을 받을 준비가 되어 있습니다.
</Info>

<Warning>
**고객 지갑 ≠ 신용 기반 청구**

고객 지갑은 청구서와 구독 결제에 적용할 수 있는 **실제 금전 잔액**(USD, INR)을 보유합니다.

가상 사용 단위(API 호출, 토큰, 컴퓨팅 시간 등)를 추적하려면 대신 [Credit-Based Billing](/features/credit-based-billing)을 참조하세요.
</Warning>

## 작동 방식

고객 지갑은 고객이 구매에 적용할 수 있는 자금을 보유합니다. 고객에게 청구서나 반복 구독 요금이 있을 때, 먼저 지갑 잔액을 확인합니다. 이용 가능한 자금이 있다면 기본 결제 수단에 과금하기 전에 자동으로 청구서에 적용됩니다.

### 자동 설정

새 고객을 생성하면 Dodo Payments가 잔액 0인 지갑을 자동으로 생성합니다. API를 통해 즉시 자금을 받을 준비가 되어 있습니다.

### 다중 통화 지원

각 지갑은 다음과 같은 서로 다른 통화 잔액을 보유할 수 있습니다:

<ResponseField name="USD Balance" type="integer">
미국 달러 잔액(센트 단위로 저장)
</ResponseField>

<ResponseField name="INR Balance" type="integer">
인도 루피 잔액(파이즈 단위로 저장)
</ResponseField>

<Info>
현재는 **USD** 및 **INR** 잔액만 제공됩니다. 더 많은 통화는 곧 지원될 예정입니다.
</Info>

## 실제 사례

### 지갑으로 환불

고객이 환불을 요청하면 전통적인 카드 환불 대신 그 금액을 지갑 잔액에 추가할 수 있습니다. 이렇게 하면 향후 구매를 위해 자금이 귀하의 생태계 내에 유지됩니다.

```javascript
async function refundToWallet(customerId, refundAmount, originalPaymentId) {
  await client.customers.wallets.ledgerEntries.create(customerId, {
    amount: refundAmount, // Amount in cents
    currency: 'USD',
    entry_type: 'credit',
    reason: `Refund for payment ${originalPaymentId}`,
    idempotency_key: `refund_${originalPaymentId}`
  });
}
```

### 환영 보너스 / 프로모션 잔액

신규 고객에게 첫 구매를 유도하기 위해 금전적 환영 보너스를 제공하세요.

```javascript
async function addWelcomeBonus(customerId) {
  await client.customers.wallets.ledgerEntries.create(customerId, {
    amount: 1000, // $10.00 promotional balance
    currency: 'USD',
    entry_type: 'credit',
    reason: 'Welcome bonus - $10 promotional balance',
    idempotency_key: `welcome_${customerId}`
  });
}
```

### 지갑에서 구독 결제

구독 요금 또는 수동 구매를 충당하기 위해 지갑에서 자금을 차감하세요.

```javascript
async function deductForPurchase(customerId, purchaseAmount, purchaseId) {
  try {
    await client.customers.wallets.ledgerEntries.create(customerId, {
      amount: purchaseAmount,
      currency: 'USD',
      entry_type: 'debit',
      reason: `Subscription charge - monthly billing`,
      idempotency_key: `charge_${purchaseId}`
    });
  } catch (error) {
    if (error.status === 400) {
      console.log('Insufficient wallet balance');
    }
  }
}
```

## 모범 사례

### 중복 거래 방지

동일한 이벤트에 대해 자금을 두 번 추가하거나 차감하지 않도록 멱등성 키를 사용하세요.

```javascript
async function addFundsSafely(customerId, amount, reason) {
  const idempotencyKey = `${reason}_${customerId}_${Date.now()}`;
  
  try {
    const result = await client.customers.wallets.ledgerEntries.create(customerId, {
      amount: amount,
      currency: 'USD',
      entry_type: 'credit',
      reason: reason,
      idempotency_key: idempotencyKey
    });
    
    return { success: true, wallet: result };
  } catch (error) {
    if (error.status === 409) {
      // Transaction already processed
      return { success: true, wallet: null, duplicate: true };
    }
    
    throw error;
  }
}
```

### 과금 전에 잔액 확인

지갑에서 대규모 거래를 처리하기 전에 고객에게 충분한 자금이 있는지 확인하세요.

```javascript
async function checkBalanceBeforeOperation(customerId, requiredAmount) {
  const wallets = await client.customers.wallets.list(customerId);
  const usdWallet = wallets.items.find(w => w.currency === 'USD');
  
  if (!usdWallet || usdWallet.balance < requiredAmount) {
    throw new Error('Insufficient funds for this operation');
  }
  
  return usdWallet.balance;
}
```
