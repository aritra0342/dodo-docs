## 개요

모델 컨텍스트 프로토콜(MCP)은 AI 애플리케이션이 외부 데이터 소스 및 API에 안전하게 연결할 수 있도록 하는 개방형 표준입니다. Dodo Payments MCP 서버는 Claude, Cursor 및 기타 MCP 호환 클라이언트와 같은 AI 어시스턴트에게 귀하의 결제 인프라에 대한 구조화된 접근을 제공합니다.

Dodo Payments MCP 서버는 **Code Mode** 아키텍처를 사용합니다. 모든 API 엔드포인트에 대해 수백 개의 개별 도구를 노출하는 대신, Code Mode는 AI 에이전트가 격리된 샌드박스 환경에서 Dodo Payments SDK를 대상으로 TypeScript 코드를 작성하고 실행할 수 있게 합니다.

### 주요 기능

- **결제 작업**: 결제 및 환불 생성, 검색 및 관리
- **구독 관리**: 반복 청구, 업그레이드 및 취소 처리
- **고객 관리**: 고객 데이터 및 포털 접근 관리
- **제품 카탈로그**: 제품, 가격 및 할인 생성 및 업데이트
- **라이선스 관리**: 소프트웨어 라이선스 활성화, 검증 및 관리
- **사용 기반 청구**: 측정된 사용량 추적 및 청구

## Code Mode 작동 방식

Dodo Payments MCP 서버는 AI 에이전트에 정확히 두 가지 도구를 제공합니다:

1. **Docs Search Tool**: Dodo Payments API 및 SDK에 대한 문서를 조회하여 사용 가능한 작업과 매개변수를 파악합니다.
2. **Code Execution Tool**: SDK를 대상으로 TypeScript 코드를 작성하고 보안 샌드박스 환경에서 실행합니다.

이 아키텍처를 통해 에이전트는 단일 호출로 복잡한 다단계 작업을 수행할 수 있습니다. 예를 들어, 에이전트는 모든 활성 구독을 나열하고 특정 기준으로 필터링한 다음 각 구독에 할인을 적용하는 작업을 하나의 스크립트 내에서 수행할 수 있습니다.

<Tip>
Code Mode를 사용하면 에이전트는 여러 API 호출을 연결하고 조건부 로직을 처리하며 여러 번 LLM과 왕복할 필요 없이 계산을 수행할 수 있습니다.
</Tip>

### 빠른 설정

AI 클라이언트에서 Dodo Payments MCP 서버에 연결하세요:

<Tabs>
<Tab title="Cursor">
다음 내용을 `~/.cursor/mcp.json`에 추가하세요:

```json
{
  "mcpServers": {
    "dodopayments": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://mcp.dodopayments.com/sse"]
    }
  }
}
```

</Tab>

<Tab title="Claude Desktop">
Claude Desktop 구성에 다음을 추가하세요:

```json
{
  "mcpServers": {
    "dodopayments": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://mcp.dodopayments.com/sse"]
    }
  }
}
```

구성 파일 위치:
- **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`
- **Linux**: `~/.config/Claude/claude_desktop_config.json`
</Tab>

<Tab title="Windsurf">

```json
{
```

</Tab>

<Tab title="Claude Code">

</Tab>

</Note>

## Dodo Knowledge MCP

Dodo Payments MCP 서버(API 작업 실행용) 외에도 AI 어시스턴트가 Dodo Payments 문서 및 지식 베이스에 즉시 액세스할 수 있도록 하는 **Dodo Knowledge MCP**라는 의미 기반 검색 서버를 제공합니다.

<Info>
**[ContextMCP.ai](https://contextmcp.ai) 기반**: Dodo Knowledge MCP는 ContextMCP를 통해 벡터 임베딩을 사용하여 문서 전체에 걸친 빠른 의미 기반 검색을 지원합니다.
</Info>

### Dodo Knowledge MCP란?

Dodo Knowledge MCP는 원격 MCP 서버로 다음을 제공합니다:

- **의미 기반 문서 검색**: 자연어 쿼리를 사용하여 관련 문서를 찾습니다.
- **문맥 기반 응답**: AI 어시스턴트는 Dodo Payments에 대한 정확하고 최신 정보를 얻습니다.
- **설정 불필요**: API 키나 로컬 설치 없이 바로 연결하고 쿼리를 실행할 수 있습니다.

### 두 MCP 서버를 함께 사용하기

최상의 AI 지원 개발 경험을 위해 두 MCP 서버를 함께 사용하는 것이 좋습니다:

| 서버 | 목적 | 사용 사례 |
|--------|---------|----------|
| **Dodo Knowledge MCP** | 문서 검색 | "웹훅을 어떻게 처리하나요?", "어떤 결제 수단을 지원하나요?" |
| **Dodo Payments MCP** | API 작업 | 결제 생성, 구독 관리, 환불 처리 |

```json
{
  "mcpServers": {
    "dodo-knowledge": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://knowledge.dodopayments.com/mcp"]
    },
    "dodopayments": {
      "command": "npx",
      "args": ["-y", "mcp-remote@latest", "https://mcp.dodopayments.com/sse"]
    }
  }
}
```

<Tip>
두 서버를 모두 구성하면 AI 어시스턴트는 기능을 구현하는 방법을 먼저 문서를 검색하여 이해하고, 실제 API 호출을 단일 대화 내에서 실행할 수 있습니다.
</Tip>

### Knowledge MCP 문제 해결

연결 문제가 발생하면 다음을 시도하세요:

1. **MCP 인증 캐시 지우기**: `rm -rf ~/.mcp-auth`
2. **클라이언트 애플리케이션 재시작**
3. **클라이언트 로그 확인**
4. **Node.js 버전 확인**: Node.js 18 이상 필요

### NPM 패키지

NPM을 사용하여 로컬에서 MCP 서버를 설치하고 실행하세요.

<Tabs>
<Tab title="NPX (No Installation)">

```bash
# Set your environment variables
export DODO_PAYMENTS_API_KEY="dodo_test_..."
export DODO_PAYMENTS_WEBHOOK_KEY="your_webhook_key"
export DODO_PAYMENTS_ENVIRONMENT="live_mode"

# Run the latest version
npx -y dodopayments-mcp@latest
```

</Tab>

<Tab title="MCP Client Configuration">

```json
{
  "mcpServers": {
    "dodopayments_api": {
      "command": "npx",
      "args": ["-y", "dodopayments-mcp@latest"],
      "env": {
        "DODO_PAYMENTS_API_KEY": "dodo_test_...",
        "DODO_PAYMENTS_WEBHOOK_KEY": "your_webhook_key",
        "DODO_PAYMENTS_ENVIRONMENT": "live_mode"
      }
    }
  }
}
```

</Tab>
</Tabs>

<Tip>
항상 최신 버전을 사용하려면 `@latest`를 사용하거나 안정성을 위해 `@1.0.0`과 같은 특정 버전으로 고정하세요.
</Tip>

### Docker

일관된 배포를 위해 컨테이너화된 환경에서 MCP 서버를 실행하세요.

<CodeGroup>

```bash Docker
# Pull the latest image
docker pull ghcr.io/dodopayments/mcp:latest

# Run the container
docker run -e DODO_PAYMENTS_API_KEY="dodo_test_..." \
  -e DODO_PAYMENTS_WEBHOOK_KEY="your_webhook_key" \
  -e DODO_PAYMENTS_ENVIRONMENT="live_mode" \
  ghcr.io/dodopayments/mcp:latest
```

```yaml Docker Compose
services:
  dodopayments-mcp:
    image: ghcr.io/dodopayments/mcp:latest
    environment:
      - DODO_PAYMENTS_API_KEY=dodo_test_...
      - DODO_PAYMENTS_WEBHOOK_KEY=your_webhook_key
      - DODO_PAYMENTS_ENVIRONMENT=live_mode
    restart: unless-stopped
```

</CodeGroup>

<Note>
Docker 이미지는 [GitHub Container Registry](https://github.com/orgs/dodopayments/packages/container/package/mcp)에서 사용할 수 있습니다.
</Note>

## 환경 변수

환경 변수를 사용하여 MCP 서버 동작을 구성하세요.

| 변수 | 설명 | 필수 여부 |
|----------|-------------|----------|
| `DODO_PAYMENTS_API_KEY` | 귀하의 Dodo Payments API 키 | 예 |
| `DODO_PAYMENTS_WEBHOOK_KEY` | 웹훅 서명 키 | 아니요 |
| `DODO_PAYMENTS_ENVIRONMENT` | 프로덕션용으로 `live_mode` 값 설정 | 아니요 |

## 원격 실행

MCP 서버를 웹 기반 클라이언트 또는 에이전트 워크플로를 위한 원격 HTTP 서버로 배포하세요.

```bash
# Start as remote server
npx -y dodopayments-mcp --transport=http

# Specify a custom port
npx -y dodopayments-mcp --transport=http --port=3000
```

### 원격 서버 구성

배포 후, 클라이언트는 다음 서버 URL을 사용하여 연결할 수 있습니다:

```json
{
  "mcpServers": {
    "dodopayments_api": {
      "url": "http://localhost:3000",
      "headers": {
        "Authorization": "Bearer your_api_key"
      }
    }
  }
}
```

### 인증 헤더

원격 서버는 다음 헤더를 통해 인증을 허용합니다:

| 헤더 | 설명 |
|--------|-------------|
| `Authorization` | Bearer 토큰 인증 |
| `x-dodo-payments-api-key` | 직접 API 키 헤더 |

Code Mode는 샌드박스 환경에서 코드를 실행하고 서버 측에서 API 키를 주입함으로써 고유한 보안을 제공합니다. 자격 증명을 보호하기 위해 다음 추가 모범 사례도 따르세요.

<AccordionGroup>
<Accordion title="API Key Management">
**자격 증명을 버전 관리에 커밋하지 마세요**

API 키를 환경 변수 또는 보안 비밀 관리 시스템에 저장하세요.

```bash
# Use environment variables
```

**개발에는 테스트 키 사용**

개발 중에는 항상 테스트 모드 API 키를 사용하여 프로덕션 데이터를 변경하지 않도록 하세요.

**API 사용량 모니터링**

Dodo Payments 대시보드를 통해 MCP 서버 활동을 추적하고 이상 패턴에 대한 알림을 설정하세요.

<Accordion title="Network Security">

**요청 제한 구현**

MCP 서버와 API 수준 모두에서 요청 제한을 구현하여 남용을 방지하세요.

**네트워크 액세스 제한**

방화벽 규칙을 구성하여 MCP 서버에 연결할 수 있는 클라이언트를 제한하세요.
</Accordion>

## 문제 해결

<AccordionGroup>
<Accordion title="Connection issues">
**API 키 확인**

API 키가 올바르게 설정되어 있으며 필요한 권한을 가지고 있는지 확인하세요.

```bash
# Test your API key
curl -H "Authorization: Bearer dodo_test_..." \
  https://test.dodopayments.com/payments
```

**네트워크 연결 확인**

Dodo Payments API 엔드포인트에 접근 가능한지 확인하세요.

**클라이언트 로그 검토**

MCP 클라이언트에서 자세한 로깅을 활성화하여 연결 문제를 진단하세요.
</Accordion>

<Accordion title="Authentication errors">
**API 키 환경 확인**

테스트 엔드포인트에는 테스트 키를, 프로덕션 엔드포인트에는 라이브 키를 사용하고 있는지 확인하세요.

**환경 변수 확인**

`DODO_PAYMENTS_ENVIRONMENT`이 올바르게 설정되어 있는지 확인하세요(`live_mode`은 프로덕션용).

**자격 증명 재생성**

문제가 지속되면 대시보드를 통해 새 API 키를 생성하세요.
</Accordion>

<Accordion title="Tool execution failures">
**입력 매개변수 검증**

AI 어시스턴트가 각 도구에 대해 올바르게 형식화된 매개변수를 제공하는지 확인하세요.

**오류 메시지 검토**

API의 오류 응답을 확인하여 문제가 무엇인지 구체적인 지침을 받으세요.

**API 직접 테스트**

curl 또는 Postman을 통해 Dodo Payments API를 직접 호출하여 작업이 작동하는지 확인하세요.
</Accordion>
</AccordionGroup>

## 왜 Code Mode인가

기존 MCP 구현은 종종 각 API 엔드포인트를 개별 도구로 노출하는 "도구 증가" 문제를 겪습니다. Code Mode는 다음과 같은 여러 이유로 우수한 접근 방식입니다:

### LLM은 도구를 호출하기보다 코드를 작성하는 데 더 능숙합니다
LLM은 실제 코드 수백만 줄을 기반으로 훈련되어 자연스럽게 스크립트를 작성하는 데 숙련되어 있습니다. 반면 도구 호출은 종종 합성 예제에 기반합니다.

<Info>
"LLM에게 도구 호출로 작업을 수행하게 하는 것은 셰익스피어에게 한 달간 중국어 수업을 듣게 한 뒤 그 언어로 희곡을 쓰라고 하는 것과 같다." — [Cloudflare](https://blog.cloudflare.com/code-mode/)
</Info>

### 컨텍스트 창 부풀림 제거
기존 접근 방식에서는 모든 도구 정의가 대화가 시작되기 전에 토큰을 소비합니다. 50개 이상의 도구를 노출하면 쉽게 55K~100K+ 토큰이 소비됩니다. Anthropic은 도구 정의가 최적화 이전에 최대 134K 토큰을 소비할 수 있다는 사실을 발견했습니다.

Code Mode에서는 단 두 개의 도구 정의만 로드되며(약 1K 토큰) 에이전트는 필요한 문서를 사용 시점에 검색합니다. Anthropic의 Tool Search Tool은 컨텍스트 창의 95%를 유지하며 오버헤드를 77K에서 8.7K 토큰으로 감소시켰습니다.

### 프로그램 방식 오케스트레이션으로 대기 시간 감소
기존 도구 호출은 각 작업마다 모델 추론을 위해 왕복이 필요합니다. 작업에 20개의 API 호출이 필요한 경우 20번의 왕복이 필요합니다. Code Mode에서는 에이전트가 모든 호출을 수행하고 최종 결과만 반환하는 하나의 스크립트를 작성합니다.

Anthropic은 이 프로그램 방식 접근 방식을 통해 토큰 사용량을 37% 줄이고 정확도를 향상시켰으며(지식 검색은 25.6%에서 28.5%로 향상) 향상된 결과를 관찰했습니다.

### 설계상 더 안전함
Code Mode는 다음과 같은 고유 보안 이점을 제공합니다:
- **매개변수에 API 키 없음**: API 키는 서버 측에서 주입되며 도구 매개변수로 LLM에 노출되지 않습니다.
- **격리된 샌드박스**: 코드는 네트워크나 호스트 파일 시스템에 액세스할 수 없는 안전한 환경에서 실행됩니다.
- **제어된 SDK**: 에이전트는 인증된 SDK 메서드만 사용할 수 있습니다.

### 모든 API 크기로 확장 가능
API가 성장함에 따라 더 많은 도구를 컨텍스트에 로드해야 하기 때문에 기존 MCP 성능이 저하됩니다. Code Mode는 API 표면적에 관계없이 항상 두 개의 도구로 일정합니다. Cloudflare는 2,500개 이상의 API 엔드포인트를 단 두 개의 도구와 약 1,000 토큰의 컨텍스트로 성공적으로 통합했습니다.

<Info>
이 아키텍처의 이점에 대한 자세한 내용은 [Anthropic](https://www.anthropic.com/engineering/advanced-tool-use) 및 [Cloudflare](https://blog.cloudflare.com/code-mode/)의 엔지니어링 블로그와 Claude의 [Programmatic Tool Calling](https://platform.claude.com/docs/en/agents-and-tools/tool-use/programmatic-tool-calling) 문서를 참조하세요.
</Info>
