## 개요

Next.js 15, React 19, Supabase, Drizzle ORM 및 Dodo Payments를 사용하는 프로덕션 준비 완료 구독 보일러플레이트입니다. Google OAuth, 구독 체크아웃, 웹훅 처리, 데이터베이스 스키마 및 기본 대시보드가 포함되어 있습니다.

<Info>
기존 앱에 라우트 핸들러만 필요하다면 전용 어댑터인 <a href="/developer-resources/nextjs-adaptor">Next.js Adaptor</a>와 <a href="/developer-resources/express-adaptor">Express Adaptor</a>를 참고하세요.
</Info>

## 필수 조건

- Node.js 18+ (또는 Bun 1.0+)
- Supabase 프로젝트 (URL, Anon 키, 서비스 역할 키, 데이터베이스 URL)
- Dodo Payments 계정 (API 키, 웹훅 비밀)
- Google Cloud OAuth 클라이언트 (클라이언트 ID 및 비밀)

## 빠른 시작

<Steps>
{/* LOCKED_PATTERN_587a3a4c926aec89f48dfc1e7c21054e */}

```bash
git clone https://github.com/dodopayments/dodo-supabase-subscription-starter.git
cd dodo-supabase-subscription-starter
# choose one
bun install
# or
npm install
# or
pnpm install
```

</Step>

{/* LOCKED_PATTERN_ea54c61e84e28cd275c0eed6d7634e0f */}
Supabase 프로젝트를 생성하고 다음 항목을 복사하세요:
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
- SUPABASE_SERVICE_ROLE_KEY
- DATABASE_URL (Connection string)
</Step>

{/* LOCKED_PATTERN_42bf2f0f0010fd47b77c88325d34313c */}
Google Cloud에서 리디렉션 URI를 `https://[your-project-ref].supabase.co/auth/v1/callback`으로 설정한 후, 클라이언트 ID와 시크릿을 사용해 Supabase Auth에서 Google 공급자를 활성화하세요.
</Step>

{/* LOCKED_PATTERN_f32b65d8a917a558befa771bede82115 */}
Dodo 대시보드에서 API 키와 Webhook 시크릿을 생성하세요. 개발 중에는 환경을 `test_mode`으로 설정하세요.
</Step>

{/* LOCKED_PATTERN_a34a8e162789e5a79d108d9eb3c86df5 */}

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role

# Database
DATABASE_URL=postgresql://postgres:[password]@db.[project-ref].supabase.co:5432/postgres

# Dodo Payments
DODO_PAYMENTS_API_KEY=your-dodo-api-key
DODO_WEBHOOK_SECRET=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT=test_mode
```

<Warning>
시크릿을 커밋하지 마세요. 배포 환경에서는 환경 변수를 사용하세요.
</Warning>
</Step>

{/* LOCKED_PATTERN_c35fe33eae6aa68b4bc39bd740632964 */}

```bash
bun run db:push
# or
npm run db:push
# or
pnpm run db:push
```

<Check>
생성된 테이블: `users`, `subscriptions`, `payments`.
</Check>
</Step>

{/* LOCKED_PATTERN_560d1d81fad22bd0622f0c8dcb3c706d */}

```bash
# login (one-time)
bunx supabase login
# or
npx supabase login

# deploy the edge function
bun run deploy:webhook --project-ref [your-project-ref]
# or
npm run deploy:webhook -- --project-ref [your-project-ref]
# or
pnpm run deploy:webhook --project-ref [your-project-ref]
```

```bash cURL
curl -X POST \
  'https://[your-project-ref].supabase.co/functions/v1/dodo-webhook' \
  -H 'Content-Type: application/json' \
  -H 'Dodo-Signature: <signature>' \
  -d '{"type":"payment.succeeded","data":{}}'
```

</Step>

{/* LOCKED_PATTERN_dd5db8f3734010c91f70fdf794c4e2a2 */}
엔드포인트 URL을 다음으로 설정하세요:
</Step>

```text

```

결제 및 구독 이벤트를 선택하세요.
</Step>

{/* LOCKED_PATTERN_43dfdbde6ac180daa9987aee2baf2da7 */}
Dodo 대시보드 → Products → Create Product로 이동합니다. 선택적으로 메타데이터를 추가하세요:
</Step>

```json

  "features": ["Feature 1", "Feature 2", "Feature 3"]
}
```

요금제 UI는 이 `features` 배열을 읽어 동적으로 렌더링합니다.
</Step>

{/* LOCKED_PATTERN_e437399605995e5a4845b4d749b71d01 */}

```bash

# or
```

http://localhost:3000을 엽니다.
</Step>
</Steps>

## 포함된 내용

- Supabase를 통한 인증 (Google OAuth 구성 완료)
- Dodo Payments를 통한 구독 체크아웃
- 웹훅용 Supabase Edge Function (`dodo-webhook`)
- Drizzle ORM 스키마 및 마이그레이션
- 청구서, 구독 상태, 요금제 기능을 제공하는 대시보드

<Tip>
종단 간 테스트를 완료할 때까지 `DODO_PAYMENTS_ENVIRONMENT`을 `test_mode`으로 유지하세요.
</Tip>

## 검증 및 문제 해결

<AccordionGroup>
{/* LOCKED_PATTERN_6b286e9e2bbacd5c7684f8442203f10e */}
- `DODO_WEBHOOK_SECRET`이 Dodo 대시보드의 값과 일치하는지 확인하세요
- 최신 `dodo-webhook` 함수를 배포했는지 확인하세요
- 함수에서 헤더 이름(Dodo-Signature)이 올바른지 확인하세요
</Accordion>
{/* LOCKED_PATTERN_c270a2f8797d0168d608bb07bd40f056 */}
- `DATABASE_URL` 구문과 Supabase 네트워크 이그레스를 확인하세요
- 프로젝트 생성 후 첫 번째 푸시 전에 약 2~3분 기다리세요
</Accordion>
{/* LOCKED_PATTERN_276d2f8137bb346c423586f69d6b7cae */}
- 리디렉션 URI는 `https://[ref].supabase.co/auth/v1/callback`이어야 합니다
- Google Cloud와 Supabase Auth 공급자 모두에서 동일한지 확인하세요
</Accordion>
</AccordionGroup>
