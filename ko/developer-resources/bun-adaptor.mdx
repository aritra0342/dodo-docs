## 설치

<Steps>
<Step title="Install the package">
프로젝트 루트에서 다음 명령을 실행하세요:

```bash
bun add @dodopayments/bun
```

</Step>
<Step title="Set up environment variables">
프로젝트 루트에 <code>.env</code> 파일을 생성하세요:

```env expandable
DODO_PAYMENTS_API_KEY=your-api-key
DODO_PAYMENTS_WEBHOOK_KEY=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT="test_mode" or "live_mode"
DODO_PAYMENTS_RETURN_URL=your-return-url
```

<Warning>
절대 <code>.env</code> 파일이나 비밀을 버전 관리 시스템에 커밋하지 마세요.
</Warning>
</Step>
</Steps>

## 라우트 핸들러 예시

<Info>
모든 예시는 <code>Bun.serve()</code>와 함께 Bun의 네이티브 서버를 사용한다고 가정합니다.
</Info>

<Tabs>
<Tab title="Checkout Handler">
  <Info>
  이 핸들러를 사용해 Dodo Payments 체크아웃을 Bun 서버에 통합하세요. 정적(GET), 동적(POST), 세션(POST) 결제 흐름을 모두 지원합니다.
  </Info>
  <CodeGroup>

    ```typescript Bun Server Handler expandable
    import { Checkout } from '@dodopayments/bun';

    const staticCheckoutHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "static"
    });

    const sessionCheckoutHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "session"
    });

    const dynamicCheckoutHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "dynamic"
    });

    Bun.serve({
      port: 3000,
      fetch(request) {
        const url = new URL(request.url);
        
        if (url.pathname === "/api/checkout") {
          if (request.method === "GET") {
            return staticCheckoutHandler(request);
          }
          if (request.method === "POST") {
            return sessionCheckoutHandler(request);
            // or return dynamicCheckoutHandler(request);
          }
        }
        
        return new Response("Not Found", { status: 404 });
      },
    });
    ```

  </CodeGroup>

## 체크아웃 라우트 핸들러

<Info>
Dodo Payments는 사이트에 결제를 통합하기 위한 세 가지 유형의 결제 흐름을 지원하며, 이 어댑터는 모든 결제 흐름을 지원합니다.
</Info>

- **Static Payment Links:** 빠르고 코드 없이 결제 수집이 가능한 즉시 공유 가능한 URL입니다.
- **Dynamic Payment Links:** API 또는 SDK를 사용하여 커스텀 세부 정보로 결제 링크를 프로그래밍 방식으로 생성합니다.
- **Checkout Sessions:** 사전 구성된 상품 카트와 고객 정보를 사용하여 보안성과 사용자 정의가 강화된 체크아웃 환경을 생성합니다.

## 고객 포털 라우트 핸들러

고객 포털 라우트 핸들러는 Dodo Payments 고객 포털을 Bun 서버 애플리케이션에 원활하게 통합할 수 있도록 합니다.

### 쿼리 매개변수
<ParamField query="customer_id" type="string" required>
포털 세션에 사용할 고객 ID(예: <code>?customer_id=cus_123</code>).
</ParamField>
<ParamField query="send_email" type="boolean">
<code>true</code>로 설정하면 고객에게 포털 링크가 포함된 이메일을 전송합니다.
</ParamField>

<Warning>
<code>customer_id</code>가 없으면 400을 반환합니다.
</Warning>

## 웹훅 라우트 핸들러

- **메서드:** POST 요청만 지원됩니다. 다른 메서드는 405를 반환합니다.
- **서명 검증:** <code>webhookKey</code>를 사용하여 웹훅 서명을 검증합니다. 검증 실패 시 401을 반환합니다.
- **페이로드 검증:** Zod로 검증합니다. 잘못된 페이로드는 400을 반환합니다.
- **오류 처리:**
  - 401: 잘못된 서명
  - 400: 잘못된 페이로드
  - 500: 검증 중 내부 오류
- **이벤트 라우팅:** 페이로드 유형에 따라 적절한 이벤트 핸들러를 호출합니다.
