Dodo Payments는 거래, 구독, 환불, 분쟁 및 라이선스 키 관리와 관련된 중요한 이벤트에 대한 실시간 웹훅 알림을 제공합니다. 아래는 이러한 이벤트에 대한 자세한 가이드로, 명확성을 위해 분류되어 있으며, 구현 팁과 고려 사항이 포함되어 있습니다.

## **결제 이벤트**

| **이벤트 이름**           | **이벤트 유형**           | **설명**                                                                 |
|--------------------------|--------------------------|---------------------------------------------------------------------------------|
| **결제 성공**            | `payment.succeeded`      | 결제가 성공적으로 처리될 때 발생합니다.                                     |
| **결제 실패**            | `payment.failed`         | 오류, 거부된 카드 또는 기타 문제로 인해 결제 시도가 실패할 때 발생합니다. |
| **결제 처리 중**         | `payment.processing`     | 결제가 현재 처리 중임을 나타냅니다.                                          |
| **결제 취소됨**          | `payment.cancelled`      | 결제가 완료되기 전에 취소될 때 발생합니다.                                  |

## **환불 이벤트**

| **이벤트 이름**           | **이벤트 유형**           | **설명**                                                                 |
|--------------------------|--------------------------|---------------------------------------------------------------------------------|
| **환불 성공**            | `refund.succeeded`       | 환불이 성공적으로 처리될 때 발생합니다.                                      |
| **환불 실패**            | `refund.failed`          | 처리 오류 또는 기타 문제로 인해 환불 시도가 실패할 때 발생합니다.          |

## **분쟁 이벤트**

| **이벤트 이름**           | **이벤트 유형**           | **설명**                                                                 |
|--------------------------|--------------------------|---------------------------------------------------------------------------------|
| **분쟁 시작됨**          | `dispute.opened`         | 고객이 분쟁을 시작할 때 발생합니다.                                          |
| **분쟁 만료됨**          | `dispute.expired`        | 분쟁이 해결되지 않고 만료되었음을 나타냅니다.                                |
| **분쟁 수락됨**          | `dispute.accepted`       | 상인이 분쟁을 수락할 때 발생합니다.                                          |
| **분쟁 취소됨**          | `dispute.cancelled`      | 고객이나 시스템에 의해 분쟁이 취소될 때 발생합니다.                         |
| **분쟁 도전됨**          | `dispute.challenged`     | 상인이 분쟁에 도전할 때 발생합니다.                                          |
| **분쟁 승리**            | `dispute.won`            | 상인이 분쟁에서 성공적으로 승리했음을 나타냅니다.                          |
| **분쟁 패배**            | `dispute.lost`           | 상인이 분쟁에서 패배할 때 발생합니다.                                       |

## **구독 이벤트**

| **이벤트 이름**           | **이벤트 유형**           | **설명**                                                                 |
|--------------------------|--------------------------|---------------------------------------------------------------------------------|
| **구독 활성화**          | `subscription.active`    | 구독이 활성화되었으며 반복 청구가 예정되어 있음을 나타냅니다.               |
| **구독 업데이트됨**      | `subscription.updated`   | 구독 필드가 업데이트될 때 발생합니다(폴링 없이 실시간 동기화).               |
| **구독 보류 중**         | `subscription.on_hold`   | 갱신 실패로 인해 구독이 일시적으로 보류될 때 발생합니다.                   |
| **구독 갱신됨**          | `subscription.renewed`   | 구독이 성공적으로 갱신될 때 발생합니다.                                      |
| **구독 플랜 변경됨**      | `subscription.plan_changed` | 구독이 업그레이드, 다운그레이드되거나 다른 추가 기능으로 수정될 때 발생합니다. |
| **구독 취소됨**          | `subscription.cancelled` | 상인이나 고객에 의해 구독이 취소될 때 발생합니다.                          |
| **구독 실패**            | `subscription.failed`    | 구독이 실패했음을 나타냅니다. 이는 우리가 위임을 생성할 수 없음을 의미합니다. |
| **구독 만료됨**          | `subscription.expired`   | 구독이 기간이 끝나고 만료될 때 발생합니다.                                   |

## **라이선스 키 이벤트**

| **이벤트 이름**           | **이벤트 유형**           | **설명**                                                                 |
|--------------------------|--------------------------|---------------------------------------------------------------------------------|
| **라이선스 키 생성됨**    | `license_key.created`    | 제품에 대한 새로운 라이선스 키가 생성될 때 발생합니다.                     |

이 웹훅 이벤트는 상인이 운영을 효과적으로 관리할 수 있도록 중요한 업데이트를 제공하기 위해 설계되었습니다. 웹훅 통합에서 각 관련 이벤트에 대해 강력한 핸들러를 구현하여 원활하고 신뢰할 수 있는 기능을 보장하십시오.
