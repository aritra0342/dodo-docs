Digital wallets let customers pay with a single tap using saved payment methods. They offer higher conversion rates and stronger security through tokenization.

## Supported Wallets

| Wallet | Availability | Currencies | Subscriptions |
| :----- | :----------- | :--------- | :-----------: |
| **Apple Pay** | Global (excl. India) | All | Yes |
| **Google Pay** | Global (excl. India) | All | Yes |
| **Amazon Pay** | Global (excl. India) | USD | No |
| **Cash App Pay** | US only | USD | No |
| **RevolutPay** | Global | GBP | No |

<Warning>
**PayPal Support Paused:** PayPal is temporarily unavailable.
</Warning>

## Configuration

<AccordionGroup>
<Accordion title="Apple Pay" defaultOpen>
Requires domain verification before appearing in production.

**Setup:**
1. Download the [domain association file](http://checkout.dodopayments.com/.well-known/apple-developer-merchantid-domain-association)
2. Host at `https://yourdomain.com/.well-known/apple-developer-merchantid-domain-association`
3. Email **support@dodopayments.com** to request activation

```javascript
allowed_payment_method_types: ['apple_pay', 'credit', 'debit']
```

<Note>
Requires iOS 17+, iPadOS 17+, or Safari 17+ on macOS. Must be served over HTTPS.
</Note>
</Accordion>

<Accordion title="Google Pay">
Works automatically once your account is live. No domain verification needed.

```javascript
allowed_payment_method_types: ['google_pay', 'credit', 'debit']
```

<Note>
Requires Chrome or Android device with Google Pay app.
</Note>
</Accordion>

<Accordion title="Amazon Pay">
Customers use their Amazon credentials for checkout. USD only, no subscriptions.

```javascript
allowed_payment_method_types: ['amazon_pay', 'credit', 'debit']
```
</Accordion>

<Accordion title="Cash App Pay">
Popular among US consumers. US only, USD only, no subscriptions.

```javascript
allowed_payment_method_types: ['cashapp', 'credit', 'debit']
```
</Accordion>

<Accordion title="RevolutPay">
Fast payments for Revolut users. GBP only, no subscriptions.

```javascript
allowed_payment_method_types: ['revolut_pay', 'credit', 'debit']
```
</Accordion>
</AccordionGroup>

## Testing

<AccordionGroup>
<Accordion title="Apple Pay">
1. Enable test mode in your dashboard
2. Add a real card to Apple Wallet (won't be charged in test mode)
3. Open checkout on an Apple device and complete the flow

Must be served over HTTPS.
</Accordion>

<Accordion title="Google Pay">
1. [Join the test card group](https://groups.google.com/g/googlepay-test-mode-stub-data) to get test cards
2. Use test API keys
3. Select a test card in Google Pay to complete the transaction
</Accordion>

<Accordion title="Other Wallets">
Use test API keys and follow the standard checkout flow. Transactions will be simulated.
</Accordion>
</AccordionGroup>

## SDK Support

<Warning>
**React Native:** Apple Pay, Google Pay, and Cash App Pay are not currently supported. Use web-based checkout for these methods.
</Warning>

## Troubleshooting

<AccordionGroup>
<Accordion title="Apple Pay not appearing">
Check: domain verified? HTTPS? Compatible device (iOS 17+, Safari 17+)? Cards in Apple Wallet?
</Accordion>

<Accordion title="Google Pay not appearing">
Check: Chrome or compatible browser? Cards in Google Pay? Chrome payment settings enabled?
</Accordion>

<Accordion title="Payment failed">
Card in wallet may be expired or invalid. Customer should verify card or try different method.
</Accordion>
</AccordionGroup>
