India's payment landscape is dominated by UPI (60%+ of digital transactions). Dodo Payments supports UPI and Rupay with full RBI compliance for subscriptions.

## Supported Methods

| Method | Type | Subscriptions | Currency |
| :----- | :--- | :-----------: | :------- |
| **UPI Collect** | QR code / VPA | Yes* | INR |
| **Rupay** | Credit/Debit cards | Yes* | INR |

*Subscriptions require RBI-compliant mandates with 48-hour processing delay.

## Configuration

| Type | Description |
| :--- | :---------- |
| `upi_collect` | UPI via QR code or VPA |
| `credit` | Credit cards (including Rupay) |
| `debit` | Debit cards (including Rupay) |

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: ['upi_collect', 'credit', 'debit'],
  billing_currency: 'INR',
  billing_address: { country: 'IN', zipcode: '560001' },
  return_url: 'https://example.com/success'
});
```

<Warning>
For non-Indian merchants, **Adaptive Currency** must be enabled for UPI to appear.
</Warning>

## RBI Subscription Mandates

Indian subscriptions operate under RBI regulations with unique requirements:

<AccordionGroup>
<Accordion title="Mandate Limits" defaultOpen>
| Subscription Amount | Mandate Type |
| :------------------ | :----------- |
| Below Rs 15,000 | On-demand mandate (Rs 15,000 limit) |
| Rs 15,000 or above | Fixed-amount mandate |

If an upgrade exceeds the mandate limit, customer must re-authorize.
</Accordion>

<Accordion title="48-Hour Processing Delay">
1. **Day 0:** Charge initiated on scheduled date
2. **48 hours:** Customer can cancel mandate during this window
3. **~48-51 hours:** Actual deduction occurs

**Do not grant benefits until `payment.succeeded` webhook is received.**
</Accordion>

<Accordion title="Webhook Handling">
```javascript
// Wait for payment confirmation
if (event.type === 'payment.succeeded') {
  grantAccess(event.data.customer_id); // ✅ Correct
}

// DON'T grant access at charge initiation
```

| Event | When | Action |
| :---- | :--- | :----- |
| `payment.succeeded` | ~48h after charge | Grant access |
| `payment.failed` | Debit failed | Pause access |
| `subscription.on_hold` | Payment failed | Prompt payment update |
</Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
<Accordion title="UPI not appearing">
Check: Billing country set to `IN`? Currency set to `INR`? Adaptive Currency enabled (for non-Indian merchants)? `upi_collect` in allowed types?
</Accordion>

<Accordion title="Subscription charge failed after upgrade">
New amount may exceed mandate limit. Customer must update payment method to establish new mandate.
</Accordion>

<Accordion title="Payment delayed beyond 48 hours">
Bank API delays can extend processing by 2-3 hours. Build systems to handle up to ~51 hours total.
</Accordion>
</AccordionGroup>
