<Note>
This guide exclusively covers **one-time payments** involving **cards, Apple Pay and Google Pay**.
</Note>

## API Endpoints

| Environment | Base URL | Confirm Payment Endpoint |
| --- | --- | --- |
| **Test Mode** | `https://test.dodopayments.com` | `https://api.custom.hs.dodopayments.com/payments/<payment_id>/confirm` |
| **Live Mode** | `https://live.dodopayments.com` | `https://api.hs.dodopayments.com/payments/<payment_id>/confirm` |

## Step-by-Step Card Payment Flow

### Step 1: Obtain the Secret Key

Begin by creating a payment using the following API:

- **Payments**: [Create Payment API](/api-reference/payments/post-payments)

This API returns a **`secret_key`**, required for subsequent steps.

### Step 2: Confirm the Payment

Confirm the payment by calling the `payments confirm` endpoint:

- **Test Mode**: `POST https://api.custom.hs.dodopayments.com/payments/<payment_id>/confirm`
- **Live Mode**: `POST https://api.hs.dodopayments.com/payments/<payment_id>/confirm`

Include the following details in the request body:

- `client_secret`: Secret key from Step 1
- `payment_method`: Set to `card`
- `payment_method_data`: Card and billing details
- `browser_info`: Browser-specific data (user agent, screen details, etc.)
- `payment_type`: Set explicitly to `normal` (for one-time payments)

Headers required:

- Obtain your publishable key from DodoPayments support
- `api-key`: Your publishable key (Get this from Dodo Payments Team)
- `content-type`: `application/json`

**Example Request Body:**

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "card",
  "payment_method_data": {
    "card": {
      "card_number": "<card_number>",
      "card_exp_month": "<exp_month>",
      "card_exp_year": "<exp_year>",
      "card_holder_name": "<cardholder_name>",
      "card_cvc": "<cvc>",
      "card_network": "<network>"
    },
    "billing": {
      "address": {
        "first_name": "<first_name>",
        "last_name": "<last_name>"
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
    "time_zone": "<time_zone>",
    "java_enabled": true,
    "java_script_enabled": true,
    "ip_address": "<ip_address_of_client>",
    "device_model": "<device_model>",
    "os_type": "<os_type>",
    "os_version": "<version>"
  },
  "payment_type": "normal"
}
```

### Handling the Response

- If the response **`status`** is **`requires_customer_action`**, redirect the customer to the URL provided in the **`next_action`** field for further authentication.

<Warning>
Ensure adherence to PCI compliance guidelines when handling raw card data.
</Warning>

## Apple Pay Integration

To use Apple Pay, set `payment_method` to `wallet` and include the Apple Pay data in `payment_method_data`.

### Request Body Structure

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "wallet",
  "payment_method_data": {
    "wallet": {
      "apple_pay": {
        "payment_data": "<decrypted_payment_data>",
        "payment_method": {
          "display_name": "Visa 1234",
          "network": "Visa",
          "type": "debit"
        },
        "transaction_identifier": "UNIQUE_TRANSACTION_ID_FROM_APPLE"
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
    "time_zone": "<time_zone>",
    "java_enabled": true,
    "java_script_enabled": true,
    "ip_address": "<ip_address_of_client>",
    "device_model": "<device_model>",
    "os_type": "<os_type>",
    "os_version": "<version>"
  },
  "payment_type": "normal"
}
```

### Apple Pay Fields Reference

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `payment_data` | string | Yes | This enum is used to represent the Apple Pay payment data, which can either be encrypted or decrypted. |
| `payment_method.display_name` | string | Yes | Display name shown on Apple Pay button (e.g., "Visa 1234") |
| `payment_method.network` | string | Yes | Card network (e.g., "Visa", "Mastercard", "Amex") |
| `payment_method.type` | string | Yes | Payment method type (e.g., "debit", "credit") |
| `payment_method.card_exp_month` | string | No | Card expiration month (e.g., "12") |
| `payment_method.card_exp_year` | string | No | Card expiration year (e.g., "25") |
| `transaction_identifier` | string | Yes | Unique transaction identifier from Apple Pay |

## Google Pay Integration

To use Google Pay, set `payment_method` to `wallet` and include the Google Pay data in `payment_method_data`.

### Request Body Structure

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "wallet",
  "payment_method_data": {
    "wallet": {
      "google_pay": {
        "type": "CARD",
        "description": "Visa ending in 1234",
        "info": {
          "card_network": "VISA",
          "card_details": "1234"
        },
        "tokenization_data": {
          "type": "PAYMENT_GATEWAY",
          "token": "<encrypted_payment_token>"
        }
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
    "time_zone": "<time_zone>",
    "java_enabled": true,
    "java_script_enabled": true,
    "ip_address": "<ip_address_of_client>",
    "device_model": "<device_model>",
    "os_type": "<os_type>",
    "os_version": "<version>"
  },
  "payment_type": "normal"
}
```

### Google Pay Fields Reference

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `type` | string | Yes | Payment method type (typically "CARD") |
| `description` | string | Yes | User-facing description of the payment method |
| `info.card_network` | string | Yes | Card network (e.g., "VISA", "MASTERCARD", "AMEX") |
| `info.card_details` | string | Yes | Last 4 digits of the card |
| `info.assurance_details` | object | No | Authentication assurance details |
| `tokenization_data.type` | string | Yes | Tokenization type (e.g., "PAYMENT_GATEWAY") |
| `tokenization_data.token` | string | Yes | Encrypted payment token from Google Pay |
