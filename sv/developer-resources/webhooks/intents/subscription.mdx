## Prenumerations Webhook Händelser

Följande webhook-händelser är tillgängliga för att spåra förändringar i prenumerationens livscykel:

| Event | Description |
|-------|-------------|
| `subscription.active` | Prenumerationen aktiveras framgångsrikt |
| `subscription.updated` | Prenumerationsobjektet uppdaterades (ändring i vilket fält som helst utlöser det här händelsen) |
| `subscription.on_hold` | Prenumerationen pausas på grund av misslyckad förnyelse |
| `subscription.renewed` | Prenumerationen förnyas för nästa faktureringsperiod |
| `subscription.plan_changed` | Prenumerationsplanen uppgraderades, nedgraderades eller ändrades |
| `subscription.cancelled` | Prenumerationen avbryts |
| `subscription.failed` | Prenumerationsskapandet misslyckades under mandatets skapande |
| `subscription.expired` | Prenumerationen nådde slutet av sin löptid |

### Använda `subscription.updated` för synkronisering i realtid

Webhooken `subscription.updated` utlöses när som helst ett prenumerationsfält ändras, vilket gör att du kan hålla din applikationsstatus synkroniserad utan att behöva fråga API:et upprepade gånger:

```javascript
app.post('/webhooks/dodo', async (req, res) => {
  const event = req.body;
  
  if (event.type === 'subscription.updated') {
    const subscription = event.data;
    
    // Sync subscription changes to your database
    await syncSubscription(subscription.subscription_id, {
      status: subscription.status,
      next_billing_date: subscription.next_billing_date,
      metadata: subscription.metadata,
      // ... other fields you want to track
    });
    
    console.log(`Subscription ${subscription.subscription_id} updated`);
  }
  
  res.json({ received: true });
});
```

<Tip>
Prenumerera på `subscription.updated` för att få meddelanden i realtid om alla prenumerationsändringar, vilket eliminerar behovet av att fråga API:et om uppdateringar.
</Tip>
