## Snabbstart

Kom igång med automatisk LLM-tokenövervakning på bara 2 minuter:

<Steps>
{/* LOCKED_PATTERN_776027320500bde6b99bac6bed1cc64d */}
Installera Dodo Payments Ingestion Blueprints:

```bash
npm install @dodopayments/ingestion-blueprints
```

</Step>

{/* LOCKED_PATTERN_e608972569617212f31c0b7d72677865 */}
Du behöver två API-nycklar:

- **Dodo Payments API-nyckel**: Hämta den från [Dodo Payments Dashboard](https://app.dodopayments.com/developer/api-keys)
- **LLM-leverantörs API-nyckel**: Från AI SDK, OpenAI, Anthropic, Groq, etc.

<Tip>
Spara dina API-nycklar säkert i miljövariabler. Checka aldrig in dem i versionshantering.
</Tip>
</Step>

{/* LOCKED_PATTERN_3c26a22b684f0fb6b635e5bf1e61dd55 */}
Innan du börjar spåra användning, skapa en mätare i din Dodo Payments-instrumentpanel:

1. **Logga in** till [Dodo Payments Dashboard](https://app.dodopayments.com/)
2. **Navigera till** Products → Meters
3. **Klicka på** "Create Meter"
4. **Konfigurera din mätare**:
   - **Mätarnamn**: Välj ett beskrivande namn (t.ex. "LLM Token Usage")
   - **Händelsenamn**: Ställ in en unik händelseidentifierare (t.ex. `llm.chat_completion`)
   - **Aggregeringstyp**: Välj `sum` för att addera tokenräkningar
   - **Över egenskap**: Välj vad som ska spåras:
     - `inputTokens` - Spåra input-/prompttokens
     - `outputTokens` - Spåra output-/completion-tokens (inklusive resonemangstokens när det är tillämpligt)
     - `totalTokens` - Spåra kombinerade input + output-tokens

<Info>
**Händelsenamn** du ställer in här måste matcha exakt det du skickar till SDK:n (skiftlägeskänsligt).
</Info>

För detaljerade instruktioner, se [Usage-Based Billing Guide](/developer-resources/usage-based-billing-guide).
</Step>

## Konfiguration

### Tracker-konfiguration

Skapa en tracker en gång vid applikationsstart med dessa obligatoriska parametrar:

<ParamField path="apiKey" type="string" required>
Din Dodo Payments API-nyckel. Hämta den från [API Keys page](https://app.dodopayments.com/developer/api-keys).

```javascript
apiKey: process.env.DODO_PAYMENTS_API_KEY
```

</ParamField>

<ParamField path="environment" type="string" required>
Miljöläge för spåraren.

- `test_mode` - Använd för utveckling och testning
- `live_mode` - Använd för produktion

```javascript
environment: 'test_mode' // or 'live_mode'
```

<Warning>
Använd alltid `test_mode` under utveckling för att inte påverka produktionsmått.
</Warning>
</ParamField>

<ParamField path="eventName" type="string" required>
Händelsenamnet som triggar din mätare. Måste matcha exakt det du konfigurerade i din Dodo Payments-mätare (skiftlägeskänsligt).

```javascript
eventName: 'llm.chat_completion'
```

<Info>
Detta händelsenamn länkar din spårade användning till rätt mätare för faktureringsberäkningar.
</Info>
</ParamField>

### Wrapper-konfiguration

När du wrappar din LLM-klient, ange dessa parametrar:

<ParamField path="client" type="object" required>
Din LLM-klientinstans (OpenAI, Anthropic, Groq, med flera).

```javascript
client: openai
```

</ParamField>

<ParamField path="customerId" type="string" required>
Den unika kundidentifieraren för fakturering. Detta bör matcha ditt kund-ID i Dodo Payments.

```javascript
customerId: 'customer_123'
```

<Tip>
Använd ditt applikationsanvändar-ID eller kund-ID för att säkerställa korrekt fakturering per kund.
</Tip>
</ParamField>

<ParamField path="metadata" type="object">
Valfri ytterligare data att bifoga till spårningshändelsen. Användbart för filtrering och analys.

```javascript
metadata: {
  feature: 'chat',
  userTier: 'premium',
  sessionId: 'session_123',
  modelVersion: 'gpt-4'
}
```

</ParamField>

### Fullständig konfigurations exempel

<CodeGroup>

```javascript Full Configuration
import { createLLMTracker } from "@dodopayments/ingestion-blueprints";
import { generateText } from "ai";
import { google } from "@ai-sdk/google";
import "dotenv/config";

async function aiSdkExample() {
  console.log("🤖 AI SDK Simple Usage Example\n");

  try {
    // 1. Create tracker
    const llmTracker = createLLMTracker({
      apiKey: process.env.DODO_PAYMENTS_API_KEY!,
      environment: "test_mode",
      eventName: "your_meter_event_name",
    });

    // 2. Wrap the ai-sdk methods
    const client = llmTracker.wrap({
      client: { generateText },
      customerId: "customer_123",
      metadata: {
        provider: "ai-sdk",
      },
    });

    // 3. Use the wrapped function
    const response = await client.generateText({
      model: google("gemini-2.5-flash"),
      prompt: "Hello, I am a cool guy! Tell me a fun fact.",
      maxOutputTokens: 500,
    });

    console.log(response);
    console.log(response.usage);
    console.log("✅ Automatically tracked for customer\n");
  } catch (error) {
    console.error(error);
  }
}

aiSdkExample().catch(console.error);
```

</CodeGroup>

<Info>
**Automatisk spårning:** SDK:n spårar tokenanvändning i bakgrunden utan att ändra svaret. Din kod förblir ren och identisk med att använda de ursprungliga provider-SDK:erna.
</Info>

### Express.js API-integration

Fullständigt exempel på att integrera LLM-spårning i en Express.js API:

<CodeGroup>

```javascript Express.js Server
import express from 'express';
import { createLLMTracker } from '@dodopayments/ingestion-blueprints';
import OpenAI from 'openai';

const app = express();
app.use(express.json());

// Initialize OpenAI client
const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

// Create tracker once at startup
const tracker = createLLMTracker({
  apiKey: process.env.DODO_PAYMENTS_API_KEY,
  environment: process.env.NODE_ENV === 'production' ? 'live_mode' : 'test_mode',
  eventName: 'api.chat_completion'
});

// Chat endpoint with automatic tracking
app.post('/api/chat', async (req, res) => {
  try {
    const { message, userId } = req.body;
    
    // Validate input
    if (!message || !userId) {
      return res.status(400).json({ error: 'Missing message or userId' });
    }
    
    // Wrap client for this specific user
    const trackedClient = tracker.wrap({
      client: openai,
      customerId: userId,
      metadata: { 
        endpoint: '/api/chat',
        timestamp: new Date().toISOString()
      }
    });
    
    // Make LLM request - automatically tracked
    const response = await trackedClient.chat.completions.create({
      model: 'gpt-4',
      messages: [{ role: 'user', content: message }],
      temperature: 0.7
    });
    
    const completion = response.choices[0].message.content;
    
    res.json({ 
      message: completion,
      usage: response.usage
    });
  } catch (error) {
    console.error('Chat error:', error);
    res.status(500).json({ error: 'Internal server error' });
  }
});

app.listen(3000, () => {
  console.log('Server running on port 3000');
});
```

</CodeGroup>

</CodeGroup>

### Händelsefält

<ParamField path="event_id" type="string">
Unik identifierare för denna specifika händelse. Genereras automatiskt av SDK:n.

Format: `llm_[timestamp]_[random]`
</ParamField>

<ParamField path="customer_id" type="string">
Kund-ID:t du angav när du omslöt klienten. Används för fakturering.

<ParamField path="event_name" type="string">
Händelsenamnet som triggar din mätare. Matchar din spårarkonfiguration.
</ParamField>

<ParamField path="timestamp" type="string">
ISO 8601-tidsstämpel när händelsen inträffade.
</ParamField>

<ParamField path="metadata" type="object">
Tokenanvändning och ytterligare spårningsdata:

- `inputTokens` - Antal input-/prompttokens som användes
- `outputTokens` - Antal output-/completion-tokens som användes (inkluderar resonemangstokens när tillämpligt)
- `totalTokens` - Totalt antal tokens (input + output)
- `model` - Den använda LLM-modellen (t.ex. "gpt-4")
- `provider` - LLM-leverantören (om inkluderad i wrapper-metadata)
- All anpassad metadata du angav när du omslöt klienten

<Note>
**Resonemangstokens:** För modeller med resonemangskapacitet inkluderar `outputTokens` automatiskt både completion-tokens och resonemangstokens.
</Note>
</ParamField>

<Info>
Din Dodo Payments-mätare använder `metadata`-fälten (särskilt `inputTokens`, `outputTokens` eller `totalTokens`) för att beräkna användning och fakturering.
</Info>
