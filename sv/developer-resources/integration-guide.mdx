## Förutsättningar

För att integrera Dodo Payments API behöver du:
- Ett Dodo Payments handelskonto
- API-uppgifter (API-nyckel och webhook hemlig nyckel) från instrumentpanelen

## Inställning av instrumentpanelen

1. Navigera till [Dodo Payments Dashboard](https://app.dodopayments.com/)

2. Skapa en produkt (engångsbetalning eller prenumeration)

4. Generera din API-nyckel:
   - Gå till Utvecklare > API
   - [Detaljerad guide](/api-reference/introduction#api-key-generation)
   - Kopiera API-nyckeln i miljön med namnet DODO_PAYMENTS_API_KEY

5. Konfigurera webhooks:
   - Gå till Utvecklare > Webhooks
   - Skapa en webhook-URL för betalningsnotifikationer
   - Kopiera den hemliga webhook-nyckeln i miljön 

## Integration

### Betalningslänkar

Välj den integrationsväg som passar ditt användningsfall:
- **Checkout Sessions (rekommenderas)**: Bäst för de flesta integrationer. Skapa en session på din server och omdirigera kunder till en säker, hostad checkout.
- **Overlay Checkout (inbäddad)**: Används när du behöver en in-page-upplevelse som bäddar in den hostade checkouten på din webbplats.
- **Statisk betalningslänk**: Ingen kod, omedelbart delbara URL:er för snabb betalningsinsamling.
- **Dynamiska betalningslänkar**: Programmässigt skapade länkar. Checkout Sessions rekommenderas och ger mer flexibilitet.

#### 1. Checkout Sessions

Använd Checkout Sessions för att skapa en säker, hostad checkoutupplevelse för engångsbetalningar eller prenumerationer. Du skapar en session på din server, och omdirigerar sedan kunden till den returnerade `checkout_url`.

<Info>
Checkout-sessions är giltiga i 24 timmar som standard. Om du skickar <code>confirm=true</code>, är sessionerna giltiga i 15 minuter och alla obligatoriska fält måste anges.
</Info>

<Steps>
<Step title="Skapa en checkout-session">
  Välj ditt föredragna SDK eller anropa REST API.

  <Tabs>
  <Tab title="Node.js SDK">

  ```javascript
  import DodoPayments from 'dodopayments';

  const client = new DodoPayments({
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    environment: 'test_mode', // defaults to 'live_mode'
  });

  const session = await client.checkoutSessions.create({
    product_cart: [{ product_id: 'prod_123', quantity: 1 }],
    customer: { email: 'customer@example.com', name: 'John Doe' },
    return_url: 'https://yourapp.com/checkout/success',
  });
  ```

  </Tab>

  <Tab title="Python SDK">

  ```python

      product_cart=[{"product_id": "prod_123", "quantity": 1}],
      customer={"email": "customer@example.com", "name": "John Doe"},
      return_url="https://yourapp.com/checkout/success",
  ```

#### 2. Overlay Checkout

För en sömlös in-page checkoutupplevelse, utforska vår [Overlay Checkout](/developer-resources/overlay-checkout) integration som gör att kunder kan slutföra betalningar utan att lämna din webbplats.

#### 3. Statisk betalningslänk

Statisk betalningslänkar låter dig snabbt ta emot betalningar genom att dela en enkel URL. Du kan anpassa checkoutupplevelsen genom att skicka med frågeparametrar för att förfylla kunduppgifter, kontrollera formulärfält och lägga till anpassad metadata.

<Steps>
<Step title="Konstruera din betalningslänk">
  Börja med bas-URL:en och lägg till din produkt-ID:

  ```text
  https://checkout.dodopayments.com/buy/{productid}
  ```

</Step>
<Step title="Lägg till kärnparametrar">
  Inkludera viktiga frågeparametrar:
  - <ParamField query="quantity" type="integer" default="1">Antal artiklar att köpa.</ParamField>
  - <ParamField query="redirect_url" type="string" required>URL att omdirigera efter betalningens slutförande.</ParamField>
  <Note>
  Omdirigerings-URL:en kommer att inkludera betalningsdetaljer som frågeparametrar, till exempel:<br/>
  <code>https://example.com/?payment_id=pay_ts2ySpzg07phGeBZqePbH&status=succeeded</code>
  </Note>
</Step>
<Step title="Förfylla kundinformation (valfritt)">
  Lägg till kund- eller faktureringsfält som frågeparametrar för att strömlinjeforma checkout.
  <AccordionGroup>
    <Accordion title="Stödda kundfält">
      - <ParamField query="fullName" type="string">Kundens fullständiga namn (ignoreras om firstName eller lastName anges).</ParamField>
      - <ParamField query="firstName" type="string">Kundens förnamn.</ParamField>
      - <ParamField query="lastName" type="string">Kundens efternamn.</ParamField>
      - <ParamField query="email" type="string">Kundens e-postadress.</ParamField>
      - <ParamField query="country" type="string">Kundens land.</ParamField>
      - <ParamField query="addressLine" type="string">Gatuadress.</ParamField>
      - <ParamField query="city" type="string">Stad.</ParamField>
      - <ParamField query="state" type="string">Stat eller provins.</ParamField>
      - <ParamField query="zipCode" type="string">Postnummer/ZIP-kod.</ParamField>
      - <ParamField query="showDiscounts" type="boolean">true eller false</ParamField>
    </Accordion>
  </AccordionGroup>
</Step>
<Step title="Kontrollera formulärfält (valfritt)">
  <Info>
  Du kan inaktivera specifika fält för att göra dem skrivskyddade för kunden. Detta är användbart när du redan har kundens uppgifter (t.ex. inloggade användare).
  </Info>
  För att inaktivera ett fält, ange dess värde och ställ in motsvarande <code>disable...</code> flagga till <code>true</code>:
  <CodeGroup>

  ```text Example
  &email=alice@example.com&disableEmail=true
  ```

  </CodeGroup>
  <Tabs>
    <Tab title="Inaktiveringsflaggor Tabell">
      | Fält          | Inaktiveringsflagga   | Obligatorisk parameter |
      |---------------|---------------------|-------------------|
      | Fullständigt namn | `disableFullName`   | `fullName`        |
      | Förnamn      | `disableFirstName`  | `firstName`       |
      | Efternamn    | `disableLastName`   | `lastName`        |
      | E-post       | `disableEmail`      | `email`           |
      | Land         | `disableCountry`    | `country`         |
      | Adressrad    | `disableAddressLine`| `addressLine`     |
      | Stad         | `disableCity`       | `city`            |
      | Stat         | `disableState`      | `state`           |
      | ZIP-kod      | `disableZipCode`    | `zipCode`         |
    </Tab>
  </Tabs>
  <Tip>
  Inaktivering av fält hjälper till att förhindra oavsiktliga ändringar och säkerställer datakonsistens.
  </Tip>
  <Note>
  Att ställa in <code>showDiscounts=false</code> kommer att inaktivera och dölja rabatteringssektionen i checkoutformuläret. Använd detta om du vill förhindra att kunder anger kupong- eller kampanjkoder under checkout.
  </Note>

</Step>
<Step title="Lägg till avancerade kontroller (valfritt)">
  - <ParamField query="paymentCurrency" type="string">Anger betalningsvalutan. Standard är den fakturerande landets valuta.</ParamField>
  - <ParamField query="showCurrencySelector" type="boolean" default="true">Visa eller dölja valutaväljaren.</ParamField>
  - <ParamField query="paymentAmount" type="integer">Belopp i cent (endast för Pay What You Want-prissättning).</ParamField>
  - <ParamField query="metadata_*" type="string">Anpassade metadatafält (t.ex. <code>metadata_orderId=123</code>).</ParamField>
</Step>
<Step title="Dela länken">
  Skicka den färdiga betalningslänken till din kund. När de besöker, samlas alla frågeparametrar in och lagras med ett session-ID. URL:en förenklas sedan för att endast inkludera sessionsparametern (t.ex. <code>?session=sess_1a2b3c4d</code>). Den lagrade informationen kvarstår genom siduppdateringar och är tillgänglig under hela checkoutprocessen.
  <Check>
  Kundens checkoutupplevelse är nu strömlinjeformad och personlig baserat på dina parametrar.
  </Check>
</Step>
</Steps>

### Implementera Webhooks

Ställ in en API-slutpunkt för att ta emot betalningsnotifikationer. Här är ett exempel med Next.js:

```javascript
import { Webhook } from "standardwebhooks";
import { headers } from "next/headers";
import { WebhookPayload } from "@/types/api-types";

const webhook = new Webhook(process.env.DODO_WEBHOOK_KEY!); // Replace with your secret key generated from the Dodo Payments Dashboard

export async function POST(request: Request) {
  const headersList = headers();
  const rawBody = await request.text();

  const webhookHeaders = {
    "webhook-id": headersList.get("webhook-id") || "",
    "webhook-signature": headersList.get("webhook-signature") || "",
    "webhook-timestamp": headersList.get("webhook-timestamp") || "",
  };

  await webhook.verify(rawBody, webhookHeaders);
  const payload = JSON.parse(rawBody) as WebhookPayload;
  
  // Process the payload according to your business logic
}
```

Vår webhook-implementering följer [Standard Webhooks](https://standardwebhooks.com/) specifikationen. För definitioner av webhook-typer, se vår [Webhook Event Guide](/developer-resources/webhooks/intents/webhook-events-guide).

Du kan hänvisa till detta projekt med demoimplementering på [GitHub](https://github.com/dodopayments/dodo-checkout-demo) med Next.js och TypeScript.

Du kan kolla in den live-implementeringen [här](https://atlas.dodopayments.com/).
