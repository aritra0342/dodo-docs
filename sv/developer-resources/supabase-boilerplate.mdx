## Översikt

Ett produktionsklart boilerplate för prenumerationer med Next.js 15, React 19, Supabase, Drizzle ORM och Dodo Payments. Det levereras med Google OAuth, prenumerationskassa, webhook-hantering, databasstruktur och en grundläggande instrumentpanel.

<Info>
Om du bara behöver route handlers för en befintlig app, se de dedikerade adaptrarna: <a href="/developer-resources/nextjs-adaptor">Next.js Adaptor</a> och <a href="/developer-resources/express-adaptor">Express Adaptor</a>.
</Info>

## Förutsättningar

- Node.js 18+ (eller Bun 1.0+)
- Supabase-projekt (URL, Anon-nyckel, Service roll-nyckel, Databas-URL)
- Dodo Payments-konto (API-nyckel, Webhook-hemlighet)
- Google Cloud OAuth-klient (Klient-ID och hemlighet)

## Snabbstart

<Steps>
{/* LOCKED_PATTERN_587a3a4c926aec89f48dfc1e7c21054e */}

```bash
git clone https://github.com/dodopayments/dodo-supabase-subscription-starter.git
cd dodo-supabase-subscription-starter
# choose one
bun install
# or
npm install
# or
pnpm install
```

</Step>

{/* LOCKED_PATTERN_ea54c61e84e28cd275c0eed6d7634e0f */}
Skapa ett Supabase-projekt och kopiera:
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
- SUPABASE_SERVICE_ROLE_KEY
- DATABASE_URL (Connection string)
</Step>

{/* LOCKED_PATTERN_42bf2f0f0010fd47b77c88325d34313c */}
Ställ in redirect-URI till: `https://[your-project-ref].supabase.co/auth/v1/callback` i Google Cloud, och aktivera sedan Google-leverantören i Supabase Auth med din Client ID och Secret.
</Step>

{/* LOCKED_PATTERN_f32b65d8a917a558befa771bede82115 */}
Generera en API-nyckel och Webhook-hemlighet från Dodo-instrumentpanelen. Ställ in miljön på `test_mode` under utvecklingen.
</Step>

{/* LOCKED_PATTERN_a34a8e162789e5a79d108d9eb3c86df5 */}

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role

# Database
DATABASE_URL=postgresql://postgres:[password]@db.[project-ref].supabase.co:5432/postgres

# Dodo Payments
DODO_PAYMENTS_API_KEY=your-dodo-api-key
DODO_WEBHOOK_SECRET=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT=test_mode
```

<Warning>
Gör aldrig commit med hemligheter. Använd miljövariabler i deploymentsmiljöer.
</Warning>
</Step>

{/* LOCKED_PATTERN_c35fe33eae6aa68b4bc39bd740632964 */}

```bash
bun run db:push
# or
npm run db:push
# or
pnpm run db:push
```

<Check>
Tabeller skapade: `users`, `subscriptions`, `payments`.
</Check>
</Step>

{/* LOCKED_PATTERN_560d1d81fad22bd0622f0c8dcb3c706d */}

```bash
# login (one-time)
bunx supabase login
# or
npx supabase login

# deploy the edge function
bun run deploy:webhook --project-ref [your-project-ref]
# or
npm run deploy:webhook -- --project-ref [your-project-ref]
# or
pnpm run deploy:webhook --project-ref [your-project-ref]
```

```bash cURL
curl -X POST \
  'https://[your-project-ref].supabase.co/functions/v1/dodo-webhook' \
  -H 'Content-Type: application/json' \
  -H 'Dodo-Signature: <signature>' \
  -d '{"type":"payment.succeeded","data":{}}'
```

</Step>

{/* LOCKED_PATTERN_dd5db8f3734010c91f70fdf794c4e2a2 */}
Ställ in slutpunkts-URL till:


```text
https://[your-project-ref].supabase.co/functions/v1/dodo-webhook
```

Välj betalnings- och prenumerationsevenemang.
</Step>

Prissättningsgränssnittet läser den här `features`-matrisen och renderar den dynamiskt.
</Step>

{/* LOCKED_PATTERN_e437399605995e5a4845b4d749b71d01 */}

```bash

# or
```

Öppna http://localhost:3000.
</Step>
</Steps>

## Vad som ingår

- Autentisering via Supabase (Google OAuth konfigurerat)
- Prenumerationskassa via Dodo Payments
- Supabase Edge Function för webhooks (`dodo-webhook`)
- Drizzle ORM-schema och migrationer
- Instrumentpanel med fakturor, prenumerationsstatus och planfunktioner

<Tip>
Behåll `DODO_PAYMENTS_ENVIRONMENT` som `test_mode` tills du har slutfört end-to-end-tester.
</Tip>

## Verifiering och felsökning

<AccordionGroup>
{/* LOCKED_PATTERN_6b286e9e2bbacd5c7684f8442203f10e */}
- Se till att `DODO_WEBHOOK_SECRET` matchar värdet från Dodo-instrumentpanelen
- Bekräfta att du deployerat den senaste `dodo-webhook`-funktionen
- Verifiera att rubriknamnet är korrekt i din funktion (Dodo-Signature)
</Accordion>
{/* LOCKED_PATTERN_c270a2f8797d0168d608bb07bd40f056 */}
- Kontrollera `DATABASE_URL`-syntaxen och Supabase-nätverkets egress
- Vänta ca 2–3 minuter efter att projektet skapats innan första push
</Accordion>
{/* LOCKED_PATTERN_276d2f8137bb346c423586f69d6b7cae */}
- Redirect-URI måste vara `https://[ref].supabase.co/auth/v1/callback`
- Säkerställ samma i Google Cloud och Supabase Auth-leverantören
</Accordion>
</AccordionGroup>
