## Installation

<Steps>
<Step title="Installera paketet">
Kör följande kommando i din projektrot:

```bash
npm install @dodopayments/remix
```

</Step>
<Step title="Ställ in miljövariabler">
Skapa en <code>.env</code> fil i din projektrot:

```env expandable
DODO_PAYMENTS_API_KEY=your-api-key
DODO_PAYMENTS_WEBHOOK_SECRET=your-webhook-secret
DODO_PAYMENTS_RETURN_URL=https://yourdomain.com/checkout/success
DODO_PAYMENTS_ENVIRONMENT="test_mode" or "live_mode"
```

<Warning>
Kom ihåg att aldrig begå din <code>.env</code> fil eller hemligheter till versionskontroll.
</Warning>
</Step>
</Steps>

<Tab title="Kassa-hanterare">
  <Info>
  Använd denna hanterare för att integrera Dodo Payments kassa i din Remix-app. Stöder statiska (GET), dynamiska (POST) och sessions (POST) betalningsflöden.
  </Info>
  <CodeGroup>

    ```typescript Remix Route Handler expandable
    // app/routes/api.checkout.tsx
    import { Checkout } from "@dodopayments/remix";
    import type { LoaderFunctionArgs } from "@remix-run/node";

    const checkoutGetHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "static"
    });

    const checkoutPostHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "dynamic", // for dynamic checkout
    });

    const checkoutSessionHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "session", // for checkout sessions
    });

    export const loader = ({ request }: LoaderFunctionArgs) => checkoutGetHandler(request);
    export const action = ({ request }: LoaderFunctionArgs) => {
        // You can conditionally route to different handlers based on your needs
        // For checkout sessions, use checkoutSessionHandler(request)
        return checkoutPostHandler(request);
    };
    ```

## Kassa-rutt-hanterare

<Info>
Dodo Payments stöder tre typer av betalningsflöden för att integrera betalningar på din webbplats, denna adaptor stöder alla typer av betalningsflöden.
</Info>

- **Statiska betalningslänkar:** Omedelbart delbara URL:er för snabb, kodfri insamling av betalningar.
- **Dynamiska betalningslänkar:** Programmera generera betalningslänkar med anpassade detaljer med hjälp av API:er eller SDK:er.
- **Kassasessioner:** Skapa säkra, anpassningsbara kassaupplevelser med förkonfigurerade produktvagnar och kunddetaljer.

Statisk kassa returnerar ett JSON-svar med kassa-URL:en:


<Accordion title="Dynamisk kassa (POST)">

  - [Begäran kropp för en engångsbetalningsprodukt](https://docs.dodopayments.com/api-reference/payments/post-payments)

### Svarformat
Dynamisk kassa returnerar ett JSON-svar med kassa-URL:en:

```json

}
```

</Accordion>
<Accordion title="Kassasessioner (POST)">
Kassasessioner erbjuder en säkrare, värd kassaupplevelse som hanterar hela betalningsflödet för både engångsköp och prenumerationer med full anpassningskontroll.

## Kundportal-rutt-hanterare

Kundportal-rutt-hanteraren gör det möjligt för dig att sömlöst integrera Dodo Payments kundportal i din Remix-applikation.

### Frågeparametrar
<ParamField query="customer_id" type="string" required>
Kund-ID för portal-sessionen (t.ex. <code>?customer_id=cus_123</code>).
</ParamField>
<ParamField query="send_email" type="boolean">
Om inställt på <code>true</code>, skickar ett e-postmeddelande till kunden med portal-länken.
</ParamField>

<Warning>
Returnerar 400 om <code>customer_id</code> saknas.
</Warning>

## Webhook-rutt-hanterare

- **Metod:** Endast POST-begärningar stöds. Andra metoder returnerar 405.
- **Signaturverifiering:** Verifierar webhook-signaturen med <code>webhookKey</code>. Returnerar 401 om verifieringen misslyckas.
- **Payload-validering:** Valideras med Zod. Returnerar 400 för ogiltiga payloads.
- **Felhantering:**
  - 401: Ogiltig signatur
  - 400: Ogiltig payload
  - 500: Internt fel under verifiering
- **Händelse-routing:** Anropar den lämpliga händelsehanteraren baserat på payload-typen.
