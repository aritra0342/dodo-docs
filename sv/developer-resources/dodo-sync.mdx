Dodo Payments erbjuder en inbyggd databas synkroniseringsfunktion som automatiskt synkroniserar dina betalningsdata med din egen databas. Du kan synkronisera **betalningar**, **kunder**, **prenumerationer** och **licenser** för att behålla en lokal kopia av dina data för analys, rapportering eller integration med andra system.

<Info>
**Implementering**: Finns via [npm package](https://www.npmjs.com/package/dodo-sync) | **Source Code**: [GitHub](https://github.com/dodopayments/dodo-sync)
</Info>

## Databasstöd

Vi stöder för närvarande **MongoDB**, **PostgreSQL**, **MySQL** och **ClickHouse**.

Vi arbetar aktivt med att utöka stödet för:
- **Databaser**: Snowflake och andra.
- **Pipelines**: ETL-pipelines, Realtids synkronisering.

<Tip>
Vi utökar kontinuerligt databassupporten. Om du vill bidra med en ny databasintegration, skicka gärna en Pull Request till vårt [GitHub repository](https://github.com/dodopayments/dodo-sync).
</Tip>

## Komma Igång

Du kan använda vår databas synkroniseringsfunktion via **CLI** eller programmässigt i din **kod**. Båda metoderna erbjuder samma funktionalitet—välj den som passar din arbetsflöde bäst.

### Använda CLI

CLI-verktyget erbjuder ett snabbt sätt att ställa in och köra databas synkronisering. Installera det globalt för att använda det från var som helst i din terminal:

<CodeGroup>

```bash npm
npm install -g dodo-sync
```

```bash bun
bun add -g dodo-sync
```

</CodeGroup>

#### Köra CLI

CLI stöder två lägen: **Interaktivt Läge** för guidad installation, och **Manuellt Läge** för direkt konfiguration.

**Interaktivt Läge**: Kör helt enkelt kommandot utan argument för att starta den interaktiva installationsguiden.

```bash
dodo-sync
```

**Manuellt Läge**: Skicka argument direkt för att hoppa över guiden.

```bash
dodo-sync -i [interval] -d [database] -u [database_uri] --scopes [scopes] --api-key [api_key] --env [environment]
```

**Exempel:**

```bash
# MongoDB
dodo-sync -i 600 -d mongodb -u mongodb://mymongodb.url --scopes "licences,payments,customers,subscriptions" --api-key YOUR_API_KEY --env test_mode

# PostgreSQL
dodo-sync -i 600 -d postgres -u postgresql://user:password@localhost:5432/mydb --scopes "licences,payments,customers,subscriptions" --api-key YOUR_API_KEY --env test_mode

# MySQL
dodo-sync -i 600 -d mysql -u mysql://user:password@localhost:3306/mydb --scopes "licences,payments,customers,subscriptions" --api-key YOUR_API_KEY --env test_mode

# ClickHouse
dodo-sync -i 600 -d clickhouse -u http://localhost:8123 --scopes "licences,payments,customers,subscriptions" --api-key YOUR_API_KEY --env test_mode
```

#### CLI-Argument

<ParamField path="--interval" type="number" alias="-i">
Synkroniseringsintervall i sekunder. Bestämmer hur ofta synkroniseringsoperationen körs. Om det inte anges körs synkroniseringen en gång och avslutas.
</ParamField>

<ParamField path="--database" type="string" alias="-d" required>
Databastyp att använda. Stödda värden: `"mongodb"`, `"postgres"`, `"mysql"`, eller `"clickhouse"`.
</ParamField>

<ParamField path="--database-uri" type="string" alias="-u" required>
Anslutnings-URI för din databas:
- **MongoDB**: `mongodb://localhost:27017` eller `mongodb+srv://user:pass@cluster.mongodb.net/`
- **PostgreSQL**: `postgresql://user:password@localhost:5432/mydb`
- **MySQL**: `mysql://user:password@localhost:3306/mydb`
- **ClickHouse**: `http://localhost:8123`
</ParamField>

<ParamField path="--scopes" type="string" required>
Kommaavgränsad lista över dataenheter att synkronisera. Tillgängliga scopar: `licences`, `payments`, `customers`, `subscriptions`. Exempel: `"payments,customers"`.
</ParamField>

<ParamField path="--api-key" type="string" required>
Din Dodo Payments API-nyckel. Bör börja med `dp_live_` för live-läge eller `dp_test_` för testläge.
</ParamField>

<ParamField path="--env" type="string" required>
Målmiljö. Måste vara antingen `"live_mode"` eller `"test_mode"`. Detta bestämmer vilken Dodo Payments-miljö som ska synkas från.
</ParamField>

<ParamField path="--rate-limit" type="number" alias="--rl">
Hastighetsbegränsning i förfrågningar per sekund. Kontrollerar hur snabbt synkroniseringsmotorn gör API-anrop för att undvika att överbelasta Dodo Payments API.
</ParamField>

### Använda i Din Kod

För programmatisk kontroll, integrera synkroniseringsfunktionen direkt i din applikation. Installera det som ett beroende i ditt projekt:

<CodeGroup>

```bash npm
npm install dodo-sync
```

```bash bun
bun add dodo-sync
```

</CodeGroup>

#### Automatisk Synk (Intervallbaserad)

Använd automatisk synk när du vill att synkroniseringen ska köras kontinuerligt med regelbundna intervall:

```typescript
import { DodoSync } from 'dodo-sync';

const syncDodoPayments = new DodoSync({
  interval: 60, // Sync every 60 seconds
  database: 'mongodb',
  databaseURI: process.env.MONGODB_URI, // e.g., 'mongodb://localhost:27017'
  scopes: ['licences', 'payments', 'customers', 'subscriptions'],
  dodoPaymentsOptions: {
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    environment: 'test_mode' // or 'live_mode'
  }
});

// Initialize connection
await syncDodoPayments.init();

// Start the sync loop
syncDodoPayments.start();
```

<Tip>
Alternativet `interval` krävs när du använder `.start()` för automatisk synkning. Synkningen kommer att köras kontinuerligt med det angivna intervallet tills processen stoppas.
</Tip>

#### Manuell Synk

Använd manuell synk när du vill utlösa synkroniseringsoperationer på begäran (t.ex. från ett cron-jobb eller API-slutpunkt):

```typescript
import { DodoSync } from 'dodo-sync';

const syncDodoPayments = new DodoSync({
  database: 'mongodb',
  databaseURI: process.env.MONGODB_URI,
  scopes: ['licences', 'payments', 'customers', 'subscriptions'],
  dodoPaymentsOptions: {
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    environment: 'test_mode'
  }
});

// Initialize connection
await syncDodoPayments.init();

// Trigger a single sync operation
await syncDodoPayments.run();
```

<Tip>
När du använder manuell synkning krävs inte alternativet `interval`. Du kan anropa `.run()` när du behöver utföra en synkroniseringsoperation.
</Tip>

## Viktig Information

**MongoDB**: En databas med namnet `dodopayments_sync` kommer att skapas automatiskt på din databasserver. All synkdata lagras där. Detta databasnamn är för närvarande fast och kan inte ändras.

**PostgreSQL**: Tabeller (`Subscriptions`, `Payments`, `Licenses`, `Customers`) kommer att skapas i databasen som anges i din anslutnings-URI. Data lagras som JSONB.

**MySQL**: Tabeller (`Subscriptions`, `Payments`, `Licenses`, `Customers`) kommer att skapas i databasen som anges i din anslutnings-URI. Data lagras som JSON.

**ClickHouse**: Tabeller (`Subscriptions`, `Payments`, `Licenses`, `Customers`) kommer att skapas med ReplacingMergeTree-motorn. Vid frågor, använd nyckelordet `FINAL` för att säkerställa deduplicerade resultat.
</Warning>

<Info>
Synkroniseringsmotorn spårar ändringar och synkroniserar endast nya eller uppdaterade poster, vilket gör efterföljande synkningar effektiva även med stora datamängder.
</Info>
