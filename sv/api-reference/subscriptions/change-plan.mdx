## Hantering av betalningsfel

Använd `on_payment_failure`-parametern för att styra vad som händer när betalning för planändring misslyckas:

| Value | Behavior |
|-------|----------|
| `prevent_change` | Behåll prenumerationen på nuvarande plan tills betalningen lyckas. Planändringen förblir väntande. |
| `apply_change` | Tillämpa planändringen omedelbart oavsett betalningsresultat. Detta är standard. |

<Info>
Om `on_payment_failure` inte anges, återgår beteendet till din företagsinställning som konfigurerats i instrumentpanelen.
</Info>

<Tip>
Använd `prevent_change` för kritiska uppgraderingar där du vill säkerställa betalning innan åtkomst till premiumfunktioner ges.
</Tip>
