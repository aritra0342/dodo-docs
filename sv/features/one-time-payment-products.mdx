## Vad är engångsbetalningar?

Engångsbetalningar är fasta, förskottsbetalningar - idealiska för:

- **Programvarulicenser**: Livstids- eller större versionsupplåsningar
- **Digitala nedladdningar**: E-böcker, mallar, grafik eller plugins
- **Tjänster**: Granskningar, onboardingpaket eller engångskonsultationer
- **Kurser och biljetter**: Kurser, workshops eller live-sessioner

## Nyckelfördelar

- **Enkel kassa**: Inga förnyelser eller proportionell logik
- **Omedelbar uppfyllelse**: Leverera licenser eller filer direkt
- **Flexibel prissättning**: Rabatter och skatter stöds
- **Utvecklarvänlig**: Tydliga API:er för skapande, återbetalningar och webhooks

## Skapa engångsprodukter

Skapa produkter i din Dodo Payments-instrumentpanel, och sälj dem via värd-kassa, betalningslänkar eller ditt API.

### Produktkonfiguration

Konfigurera fälten i instrumentpanelen för att definiera hur din produkt visas och säljs.

#### Produktdetaljer

- **Produktnamn** (obligatoriskt): Visningsnamn som visas i kassan och fakturor
- **Produktbeskrivning** (obligatoriskt): Tydligt värdeuttalande som kunder ser innan de betalar
- **Produktbild** (obligatoriskt): PNG/JPG/WebP upp till 3 MB
- **Varumärke**: Koppla produkten till ett specifikt varumärke för temat och e-post
- **Skattekategori** (obligatoriskt): Välj rätt kategori för att bestämma skatteregler

<Tip>
Välj den mest korrekta skatte kategorin för att säkerställa korrekt skatteuppbörd per region.
</Tip>

#### Prissättning

- **Prissättningstyp**: Välj <b>Engångsbetalning</b>
- **Pris** (obligatoriskt): Fast avgift med valuta
- **Rabatt tillämplig (%)**: Valfri procentuell rabatt som visas i kassan och fakturor

<Warning>
Att ändra produktpriser påverkar endast nya köp. Befintliga kunder påverkas inte.
</Warning>

#### Avancerade inställningar

- **Skatteinkluderande prissättning**: Visa priser inklusive tillämpliga skatter
- **Generera licensnycklar**: Utfärda en unik nyckel efter köp. Se <a href="/features/license-keys">Licensnycklar</a>
- **Digital produktleverans**: Leverera filer eller innehåll automatiskt. Se <a href="/features/digital-product-delivery">Digital produktleverans</a>
- **Metadata**: Bifoga anpassade nyckel-värde-par för intern taggning eller integrationer. Se <a href="/api-reference/metadata">Metadata</a>

<Tip>
Använd metadata för att lagra identifierare från ditt system (t.ex. orderId) för att stämma av händelser och kvitton senare.
</Tip>

## Vanliga användningsfall

- **Licensbaserade verktyg**: Lås upp funktioner med en engångsnyckel
- **Digitala tillgångar**: Teman, mallar, ikonset eller media
- **Professionella tjänster**: Fast avgift för implementering eller installation
- **Utbildning**: Kursbiljetter och engångsworkshops

## Integreringsexempel

### Kassa-sessioner (engångsbetalning)
Skapa en kassa-session med en engångsprodukt i kundvagnen:

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    { product_id: 'prod_one_time', quantity: 1 }
  ]
});
```

## Bästa praxis

- **Klargör leveranser**: Berätta för köpare exakt vad de får och hur det levereras
- **Leverera omedelbart**: Använd licensnycklar eller digital leverans för omedelbar åtkomst
- **Spåra med metadata**: Lagra dina ID:n för enkel avstämning

<Info>
Engångsbetalningar är den snabbaste vägen till att sälja produkter med fast pris. Börja enkelt, verifiera leveransen och iterera ditt erbjudande baserat på konvertering och feedback.
</Info>
