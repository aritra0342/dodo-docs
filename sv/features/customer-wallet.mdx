## What are Customer Wallets?

Kundplånböcker är konton för penningbalanser som håller riktiga medel åt dina användare. Varje kund får en automatiskt när du skapar deras konto. Du kan använda dessa plånböcker till att:

- **Lagra förbetalda pengar** för framtida prenumerationsbetalningar
- **Hantera återbetalningar** som plånbokssaldo istället för kortåterbetalningar
- **Utfärda kampanjbalanser** som välkomstbonusar eller lojalitetsbelöningar
- **Tillämpa plånboksmedel på fakturor** automatiskt vid fakturering
- **Spåra penningtransaktioner** med en detaljerad historik

<Info>
Varje kund får automatiskt en plånbok när du skapar deras konto. Plånböcker stöder USD- och INR-valutor med separata penningbalanser för varje.
</Info>

<Warning>
**Kundplånböcker ≠ Kreditbaserad fakturering**

Kundplånböcker håller **riktiga penningbalanser** (USD, INR) som kan tillämpas på fakturor och prenumerationsbetalningar.

Om du vill spåra virtuella användningsenheter (API-anrop, token, beräkningstimmar) se [Credit-Based Billing](/features/credit-based-billing) istället.
</Warning>

## Hur det fungerar

Kundplånböcker håller medel som kunderna kan använda för sina köp. När en kund har en faktura eller en återkommande prenumerationsavgift kontrolleras deras plånbokssaldo först. Alla tillgängliga medel används automatiskt till fakturan innan deras primära betalningsmetod belastas.

### Automatisk konfiguration

När du skapar en ny kund skapar Dodo Payments automatiskt en plånbok med nollsaldo. Den är redo att ta emot medel omedelbart via vårt API.

### Fler valutor

Varje plånbok kan hålla saldon i olika valutor:

<ResponseField name="USD Balance" type="integer">
Saldo i amerikanska dollar (lagrat i cent)
</ResponseField>

<ResponseField name="INR Balance" type="integer">
Saldo i indiska rupier (lagrat i paise)
</ResponseField>

<Info>
För närvarande finns endast **USD**- och **INR**-saldon tillgängliga. Fler valutor kommer snart.
</Info>

## Verkliga exempel

### Återbetalning till plånbok

När en kund begär en återbetalning kan du lägga till beloppet i deras plånbokssaldo istället för att göra en traditionell kortåterbetalning. Detta håller pengarna inom ditt ekosystem för framtida köp.

```javascript
async function refundToWallet(customerId, refundAmount, originalPaymentId) {
  await client.customers.wallets.ledgerEntries.create(customerId, {
    amount: refundAmount, // Amount in cents
    currency: 'USD',
    entry_type: 'credit',
    reason: `Refund for payment ${originalPaymentId}`,
    idempotency_key: `refund_${originalPaymentId}`
  });
}
```

### Välkomstbonus / kampanjbalans

Ge nya kunder en monetär välkomstbonus för att uppmuntra deras första köp.

```javascript
async function addWelcomeBonus(customerId) {
  await client.customers.wallets.ledgerEntries.create(customerId, {
    amount: 1000, // $10.00 promotional balance
    currency: 'USD',
    entry_type: 'credit',
    reason: 'Welcome bonus - $10 promotional balance',
    idempotency_key: `welcome_${customerId}`
  });
}
```

### Prenumerationsbetalning från plånbok

Dra av pengar från en plånbok för att täcka en prenumerationsavgift eller ett manuellt köp.

```javascript
async function deductForPurchase(customerId, purchaseAmount, purchaseId) {
  try {
    await client.customers.wallets.ledgerEntries.create(customerId, {
      amount: purchaseAmount,
      currency: 'USD',
      entry_type: 'debit',
      reason: `Subscription charge - monthly billing`,
      idempotency_key: `charge_${purchaseId}`
    });
  } catch (error) {
    if (error.status === 400) {
      console.log('Insufficient wallet balance');
    }
  }
}
```

## Bästa praxis

### Förhindra dubbla transaktioner

Använd idempotensnycklar för att säkerställa att du inte av misstag lägger till eller drar av pengar två gånger för samma händelse.

```javascript
async function addFundsSafely(customerId, amount, reason) {
  const idempotencyKey = `${reason}_${customerId}_${Date.now()}`;
  
  try {
    const result = await client.customers.wallets.ledgerEntries.create(customerId, {
      amount: amount,
      currency: 'USD',
      entry_type: 'credit',
      reason: reason,
      idempotency_key: idempotencyKey
    });
    
    return { success: true, wallet: result };
  } catch (error) {
    if (error.status === 409) {
      // Transaction already processed
      return { success: true, wallet: null, duplicate: true };
    }
    
    throw error;
  }
}
```

### Kontrollera saldon innan debitering

Verifiera att kunderna har tillräckliga medel innan du försöker behandla stora transaktioner från plånboken.

```javascript
async function checkBalanceBeforeOperation(customerId, requiredAmount) {
  const wallets = await client.customers.wallets.list(customerId);
  const usdWallet = wallets.items.find(w => w.currency === 'USD');
  
  if (!usdWallet || usdWallet.balance < requiredAmount) {
    throw new Error('Insufficient funds for this operation');
  }
  
  return usdWallet.balance;
}
```

## Vad som kommer härnäst

<Warning>
Dessa funktioner är planerade för framtida släpp:
</Warning>

- **Saldoutgång**: Ställ in så att pengar upphör efter en viss tidsperiod
- **Bättre analys**: Detaljerade utgiftsrapporter och balansutveckling
- **Fler webhooks**: Realtidsaviseringar om balansförändringar och låg balans-varningar

<Tip>
Börja med grundläggande finansierings- och avdragsoperationer, och integrera sedan mer komplexa automatiserade faktureringsarbetsflöden när ditt företag växer.
</Tip>
