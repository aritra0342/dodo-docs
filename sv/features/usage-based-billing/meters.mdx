<Info>
Mätare omvandlar råa händelser till fakturerbara kvantiteter. De filtrerar händelser och tillämpar aggregationsfunktioner (Räkna, Summa, Max, Senast) för att beräkna användning per kund.
</Info>

## Skapa en Mätare

<Steps>
<Step title="Grundläggande Information">
<ParamField path="Mätarnamn" type="string" required>
Beskrivande namn (t.ex. "API-förfrågningar", "Tokenanvändning")
</ParamField>

<ParamField path="Event Name" type="string" required>
Exakt händelsenamn att matcha (skiftlägeskänslig). Exempel: `api.call`, `image.generated`
</ParamField>
</Step>

<Step title="Aggregat">
<ParamField path="Aggregationstyp" type="string" required>
Välj hur händelser aggregeras:

- **Räkna**: Totalt antal händelser (API-anrop, uppladdningar)
- **Summa**: Lägg till numeriska värden (tokens, byte)
- **Max**: Högsta värde under perioden (top-användare)
- **Senast**: Senaste värde
</ParamField>

<ParamField path="Over Property" type="string">
Metadata-nyckel att aggregera (obligatorisk för alla typer utom Count). Exempel: `tokens`, `bytes`, `duration_ms`
</ParamField>

<ParamField path="Measurement Unit" type="string" required>
Enhetsbeteckning för fakturor. Exempel: `calls`, `tokens`, `GB`, `hours`
</ParamField>
</Step>

<Step title="Filtrering (Valfritt)">
<Frame>
<img src="/images/usage-based/UBB-3.png" alt="Händelsefiltrering" />
</Frame>

Lägg till villkor för att filtrera vilka händelser som räknas:
- **OCH-logik**: Alla villkor måste matcha
- **ELLER-logik**: Vilket som helst villkor kan matcha

**Jämförare**: lika med, inte lika med, större än, mindre än, innehåller

Aktivera filtrering, välj logik, lägg till villkor med egenskapsnyckel, jämförare och värde.
</Step>

<Step title="Skapa">
Granska konfigurationen och klicka på **Skapa Mätare**.
</Step>
</Steps>

## Felsökning

<AccordionGroup>
<Accordion title="Händelser visas inte">
- Händelsenamn måste matcha exakt (skiftlägeskänsligt)
- Kontrollera att mätarfiltret inte utesluter händelser
- Verifiera att kund-ID:n finns
- Temporärt inaktivera filter för att testa
</Accordion>

<Accordion title="Aggregering fungerar inte">
- Verifiera att Over Property matchar metadata-nyckeln exakt
- Använd siffror, inte strängar: `tokens: 150` inte `"150"`
- Inkludera obligatoriska egenskaper i alla händelser
</Accordion>

<Accordion title="Filter fungerar inte">
- Matcha skiftläge exakt
- Använd korrekta operatorer för datatyp
- Se till att händelser inkluderar filtrerade egenskaper
</Accordion>

<Accordion title="Felaktiga användningstotaler">
- Kontrollera Händelser-fliken för att räkna faktiska händelser som mottagits
- Verifiera aggregationstyp (Räkna vs Summa)
- Se till att värden är numeriska för Summa/Max
</Accordion>
</AccordionGroup>
