Ingestionsblåkopior är förbyggda mönster för att spåra användning i vanliga scenarier. Istället för att ta reda på händelsestrukturer och mätkonfigurationer från grunden, börja med en beprövad blåkopi och anpassa den efter dina behov.

## Vad är blåkopior?

Varje blåkopi inkluderar:
- **Rekommendationer för händelsestruktur**: Optimal namngivning av händelser och design av metadata
- **Mätkonfiguration**: Föreslagna aggregeringstyper och filter
- **Exempel på kod**: Funktionella implementations exempel i flera språk
- **Bästa praxis**: Tips för noggrannhet, prestanda och kostnadsoptimering

<Tip>
Blueprints sparar timmar av uppsättningstid och hjälper dig undvika vanliga fallgropar. Börja med en blueprint och anpassa den sedan efter dina specifika krav.
</Tip>

## Jämförelse av blåkopior

Osäker på vilken blåkopi som passar dina behov? Här är en snabb jämförelse:

| Användningsfall | Blåkopi | Aggregering | Vanlig metadata |
|----------|-----------|-------------|-----------------|
| AI chattapp | LLM | Summa (tokens) | modell, inmatningstokens, utmatningstokens |
| REST API | API Gateway | Antal | endpoint, metod, statuskod |
| Fil lagring | Objektlagring | Summa (bytes) | filstorlek, filtyp, operation |
| Videoplattform | Stream | Summa (bandbredd) | längd, kvalitet, bitrate |
| Serverlös | Tidsintervall | Summa (längd) | funktionsnamn, exekveringstid_ms |

## Anpassade implementationer

Ser du inte en blåkopi för ditt exakta användningsfall? Inga problem:

1. **Börja med den närmaste blåkopin**: Välj den som är mest lik dina behov
2. **Anpassa händelsestrukturen**: Ändra metadatafält för att matcha dina krav
3. **Justera mätkonfigurationen**: Ändra aggregeringstyp eller filter vid behov
4. **Testa noggrant**: Verifiera faktureringsberäkningar med exempelanvändning

<Info>
Alla blueprints är utgångspunkter. Känn dig fri att blanda koncept från olika blueprints eller skapa helt egna implementationer.
</Info>
