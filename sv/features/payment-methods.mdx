Dodo Payments stöder ett brett utbud av betalningsmetoder för att hjälpa dig att betjäna kunder globalt. Alla listade betalningsmetoder blir automatiskt tillgängliga när ditt säljarkonto är verifierat och live-läget är aktiverat.

<Info>
**Automatisk aktivering:** Ingen ytterligare konfiguration behövs från din sida. Alla betalningsmetoder är förkonfigurerade och hanteras av Dodo Payments. Du kan valfritt begränsa vilka metoder som visas vid kassan med hjälp av API-konfigurationen som beskrivs nedan.
</Info>

<Warning>
**PayPal-support pausad:** Stödet för PayPal-betalningsmetoden har tillfälligt pausats. PayPal är för närvarande inte tillgängligt för nya transaktioner. Vi kommer att uppdatera denna sida när PayPal-stödet återställs.
</Warning>

<Note>
Varje transaktion måste uppfylla det <strong>minimi belopp som krävs för varje valuta</strong>. För USD är minimum <strong>0,50 USD</strong>. Minimum för andra valutor varierar beroende på region.
[Se stödda valutor &amp; minimum](/features/adaptive-currency#supported-currencies)
</Note>

<Note>
Klarna och Afterpay kräver ett minimi transaktionsbelopp på <strong>50,01 USD</strong> (eller motsvarande i stödda valutor) för berättigande vid kassan. Transaktioner under denna gräns kommer inte att vara berättigade för dessa BNPL-betalningsalternativ.

Som standard visas alla berättigade betalningsmetoder automatiskt. Använd `allowed_payment_method_types` för att begränsa vilka metoder som visas:

<Tabs>
{/* LOCKED_PATTERN_7202147f6fc990d9a8a4b3a4cdc10768 */}

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Step title="Lägg till parameter i API-förfrågan">
Inkludera `allowed_payment_method_types` fältet i din betalnings- eller prenumerationsskapande förfrågan:

| Typ | Beskrivning |
| :--- | :---------- |
| `credit` | Kreditkort (Visa, Mastercard, Amex etc.) |
| `debit` | Betalkort |
| `apple_pay` | Apple Pay |
| `google_pay` | Google Pay |
| `amazon_pay` | Amazon Pay |
| `cashapp` | Cash App Pay |
| `klarna` | Klarna BNPL |
| `afterpay_clearpay` | Afterpay / Clearpay |
| `upi_collect` | UPI Collect (QR-kod) |
| `ideal` | iDEAL (Nederländerna) |
| `bancontact_card` | Bancontact (Belgien) |
| `eps` | EPS (Österrike) |
| `multibanco` | Multibanco (Portugal) |
| `revolut_pay` | RevolutPay |
| `billie` | Billie B2B BNPL |

<Warning>
Inkludera alltid `credit` och `debit` som reservalternativ. Om alla angivna betalningsmetoder är otillgängliga för en kund misslyckas köpet.
</Warning>

### API-slutpunkter

<AccordionGroup>
{/* LOCKED_PATTERN_63da960e31e014d866efe70101f28b99 */}
Inkludera regionala betalningsmetoder för dina målmarknader. iDEAL står för ~60 % av onlinebetalningarna i Nederländerna; UPI dominerar i Indien. Att utelämna dessa metoder skadar konverteringen avsevärt.
</Accordion>

{/* LOCKED_PATTERN_78f59c035688fa14218708818c9f0fca */}
Även när du riktar in dig på specifika metoder, inkludera `credit` och `debit` som reservalternativ. Inte alla kunder har tillgång till varje betalningsmetod.
</Accordion>

{/* LOCKED_PATTERN_564640b27644e47b9e4e7faaec6c2d88 */}
BNPL-metoder (Klarna, Afterpay) kräver minst 50,01 USD. Inkludera dem inte för lågprissatta produkter – de visas helt enkelt inte, vilket är okej.
</Accordion>

{/* LOCKED_PATTERN_ea5461a635a5291807627a6b40d7922e */}
Använd testläge för att verifiera tillgängligheten för betalningsmetoder för olika kundplatser, valutor och enhetstyper innan du går live.
</Accordion>
</AccordionGroup>

<AccordionGroup>
{/* LOCKED_PATTERN_fbf2d38e7725fb97384bef01324b59a1 */}
**Kontrollera dessa faktorer:**
1. Ingår metoden i `allowed_payment_method_types`?
2. Matchar kundens plats metodens stödjda region?
3. Är transaktionsvalutan stödd av metoden?
4. Uppfyller transaktionsbeloppet minimi (t.ex. 50,01 USD för BNPL)?
5. För prenumerationer: stödjer metoden återkommande betalningar?

**Lösning:** Ta tillfälligt bort `allowed_payment_method_types` för att se alla berättigade metoder, och smalna sedan av problemet.
</Accordion>

{/* LOCKED_PATTERN_57385d59bd851fb1e8c26266b489efd4 */}
**Möjliga orsaker:**
- `allowed_payment_method_types` skickas inte korrekt
- Parametern formaterad felaktigt (måste vara en array med strängar)

**Lösning:** Logga din API-förfrågan och verifiera parameterstrukturen.
</Accordion>

{/* LOCKED_PATTERN_6d1a1b1a59cbe2789bbc9434eb1ec153 */}
**Kontrollera:**
- Kundens faktureringsadress matchar metodens region
- Valutan är korrekt inställd (t.ex. EUR för iDEAL)
- Adaptiv valuta är aktiverad om kunden är internationell

**Lösning:** Verifiera landkod för faktureringsadress och valuta i din API-förfrågan.
</Accordion>

{/* LOCKED_PATTERN_52f3532b80d2def876ef3cc78b55f9f6 */}
UPI- och Rupay-prenumerationer har unika krav som inkluderar 48-timmarsbearbetningsfördröjningar och RBI-mandatgränser.

**Se:** <a href="/features/payment-methods/india">India Payment Methods</a> för komplett vägledning.
</Accordion>
</AccordionGroup>

<Accordion title="Standardbeteende">
Om `allowed_payment_method_types` inte anges, kommer alla berättigade betalningsmetoder att vara tillgängliga baserat på kundens plats, valuta och transaktionstyp. Använd endast denna parameter när du behöver begränsa betalningsmetoder.
</Accordion>
