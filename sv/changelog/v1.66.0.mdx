## Nya funktioner 🚀

1. **`subscription.updated` Webhook**  
   En ny webhook-händelse som triggas vid varje uppdatering av ett prenumerationsobjekt, vilket ger dig realtidsnotiser om alla ändringar i dina prenumerationer.
   
   Använd denna webhook för att:
   - Få omedelbara notiser när prenumerationsuppgifter ändras
   - Hålla din applikationsstatus synkroniserad med Dodo Payments
   - Reagera på prenumerationsuppdateringar utan att pollera API:et
   
   <Tip>
   Prenumerera på `subscription.updated` för att eliminera behovet av pollning och hålla ditt system synkat med realtidsändringar i prenumerationer.
   </Tip>
   
   Läs mer: [Prenumerationswebhooks](/developer-resources/webhooks/intents/subscription)

2. **Minimal adressstöd i kassan**  

   <Frame>
   <img src="/images/changelog/minimal-address.png" alt="Stöd för minimal adress i kassan" />
   </Frame>

   Minska friktionen i kassan med det nya läget för minimal adressinsamling. Istället för att samla in fullständiga adressuppgifter samlar läget för minimal adress endast in:
   
   - **Land**: Alltid nödvändigt för skatteberäkning
   - **Postnummer**: Samlas endast in i regioner där det krävs för beräkning av försäljningsskatt, moms eller GST
   
   <Check>
   Resultat: Betydligt snabbare kassaprocess med färre formulärfält för kunderna.
   </Check>
   
   **Aktivera minimal adress:**
   
   <Tabs>
   {/* LOCKED_PATTERN_0090a508f27e87d00759ddcd3b9fe94c */}

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [{ product_id: 'prod_123', quantity: 1 }],
     minimal_address: true
   });
   ```

   </Tab>
   
   {/* LOCKED_PATTERN_ed987f4815610e04ce90e6f11a8998dc */}
   Lägg till `minimalAddress=true` i din betalningslänk-URL:
   

   ```
   https://checkout.dodopayments.com/buy/prod_123?minimalAddress=true
   ```

   </Tab>
   </Tabs>
   
   <Info>
   Som standard fortsätter kassan att samla in kompletta adresser. Vi kommer gradvis att överföra alla kassalänkar till läget för minimal adress. Aktivera funktionsflaggan nu för att börja dra nytta av minskad friktion i kassan.
   </Info>

3. **Endast läsbehörighet för API-nycklar**  
   Skapa API-nycklar med skrivskyddade behörigheter för säkrare integrationer. När du skapar en ny API-nyckel avmarkerar du **Enable write access** för att skapa en skrivskyddad nyckel.
   
   Skrivskyddade nycklar kan:
   - Hämta betalnings-, prenumerations- och kunddata
   - Lista produkter och hämta detaljer
   - Komma åt rapporter och analyser
   
   Skrivskyddade nycklar **kan inte**:
   - Skapa eller ändra betalningar
   - Uppdatera prenumerationer
   - Ändra produktkonfigurationer
   
   <Tip>
   Avmarkera "Enable write access" för instrumentpanelintegrationer, analyssystem och alla system som bara behöver visa data utan att göra ändringar.
   </Tip>

4. **UPI AutoPay och RuPay-kortprenumerationer – full utrullning**  
   UPI AutoPay och RuPay-kortprenumerationer är nu tillgängliga för alla företag. Detta möjliggör återkommande betalningar för indiska kunder med:
   
   - **UPI AutoPay**: Automatiska återkommande avgifter via UPI-mandat
   - **RuPay-kort**: Prenumerationsstöd för Indiens inhemska kortnätverk
   
   <Check>
   Resultat: Utökade betalningsalternativ för prenumerationer på den indiska marknaden med RBI-kompatibel hantering av mandat.
   </Check>
   
   Läs mer: [Prenumerationer med RBI-kompatibla mandat](/features/subscription#subscriptions-with-rbi-compliant-mandates)

5. **API för förhandsgranskning av planändringar**  
   En ny API-slutpunkt för att förhandsgranska ändringar av prenumerationsplaner innan du åtar dig dem. Få den exakta avgiftsbeloppet och se hur prenumerationen kommer att se ut efter ändringen - utan att göra några faktiska modifieringar.
   

   ```javascript
   const preview = await client.subscriptions.previewChangePlan('sub_123', {
     product_id: 'prod_pro_plan',
     proration_billing_mode: 'prorated_immediately',
     quantity: 1
   });
   
   console.log('Charge amount:', preview.immediate_charge.summary);
   console.log('New plan details:', preview.new_plan);
   ```

   
   Använd detta för att:
   - Visa kunder exakt vad de kommer att betala innan de bekräftar uppgraderingar/nedgraderingar
   - Bygga priskalkylatorer och planjämförelseverktyg
   - Minska supportärenden genom att förbättra pristransparensen
   
   Läs mer: [API för förhandsgranskning av planändring](/api-reference/subscriptions/preview-change-plan)
