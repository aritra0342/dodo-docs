## Nya funktioner 🚀

1. **Mixed Cart Checkout: Engångs- och prenumerationsprodukter i en enda kundvagn**  
   Checkout stödjer nu att man kombinerar både engångsprodukter och prenumerationsprodukter i en enda kundvagn, vilket öppnar upp kraftfulla nya faktureringsscenarier.
   
   <Frame>
   <img src="/images/changelog/mixed-checkout.png" alt="Mixed checkout with one-time and subscription products" />
   </Frame>
   
   ### Vad detta möjliggör
   
   - **Installationsavgifter + prenumerationer**: Ta ut en engångsinställningsavgift tillsammans med en återkommande prenumeration i ett enda kassaflöde
   - **Paket med återkommande komponenter**: Sälj en produkt (engångs) ihop med tillgång till en pågående tjänst (prenumeration)
   - **Kurs- och medlemskombinationer**: Erbjud ett kursköp med en valfri prenumeration på community-medlemskap
   - **Livstid + tilläggsprenumerationer**: Kombinera livstidsåtkomst till en produkt med valbara återkommande premiumfunktioner
   
   ### Så fungerar det
   
   Lägg helt enkelt till både engångs- och prenumerationsprodukter i din kassasession:
   

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [
       // One-time setup fee
       { product_id: 'prod_setup_fee', quantity: 1 },
       // Monthly subscription
       { product_id: 'prod_monthly_plan', quantity: 1 }
     ],
     customer: {
       email: 'customer@example.com',
       name: 'Jane Doe'
     },
     return_url: 'https://yourapp.com/success'
   });
   ```

   
   <Check>
   Resultat: Ett enda kassaflöde som hanterar engångsbetalningen och initierar prenumerationen — vilket minskar friktion och ökar konverteringsgraden.
   </Check>
   
   <Info>
   När du blandar produkttyper debiteras engångsbetalningen omedelbart medan prenumerationen följer sin konfigurerade faktureringscykel (inklusive eventuell testperiod du har angett).
   </Info>
   
   Läs mer: [Checkout Sessions](/developer-resources/checkout-session)

2. **Unified Customer Portal**  
   
   En ny centraliserad kundportal på [customer.dodopayments.com](https://customer.dodopayments.com) där kunder kan se och hantera alla sina köp och prenumerationer från olika företag på Dodo Payments.
   
   <Frame>
   <img src="/images/changelog/unified-customer-portal.png" alt="Unified Customer Portal" />
   </Frame>
   
   ### Portalens funktioner
   
   - **Övergripande företagsöversikt**: Visa alla köp och prenumerationer från varje företag som använder Dodo Payments på ett och samma ställe
   - **Centraliserad prenumerationshantering**: Hantera prenumerationer över olika handlare från en enda instrumentpanel
   - **Enhetlig fakturahistorik**: Kom åt fakturor och betalningshistorik från alla dina köp
   - **Single sign-on-upplevelse**: Logga in en gång med din e-post för att komma åt köp från alla Dodo Payments-handlare
   
   ### Så fungerar det
   
   Kunder besöker helt enkelt [customer.dodopayments.com](https://customer.dodopayments.com) och anger sin e-postadress. De får en säker inloggningslänk som ger tillgång till alla deras köp inom Dodo Payments-ekosystemet.
   
   <Tip>
   Unified Customer Portal kompletterar företagsunika kundportaler. Kunder kan fortfarande besöka individuella handlarportaler för en varumärkesanpassad upplevelse eller använda den enhetliga portalen för en samlad översikt.
   </Tip>
   
   <Note>
   Den enhetliga portalen använder samma e-postbaserade autentisering som företagsunika portaler, vilket säkerställer en konsekvent och säker upplevelse.
   </Note>

3. **Invoice ID in Payment Response**  
   Betalningssvaret innehåller nu ett `invoice_id`-fält, vilket gör det enklare att stämma av betalningar med fakturor och integrera med ekonomisystem.
   

   ```json
   {
     "payment_id": "pay_2IjeQm4hqU6RA4Z4kwDee",
     "status": "succeeded",
     "invoice_id": "inv_2IsUnWGtRKFLxk7xAQeyt",
     "amount": 9900,
     "currency": "USD",
     // ... other fields
   }
   ```

   
   ### Användningsfall
   
   - **Redovisningsintegration**: Koppla betalningar direkt till fakturor i ditt redovisningsprogram
   - **Kundsupport**: Hitta snabbt fakturan som är kopplad till en betalning när du hanterar supportärenden
   - **Avstämning**: Förenkla den ekonomiska avstämningen genom att matcha betalningar mot fakturor programmässigt
   - **Efterlevnad**: Behåll korrekta register som länkar betalningar till deras motsvarande skattefakturor
   
   <Info>
   `invoice_id`-fältet använder Indien-specifika faktura-ID:n när sådana finns tillgängliga, vilket säkerställer efterlevnad av lokala faktureringskrav.
   </Info>
   
   Läs mer: [Get Payment Detail API](/api-reference/payments/get-payments-1)
