## Vad är en prenumerationsmigration?

En prenumerationsmigration överför dina befintliga prenumeranter från din gamla betalningsprocessor till Dodo Payments, vilket ger dig en enda källa för all din kunddata. Utan en migration förblir kunder som du har förvärvat hittills i ditt gamla system, och endast nya kunder skulle vara synliga i Dodo Payments.

Migrationsprocessen säkerställer en sömlös faktureringsupplevelse för dina kunder—de behöver inte ange sina betalningsuppgifter på nytt eller uppleva några avbrott i sina prenumerationer.

<Warning>
För närvarande stöder vi endast migration för **korttoken**. Apple Pay, Google Pay och andra plånbok-betalningsmetoder kan inte överföras mellan betalningsleverantörer på grund av de tokeniseringsbegränsningar som dessa leverantörer ställer.
</Warning>

## Översikt av migrationsprocessen

Prenumerationsmigrationsprocessen består av tre nyckeldelar:

1. **Överföring av betalningsmetod** — Säker överföring av känsliga kortdata från din nuvarande processor
2. **Överföring av prenumerantdata** — Export av kunduppgifter, prenumerationsplaner och faktureringsscheman
3. **Import och validering** — Sammanfoga betalningsmetoder och prenumerantdata i Dodo Payments

<Note>
För att påbörja en migrationsprocess för prenumerationer, kontakta vårt [supportteam](mailto:support@dodopayments.com). De kommer koppla ihop dig med en dedikerad Solutions Engineer som guidar dig genom hela processen.
</Note>

## Krav

Innan du påbörjar en migration, se till att du uppfyller följande krav:

<AccordionGroup>
<Accordion title="PCI DSS compliance verification">
Dodo Payments är PCI DSS Level 1-kompatibelt. För att uppfylla efterlevnadskraven kan vi endast ta emot kortdata från en annan betalningsleverantör som också uppfyller PCI DSS Level 1.

Din nuvarande processor måste tillhandahålla:
- Deras aktuella PCI Attestation of Compliance (AOC), eller deras listning på Visas globala register över tjänsteleverantörer
- Bekräftelse på att de stöder säkra kortdataexporter

Visa vårt [Attestation of Compliance (AOC)](https://pub-2a89184e4c954b599464449d23f506bb.r2.dev/dodo-payments-pci-dss-aoc.pdf) för att dela med din nuvarande leverantör.
</Accordion>

<Accordion title="Dodo Payments integration">
Slutför din integration med Dodo Payments och säkerställ att alla nya kunder dirigeras till Dodo Payments kassasida innan du startar migrationen. Detta förhindrar datainkonsekvenser under övergångsperioden.
</Accordion>

<Accordion title="Data access authorization">
Säkerställ att du har nödvändig auktorisation för att begära en dataexport från din nuvarande leverantör. Detta kräver vanligtvis:
- Verifiering av kontoägare
- Skriftlig auktorisation för åtkomst till känslig data
- Slutförande av din leverantörs säkerhetsverifieringssteg
</Accordion>
</AccordionGroup>

## Steg-för-steg migrationsguide

<Steps>
<Step title="Request payment method export from your current processor">
Kontakta supportteamet hos din nuvarande betalningsleverantör och lämna en formell begäran om en fullständig export av kundernas korttoken.

Du behöver:
- Tillhandahålla verifiering av din identitet och behörighet
- Begära att de krypterar exporten med Dodo Payments PGP offentliga nyckel (anges nedan)
- Be dem att överföra den krypterade filen direkt till vår säkra SFTP-server

<Info>
Säkra överföringar sker direkt server till server via SFTP. Din Solutions Engineer kommer att tillhandahålla SFTP-uppgifterna som du delar med din nuvarande leverantör.
</Info>

I genomsnitt tar den här processen cirka **2 veckor**, även om tidsramen varierar beroende på din nuvarande leverantörs policyer och svarstider.
</Step>

<Step title="Prepare subscriber data export">
Medan överföringen av betalningsmetoder pågår, exportera din prenumerantdata från ditt gamla system. Din Solutions Engineer kommer att ge dig en CSV-mall och vägleda dig genom de obligatoriska fälten.

### Nödvändig prenumerantdata

| Field | Description |
|-------|-------------|
| `customer_id` | Ditt interna kund-ID |
| `email` | Kundens e-postadress |
| `name` | Kundens fullständiga namn |
| `country` | Tvåbokstavskod för land (ISO 3166-1 alpha-2) |
| `currency` | Trehjärnig valutakod (ISO 4217) |
| `payment_amount` | Prenumerationsbelopp i minsta valutaanhet |
| `payment_interval` | Faktureringsfrekvens (månatlig, årlig etc.) |
| `next_billing_date` | Nästa planerade faktureringsdatum (ISO 8601) |
| `card_token` | Unik token från din nuvarande leverantör |

<Warning>
Fältet `card_token` är avgörande—det används för att matcha prenumerantposter med deras betalningsmetoder under importen. Se till att detta matchar exakt token i exporten av betalningsmetoder.
</Warning>

### Exempel på dataformat

```json
{
  "customers": [
    {
      "customer_id": "cus_abc123",
      "email": "jane.doe@example.com",
      "name": "Jane Doe",
      "country": "US",
      "currency": "USD",
      "payment_amount": 2999,
      "payment_interval": "monthly",
      "next_billing_date": "2025-01-15T00:00:00Z",
      "card_token": "tok_xyz789",
      "metadata": {
        "plan": "pro",
        "signup_source": "website"
      }
    }
  ]
}
```

</Step>

<Step title="Validation and testing">
Innan importdatumet kommer vårt team att:

- Verifiera filens integritet och kryptering av betalningsmetodens data
- Validera din prenumerant-CSV för korrekt formatering
- Köra en testimport i vår sandbox-miljö
- Identifiera och lösa eventuella datakartläggningsproblem

<Check>
Du kommer att få en valideringsrapport som bekräftar att allt är redo för produktionsimporten.
</Check>
</Step>

<Step title="Production import">
På det överenskomna importdatumet kommer vårt tekniska team att:

1. Importera betalningsmetodens data till vårt PCI-kompatibla Card Vault
2. Importera dina prenumerantdata
3. Matcha prenumeranter med deras betalningsmetoder med hjälp av korttoken
4. Aktivera alla migrerade prenumerationer

<Info>
Typisk behandlingstid är **2-3 arbetsdagar** för standardmigreringar, beroende på antalet prenumeranter.
</Info>
</Step>

<Step title="Post-migration verification">
Efter att importen slutförts kommer du att få:

- En omfattande migrationsrapport
- En CSV som kartlägger dina gamla prenumerations-ID:n till nya Dodo Payments-ID:n
- Tillgång till alla migrerade kunder i din Dodo Payments-instrumentpanel

Vi rekommenderar att du:
- Verifierar ett urval av migrerade prenumerationer i din kontrollpanel
- Testar några prenumerationsförnyelser i sandbox-läge
- Uppdaterar interna databassreferenser med de nya ID:na
- Kör parallell övervakning under den första faktureringscykeln
</Step>
</Steps>

## Migrerbar data

Följande data kan överföras under en prenumerationsmigration:

| Datatyp | Stöds | Anteckningar |
|-----------|-----------|-------|
| Kredit-/betalkortnummer | ✅ | Krypterat via PGP |
| Kortets utgångsdatum | ✅ | |
| Kortinnehavarens namn | ✅ | |
| Faktureringsadresser | ✅ | |
| Kundens e-postadresser | ✅ | |
| Kundmetadata | ✅ | Anpassade nyckel-värdepar |
| Prenumerationsdetaljer | ✅ | Plan, belopp, intervall |
| Apple Pay-token | ❌ | Kan inte överföras |
| Google Pay-token | ❌ | Kan inte överföras |
| Andra plånbokstoken | ❌ | Kan inte överföras |
| Betalningshistorik | ❌ | Exportera från din nuvarande processor |

<Note>
Betalningshistorik och transaktionsregister kan inte migreras. Vi rekommenderar att du exporterar denna data från din nuvarande leverantör för dina arkiv innan du slutför övergången.
</Note>

## Tidslinje för förväntningar

| Fas | Varaktighet |
|-------|----------|
| Inledande installation och planering | 1-2 dagar |
| Överföring av betalningsmetod | ~2 veckor |
| Förberedelse av prenumerantdata | 3-5 dagar (parallellt) |
| Validering och testning | 2-3 dagar |
| Produktionsimport | 2-3 arbetsdagar |
| Verifiering efter migration | 1-2 dagar |

<Tip>
Den totala migreringen tar vanligtvis **3-4 veckor** från start till mål. Att börja förbereda prenumerantdatan medan överföringen av betalningsmetoder pågår hjälper till att minimera den totala tidsramen.
</Tip>
