## Introduktion

Skicka transaktionse-post och hantera kundkommunikation automatiskt när betalningshändelser inträffar. Leverera betalningsbekräftelser, abonnemangsuppdateringar och viktiga meddelanden med Loopars e-postinfrastruktur.

<Info>
Denna integration kräver din Loops API-nyckel för autentisering.
</Info>

## Komma igång

<Steps>
<Step title="Open the Webhook Section">
  I din Dodo Payments-instrumentpanel, navigera till <b>Webhooks → + Add Endpoint</b> och expandera integrations-rullgardinsmenyn.
  <Frame>
    <img src="/images/integrations/loops.png" alt="Add Endpoint and integrations dropdown" />
  </Frame>
</Step>

<Step title="Select Loops">
  Välj <b>Loops</b>-integrationskortet.
</Step>

<Step title="Enter API Key">
  Ange din Loops API-nyckel i konfigurationen.
</Step>

<Step title="Configure Transformation">
  Redigera omvandlingskoden för att formatera e-postmeddelanden för Loops API.
</Step>

<Step title="Test & Create">
  Testa med exempeldata och klicka på <b>Skapa</b> för att aktivera e-postutskicket.
</Step>

<Step title="Done!">
  🎉 Betalningsevenemang kommer nu automatiskt att utlösa transaktionella e-postmeddelanden via Loops.
</Step>
</Steps>

## Exempel på transformationskod

### Betalningsbekräftelse-e-post

```javascript payment_confirmation.js icon="js" expandable
function handler(webhook) {
  if (webhook.eventType === "payment.succeeded") {
    const p = webhook.payload.data;
    webhook.url = "https://api.loops.so/v1/events/send";
    webhook.payload = {
      eventName: "payment_confirmation",
      email: p.customer.email,
      properties: {
        customer_name: p.customer.name,
        payment_id: p.payment_id,
        amount: (p.total_amount / 100).toFixed(2),
        currency: p.currency || "USD",
        payment_method: p.payment_method || "unknown",
        payment_date: new Date(webhook.payload.timestamp).toLocaleDateString()
      }
    };
  }
  return webhook;
}
```

### Välkomst-e-post för abonnemang

```javascript subscription_welcome.js icon="js" expandable
function handler(webhook) {
  if (webhook.eventType === "subscription.active") {
    const s = webhook.payload.data;
    webhook.url = "https://api.loops.so/v1/events/send";
    webhook.payload = {
      eventName: "subscription_welcome",
      email: s.customer.email,
      properties: {
        customer_name: s.customer.name,
        subscription_id: s.subscription_id,
        product_id: s.product_id,
        amount: (s.recurring_pre_tax_amount / 100).toFixed(2),
        frequency: s.payment_frequency_interval,
        next_billing: s.next_billing_date
      }
    };
  }
  return webhook;
}
```

### Meddelande om betalningsfel

```javascript payment_failure.js icon="js" expandable
function handler(webhook) {
  if (webhook.eventType === "payment.failed") {
    const p = webhook.payload.data;
    webhook.url = "https://api.loops.so/v1/events/send";
    webhook.payload = {
      eventName: "payment_failed",
      email: p.customer.email,
      properties: {
        customer_name: p.customer.name,
        payment_id: p.payment_id,
        amount: (p.total_amount / 100).toFixed(2),
        error_message: p.error_message || "Payment processing failed",
        retry_link: `https://yourdomain.com/retry-payment/${p.payment_id}`
      }
    };
  }
  return webhook;
}
```

## Tips

- Använd beskrivande händelsenamn för bättre organisering av e-postmallar
- Inkludera relevanta kundegenskaper för personalisering
- Ställ in e-postmallar i Loopars instrumentpanel för varje händelse
- Använd konsekvent namngivning av egenskaper över händelser
- Testa e-postleverans innan du går live

## Felsökning

<AccordionGroup>
<Accordion title="Emails not being sent">
- Verifiera att API-nyckeln är korrekt och aktiv
- Kontrollera att händelsenamnen matchar dina Loops-mallar
- Säkerställ att mottagarnas e-postadresser är giltiga
- Granska Loops sändningsgränser och kvoter
</Accordion>

<Accordion title="Transformation errors">
- Validera att JSON-strukturen matchar Loops API-formatet
- Kontrollera att alla obligatoriska fält finns
- Säkerställ att händelsenamnen är korrekt formaterade
- Verifiera API-nyckelns behörigheter
</Accordion>
</AccordionGroup>
