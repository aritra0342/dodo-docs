## Prasyarat

Untuk mengintegrasikan API Pembayaran Dodo, Anda memerlukan:
- Akun pedagang Pembayaran Dodo
- Kredensial API (kunci API dan kunci rahasia webhook) dari dasbor

## Pengaturan Dasbor

1. Arahkan ke [Dasbor Pembayaran Dodo](https://app.dodopayments.com/)

2. Buat produk (pembayaran sekali atau langganan)

4. Hasilkan kunci API Anda:
   - Pergi ke Developer > API
   - [Panduan Detail](/api-reference/introduction#api-key-generation)
   - Salin kunci API di env bernama DODO_PAYMENTS_API_KEY

5. Konfigurasi webhook:
   - Pergi ke Developer > Webhooks
   - Buat URL webhook untuk notifikasi pembayaran
   - Salin kunci rahasia webhook di env 

## Integrasi

### Tautan Pembayaran

Pilih jalur integrasi yang sesuai dengan kasus penggunaan Anda:
- **Sesi Checkout (direkomendasikan)**: Terbaik untuk sebagian besar integrasi. Buat sesi di server Anda dan arahkan pelanggan ke checkout yang aman dan dihosting.
- **Overlay Checkout**: Gunakan saat Anda memerlukan pengalaman dalam halaman yang membuka checkout sebagai overlay modal di situs Anda.
- **Inline Checkout**: Sematkan checkout langsung ke dalam tata letak halaman Anda untuk pengalaman checkout yang sepenuhnya terintegrasi dan bermerek.
- **Tautan Pembayaran Statis**: URL yang dapat dibagikan tanpa kode untuk pengumpulan pembayaran yang cepat.
- **Tautan Pembayaran Dinamis**: Tautan yang dibuat secara programatis. Namun, Sesi Checkout direkomendasikan dan memberikan lebih banyak fleksibilitas.

#### 1. Sesi Checkout

Gunakan Sesi Checkout untuk membuat pengalaman checkout yang aman dan dihosting untuk pembayaran sekali atau langganan. Anda membuat sesi di server Anda, lalu mengarahkan pelanggan ke `checkout_url` yang dikembalikan.

<Info>
Sesi checkout berlaku selama 24 jam secara default. Jika Anda mengirim <code>confirm=true</code>, sesi berlaku selama 15 menit dan semua bidang yang diperlukan harus disediakan.
</Info>

<Steps>
<Step title="Buat sesi checkout">
  Pilih SDK pilihan Anda atau panggil REST API.

  <Tabs>

  ```javascript
  const client = new DodoPayments({
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    environment: 'test_mode', // defaults to 'live_mode'

  const session = await client.checkoutSessions.create({
    product_cart: [{ product_id: 'prod_123', quantity: 1 }],
    customer: { email: 'customer@example.com', name: 'John Doe' },
    return_url: 'https://yourapp.com/checkout/success',
  });
  ```

<Tip>
Sebaiknya gunakan Sesi Checkout untuk cara tercepat dan paling andal dalam mulai menerima pembayaran. Untuk kustomisasi lanjutan, lihat <a href="/developer-resources/checkout-session">panduan Sesi Checkout</a> dan <a href="/api-reference/checkout-sessions/create">Referensi API</a>.

#### 2. Checkout Overlay

Untuk pengalaman checkout dalam halaman yang mulus, jelajahi integrasi [Checkout Overlay](/developer-resources/overlay-checkout) kami yang memungkinkan pelanggan menyelesaikan pembayaran tanpa meninggalkan situs web Anda.

#### 3. Inline Checkout

Untuk pengalaman checkout yang sepenuhnya terintegrasi yang disematkan langsung di halaman Anda, gunakan integrasi [Inline Checkout](/developer-resources/inline-checkout) kami. Ini memungkinkan Anda untuk membangun ringkasan pesanan kustom dan memiliki kontrol penuh atas tata letak checkout sementara Dodo Payments menangani pengumpulan pembayaran dengan aman.

#### 4. Tautan Pembayaran Statis

Tautan pembayaran statis memungkinkan Anda untuk dengan cepat menerima pembayaran dengan membagikan URL sederhana. Anda dapat menyesuaikan pengalaman checkout dengan mengirimkan parameter kueri untuk mengisi detail pelanggan, mengontrol bidang formulir, dan menambahkan metadata kustom.

<Steps>
<Step title="Buat tautan pembayaran Anda">
  Mulailah dengan URL dasar dan tambahkan ID produk Anda:

  ```text
  https://checkout.dodopayments.com/buy/{productid}
  ```

</Step>
<Step title="Tambahkan parameter inti">
  Sertakan parameter kueri penting:
  - <ParamField query="quantity" type="integer" default="1">Jumlah barang yang akan dibeli.</ParamField>
  - <ParamField query="redirect_url" type="string" required>URL untuk mengarahkan setelah pembayaran selesai.</ParamField>
  <Note>
  URL pengalihan akan menyertakan detail pembayaran sebagai parameter kueri, misalnya:<br/>
  <code>https://example.com/?payment_id=pay_ts2ySpzg07phGeBZqePbH&status=succeeded</code>
  </Note>
</Step>
<Step title="Isi informasi pelanggan (opsional)">
  Tambahkan bidang pelanggan atau penagihan sebagai parameter kueri untuk memperlancar checkout.
  <AccordionGroup>
    <Accordion title="Bidang Pelanggan yang Didukung">
      - <ParamField query="fullName" type="string">Nama lengkap pelanggan (diabaikan jika firstName atau lastName disediakan).</ParamField>
      - <ParamField query="firstName" type="string">Nama depan pelanggan.</ParamField>
      - <ParamField query="lastName" type="string">Nama belakang pelanggan.</ParamField>
      - <ParamField query="email" type="string">Alamat email pelanggan.</ParamField>
      - <ParamField query="country" type="string">Negara pelanggan.</ParamField>
      - <ParamField query="addressLine" type="string">Alamat jalan.</ParamField>
      - <ParamField query="city" type="string">Kota.</ParamField>
      - <ParamField query="state" type="string">Negara bagian atau provinsi.</ParamField>
      - <ParamField query="zipCode" type="string">Kode pos/ZIP.</ParamField>
      - <ParamField query="showDiscounts" type="boolean">true atau false</ParamField>
    </Accordion>
  </AccordionGroup>
</Step>
<Step title="Kontrol bidang formulir (opsional)">
  <Info>
  Anda dapat menonaktifkan bidang tertentu agar hanya dapat dibaca oleh pelanggan. Ini berguna ketika Anda sudah memiliki detail pelanggan (misalnya, pengguna yang sudah masuk).
  </Info>
  Untuk menonaktifkan sebuah bidang, berikan nilainya dan atur bendera <code>disable...</code> yang sesuai ke <code>true</code>:
  <CodeGroup>

  ```text Example
  &email=alice@example.com&disableEmail=true
  ```

  </CodeGroup>
  <Tabs>
    <Tab title="Tabel Bendera Nonaktif">
      | Bidang         | Bendera Nonaktif         | Parameter Diperlukan |
      |---------------|---------------------|-------------------|
      | Nama Lengkap     | `disableFullName`   | `fullName`        |
      | Nama Depan    | `disableFirstName`  | `firstName`       |
      | Nama Belakang     | `disableLastName`   | `lastName`        |
      | Email         | `disableEmail`      | `email`           |
      | Negara       | `disableCountry`    | `country`         |
      | Alamat Jalan  | `disableAddressLine`| `addressLine`     |
      | Kota          | `disableCity`       | `city`            |
      | Negara Bagian         | `disableState`      | `state`           |
      | Kode Pos      | `disableZipCode`    | `zipCode`         |
    </Tab>
  </Tabs>
  <Tip>
  Menonaktifkan bidang membantu mencegah perubahan yang tidak disengaja dan memastikan konsistensi data.
  </Tip>
  <Note>
  Mengatur <code>showDiscounts=false</code> akan menonaktifkan dan menyembunyikan bagian diskon di formulir checkout. Gunakan ini jika Anda ingin mencegah pelanggan memasukkan kode kupon atau promo selama checkout.
  </Note>

</Step>
<Step title="Tambahkan kontrol lanjutan (opsional)">
  - <ParamField query="paymentCurrency" type="string">Menentukan mata uang pembayaran. Secara default menggunakan mata uang negara penagihan.</ParamField>
  - <ParamField query="showCurrencySelector" type="boolean" default="true">Tampilkan atau sembunyikan pemilih mata uang.</ParamField>
  - <ParamField query="paymentAmount" type="integer">Jumlah dalam sen (hanya untuk harga Pay What You Want).</ParamField>
  - <ParamField query="metadata_*" type="string">Bidang metadata kustom (misalnya, <code>metadata_orderId=123</code>).</ParamField>
</Step>
<Step title="Bagikan tautan">
  Kirim tautan pembayaran yang telah selesai kepada pelanggan Anda. Ketika mereka mengunjungi, semua parameter kueri dikumpulkan dan disimpan dengan ID sesi. URL kemudian disederhanakan untuk hanya menyertakan parameter sesi (misalnya, <code>?session=sess_1a2b3c4d</code>). Informasi yang disimpan bertahan melalui penyegaran halaman dan dapat diakses sepanjang proses checkout.
  <Check>
  Pengalaman checkout pelanggan sekarang lebih lancar dan dipersonalisasi berdasarkan parameter Anda.
  </Check>
</Step>
</Steps>

    ```javascript
      {
        error: err instanceof Error ? err.message : "An unknown error occurred",

      { status: 500 }
    );
```

### Mengimplementasikan Webhook

Siapkan endpoint API untuk menerima notifikasi pembayaran. Berikut adalah contoh menggunakan Next.js:

```javascript
import { Webhook } from "standardwebhooks";
import { headers } from "next/headers";
import { WebhookPayload } from "@/types/api-types";

const webhook = new Webhook(process.env.DODO_WEBHOOK_KEY!); // Replace with your secret key generated from the Dodo Payments Dashboard

export async function POST(request: Request) {
  const headersList = headers();
  const rawBody = await request.text();

  const webhookHeaders = {
    "webhook-id": headersList.get("webhook-id") || "",
    "webhook-signature": headersList.get("webhook-signature") || "",
    "webhook-timestamp": headersList.get("webhook-timestamp") || "",
  };

  await webhook.verify(rawBody, webhookHeaders);
  const payload = JSON.parse(rawBody) as WebhookPayload;
  
  // Process the payload according to your business logic
}
```

Implementasi webhook kami mengikuti spesifikasi [Standard Webhooks](https://standardwebhooks.com/). Untuk definisi tipe webhook, lihat [Panduan Acara Webhook](/developer-resources/webhooks/intents/webhook-events-guide).

Anda dapat merujuk proyek ini dengan implementasi demo di [GitHub](https://github.com/dodopayments/dodo-checkout-demo) menggunakan Next.js dan TypeScript.

Anda dapat melihat implementasi langsung [di sini](https://atlas.dodopayments.com/).
