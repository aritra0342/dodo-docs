<Note>
Panduan ini secara eksklusif mencakup **pembayaran sekali saja** yang melibatkan **kartu, Apple Pay, dan Google Pay**.
</Note>

## Endpoint API

| Environment | Base URL | Confirm Payment Endpoint |
| --- | --- | --- |
| **Test Mode** | `https://test.dodopayments.com` | `https://api.custom.hs.dodopayments.com/payments/<payment_id>/confirm` |
| **Live Mode** | `https://live.dodopayments.com` | `https://api.hs.dodopayments.com/payments/<payment_id>/confirm` |

## Alur Pembayaran Kartu Langkah demi Langkah

### Langkah 1: Dapatkan Secret Key

Mulailah dengan membuat pembayaran menggunakan API berikut:

- **Payments**: [Create Payment API](/api-reference/payments/post-payments)

API ini mengembalikan **`secret_key`**, yang diperlukan untuk langkah-langkah berikutnya.

### Langkah 2: Konfirmasi Pembayaran

Konfirmasikan pembayaran dengan memanggil endpoint `payments confirm`:

- **Test Mode**: `POST https://api.custom.hs.dodopayments.com/payments/<payment_id>/confirm`
- **Live Mode**: `POST https://api.hs.dodopayments.com/payments/<payment_id>/confirm`

Cantumkan detail berikut dalam body permintaan:

- `client_secret`: Secret key dari Langkah 1
- `payment_method`: Tetapkan ke `card`
- `payment_method_data`: Detail kartu dan penagihan
- `browser_info`: Data spesifik browser (user agent, detail layar, dll.)
- `payment_type`: Tetapkan secara eksplisit ke `normal` (untuk pembayaran sekali saja)

Headers yang diperlukan:

- Dapatkan publishable key Anda dari dukungan DodoPayments
- `api-key`: Publishable key Anda (Dapatkan ini dari Tim Dodo Payments)
- `content-type`: `application/json`

### Menangani Respons

- Jika respons **`status`** adalah **`requires_customer_action`**, arahkan ulang pelanggan ke URL yang disediakan di bidang **`next_action`** untuk otentikasi lebih lanjut.

<Warning>
Pastikan kepatuhan terhadap pedoman PCI saat menangani data kartu mentah.
</Warning>

## Integrasi Apple Pay

Untuk menggunakan Apple Pay, tetapkan `payment_method` ke `wallet` dan sertakan data Apple Pay dalam `payment_method_data`.

### Struktur Body Permintaan

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "wallet",
  "payment_method_data": {
    "wallet": {
      "apple_pay": {
        "payment_data": "<decrypted_payment_data>",
        "payment_method": {
          "display_name": "Visa 1234",
          "network": "Visa",
          "type": "debit"
        },
        "transaction_identifier": "UNIQUE_TRANSACTION_ID_FROM_APPLE"
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
    "time_zone": "<time_zone>",
    "java_enabled": true,
    "java_script_enabled": true,
    "ip_address": "<ip_address_of_client>",
    "device_model": "<device_model>",
    "os_type": "<os_type>",
    "os_version": "<version>"
  },
  "payment_type": "normal"
}
```

### Referensi Field Apple Pay

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `payment_data` | string | Yes | Enum ini digunakan untuk mewakili data pembayaran Apple Pay, yang dapat berupa terenkripsi atau tidak. |
| `payment_method.display_name` | string | Yes | Nama yang ditampilkan pada tombol Apple Pay (misalnya, "Visa 1234") |
| `payment_method.network` | string | Yes | Jaringan kartu (misalnya, "Visa", "Mastercard", "Amex") |
| `payment_method.type` | string | Yes | Jenis metode pembayaran (misalnya, "debit", "credit") |
| `payment_method.card_exp_month` | string | No | Bulan kedaluwarsa kartu (misalnya, "12") |
| `payment_method.card_exp_year` | string | No | Tahun kedaluwarsa kartu (misalnya, "25") |
| `transaction_identifier` | string | Yes | Identifier transaksi unik dari Apple Pay |

## Integrasi Google Pay

Untuk menggunakan Google Pay, tetapkan `payment_method` ke `wallet` dan sertakan data Google Pay dalam `payment_method_data`.

### Struktur Body Permintaan

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "wallet",
  "payment_method_data": {
    "wallet": {
      "google_pay": {
        "type": "CARD",
        "description": "Visa ending in 1234",
        "info": {
          "card_network": "VISA",
          "card_details": "1234"
        },
        "tokenization_data": {
          "type": "PAYMENT_GATEWAY",
          "token": "<encrypted_payment_token>"
        }
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
    "time_zone": "<time_zone>",
    "java_enabled": true,
    "java_script_enabled": true,
    "ip_address": "<ip_address_of_client>",
    "device_model": "<device_model>",
    "os_type": "<os_type>",
    "os_version": "<version>"
  },
  "payment_type": "normal"
}
```

### Referensi Field Google Pay

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `type` | string | Yes | Jenis metode pembayaran (biasanya "CARD") |
| `description` | string | Yes | Deskripsi metode pembayaran yang ditujukan untuk pengguna |
| `info.card_network` | string | Yes | Jaringan kartu (misalnya, "VISA", "MASTERCARD", "AMEX") |
| `info.card_details` | string | Yes | 4 digit terakhir kartu |
| `info.assurance_details` | object | No | Detail jaminan otentikasi |
| `tokenization_data.type` | string | Yes | Jenis tokenisasi (misalnya, "PAYMENT_GATEWAY") |
| `tokenization_data.token` | string | Yes | Token pembayaran terenkripsi dari Google Pay |
