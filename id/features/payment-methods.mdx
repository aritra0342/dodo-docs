## Cara Kerjanya

Metode pembayaran secara otomatis ditampilkan kepada pelanggan berdasarkan:

- **Lokasi Pelanggan** — Metode regional seperti iDEAL (Belanda) atau UPI (India) hanya muncul untuk pelanggan di wilayah tersebut
- **Mata Uang Transaksi** — Metode pembayaran difilter berdasarkan mata uang yang didukung
- **Tipe Perangkat** — Apple Pay hanya muncul pada perangkat dan browser Apple yang kompatibel
- **Tipe Transaksi** — Beberapa metode tidak mendukung langganan atau memiliki jumlah minimum

<Info>
**Aktivasi Otomatis:** Semua metode pembayaran telah dikonfigurasi sebelumnya dan dikelola oleh Dodo Payments. Tidak perlu pengaturan tambahan — mereka tersedia setelah akun Anda diverifikasi dan mode langsung diaktifkan.
</Info>

## Mengonfigurasi Metode Pembayaran

Secara default, semua metode pembayaran yang memenuhi syarat muncul secara otomatis. Gunakan `allowed_payment_method_types` untuk membatasi metode mana yang ditampilkan:

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://test.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

Selalu sertakan `credit` dan `debit` sebagai cadangan. Jika semua metode pembayaran yang ditentukan tidak tersedia untuk pelanggan, proses checkout akan gagal.
</Warning>

## Praktik Terbaik

<AccordionGroup>
{/* LOCKED_PATTERN_63da960e31e014d866efe70101f28b99 */}
Sertakan metode pembayaran regional untuk pasar sasaran Anda. iDEAL menangani ~60% pembayaran online di Belanda; UPI mendominasi di India. Menghilangkan metode-metode ini sangat merugikan konversi.
</Accordion>

{/* LOCKED_PATTERN_78f59c035688fa14218708818c9f0fca */}
Bahkan saat menargetkan metode tertentu, sertakan `credit` dan `debit` sebagai cadangan. Tidak semua pelanggan memiliki akses ke setiap metode pembayaran.
</Accordion>

{/* LOCKED_PATTERN_564640b27644e47b9e4e7faaec6c2d88 */}
Metode BNPL (Klarna, Afterpay) memerlukan minimal $50,01. Jangan sertakan untuk produk bernilai rendah — metode tersebut tidak akan muncul, dan itu tidak apa-apa.
</Accordion>

{/* LOCKED_PATTERN_ea5461a635a5291807627a6b40d7922e */}
Gunakan mode pengujian untuk memverifikasi ketersediaan metode pembayaran untuk lokasi pelanggan, mata uang, dan jenis perangkat yang berbeda sebelum go live.
</Accordion>
</AccordionGroup>

## Pemecahan Masalah

<AccordionGroup>
{/* LOCKED_PATTERN_fbf2d38e7725fb97384bef01324b59a1 */}
**Periksa faktor-faktor ini:**
1. Metode termasuk dalam `allowed_payment_method_types`?
2. Lokasi pelanggan sesuai dengan wilayah yang didukung metode?
3. Mata uang transaksi didukung oleh metode?
4. Jumlah transaksi memenuhi minimum (misal, $50,01 untuk BNPL)?
5. Untuk langganan: metode mendukung pembayaran berulang?


**Solusi:** Hapus `allowed_payment_method_types` sementara untuk melihat semua metode yang memenuhi syarat, lalu persempit masalahnya.
</Accordion>

{/* LOCKED_PATTERN_57385d59bd851fb1e8c26266b489efd4 */}
**Penyebab yang mungkin:**
- `allowed_payment_method_types` tidak dikirim dengan benar
- Parameter diformat salah (harus array string)


**Solusi:** Catat payload permintaan API Anda dan verifikasi struktur parameternya.
</Accordion>

{/* LOCKED_PATTERN_6d1a1b1a59cbe2789bbc9434eb1ec153 */}
**Periksa:**
- Alamat penagihan pelanggan sesuai wilayah metode
- Mata uang diatur dengan benar (misal, EUR untuk iDEAL)
- Mata Uang Adaptif diaktifkan jika pelanggan internasional


**Solusi:** Verifikasi kode negara alamat penagihan dan mata uang dalam permintaan API Anda.
</Accordion>

{/* LOCKED_PATTERN_52f3532b80d2def876ef3cc78b55f9f6 */}
Langganan UPI dan Rupay memiliki persyaratan unik termasuk penundaan pemrosesan 48 jam dan batas mandat RBI.


**Lihat:** <a href="/features/payment-methods/india">India Payment Methods</a> untuk panduan lengkap.
</Accordion>
</AccordionGroup>
