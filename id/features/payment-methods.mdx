Dodo Payments mendukung berbagai metode pembayaran untuk membantu Anda melayani pelanggan secara global. Aktifkan metode pembayaran baru secara instan — tidak ada kontrak tambahan, integrasi, atau akun pedagang yang dibutuhkan.

## Cara Kerjanya

Metode pembayaran secara otomatis ditampilkan kepada pelanggan berdasarkan:

- **Lokasi Pelanggan** — Metode regional seperti iDEAL (Belanda) atau UPI (India) hanya muncul untuk pelanggan di wilayah tersebut
- **Mata Uang Transaksi** — Metode pembayaran difilter berdasarkan mata uang yang didukung
- **Tipe Perangkat** — Apple Pay hanya muncul pada perangkat dan browser Apple yang kompatibel
- **Tipe Transaksi** — Beberapa metode tidak mendukung langganan atau memiliki jumlah minimum

<Info>
**Aktivasi Otomatis:** Semua metode pembayaran sudah pra-konfigurasi dan dikelola oleh Dodo Payments. Tidak ada pengaturan tambahan yang diperlukan — mereka menjadi tersedia setelah akun Anda diverifikasi dan mode live diaktifkan.
</Info>

## Mengonfigurasi Metode Pembayaran

Secara default, semua metode pembayaran yang memenuhi syarat muncul secara otomatis. Gunakan `allowed_payment_method_types` untuk membatasi metode yang ditampilkan:

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://test.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

## Praktik Terbaik

<AccordionGroup>
<Accordion title="Sesuaikan metode dengan pasar Anda">
Sertakan metode pembayaran regional untuk pasar target Anda. iDEAL menangkap ~60% dari pembayaran online di Belanda; UPI mendominasi di India. Menghilangkan metode ini secara signifikan merugikan konversi.
</Accordion>

<Accordion title="Selalu sediakan cadangan kartu">
Bahkan saat menargetkan metode tertentu, sertakan `credit` dan `debit` sebagai cadangan. Tidak semua pelanggan memiliki akses ke setiap metode pembayaran.
</Accordion>

<Accordion title="Pertimbangkan jumlah minimum transaksi">
Metode BNPL (Klarna, Afterpay) memerlukan minimum $50.01. Jangan sertakan mereka untuk produk bernilai rendah — mereka tidak akan muncul, yang baik-baik saja.
</Accordion>

<Accordion title="Uji berbagai wilayah dan perangkat">
Gunakan mode uji untuk memverifikasi ketersediaan metode pembayaran untuk lokasi pelanggan yang berbeda, mata uang, dan jenis perangkat sebelum go-live.
</Accordion>
</AccordionGroup>

## Pemecahan Masalah

<AccordionGroup>
<Accordion title="Metode pembayaran tidak muncul">
**Periksa faktor-faktor ini:**
1. Metode disertakan dalam `allowed_payment_method_types`?
2. Lokasi pelanggan cocok dengan wilayah yang didukung oleh metode?
3. Mata uang transaksi didukung oleh metode?
4. Jumlah transaksi memenuhi minimum (e.g., $50.01 untuk BNPL)?
5. Untuk langganan: metode mendukung pembayaran berulang?

**Solusi:** Hapus `allowed_payment_method_types` sementara untuk melihat semua metode yang memenuhi syarat, lalu cari tahu masalahnya.
</Accordion>

<Accordion title="Metode yang tidak terduga muncul">
**Penyebab mungkin:**
- `allowed_payment_method_types` tidak dikirim dengan benar
- Parameter diformat tidak benar (harus berupa array string)

**Solusi:** Catat payload permintaan API Anda dan verifikasi struktur parameter.
</Accordion>

<Accordion title="Metode regional tidak tersedia">
**Periksa:**
- Alamat penagihan pelanggan cocok dengan wilayah metode
- Mata uang diatur dengan benar (misalnya EUR untuk iDEAL)
- Mata Uang Adaptif diaktifkan jika pelanggan berasal dari luar negeri

**Solusi:** Verifikasi kode negara alamat penagihan dan mata uang dalam permintaan API Anda.
</Accordion>

<Accordion title="Masalah metode pembayaran India">
Langganan UPI dan Rupay memiliki persyaratan unik termasuk penundaan pemrosesan 48 jam dan batas mandat RBI.

**Lihat:** <a href="/features/payment-methods/india">Metode Pembayaran India</a> untuk panduan lengkap.
</Accordion>
</AccordionGroup>
