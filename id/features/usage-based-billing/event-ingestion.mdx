Acara adalah dasar dari penagihan berbasis penggunaan. Kirim acara saat tindakan yang dapat ditagih terjadi, dan meter mengagregasi mereka menjadi biaya.

{/* LOCKED_PATTERN_f67275d778880e9e57a8db8b7973e6d3 */}
Dokumentasi API lengkap dengan contoh dan kode respons.
</Card>

## Struktur Acara

{/* LOCKED_PATTERN_4d7f2e9ce45e7323dd2d626df4a7c5b8 */}
Pengidentifikasi unik. Gunakan UUID atau gabungkan ID pelanggan + timestamp + tindakan.
</ParamField>

<ParamField body="customer_id" type="string" required>
ID pelanggan Dodo Payments. Harus pelanggan yang valid dan sudah ada.
</ParamField>

<ParamField body="event_name" type="string" required>
Jenis event yang cocok dengan nama event meter Anda (sensitif terhadap huruf). Contoh: `api.call`, `image.generated`
</ParamField>
</Accordion>

{/* LOCKED_PATTERN_94ee55474fc7877494948c444fe08c91 */}

<ParamField body="timestamp" type="string">
Cap waktu ISO 8601. Default ke waktu server jika tidak disertakan. Sertakan untuk penagihan akurat pada event tertunda/batch.
</ParamField>

<ParamField body="metadata" type="object">
Properti tambahan untuk agregasi dan pemfilteran:
- Nilai numerik: `bytes`, `tokens`, `duration_ms`
- Filter: `endpoint`, `method`, `quality`

```javascript
metadata: {
  endpoint: "/v1/orders",
  method: "POST",
  tokens: 1024
}
```

</ParamField>
</Accordion>

## Mengirim Acara

<CodeGroup>

```javascript Single Event
await fetch('https://test.dodopayments.com/events/ingest', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.DODO_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    events: [{
      event_id: "api_call_1234",
      customer_id: "cus_abc123",
      event_name: "api.call",
      metadata: { endpoint: "/v1/orders" }
    }]
  })
});
```

```javascript Batch Events
await fetch('https://test.dodopayments.com/events/ingest', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.DODO_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    events: [
      { event_id: "call_1", customer_id: "cus_abc", event_name: "api.call" },
      { event_id: "call_2", customer_id: "cus_abc", event_name: "api.call" },
      { event_id: "call_3", customer_id: "cus_abc", event_name: "api.call" }
    ]
  })
});
```

```python Python
import requests

requests.post(
    'https://test.dodopayments.com/events/ingest',
    headers={'Authorization': f'Bearer {api_key}', 'Content-Type': 'application/json'},
    json={'events': [{'event_id': 'call_1', 'customer_id': 'cus_abc', 'event_name': 'api.call'}]}
)
```

```curl cURL
curl -X POST 'https://test.dodopayments.com/events/ingest' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{"events": [{"event_id": "call_1", "customer_id": "cus_abc", "event_name": "api.call"}]}'
```

</CodeGroup>

<Tip>
Kirim batch hingga 100 event per permintaan untuk kinerja lebih baik.
</Tip>

## Praktik Terbaik

<AccordionGroup>
{/* LOCKED_PATTERN_c11962fb3ea9e1a82d1994f0860ee77a */}
Gunakan ID deterministik untuk mencegah duplikasi: `${customerId}_${action}_${timestamp}`
</Accordion>

{/* LOCKED_PATTERN_d4b029e15b52ca6ac907535b9dad9ee9 */}
Coba ulang pada error 5xx dengan backoff eksponensial. Jangan coba ulang error 4xx.
</Accordion>

{/* LOCKED_PATTERN_90e261cbcc49667becacc297210d42c3 */}
Hapus untuk event waktu nyata. Sertakan untuk event tertunda/batch demi akurasi.
</Accordion>

{/* LOCKED_PATTERN_fa83e100effc6c87ff1c68c56ea8afa9 */}
Lacak tingkat keberhasilan dan antri event gagal untuk dicoba ulang.
</Accordion>
</AccordionGroup>

## Pemecahan Masalah

<AccordionGroup>
{/* LOCKED_PATTERN_fc1788ddf0982a3a2fe9f6e81812c541 */}
- Nama event harus benar-benar cocok dengan meter (sensitif huruf)
- ID pelanggan harus ada
- Periksa filter meter tidak mengecualikan event
- Pastikan cap waktu baru-baru ini
</Accordion>

{/* LOCKED_PATTERN_1af6f154c82eaebbb38a748cef2be54a */}
Verifikasi kunci API benar dan gunakan format: `Bearer YOUR_API_KEY`
</Accordion>

{/* LOCKED_PATTERN_bfe0427c6207c44242acb102b2467be5 */}
Pastikan semua kolom wajib ada: `event_id`, `customer_id`, `event_name`
</Accordion>

{/* LOCKED_PATTERN_61cd934926cd73f663eac6935ef56a1b */}
- Kunci metadata harus cocok persis dengan "Over Property" meter
- Gunakan angka, bukan string: `tokens: 150` bukan `tokens: "150"`
</Accordion>
</AccordionGroup>
