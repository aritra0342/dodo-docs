## Apa itu Dompet Pelanggan?

Dompet pelanggan adalah akun saldo moneter yang menyimpan dana nyata untuk pengguna Anda. Setiap pelanggan mendapatkannya secara otomatis saat Anda membuat akun mereka. Anda dapat menggunakan dompet ini untuk:

- **Menyimpan dana prabayar** untuk pembayaran langganan di masa depan
- **Menangani pengembalian dana** sebagai saldo dompet alih-alih pengembalian kartu
- **Mengeluarkan saldo promosi** seperti bonus sambutan atau hadiah loyalitas
- **Menerapkan dana dompet ke faktur** secara otomatis selama penagihan
- **Melacak transaksi moneter** dengan riwayat buku besar yang rinci

<Info>
Setiap pelanggan secara otomatis mendapatkan dompet saat Anda membuat akun mereka. Dompet mendukung mata uang USD dan INR dengan saldo moneter terpisah untuk masing-masing.
</Info>

<Warning>
**Dompet Pelanggan ≠ Penagihan Berbasis Kredit**

Dompet Pelanggan menyimpan **saldo moneter nyata** (USD, INR) yang dapat diterapkan ke faktur dan pembayaran langganan.

Jika Anda ingin melacak unit penggunaan virtual (panggilan API, token, jam komputasi), lihat [Credit-Based Billing](/features/credit-based-billing) sebagai gantinya.
</Warning>

## Cara Kerjanya

Dompet pelanggan menyimpan dana yang dapat digunakan pelanggan untuk pembelian mereka. Ketika pelanggan memiliki faktur atau tagihan langganan berulang, saldo dompet mereka akan diperiksa terlebih dahulu. Dana yang tersedia diterapkan secara otomatis ke faktur sebelum membebankan metode pembayaran utama mereka.

### Pengaturan Otomatis

Saat Anda membuat pelanggan baru, Dodo Payments secara otomatis membuat dompet dengan saldo nol. Dompet siap menerima dana segera melalui API kami.

### Dukungan Multi-Mata Uang

Setiap dompet dapat menyimpan saldo dalam mata uang yang berbeda:

<ResponseField name="USD Balance" type="integer">
Saldo dalam Dolar AS (disimpan dalam sen)
</ResponseField>

<ResponseField name="INR Balance" type="integer">
Saldo dalam Rupee India (disimpan dalam paise)
</ResponseField>

<Info>
Saat ini, hanya saldo **USD** dan **INR** yang tersedia. Mata uang tambahan akan hadir segera.
</Info>

## Contoh Dunia Nyata

### Pengembalian Dana ke Dompet

Ketika pelanggan meminta pengembalian dana, Anda dapat menambahkan jumlah tersebut ke saldo dompet mereka alih-alih melakukan pengembalian dana kartu tradisional. Ini menjaga dana tetap berada dalam ekosistem Anda untuk pembelian di masa depan.

```javascript
async function refundToWallet(customerId, refundAmount, originalPaymentId) {
  await client.customers.wallets.ledgerEntries.create(customerId, {
    amount: refundAmount, // Amount in cents
    currency: 'USD',
    entry_type: 'credit',
    reason: `Refund for payment ${originalPaymentId}`,
    idempotency_key: `refund_${originalPaymentId}`
  });
}
```

### Bonus Sambutan / Saldo Promosi

Berikan bonus sambutan moneter kepada pelanggan baru untuk mendorong pembelian pertama mereka.

```javascript
async function addWelcomeBonus(customerId) {
  await client.customers.wallets.ledgerEntries.create(customerId, {
    amount: 1000, // $10.00 promotional balance
    currency: 'USD',
    entry_type: 'credit',
    reason: 'Welcome bonus - $10 promotional balance',
    idempotency_key: `welcome_${customerId}`
  });
}
```

### Pembayaran Langganan dari Dompet

Kurangi dana dari dompet untuk menutupi biaya langganan atau pembelian manual.

```javascript
async function deductForPurchase(customerId, purchaseAmount, purchaseId) {
  try {
    await client.customers.wallets.ledgerEntries.create(customerId, {
      amount: purchaseAmount,
      currency: 'USD',
      entry_type: 'debit',
      reason: `Subscription charge - monthly billing`,
      idempotency_key: `charge_${purchaseId}`
    });
  } catch (error) {
    if (error.status === 400) {
      console.log('Insufficient wallet balance');
    }
  }
}
```

## Praktik Terbaik

### Mencegah Transaksi Duplikat

Gunakan kunci idempoten untuk memastikan Anda tidak secara tidak sengaja menambahkan atau mengurangi dana dua kali untuk peristiwa yang sama.

```javascript
async function addFundsSafely(customerId, amount, reason) {
  const idempotencyKey = `${reason}_${customerId}_${Date.now()}`;
  
  try {
    const result = await client.customers.wallets.ledgerEntries.create(customerId, {
      amount: amount,
      currency: 'USD',
      entry_type: 'credit',
      reason: reason,
      idempotency_key: idempotencyKey
    });
    
    return { success: true, wallet: result };
  } catch (error) {
    if (error.status === 409) {
      // Transaction already processed
      return { success: true, wallet: null, duplicate: true };
    }
    
    throw error;
  }
}
```

### Periksa Saldo Sebelum Membebankan

Verifikasi pelanggan memiliki dana yang cukup sebelum mencoba memproses transaksi besar dari dompet.

```javascript
async function checkBalanceBeforeOperation(customerId, requiredAmount) {
  const wallets = await client.customers.wallets.list(customerId);
  const usdWallet = wallets.items.find(w => w.currency === 'USD');
  
  if (!usdWallet || usdWallet.balance < requiredAmount) {
    throw new Error('Insufficient funds for this operation');
  }
  
  return usdWallet.balance;
}
```

## Apa yang Akan Datang

<Warning>
Fitur-fitur berikut direncanakan untuk rilis mendatang:
</Warning>

- **Kedaluwarsa Saldo**: Tetapkan dana untuk kedaluwarsa setelah periode tertentu
- **Analitik yang Lebih Baik**: Laporan pengeluaran rinci dan tren saldo
- **Lebih Banyak Webhook**: Notifikasi waktu nyata untuk perubahan saldo dan peringatan saldo rendah

<Tip>
Mulai dengan operasi pendanaan dan pengurangan dasar, lalu integrasikan alur kerja penagihan otomatis yang lebih kompleks seiring pertumbuhan bisnis Anda.
</Tip>
