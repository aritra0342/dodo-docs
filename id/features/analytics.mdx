Lacak perjalanan pelanggan dari etalase melalui proses checkout hingga penyelesaian pembelian. Dodo Payments terintegrasi dengan Google Analytics 4, Google Tag Manager, dan Meta Pixel untuk menyediakan analitik komprehensif di semua halaman yang dihosting.

<Info>
Pelacakan analitik diaktifkan secara otomatis di Etalase, halaman Checkout, dan Portal Pelanggan Anda setelah Anda menambahkan ID pelacakan di dashboard.
</Info>

<Warning>
Pemblokir iklan di browser pelanggan dapat mencegah pelacakan analitik berfungsi untuk sebagian pengunjung toko Anda.
</Warning>

## Getting Started

<Steps>
<Step title="Open Business Settings">
  Di dashboard Dodo Payments Anda, buka **Settings** dan gulir ke bagian **Analytics**.
</Step>

<Step title="Enable Your Tracking Platform">
  Aktifkan platform analitik yang ingin Anda gunakan dan masukkan ID pelacakan Anda:
  - **Google Analytics**: Masukkan Measurement ID Anda (format: `G-XXXXXXXXXX`)
  - **Google Tag Manager**: Masukkan Container ID Anda (format: `GTM-XXXXXXX`)
  - **Meta Pixel**: Masukkan Pixel ID Anda (numerik, 10-20 digit)
</Step>

<Step title="Save Your Settings">
  Klik **Save** untuk menerapkan konfigurasi pelacakan Anda.
</Step>

<Step title="Verify Tracking">
  Kunjungi etalase Anda atau buat checkout uji untuk memverifikasi acara dilacak di dashboard analitik Anda.
</Step>
</Steps>

## Google Analytics 4

[Google Analytics 4](https://marketingplatform.google.com/about/analytics/) menyediakan analitik web menyeluruh dengan kemampuan pelacakan e-niaga. Dengan integrasi Dodo Payments, Anda dapat melacak seluruh perjalanan pelanggan mulai dari menjelajahi toko hingga menyelesaikan pembelian.

### Add Your Google Analytics Measurement ID

1. Di Google Analytics, buka **Admin** → **Data Streams** → pilih stream web Anda
2. Salin **Measurement ID** (dimulai dengan `G-`)
3. Di dashboard Dodo Payments, buka **Settings** → **Analytics**
4. Aktifkan **Google Analytics** dan tempel Measurement ID Anda
5. Klik **Save**

<Tip>
Jika Anda belum memiliki properti Google Analytics, ikuti [panduan penyiapan Google](https://support.google.com/analytics/answer/9304153) untuk membuatnya.
</Tip>

### Events Tracked in Google Analytics

Kami melacak acara berikut di etalase dan halaman checkout yang dihosting. Data acara termasuk informasi produk, total transaksi, dan mata uang dikirim bersama setiap acara.

#### Storefront Events

| Event | Description |
|-------|-------------|
| `view_item_list` | Terpicu ketika halaman etalase dimuat dengan produk |
| `select_item` | Terpicu ketika pelanggan mengklik produk untuk memulai checkout |

#### Checkout Events

| Event | Description |
|-------|-------------|
| `view_item` | Terpicu ketika halaman checkout dimuat |
| `view_cart` | Terpicu ketika checkout dimuat dan ketika varian dipilih |
| `apply_discount`* | Terpicu ketika kode diskon valid berhasil diterapkan |
| `remove_discount`* | Terpicu ketika kode diskon dihapus |
| `add_payment_info` | Terpicu ketika pelanggan mengirimkan formulir pembayaran |
| `purchase` | Terpicu ketika pembayaran berhasil |

<Note>
`apply_discount` dan `remove_discount` adalah acara kustom yang tidak termasuk dalam [acara e-niaga standar Google Analytics](https://support.google.com/analytics/answer/9267735).
</Note>

### Using GA4 with Overlay/Inline Checkout

Untuk melacak acara Google Analytics di overlay atau inline checkout yang disematkan di situs Anda, Anda perlu memasang kode pelacakan Google Analytics di situs tempat checkout disematkan.

Setelah terpasang, semua acara pelacakan yang tercantum di atas akan dikirim ke akun Google Analytics Anda secara otomatis - tanpa konfigurasi tambahan.

## Google Tag Manager

[Google Tag Manager](https://marketingplatform.google.com/about/tag-manager/) memungkinkan Anda mengelola tag pemasaran dan analitik tanpa mengubah kode. Tag yang diterapkan melalui GTM akan berjalan di semua halaman yang dihosting Dodo Payments.

### Add Your GTM Container ID

1. Di Google Tag Manager, salin **Container ID** Anda (dimulai dengan `GTM-`)
2. Di dashboard Dodo Payments, buka **Settings** → **Analytics**
3. Aktifkan **Google Tag Manager** dan tempel Container ID Anda
4. Klik **Save**

<Info>
Saat menggunakan GTM, Anda dapat mengonfigurasi pelacakan tambahan (seperti konversi Google Ads) langsung di kontainer GTM Anda. Integrasi Dodo Payments menyuntikkan skrip GTM dan fallback noscript untuk kompatibilitas maksimal.
</Info>

### Using GTM with Overlay/Inline Checkout

Untuk overlay atau inline checkout yang disematkan di situs Anda, pasang GTM di situs Anda sendiri. Checkout akan mewarisi konfigurasi GTM Anda dan menjalankan acara dalam bingkai yang disematkan.

## Meta Pixel

[Meta Pixel](https://www.facebook.com/business/tools/meta-pixel/) melacak tindakan pelanggan di toko Anda untuk mengukur efektivitas kampanye iklan Facebook dan Instagram Anda. Lacak konversi dari klik iklan hingga penyelesaian pembelian.

### Add Your Meta Pixel ID

1. Di [Meta Events Manager](https://www.facebook.com/events_manager2/), buat atau pilih Pixel Anda
2. Salin **Pixel ID** (numerik, umumnya 15-16 digit)
3. Di dashboard Dodo Payments, buka **Settings** → **Analytics**
4. Aktifkan **Meta Pixel** dan tempel Pixel ID Anda
5. Klik **Save**

<Tip>
Ikuti [panduan penyiapan Pixel](https://www.facebook.com/business/help/952192354843755) dari Meta jika Anda belum membuat Pixel.
</Tip>

### Events Tracked with Meta Pixel

Kami melacak acara berikut yang muncul di [Meta Events Manager](https://www.facebook.com/events_manager2/). Data acara termasuk ID konten, nilai, dan mata uang dikirim bersama setiap acara.

<Note>
Acara mungkin tidak muncul segera di Meta Events Manager - bisa ada jeda beberapa menit.
</Note>

#### Storefront Events

| Event | Description |
|-------|-------------|
| `PageView` | Terpicu ketika halaman etalase dimuat |
| `ViewContent` | Terpicu ketika etalase dimuat dengan produk |
| `InitiateCheckout` | Terpicu ketika pelanggan mengklik produk untuk memulai checkout |

#### Checkout Events

| Event | Description |
|-------|-------------|
| `PageView` | Terpicu ketika halaman checkout dimuat |
| `AddToCart` | Terpicu ketika checkout dimuat dan ketika varian dipilih |
| `ApplyDiscount`* | Terpicu ketika kode diskon valid berhasil diterapkan |
| `RemoveDiscount`* | Terpicu ketika kode diskon dihapus |
| `AddPaymentInfo` | Terpicu ketika pelanggan mengirimkan formulir pembayaran |
| `Purchase` | Terpicu ketika pembayaran berhasil |

<Note>
`ApplyDiscount` dan `RemoveDiscount` adalah acara kustom yang tidak termasuk dalam [acara standar Meta](https://developers.facebook.com/docs/meta-pixel/reference#standard-events).
</Note>

## Complete Events Reference

Tabel ini menunjukkan semua acara yang dilacak di Google Analytics 4 dan Meta Pixel:

| Trigger | GA4 Event | Meta Pixel Event | Location |
|---------|-----------|------------------|----------|
| Storefront page loads | `view_item_list` | `ViewContent` | Storefront |
| Customer clicks product | `select_item` | `InitiateCheckout` | Storefront |
| Checkout page loads | `view_item` | `AddToCart` | Checkout |
| Cart viewed/variant selected | `view_cart` | - | Checkout |
| Discount code applied | `apply_discount` | `ApplyDiscount` | Checkout |
| Discount code removed | `remove_discount` | `RemoveDiscount` | Checkout |
| Payment form submitted | `add_payment_info` | `AddPaymentInfo` | Checkout |
| Payment successful | `purchase` | `Purchase` | Checkout |

## Platform Coverage

Pelacakan analitik tersedia di berbagai permukaan Dodo Payments:

| Surface | Google Analytics 4 | Google Tag Manager | Meta Pixel |
|---------|-------------------|-------------------|------------|
| Storefront | Ya | Ya | Ya |
| Checkout (Hosted) | Ya | Ya | Ya |
| Checkout (Overlay/Inline)* | Ya | Ya | Ya |
| Customer Portal | Ya | Ya | Tidak |

<Info>
*Untuk checkout overlay dan inline, pasang kode pelacakan di situs Anda sendiri agar atribusi lengkap. Acara dari checkout yang disematkan akan terlacak otomatis.
</Info>

## Event Data Included

Setiap acara pelacakan menyertakan data e-niaga relevan untuk membantu Anda menganalisis perilaku pelanggan:

### Product Data
- Product ID
- Nama produk
- Harga
- Mata uang
- Jumlah

### Transaction Data (Purchase Events)
- ID Transaksi/Pembayaran
- Total jumlah
- Jumlah pajak
- Mata uang
- Diskon diterapkan

### Customer Data
- ID Pelanggan (untuk pelanggan yang kembali)
- Email (di-hash untuk privasi bila berlaku)

## Best Practices

<AccordionGroup>
<Accordion title="Test Your Integration">
Setelah menambahkan ID pelacakan Anda, kunjungi etalase Anda dan selesaikan checkout uji untuk memverifikasi acara muncul di dashboard analitik Anda. Gunakan laporan Realtime Google Analytics atau fitur Test Events di Meta Events Manager.
</Accordion>

<Accordion title="Use UTM Parameters">
Tambahkan parameter UTM ke tautan pemasaran Anda untuk melacak kampanye mana yang mendorong konversi. Dodo Payments mempertahankan parameter UTM sepanjang alur checkout.
</Accordion>

<Accordion title="Set Up Conversion Goals">
Di Google Analytics, buat acara konversi untuk `purchase` guna melacak rasio konversi Anda. Di Meta Ads Manager, gunakan acara `Purchase` sebagai tujuan optimasi kampanye.
</Accordion>

<Accordion title="Enable Enhanced E-commerce">
Jika menggunakan Google Tag Manager, pertimbangkan untuk menyiapkan laporan Enhanced E-commerce guna analisis funnel rinci dan metrik kinerja produk.
</Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
<Accordion title="Events not appearing in Google Analytics">
- Verifikasi Measurement ID Anda benar (format: `G-XXXXXXXXXX`)
- Periksa laporan Realtime Google Analytics - acara muncul di sana terlebih dahulu
- Pastikan toggle diaktifkan di pengaturan Dodo Payments Anda
- Tunggu beberapa menit karena bisa ada keterlambatan pemrosesan
- Uji di jendela incognito untuk mengesampingkan pemblokir iklan
</Accordion>

<Accordion title="Events not appearing in Meta Events Manager">
- Verifikasi Pixel ID Anda benar (numerik, 10-20 digit)
- Gunakan ekstensi browser Meta Pixel Helper untuk debugging
- Periksa tab Test Events di Events Manager
- Acara dapat membutuhkan beberapa menit untuk muncul
- Pastikan pemblokir iklan dinonaktifkan saat pengujian
</Accordion>

<Accordion title="GTM tags not firing">
- Konfirmasi Container ID Anda benar (format: `GTM-XXXXXXX`)
- Gunakan mode Preview GTM untuk debugging pemicu tag
- Periksa bahwa trigger dikonfigurasi dengan benar di kontainer GTM Anda
- Pastikan kontainer dipublikasikan (tidak hanya draft)
</Accordion>

<Accordion title="Overlay/Inline checkout not tracking">
- Pasang kode pelacakan di situs Anda sendiri terlebih dahulu
- Pastikan pelacakan dimuat sebelum overlay checkout terbuka
- Periksa konsol browser untuk kesalahan JavaScript
- Verifikasi pelacakan lintas domain dikonfigurasi jika menggunakan GA4
</Accordion>

<Accordion title="Purchase events showing wrong amounts">
- Jumlah dikirim dalam satuan mata uang terkecil (sen)
- Verifikasi alat analitik Anda dikonfigurasi untuk menampilkan mata uang yang benar
- Periksa apakah pajak termasuk/eksklusif sesuai yang diharapkan
</Accordion>
</AccordionGroup>
