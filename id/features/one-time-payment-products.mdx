## Apa Itu Pembayaran Sekali?

Pembayaran sekali adalah pembelian tetap di muka - ideal untuk:

- **Lisensi perangkat lunak**: Pembukaan seumur hidup atau versi utama
- **Unduhan digital**: E-book, template, grafik, atau plugin
- **Layanan**: Audit, paket onboarding, atau konsultasi sekali
- **Kursus dan tiket**: Kursus, lokakarya, atau sesi langsung

## Manfaat Utama

- **Checkout sederhana**: Tidak ada pembaruan atau logika prorata
- **Pemenuhan instan**: Kirim lisensi atau file segera
- **Harga fleksibel**: Diskon dan pajak didukung
- **Developer-pertama**: API yang jelas untuk pembuatan, pengembalian dana, dan webhook

## Membuat Produk Sekali

Buat produk di dasbor Dodo Payments Anda, lalu jual melalui checkout yang dihosting, tautan pembayaran, atau API Anda.

### Konfigurasi Produk

Konfigurasikan bidang di dasbor untuk mendefinisikan bagaimana produk Anda muncul dan dijual.

#### Detail Produk

- **Nama Produk** (wajib): Nama tampilan yang ditampilkan di checkout dan faktur
- **Deskripsi Produk** (wajib): Pernyataan nilai yang jelas yang dilihat pelanggan sebelum membayar
- **Gambar Produk** (wajib): PNG/JPG/WebP hingga 3 MB
- **Merek**: Kaitkan produk dengan merek tertentu untuk tema dan email
- **Kategori Pajak** (wajib): Pilih kategori yang benar untuk menentukan aturan pajak

<Tip>
Pilih kategori pajak yang paling akurat untuk memastikan pengumpulan pajak yang benar per wilayah.
</Tip>

#### Harga

- **Tipe Harga**: Pilih <b>Pembayaran Tunggal</b>
- **Harga** (wajib): Biaya tetap dengan mata uang
- **Diskon Berlaku (%)**: Persentase diskon opsional yang ditampilkan di checkout dan faktur

<Warning>
Mengubah harga produk hanya mempengaruhi pembelian baru. Pelanggan yang ada tidak terpengaruh.
</Warning>

#### Pengaturan Lanjutan

- **Harga Termasuk Pajak**: Tampilkan harga termasuk pajak yang berlaku
- **Hasilkan kunci lisensi**: Keluarkan kunci unik setelah pembelian. Lihat <a href="/features/license-keys">Kunci Lisensi</a>
- **Pengiriman Produk Digital**: Kirim file atau konten secara otomatis. Lihat <a href="/features/digital-product-delivery">Pengiriman Produk Digital</a>
- **Metadata**: Lampirkan pasangan kunci-nilai kustom untuk penandaan internal atau integrasi. Lihat <a href="/api-reference/metadata">Metadata</a>

<Tip>
Gunakan metadata untuk menyimpan pengidentifikasi dari sistem Anda (misalnya, orderId) untuk merekonsiliasi peristiwa dan penerimaan nanti.
</Tip>

## Kasus Penggunaan Umum

- **Alat berbasis lisensi**: Buka fitur dengan kunci sekali pakai
- **Aset digital**: Tema, template, set ikon, atau media
- **Layanan profesional**: Implementasi atau pengaturan dengan biaya tetap
- **Pendidikan**: Tiket kursus dan lokakarya sekali

## Contoh Integrasi

### Sesi Checkout (pembayaran tunggal)
Buat sesi checkout dengan produk sekali pakai di keranjang:

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    { product_id: 'prod_one_time', quantity: 1 }
  ]
});
```

## Praktik Terbaik

- **Jelaskan hasil**: Beritahu pembeli dengan tepat apa yang mereka dapatkan dan bagaimana itu dikirimkan
- **Kirim segera**: Gunakan kunci lisensi atau pengiriman digital untuk akses instan
- **Lacak dengan metadata**: Simpan ID Anda untuk rekonsiliasi yang mudah

<Info>
Pembayaran sekali adalah jalur tercepat untuk menjual produk dengan harga tetap. Mulailah dengan sederhana, verifikasi pengiriman, dan iterasi penawaran Anda berdasarkan konversi dan umpan balik.
</Info>
