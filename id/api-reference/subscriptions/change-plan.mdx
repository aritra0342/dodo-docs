## Penanganan Kegagalan Pembayaran

Gunakan parameter `on_payment_failure` untuk mengendalikan apa yang terjadi ketika pembayaran perubahan paket gagal:

| Value | Behavior |
|-------|----------|
| `prevent_change` | Pertahankan langganan pada paket saat ini hingga pembayaran berhasil. Perubahan paket tetap tertunda. |
| `apply_change` | Terapkan perubahan paket segera terlepas dari hasil pembayaran. Ini adalah bawaan. |

<Info>
Jika `on_payment_failure` tidak ditentukan, perilaku default mengikuti pengaturan tingkat bisnis Anda yang dikonfigurasi di dashboard.
</Info>

<Tip>
Gunakan `prevent_change` untuk peningkatan kritis di mana Anda ingin memastikan pembayaran sebelum memberikan akses ke fitur premium.
</Tip>
