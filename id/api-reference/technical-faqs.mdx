<Accordion title="Q1: Mengapa saya tidak menerima product id dalam webhook payment.succeeded untuk produk langganan?">
  A: Event webhook payment.succeeded untuk produk langganan tidak menyertakan product_id. Sebagai gantinya, ia menyediakan `subscription_id` yang terkait dengan langganan, bersama dengan detail relevan lainnya.
</Accordion>

<Accordion title="Q2: Bagaimana cara memverifikasi bahwa webhook yang saya terima benar-benar dari Dodo Payments dan bukan dari aktor jahat?">
  A: Untuk memverifikasi bahwa webhook yang Anda terima benar-benar dari Dodo Payments dan bukan dari aktor jahat, ikuti langkah-langkah berikut:
    1. **Periksa Tanda Tangan**: Setiap webhook yang dikirim oleh Dodo Payments menyertakan tanda tangan unik dalam header **`webhook-signature`**. Tanda tangan ini dihasilkan menggunakan kunci rahasia webhook Anda dan payload dari event tersebut.
    2. **Ambil Kunci Rahasia**: Pastikan Anda memiliki kunci rahasia untuk webhook Anda, yang dapat ditemukan di dasbor Dodo Payments Anda di bawah pengaturan Webhooks.
    3. **Hitung Ulang Tanda Tangan**: Gunakan algoritma penandatanganan yang sama (HMAC-SHA256) seperti Dodo Payments untuk menghitung tanda tangan di server Anda. Gunakan payload dari webhook dan kunci rahasia webhook Anda.
    4. **Bandingkan Tanda Tangan**: Bandingkan tanda tangan yang dihitung ulang dengan yang diberikan dalam header **`webhook-signature`**. Jika cocok, webhook tersebut autentik; jika tidak, harus ditolak.
    5. **Gunakan HTTPS**: Pastikan endpoint webhook Anda menggunakan HTTPS untuk mengamankan komunikasi dan mencegah manipulasi selama transmisi data.
  Dengan menerapkan langkah-langkah ini, Anda dapat dengan percaya diri memverifikasi keaslian webhook yang dikirim oleh Dodo Payments.
</Accordion>

<Accordion title="Q3: Mengapa saya menerima webhook subscription.renew bahkan untuk pembayaran bulan pertama?">
  A: Untuk langganan, `subscription.renew` akan dipicu setiap kali jumlah langganan dipotong. Jadi ya, ini adalah perilaku yang diharapkan.
</Accordion>

<Accordion title="Q5: Bagaimana cara menguji webhook secara lokal?">
  A: Untuk menguji webhook secara lokal, Anda dapat mengikuti langkah-langkah berikut:
  
1. **Gunakan Alat Tunneling**: Alat seperti [ngrok](https://ngrok.com/), [LocalTunnel](https://localtunnel.github.io/www/), atau [Cloudflare Tunnel](https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/) dapat mengekspos server lokal Anda ke internet. Ini menyediakan URL publik yang dapat digunakan Dodo Payments untuk mengirim event webhook ke lingkungan lokal Anda.
2. **Siapkan Endpoint Webhook**: Buat endpoint webhook di aplikasi lokal Anda untuk menangani event webhook yang masuk. Pastikan ia mendengarkan permintaan HTTP POST.
3. **Konfigurasi Webhook di Dodo Payments**: Masuk ke dasbor Dodo Payments Anda, navigasikan ke pengaturan Webhooks, dan perbarui URL webhook ke URL publik yang dihasilkan oleh alat tunneling.
4. **Trigger Event**: Lakukan tindakan di lingkungan Anda untuk memicu event webhook. Misalnya, Anda dapat mensimulasikan pembayaran atau pembaruan langganan.
5. **Periksa Permintaan yang Masuk**: Gunakan log alat tunneling atau logging aplikasi Anda untuk memeriksa payload webhook yang masuk. Ini memungkinkan Anda memverifikasi bahwa endpoint Anda menerima event dan memprosesnya dengan benar.
6. **Verifikasi Tanda Tangan**: Pastikan Anda memvalidasi header **`webhook-signature`** untuk mengonfirmasi bahwa event webhook adalah autentik dan dikirim oleh Dodo Payments.

Dengan mengikuti langkah-langkah ini, Anda dapat dengan mudah menguji dan memecahkan masalah webhook di lingkungan pengembangan lokal Anda.
</Accordion>
