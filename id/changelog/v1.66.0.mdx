## Fitur Baru 🚀

1. **`subscription.updated` Webhook**  
   Peristiwa webhook baru yang aktif setiap kali objek langganan diperbarui, memberi Anda notifikasi waktu nyata tentang setiap perubahan pada langganan Anda.
   
   Gunakan webhook ini untuk:
   - Mendapatkan notifikasi instan ketika detail langganan berubah
   - Menjaga keadaan aplikasi Anda tetap sinkron dengan Dodo Payments
   - Merespons pembaruan langganan tanpa polling API
   
   <Tip>
   Berlangganan pada `subscription.updated` untuk menghilangkan kebutuhan polling dan menjaga sistem Anda tetap sinkron dengan perubahan langganan waktu nyata.
   </Tip>
   
   Pelajari selengkapnya: [Subscription Webhooks](/developer-resources/webhooks/intents/subscription)

2. **Dukungan Alamat Minimal di Checkout**  

   <Frame>
   <img src="/images/changelog/minimal-address.png" alt="Minimal Address Support in Checkout" />
   </Frame>

   Kurangi gesekan checkout dengan mode pengumpulan alamat minimal baru. Alih-alih mengumpulkan detail alamat lengkap, mode alamat minimal hanya mengumpulkan:
   
   - **Negara**: Selalu diperlukan untuk penentuan pajak
   - **Kode POS/ZIP**: Hanya dikumpulkan di wilayah di mana diperlukan untuk perhitungan pajak penjualan, PPN, atau GST
   
   <Check>
   Hasil: Penyelesaian checkout yang jauh lebih cepat dengan lebih sedikit kolom formulir bagi pelanggan.
   </Check>
   
   **Aktifkan alamat minimal:**
   
   <Tabs>
   {/* LOCKED_PATTERN_0090a508f27e87d00759ddcd3b9fe94c */}

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [{ product_id: 'prod_123', quantity: 1 }],
     minimal_address: true
   });
   ```

   </Tab>
   
   {/* LOCKED_PATTERN_ed987f4815610e04ce90e6f11a8998dc */}
   Tambahkan `minimalAddress=true` ke URL tautan pembayaran Anda:
   

   ```
   https://checkout.dodopayments.com/buy/prod_123?minimalAddress=true
   ```

   </Tab>
   </Tabs>
   
   <Info>
   Secara default, checkout tetap mengumpulkan alamat lengkap. Kami akan secara bertahap memindahkan semua tautan checkout ke mode alamat minimal. Aktifkan flag fitur sekarang untuk mulai menikmati manfaat pengurangan gesekan checkout.
   </Info>

3. **Kunci API Hanya-Baca**  
   Buat kunci API dengan izin hanya-baca untuk integrasi yang lebih aman. Saat membuat kunci API baru, kosongkan centang **Enable write access** untuk membuat kunci hanya-baca.
   
   Kunci hanya-baca dapat:
   - Mengambil data pembayaran, langganan, dan pelanggan
   - Daftar produk dan mengambil detailnya
   - Mengakses laporan dan analitik
   
   Kunci hanya-baca **tidak dapat**:
   - Membuat atau memodifikasi pembayaran
   - Memperbarui langganan
   - Mengubah konfigurasi produk
   
   <Tip>
   Kosongkan centang "Enable write access" untuk integrasi dashboard, alat analitik, dan sistem apa pun yang hanya perlu melihat data tanpa melakukan perubahan.
   </Tip>

4. **UPI AutoPay & Langganan Kartu RuPay - Peluncuran Penuh**  
   UPI AutoPay dan langganan kartu RuPay kini tersedia untuk semua bisnis. Ini memungkinkan pembayaran berulang bagi pelanggan India dengan:
   
   - **UPI AutoPay**: Penagihan berulang otomatis melalui mandat UPI
   - **Kartu RuPay**: Dukungan langganan untuk jaringan kartu domestik India
   
   <Check>
   Hasil: Pilihan pembayaran langganan yang diperluas untuk pasar India dengan penanganan mandat sesuai RBI.
   </Check>
   
   Pelajari selengkapnya: [Subscriptions with RBI-Compliant Mandates](/features/subscription#subscriptions-with-rbi-compliant-mandates)

5. **API Prabaca Perubahan Rencana**  
   Sebuah endpoint API baru untuk prabaca perubahan rencana langganan sebelum berkomitmen pada mereka. Dapatkan jumlah biaya yang tepat dan lihat bagaimana langganan akan terlihat setelah perubahan - tanpa melakukan modifikasi nyata.
   

   ```javascript
   const preview = await client.subscriptions.previewChangePlan('sub_123', {
     product_id: 'prod_pro_plan',
     proration_billing_mode: 'prorated_immediately',
     quantity: 1
   });
   
   console.log('Charge amount:', preview.immediate_charge.summary);
   console.log('New plan details:', preview.new_plan);
   ```

   
   Gunakan ini untuk:
   - Menunjukkan kepada pelanggan persis apa yang akan mereka bayar sebelum mengonfirmasi peningkatan/penurunan
   - Membangun kalkulator harga dan alat perbandingan rencana
   - Mengurangi tiket dukungan dengan meningkatkan transparansi harga
   
   Pelajari lebih lanjut: [API Prabaca Perubahan Rencana](/api-reference/subscriptions/preview-change-plan)
