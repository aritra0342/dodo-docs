## Fitur Baru 🚀

### Penagihan Berdasarkan Penggunaan 
Tagih pelanggan secara tepat berdasarkan konsumsi aktual. Definisikan meter, masukkan peristiwa, dan konversi penggunaan menjadi biaya secara otomatis.

- **Agregasi fleksibel**: hitung, jumlah (berdasarkan properti), unik, maksimum, terakhir
- **Penyaringan peristiwa**: logika AND/OR dengan pembanding kaya untuk menyertakan hanya peristiwa yang relevan
- **Keterkaitan produk**: lampirkan meter ke produk dengan harga per unit dan ambang batas gratis opsional
- **Analitik**: lacak tren, aliran peristiwa, dan konsumsi per pelanggan

### Diskon 100% untuk Langganan
Terapkan diskon 100% untuk sejumlah siklus penagihan untuk mendukung periode gratis, sponsor, atau kampanye promosi. Faktur mencerminkan diskon penuh dan prorata mengikuti pengaturan rencana Anda.

### Peningkatan Kecil

- API Penerimaan Pengembalian kini dapat diakses secara publik. Lihat dokumen: [/api-reference/refunds/get-refund-receipt](/api-reference/refunds/get-refund-receipt).
- Menambahkan `checkout session ID` ke respons API Get Payment dan webhook pembayaran untuk membantu mengaitkan pembayaran dengan sesi checkout asalnya.
