## Fitur Baru 🚀

### Peningkatan API
- Diluncurkan Invoice API, memungkinkan pedagang dan pelanggan untuk mengunduh faktur menggunakan `payment_id`. Kunjungi dokumentasi [Invoice API](/api-reference/payments/get-invoice) untuk detail lebih lanjut.
- Diimplementasikan acara webhook baru `subscription.renewed` untuk notifikasi perpanjangan langganan.

### Pembaruan Platform
- Memperkenalkan Dodo Payments [Wall of Love](https://dodopayments.com/wall-of-love), menampilkan testimoni pelanggan.
- Menambahkan fungsionalitas berbagi sosial untuk transaksi yang berhasil.
    <img src="/images/changelog/share-transaction.png" alt="Fitur Berbagi Transaksi"/>
- Merilis alat Pembuat Kebijakan untuk pembuatan otomatis dokumen kebijakan privasi, syarat layanan, dan kebijakan pengembalian. [Coba Pembuat Kebijakan](https://dodopayments.com/tools/policy-generator).

### Peningkatan Dokumentasi
- Menerbitkan panduan komprehensif:
    - [Panduan Integrasi Langganan](/developer-resources/subscription-integration-guide)
    - [Panduan Acara Webhook](/developer-resources/webhooks/intents/webhook-events-guide)

## Peningkatan dan Perbaikan Bug 🔧

### Pembaruan API
- Menambahkan parameter `trial_period_days` ke [Create Subscription API](/api-reference/subscriptions/post-subscriptions) untuk menimpa periode percobaan tingkat produk.
- Meningkatkan logika identifikasi pelanggan: sekarang menggunakan `customer_id` yang konsisten di seluruh pembayaran dan langganan untuk email yang sama.
- Diimplementasikan berbagai optimasi kinerja dan memperbaiki bug kecil.
