### Ketergantungan Lingkungan

**Bergantung pada Lingkungan:** Semua fitur yang Bergantung pada Lingkungan akan memiliki data yang independen di Mode Uji dan Mode Langsung. Setiap perubahan yang dilakukan di satu lingkungan tidak akan mempengaruhi lingkungan lainnya. Misalnya, Kunci API yang dihasilkan di Mode Uji akan independen dari kunci API yang dihasilkan di Mode Langsung dan sebaliknya.

**Tidak Bergantung pada Lingkungan:** Semua fitur yang Tidak Bergantung pada Lingkungan akan memiliki data yang sama di kedua mode langsung dan mode uji. Misalnya, bagian Akun Saya akan memiliki data yang sama di mode uji dan mode langsung. Perubahan yang dilakukan di satu lingkungan akan mempengaruhi lingkungan lainnya.

Mode Langsung hanya akan tersedia setelah berhasil menyelesaikan verifikasi Identitas dan verifikasi Bisnis. Anda hanya dapat menerima pembayaran nyata di \`live mode\`.

Hostname API untuk Mode Uji dan Mode Langsung adalah di bawah ini:
- Mode Uji: [`https://test.dodopayments.com`](https://test.dodopayments.com/)
- Mode Langsung: [`https://live.dodopayments.com`](https://live.dodopayments.com/)

## Sorotan Utama

1. Semua produk yang dibuat di mode uji atau mode langsung hanya akan tersedia di lingkungan masing-masing. Ada opsi untuk menyalin ke mode langsung di lingkungan mode uji. Ini akan menyalin produk dengan semua detail relevan dan membuat yang sama di mode langsung.

2. Kunci API dan Webhook yang dibuat dalam mode uji hanya akan terbatas pada lingkungan mode uji dan tidak akan memengaruhi penyebaran mode langsung, begitu pula sebaliknya. Anda dapat menggunakan [CLI webhook listener](/developer-resources/sdks/cli#listen-for-webhooks) (`dodo wh listen`) untuk meneruskan webhook mode uji ke server pengembangan lokal Anda secara real time

3. Semua dokumen yang diajukan untuk verifikasi dan detail rekening bank yang diajukan akan sama di mode uji dan mode langsung. Perubahan informasi ini di lingkungan mana pun juga akan mengubahnya di lingkungan lainnya.

## Kartu Uji dan ID UPI

Untuk informasi lebih lanjut tentang kartu uji, ID UPI, dan metode pengujian lainnya, silakan merujuk ke panduan komprehensif kami tentang [Proses Pengujian](/miscellaneous/testing-process).

Panduan ini mencakup:
- Nomor kartu uji untuk berbagai skenario
- ID UPI uji untuk mensimulasikan pembayaran
- Instruksi untuk menguji berbagai hasil pembayaran
- Praktik terbaik untuk menguji integrasi Anda

<Note>
Metode pembayaran uji hanya berlaku dalam mode uji. Jangan pernah menggunakannya untuk pembayaran nyata.
</Note>
