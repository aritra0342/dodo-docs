## Memulai & Pengaturan Akun

<Accordion title="Q1: Bagaimana cara saya masuk ke Dasbor Dodo Payments?">
  A: Kunjungi dasbor kami dan masukkan email dan kata sandi yang dibagikan kepada Anda dalam email sambutan. Jika Anda mengalami masalah, silakan hubungi tim dukungan kami.
</Accordion>

<Accordion title="Q2: Dokumen apa yang saya butuhkan untuk menyelesaikan proses verifikasi?">
  A: Dokumen yang diperlukan untuk verifikasi termasuk ID yang dikeluarkan pemerintah yang valid (untuk verifikasi identitas) dan dokumen pendaftaran bisnis (untuk verifikasi bisnis). Untuk detail lebih lanjut, lihat [halaman proses verifikasi](/miscellaneous/verification-process).
</Accordion>

<Accordion title="Q3: Apa yang terjadi jika verifikasi saya ditolak?">
  A: Jika verifikasi Anda ditolak, Anda akan menerima email dengan alasan penolakan. Anda dapat menghubungi tim dukungan kami untuk menyelesaikan masalah, memperbaiki dokumentasi, dan mengajukan kembali. Opsi pengajuan ulang akan diaktifkan oleh tim verifikasi kami jika diperlukan.
</Accordion>

<Accordion title="Q4: Berapa lama proses verifikasi biasanya berlangsung?">
  A: Proses verifikasi biasanya memakan waktu **24-72 jam kerja**. Namun, jika dokumen tambahan diperlukan, prosesnya mungkin memakan waktu lebih lama.
</Accordion>

<Accordion title="Q5: Saat ini saya tidak memiliki situs web untuk bisnis saya. Bisakah saya mengirimkan halaman linkedin perusahaan saya sebagai gantinya?">
  A: Kami memerlukan situs web Anda untuk memahami jenis produk yang Anda jual dan memverifikasi bisnis Anda untuk akses langsung. Jika Anda tidak memiliki situs web atau sedang dalam proses membangunnya, silakan hubungi kami di support@dodopayments.com
</Accordion>

<Accordion title="Q6: Saya adalah individu dan tidak memiliki bisnis terdaftar. Bisakah saya tetap menggunakan Dodo Payments?">
  A: Ya, kami juga mendukung bisnis yang tidak terdaftar. Anda tidak perlu memiliki bisnis terdaftar untuk menggunakan Dodo Payments. Anda dapat mendaftar sebagai individu dan mulai menerima pembayaran internasional tanpa kesulitan. Anda tetap perlu mengajukan formulir Verifikasi Identitas dan formulir Verifikasi Bisnis sebagai bisnis yang tidak terdaftar agar kami dapat mendaftar Anda.
</Accordion>

<Accordion title="Q7: Saya ingin mengintegrasikan bisnis lain saya dengan Dodo Payments. Haruskah saya membuat akun baru?">
  A: Tidak, Anda dapat mengelola beberapa bisnis melalui satu akun. Untuk menambahkan bisnis baru, klik ikon + di sudut kiri bawah dasbor dan pilih opsi Tambah Bisnis.
Anda dapat merujuk ke /miscellaneous/accounts#managing-multiple-businesses untuk detail lebih lanjut
</Accordion>

<Accordion title="Q8: Haruskah saya menunggu verifikasi selesai sebelum mengintegrasikan Dodo Payments ke dalam sistem saya?">
  A: Tidak, Anda tidak perlu menunggu. Anda dapat mulai mengintegrasikan menggunakan mode Uji dan setelah verifikasi Anda berhasil, Anda dapat mulai melakukan pembayaran Langsung di produksi. Pelajari lebih lanjut tentang mode Uji dan Langsung di sini,
/miscellaneous/test-mode-vs-live-mode
</Accordion>

<Accordion title="Q9: Bagaimana cara mendapatkan akses mode langsung?">
  A: Setelah akun Anda dibuat, Anda akan secara otomatis mendapatkan akses ke mode langsung.
</Accordion>

  A: Ya, kami menawarkan program afiliasi!

<Accordion title="Q11: Bagaimana cara menghapus produk/pelanggan yang telah saya buat?">

</Accordion>

<Accordion title="Q12: Bagaimana cara saya memiliki UI kustom untuk halaman pembayaran?">

  - Memperbarui logo, nama merek, dan email dukungan Anda.

  Sebagai gantinya:
  - Alihkan pengguna ke URL checkout Dodo
  - Atau gunakan Fitur Overlay Checkout kami. [Dokumentasi Overlay Checkout](/developer-resources/overlay-checkout)
</Accordion>

<Accordion title="Q15: Bisakah saya membuat pembayaran checkout yang dihosting satu kali tanpa menambahkan produk ke Dodo Payments?">
  A: Tidak. Produk adalah wajib dalam sistem Dodo Payments.
</Accordion>

<Accordion title="Q16: Apakah Anda mendukung situs web e-commerce?">
  A: Tidak, kami tidak mendukung situs web e-commerce.
</Accordion>

<Accordion title="Q17: Bisakah saya beralih antara merek di Dodo Payments?">
  A: Ya, Anda dapat beralih antara merek untuk produk. Anda dapat memilih merek yang diinginkan saat membuat atau mengedit produk di dasbor Anda.
</Accordion>

<Accordion title="Q18: Bisakah saya menghapus merek atau memperbarui URL Merek di Dodo Payments?">
  A: Tidak, Dodo Payments saat ini tidak mendukung:
  - **Menghapus merek**: Jika merek tidak digunakan, Anda dapat mengabaikannya — itu tidak akan mempengaruhi akun atau operasi Anda
  - **Mengubah atau memperbarui URL Merek**: Setelah dibuat, URL Merek tidak dapat dimodifikasi. Bahkan jika URL salah, itu tidak mempengaruhi pengaturan atau fungsionalitas Anda

</Accordion>

## Verifikasi, Pajak & Kepatuhan

<Accordion title="Q21: Mengapa verifikasi saya memakan waktu lama?">
  A: Dodo Payments biasanya memerlukan waktu 24–72 jam kerja untuk memproses verifikasi. Namun, penundaan dapat terjadi karena:
  - Volume aplikasi yang tinggi
  - Akhir pekan dan hari libur
  - Pengajuan yang tidak lengkap atau salah
  - Tinjauan manual untuk risiko atau kepatuhan

  <Note>
  Jika sudah lebih dari 72 jam, silakan periksa email Anda atau dasbor Dodo Payments untuk tindakan yang diperlukan. Anda juga dapat mengirim DM ke email terdaftar Anda kepada tim di Discord untuk menindaklanjuti.
  </Note>
</Accordion>

<Accordion title="Q22: Saya telah mengirimkan formulir W-8BEN saya, tetapi bagian tanda tangan tidak berfungsi dengan baik. Apa yang harus saya lakukan?">
  A: Gunakan editor PDF yang tepat seperti:
  - Adobe Acrobat
  - Mac Preview (bekerja untuk sebagian besar)
  - Smallpdf atau PDFescape (editor online)

  Jika ada tanda silang pada tanda tangan, pastikan:
  - Anda menyimpan file setelah menerapkan tanda tangan.
  - Alat tersebut meratakan lapisan tanda tangan (beberapa editor tidak).

  <Tip>
  Hindari menandatangani di penampil browser — mereka tidak selalu menyematkan tanda tangan digital dengan benar.
  </Tip>
</Accordion>

<Accordion title="Q23: Saya memilih jenis pendaftaran yang salah (bisnis vs individu). Bagaimana cara memperbaikinya?">
  A: Jika Anda memilih jenis entitas yang salah (misalnya, memilih "Organisasi" tetapi Anda adalah individu atau Pemilik Tunggal dengan GST), verifikasi Anda mungkin ditahan, memerlukan Anda untuk mengubah atau memperbarui dokumentasi. 

  Langkah-langkah untuk memperbaiki:
  Langkah 1: Informasikan dukungan melalui Intercom atau Discord yang akan membantu Anda mengaktifkan pengajuan ulang untuk formulir Kepatuhan Payout.
  Langkah 2: Pilih jenis pendaftaran yang benar, dan kirim ulang dokumen yang diperlukan.

</Accordion>

<Accordion title="Q24: Verifikasi saya ditolak karena ketidakcocokan entitas. Apa yang harus saya lakukan?">
  A: Jika verifikasi Anda ditolak karena ketidakcocokan entitas (misalnya, Anda memilih "Organisasi" tetapi Anda adalah individu atau Pemilik Tunggal dengan GST), Anda perlu memperbaiki jenis pendaftaran dan mengirim ulang dokumen Anda.

  Langkah-langkah untuk memperbaiki:
  Langkah 1: Periksa dasbor atau email Anda untuk alasan spesifik penolakan.
  Langkah 2: Hubungi dukungan melalui Intercom atau email dukungan (support@dodopayments.com) untuk meminta akses pengajuan ulang untuk formulir Kepatuhan Payout.
  Langkah 3: Pilih jenis pendaftaran yang benar (Individu vs Organisasi) dan kirim ulang semua dokumen yang diperlukan.
  Langkah 4: Tunggu tim kepatuhan untuk meninjau pengajuan ulang Anda.

  <Note>
  Opsi pengajuan ulang akan diaktifkan oleh tim verifikasi kami setelah Anda menghubungi dukungan. Pastikan untuk memilih jenis entitas yang benar kali ini untuk menghindari penundaan lebih lanjut.
  </Note>
</Accordion>

<Accordion title="Q25: Bisakah saya menggunakan nama yang berbeda (seperti nama orang tua saya) untuk mendaftar dan mendapatkan verifikasi?">
  A: Tidak. Orang yang mendaftar harus sesuai dengan pemilik sah bisnis dan semua dokumen yang diajukan.

  Jika orang tua Anda adalah pemilik bisnis yang sebenarnya:
  - Mereka harus mendaftar menggunakan email dan nomor ponsel mereka.
  - Semua dokumen harus atas nama mereka.

  <Note>
  Menggunakan nama orang lain untuk verifikasi sementara Anda mengelola bisnis dapat menyebabkan penolakan kepatuhan.
  </Note>
</Accordion>

<Accordion title="Q26: Bagaimana cara memeriksa alasan penolakan verifikasi?">
  A: Anda dapat menemukan alasan di:
  - Dasbor Dodo Payments, di bagian verifikasi.
  - Notifikasi email (jika diaktifkan)

  <Tip>
  Jika tidak terlihat, hubungi compliance@dodopayments.com atau DM ID email terdaftar Anda kepada anggota tim.
  </Tip>
</Accordion>

<Accordion title="Q27: Bisakah saya membuat akun bisnis baru jika akun saya sebelumnya ditolak?">
  A: Ya — tetapi hanya jika:
  - Anda telah memperbaiki masalah yang menyebabkan penolakan.
  - Anda tidak mencoba untuk menghindari keputusan risiko (misalnya, deboarding karena masalah penipuan).

  <Note>
  Membuat beberapa akun tanpa menyelesaikan masalah yang mendasari kemungkinan akan mengakibatkan akun baru juga ditolak.
  </Note>
</Accordion>

<Accordion title="Q28: Saya diberitahu bahwa akun saya sebelumnya telah dideboard. Apa artinya ini?">
  A: Jika Anda melihat pesan yang menyatakan akun Anda sebelumnya telah dideboard oleh tim risiko:
  - Itu berarti Dodo Payments menemukan masalah kepatuhan atau risiko yang signifikan dengan aktivitas atau bisnis Anda.
  - Mencoba membuat akun baru menggunakan identitas atau detail bisnis yang sama kemungkinan akan ditolak lagi.

  <Note>
  Jika Anda percaya ini adalah kesalahan, kirim email ke compliance@dodopayments.com dan minta tinjauan — sertakan bukti yang jelas untuk mendukung kasus Anda.
  </Note>
</Accordion>

<Accordion title="Q29: Bagaimana cara mempercepat verifikasi payout saya?">
  A: Untuk menghindari penundaan:
  - Kirim dokumen yang jelas dan terformat dengan baik
  - Hindari pesan/tags berulang di Discord — ini memperlambat antrean
  - Sebutkan jika Anda telah melewati ambang payout atau memiliki tenggat waktu mendesak
  - DM ID email terdaftar Anda jika diminta

  <Tip>
  Sebagian besar tinjauan payout diselesaikan dalam waktu 72 jam kerja, kecuali ada masalah dengan pengajuan Anda.
  </Tip>
</Accordion>

<Accordion title="Q30: Apa yang terjadi jika saya memilih formulir yang salah (W-8BEN vs W-8BEN-E)?">
  A: Formulir ini terkait dengan jenis entitas Anda:
  - W-8BEN adalah untuk Individu
  - W-8BEN-E adalah untuk Organisasi

  Jika Anda mengirimkan yang salah:
  - Hubungi dukungan untuk mereset formulir Anda.
  - Isi dan unggah jenis formulir yang benar.
  - Pastikan PAN ada di kolom yang benar (Baris 6a untuk individu).

  <Tip>
  Rujuk ke panduan formulir resmi IRS jika bingung.
  </Tip>
</Accordion>

<Accordion title="Q31: Saya memiliki GST dengan nama dagang. Apakah itu dianggap sebagai organisasi?">
  A: Tidak. Jika Anda terdaftar dengan GST sebagai individu (pemilik tunggal), Anda masih dianggap sebagai Individu.

  ✅ Gunakan W-8BEN
  ✅ Pilih "Individu" dalam pendaftaran
  ❌ Jangan mengisi W-8BEN-E atau memilih "Organisasi" kecuali Anda adalah entitas hukum yang terdaftar
</Accordion>

<Accordion title="Q32: Akun saya telah dideboard. Bisakah saya hanya membuat yang baru?">
  A: Membuat akun baru untuk menghindari deboarding sebelumnya tidak diperbolehkan. Jika Anda mencoba ini:
  - Akun baru juga mungkin akan ditandai atau ditolak
  - Keputusan risiko dibawa ke seluruh pendaftaran jika bisnis atau orang yang mendasarinya sama

  <Tip>
  Jika Anda percaya Anda telah dideboard secara tidak adil, kirim email ke compliance@dodopayments.com dengan kasus Anda.
  </Tip>
</Accordion>

<Accordion title="Q33: Bagaimana Dodo Payments menangani GST untuk pedagang SaaS India?">
  A: Dodo Payments secara otomatis menghitung dan menerapkan **GST** untuk pedagang India berdasarkan lokasi pelanggan Anda.
</Accordion>

<Accordion title="Q34: Apakah Dodo Payments menangani penjualan internasional?">
  A: Ya, Dodo Payments mendukung penjualan internasional dan mengelola semua kepatuhan yang terkait.
</Accordion>

<Accordion title="Q35: Apakah data alamat seperti kota, negara bagian, dan kode pos wajib? Bisakah saya mengecualikan permintaan nilai tersebut dari pelanggan saya?">
  A: Ya, itu wajib. Karena kami adalah Merchant of Record dan menghasilkan faktur, mengajukan pajak, dan memastikan semua pembelian sesuai, kami memerlukan alamat penagihan pelanggan. Anda dapat menerapkan alur checkout berbasis API kami di mana Anda dapat mengisi detail pelanggan sebelumnya jika Anda sudah mengumpulkannya untuk memastikan pengalaman yang lebih lancar bagi pelanggan Anda, tetapi kami akan memerlukan detail ini untuk setiap pembelian.
</Accordion>

<Accordion title="Q36: Apakah saya perlu mengajukan pajak di AS jika saya menjual di sana?">
  A: Tidak secara langsung. Dodo Payments bertindak sebagai Merchant of Record, menangani pajak sisi pelanggan, dan membayar di mana berlaku. Anda masih perlu:
  - Mengajukan pajak penghasilan di negara asal Anda (misalnya, India)
  - Mengirimkan W-8BEN(-E) untuk menghindari pemotongan pajak 30% AS secara default
</Accordion>

<Accordion title="Q37: Saya tidak tahu apakah harus mengisi W-8BEN atau W-8BEN-E — bantu?">
  A: Gunakan logika berikut untuk menentukan formulir mana yang Anda butuhkan:

  | Anda adalah... | Formulir yang Harus Dikirim | Jenis Merchant |
  |------------|----------------|---------------|
  | Pemilik Tunggal, Freelancer, Individu | W-8BEN | Individu |
  | Pvt Ltd, LLP, Organisasi Terdaftar | W-8BEN-E | Organisasi |

  Masih tidak yakin? Pertimbangkan indikator ini:
  - **Jika rekening bank Anda atas nama pribadi Anda**, Anda kemungkinan adalah Individu → Gunakan W-8BEN
  - **Jika rekening bank Anda atas nama perusahaan**, Anda adalah Organisasi → Gunakan W-8BEN-E
  - **Jika Anda memiliki GST tetapi beroperasi dengan rekening bank pribadi**, Anda masih dianggap sebagai Individu → Gunakan W-8BEN

  <Tip>
  Jika Anda adalah pemilik tunggal dengan GST, Anda masih seorang Individu. Hanya entitas hukum terdaftar (Pvt Ltd, LLP, dll.) yang harus menggunakan W-8BEN-E.
  </Tip>
</Accordion>

<Accordion title="Q38: Apakah saya perlu PAN atau GST untuk menerima pembayaran?">
  A: Untuk Individu, PAN adalah persyaratan wajib (untuk KYC dan tujuan pajak). Untuk bisnis terdaftar, baik PAN Bisnis atau GSTIN dapat diajukan. GST bersifat opsional kecuali Anda melebihi ambang pendaftaran atau ingin mendapatkan kredit masukan
</Accordion>

<Accordion title="Q39: Saya memiliki sertifikat Udyam/MSME. Haruskah saya mengirimkan W-8BEN atau W-8BEN-E?">
  A: Memiliki sertifikat Udyam/MSME tidak secara otomatis menjadikan Anda sebagai Organisasi. Formulir yang Anda kirimkan tergantung pada jenis rekening bank Anda:

  - **Jika Anda beroperasi dengan rekening bank pribadi** (bahkan dengan sertifikat Udyam/MSME): Anda dianggap sebagai **Individu** → Kirim **W-8BEN**
  - **Jika Anda beroperasi dengan rekening bank perusahaan** (entitas terdaftar): Anda adalah **Organisasi** → Kirim **W-8BEN-E**

  <Note>
  Faktor kunci adalah apakah rekening bank Anda atas nama pribadi Anda atau nama perusahaan Anda, bukan apakah Anda memiliki sertifikat pendaftaran bisnis.
  </Note>
</Accordion>

<Accordion title="Q40: Jika Dodo menangani pajak, mengapa saya perlu mengajukan W-8BEN(-E)?">
  A: Dodo menangani pajak penjualan untuk pelanggan Anda. W-8BEN(-E) diperlukan untuk pajak pemotongan pada pembayaran kepada Anda.

  <Note>
  Mengajukan W-8BEN dengan benar dapat membantu Anda menghindari pajak berganda berdasarkan perjanjian pajak India–AS.
  </Note>
</Accordion>

## Pembayaran & Transaksi

<Accordion title="Q41: Jenis pembayaran apa yang dapat saya terima dengan Dodo Payments?">
  A: Dodo Payments mendukung berbagai metode pembayaran, termasuk **kartu kredit**, **kartu debit**, dan **dompet digital**. Kami juga mendukung pembayaran satu kali dan pembayaran langganan berulang untuk produk digital.
</Accordion>

<Accordion title="Q42: Mengapa pembayaran pertama ditandai berhasil dengan jumlah $0 untuk langganan?">
  A: Jika produk memiliki periode percobaan, pembayaran pertama secara otomatis menjadi pembayaran mandat sebesar $0. Ini adalah perilaku yang diharapkan dan mengonfirmasi bahwa mandat telah berhasil dibuat.
</Accordion>

<Accordion title="Q43: Setelah periode percobaan berakhir, apakah pelanggan perlu membayar (harga produk (misalnya $25)) lagi secara manual?">
  A: Tidak. Setelah percobaan berakhir, sistem secara otomatis menghasilkan harga produk (misalnya $25) untuk pelanggan. Mereka tidak perlu membayarnya secara manual.
</Accordion>

<Accordion title="Q44: Apa yang terjadi jika periode percobaan menunjukkan berbeda di aplikasi saya vs. dasbor Dodo Payments?">
  A: Ini biasanya menunjukkan masalah pengaturan atau implementasi di pihak Anda. Uji alur percobaan + langganan dalam mode uji terlebih dahulu, konfirmasi itu berfungsi, dan kemudian beralih ke mode langsung.
</Accordion>

<Accordion title="Q45: Apakah saya perlu menguji fitur percobaan sebelum pergi langsung?">
  A: Ya. Disarankan untuk sepenuhnya menerapkan dan menguji fitur dalam mode uji untuk memastikan semuanya berfungsi dengan benar sebelum beralih ke mode langsung.
</Accordion>

<Accordion title="Q46: Bagaimana saya dapat melacak status pembayaran?">
  A: Anda dapat melacak status pembayaran dari bagian **Pembayaran** di dasbor Anda. Setiap transaksi terdaftar dengan indikator status: **Dalam Proses**, **Gagal**, atau **Berhasil**.
</Accordion>

<Accordion title="Q47: Apa yang harus saya lakukan jika pembayaran gagal?">
  A: Jika pembayaran gagal, periksa **Halaman Detail Pembayaran** untuk informasi lebih lanjut. Alasan umum termasuk detail pembayaran yang salah, dana tidak mencukupi, atau masalah jaringan. Anda dapat mengirim ulang tautan pembayaran kepada pelanggan Anda untuk mencoba pembayaran lagi.
</Accordion>

<Accordion title="Q48: Apakah pedagang diberi tahu ketika pembayaran pengguna gagal?">
  A: Ya. Dodo Payments memberi tahu pedagang ketika pembayaran pengguna gagal melalui beberapa saluran dalam mode langsung:
  - Notifikasi email
  - Peristiwa webhook
  - Notifikasi seluler
  - Notifikasi dasbor
Ini memastikan pedagang tetap terinformasi tentang transaksi yang gagal.
</Accordion>

<Accordion title="Q49: Apa yang terjadi pada langganan jika pembayaran gagal?">
  A: Langganan tidak dibatalkan segera. Sebaliknya, statusnya dipindahkan ke ditahan sampai pembayaran yang berhasil dilakukan.

  Jika Anda memiliki masalah dengan kartu, Anda dapat memperbarui metode pembayaran dengan:
  - Mengklik langganan yang ditahan di dasbor Anda dan menggunakan tombol **Perbarui Metode Pembayaran**
  - Atau menggunakan endpoint API: https://docs.dodopayments.com/api-reference/subscriptions/update-payment-method
</Accordion>

<Accordion title="Q51: Bagaimana pembayaran langganan bisa gagal dengan INCORRECT_NUMBER jika siklus sebelumnya berhasil?">

Dodo Payments hanya menampilkan pesan kesalahan yang dikembalikan oleh penyedia pembayaran.

<Accordion title="Q52: Apakah chargeback berlaku untuk bisnis meskipun bisnis saya memiliki kebijakan tidak ada pengembalian dana?">
  A: Ya. Bahkan jika Anda menyatakan "Tidak ada pengembalian dana" dalam syarat atau situs web Anda, chargeback sebagian besar diatur oleh bank pemegang kartu dan aturan jaringan kartu (misalnya, Visa, Mastercard). Kebijakan "Tidak ada pengembalian dana" tidak membatalkan proses penyelesaian sengketa jaringan kartu. Jika seorang pelanggan mengajukan sengketa, bisnis Anda tetap tunduk pada penyelidikan chargeback. Ini berarti Anda harus merespons dengan bukti (bukti pengiriman, log penggunaan, penerimaan syarat, dll.) untuk melawan atau mengelola sengketa. Jika bank pelanggan memutuskan mendukung pemegang kartu, biaya chargeback dan pembalikan transaksi akan berlaku terlepas dari kebijakan yang Anda nyatakan. Kami merekomendasikan untuk memiliki komunikasi yang jelas, melacak penggunaan pelanggan, dan memberikan bukti yang meyakinkan ketika chargeback muncul.
</Accordion>

<Accordion title="Q53: Mengapa payment.status terkadang null?">

  - Tepat setelah pembuatan tetapi sebelum pemrosesan selesai

  Akhirnya, itu menjadi salah satu dari: berhasil, gagal, atau dibatalkan.

  <Note>
  Gunakan peristiwa webhook seperti payment.succeeded atau payment.failed untuk pelacakan status yang lebih dapat diandalkan.

</Accordion>

  - Adaptive Currency dinonaktifkan.  

  Catatan: Jika Anda adalah pedagang non-India dan Adaptive Currency tidak diaktifkan, UPI tidak akan tersedia untuk pelanggan Anda.

<Accordion title="Q59: Apakah Dodo mendukung UPI untuk langganan?">
  A: Ya, langganan UPI didukung dengan mandat yang sesuai dengan RBI. Langganan UPI beroperasi di bawah regulasi RBI (Reserve Bank of India) dengan persyaratan tertentu:

  - **Batas Mandat**:
    - Biaya di bawah Rs 15.000: Mandat sesuai permintaan untuk Rs 15.000 INR dibuat
    - Biaya Rs 15.000 atau lebih: Mandat langganan untuk jumlah langganan yang tepat dibuat
  - **Penundaan Pemrosesan 48 Jam**: Biaya dimulai pada tanggal yang dijadwalkan, tetapi pemotongan sebenarnya terjadi 48 jam setelah inisiasi (dapat diperpanjang hingga 2-3 jam tambahan)
  - **Pembatalan Mandat**: Pelanggan dapat membatalkan mandat melalui aplikasi perbankan mereka selama jendela 48 jam

  <Note>
  Langganan UPI memiliki persyaratan mandat yang sesuai dengan RBI yang sama seperti langganan kartu India. Untuk informasi lebih lanjut, lihat dokumentasi [Metode Pembayaran](/features/payment-methods#upi-and-rupay-india).
  </Note>
</Accordion>

<Accordion title="Q63: Apa perbedaan antara tautan pembayaran statis dan dinamis?">

  - Dikonfigurasi dari dasbor
  - Secara otomatis mengumpulkan info penagihan
  - Tidak dapat disesuaikan per pelanggan/sesi

  **Tautan pembayaran dinamis** dibuat melalui API. Mereka:
  - Memerlukan Anda untuk mengirimkan info penagihan secara eksplisit
  - Dapat disesuaikan secara real-time (harga, ID pengguna, URL pengalihan, dll.)

  <Tip>
  Jika Anda tidak ingin menulis kode, pilih tautan statis. Jika Anda memerlukan fleksibilitas atau ingin membangun alur kerja kustom, gunakan tautan dinamis.

</Accordion>

<Accordion title="Q65: Apa arti subscription.cancelled_at? Apakah itu pembatalan terjadwal?">

Biaya dimulai pada tanggal yang dijadwalkan, tetapi pemotongan sebenarnya dari rekening pelanggan terjadi 48 jam setelah inisiasi. Jendela 48 jam ini dapat diperpanjang hingga 2-3 jam tambahan tergantung pada respons API bank.

  <Note>

## Payout & Integrasi Bank

<Accordion title="Q75: Bagaimana cara kerja payout dengan Dodo Payments?">
  A: Payout adalah transfer pendapatan penjualan yang Anda kumpulkan ke rekening bank yang terhubung. Dodo Payments mendukung beberapa siklus payout:

  - **Dua Bulanan (Default)**:
    - Periode Penagihan 1: 1 hingga 15 → Payout pada tanggal 18 bulan yang sama
    - Periode Penagihan 2: 16 hingga akhir bulan → Payout pada tanggal 4 bulan berikutnya
  - **Mingguan**: Tersedia untuk bisnis dengan volume transaksi yang lebih tinggi (hubungi dukungan untuk meminta)
  - **Bulanan**:
    - Periode Penagihan: 1 hingga akhir bulan → Payout pada tanggal 11 bulan berikutnya

  Payout hanya dimulai jika semua verifikasi berhasil dan ambang payout terpenuhi. Biasanya memerlukan waktu 1-2 hari kerja agar dana muncul di rekening bank Anda setelah payout dimulai.

  <Note>
  Penundaan dapat terjadi jika tanggal pemrosesan payout jatuh pada akhir pekan atau hari libur bank. Siklus payout default untuk semua bisnis adalah dua bulanan.
  </Note>
</Accordion>

<Accordion title="Q76: Apa ambang batas minimum setelah biaya & pajak untuk memproses payout?">
  A: Ambang payout minimum adalah **$50 USD** (atau Rs 1.000 INR) setelah pemotongan biaya platform dan pajak. Anda dapat menyesuaikan ambang dompet USD Anda ke jumlah **≥ $50** langsung dari dasbor Anda. Jika saldo Anda tidak memenuhi ambang, itu akan dibawa ke siklus payout berikutnya.

  Anda dapat melihat jumlah dompet yang tepat dan mengelola ambang payout Anda di [bagian payout](https://app.dodopayments.com/business/payouts) dari dasbor Anda.

  <Note>
  Fitur ambang fleksibel saat ini tersedia untuk dompet USD/EUR/GBP. Ambang dompet INR tetap tetap di Rs 1.000.
  </Note>
</Accordion>

<Accordion title="Q77: Bagaimana cara menghubungkan rekening bank saya untuk menerima payout?">
  A: Untuk menghubungkan rekening bank Anda, navigasikan ke bagian **Pengaturan Payout** di dasbor Anda dan klik **Tambahkan Rekening Bank**. Anda perlu memasukkan detail bank Anda dan memverifikasi akun tersebut.
</Accordion>

<Accordion title="Q79: Apa itu biaya payout dan berapa biayanya?">

Kami terus melakukan perbaikan untuk mengurangi biaya payout dengan bermitra dengan mitra lokal di negara Anda. Silakan lihat [harga](https://dodopayments.com/pricing) untuk informasi lebih lanjut.
</Accordion>

<Accordion title="Q80: Saya menerima email &quot;payout diproses&quot; — kapan dana akan tiba?">
  A: Payout biasanya memerlukan waktu 1–2 hari kerja untuk tercermin di rekening bank Anda.

  <Note>
  Transfer tidak diselesaikan pada akhir pekan atau hari libur bank. Jika payout Anda diproses pada hari Jumat, harapkan itu pada hari Senin atau Selasa.

<Accordion title="Q81: Bagaimana cara menambahkan Payoneer sebagai akun payout?">

<Accordion title="Q88: Negara saya tidak menggunakan kode SWIFT. Bagaimana hal itu akan mempengaruhi verifikasi saya?">
  A: Jika negara Anda tidak mengeluarkan kode SWIFT, ini tidak akan menghalangi verifikasi Anda, tetapi akan memerlukan tinjauan manual oleh tim kepatuhan kami.

  - Kirim dokumen resmi bank yang mengonfirmasi:
    - Nomor rekening Anda
    - Nama pemegang rekening
    - Bahwa tidak ada kode SWIFT yang dikeluarkan di negara Anda
  - Dalam beberapa kasus, kode SWIFT placeholder generik dapat digunakan berdasarkan negara Anda

  <Note>
  Kasus ini akan melalui tinjauan kepatuhan manual. Untuk pemrosesan yang lebih cepat, tulis ke compliance@dodopayments.com untuk eskalasi.

<Accordion title="Q89: Dokumentasi alternatif apa yang dapat saya berikan jika negara saya tidak menggunakan kode SWIFT?">
  A: Jika negara Anda tidak menggunakan kode SWIFT, berikan dokumentasi alternatif berikut:

  - Pernyataan bank resmi atau sertifikat yang menunjukkan detail rekening Anda
  - Surat bank yang mengonfirmasi nomor rekening Anda, nama pemegang rekening, dan bahwa kode SWIFT tidak dikeluarkan
  - Dokumen yang dikeluarkan pemerintah yang mengonfirmasi format informasi routing bank Anda

  <Tip>
  Hubungi support@dodopayments.com sebelum mengirimkan untuk memastikan Anda memberikan format dokumentasi yang benar untuk negara Anda.
  </Tip>
</Accordion>

<Accordion title="Q101: Bisakah saya memverifikasi webhook Dodo Payments secara manual dalam Python?">

  ```python
  from standardwebhooks.webhooks import Webhook
  
  wh = Webhook(webhook_secret_key)
  payload = request.get_data(as_text=True)
  headers = request.headers
  
  if not wh.verify(payload, headers):
      return jsonify({'error': 'Invalid webhook signature'}), 401
 
  Then parse the payload as needed.
  ```

  </Accordion>

<Accordion title="Q108: Bagaimana cara memverifikasi bahwa webhook yang saya terima benar-benar dari Dodo Payments dan bukan dari aktor jahat?">
A: Untuk memverifikasi bahwa webhook yang Anda terima benar-benar dari Dodo Payments dan bukan dari aktor jahat, ikuti langkah-langkah ini:
1. **Periksa Tanda Tangan**: Setiap webhook yang dikirim oleh Dodo Payments menyertakan tanda tangan unik dalam header **`webhook-signature`**. Tanda tangan ini dihasilkan menggunakan kunci rahasia webhook Anda dan payload peristiwa.
2. **Ambil Kunci Rahasia**: Pastikan Anda memiliki kunci rahasia untuk webhook Anda, yang dapat ditemukan di dasbor Dodo Payments Anda di bawah pengaturan Webhooks.
3. **Hitung Ulang Tanda Tangan**: Gunakan algoritma penandatanganan yang sama (HMAC-SHA256) seperti Dodo Payments untuk menghitung tanda tangan di server Anda. Gunakan payload webhook dan kunci rahasia webhook Anda.
4. **Bandingkan Tanda Tangan**: Bandingkan tanda tangan yang dihitung ulang dengan yang diberikan dalam header **`webhook-signature`**. Jika cocok, webhook tersebut otentik; jika tidak, harus ditolak.
5. **Gunakan HTTPS**: Pastikan endpoint webhook Anda menggunakan HTTPS untuk mengamankan komunikasi dan mencegah manipulasi selama transmisi data.
Dengan menerapkan langkah-langkah ini, Anda dapat dengan percaya diri memverifikasi keaslian webhook yang dikirim oleh Dodo Payments.

<Tip>Kami merekomendasikan menggunakan paket yang disediakan oleh StandardWebhooks untuk memverifikasi tanda tangan dan tidak menyusunnya sendiri.</Tip>
</Accordion>

<Accordion title="Q109: Mengapa saya menerima webhook subscription.renew bahkan untuk pembayaran bulan pertama?">
A: Untuk langganan, `subscription.renew` akan dipicu setiap kali jumlah langganan dipotong. Jadi ya, ini adalah perilaku yang diharapkan.
</Accordion>

<Accordion title="Q110: Bagaimana cara menguji webhook secara lokal?">
A: Untuk menguji webhook secara lokal, Anda dapat mengikuti langkah-langkah ini:

1. **Gunakan Alat Tunneling**: Alat seperti [ngrok](https://ngrok.com/), [LocalTunnel](https://localtunnel.github.io/www/), atau [Cloudflare Tunnel](https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/) dapat mengekspos server lokal Anda ke internet. Ini menyediakan URL publik yang dapat digunakan Dodo Payments untuk mengirimkan peristiwa webhook ke lingkungan lokal Anda.
2. **Siapkan Endpoint Webhook**: Buat endpoint webhook di aplikasi lokal Anda untuk menangani peristiwa webhook yang masuk. Pastikan itu mendengarkan permintaan HTTP POST.
3. **Konfigurasi Webhook di Dodo Payments**: Pergi ke dasbor Dodo Payments Anda, navigasikan ke pengaturan Webhooks, dan perbarui URL webhook ke URL publik yang dihasilkan oleh alat tunneling.

</Accordion>
