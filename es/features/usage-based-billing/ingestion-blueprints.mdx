Los planos de ingesta son patrones preconstruidos para rastrear el uso en escenarios comunes. En lugar de averiguar las estructuras de eventos y las configuraciones de medidores desde cero, comienza con un plano probado y personalízalo según tus necesidades.

## ¿Qué son los Planos?

Cada plano incluye:
- **Recomendaciones de estructura de eventos**: Nombres de eventos óptimos y diseño de metadatos
- **Configuración del medidor**: Tipos de agregación y filtros sugeridos
- **Código de ejemplo**: Ejemplos de implementación funcional en múltiples lenguajes
- **Mejores prácticas**: Consejos para precisión, rendimiento y optimización de costos

<Tip>
Los planos ahorran horas de tiempo de configuración y te ayudan a evitar errores comunes. Comienza con un plano y luego adáptalo a tus requisitos específicos.
</Tip>

## Cómo Usar un Plano

<Steps>
<Step title="Elige tu plano">
Selecciona el plano que mejor se adapte a tu escenario de facturación. Cada página de plano incluye guías de implementación detalladas.
</Step>

<Step title="Revisa la estructura del evento">
Entiende el diseño recomendado para el nombre del evento y los metadatos. Copia la estructura del evento para tu caso de uso.
</Step>

<Step title="Crea el medidor">
Sigue las instrucciones de configuración del medidor del plano. Usa el tipo de agregación y los filtros sugeridos.
</Step>

<Step title="Implementa el envío de eventos">
Copia el código de ejemplo para tu lenguaje/framework. Reemplaza los valores de marcador de posición con tus claves de API y IDs de cliente reales.
</Step>

<Step title="Prueba y verifica">
Envía eventos de prueba y verifica que aparezcan en tu panel de medidores con la agregación correcta.
</Step>

<Step title="Personaliza según sea necesario">
Adapta el plano a tus requisitos específicos: agrega metadatos personalizados, ajusta filtros o modifica precios.
</Step>
</Steps>

## Comparación de Planos

¿No estás seguro de qué plano se ajusta a tus necesidades? Aquí tienes una comparación rápida:

| Caso de Uso | Plano | Agregación | Metadatos Comunes |
|----------|-----------|-------------|-----------------|
| App de chat de IA | LLM | Suma (tokens) | modelo, input_tokens, output_tokens |
| API REST | API Gateway | Conteo | endpoint, method, status_code |
| Almacenamiento de archivos | Object Storage | Suma (bytes) | file_size, file_type, operation |
| Plataforma de video | Stream | Suma (ancho de banda) | duration, quality, bitrate |
| Sin servidor | Time Range | Suma (duración) | function_name, execution_time_ms |

## Implementaciones Personalizadas

¿No ves un plano para tu caso de uso exacto? No hay problema:

1. **Comienza con el plano más cercano**: Elige el que sea más similar a tus necesidades
2. **Adapta la estructura del evento**: Modifica los campos de metadatos para que coincidan con tus requisitos
3. **Ajusta la configuración del medidor**: Cambia el tipo de agregación o los filtros según sea necesario
4. **Prueba a fondo**: Verifica los cálculos de facturación con el uso de muestra

<Info>
Todos los planos son puntos de partida. Siéntete libre de mezclar conceptos de diferentes planos o crear implementaciones completamente personalizadas.
</Info>
