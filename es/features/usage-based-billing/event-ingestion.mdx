Los eventos son la base de la facturación basada en el uso. Envía eventos cuando ocurren acciones facturables, y los medidores los agregan en cargos.

<Card title="Referencia de API - Ingesta de Eventos" icon="code" href="/api-reference/usage-events/ingest-events">
Documentación completa de la API con ejemplos y códigos de respuesta.
</Card>

## Estructura del Evento

<Accordion title="Campos Requeridos">
<ParamField body="event_id" type="string" required>
Identificador único. Usa UUIDs o combina el ID del cliente + timestamp + acción.
</ParamField>

<ParamField body="customer_id" type="string" required>
ID del cliente de Dodo Payments. Debe ser un cliente existente y válido.
</ParamField>

<ParamField body="event_name" type="string" required>
Tipo de evento que coincide con el nombre del evento de su medidor (sensible a mayúsculas y minúsculas). Ejemplos: `api.call`, `image.generated`
</ParamField>
</Accordion>

<Accordion title="Campos Opcionales">

<ParamField body="timestamp" type="string">
Timestamp en formato ISO 8601. Por defecto, se utiliza la hora del servidor si se omite. Inclúyelo para una facturación precisa con eventos retrasados/lote.
</ParamField>

<ParamField body="metadata" type="object">
Propiedades adicionales para agregación y filtrado:
- Valores numéricos: `bytes`, `tokens`, `duration_ms`
- Filtros: `endpoint`, `method`, `quality`

```javascript
metadata: {
  endpoint: "/v1/orders",
  method: "POST",
  tokens: 1024
}
```

</ParamField>
</Accordion>

## Envío de Eventos

<CodeGroup>

```javascript Single Event
await fetch('https://test.dodopayments.com/events/ingest', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.DODO_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    events: [{
      event_id: "api_call_1234",
      customer_id: "cus_abc123",
      event_name: "api.call",
      metadata: { endpoint: "/v1/orders" }
    }]
  })
});
```

```javascript Batch Events
await fetch('https://test.dodopayments.com/events/ingest', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.DODO_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    events: [
      { event_id: "call_1", customer_id: "cus_abc", event_name: "api.call" },
      { event_id: "call_2", customer_id: "cus_abc", event_name: "api.call" },
      { event_id: "call_3", customer_id: "cus_abc", event_name: "api.call" }
    ]
  })
});
```

```python Python
import requests

requests.post(
    'https://test.dodopayments.com/events/ingest',
    headers={'Authorization': f'Bearer {api_key}', 'Content-Type': 'application/json'},
    json={'events': [{'event_id': 'call_1', 'customer_id': 'cus_abc', 'event_name': 'api.call'}]}
)
```

```curl cURL
curl -X POST 'https://test.dodopayments.com/events/ingest' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{"events": [{"event_id": "call_1", "customer_id": "cus_abc", "event_name": "api.call"}]}'
```

</CodeGroup>

<Tip>
Agrupa hasta 100 eventos por solicitud para un mejor rendimiento.
</Tip>

## Mejores Prácticas

<AccordionGroup>
<Accordion title="Usar IDs de evento únicos">
Utilice IDs deterministas para evitar duplicados: `${customerId}_${action}_${timestamp}`
</Accordion>

<Accordion title="Implementa Reintentos">
Reintenta en errores 5xx con retroceso exponencial. No reintentes errores 4xx.
</Accordion>

<Accordion title="Incluye Timestamps">
Omitir para eventos en tiempo real. Incluir para eventos retrasados/lote para precisión.
</Accordion>

<Accordion title="Monitorea la Entrega">
Rastrea tasas de éxito y coloca eventos fallidos en cola para reintento.
</Accordion>
</AccordionGroup>

## Solución de Problemas

<AccordionGroup>
<Accordion title="Eventos no aparecen">
- El nombre del evento debe coincidir exactamente con el medidor (sensible a mayúsculas)
- El ID del cliente debe existir
- Verifica que los filtros del medidor no estén excluyendo eventos
- Verifica que los timestamps sean recientes
</Accordion>

<Accordion title="Errores de autenticación (401)">
Verifique que la clave de API sea correcta y use el formato: `Bearer YOUR_API_KEY`
</Accordion>

<Accordion title="Errores de validación (400)">
Asegúrese de que todos los campos requeridos estén presentes: `event_id`, `customer_id`, `event_name`
</Accordion>

<Accordion title="Metadatos no agregando">
- Las claves de metadatos deben coincidir exactamente con "Over Property" del medidor
- Use números, no cadenas: `tokens: 150` no `tokens: "150"`
</Accordion>
</AccordionGroup>
