## Cómo funciona

Los métodos de pago se presentan automáticamente a los clientes según:

- **Ubicación del cliente** — Métodos regionales como iDEAL (Países Bajos) o UPI (India) solo aparecen para clientes en esas regiones.
- **Moneda de transacción** — Los métodos de pago se filtran por las monedas soportadas.
- **Tipo de dispositivo** — Apple Pay solo aparece en dispositivos y navegadores Apple compatibles.
- **Tipo de transacción** — Algunos métodos no soportan suscripciones o tienen montos mínimos.

<Info>
**Activación automática:** Todos los métodos de pago están preconfigurados y gestionados por Dodo Payments. No se necesita ninguna configuración adicional: están disponibles una vez que se verifica la cuenta y se habilita el modo real.
</Info>

## Configuración de Métodos de Pago

De forma predeterminada, todos los métodos de pago elegibles aparecen automáticamente. Usa `allowed_payment_method_types` para limitar qué métodos se muestran:

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://test.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

### Tipos de Métodos Disponibles

| Type | Description |
| :--- | :---------- |
| `credit` | Tarjetas de crédito (Visa, Mastercard, Amex, etc.) |
| `debit` | Tarjetas de débito |
| `apple_pay` | Apple Pay |
| `google_pay` | Google Pay |
| `amazon_pay` | Amazon Pay |
| `cashapp` | Cash App Pay |
| `klarna` | Klarna BNPL |
| `afterpay_clearpay` | Afterpay / Clearpay |
| `upi_collect` | UPI Collect (código QR) |
| `ideal` | iDEAL (Países Bajos) |
| `bancontact_card` | Bancontact (Bélgica) |
| `eps` | EPS (Austria) |
| `multibanco` | Multibanco (Portugal) |
| `revolut_pay` | RevolutPay |
| `billie` | Billie B2B BNPL |

<Warning>
Incluye siempre `credit` e `debit` como opciones de respaldo. Si todos los métodos de pago especificados no están disponibles para un cliente, el checkout fallará.
</Warning>

## Mejores Prácticas

<AccordionGroup>
{/* LOCKED_PATTERN_63da960e31e014d866efe70101f28b99 */}
Incluye métodos de pago regionales para tus mercados objetivo. iDEAL representa aproximadamente el 60 % de los pagos en línea en los Países Bajos; UPI domina en India. Omitir estos métodos perjudica significativamente la conversión.
</Accordion>

{/* LOCKED_PATTERN_78f59c035688fa14218708818c9f0fca */}
Incluso al orientar métodos específicos, incluye `credit` e `debit` como opciones de respaldo. No todos los clientes tienen acceso a todos los métodos de pago.
</Accordion>

{/* LOCKED_PATTERN_564640b27644e47b9e4e7faaec6c2d88 */}
Los métodos BNPL (Klarna, Afterpay) requieren un mínimo de $50.01. No los incluyas para productos de bajo valor: simplemente no aparecerán, y eso está bien.
</Accordion>

</Accordion>
</AccordionGroup>

## Resolución de Problemas

<AccordionGroup>
{/* LOCKED_PATTERN_fbf2d38e7725fb97384bef01324b59a1 */}
**Verifica estos factores:**
1. ¿El método está incluido en `allowed_payment_method_types`?
2. ¿La ubicación del cliente coincide con la región compatible del método?
3. ¿La moneda de la transacción está soportada por el método?
4. ¿El importe de la transacción cumple con el mínimo (p. ej., $50.01 para BNPL)?
5. Para suscripciones: ¿el método admite pagos recurrentes?

**Solución:** Elimina temporalmente `allowed_payment_method_types` para ver todos los métodos elegibles y luego delimita el problema.
</Accordion>

{/* LOCKED_PATTERN_57385d59bd851fb1e8c26266b489efd4 */}
**Causas posibles:**
- `allowed_payment_method_types` no se está pasando correctamente
- Parámetro formateado incorrectamente (debe ser un array de cadenas)

**Solución:** Registra la carga útil de tu solicitud API y verifica la estructura del parámetro.
</Accordion>

{/* LOCKED_PATTERN_6d1a1b1a59cbe2789bbc9434eb1ec153 */}
**Verifica:**
- La dirección de facturación del cliente coincide con la región del método
- La moneda está configurada correctamente (p. ej., EUR para iDEAL)
- La Moneda Adaptativa está habilitada si el cliente es internacional

**Solución:** Verifica el código de país de la dirección de facturación y la moneda en tu solicitud API.
</Accordion>

{/* LOCKED_PATTERN_52f3532b80d2def876ef3cc78b55f9f6 */}
Las suscripciones UPI y Rupay tienen requisitos únicos, incluidos retrasos de procesamiento de 48 horas y límites de mandato del RBI.

**Consulta:** <a href="/features/payment-methods/india">India Payment Methods</a> para obtener orientación completa.
</Accordion>
</AccordionGroup>
