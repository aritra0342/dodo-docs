## Cómo funciona

Los métodos de pago se presentan automáticamente a los clientes según:

- **Ubicación del cliente** — Métodos regionales como iDEAL (Países Bajos) o UPI (India) solo aparecen para clientes en esas regiones.
- **Moneda de transacción** — Los métodos de pago se filtran por las monedas soportadas.
- **Tipo de dispositivo** — Apple Pay solo aparece en dispositivos y navegadores Apple compatibles.
- **Tipo de transacción** — Algunos métodos no soportan suscripciones o tienen montos mínimos.

<Info>
**Activación automática:** Todos los métodos de pago están preconfigurados y gestionados por Dodo Payments. No se necesita configuración adicional: se vuelven disponibles una vez que tu cuenta está verificada y el modo en vivo está habilitado.
</Info>

## Configuración de Métodos de Pago

Por defecto, todos los métodos de pago elegibles aparecen automáticamente. Utiliza `allowed_payment_method_types` para restringir qué métodos se muestran:

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://test.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

### Tipos de Métodos Disponibles

| Tipo | Descripción |
| :--- | :---------- |
| `credit` | Tarjetas de crédito (Visa, Mastercard, Amex, etc.) |
| `debit` | Tarjetas de débito |
| `apple_pay` | Apple Pay |
| `google_pay` | Google Pay |
| `amazon_pay` | Amazon Pay |
| `cashapp` | Cash App Pay |
| `klarna` | Klarna BNPL |
| `afterpay_clearpay` | Afterpay / Clearpay |
| `upi_collect` | UPI Collect (código QR) |
| `ideal` | iDEAL (Países Bajos) |
| `bancontact_card` | Bancontact (Bélgica) |
| `eps` | EPS (Austria) |
| `multibanco` | Multibanco (Portugal) |
| `revolut_pay` | RevolutPay |
| `billie` | Billie B2B BNPL |

<Warning>
Siempre incluye `credit` y `debit` como alternativas. Si todos los métodos de pago especificados no están disponibles para un cliente, el proceso de compra fallará.
</Warning>

## Mejores Prácticas

<AccordionGroup>
<Accordion title="Ajusta métodos a tu mercado">
Incluye métodos de pago regionales para tus mercados objetivo. iDEAL captura aproximadamente el 60% de los pagos en línea en los Países Bajos; UPI domina en India. Omitir estos métodos perjudica significativamente la conversión.
</Accordion>

<Accordion title="Siempre proporciona alternativas de tarjeta">
Incluso cuando te enfocas en métodos específicos, incluye `credit` y `debit` como alternativas. No todos los clientes tienen acceso a cada método de pago.
</Accordion>

<Accordion title="Considera los mínimos de transacción">
Los métodos BNPL (Klarna, Afterpay) requieren un mínimo de $50.01. No los incluyas para productos de bajo valor; simplemente no aparecerán, lo cual está bien.
</Accordion>

<Accordion title="Prueba en diferentes regiones y dispositivos">
Utiliza el modo de prueba para verificar la disponibilidad de métodos de pago para diferentes ubicaciones de cliente, monedas y tipos de dispositivos antes de activarlo en vivo.
</Accordion>
</AccordionGroup>

## Resolución de Problemas

<AccordionGroup>
<Accordion title="Método de pago no apareciendo">
**Verifica estos factores:**
1. ¿El método está incluido en `allowed_payment_method_types`?
2. ¿La ubicación del cliente coincide con la región soportada por el método?
3. ¿La moneda de la transacción es soportada por el método?
4. ¿El monto de la transacción cumple con el mínimo (p. ej., $50.01 para BNPL)?
5. Para suscripciones: ¿el método soporta pagos recurrentes?

**Solución:** Elimina temporalmente `allowed_payment_method_types` para ver todos los métodos elegibles y luego identifica el problema.
</Accordion>

<Accordion title="Métodos inesperados apareciendo">
**Posibles causas:**
- `allowed_payment_method_types` no se está pasando correctamente.
- Parámetro formateado incorrectamente (debe ser un array de cadenas).

**Solución:** Registra la carga útil de tu petición API y verifica la estructura del parámetro.
</Accordion>

<Accordion title="Método regional no disponible">
**Verifica:**
- La dirección de facturación del cliente coincide con la región del método.
- La moneda está configurada correctamente (p. ej., EUR para iDEAL).
- La Moneda Adaptativa está habilitada si el cliente es internacional.

**Solución:** Verifica el código de país de la dirección de facturación y la moneda en tu petición API.
</Accordion>

<Accordion title="Problemas con métodos de pago indios">
Las suscripciones de UPI y Rupay tienen requisitos únicos, incluyendo retrasos de procesamiento de 48 horas y límites de mandato del RBI.

**Consulta:** <a href="/features/payment-methods/india">Métodos de Pago en India</a> para obtener orientación completa.
</Accordion>
</AccordionGroup>
