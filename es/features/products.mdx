## Crear un producto

Puedes crear productos desde el panel de control o a través de la API. Elige el modelo de precios de antemano: Una sola vez, Suscripción o Basado en uso y luego configura los detalles. El modelo de precios no se puede cambiar más tarde; crea un nuevo producto si necesitas un modelo diferente.

<Steps>
{/* LOCKED_PATTERN_650716067d22ce9b2fc10037a079cf41 */}
Proporciona un título claro y una descripción concisa orientada al valor. En las descripciones se admite Markdown.

<Tip>
Mantén la primera frase orientada al cliente y a los resultados; aparece de forma prominente en el checkout.
</Tip>

<Frame>
  <img src="/images/products.png" alt="Productos"/>
</Frame>

</Step>

{/* LOCKED_PATTERN_cbd9515b00efe13b7a75ec5b61fe0c6b */}
Selecciona el modelo de precios:

- **Una sola vez**: Precio fijo pagado una vez.
- **Suscripción**: Precio recurrente con intervalo y prueba opcional.
- **Basado en uso**: Precio derivado de eventos medidos.

Luego establece el precio:

- **Precio**: Monto base y moneda.
- **Descuento (%)**: Descuento opcional en línea mostrado en el pago y facturas.
- Para suscripciones, establece **Repetir cada** (por ejemplo, 1 mes o 1 año) y **Días de prueba** si es necesario.

<Warning>
Cambiar el precio afecta solo a las nuevas compras. Las suscripciones existentes siguen las reglas de cambio de plan.
</Warning>
</Step>

{/* LOCKED_PATTERN_7ec2f27e00061fa989770e1c689230ca */}
Sube imágenes para mostrar el producto en el checkout y en las facturas. Se admiten PNG/JPG/WebP de hasta 3 MB. Puedes reordenarlas o reemplazarlas en cualquier momento.
</Step>

{/* LOCKED_PATTERN_fb732d1baf26a73e83e826ecbb616b8f */}
Asocia un cumplimiento que se active automáticamente después del pago:
</CardGroup>

- **Claves de licencia**: Emite y valida claves únicas
- **Descargas de archivos**: Concede acceso seguro a archivos
- **Personalizado**: Activa tu propia lógica de derechos a través de webhooks

Agrega o elimina beneficios según evolucione tu oferta. Los suscriptores existentes ganan o pierden acceso en consecuencia.
</Step>
</Steps>

## Variantes y opciones de precios

En lugar de variantes bajo un solo producto, crea productos separados para cada opción de precio (por ejemplo, Mensual y Anual). Luego, agrúpalos en una **Colección de Productos** para presentar todas las opciones en un solo pago y permitir el cambio de plan en el Portal del Cliente.

<Frame>
  <img src="/images/product-collection/checkout-page.png" alt="Colecciones de productos"/>
</Frame>

### ¿Por qué este enfoque?

- **Modelos de precios claros**: Cada producto tiene un modelo de precios único y bien definido (pago único, suscripción o basado en uso)
- **APIs predecibles**: Integraciones más simples sin lógica de variantes anidadas
- **Informes más fáciles**: Realiza un seguimiento de los ingresos y métricas por producto sin la agregación de variantes
- **Pago flexible**: Muestra múltiples productos uno al lado del otro, permitiendo a los clientes comparar y elegir

### Cómo funcionan las Colecciones de Productos

1. **Crea productos**: Configura productos individuales para cada plan (por ejemplo, Starter Mensual, Starter Anual, Pro Mensual, Pro Anual)
2. **Agrupa en una colección**: Agrega productos relacionados a una Colección de Productos
3. **Pago unificado**: Los clientes ven todas las opciones en un solo pago y seleccionan su plan preferido
4. **Cambio de plan**: Los clientes pueden actualizar o degradar entre productos en la misma colección a través del Portal del Cliente

## Gestión de productos

Puedes gestionar productos a través del panel o programáticamente mediante la API. La API proporciona control total sobre la creación, actualizaciones, recuperación, carga de imágenes y archivo de productos.

### Gestión desde el Panel

- **Actualizar**: Edita el nombre, descripción, imágenes, precio, campos y beneficios en cualquier momento (el modelo de precios es inmutable).
- **Archivar**: Oculta un producto de nuevas compras sin interrumpir a los clientes existentes. Puedes desarchivarlo más tarde.

### Gestión de API

Las siguientes instrucciones te permiten crear, actualizar, gestionar y recuperar productos, incluyendo la carga de imágenes.

<AccordionGroup>
{/* LOCKED_PATTERN_eddf4dc624fa14b64d50b00a7b62a243 */}
Un producto puede ser un artículo de pago único o un servicio basado en suscripción. Para crear un nuevo producto, envía una solicitud `POST` al endpoint `/products` con detalles como nombre, descripción, precio, moneda y si es un producto recurrente.

Para productos recurrentes, configura `recurring: true` y especifica un `billing_cycle` (`daily`, `weekly`, `monthly`, `yearly`).

{/* LOCKED_PATTERN_928df4172f546004633c66ee1394fc66 */}
  Consulta la estructura detallada de solicitud y respuesta en la documentación de la API Crear producto.
</Card>
</Accordion>

{/* LOCKED_PATTERN_5b0241a7e30101c362a1fa23c61ba5c9 */}
Para modificar un producto existente, envía una solicitud `PATCH` al endpoint `/products/{product_id}`. Puedes actualizar propiedades como el nombre, el precio y la descripción mientras mantienes otros detalles sin cambios.

Asegúrate de que el `product_id` en el endpoint coincida con un producto existente.

{/* LOCKED_PATTERN_547a8a1f64e1600671c88ac42f733bf5 */}
  Consulta la estructura detallada de solicitud y respuesta en la documentación de la API Actualizar producto.
</Card>
</Accordion>

{/* LOCKED_PATTERN_bb5bf0f0b68e143d62ace39a4c2918f1 */}
Puedes obtener una lista de productos almacenados en tu cuenta usando una solicitud `GET` al endpoint `/products`. Esto te permite recuperar detalles de los productos, incluidos los activos y los archivados.

{/* LOCKED_PATTERN_49a116775840cfaf0a655cdd67c21416 */}
  Consulta la estructura detallada de solicitud y respuesta en la documentación de la API Recuperar productos.
</Card>
</Accordion>

{/* LOCKED_PATTERN_bb5d7818f6659dbe681096a03760fcba */}
Puedes asociar una imagen con un producto subiéndola a AWS S3 mediante una URL prefirmada proporcionada por la API. Primero, solicita una URL de carga de imagen al endpoint `/products/{product_id}/images` y luego usa la URL proporcionada para subir la imagen dentro de 60 segundos.

<Warning>
La URL prefirmada expira en 60 segundos, por lo que la imagen debe cargarse dentro de ese plazo.
</Warning>

Una vez recibida la URL prefirmada de la API, sube la imagen usando el método `PUT`. Esto garantiza acceso seguro y temporal a AWS S3 para subir la imagen.

**Bibliotecas compatibles para subir a S3:**
- **Node.js**: `axios`, `node-fetch`
- **Python**: `requests`, `boto3`
- **Go**: `net/http`
- **PHP**: `GuzzleHttp`
- **Ruby**: `rest-client`

Si la carga es exitosa, AWS S3 devolverá un estado `200 OK`, lo que indica que la imagen se ha almacenado correctamente.

{/* LOCKED_PATTERN_0e8c82409b28128d958fc51adde377a8 */}
  Consulta la estructura detallada de solicitud y respuesta en la documentación de la API Subir imagen de producto.
</Card>
</Accordion>

{/* LOCKED_PATTERN_e39596eca26d22865eed77998ed1d61c */}
Si ya no deseas mostrar o usar un producto, puedes archivarlo usando una solicitud `POST` al endpoint `/products/{product_id}/archive`. Esta acción oculta el producto pero no lo elimina permanentemente.

{/* LOCKED_PATTERN_a26cda50522e246415d276751f0deefc */}

</Card>

{/* LOCKED_PATTERN_6ae04208a8e6089a2c7fee7a325fb767 */}
Si necesitas restaurar un producto archivado, envía una solicitud `POST` al endpoint `/products/{product_id}/unarchive`. Esto reactivará el producto y lo dejará disponible nuevamente.

## Mejores prácticas

- **Comienza con claridad**: Productos separados para cada opción de precio (Mensual vs Anual)
- **Utiliza pruebas de manera reflexiva**: Combina pruebas con incorporación para impulsar la activación
- **Automatiza el cumplimiento**: Utiliza beneficios y webhooks para entregar instantáneamente
- **Etiqueta con metadatos**: Almacena tus IDs del sistema para reconciliación
