## ¿Qué son los Bolsillos de Clientes?

Los monederos de cliente son cuentas de saldo monetario que contienen fondos reales para tus usuarios. Cada cliente obtiene uno automáticamente cuando creas su cuenta. Puedes usar estos monederos para:

- **Almacenar fondos prepago** para futuros pagos de suscripción
- **Gestionar reembolsos** como saldo del monedero en lugar de reembolsos a tarjeta
- **Emitir saldos promocionales** como bonificaciones de bienvenida o recompensas de fidelidad
- **Aplicar fondos del monedero a las facturas** automáticamente durante la facturación
- **Hacer seguimiento de transacciones monetarias** con un historial de libro mayor detallado

<Info>
Cada cliente recibe automáticamente un monedero cuando creas su cuenta. Los monederos admiten monedas USD e INR con saldos monetarios separados para cada una.
</Info>

<Warning>
**Monederos de Cliente ≠ Facturación basada en créditos**

Los Monederos de Cliente contienen **saldos monetarios reales** (USD, INR) que pueden aplicarse a facturas y pagos de suscripción.

Si buscas hacer seguimiento de unidades de uso virtual (llamadas a la API, tokens, horas de cómputo), consulta [Credit-Based Billing](/features/credit-based-billing) en su lugar.
</Warning>

## Cómo funciona

Los monederos de cliente almacenan fondos que los clientes pueden aplicar a sus compras. Cuando un cliente tiene una factura o un cargo recurrente de suscripción, primero se verifica el saldo de su monedero. Los fondos disponibles se aplican automáticamente a la factura antes de cargar su método de pago principal.

### Configuración automática

Cuando creas un cliente nuevo, Dodo Payments crea automáticamente un monedero con saldo cero. Está listo para recibir fondos de inmediato a través de nuestra API.

### Soporte multimoneda

Cada monedero puede contener saldos en distintas monedas:

<ResponseField name="USD Balance" type="integer">
Saldo en dólares estadounidenses (almacenado en centavos)
</ResponseField>

<ResponseField name="INR Balance" type="integer">
Saldo en rupias indias (almacenado en paise)
</ResponseField>

<Info>
Actualmente, sólo los saldos en **USD** e **INR** están disponibles. Próximamente habrá más monedas.
</Info>

## Ejemplos del mundo real

### Reembolso al monedero

Cuando un cliente solicita un reembolso, puedes añadir el monto al saldo de su monedero en lugar de realizar un reembolso tradicional a tarjeta. Esto mantiene los fondos dentro de tu ecosistema para compras futuras.

```javascript
async function refundToWallet(customerId, refundAmount, originalPaymentId) {
  await client.customers.wallets.ledgerEntries.create(customerId, {
    amount: refundAmount, // Amount in cents
    currency: 'USD',
    entry_type: 'credit',
    reason: `Refund for payment ${originalPaymentId}`,
    idempotency_key: `refund_${originalPaymentId}`
  });
}
```

### Bono de bienvenida / Saldo promocional

Ofrece a los clientes nuevos un bono monetario de bienvenida para incentivar su primera compra.

```javascript
async function addWelcomeBonus(customerId) {
  await client.customers.wallets.ledgerEntries.create(customerId, {
    amount: 1000, // $10.00 promotional balance
    currency: 'USD',
    entry_type: 'credit',
    reason: 'Welcome bonus - $10 promotional balance',
    idempotency_key: `welcome_${customerId}`
  });
}
```

### Pago de suscripción desde el monedero

Deduce fondos de un monedero para cubrir un cargo de suscripción o una compra manual.

```javascript
async function deductForPurchase(customerId, purchaseAmount, purchaseId) {
  try {
    await client.customers.wallets.ledgerEntries.create(customerId, {
      amount: purchaseAmount,
      currency: 'USD',
      entry_type: 'debit',
      reason: `Subscription charge - monthly billing`,
      idempotency_key: `charge_${purchaseId}`
    });
  } catch (error) {
    if (error.status === 400) {
      console.log('Insufficient wallet balance');
    }
  }
}
```

## Mejores prácticas

### Evitar transacciones duplicadas

Utiliza claves de idempotencia para asegurarte de no añadir o deducir fondos dos veces por el mismo evento.

```javascript
async function addFundsSafely(customerId, amount, reason) {
  const idempotencyKey = `${reason}_${customerId}_${Date.now()}`;
  
  try {
    const result = await client.customers.wallets.ledgerEntries.create(customerId, {
      amount: amount,
      currency: 'USD',
      entry_type: 'credit',
      reason: reason,
      idempotency_key: idempotencyKey
    });
    
    return { success: true, wallet: result };
  } catch (error) {
    if (error.status === 409) {
      // Transaction already processed
      return { success: true, wallet: null, duplicate: true };
    }
    
    throw error;
  }
}
```

### Verificar saldos antes de cobrar

Confirma que los clientes tienen fondos suficientes antes de intentar procesar transacciones grandes desde el monedero.

```javascript
async function checkBalanceBeforeOperation(customerId, requiredAmount) {
  const wallets = await client.customers.wallets.list(customerId);
  const usdWallet = wallets.items.find(w => w.currency === 'USD');
  
  if (!usdWallet || usdWallet.balance < requiredAmount) {
    throw new Error('Insufficient funds for this operation');
  }
  
  return usdWallet.balance;
}
```
