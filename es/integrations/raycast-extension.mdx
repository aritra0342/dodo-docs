Tu panel de Dodo Payments, a unos pocos pulsos de teclado. Usa Raycast para ver rápidamente pagos, suscripciones, clientes, productos, códigos de descuento, claves de licencia y reembolsos sin abrir el panel.

## Instalar

<Card title="Instalar desde la tienda de Raycast →" icon="puzzle-piece" href="https://www.raycast.com/dodopayments/dodo-payments">
  Abre la página de la extensión de Dodo Payments en la tienda de Raycast.
</Card>

<Info>
Requiere Raycast en macOS y una clave API de Dodo Payments.
</Info>

## Configuración

<Steps>
<Step title="Obtén tu clave API">
1. Inicia sesión en tu panel de Dodo Payments.
2. Ve a Configuración › Claves API.
3. Copia tu clave API.
</Step>

<Step title="Configura la extensión">
1. Abre Raycast.
2. Busca "Dodo Payments" y selecciona cualquier comando.
3. Cuando se te pida, ingresa tu clave API en el campo de clave API.
4. Elige el modo API:
   - Prueba: usa para desarrollo y pruebas.
   - Producción: usa para datos en producción.
5. Guarda la configuración.
</Step>
</Steps>

<Tip>
Comienza en modo de prueba para validar tu configuración, luego cambia a producción.
</Tip>

<Warning>
Trata tu clave API como una contraseña. No la compartas ni la almacenes en texto plano.
</Warning>

## Usa la extensión

- **Buscar**: abre Raycast, escribe "Dodo Payments" y elige un comando.
- **Filtrar**: reduce los resultados escribiendo partes de IDs, correos electrónicos, nombres o estados.
- **Abrir detalles**: presiona Enter en un elemento de la lista para ver más información.

## Solución de problemas

<AccordionGroup>
<Accordion title="Error de autenticación o errores 401/403">
Confirma que la clave API y el modo (Prueba vs Producción) coincidan con los datos de tu panel. Reabre el comando y vuelve a ingresar la clave si es necesario.
</Accordion>
<Accordion title="No aparecen resultados">
Asegúrate de que el modo seleccionado tenga datos. Intenta cambiar entre Prueba y Producción.
</Accordion>
<Accordion title="Límites de tasa o tiempos de espera">
Espera un momento y vuelve a intentarlo. Si el problema persiste, verifica tu conectividad de red.
</Accordion>
</AccordionGroup>
