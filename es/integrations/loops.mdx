## Introducción

Envía correos electrónicos transaccionales y gestiona automáticamente la comunicación con los clientes cuando ocurren eventos de pago. Entrega confirmaciones de pago, actualizaciones de suscripción y notificaciones importantes con la infraestructura de correo electrónico de Bucles.

<Info>
Esta integración requiere su clave de API de Loops para la autenticación.
</Info>

## Comenzando

<Steps>
<Step title="Open the Webhook Section">
  En el panel de Dodo Payments, vaya a <b>Webhooks → + Agregar punto de conexión</b> y expanda el desplegable de integraciones.
  <Frame>
    <img src="/images/integrations/loops.png" alt="Agregar punto de conexión y desplegable de integraciones" />
  </Frame>
</Step>

<Step title="Select Loops">
  Elija la tarjeta de integración de <b>Loops</b>.
</Step>

<Step title="Enter API Key">
  Proporcione su clave de API de Loops en la configuración.
</Step>

<Step title="Configure Transformation">
  Edite el código de transformación para formatear los correos electrónicos para la API de Loops.
</Step>

<Step title="Test & Create">
  Pruebe con cargas útiles de ejemplo y haga clic en <b>Crear</b> para activar el envío de correos.
</Step>

<Step title="Done!">
  🎉 Los eventos de pago ahora activarán automáticamente correos transaccionales vía Loops.
</Step>
</Steps>

## Ejemplos de Código de Transformación

### Correo Electrónico de Confirmación de Pago

```javascript payment_confirmation.js icon="js" expandable
function handler(webhook) {
  if (webhook.eventType === "payment.succeeded") {
    const p = webhook.payload.data;
    webhook.url = "https://api.loops.so/v1/events/send";
    webhook.payload = {
      eventName: "payment_confirmation",
      email: p.customer.email,
      properties: {
        customer_name: p.customer.name,
        payment_id: p.payment_id,
        amount: (p.total_amount / 100).toFixed(2),
        currency: p.currency || "USD",
        payment_method: p.payment_method || "unknown",
        payment_date: new Date(webhook.payload.timestamp).toLocaleDateString()
      }
    };
  }
  return webhook;
}
```

### Correo Electrónico de Bienvenida a la Suscripción

```javascript subscription_welcome.js icon="js" expandable
function handler(webhook) {
  if (webhook.eventType === "subscription.active") {
    const s = webhook.payload.data;
    webhook.url = "https://api.loops.so/v1/events/send";
    webhook.payload = {
      eventName: "subscription_welcome",
      email: s.customer.email,
      properties: {
        customer_name: s.customer.name,
        subscription_id: s.subscription_id,
        product_id: s.product_id,
        amount: (s.recurring_pre_tax_amount / 100).toFixed(2),
        frequency: s.payment_frequency_interval,
        next_billing: s.next_billing_date
      }
    };
  }
  return webhook;
}
```

### Notificación de Fallo de Pago

```javascript payment_failure.js icon="js" expandable
function handler(webhook) {
  if (webhook.eventType === "payment.failed") {
    const p = webhook.payload.data;
    webhook.url = "https://api.loops.so/v1/events/send";
    webhook.payload = {
      eventName: "payment_failed",
      email: p.customer.email,
      properties: {
        customer_name: p.customer.name,
        payment_id: p.payment_id,
        amount: (p.total_amount / 100).toFixed(2),
        error_message: p.error_message || "Payment processing failed",
        retry_link: `https://yourdomain.com/retry-payment/${p.payment_id}`
      }
    };
  }
  return webhook;
}
```

## Consejos

- Usa nombres de eventos descriptivos para una mejor organización de las plantillas de correo electrónico
- Incluye propiedades relevantes del cliente para la personalización
- Configura plantillas de correo electrónico en el panel de Bucles para cada evento
- Usa nombres de propiedades consistentes en todos los eventos
- Prueba la entrega de correos electrónicos antes de ponerlo en producción

## Solución de Problemas

<AccordionGroup>
<Accordion title="Emails not being sent">
- Verifique que la clave de API sea correcta y esté activa
- Compruebe que los nombres de los eventos coincidan con sus plantillas de Loops
- Asegúrese de que las direcciones de correo de los destinatarios sean válidas
- Revise los límites y cuotas de envío de Loops
</Accordion>

<Accordion title="Transformation errors">
- Valide que la estructura JSON coincida con el formato de la API de Loops
- Compruebe que todos los campos obligatorios estén presentes
- Asegúrese de que los nombres de los eventos estén correctamente formateados
- Verifique los permisos de la clave de API
</Accordion>
</AccordionGroup>
