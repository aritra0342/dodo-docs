---
title: Envía tu proyecto
description: Cómo agregar tu proyecto de código abierto al directorio de la
  comunidad de Dodo Payments.
icon: plus
---
<Steps>
<Step title="Check eligibility">
- El proyecto es de código abierto con una licencia permisiva.
- Es relevante para Dodo Payments (SDK, complemento, integración, plantilla).
- Cuenta con un README que incluye instalación, uso y estado de mantenimiento.
</Step>

<Step title="Add entry via PR">
Edit `community/projects.mdx` y añade un nuevo `<Card>` bajo la pestaña más relevante.

```mdx
<Card title="PROJECT_NAME (LANG)" href="REPO_URL" icon="code">
  One-sentence description of your project.
  <br/>By <a href="GITHUB_PROFILE_URL">@github_username</a>
  <br/><br/>`LANGUAGE` `FRAMEWORK` `CATEGORY`
</Card>
```

</Step>

<Step title="Open submission issue (optional)">
Si lo prefieres, abre un issue usando la plantilla "Community Project Submission" con:
- URL del repositorio, descripción, categoría (SDK/Plugin/Boilerplate)
- Lenguaje/marco principal
- Enlace al fragmento de instalación/uso
- Estado de mantenimiento y última actualización
</Step>

<Step title="Review & publish">
Revisamos por relevancia, claridad y seguridad. Las entradas aprobadas se fusionan y publican.
</Step>
</Steps>

<Warning>
Podemos eliminar entradas que no se mantengan, sean inseguras, confusas o infrinjan la política.
</Warning>
