---
title: Envía tu proyecto
description: Cómo agregar tu proyecto de código abierto al directorio de la
  comunidad de Dodo Payments.
icon: plus
---

<Steps>
<Step title="Verificar elegibilidad">
- El proyecto es de código abierto con una licencia permisiva.
- Es relevante para Dodo Payments (SDK, plugin, integración, plantilla).
- Tiene un README con instalación, uso y estado de mantenimiento.
</Step>

<Step title="Agregar entrada a través de PR">
Edit `community/projects.mdx` y agrega un nuevo `<Card>` bajo la pestaña más relevante.

```mdx
<Card title="PROJECT_NAME (LANG)" href="REPO_URL" icon="code">
  One-sentence description of your project.
  <br/>By <a href="GITHUB_PROFILE_URL">@github_username</a>
  <br/><br/>`LANGUAGE` `FRAMEWORK` `CATEGORY`
</Card>
```

</Step>

<Step title="Abrir un issue de envío (opcional)">
Si lo prefieres, abre un issue usando la plantilla "Envío de Proyecto Comunitario" con:
- URL del repositorio, descripción, categoría (SDK/Plugin/Boilerplate)
- Lenguaje/framework principal
- Enlace al fragmento de instalación/uso
- Estado de mantenimiento y última actualización
</Step>

<Step title="Revisar y publicar">
Revisamos la relevancia, claridad y seguridad. Las entradas aprobadas se fusionan y publican.
</Step>
</Steps>

<Warning>
Podemos eliminar entradas que no estén mantenidas, sean inseguras, engañosas o violen la política.
</Warning>
