## ¿Qué es una migración de suscripciones?

Una migración de suscripciones transfiere tus suscriptores existentes desde tu procesador de pagos heredado a Dodo Payments, dándote una única fuente de verdad para todos tus datos de clientes. Sin una migración, los clientes que has adquirido hasta la fecha permanecen en tu sistema heredado, y solo los nuevos clientes serían visibles en Dodo Payments.

El proceso de migración asegura una experiencia de facturación sin interrupciones para tus clientes: no necesitarán volver a ingresar sus detalles de pago ni experimentar ninguna interrupción en sus suscripciones.

<Warning>
Actualmente, solo soportamos la migración de **tokens de tarjeta**. Apple Pay, Google Pay y otros métodos de pago de billetera no pueden ser transferidos entre procesadores de pagos debido a restricciones de tokenización impuestas por estos proveedores.
</Warning>

## Resumen del proceso de migración

El proceso de migración de suscripciones consta de tres partes clave:

1. **Transferencia de método de pago** — Transferencia segura de datos sensibles de tarjetas desde tu procesador actual
2. **Transferencia de datos de suscriptores** — Exportación de detalles de clientes, planes de suscripción y horarios de facturación
3. **Importación y validación** — Unificación de métodos de pago y datos de suscriptores en Dodo Payments

<Note>
Para comenzar una migración de suscripciones, contacta a nuestro [equipo de soporte](mailto:support@dodopayments.com). Ellos te conectarán con un Ingeniero de Soluciones dedicado que te guiará a través de todo el proceso.
</Note>

## Requisitos

Antes de iniciar una migración, asegúrate de cumplir con los siguientes requisitos:

<AccordionGroup>
<Accordion title="Verificación de cumplimiento de PCI DSS">
Dodo Payments es compatible con PCI DSS Nivel 1. Para cumplir con las obligaciones de cumplimiento, solo podemos recibir datos de tarjetas de otro procesador de pagos que sea compatible con PCI DSS Nivel 1.

Tu procesador actual debe proporcionar:
- Su actual Certificación de Cumplimiento de PCI (AOC), o su listado en el Registro Global de Proveedores de Servicios de Visa
- Confirmación de que soportan exportaciones seguras de datos de tarjetas

Consulta nuestra [Certificación de Cumplimiento (AOC)](/assets/dodo-payments-pci-dss-aoc.pdf) para compartir con tu procesador actual.
</Accordion>

<Accordion title="Integración de Dodo Payments">
Completa tu integración de Dodo Payments y asegúrate de que todos los nuevos clientes sean dirigidos al proceso de pago de Dodo Payments antes de comenzar la migración. Esto previene inconsistencias de datos durante el período de transición.
</Accordion>

<Accordion title="Autorización de acceso a datos">
Asegúrate de tener la autorización necesaria para solicitar una exportación de datos de tu procesador actual. Esto generalmente requiere:
- Verificación del propietario de la cuenta
- Autorización por escrito para el acceso a datos sensibles
- Completar los pasos de verificación de seguridad de tu procesador
</Accordion>
</AccordionGroup>

## Guía de migración paso a paso

<Steps>
<Step title="Solicitar exportación de método de pago a tu procesador actual">
Contacta al equipo de soporte de tu procesador de pagos actual y presenta una solicitud formal para una exportación completa de los tokens de tarjeta de tus clientes.

Necesitarás:
- Proporcionar verificación de tu identidad y autorización
- Solicitar que encripten la exportación utilizando la clave pública PGP de Dodo Payments (proporcionada a continuación)
- Pedirles que transfieran el archivo encriptado directamente a nuestro servidor SFTP seguro

<Info>
Las transferencias seguras se realizan directamente de servidor a servidor a través de SFTP. Tu Ingeniero de Soluciones proporcionará las credenciales SFTP para compartir con tu procesador actual.
</Info>

En promedio, este proceso toma alrededor de **2 semanas**, aunque el tiempo varía dependiendo de las políticas y tiempos de respuesta de tu procesador actual.
</Step>

<Step title="Preparar exportación de datos de suscriptores">
Mientras la transferencia del método de pago está en progreso, exporta los datos de tus suscriptores desde tu sistema heredado. Tu Ingeniero de Soluciones proporcionará una plantilla CSV y te guiará a través de los campos requeridos.

### Datos de suscriptores requeridos

| Campo | Descripción |
|-------|-------------|
| `customer_id` | Tu identificador interno de cliente |
| `email` | Dirección de correo electrónico del cliente |
| `name` | Nombre completo del cliente |
| `country` | Código de país de dos letras (ISO 3166-1 alpha-2) |
| `currency` | Código de moneda de tres letras (ISO 4217) |
| `payment_amount` | Monto de suscripción en la unidad monetaria más pequeña |
| `payment_interval` | Frecuencia de facturación (mensual, anual, etc.) |
| `next_billing_date` | Próxima fecha de facturación programada (ISO 8601) |
| `card_token` | Token único de tu procesador actual |

<Warning>
El campo `card_token` es crítico: se utiliza para emparejar los registros de suscriptores con sus métodos de pago durante la importación. Asegúrate de que coincida exactamente con el token en la exportación del método de pago.
</Warning>

### Formato de datos de ejemplo

```json
{
  "customers": [
    {
      "customer_id": "cus_abc123",
      "email": "jane.doe@example.com",
      "name": "Jane Doe",
      "country": "US",
      "currency": "USD",
      "payment_amount": 2999,
      "payment_interval": "monthly",
      "next_billing_date": "2025-01-15T00:00:00Z",
      "card_token": "tok_xyz789",
      "metadata": {
        "plan": "pro",
        "signup_source": "website"
      }
    }
  ]
}
```

</Step>

<Step title="Validación y pruebas">
Antes de la fecha de importación, nuestro equipo realizará:

- Verificación de la integridad del archivo y encriptación de los datos del método de pago
- Validación de tu CSV de suscriptores para el formato correcto
- Ejecución de una importación de prueba en nuestro entorno de sandbox
- Identificación y resolución de cualquier problema de mapeo de datos

<Check>
Recibirás un informe de validación confirmando que todo está listo para la importación en producción.
</Check>
</Step>

<Step title="Importación en producción">
En la fecha de importación acordada, nuestro equipo de ingeniería realizará:

1. Importar los datos del método de pago en nuestro Vault de Tarjetas compatible con PCI
2. Importar tus datos de suscriptores
3. Emparejar suscriptores con sus métodos de pago utilizando los tokens de tarjeta
4. Activar todas las suscripciones migradas

<Info>
El tiempo de procesamiento típico es de **2-3 días hábiles** para migraciones estándar, dependiendo del volumen de suscriptores.
</Info>
</Step>

<Step title="Verificación post-migración">
Después de que se complete la importación, recibirás:

- Un informe de migración completo
- Un CSV que mapea tus antiguos IDs de suscripción a nuevos IDs de Dodo Payments
- Acceso a todos los clientes migrados en tu panel de Dodo Payments

Recomendamos:
- Verificar una muestra de suscripciones migradas en tu panel
- Probar algunas renovaciones de suscripción en modo sandbox
- Actualizar las referencias de tu base de datos interna con los nuevos IDs
- Realizar un monitoreo paralelo durante el primer ciclo de facturación
</Step>
</Steps>

## Datos migrables

Los siguientes datos pueden ser transferidos durante una migración de suscripciones:

| Tipo de dato | Soportado | Notas |
|-----------|-----------|-------|
| Números de tarjetas de crédito/débito | ✅ | Encriptados a través de PGP |
| Fechas de expiración de tarjetas | ✅ | |
| Nombres de titulares de tarjetas | ✅ | |
| Direcciones de facturación | ✅ | |
| Direcciones de correo electrónico de clientes | ✅ | |
| Metadatos de clientes | ✅ | Pares clave-valor personalizados |
| Detalles de suscripción | ✅ | Plan, monto, intervalo |
| Tokens de Apple Pay | ❌ | No se pueden transferir |
| Tokens de Google Pay | ❌ | No se pueden transferir |
| Otros tokens de billetera | ❌ | No se pueden transferir |
| Historial de pagos | ❌ | Exportar desde tu procesador actual |

<Note>
El historial de pagos y los registros de transacciones no pueden ser migrados. Recomendamos exportar estos datos de tu procesador actual para tus registros antes de completar la transición.
</Note>

## Expectativas de cronograma

| Fase | Duración |
|-------|----------|
| Configuración inicial y planificación | 1-2 días |
| Transferencia de método de pago | ~2 semanas |
| Preparación de datos de suscriptores | 3-5 días (paralelo) |
| Validación y pruebas | 2-3 días |
| Importación en producción | 2-3 días hábiles |
| Verificación post-migración | 1-2 días |

<Tip>
La migración total generalmente toma **3-4 semanas** de principio a fin. Comenzar la preparación de datos de suscriptores mientras la transferencia del método de pago está en progreso ayuda a minimizar el cronograma general.
</Tip>
