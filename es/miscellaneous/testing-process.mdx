## Resumen

El modo de prueba te permite simular pagos sin procesar transacciones reales. Todas las credenciales de prueba solo funcionan en modo de prueba; nunca las uses en producción.

<CardGroup cols={3}>
<Card title="Sin Cargos Reales" icon="shield-check">
Las transacciones de prueba no procesan pagos reales ni afectan cuentas reales.
</Card>

<Card title="Acceso Completo a la API" icon="code">
Todos los puntos finales de la API funcionan de manera idéntica en los modos de prueba y en vivo.
</Card>

<Card title="Pruebas de Webhook" icon="webhook">
Los webhooks se activan para transacciones de prueba al igual que en producción.
</Card>
</CardGroup>

<Warning>
Las credenciales de prueba solo funcionan en modo de prueba. Intentar usarlas en modo en vivo resultará en fallos de pago.
</Warning>

## Prueba de UPI

Las pruebas de UPI utilizan identificadores VPA (Dirección de Pago Virtual) especiales que simulan diferentes resultados de pago.

| Estado | ID de UPI |
| :----- | :----- |
| Exito | `success@upi` |
| Fallo | `failure@upi` |

### Requisitos para la Prueba de UPI

- El país de facturación debe estar configurado como `IN`
- La moneda debe ser `INR`
- Para comerciantes no indios: La moneda adaptativa debe estar habilitada

Para la documentación completa de UPI, incluida la prueba de mandato de RBI para suscripciones, consulta la página de [Métodos de Pago de India](/features/payment-methods/india).

</Note>

### Pruebas de Afterpay

<Steps>
<Step title="Selecciona Afterpay">
Elige Afterpay como método de pago en la caja y haz clic en Pagar.
</Step>

<Step title="Prueba de pago exitoso">
Usa cualquier dirección de correo electrónico y dirección de envío válida para pagos exitosos.
</Step>

<Step title="Prueba de autenticación fallida">
Para simular el fallo, cierra la ventana modal de Afterpay en la página de redirección. La transición de pago va de `requires_action` a `requires_payment_method`.
</Step>
</Steps>

<Note>
Afterpay requiere un importe mínimo de transacción de **$50.01** para aparecer como opción de pago.
</Note>

Para la documentación completa de BNPL, incluida la prueba de Billie B2B, consulta la página de [Buy Now Pay Later](/features/payment-methods/bnpl).

## Prueba de Billeteras Digitales

### Apple Pay

<Steps>
<Step title="Habilitar modo de prueba">
Usa tus claves API de prueba de Dodo Payments.
</Step>

<Step title="Agrega una tarjeta a Apple Wallet">
Agrega una tarjeta real a tu Apple Wallet. En modo de prueba, la tarjeta no se cargará.
</Step>

<Step title="Completa la compra de prueba">
Abre la caja en un dispositivo Apple y completa el flujo de Apple Pay.
</Step>
</Steps>

<Warning>
Apple Pay requiere HTTPS. No aparecerá en `localhost` sin una configuración adecuada de SSL. La verificación de dominio también debe estar completa.
</Warning>

### Google Pay

<Steps>
<Step title="Únete al grupo de tarjetas de prueba">
[Únete al grupo de tarjetas de prueba de Google Pay](https://groups.google.com/g/googlepay-test-mode-stub-data) para que se añadan automáticamente tarjetas de prueba a tu billetera.
</Step>

<Step title="Habilitar modo de prueba">
Usa tus claves API de prueba de Dodo Payments.
</Step>

<Step title="Completa la compra de prueba">
Selecciona una de las tarjetas de prueba en Google Pay para completar la transacción.
</Step>
</Steps>

### Amazon Pay, Cash App Pay y RevolutPay

Usa tus claves API de prueba y sigue el flujo de caja estándar. Las transacciones de prueba se simulan sin cargos reales.

Para la documentación completa de billeteras digitales, incluida la verificación de dominio para Apple Pay, consulta la página de [Billeteras Digitales](/features/payment-methods/digital-wallets).

## Pruebas de Métodos Europeos

Los métodos de pago europeos (iDEAL, Bancontact, EPS, Multibanco) se pueden probar en modo sandbox.

<Steps>
<Step title="Habilitar modo de prueba">
Usa tus claves API de prueba de Dodo Payments.
</Step>

<Step title="Establecer dirección de facturación">
Configura el país de la dirección de facturación para que coincida con el método de pago:
- `NL` para iDEAL
- `BE` para Bancontact
- `AT` para EPS
- `PT` para Multibanco
</Step>

<Step title="Establecer moneda">
Los métodos europeos requieren la moneda EUR.
</Step>

<Step title="Completa el flujo de prueba">
Sigue el flujo de autenticación bancaria simulado en el entorno de prueba.
</Step>
</Steps>

Para la documentación completa de métodos de pago europeos, consulta la página de [Europa](/features/payment-methods/europe).

## Mejores Prácticas de Pruebas

<AccordionGroup>
<Accordion title="Prueba todos los escenarios de pago">
No solo pruebes pagos exitosos. Prueba rechazos, cancelaciones y casos extremos como fondos insuficientes.
</Accordion>

<Accordion title="Verifica el manejo de webhooks">
Asegúrate de que tus puntos finales de webhook procesen correctamente todos los tipos de eventos, especialmente `payment.succeeded`, `payment.failed`, y eventos de suscripción.
</Accordion>

<Accordion title="Prueba en dispositivos reales">
Para Apple Pay y Google Pay, prueba en dispositivos iOS y Android reales. Los simuladores no replican completamente el comportamiento de la billetera.
</Accordion>

<Accordion title="Prueba métodos regionales con direcciones correctas">
Los métodos de pago regionales (UPI, iDEAL, etc.) requieren direcciones de facturación coincidentes. Una dirección de facturación de EE. UU. no mostrará iDEAL.
</Accordion>

<Accordion title="Verifica los importes mínimos">
Los métodos BNPL requieren un mínimo de $50.01. Prueba que aparezcan o se oculten correctamente según el total del carrito.
</Accordion>
</AccordionGroup>
