## Detalles de la Tarjeta de Prueba

Dodo Payments proporciona un conjunto de números de Tarjeta de Prueba que puedes usar para simular varios escenarios de pago. Puedes usar estos números de tarjeta de prueba para crear pagos simulados en modo de prueba sin procesar pagos o cargos reales.

Los números de tarjeta de prueba solo son válidos en modo de prueba. No los uses para pagos reales.

## Detalles de Afterpay de Prueba

Al probar tu integración de Checkout, selecciona Afterpay como el método de pago y haz clic en el botón Pagar.

### Pago Exitoso
- Usa cualquier dirección de correo electrónico y dirección de envío válida para pagos exitosos.

### Autenticación Fallida
- Para probar el caso en el que el usuario no logra autenticarse, usa tus claves API de prueba y visualiza la página de redirección. En la página de redirección, cierra la ventana modal de Afterpay y verifica que el pago falló. El PaymentIntent transiciona de `requires_action` a `requires_payment_method`.

**Nota**: El monto mínimo requerido para transacciones de Afterpay es de $50.

Para PaymentIntents de captura manual en modo de prueba, el PaymentIntent no capturado caduca automáticamente 10 minutos después de la autorización exitosa.

## Otros Métodos de Pago

Para probar otros métodos de pago como Amazon Pay y Cash App, sigue los procedimientos de prueba habituales. Ten en cuenta que Apple Pay y Google Pay no funcionarán en modo de prueba.
