## Resumen

El modo de prueba te permite simular pagos sin procesar transacciones reales. Todas las credenciales de prueba solo funcionan en modo de prueba; nunca las uses en producción.

<CardGroup cols={3}>
{/* LOCKED_PATTERN_717af1b1289e4c0b6d1c95c8c204d71e */}
Las transacciones de prueba no procesan pagos reales ni afectan cuentas reales.
</Card>

{/* LOCKED_PATTERN_5ba03ec58a56547ac9750ef18da96a7f */}
Todos los endpoints de la API funcionan idénticamente en los modos de prueba y en vivo.
</Card>

{/* LOCKED_PATTERN_1f50b75a756a498817926e42de6e14a9 */}
Los webhooks se activan para las transacciones de prueba igual que en producción.
</Card>
</CardGroup>

<Warning>
Las credenciales de prueba solo funcionan en modo de prueba. Si intentas usarlas en modo en vivo, los pagos fallarán.
</Warning>

## Prueba de UPI

Las pruebas de UPI utilizan identificadores VPA (Dirección de Pago Virtual) especiales que simulan diferentes resultados de pago.

| Estado | ID de UPI |
| :----- | :----- |
| Éxito | `success@upi` |
| Fallo | `failure@upi` |

### Requisitos para la Prueba de UPI

- El país de facturación debe establecerse en `IN`
- La moneda debe ser `INR`
- Para comerciantes fuera de India: la Moneda Adaptativa debe estar habilitada

Para la documentación completa de UPI, incluida la prueba de mandato de RBI para suscripciones, consulta la página de [Métodos de Pago de India](/features/payment-methods/india).

## Prueba de BNPL

Los proveedores de Buy Now Pay Later tienen requisitos específicos de datos para pruebas.

### Datos de Prueba de Klarna

Usa estos detalles para simular pagos con Klarna en modo de prueba:

| Campo | Aprobado | Denegado |
| :---- | :------- | :----- |
| **Fecha de Nacimiento** | 07-10-1970 | 07-10-1970 |
| **Nombre** | Prueba | Prueba |
| **Apellido** | Persona-us | Persona-us |
| **Correo Electrónico** | customer@email.us | customer+denied@email.us |
| **Calle** | Amsterdam Ave | Amsterdam Ave |
| **Número de Casa** | 509 | 509 |
| **Ciudad** | Nueva York | Nueva York |
| **Estado** | Nueva York | Nueva York |
| **Código Postal** | 10024-3941 | 10024-3941 |
| **Teléfono** | +13106683312 | +13106354386 |

<Note>
Klarna requiere un monto mínimo de transacción de **$50.01** para aparecer como opción de pago.
</Note>

### Apple Pay

<Steps>
{/* LOCKED_PATTERN_540056f13df545529727751bb5b93f77 */}
Usa tus claves de API de prueba de Dodo Payments.
</Step>

{/* LOCKED_PATTERN_837aa82e384540b5d8217cef3491dd36 */}
Agrega una tarjeta real a tu Apple Wallet. En modo de prueba, la tarjeta no se cobrará.

<Warning>
Apple Pay requiere HTTPS. No aparecerá en `localhost` sin una configuración SSL adecuada. También debe completarse la verificación del dominio.
</Warning>

## Pruebas de Métodos Europeos

Los métodos de pago europeos (iDEAL, Bancontact, EPS, Multibanco) se pueden probar en modo sandbox.

<Steps>
{/* LOCKED_PATTERN_540056f13df545529727751bb5b93f77 */}
Usa tus claves de API de prueba de Dodo Payments.
</Step>

{/* LOCKED_PATTERN_73a737ada9108be9fcbfe7701d196c25 */}
Establece el país de la dirección de facturación para que coincida con el método de pago:
- `NL` para iDEAL
- `BE` para Bancontact
- `AT` para EPS
- `PT` para Multibanco
</Step>

{/* LOCKED_PATTERN_111b236b56d1ef796758b64d1fe1fcd1 */}
Los métodos europeos requieren moneda EUR.
</Step>

{/* LOCKED_PATTERN_3efbb33511174ca4755c102c30869543 */}
Sigue el flujo simulado de autenticación bancaria en el entorno de prueba.
</Step>
</Steps>

Para la documentación completa de métodos de pago europeos, consulta la página de [Europa](/features/payment-methods/europe).

## Mejores Prácticas de Pruebas

<AccordionGroup>
{/* LOCKED_PATTERN_065a271b5abfd247b6320fbca34af90c */}
No solo pruebes pagos exitosos. Prueba rechazos, cancelaciones y casos límite como fondos insuficientes.
</Accordion>

{/* LOCKED_PATTERN_b9eb520012676a2badc82027085ca781 */}
Asegúrate de que tus endpoints de webhook procesen correctamente todos los tipos de eventos, especialmente `payment.succeeded`, `payment.failed` y eventos de suscripción.

Usa el [CLI de Dodo Payments](/developer-resources/sdks/cli#webhook-testing) para probar webhooks localmente:
- `dodo wh listen` reenvía webhooks de prueba en vivo a tu servidor local
- `dodo wh trigger` envía cargas simuladas para los 22 tipos de eventos
</Accordion>

{/* LOCKED_PATTERN_7fd9589439eedb7dd6977012ac791713 */}
Para Apple Pay y Google Pay, prueba en dispositivos iOS y Android reales. Los simuladores no replican completamente el comportamiento del wallet.
</Accordion>

{/* LOCKED_PATTERN_ef9afd776f8103114546b31cb4fef641 */}
Los métodos regionales de pago (UPI, iDEAL, etc.) requieren direcciones de facturación coincidentes. Una dirección de facturación de EE. UU. no mostrará iDEAL.
</Accordion>

{/* LOCKED_PATTERN_52e65b7bb9610a0aa115d3b6ba276057 */}
Los métodos BNPL requieren un mínimo de $50.01. Verifica que aparezcan u oculten correctamente según el total del carrito.
</Accordion>
</AccordionGroup>
