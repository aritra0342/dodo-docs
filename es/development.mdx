<Info>
  **Requisito previo**: Por favor, instala Node.js (versión 19 o superior) antes de continuar.
</Info>

Sigue estos pasos para instalar y ejecutar Mintlify en tu sistema operativo:

**Paso 1**: Instala Mintlify:

<CodeGroup>

  ```bash npm
  npm i -g mintlify
  ```

```bash yarn
yarn global add mintlify
```

</CodeGroup>

**Paso 2**: Navega al directorio de docs (donde se encuentra el archivo `mint.json`) y ejecuta el siguiente comando:

```bash
mintlify dev
```

Una vista previa local de tu documentación estará disponible en `http://localhost:3000`.

### Puertos Personalizados

Por defecto, Mintlify utiliza el puerto 3000. Puedes personalizar el puerto en el que se ejecuta Mintlify utilizando la bandera `--port`. Para ejecutar Mintlify en el puerto 3333, por ejemplo, usa este comando:

```bash
mintlify dev --port 3333
```

Si intentas ejecutar Mintlify en un puerto que ya está en uso, utilizará el siguiente puerto disponible:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Versiones de Mintlify

Ten en cuenta que cada lanzamiento de CLI está asociado con una versión específica de Mintlify. Si tu sitio web local no coincide con la versión de producción, actualiza el CLI:

<CodeGroup>

```bash npm
npm i -g mintlify@latest
```

```bash yarn
yarn global upgrade mintlify
```

</CodeGroup>

## Validación de Enlaces

El CLI puede ayudar a validar los enlaces de referencia realizados en tu documentación. Para identificar enlaces rotos, utiliza el siguiente comando:

```bash
mintlify broken-links
```

## Formateo de Código

Sugerimos usar extensiones en tu IDE para reconocer y formatear MDX. Si eres usuario de VSCode, considera la [extensión MDX para VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) para resaltar la sintaxis, y [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) para el formateo de código.

## Solución de Problemas

<AccordionGroup>
  <Accordion title='Error: No se pudo cargar el módulo "sharp" utilizando el runtime darwin-arm64'>

    Esto puede deberse a una versión desactualizada de node. Intenta lo siguiente:
    1. Elimina la versión actualmente instalada de mintlify: `npm remove -g mintlify`
    2. Actualiza a Node v19 o superior.
    3. Reinstala mintlify: `npm install -g mintlify`
  </Accordion>

  <Accordion title="Problema: Encuentro un error desconocido">
  
    Solución: Ve a la raíz de tu dispositivo y elimina la carpeta \~/.mintlify. Después, ejecuta `mintlify dev` nuevamente.
  </Accordion>
</AccordionGroup>

¿Tienes curiosidad sobre qué cambió en la versión del CLI? [Consulta el registro de cambios del CLI.](https://www.npmjs.com/package/mintlify?activeTab=versions)
