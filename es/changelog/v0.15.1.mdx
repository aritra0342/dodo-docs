## Nuevas Funciones 🚀

### Mejoras de la API
- Lanzada la API de Facturas, que permite a los comerciantes y clientes descargar facturas usando `payment_id`. Visita la documentación de la [API de Facturas](/api-reference/payments/get-invoice) para más detalles.
- Implementado un nuevo evento de webhook `subscription.renewed` para notificaciones de renovación de suscripciones.

### Actualizaciones de la Plataforma
- Introducido el [Wall of Love](https://dodopayments.com/wall-of-love) de Dodo Payments, mostrando testimonios de clientes.
- Añadida funcionalidad de compartición social para transacciones exitosas.
    <img src="/images/changelog/share-transaction.png" alt="Función de Compartir Transacción"/>
- Lanzada la herramienta Generador de Políticas para la generación automática de documentos de política de privacidad, términos de servicio y política de reembolso. [Prueba el Generador de Políticas](https://dodopayments.com/tools/policy-generator).

### Mejoras en la Documentación
- Publicadas guías completas:
    - [Guía de Integración de Suscripciones](/developer-resources/subscription-integration-guide)
    - [Guía de Eventos de Webhook](/developer-resources/webhooks/intents/webhook-events-guide)

## Mejoras y Correcciones de Errores 🔧

### Actualizaciones de la API
- Añadido el parámetro `trial_period_days` a la [API de Crear Suscripción](/api-reference/subscriptions/post-subscriptions) para anular los períodos de prueba a nivel de producto.
- Mejorada la lógica de identificación del cliente: ahora se utiliza un `customer_id` consistente en pagos y suscripciones para el mismo correo electrónico.
- Implementadas diversas optimizaciones de rendimiento y corregidos errores menores.
