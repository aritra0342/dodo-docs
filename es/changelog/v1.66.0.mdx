## Nuevas Funciones 🚀

1. **`subscription.updated` Webhook**  
   Un nuevo evento de webhook que se activa en cada actualización del objeto de suscripción, brindándote notificaciones en tiempo real sobre cualquier cambio en tus suscripciones.
   
   Utiliza este webhook para:
   - Obtener notificaciones instantáneas cuando cambian los detalles de la suscripción
   - Mantener el estado de tu aplicación sincronizado con Dodo Payments
   - Reaccionar a actualizaciones de suscripciones sin sondear la API
   
   <Tip>
   Suscríbete a `subscription.updated` para eliminar la necesidad de sondear y mantener tu sistema sincronizado con los cambios en tiempo real de las suscripciones.
   </Tip>
   
   Más información: [Webhooks de suscripción](/developer-resources/webhooks/intents/subscription)

2. **Soporte de Dirección Mínima en Checkout**  

   <Frame>
   <img src="/images/changelog/minimal-address.png" alt="Soporte de dirección mínima en el pago" />
   </Frame>

   Reduce la fricción del pago con el nuevo modo de recolección de dirección mínima. En lugar de recopilar los datos completos de la dirección, el modo de dirección mínima solo recopila:
   
   - **País**: Siempre requerido para la determinación de impuestos
   - **Código postal**: Solo se recopila en regiones donde sea necesario para el cálculo de impuestos sobre las ventas, IVA o GST
   
   <Check>
   Resultado: finalización del pago significativamente más rápida con menos campos de formulario para los clientes.
   </Check>
   
   **Activa la dirección mínima:**
   
   <Tabs>
   <Tab title="Checkout Sessions">

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [{ product_id: 'prod_123', quantity: 1 }],
     minimal_address: true
   });
   ```

   </Tab>
   
   <Tab title="Static Payment Links">
   Añade `minimalAddress=true` a la URL de tu enlace de pago:
   

   ```
   https://checkout.dodopayments.com/buy/prod_123?minimalAddress=true
   ```

   </Tab>
   </Tabs>
   
   <Info>
   De forma predeterminada, el checkout sigue recopilando direcciones completas. Gradualmente pasaremos todos los enlaces de pago al modo de dirección mínima. Activa la bandera de función ahora para comenzar a beneficiarte de la reducción de fricción en el pago.
   </Info>

3. **Claves de API de solo lectura**  
   Crea claves de API con permisos de solo lectura para integraciones más seguras. Al crear una nueva clave de API, desmarca **Enable write access** para generar una clave de solo lectura.
   
   Las claves de solo lectura pueden:
   - Obtener datos de pagos, suscripciones y clientes
   - Listar productos y recuperar detalles
   - Acceder a informes y análisis
   
   Las claves de solo lectura **no pueden**:
   - Crear o modificar pagos
   - Actualizar suscripciones
   - Cambiar configuraciones de productos
   
   <Tip>
   Desmarca “Enable write access” para integraciones del panel de control, herramientas de análisis y cualquier sistema que solo necesite ver datos sin realizar cambios.
   </Tip>

4. **UPI AutoPay y suscripciones con tarjeta RuPay: implementación completa**  
   UPI AutoPay y las suscripciones con tarjeta RuPay ya están disponibles para todos los negocios. Esto permite pagos recurrentes para clientes indios con:
   
   - **UPI AutoPay**: Cobros recurrentes automatizados mediante mandato UPI
   - **Tarjetas RuPay**: Compatibilidad con suscripciones para la red de tarjetas nacional de la India
   
   <Check>
   Resultado: opciones ampliadas de pago por suscripción para el mercado indio con gestión de mandatos compatible con el RBI.
   </Check>
   
   Más información: [Subscriptions with RBI-Compliant Mandates](/features/subscription#subscriptions-with-rbi-compliant-mandates)

5. **API de Vista Previa de Cambio de Plan**  
   Un nuevo endpoint de API para previsualizar cambios en el plan de suscripción antes de comprometerse a ellos. Obtén el monto exacto del cargo y ve cómo se verá la suscripción después del cambio, sin hacer modificaciones reales.
   

   ```javascript
   const preview = await client.subscriptions.previewChangePlan('sub_123', {
     product_id: 'prod_pro_plan',
     proration_billing_mode: 'prorated_immediately',
     quantity: 1
   });
   
   console.log('Charge amount:', preview.immediate_charge.summary);
   console.log('New plan details:', preview.new_plan);
   ```

   
   Usa esto para:
   - Mostrar a los clientes exactamente lo que pagarán antes de confirmar mejoras/reducciones
   - Construir calculadoras de precios y herramientas de comparación de planes
   - Reducir tickets de soporte mejorando la transparencia de precios
   
   Aprende más: [API de Vista Previa de Cambio de Plan](/api-reference/subscriptions/preview-change-plan)
