## Nuevas Funciones 🚀

1. **`subscription.updated` Webhook**  
   Un nuevo evento de webhook que se activa en cada actualización del objeto de suscripción, brindándote notificaciones en tiempo real sobre cualquier cambio en tus suscripciones.
   
   Usa este webhook para:
   - Obtener notificaciones instantáneas cuando cambien los detalles de la suscripción
   - Mantener el estado de tu aplicación sincronizado con Dodo Payments
   - Reaccionar a las actualizaciones de suscripción sin sondear la API
   
   <Tip>
   Suscríbete a `subscription.updated` para eliminar la necesidad de sondear y mantener tu sistema sincronizado con los cambios de suscripción en tiempo real.
   </Tip>
   
   Aprende más: [Webhooks de Suscripción](/developer-resources/webhooks/intents/subscription)

2. **Soporte de Dirección Mínima en Checkout**  

   <Frame>
   <img src="/images/changelog/minimal-address.png" alt="Soporte de Dirección Mínima en Checkout" />
   </Frame>

   Reduce la fricción en el checkout con el nuevo modo de recolección de dirección mínima. En lugar de recopilar todos los detalles de la dirección, el modo de dirección mínima solo recopila:
   
   - **País**: Siempre requerido para la determinación de impuestos
   - **Código ZIP/Código postal**: Solo se recopila en regiones donde es necesario para el cálculo del impuesto sobre ventas, IVA o GST
   
   <Check>
   Resultado: Finalización del checkout significativamente más rápida con menos campos de formulario para los clientes.
   </Check>
   
   **Habilitar dirección mínima:**
   
   <Tabs>
   <Tab title="Sesiones de Checkout">

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [{ product_id: 'prod_123', quantity: 1 }],
     minimal_address: true
   });
   ```

   </Tab>
   
   <Tab title="Enlaces de Pago Estáticos">
   Agrega `minimalAddress=true` a la URL de tu enlace de pago:
   

   ```
   https://checkout.dodopayments.com/buy/prod_123?minimalAddress=true
   ```

   </Tab>
   </Tabs>
   
   <Info>
   Por defecto, el checkout continúa recopilando direcciones completas. Gradualmente, vamos a transitar todos los enlaces de checkout al modo de dirección mínima. Habilita la bandera de función ahora para comenzar a beneficiarte de la reducción de la fricción en el checkout.
   </Info>

3. **Claves API de Solo Lectura**  
   Crea claves API con permisos de solo lectura para integraciones más seguras. Al crear una nueva clave API, desmarca **Habilitar acceso de escritura** para crear una clave de solo lectura.
   
   Las claves de solo lectura pueden:
   - Obtener datos de pagos, suscripciones y clientes
   - Listar productos y recuperar detalles
   - Acceder a informes y análisis
   
   Las claves de solo lectura **no pueden**:
   - Crear o modificar pagos
   - Actualizar suscripciones
   - Cambiar configuraciones de productos
   
   <Tip>
   Desmarca "Habilitar acceso de escritura" para integraciones de panel, herramientas de análisis y cualquier sistema que solo necesite ver datos sin hacer cambios.
   </Tip>

4. **UPI AutoPay y Suscripciones con Tarjetas RuPay - Implementación Completa**  
   UPI AutoPay y las suscripciones con tarjetas RuPay ahora están disponibles para todos los negocios. Esto permite pagos recurrentes para clientes indios con:
   
   - **UPI AutoPay**: Cargos recurrentes automatizados a través de mandato UPI
   - **Tarjetas RuPay**: Soporte de suscripción para la red de tarjetas domésticas de India
   
   <Check>
   Resultado: Opciones de pago de suscripción ampliadas para el mercado indio con manejo de mandatos conforme a RBI.
   </Check>
   
   Aprende más: [Suscripciones con Mandatos Conformes a RBI](/features/subscription#subscriptions-with-rbi-compliant-mandates)

5. **API de Vista Previa de Cambio de Plan**  
   Un nuevo endpoint de API para previsualizar cambios en el plan de suscripción antes de comprometerse a ellos. Obtén el monto exacto del cargo y ve cómo se verá la suscripción después del cambio, sin hacer modificaciones reales.
   

   ```javascript
   const preview = await client.subscriptions.previewChangePlan('sub_123', {
     product_id: 'prod_pro_plan',
     proration_billing_mode: 'prorated_immediately',
     quantity: 1
   });
   
   console.log('Charge amount:', preview.immediate_charge.summary);
   console.log('New plan details:', preview.new_plan);
   ```

   
   Usa esto para:
   - Mostrar a los clientes exactamente lo que pagarán antes de confirmar mejoras/reducciones
   - Construir calculadoras de precios y herramientas de comparación de planes
   - Reducir tickets de soporte mejorando la transparencia de precios
   
   Aprende más: [API de Vista Previa de Cambio de Plan](/api-reference/subscriptions/preview-change-plan)
