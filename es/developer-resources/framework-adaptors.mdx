Dodo Payments proporciona adaptadores de framework oficiales que simplifican la integración de pagos. Cada adaptador está diseñado para funcionar sin problemas con las convenciones de tu framework, ofreciendo manejo de checkout, portal de clientes y webhooks desde el primer momento.

<Tip>
Los adaptadores de framework te permiten integrar Dodo Payments en menos de 10 líneas de código. Gestionan la autenticación, el análisis de solicitudes y el formateo de respuestas automáticamente.
</Tip>

## Funcionalidades clave

Todos los adaptadores de framework ofrecen estas capacidades integradas:

| Característica | Descripción |
|----------------|-------------|
| **Controlador de checkout** | Compatibilidad con flujos de checkout estáticos, dinámicos y basados en sesiones |
| **Portal del cliente** | Controlador preconstruido para la gestión de suscripciones y facturación |
| **Controlador de webhooks** | Verificación segura de firmas con controladores de eventos tipados |
| **Configuración del entorno** | Configuración sencilla mediante variables de entorno |
| **Seguridad de tipos** | Compatibilidad total con TypeScript y cargas útiles tipadas |

## Inicio rápido

Comienza con cualquier adaptador de framework en tres pasos:

<Steps>
{/* LOCKED_PATTERN_d18984bd7669f0bdbf8a5e25faa0b786 */}
  Usa tu gestor de paquetes para instalar el adaptador específico del framework:

  <Tabs>
  <Tab title="Next.js">

  ```bash
  npm install @dodopayments/nextjs
  ```

  </Tab>
  
  <Tab title="Nuxt">

  ```bash
  npm install @dodopayments/nuxt
  ```

  </Tab>
  
  <Tab title="Express">

  ```bash
  npm install @dodopayments/express
  ```

  </Tab>
  
  <Tab title="Hono">

  ```bash
  npm install @dodopayments/hono
  ```

  </Tab>
  <Tab title="Astro">
  ```bash
  npm install @dodopayments/astro
  ```

  </Tab>
  <Tab title="SvelteKit">
  ```bash
  npm install @dodopayments/sveltekit
  ```

  </Tab>
  </Tabs>
</Step>

{/* LOCKED_PATTERN_f8f1c4a54590eca96bda1740ea80800b */}
  Add your Dodo Payments credentials to your environment:

  ```env
  DODO_PAYMENTS_API_KEY=your-api-key
  DODO_PAYMENTS_WEBHOOK_KEY=your-webhook-secret
  DODO_PAYMENTS_RETURN_URL=https://yourdomain.com/checkout/success
  DODO_PAYMENTS_ENVIRONMENT="test_mode" # or "live_mode"
  ```

  <Warning>
  Never commit your `.env` file or secrets to version control.
  </Warning>
</Step>

{/* LOCKED_PATTERN_fe5b6f5fd1b8c6ffa573b7d70dc83e35 */}
  Set up your checkout, customer portal, and webhook routes:

  <Tabs>
  <Tab title="Next.js">

  ```typescript
  // app/checkout/route.ts
  import { Checkout } from "@dodopayments/nextjs";

  export const GET = Checkout({
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
    environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
  });
  ```

  </Tab>
  
  <Tab title="Express">

  ```typescript
  import { checkoutHandler } from '@dodopayments/express';

  app.get('/api/checkout', checkoutHandler({
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
    environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
  }));
  ```

  </Tab>
  
  <Tab title="Hono">

  ```typescript
  import { Checkout } from "@dodopayments/hono";

  app.get('/checkout', Checkout({
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
    environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
  }));
  ```

  </Tab>
  </Tabs>

  <Check>
  Ya estás listo para procesar pagos. Visita las páginas individuales de adaptadores para guías detalladas y todas las opciones disponibles.
  </Check>
</Step>
</Steps>

## Tipos de flujo de checkout

Todos los adaptadores admiten tres tipos de flujo de checkout:

<AccordionGroup>
{/* LOCKED_PATTERN_54ec7a6c810897db7154e4dbddb9180e */}
Usa checkout estático para enlaces de pago simples y compartibles. Pasa el ID del producto como parámetro de consulta:

```
/api/checkout?productId=pdt_xxx&quantity=1
```

Admite completado previo opcional del cliente y personalización mediante parámetros de consulta.
</Accordion>

{/* LOCKED_PATTERN_3a705ba64e435adc71acb6fb44c2370a */}
Usa checkout dinámico para crear pagos programáticamente con detalles personalizados:

```json
{
  "product_id": "pdt_xxx",
  "customer": {
    "email": "customer@example.com",
    "name": "John Doe"
  },
  "quantity": 1
}
```

Admite pagos únicos y suscripciones.
</Accordion>

{/* LOCKED_PATTERN_c9e7172c02eea079bd96ccf0ae70f411 */}
Utiliza sesiones de checkout para la experiencia de checkout más flexible con soporte para carritos:

```json
{
  "product_cart": [
    { "product_id": "pdt_xxx", "quantity": 1 },
    { "product_id": "pdt_yyy", "quantity": 2 }
  ],
  "customer": {
    "email": "customer@example.com"
  }
}
```

Obtén más información en la [Guía de sesiones de checkout](/developer-resources/checkout-session).
</Accordion>
</AccordionGroup>

## Manejo de eventos de webhook

Todos los adaptadores ofrecen manejo de webhooks con tipado seguro y callbacks de eventos granulares:

```typescript
Webhooks({
  webhookKey: process.env.DODO_PAYMENTS_WEBHOOK_KEY,
  onPayload: async (payload) => {
    // Handle any webhook event
  },
  onPaymentSucceeded: async (payload) => {
    // Handle successful payments
  },
  onSubscriptionActive: async (payload) => {
    // Handle new subscriptions
  },
  // ... 20+ event types supported
});
```

<Info>
Todos los controladores de webhook verifican automáticamente las firmas y validan las cargas útiles usando esquemas Zod. Las solicitudes inválidas se rechazan con códigos de error apropiados.
</Info>
