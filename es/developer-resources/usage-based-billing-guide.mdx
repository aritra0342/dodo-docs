## Creando un Medidor

Los medidores definen cómo se agregan y miden tus eventos de uso para fines de facturación.

Antes de crear un medidor, planifica tu estrategia de seguimiento de uso:
- Identifica qué eventos de uso deseas rastrear
- Determina cómo deben agregarse los eventos (conteo, suma, etc.)
- Define cualquier requisito de filtrado para casos de uso específicos

## Vinculando el Medidor a un Producto

Una vez que hayas creado tu medidor, necesitas vincularlo a un producto para habilitar la facturación basada en el uso. Este proceso conecta los datos de uso de tu medidor con las reglas de precios para la facturación del cliente.

Vincular medidores a productos establece la conexión entre el seguimiento de uso y la facturación:
- Los productos definen las reglas de precios y el comportamiento de facturación
- Los medidores proporcionan datos de uso para los cálculos de facturación
- Se pueden vincular múltiples medidores a un solo producto para escenarios de facturación complejos

### Proceso de Configuración del Producto

Transforma tus datos de uso en cargos facturables configurando correctamente los ajustes de tu producto:

<Steps>
{/* LOCKED_PATTERN_c6c0a713c4c40a6d7afb5356f839b072 */}
Navega a la página de creación o edición de tu producto y selecciona **Usage-Based** como tipo de producto.

</Step>

{/* LOCKED_PATTERN_96b3ccc2ef72fc5c11840c638f374759 */}
Haz clic en **Associated Meter** para abrir el panel de selección de medidores desde el lateral.

Este panel te permite configurar qué medidores rastrearán el uso para este producto.
</Step>

{/* LOCKED_PATTERN_80a284cd85458e962fc3cec7c255149a */}
En el panel de selección de medidores:

1. Haz clic en **Agregar Medidores** para ver los medidores disponibles
2. Selecciona el medidor que creaste de la lista desplegable
3. El medidor seleccionado aparecerá en la configuración de tu producto

</Step>

{/* LOCKED_PATTERN_6b0ad6e340264c8baa56736989bc3e50 */}
Establece el precio para cada unidad de uso rastreada por tu medidor.

{/* LOCKED_PATTERN_213cd3b101c5416e3cd966a3d8652d15 */}
Define cuánto cobrar por cada unidad medida por tu medidor.

**Ejemplo**: Establecer `$0.50` por unidad significa:
- 1,000 unidades consumidas = 1,000 × $0.50 = 500.00 facturados
- 500 unidades consumidas = 500 × $0.50 = 250.00 facturados
- 100 unidades consumidas = 100 × $0.50 = 50.00 facturados
</ParamField>

</Step>

{/* LOCKED_PATTERN_c1485ee2c65c25ecef83c7e0c270509b */}
Configura una asignación de uso gratuita antes de que comience la facturación.

{/* LOCKED_PATTERN_a57cc2829e5cf6a714d2d75f1b5361ce */}
Número de unidades que los clientes pueden consumir sin cargo antes de que comience el cálculo del uso pagado.

**Cómo funciona**:
- Umbral gratuito: 100 unidades
- Precio por unidad: $0.50
- Uso del cliente: 250 unidades
- **Cálculo**: (250 - 100) × $0.50 = **$75.00** facturados
</ParamField>

<Info>
Los umbrales gratuitos son ideales para modelos freemium, periodos de prueba o para proporcionar a los clientes una asignación base incluida en su plan.
</Info>

<Check>
El umbral gratuito se aplica a cada ciclo de facturación, dando a los clientes nuevas asignaciones mensualmente o según tu calendario de facturación.
</Check>
</Step>

{/* LOCKED_PATTERN_08395c29a4a5c780a5cf3de53acf78b9 */}
Revisa la configuración del medidor y de precios, luego haz clic en **Save Changes** para finalizar la configuración.

<Check>
Tu producto ahora está configurado para facturación basada en el uso y cobrará automáticamente a los clientes según su consumo medido.
</Check>

**Qué sucede a continuación**:
- Los eventos de uso enviados a tu medidor se rastrearán y agregarán
- Los cálculos de facturación aplicarán automáticamente tus reglas de precios
- Los clientes se les cobrará según el consumo real durante cada ciclo de facturación
</Step>
</Steps>

<Note>
Recuerda que puedes agregar hasta 10 medidores por producto, lo que permite un seguimiento sofisticado del uso en múltiples dimensiones como llamadas a la API, almacenamiento, tiempo de cómputo y métricas personalizadas.
</Note>

## Enviando Eventos de Uso

Una vez que tu medidor esté configurado, puedes comenzar a enviar eventos de uso desde tu aplicación para rastrear el uso del cliente.

### Estructura del Evento

Cada evento de uso debe incluir estos campos requeridos:

<ParamField body="event_id" type="string" required>
Identificador único para este evento específico. Debe ser único en todos los eventos.
</ParamField>

<ParamField body="customer_id" type="string" required>
La ID de cliente de Dodo Payments a la que debe atribuirse este uso.
</ParamField>

<ParamField body="event_name" type="string" required>
El nombre del evento que coincide con la configuración de tu medidor. Los nombres de eventos activan el medidor correspondiente.
</ParamField>

<ParamField body="timestamp" type="string">
Marca de tiempo ISO 8601 cuando ocurrió el evento. Por defecto usa el tiempo actual si no se proporciona.
</ParamField>

<ParamField body="metadata" type="object">
Propiedades adicionales para filtrado y agregación. Incluye los valores a los que hace referencia la configuración "Over Property" o las condiciones de filtrado de tu medidor.
</ParamField>

### Ejemplos de API de Eventos de Uso

Envía eventos de uso a tus medidores configurados utilizando la API de Eventos:

<CodeGroup>

```javascript Node.js
const response = await fetch('https://test.dodopayments.com/events/ingest', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.DODO_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    events: [
      {
        event_id: "api_call_1234",
        customer_id: "cus_atXa1lklCRRzMicTqfiw2", 
        event_name: "api.call",
        timestamp: new Date().toISOString(),
        metadata: {
          endpoint: "/v1/orders",
          method: "POST",
          response_size: 1024
        }
      }
    ]
  })
});
```

```python Python

from datetime import datetime

        ]
    }
)
```

## Solución de Problemas

Resuelve problemas comunes con la implementación de facturación basada en el uso y asegura un seguimiento y facturación precisos.

### Problemas Comunes

La mayoría de los problemas de facturación basada en el uso caen en estas categorías:
- Problemas de entrega y procesamiento de eventos
- Problemas de configuración de medidores
- Errores de tipo de datos y formato
- Problemas de ID de cliente y autenticación

### Pasos de Depuración

Al solucionar problemas de facturación basada en el uso:
1. Verifica la entrega de eventos en la pestaña de análisis de Eventos
2. Verifica que la configuración del medidor coincida con la estructura de tu evento
3. Valida los IDs de cliente y la autenticación de la API
4. Revisa las condiciones de filtrado y los ajustes de agregación
