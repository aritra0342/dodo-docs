Dodo Payments proporciona SDKs oficiales para múltiples lenguajes de programación, cada uno diseñado con las mejores prácticas específicas del lenguaje y características modernas para una integración de pagos sin problemas.

<Tip>
Utiliza siempre la versión más reciente del SDK para acceder a las funciones e mejoras más recientes. Revisa tu gestor de paquetes periódicamente para asegurarte de tener acceso a todas las capacidades de Dodo Payments.
</Tip>

## Inicio rápido

Comienza con cualquier SDK en minutos:

<Steps>
<Step title="Install the SDK">
  Utiliza el gestor de paquetes de tu lenguaje para instalar el SDK
  
  <Tabs>
  <Tab title="TypeScript/Node.js">

  ```bash
  npm install dodopayments
  ```

  </Tab>
  
  <Tab title="Python">

  ```bash
  pip install dodopayments
  ```

  </Tab>
  
  <Tab title="PHP">

  ```bash
  composer require dodopayments/client
  ```

  </Tab>
  
  <Tab title="Go">

  ```bash
  go get github.com/dodopayments/dodopayments-go
  ```

  </Tab>
  </Tabs>
</Step>

<Step title="Initialize the Client">
  Configura el cliente con tu clave de API
  
  <Tabs>
  <Tab title="TypeScript">

  ```typescript
  import DodoPayments from 'dodopayments';
  const client = new DodoPayments({ bearerToken: 'your_api_key' });
  ```

  </Tab>
  
  <Tab title="Python">

  ```python
  from dodopayments import DodoPayments
  client = DodoPayments(bearer_token="your_api_key")
  ```

  </Tab>
  
  <Tab title="PHP">

  ```php
  use Dodopayments\Client;
  $client = new Client(bearerToken: 'your_api_key');
  ```

  </Tab>
  
  <Tab title="Go">

  ```go
  import "github.com/dodopayments/dodopayments-go"
  client := dodopayments.NewClient(option.WithBearerToken("your_api_key"))
  ```

  </Tab>
  </Tabs>
  
  <Warning>
  Siempre almacena tus claves de API de forma segura usando variables de entorno. Nunca las subas al control de versiones.
  </Warning>
</Step>

<Step title="Make Your First API Call">
  Crea una sesión de checkout o pago
  
  <Check>
  ¡Ya estás listo para procesar pagos! Visita las páginas individuales de cada SDK para guías y ejemplos detallados.
  </Check>
</Step>
</Steps>

## Características clave

Todos los SDK comparten estas capacidades fundamentales:

- **Seguridad de tipos**: Tipado fuerte para seguridad en tiempo de compilación y mejor soporte de IDE
- **Manejo de errores**: Manejo de excepciones integral con mensajes de error detallados
- **Autenticación**: Autenticación simple con clave API y soporte para variables de entorno
- **Soporte asíncrono**: Patrones modernos de async/await donde sea aplicable
- **Auto-paginación**: Paginación automática para respuestas de lista
- **Facturación basada en uso**: Soporte integrado para rastrear e ingerir eventos de uso
- **Pruebas**: Soporte completo para entornos de sandbox para desarrollo y pruebas

## SDK de React Native

Para aplicaciones móviles, proporcionamos un SDK dedicado de React Native:

<Card title="React Native SDK" icon="mobile" href="/developer-resources/react-native-integration">
  Crea experiencias de pago seguras para apps de iOS y Android con componentes de UI nativos y recopilación simplificada de datos de pago
</Card>

**Características:**
- Componentes de UI nativos para Android e iOS
- Seguridad simplificada para la recolección de datos de pago sensibles
- Soporte para múltiples métodos de pago
- Integración sin problemas con aplicaciones de React Native

<Info>
Apple Pay, Google Pay, Cash App y UPI no son compatibles actualmente en el SDK de React Native. El soporte para estos métodos de pago está planificado para futuras versiones.
</Info>

## Interfaz de línea de comandos

Para flujos de trabajo basados en terminal y automatización:

<Card title="CLI" icon="terminal" href="/developer-resources/sdks/cli">
  Interfaz de línea de comandos generada automáticamente con soporte para los 78 endpoints de la API
</Card>

**Características:**
- Estructura de comandos basada en recursos para un uso intuitivo
- Múltiples formatos de salida (JSON, YAML, bonito, interactivo)
- Compleción de shell para bash, zsh y fish
- Perfecto para scripting y automatización CI/CD

```bash
# Quick example
dodopayments payments list --format json | jq '.data[] | {id, amount}'
```
