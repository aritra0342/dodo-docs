## Requisitos Previos

Para integrar la API de Dodo Payments, necesitarás:
- Una cuenta de comerciante de Dodo Payments
- Credenciales de API (clave de API y clave secreta de webhook) desde el panel de control

## Configuración del Panel de Control

1. Navega al [Panel de Control de Dodo Payments](https://app.dodopayments.com/)

2. Crea un producto (pago único o suscripción)

4. Genera tu clave de API:
   - Ve a Desarrollador > API
   - [Guía Detallada](/api-reference/introduction#api-key-generation)
   - Copia la clave de API en la variable de entorno llamada DODO_PAYMENTS_API_KEY

5. Configura los webhooks:
   - Ve a Desarrollador > Webhooks
   - Crea una URL de webhook para notificaciones de pago
   - Copia la clave secreta de webhook en la variable de entorno

## Integración

### Enlaces de Pago

Elige la ruta de integración que se ajuste a tu caso de uso:
- **Sesiones de Pago (recomendado)**: Mejor para la mayoría de las integraciones. Crea una sesión en tu servidor y redirige a los clientes a un pago seguro y alojado.
- **Pago en Superposición**: Utiliza cuando necesites una experiencia en la página que abra el pago como una superposición modal en tu sitio.
- **Pago en Línea**: Integra el pago directamente en el diseño de tu página para experiencias de pago completamente integradas y de marca.
- **Enlaces de Pago Estáticos**: URLs compartibles instantáneamente sin código para una rápida recolección de pagos.
- **Enlaces de Pago Dinámicos**: Enlaces creados programáticamente. Sin embargo, se recomiendan las Sesiones de Pago, ya que ofrecen más flexibilidad.

#### 1. Sesiones de Pago

Utiliza las Sesiones de Pago para crear una experiencia de pago segura y alojada para pagos únicos o suscripciones. Creas una sesión en tu servidor y luego rediriges al cliente a la `checkout_url`.

<Info>
Las sesiones de pago son válidas por 24 horas por defecto. Si pasas <code>confirm=true</code>, las sesiones son válidas por 15 minutos y se deben proporcionar todos los campos requeridos.
</Info>

<Steps>
<Step title="Crear una sesión de pago">
  Elige tu SDK preferido o llama a la API REST.

#### 2. Pago Superpuesto

Para una experiencia de pago en página sin interrupciones, explora nuestra integración de [Pago Superpuesto](/developer-resources/overlay-checkout) que permite a los clientes completar pagos sin salir de tu sitio web.

#### 3. Pago en Línea

Para experiencias de pago completamente integradas incrustadas directamente en tu página, utiliza nuestra integración de [Pago en Línea](/developer-resources/inline-checkout). Esto te permite construir resúmenes de pedidos personalizados y tener control total sobre el diseño del pago mientras Dodo Payments maneja de forma segura la recolección de pagos.

#### 4. Enlaces de Pago Estáticos

Los enlaces de pago estáticos te permiten aceptar pagos rápidamente compartiendo una URL simple. Puedes personalizar la experiencia de pago pasando parámetros de consulta para prellenar detalles del cliente, controlar campos de formulario y agregar metadatos personalizados.

<Steps>
<Step title="Construye tu enlace de pago">
  Comienza con la URL base y agrega tu ID de producto:

  ```text
  https://checkout.dodopayments.com/buy/{productid}
  ```

</Step>
<Step title="Agrega parámetros básicos">
  Incluye parámetros de consulta esenciales:
  - <ParamField query="quantity" type="integer" default="1">Número de artículos a comprar.</ParamField>
  - <ParamField query="redirect_url" type="string" required>URL a la que redirigir después de completar el pago.</ParamField>
  <Note>
  La URL de redirección incluirá detalles de pago como parámetros de consulta, por ejemplo:<br/>
  <code>https://example.com/?payment_id=pay_ts2ySpzg07phGeBZqePbH&status=succeeded</code>
  </Note>
</Step>
<Step title="Prellenar información del cliente (opcional)">
  Agrega campos de cliente o facturación como parámetros de consulta para agilizar el pago.
  <AccordionGroup>
    <Accordion title="Campos de Cliente Soportados">
      - <ParamField query="fullName" type="string">Nombre completo del cliente (ignorado si se proporciona firstName o lastName).</ParamField>
      - <ParamField query="firstName" type="string">Nombre del cliente.</ParamField>
      - <ParamField query="lastName" type="string">Apellido del cliente.</ParamField>
      - <ParamField query="email" type="string">Dirección de correo electrónico del cliente.</ParamField>
      - <ParamField query="country" type="string">País del cliente.</ParamField>
      - <ParamField query="addressLine" type="string">Dirección de la calle.</ParamField>
      - <ParamField query="city" type="string">Ciudad.</ParamField>
      - <ParamField query="state" type="string">Estado o provincia.</ParamField>
      - <ParamField query="zipCode" type="string">Código postal/ZIP.</ParamField>
      - <ParamField query="showDiscounts" type="boolean">true o false</ParamField>
    </Accordion>
  </AccordionGroup>
</Step>
<Step title="Controlar campos de formulario (opcional)">
  <Info>
  Puedes deshabilitar campos específicos para que sean de solo lectura para el cliente. Esto es útil cuando ya tienes los detalles del cliente (por ejemplo, usuarios que han iniciado sesión).
  </Info>
  Para deshabilitar un campo, proporciona su valor y establece el correspondiente <code>disable...</code> flag a <code>true</code>:
  <CodeGroup>

  ```text Example
  &email=alice@example.com&disableEmail=true
  ```

  </CodeGroup>
  <Tabs>
    <Tab title="Tabla de Banderas Desactivadas">
      | Campo         | Bandera Desactivada  | Parámetro Requerido |
      |---------------|---------------------|-------------------|
      | Nombre Completo | `disableFullName`   | `fullName`        |
      | Primer Nombre | `disableFirstName`  | `firstName`       |
      | Apellido      | `disableLastName`   | `lastName`        |
      | Correo Electrónico | `disableEmail`      | `email`           |
      | País         | `disableCountry`    | `country`         |
      | Línea de Dirección | `disableAddressLine`| `addressLine`     |
      | Ciudad        | `disableCity`       | `city`            |
      | Estado        | `disableState`      | `state`           |
      | Código Postal  | `disableZipCode`    | `zipCode`         |
    </Tab>
  </Tabs>
  <Tip>
  Desactivar campos ayuda a prevenir cambios accidentales y asegura la consistencia de los datos.
  </Tip>
  <Note>
  Establecer <code>showDiscounts=false</code> desactivará y ocultará la sección de descuentos en el formulario de pago. Usa esto si deseas evitar que los clientes ingresen códigos de cupón o promoción durante el pago.
  </Note>

</Step>
<Step title="Agregar controles avanzados (opcional)">
  - <ParamField query="paymentCurrency" type="string">Especifica la moneda de pago. Por defecto, es la moneda del país de facturación.</ParamField>
  - <ParamField query="showCurrencySelector" type="boolean" default="true">Mostrar u ocultar el selector de moneda.</ParamField>
  - <ParamField query="paymentAmount" type="integer">Cantidad en centavos (solo para precios de Paga lo que quieras).</ParamField>
  - <ParamField query="metadata_*" type="string">Campos de metadatos personalizados (por ejemplo, <code>metadata_orderId=123</code>).</ParamField>
</Step>
<Step title="Comparte el enlace">
  Envía el enlace de pago completado a tu cliente. Cuando lo visiten, todos los parámetros de consulta se recopilan y se almacenan con un ID de sesión. La URL se simplifica para incluir solo el parámetro de sesión (por ejemplo, <code>?session=sess_1a2b3c4d</code>). La información almacenada persiste a través de las actualizaciones de página y es accesible durante todo el proceso de pago.
  <Check>
  La experiencia de pago del cliente ahora está optimizada y personalizada según tus parámetros.
  </Check>
</Step>
</Steps>

#### 4. Enlaces de Pago Dinámicos
<Tip>
Prefiere las Sesiones de Pago para la mayoría de los casos de uso, ofrecen más flexibilidad y control.
</Tip>

Creado a través de una llamada a la API o nuestro SDK con detalles del cliente. Aquí hay un ejemplo:

Hay dos APIs para crear enlaces de pago dinámicos:

    });

    if (!response.ok) {
      const errorData = await response.json().catch(() => null);
      return NextResponse.json(
        { error: "Payment link creation failed", details: errorData },
        { status: response.status }
      );

    const data = await response.json();
    return NextResponse.json({ paymentLink: data.payment_link });
  } catch (err) {
    console.error("Payment error:", err);
    return NextResponse.json(
      {
        error: err instanceof Error ? err.message : "An unknown error occurred",
      },
      { status: 500 }
    );
  }
}

### Implementando Webhooks

Configura un endpoint de API para recibir notificaciones de pago. Aquí hay un ejemplo usando Next.js:

```javascript
import { Webhook } from "standardwebhooks";
import { headers } from "next/headers";
import { WebhookPayload } from "@/types/api-types";

const webhook = new Webhook(process.env.DODO_WEBHOOK_KEY!); // Replace with your secret key generated from the Dodo Payments Dashboard

export async function POST(request: Request) {
  const headersList = headers();
  const rawBody = await request.text();

  const webhookHeaders = {
    "webhook-id": headersList.get("webhook-id") || "",
    "webhook-signature": headersList.get("webhook-signature") || "",
    "webhook-timestamp": headersList.get("webhook-timestamp") || "",
  };

  await webhook.verify(rawBody, webhookHeaders);
  const payload = JSON.parse(rawBody) as WebhookPayload;
  
  // Process the payload according to your business logic
}
```

Nuestra implementación de webhook sigue la especificación de [Webhooks Estándar](https://standardwebhooks.com/). Para definiciones de tipos de webhook, consulta nuestra [Guía de Eventos de Webhook](/developer-resources/webhooks/intents/webhook-events-guide).

Puedes consultar este proyecto con implementación de demostración en [GitHub](https://github.com/dodopayments/dodo-checkout-demo) usando Next.js y TypeScript.
