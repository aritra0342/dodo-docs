## Instalación

<Steps>
<Step title="Install the package">
Ejecuta el siguiente comando en la raíz de tu proyecto:

```bash
bun add @dodopayments/bun
```

</Step>
<Step title="Set up environment variables">
Crea un archivo <code>.env</code> en la raíz de tu proyecto:

```env expandable
DODO_PAYMENTS_API_KEY=your-api-key
DODO_PAYMENTS_WEBHOOK_KEY=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT="test_mode" or "live_mode"
DODO_PAYMENTS_RETURN_URL=your-return-url
```

<Warning>
Nunca publiques tu archivo <code>.env</code> ni secretos en el control de versiones.
</Warning>
</Step>
</Steps>

## Ejemplos de controladores de rutas

<Info>
Todos los ejemplos asumen que usas el servidor nativo de Bun con <code>Bun.serve()</code>.
</Info>

<Tabs>
<Tab title="Checkout Handler">
  <Info>
  Utiliza este controlador para integrar el checkout de Dodo Payments en tu servidor Bun. Admite flujos de pago estáticos (GET), dinámicos (POST) y de sesión (POST).
  </Info>
  <CodeGroup>

    ```typescript Bun Server Handler expandable
    import { Checkout } from '@dodopayments/bun';

    const staticCheckoutHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "static"
    });

    const sessionCheckoutHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "session"
    });

    const dynamicCheckoutHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "dynamic"
    });

    Bun.serve({
      port: 3000,
      fetch(request) {
        const url = new URL(request.url);
        
        if (url.pathname === "/api/checkout") {
          if (request.method === "GET") {
            return staticCheckoutHandler(request);
          }
          if (request.method === "POST") {
            return sessionCheckoutHandler(request);
            // or return dynamicCheckoutHandler(request);
          }
        }
        
        return new Response("Not Found", { status: 404 });
      },
    });
    ```

  </CodeGroup>
  <CodeGroup>

## Controlador de ruta de checkout

<Info>
Dodo Payments admite tres tipos de flujos de pago para integrar pagos en tu sitio web; este adaptador admite todos los tipos de flujos de pago.
</Info>

- **Enlaces de pago estáticos:** URLs compartibles al instante para cobros rápidos sin código.
- **Enlaces de pago dinámicos:** Genera programáticamente enlaces de pago con detalles personalizados usando la API o los SDK.
- **Sesiones de checkout:** Crea experiencias de checkout seguras y personalizables con carritos de productos preconfigurados y datos del cliente.

Si falta <code>productId</code>, el controlador devuelve una respuesta 400. Los parámetros de consulta no válidos también generan una respuesta 400.
</Warning>

### Formato de respuesta
El checkout estático devuelve una respuesta JSON con la URL de checkout:

```json
{
  "checkout_url": "https://checkout.dodopayments.com/..."
}
```

</Accordion>
<Accordion title="Dynamic Checkout (POST)">
- Envía los parámetros como un cuerpo JSON en una solicitud POST.
- Admite pagos únicos y recurrentes.
- Para una lista completa de campos admitidos en el cuerpo POST, consulta:
  - [Request body for a One Time Payment Product](https://docs.dodopayments.com/api-reference/payments/post-payments)
  - [Request body for a Subscription Product](https://docs.dodopayments.com/api-reference/subscriptions/post-subscriptions)

### Formato de respuesta
El checkout dinámico devuelve una respuesta JSON con la URL de checkout:

```json
{
  "checkout_url": "https://checkout.dodopayments.com/..."
}
```

</Accordion>
<Accordion title="Checkout Sessions (POST)">
Las sesiones de checkout ofrecen una experiencia de checkout alojada más segura que maneja todo el flujo de pago para compras únicas y suscripciones con control total de personalización.

Consulta la [Guía de integración de sesiones de checkout](https://docs.dodopayments.com/developer-resources/checkout-session) para obtener más detalles y una lista completa de campos admitidos.

### Formato de respuesta
Las sesiones de checkout devuelven una respuesta JSON con la URL de checkout:

```json
{
  "checkout_url": "https://checkout.dodopayments.com/session/..."
}
```

</Accordion>
</AccordionGroup>

## Controlador de ruta para el portal de clientes

El Controlador de ruta del portal de clientes te permite integrar de manera fluida el portal de clientes de Dodo Payments en tu aplicación de servidor Bun.

### Parámetros de consulta
<ParamField query="customer_id" type="string" required>
El ID del cliente para la sesión del portal (por ejemplo, <code>?customer_id=cus_123</code>).
</ParamField>
<ParamField query="send_email" type="boolean">
Si se establece en <code>true</code>, envía un correo al cliente con el enlace del portal.
</ParamField>

<Warning>
Devuelve 400 si falta <code>customer_id</code>.
</Warning>

## Controlador de ruta de webhooks

- **Método:** Solo se admiten solicitudes POST. Otros métodos devuelven 405.
- **Verificación de firma:** Verifica la firma del webhook usando <code>webhookKey</code>. Devuelve 401 si la verificación falla.
- **Validación de la carga útil:** Validada con Zod. Devuelve 400 para cargas útiles no válidas.
- **Gestión de errores:**
  - 401: Firma inválida
  - 400: Carga útil inválida
  - 500: Error interno durante la verificación
- **Enrutamiento de eventos:** Llama al controlador de eventos adecuado según el tipo de carga útil.

import { Checkout, CustomerPortal, Webhooks } from "@dodopayments/bun";

const staticCheckoutHandler = Checkout({
  bearerToken: process.env.DODO_PAYMENTS_API_KEY,
  returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
  environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
  type: "static",
});

const sessionCheckoutHandler = Checkout({
  bearerToken: process.env.DODO_PAYMENTS_API_KEY,
  returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
  environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
  type: "session",
});

const customerPortalHandler = CustomerPortal({
  bearerToken: process.env.DODO_PAYMENTS_API_KEY,
  environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
});

const webhookHandler = Webhooks({
  webhookKey: process.env.DODO_PAYMENTS_WEBHOOK_KEY,
  onPaymentSucceeded: async (payload) => {
    console.log("Payment succeeded:", payload);
    // Your business logic here
  },
  onSubscriptionActive: async (payload) => {
    console.log("Subscription activated:", payload);
    // Your business logic here
  },
});

Bun.serve({
  port: 3000,
  fetch(request) {
    const url = new URL(request.url);
