# Descripción general

El <b>Adaptador Better Auth</b> para Dodo Payments proporciona:

- Creación automática de clientes al registrarse
- Sesiones de pago (preferido) con mapeo de slug de producto
- Portal de clientes de autoservicio
- Puntos finales de ingestión de uso medido e informes para facturación basada en el uso
- Procesamiento de eventos webhook en tiempo real con verificación de firma
- Soporte completo para TypeScript

<Note>
  Necesitas una cuenta de Dodo Payments y claves API para usar esta integración.
</Note>

# Requisitos previos

- Node.js 20+
- Acceso a tu panel de control de Dodo Payments
- Proyecto existente que use [better-auth](https://www.npmjs.com/package/better-auth)

# Instalación

<Steps>
<Step title="Install dependencies">
Ejecuta el siguiente comando en la raíz de tu proyecto:

```bash
npm install @dodopayments/better-auth dodopayments better-auth zod
```

</Check>
</Step>

# Configuración

<Steps>
<Step title="Configure environment variables">
Agrégalos a tu archivo <code>.env</code>:


```env expandable
DODO_PAYMENTS_API_KEY=your_api_key_here
DODO_PAYMENTS_WEBHOOK_SECRET=your_webhook_secret_here
BETTER_AUTH_URL=http://localhost:3000
BETTER_AUTH_SECRET=your_better_auth_secret_here
```

<Warning>
Nunca subas claves API ni secretos al control de versiones.
</Warning>
</Step>

<Step title="Set up server-side integration">
Crea o actualiza <code>src/lib/auth.ts</code>:


```typescript expandable
import { betterAuth } from "better-auth";
import {
  dodopayments,
  checkout,
  portal,
  webhooks,
  usage,
} from "@dodopayments/better-auth";
import DodoPayments from "dodopayments";

export const dodoPayments = new DodoPayments({
  bearerToken: process.env.DODO_PAYMENTS_API_KEY!,
  environment: "test_mode", // or "live_mode" for production
});

export const { auth, endpoints, client } = BetterAuth({
  plugins: [
    dodopayments({
      client: dodoPayments,
      createCustomerOnSignUp: true,
      use: [
        checkout({
          products: [
            {
              productId: "pdt_xxxxxxxxxxxxxxxxxxxxx",
              slug: "premium-plan",
            },
          ],
          successUrl: "/dashboard/success",
          authenticatedUsersOnly: true,
        }),
        portal(),
        webhooks({
          webhookKey: process.env.DODO_PAYMENTS_WEBHOOK_SECRET!,
          onPayload: async (payload) => {
            console.log("Received webhook:", payload.event_type);
          },
        }),
        usage(),
      ],
    }),
  ],
});
```

<Tip>
Configura <code>environment</code> en <code>live_mode</code> para producción.
</Tip>
</Step>

<Step title="Set up client-side integration">
Crea o actualiza <code>src/lib/auth-client.ts</code>:


```typescript expandable
import { createAuthClient } from "better-auth/react";
import { dodopaymentsClient } from "@dodopayments/better-auth";

export const authClient = createAuthClient({
  baseURL: process.env.BETTER_AUTH_URL || "http://localhost:3000",
  plugins: [dodopaymentsClient()],
});
```

</Step>
</Steps>

<CodeGroup>

```typescript Using a slug
const { data: session, error } = await authClient.dodopayments.checkoutSession({
  // Option A: use a configured slug
  slug: "premium-plan",

// Your internal id for reference
referenceId: "order_123",
});

if (session) {
window.location.href = session.url;
}

```

`

```typescript Using product cart
const { data: session, error } = await authClient.dodopayments.checkoutSession({
  // Option B: pass a product cart/product id directly
  product_cart: [
    {
      product_id: "pdt_xxxxxxxxxxxxxxxxxxxxx",
      quantity: 1,
    }
  ],

  // Your internal id for reference
  referenceId: "order_123",
});

if (session) {
  window.location.href = session.url;
}
```

`

</CodeGroup>

<Info>
  A diferencia del método heredado <code>checkout</code>, <code>checkoutSession</code>
  no requiere información de facturación por adelantado, ya que esta se toma del
  usuario en la página de pago. Aun así, puedes sobrescribirla pasando la clave
  <code>billing</code> en el argumento.
</Info>

<Info>
  Similar al método heredado <code>checkout</code>, <code>checkoutSession</code>
  obtiene el correo electrónico y nombre del cliente desde su sesión de
  better-auth; sin embargo, puedes sobrescribirlos pasando el objeto
  <code>customer</code> con correo y nombre.
</Info>

<Info>
  Puedes pasar las mismas opciones en el argumento que el cuerpo de la solicitud
  para [Create Checkout Session](https://docs.dodopayments.com/api-reference/checkout-sessions/create).
</Info>

<Note>
  La URL de retorno se toma de <code>successUrl</code> configurado en el plugin
  del servidor. No necesitas incluir <code>return_url</code> en la carga útil del
  cliente.
</Note>

## Seguimiento del Uso Medido

Habilita el complemento <code>usage()</code> en el servidor para capturar eventos medidos y permitir que los clientes inspeccionen su facturación basada en el uso.

- <code>authClient.dodopayments.usage.ingest</code> ingesta eventos para el usuario autenticado y verificado por correo electrónico.
- <code>authClient.dodopayments.usage.meters.list</code> lista el uso reciente para los medidores vinculados a las suscripciones de ese cliente.

```typescript expandable
// Record a metered event (e.g., an API request)
const { error: ingestError } = await authClient.dodopayments.usage.ingest({
  event_id: crypto.randomUUID(),
  event_name: "api_request",
  metadata: {
    route: "/reports",
    method: "GET",
  },
  // Optional Date; defaults to now if omitted
  timestamp: new Date(),
});

if (ingestError) {
  console.error("Failed to record usage", ingestError);
}

// List recent usage for the current customer
const { data: usage, error: usageError } =
  await authClient.dodopayments.usage.meters.list({
    query: {
      page_size: 20,
      meter_id: "mtr_yourMeterId", // optional
    },
  });

if (usage?.items) {
  usage.items.forEach((event) => {
    console.log(event.event_name, event.timestamp, event.metadata);
  });
}
```

<Tip>
  Las marcas de tiempo con más de una hora de antigüedad o más de cinco minutos
  en el futuro son rechazadas al registrar el uso.
</Tip>
<Tip>
  Si omites <code>meter_id</code> al listar medidores de uso, se devuelven todos
  los medidores vinculados a las suscripciones del cliente.
</Tip>

# Webhooks

<Info>
  El plugin de webhooks procesa eventos de pago en tiempo real de Dodo Payments
  con verificación segura de firmas. El endpoint predeterminado es
  <code>/api/auth/dodopayments/webhooks</code>.
</Info>

<Steps>
<Step title="Generate and set webhook secret">
Genera un secreto de webhook para la URL de tu endpoint (por ejemplo,
<code>https://&lt;your-domain&gt;/api/auth/dodopayments/webhooks</code>) en el Panel de
Dodo Payments y configúralo en tu archivo <code>.env</code>:


```env
DODO_PAYMENTS_WEBHOOK_SECRET=your_webhook_secret_here
```

</Step>

<Step title="Handle webhook events">
Ejemplo de manejador:


```typescript expandable
webhooks({
  webhookKey: process.env.DODO_PAYMENTS_WEBHOOK_SECRET!,
  onPayload: async (payload) => {
    console.log("Received webhook:", payload.event_type);
  },
});
```

</Step>
</Steps>

# Referencia de Configuración

<AccordionGroup>
  <Accordion title="Plugin Options">
    - <b>client</b> (obligatorio): instancia del cliente DodoPayments - <b>createCustomerOnSignUp</b> (opcional): crea clientes automáticamente al registrar usuarios - <b>use</b> (obligatorio): matriz de plugins para habilitar (checkout, portal, usage, webhooks)
  </Accordion>
  <Accordion title="Checkout Plugin Options">
    - <b>products</b>: matriz de productos o función asíncrona que devuelve productos - <b>successUrl</b>: URL para redirigir después de un pago exitoso - <b>authenticatedUsersOnly</b>: requiere autenticación de usuario (predeterminado: false)
  </Accordion>
</AccordionGroup>

# Solución de Problemas y Consejos

<AccordionGroup>
  <Accordion title="Common Issues">
    - <b>Invalid API key</b>: Verifica dos veces tu <code>DODO_PAYMENTS_API_KEY</code> en <code>.env</code>. - <b>Webhook signature mismatch</b>: Asegúrate de que el secreto de webhook coincida con el establecido en el Panel de Dodo Payments. - <b>Customer not created</b>: Confirma que <code>createCustomerOnSignUp</code> esté establecido en <code>true</code>.
  </Accordion>
  <Accordion title="Best Practices">
    - Usa variables de entorno para todos los secretos y claves. - Prueba en <code>test_mode</code> antes de cambiar a <code>live_mode</code>. - Registra eventos de webhook para depuración y auditoría.
  </Accordion>
</AccordionGroup>
