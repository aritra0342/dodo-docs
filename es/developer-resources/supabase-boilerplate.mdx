## Descripción general

Una plantilla lista para producción para suscripciones utilizando Next.js 15, React 19, Supabase, Drizzle ORM y Dodo Payments. Viene con Google OAuth, checkout de suscripción, manejo de webhooks, esquema de base de datos y un panel básico.

<Info>
Si solo necesitas controladores de rutas para una aplicación existente, consulta los adaptadores dedicados: <a href="/developer-resources/nextjs-adaptor">Next.js Adaptor</a> y <a href="/developer-resources/express-adaptor">Express Adaptor</a>.
</Info>

## Requisitos previos

- Node.js 18+ (o Bun 1.0+)
- Proyecto de Supabase (URL, clave Anon, clave de rol de servicio, URL de base de datos)
- Cuenta de Dodo Payments (clave API, secreto de webhook)
- Cliente de Google Cloud OAuth (ID de cliente y secreto)

## Inicio rápido

<Steps>
{/* LOCKED_PATTERN_587a3a4c926aec89f48dfc1e7c21054e */}

```bash
git clone https://github.com/dodopayments/dodo-supabase-subscription-starter.git
cd dodo-supabase-subscription-starter
# choose one
bun install
# or
npm install
# or
pnpm install
```

</Step>

{/* LOCKED_PATTERN_ea54c61e84e28cd275c0eed6d7634e0f */}
Crea un proyecto en Supabase y copia:
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
- SUPABASE_SERVICE_ROLE_KEY
- DATABASE_URL (Connection string)
</Step>

{/* LOCKED_PATTERN_42bf2f0f0010fd47b77c88325d34313c */}
Configura la URI de redirección en: `https://[your-project-ref].supabase.co/auth/v1/callback` en Google Cloud, luego habilita el proveedor de Google en Supabase Auth usando tu Client ID y Secret.
</Step>

{/* LOCKED_PATTERN_f32b65d8a917a558befa771bede82115 */}
Genera una clave API y un secreto de webhook desde el panel de Dodo. Establece el entorno en `test_mode` mientras desarrollas.
</Step>

{/* LOCKED_PATTERN_a34a8e162789e5a79d108d9eb3c86df5 */}

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role

# Database
DATABASE_URL=postgresql://postgres:[password]@db.[project-ref].supabase.co:5432/postgres

# Dodo Payments
DODO_PAYMENTS_API_KEY=your-dodo-api-key
DODO_WEBHOOK_SECRET=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT=test_mode
```

<Warning>
Nunca compartas secretos en el repositorio. Usa variables de entorno en los entornos de despliegue.
</Warning>
</Step>

{/* LOCKED_PATTERN_c35fe33eae6aa68b4bc39bd740632964 */}

```bash
bun run db:push
# or
npm run db:push
# or
pnpm run db:push
```

<Check>
Tablas creadas: `users`, `subscriptions`, `payments`.
</Check>
</Step>

{/* LOCKED_PATTERN_560d1d81fad22bd0622f0c8dcb3c706d */}

```bash
# login (one-time)
bunx supabase login
# or
npx supabase login

# deploy the edge function
bun run deploy:webhook --project-ref [your-project-ref]
# or
npm run deploy:webhook -- --project-ref [your-project-ref]
# or
pnpm run deploy:webhook --project-ref [your-project-ref]
```

```bash cURL
curl -X POST \
  'https://[your-project-ref].supabase.co/functions/v1/dodo-webhook' \
  -H 'Content-Type: application/json' \
  -H 'Dodo-Signature: <signature>' \
  -d '{"type":"payment.succeeded","data":{}}'
```

</Step>

{/* LOCKED_PATTERN_dd5db8f3734010c91f70fdf794c4e2a2 */}
Configura la URL del endpoint en:


```text
https://[your-project-ref].supabase.co/functions/v1/dodo-webhook
```

Selecciona eventos de pago y de suscripción.
</Step>

{/* LOCKED_PATTERN_43dfdbde6ac180daa9987aee2baf2da7 */}

```json

  "features": ["Feature 1", "Feature 2", "Feature 3"]

```

La interfaz de precios lee este `features` array y lo renderiza dinámicamente.
</Step>

{/* LOCKED_PATTERN_e437399605995e5a4845b4d749b71d01 */}

```bash

# or
npm run dev
# or
pnpm run dev
```

Abre http://localhost:3000.
</Step>
</Steps>

## Qué incluye

- Autenticación a través de Supabase (OAuth de Google configurado)
- Pago de suscripción mediante Dodo Payments
- Función Edge de Supabase para webhooks (`dodo-webhook`)
- Esquema y migraciones de Drizzle ORM
- Panel con facturas, estado de suscripción y características del plan

<Tip>
Mantén `DODO_PAYMENTS_ENVIRONMENT` como `test_mode` hasta que completes las pruebas de extremo a extremo.
</Tip>

## Verificación y solución de problemas

<AccordionGroup>
{/* LOCKED_PATTERN_6b286e9e2bbacd5c7684f8442203f10e */}
- Ensure `DODO_WEBHOOK_SECRET` matches the value from Dodo dashboard
- Confirm you deployed the latest `dodo-webhook` function
- Verify the header name is correct in your function (Dodo-Signature)
</Accordion>
{/* LOCKED_PATTERN_c270a2f8797d0168d608bb07bd40f056 */}
- Check `DATABASE_URL` syntax and Supabase network egress
- Wait ~2–3 minutes after project creation before first push
</Accordion>
{/* LOCKED_PATTERN_276d2f8137bb346c423586f69d6b7cae */}
- Redirect URI must be `https://[ref].supabase.co/auth/v1/callback`
- Ensure the same in Google Cloud and Supabase Auth provider
</Accordion>
</AccordionGroup>
