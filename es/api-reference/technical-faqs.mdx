<Accordion title="P1: ¿Por qué no recibí el id del producto en el webhook payment.succeeded para un producto de suscripción?">
  R: El evento del webhook payment.succeeded para productos de suscripción no incluye un product_id. En su lugar, proporciona el `subscription_id` asociado con la suscripción, junto con otros detalles relevantes.
</Accordion>

<Accordion title="P2: ¿Cómo puedo verificar que los webhooks que recibo son realmente de Dodo Payments y no de actores maliciosos?">
  R: Para verificar que los webhooks que recibes son genuinamente de Dodo Payments y no de actores maliciosos, sigue estos pasos:
    1. **Verifica la Firma**: Cada webhook enviado por Dodo Payments incluye una firma única en el encabezado **`webhook-signature`**. Esta firma se genera utilizando la clave secreta de tu webhook y la carga útil del evento.
    2. **Recupera la Clave Secreta**: Asegúrate de tener la clave secreta para tu webhook, que se puede encontrar en tu panel de Dodo Payments bajo la configuración de Webhooks.
    3. **Recalcula la Firma**: Utiliza el mismo algoritmo de firma (HMAC-SHA256) que Dodo Payments para calcular la firma en tu servidor. Usa la carga útil del webhook y tu clave secreta de webhook.
    4. **Compara Firmas**: Compara la firma recalculada con la que se proporciona en el encabezado **`webhook-signature`**. Si coinciden, el webhook es auténtico; si no, debe ser rechazado.
    5. **Usa HTTPS**: Asegúrate de que tu endpoint de webhook use HTTPS para asegurar la comunicación y prevenir manipulaciones durante la transmisión de datos.
  Al implementar estos pasos, puedes verificar con confianza la autenticidad de los webhooks enviados por Dodo Payments. 
</Accordion>

<Accordion title="P3: ¿Por qué estoy recibiendo un webhook subscription.renew incluso para el pago del primer mes?">
  R: Para las suscripciones, `subscription.renew` se activará cada vez que se deduzca el monto de la suscripción. Así que sí, este es un comportamiento esperado.
</Accordion>

<Accordion title="P5: ¿Cómo puedo probar webhooks localmente?">
  R: Para probar webhooks localmente, puedes seguir estos pasos:
  
1. **Usa una Herramienta de Túnel**: Herramientas como [ngrok](https://ngrok.com/), [LocalTunnel](https://localtunnel.github.io/www/), o [Cloudflare Tunnel](https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/) pueden exponer tu servidor local a Internet. Esto proporciona una URL pública que Dodo Payments puede usar para enviar eventos de webhook a tu entorno local.
2. **Configura el Endpoint de Webhook**: Crea un endpoint de webhook en tu aplicación local para manejar los eventos de webhook entrantes. Asegúrate de que escuche solicitudes HTTP POST.
3. **Configura el Webhook en Dodo Payments**: Ve a tu panel de Dodo Payments, navega a la configuración de Webhooks y actualiza la URL del webhook a la URL pública generada por la herramienta de túnel.
4. **Activa Eventos**: Realiza acciones en tu entorno para activar eventos de webhook. Por ejemplo, puedes simular pagos o renovaciones de suscripción.
5. **Inspecciona Solicitudes Entrantes**: Usa los registros de la herramienta de túnel o el registro de tu aplicación para inspeccionar las cargas útiles de los webhooks entrantes. Esto te permite verificar que tu endpoint está recibiendo los eventos y procesándolos correctamente.
6. **Verifica la Firma**: Asegúrate de validar el encabezado **`webhook-signature`** para confirmar que los eventos de webhook son auténticos y enviados por Dodo Payments.

Siguiendo estos pasos, puedes probar y depurar fácilmente los webhooks en tu entorno de desarrollo local.
</Accordion>
