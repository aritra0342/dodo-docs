## Gestión de Claves API y Autenticación

<Steps>
<Step title="Acceder a las Claves API">
Navega a **Desarrollador → Claves API** en tu panel para gestionar tus claves.
</Step>

<Step title="Generar una Nueva Clave">
Selecciona **Agregar Clave API**, proporciona un nombre descriptivo y configura el acceso de escritura:

- **Habilitar acceso de escritura** (marcado): Permisos de lectura y escritura completos para todas las operaciones de la API
- **Habilitar acceso de escritura** (desmarcado): Acceso solo de lectura - solo se puede obtener datos (pagos, suscripciones, clientes, productos). No se pueden crear ni modificar recursos.

<Tip>
Desmarca "Habilitar acceso de escritura" para integraciones de panel, herramientas de análisis y cualquier sistema que solo necesite ver datos sin realizar cambios.
</Tip>
</Step>

<Step title="Almacena Tu Clave de Forma Segura">
Copia la clave generada de inmediato y asegúrate de que se almacene de forma segura.
</Step>

<Step title="Autentica Tus Solicitudes API">
Utiliza tus claves API para autenticar todas las solicitudes. Aplica el siguiente formato de autorización:

```bash 
Authorization: Bearer YOUR_API_KEY
```

<Warning>
Nunca expongas tus claves API secretas en código del lado del cliente o en repositorios públicos.
</Warning>
</Step>
</Steps>

## Formato de Respuesta

<CodeGroup>

```json Success Response
{
  "id": "pay_1234567890",
  "status": "completed",
  "amount": 2999,
  "currency": "USD",
  "created_at": "2024-01-15T10:30:00Z"
}
```

```json Error Response
{
  "code": "INVALID_REQUEST",
  "message": "The request contains invalid parameters"
}
```

</CodeGroup>

## Límites de Tasa

- **Estándar**: 1000 solicitudes por minuto por negocio
- **Ráfaga**: Hasta 100 solicitudes por segundo

<Note>
Monitorea `X-RateLimit-Remaining` encabezado para rastrear el uso.
</Note>
