Actualiza el método de pago de una suscripción. Este endpoint admite tanto suscripciones activas como suscripciones en estado `on_hold`.

<Info>
Para suscripciones en estado `on_hold`, actualizar el método de pago crea automáticamente un cargo por los adeudos restantes, genera una factura y reactiva la suscripción al estado `active` tras un pago exitoso.
</Info>

### Casos de Uso

- **Suscripciones activas**: Actualiza el método de pago cuando una tarjeta expira o el cliente quiere usar un método de pago diferente
- **Suscripciones en espera**: Reactiva suscripciones que se pusieron en espera debido a pagos fallidos actualizando el método de pago
- **Gestión de métodos de pago**: Cambia entre métodos de pago guardados o agrega nuevos

<Info>
Para listar los métodos de pago existentes de un cliente, utiliza la [List Payment Methods API](/api-reference/customers/get-customer-payment-methods). Esto te ayuda a obtener los IDs de métodos de pago disponibles cuando usas `type: "existing"` para actualizar el método de pago de una suscripción.
</Info>

### Comportamiento para Suscripciones Activas

Al actualizar el método de pago para una suscripción activa:

- El método de pago se actualiza inmediatamente
- No se crea ningún cargo
- La suscripción permanece activa
- Las renovaciones futuras utilizarán el nuevo método de pago

### Comportamiento para Suscripciones en Espera

Al actualizar el método de pago de una suscripción en estado `on_hold`:

1. Se crea automáticamente un cargo por los adeudos restantes
2. Se genera una factura por el cargo
3. El pago se procesa con el nuevo método de pago
4. Tras un pago exitoso, la suscripción se reactiva al estado `active`
5. Recibirás eventos webhook: `payment.succeeded` seguido de `subscription.active`

<Warning>
Si el pago falla después de actualizar el método de pago de una suscripción en estado `on_hold`, la suscripción permanecerá en ese estado. Supervisa los eventos webhook para seguir el estado del pago.
</Warning>

### Eventos de Webhook

Al actualizar un método de pago para una suscripción en estado `on_hold`, recibirás los siguientes eventos webhook:

1. **`payment.succeeded`** - El cargo por los adeudos restantes fue exitoso
2. **`subscription.active`** - La suscripción ha sido reactivada
