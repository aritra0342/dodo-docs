## Manejo de fallas de pago

Utiliza el parámetro `on_payment_failure` para controlar lo que ocurre cuando el pago del cambio de plan falla:

| Valor | Comportamiento |
|-------|----------------|
| `prevent_change` | Mantener la suscripción en el plan actual hasta que el pago tenga éxito. El cambio de plan permanece pendiente. |
| `apply_change` | Aplicar el cambio de plan de inmediato independientemente del resultado del pago. Esta es la opción predeterminada. |

Si `on_payment_failure` no se especifica, el comportamiento predeterminado es la configuración a nivel de negocio que hayas configurado en el panel.
