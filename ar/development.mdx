<Info>
  **المتطلبات الأساسية**: يرجى تثبيت Node.js (الإصدار 19 أو أعلى) قبل المتابعة.
</Info>

اتبع هذه الخطوات لتثبيت وتشغيل Mintlify على نظام التشغيل الخاص بك:

**الخطوة 1**: تثبيت Mintlify:

<CodeGroup>

  ```bash npm
  npm i -g mintlify
  ```

```bash yarn
yarn global add mintlify
```

</CodeGroup>

**الخطوة 2**: انتقل إلى دليل الوثائق (حيث يوجد ملف `mint.json`) ونفذ الأمر التالي:

```bash
mintlify dev
```

ستكون معاينة محلية للوثائق الخاصة بك متاحة على `http://localhost:3000`.

### المنافذ المخصصة

بشكل افتراضي، يستخدم Mintlify المنفذ 3000. يمكنك تخصيص المنفذ الذي يعمل عليه Mintlify باستخدام العلامة `--port`. لتشغيل Mintlify على المنفذ 3333، على سبيل المثال، استخدم هذا الأمر:

```bash
mintlify dev --port 3333
```

إذا حاولت تشغيل Mintlify على منفذ قيد الاستخدام بالفعل، فسوف يستخدم المنفذ المتاح التالي:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## إصدارات Mintlify

يرجى ملاحظة أن كل إصدار من CLI مرتبط بإصدار محدد من Mintlify. إذا لم يتطابق موقعك المحلي مع الإصدار الإنتاجي، يرجى تحديث CLI:

<CodeGroup>

```bash npm
npm i -g mintlify@latest
```

```bash yarn
yarn global upgrade mintlify
```

</CodeGroup>

## التحقق من الروابط

يمكن أن يساعدك CLI في التحقق من الروابط المرجعية الموجودة في الوثائق الخاصة بك. لتحديد أي روابط معطلة، استخدم الأمر التالي:

```bash
mintlify broken-links
```

## استكشاف الأخطاء وإصلاحها

<AccordionGroup>
  <Accordion title='خطأ: لم يتم تحميل وحدة "sharp" باستخدام وقت تشغيل darwin-arm64'>

    قد يكون ذلك بسبب إصدار قديم من node. جرب ما يلي:
    1. قم بإزالة الإصدار المثبت حاليًا من mintlify: `npm remove -g mintlify`
    2. قم بالترقية إلى Node v19 أو أعلى.
    3. أعد تثبيت mintlify: `npm install -g mintlify`
  </Accordion>

  <Accordion title="مشكلة: مواجهة خطأ غير معروف">
  
    الحل: انتقل إلى جذر جهازك واحذف مجلد \~/.mintlify. بعد ذلك، قم بتشغيل `mintlify dev` مرة أخرى.
  </Accordion>
</AccordionGroup>
