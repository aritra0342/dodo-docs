## نظرة عامة

تدمج إضافة Dodo Payments Framer معالجة المدفوعات بسلاسة في مواقع Framer الخاصة بك. أنشئ تجارب دفع قابلة للتخصيص مع دعم لكل من بيئات الاختبار والإنتاج.

</Card>

## دليل الإعداد

<Steps>
<Step title="إنشاء مفتاح API">
قم بزيارة [لوحة تحكم Dodo Payments](https://app.dodopayments.com/developer/api-keys) وأنشئ مفتاح API جديد.

<Tabs>

<Tab title="وضع الإنتاج">
للاستخدام في الإنتاج مع معالجة مدفوعات حقيقية.
</Tab>

<Tab title="وضع الاختبار">
مثالي للتطوير والاختبار دون أي رسوم حقيقية.
</Tab>

</Tabs>

<Check>
قم بنسخ مفتاح API الخاص بك بأمان - ستحتاج إليه للخطوة التالية.
</Check>
</Step>

<Step title="المصادقة على الإضافة">
1. افتح إضافة Dodo Payments في Framer
2. انتقل إلى صفحة المصادقة
3. ألصق مفتاح API الخاص بك
4. اختر الوضع المناسب (اختبار أو إنتاج)
5. انقر على **متابعة** للمصادقة

<Warning>
احتفظ بمفاتيح API الخاصة بك بأمان ولا تشاركها علنًا.
</Warning>
</Step>

<Step title="إدارة المنتجات">
بعد المصادقة، سترى قائمة منتجاتك:

- **المنتجات الموجودة**: تُعرض تلقائيًا
- **لا توجد منتجات**: انقر على زر **زائد (+)** بجوار 'المنتجات'

<Info>
سيتم توجيهك إلى لوحة تحكم Dodo Payments لإنشاء منتجات مع التسعير، والأوصاف، والإعدادات.
</Info>
</Step>

<Step title="إضافة أزرار الشراء">
1. اختر منتجًا من قائمتك المتاحة
2. انقر على زر `Add Buy Button`
3. سيتم إضافة زر شراء مع النص 'شراء \{product_name\}' إلى لوحتك

<Warning>
سيتم تعطيل زر إضافة شراء إذا لم يكن لديك الأذونات الصحيحة للمشروع.
</Warning>
</Step>
</Steps>

## التخصيص

<Steps>
<Step title="اختيار تجربة الدفع">
    اختر تجربة الدفع المفضلة لديك:
    - **نافذة منبثقة (مضمنة)**: تظهر عملية الدفع كنافذة منبثقة داخل مشروع Framer الخاص بك، مما يبقي المستخدمين على موقعك.
    - **مستضاف**: يتم إعادة توجيه المستخدمين إلى صفحة دفع مستضافة من Dodo Payments لتجربة سلسة.
</Step>

<Step title="تخصيص خيارات المظهر">
    خصص تجربة الدفع الخاصة بك مع إعدادات المظهر التالية:
    - **سمة الدفع**: اختر بين السمات الفاتحة أو الداكنة لتتناسب مع جمالية علامتك التجارية.
    - **نص الزر**: خصص النص (مثل 'اشترِ الآن'، 'شراء'، 'ابدأ').
    - **أنماط الأزرار**: قم بتعديل الألوان، والخطوط، والأحجام، وخصائص التنسيق الأخرى.
    - **رابط العودة**: قم بتعيين صفحة Framer ليعود إليها المستخدم بعد الدفع.
</Step>
</Steps>

## استكشاف الأخطاء وإصلاحها

<AccordionGroup>
<Accordion title="مشاكل المصادقة">
**المشكلة**: الإضافة لا تصدق

**الحلول**:
- تحقق من صحة مفتاح API الخاص بك
- تأكد من أنك تستخدم الوضع الصحيح (اختبار/إنتاج)
- تحقق من اتصال الإنترنت الخاص بك
</Accordion>

<Accordion title="مشاكل إنشاء الأزرار">
**المشكلة**: زر إضافة شراء معطل

**الحلول**:
- تحقق من أن لديك الأذونات الصحيحة في مشروع Framer الخاص بك
- تأكد من أنك مصدق بشكل صحيح
- حاول تحديث الإضافة
</Accordion>

<Accordion title="مشاكل عرض المنتجات">
**المشكلة**: المنتجات لا تظهر

**الحلول**:
- تأكد من أنك قد أنشأت منتجات في لوحة تحكم Dodo Payments الخاصة بك
- تحقق من أن مفتاح API الخاص بك لديه الأذونات الصحيحة
- حاول إعادة المصادقة
</Accordion>
</AccordionGroup>
