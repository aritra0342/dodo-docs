# نظرة عامة

يوفر <b>محول Better Auth</b> لمدفوعات دودي:

- إنشاء العملاء تلقائيًا عند التسجيل
- جلسات الدفع (المفضلة) مع خريطة منتج
- بوابة العملاء ذات الخدمة الذاتية
- نقاط إدخال واستخدام تقارير الاستخدام للفوترة المعتمدة على الاستخدام
- معالجة أحداث الويب في الوقت الفعلي مع التحقق من التوقيع
- دعم كامل لـ TypeScript

<Note>
  تحتاج إلى حساب مدفوعات دودي ومفاتيح API لاستخدام هذا التكامل.
</Note>

# المتطلبات المسبقة

- Node.js 20+
- الوصول إلى لوحة معلومات مدفوعات دودي الخاصة بك
- مشروع موجود يستخدم [better-auth](https://www.npmjs.com/package/better-auth)

```env expandable
BETTER_AUTH_URL=http://localhost:3000

```

<Warning>

</Warning>
</Step>

<Step title="إعداد التكامل من جانب الخادم">
قم بإنشاء أو تحديث <code>src/lib/auth.ts</code>:

```typescript expandable
import { betterAuth } from "better-auth";
import {
  dodopayments,
  checkout,
  portal,
  webhooks,
  usage,
} from "@dodopayments/better-auth";
import DodoPayments from "dodopayments";

export const dodoPayments = new DodoPayments({
  bearerToken: process.env.DODO_PAYMENTS_API_KEY!,
  environment: "test_mode", // or "live_mode" for production
});

export const { auth, endpoints, client } = BetterAuth({
  plugins: [
    dodopayments({
      client: dodoPayments,
      createCustomerOnSignUp: true,
      use: [
        checkout({
          products: [
            {
              productId: "pdt_xxxxxxxxxxxxxxxxxxxxx",
              slug: "premium-plan",
            },
          ],
          successUrl: "/dashboard/success",
          authenticatedUsersOnly: true,
        }),
        portal(),
        webhooks({
          webhookKey: process.env.DODO_PAYMENTS_WEBHOOK_SECRET!,
          onPayload: async (payload) => {
            console.log("Received webhook:", payload.event_type);
          },
        }),
        usage(),
      ],
    }),
  ],
});
```

<Tip>
قم بتعيين <code>environment</code> إلى <code>live_mode</code> للإنتاج.
</Tip>
</Step>

<Step title="إعداد التكامل من جانب العميل">
قم بإنشاء أو تحديث <code>src/lib/auth-client.ts</code>:

```typescript expandable
import { createAuthClient } from "better-auth/react";
import { dodopaymentsClient } from "@dodopayments/better-auth";

export const authClient = createAuthClient({
  baseURL: process.env.BETTER_AUTH_URL || "http://localhost:3000",
  plugins: [dodopaymentsClient()],

```

</Step>
</Steps>

# أمثلة الاستخدام

<Info>
  يفضل استخدام <code>authClient.dodopayments.checkoutSession</code> للتكاملات الجديدة. تم إهمال طريقة <code>checkout</code> القديمة وتبقي فقط للتوافق مع الإصدارات السابقة.
</Info>

## إنشاء جلسة دفع (المفضلة)

<CodeGroup>

```typescript Using a slug
const { data: session, error } = await authClient.dodopayments.checkoutSession({
  // Option A: use a configured slug
  slug: "premium-plan",

// Your internal id for reference
referenceId: "order_123",
});

if (session) {
window.location.href = session.url;
}

```

`

```typescript Using product cart
const { data: session, error } = await authClient.dodopayments.checkoutSession({
  // Option B: pass a product cart/product id directly
  product_cart: [
    {
      product_id: "pdt_xxxxxxxxxxxxxxxxxxxxx",
      quantity: 1,
    }
  ],

  // Your internal id for reference
  referenceId: "order_123",
});

if (session) {
  window.location.href = session.url;
}
```

`

</CodeGroup>

<Info>
  على عكس طريقة <code>checkout</code> القديمة، لا تتطلب <code>checkoutSession</code> معلومات الفوترة مسبقًا حيث سيتم أخذها من المستخدم في صفحة الدفع. يمكنك تجاوز ذلك عن طريق تمرير مفتاح <code>billing</code> في الوسيط.
</Info>

<Info>
  مشابهة لطريقة <code>checkout</code> القديمة، تأخذ <code>checkoutSession</code> بريد العميل الإلكتروني واسمهم من جلسة better-auth الخاصة بهم، ومع ذلك، يمكنك تجاوز ذلك عن طريق تمرير كائن <code>customer</code> مع البريد الإلكتروني والاسم.
</Info>

<Info>
  يمكنك تمرير نفس الخيارات في الوسيط كجسم الطلب لنقطة النهاية [إنشاء جلسة دفع](https://docs.dodopayments.com/api-reference/checkout-sessions/create).
</Info>

<Note>
  يتم أخذ عنوان URL للعودة من <code>successUrl</code> المكون في ملحق الخادم. لا تحتاج إلى تضمين <code>return_url</code> في الحمولة الخاصة بالعميل.
</Note>

## تتبع الاستخدام المعتمد على القياس

قم بتمكين ملحق <code>usage()</code> على الخادم لالتقاط الأحداث المعتمدة على القياس ودع العملاء يفحصون فواتيرهم المعتمدة على الاستخدام.

- <code>authClient.dodopayments.usage.ingest</code> يستوعب الأحداث للمستخدم المسجل، الذي تم التحقق من بريده الإلكتروني.
- <code>authClient.dodopayments.usage.meters.list</code> يسرد الاستخدام الأخير للعدادات المرتبطة باشتراكات ذلك العميل.

```typescript expandable
// Record a metered event (e.g., an API request)
const { error: ingestError } = await authClient.dodopayments.usage.ingest({
  event_id: crypto.randomUUID(),
  event_name: "api_request",
  metadata: {
    route: "/reports",
    method: "GET",
  },
  // Optional Date; defaults to now if omitted
  timestamp: new Date(),
});

if (ingestError) {
  console.error("Failed to record usage", ingestError);
}

// List recent usage for the current customer
const { data: usage, error: usageError } =
  await authClient.dodopayments.usage.meters.list({
    query: {
      page_size: 20,
      meter_id: "mtr_yourMeterId", // optional
    },
  });

if (usage?.items) {
  usage.items.forEach((event) => {
    console.log(event.event_name, event.timestamp, event.metadata);
  });
}
```

<Tip>
  يتم رفض الطوابع الزمنية التي تزيد عن ساعة واحدة أو أكثر من خمس دقائق في المستقبل عند استيعاب الاستخدام.
</Tip>
<Tip>
  إذا قمت بإغفال <code>meter_id</code> عند سرد عدادات الاستخدام، يتم إرجاع جميع العدادات المرتبطة باشتراكات العميل.
</Tip>

# أحداث الويب

<Info>
  يقوم ملحق أحداث الويب بمعالجة أحداث الدفع في الوقت الفعلي من مدفوعات دودي مع التحقق الآمن من التوقيع. نقطة النهاية الافتراضية هي <code>/api/auth/dodopayments/webhooks</code>.
</Info>

<Steps>
<Step title="توليد وتعيين سر أحداث الويب">
قم بتوليد سر أحداث الويب لعنوان URL الخاص بنقطة النهاية الخاصة بك (على سبيل المثال، <code>https://&lt;your-domain&gt;/api/auth/dodopayments/webhooks</code>) في لوحة معلومات مدفوعات دودي وقم بتعيينه في ملف <code>.env</code> الخاص بك:

```env
DODO_PAYMENTS_WEBHOOK_SECRET=your_webhook_secret_here
```

</Step>

<Step title="معالجة أحداث الويب">
مثال على المعالج:

```typescript expandable
webhooks({
  webhookKey: process.env.DODO_PAYMENTS_WEBHOOK_SECRET!,
  onPayload: async (payload) => {
    console.log("Received webhook:", payload.event_type);
  },
});
```

</Step>
</Steps>
