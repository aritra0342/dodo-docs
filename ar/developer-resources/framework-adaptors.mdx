تقدم مدفوعات دودي موصلات رسمية للإطار تسهل دمج المدفوعات. تم تصميم كل موصل للعمل بسلاسة مع تقاليد إطارك، مما يوفر معالجة الدفع، وبوابة العملاء، والتعامل مع الويب هوك بشكل مباشر.

<Tip>
تتيح لك موصلات الإطار دمج مدفوعات دودي في أقل من 10 أسطر من التعليمات البرمجية. إنها تتعامل مع المصادقة، وتحليل الطلبات، وتنسيق الاستجابات تلقائيًا.
</Tip>

## الميزات الأساسية

توفر جميع موصلات الإطار هذه القدرات المدمجة:

| الميزة | الوصف |
|---------|-------------|
| **معالج الدفع** | دعم لتدفقات الدفع الثابتة، والديناميكية، والمعتمدة على الجلسة |
| **بوابة العملاء** | معالج مسبق البناء لإدارة الاشتراكات والفواتير |
| **معالج الويب هوك** | تحقق آمن من التوقيع مع معالجات الأحداث المخصصة |
| **تكوين البيئة** | إعداد بسيط عبر متغيرات البيئة |
| **أمان النوع** | دعم كامل لـ TypeScript مع حمولات مخصصة |

## البدء السريع

ابدأ مع أي موصل إطار في ثلاث خطوات:

<Steps>
<Step title="تثبيت الموصل">
  استخدم مدير الحزم الخاص بك لتثبيت الموصل المحدد للإطار:

  <Tabs>
  <Tab title="Next.js">

  ```bash
  npm install @dodopayments/nextjs
  ```

  </Tab>
  
  <Tab title="Nuxt">

  ```bash
  npm install @dodopayments/nuxt
  ```

  </Tab>
  
  <Tab title="Express">

  ```bash
  npm install @dodopayments/express
  ```

  </Tab>
  
  <Tab title="Hono">

  ```bash
  npm install @dodopayments/hono
  ```

  </Tab>

  <Tab title="Astro">

  ```bash
  npm install @dodopayments/astro
  ```

  </Tab>

  <Tab title="SvelteKit">

  ```bash
  npm install @dodopayments/sveltekit
  ```

  </Tab>
  </Tabs>
</Step>

<Step title="تكوين متغيرات البيئة">
  أضف بيانات اعتماد مدفوعات دودي إلى بيئتك:

  ```env
  DODO_PAYMENTS_API_KEY=your-api-key
  DODO_PAYMENTS_WEBHOOK_KEY=your-webhook-secret
  DODO_PAYMENTS_RETURN_URL=https://yourdomain.com/checkout/success
  DODO_PAYMENTS_ENVIRONMENT="test_mode" # or "live_mode"
  ```

  <Warning>
  لا تقم أبداً بالتزام ملفك `.env` أو الأسرار في التحكم في الإصدار.
  </Warning>
</Step>

<Step title="إنشاء معالجات المسار">
  قم بإعداد مسارات الدفع، وبوابة العملاء، والويب هوك:

  <Tabs>
  <Tab title="Next.js">

  ```typescript
  // app/checkout/route.ts
  import { Checkout } from "@dodopayments/nextjs";

  export const GET = Checkout({
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
    environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
  });
  ```

  </Tab>
  
  <Tab title="Express">

  ```typescript
  import { checkoutHandler } from '@dodopayments/express';

  app.get('/api/checkout', checkoutHandler({
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
    environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
  }));
  ```

  </Tab>
  
  <Tab title="Hono">

  ```typescript
  import { Checkout } from "@dodopayments/hono";

  app.get('/checkout', Checkout({
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
    environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
  }));
  ```

  </Tab>
  </Tabs>

  <Check>
  أنت الآن جاهز لمعالجة المدفوعات! قم بزيارة صفحات الموصل الفردية للحصول على أدلة مفصلة وجميع الخيارات المتاحة.
  </Check>
</Step>
</Steps>

## أنواع تدفقات الدفع

تدعم جميع الموصلات ثلاثة أنواع من تدفقات الدفع:

<AccordionGroup>
<Accordion title="الدفع الثابت (GET)">
استخدم الدفع الثابت لروابط الدفع البسيطة والقابلة للمشاركة. مرر معرف المنتج كمعامل استعلام:

```
/api/checkout?productId=pdt_xxx&quantity=1
```

يدعم تعبئة العملاء الاختيارية والتخصيص عبر معاملات الاستعلام.
</Accordion>

<Accordion title="الدفع الديناميكي (POST)">
استخدم الدفع الديناميكي لإنشاء المدفوعات برمجيًا مع تفاصيل مخصصة:

```json
{
  "product_id": "pdt_xxx",
  "customer": {
    "email": "customer@example.com",
    "name": "John Doe"
  },
  "quantity": 1
}
```

يدعم كل من المدفوعات لمرة واحدة والاشتراكات.
</Accordion>

<Accordion title="جلسات الدفع (POST)">
استخدم جلسات الدفع للحصول على تجربة دفع أكثر مرونة مع دعم السلة:

```json
{
  "product_cart": [
    { "product_id": "pdt_xxx", "quantity": 1 },
    { "product_id": "pdt_yyy", "quantity": 2 }
  ],
  "customer": {
    "email": "customer@example.com"
  }
}
```

تعرف على المزيد في [دليل جلسات الدفع](/developer-resources/checkout-session).
</Accordion>
</AccordionGroup>

## معالجة أحداث الويب هوك

توفر جميع الموصلات معالجة ويب هوك آمنة من النوع مع ردود أحداث دقيقة:

```typescript
Webhooks({
  webhookKey: process.env.DODO_PAYMENTS_WEBHOOK_KEY,
  onPayload: async (payload) => {
    // Handle any webhook event
  },
  onPaymentSucceeded: async (payload) => {
    // Handle successful payments
  },
  onSubscriptionActive: async (payload) => {
    // Handle new subscriptions
  },
  // ... 20+ event types supported
});
```

<Info>
تتحقق جميع معالجات الويب هوك تلقائيًا من التوقيعات وتتحقق من صحة الحمولات باستخدام مخططات Zod. يتم رفض الطلبات غير الصالحة مع رموز خطأ مناسبة.
</Info>

## اختيار الموصل المناسب

| الإطار | الأفضل لـ | وقت التشغيل |
|-----------|----------|---------|
| **Next.js** | تطبيقات React كاملة مع موجه التطبيق | Node.js، Edge |
| **Nuxt** | تطبيقات Vue.js كاملة | Node.js |
| **Express** | واجهات برمجة التطبيقات REST وتطبيقات Node.js التقليدية | Node.js |
| **Fastify** | واجهات برمجة التطبيقات عالية الأداء | Node.js |
| **Hono** | نشرات الحافة، عمال Cloudflare | Edge، Node.js |
| **Astro** | مواقع المحتوى مع نقاط نهاية الخادم | Node.js، Edge |
| **SvelteKit** | تطبيقات Svelte كاملة | Node.js |
| **Remix** | React كامل مع توجيه متداخل | Node.js |
| **TanStack Start** | React كامل آمن من النوع | Node.js |
| **Better Auth** | التطبيقات التي تستخدم بالفعل Better Auth | متنوعة |
| **Convex** | التطبيقات التي تستخدم Convex للخلفية | وقت تشغيل Convex |
