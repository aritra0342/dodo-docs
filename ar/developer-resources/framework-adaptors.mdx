تقدم مدفوعات دودي موصلات رسمية للإطار تسهل دمج المدفوعات. تم تصميم كل موصل للعمل بسلاسة مع تقاليد إطارك، مما يوفر معالجة الدفع، وبوابة العملاء، والتعامل مع الويب هوك بشكل مباشر.

<Tip>
تتيح لك الموصلات الخاصة بالأطر دمج Dodo Payments بأقل من 10 أسطر من التعليمات البرمجية. تتولى المصادقة وتحليل الطلبات وتنسيق الاستجابات تلقائيًا.
</Tip>

## الميزات الأساسية

توفر جميع موصلات الأطر هذه القدرات المدمجة:

| الميزة | الوصف |
|---------|-------------|
| **معالج الدفع** | دعم لتدفقات الدفع الثابتة والديناميكية والتي تستند إلى الجلسات |
| **بوابة العملاء** | معالج مبني مسبقًا لإدارة الاشتراكات والفوترة |
| **معالج الويب هوكس** | التحقق من التوقيع الآمن مع معالجات أحداث مطبوعة |
| **إعداد البيئة** | إعداد بسيط عبر متغيرات البيئة |
| **أمان الأنواع** | دعم كامل لـ TypeScript مع حمولات بيانات مطبوعة |

  ```env
  DODO_PAYMENTS_API_KEY=your-api-key
  DODO_PAYMENTS_WEBHOOK_KEY=your-webhook-secret
  ```

## أنواع تدفقات الدفع

جميع الموصلات تدعم ثلاثة أنواع من تدفق الدفع:

<AccordionGroup>
{/* LOCKED_PATTERN_54ec7a6c810897db7154e4dbddb9180e */}
استخدم الدفع الثابت للروابط البسيطة والمشاركة. مرر معرف المنتج كمعامل استعلام:

```
/api/checkout?productId=pdt_xxx&quantity=1
```

يدعم ملء بيانات العميل المسبق والتخصيص اختياريًا عبر معاملات الاستعلام.
</Accordion>

{/* LOCKED_PATTERN_3a705ba64e435adc71acb6fb44c2370a */}
استخدم الدفع الديناميكي لإنشاء المدفوعات برمجيًا بتفاصيل مخصصة:

```json
{
  "product_id": "pdt_xxx",
  "customer": {
    "email": "customer@example.com",
    "name": "John Doe"
  },
  "quantity": 1
}
```

يدعم كل من المدفوعات مرة واحدة والاشتراكات.
</Accordion>

{/* LOCKED_PATTERN_c9e7172c02eea079bd96ccf0ae70f411 */}
استخدم جلسات الدفع لأكثر تجارب الدفع مرونة مع دعم السلة:

```json
{
  "product_cart": [
    { "product_id": "pdt_xxx", "quantity": 1 },
    { "product_id": "pdt_yyy", "quantity": 2 }
  ],
  "customer": {
    "email": "customer@example.com"
  }
}
```

تعرف على المزيد في [دليل جلسات الدفع](/developer-resources/checkout-session).
</Accordion>
</AccordionGroup>

## معالجة أحداث الويب هوكس

جميع الموصلات توفر معالجة ويب هوكس آمنة بأنواع مع ردود نداء للأحداث التفصيلية:

```typescript
Webhooks({
  webhookKey: process.env.DODO_PAYMENTS_WEBHOOK_KEY,
  onPayload: async (payload) => {
    // Handle any webhook event
  },
  onPaymentSucceeded: async (payload) => {
    // Handle successful payments
  },
  onSubscriptionActive: async (payload) => {
    // Handle new subscriptions
  },
  // ... 20+ event types supported
});
```

<Info>
جميع معالجات الويب هوكس تتحقق تلقائيًا من التوقيعات وتتحقق من صحة الحمولات باستخدام مخططات Zod. يتم رفض الطلبات غير الصالحة مع رموز خطأ مناسبة.
</Info>

## اختيار الموصل المناسب

| الإطار | الأفضل من أجله | وقت التشغيل |
|-----------|----------|---------|
| **Next.js** | تطبيقات React الكاملة مع App Router | Node.js، Edge |
| **Nuxt** | تطبيقات Vue.js الكاملة | Node.js |
| **Express** | واجهات برمجة التطبيقات REST والتطبيقات التقليدية | Node.js |
| **Fastify** | واجهات برمجة تطبيقات عالية الأداء | Node.js |
| **Hono** | عمليات النشر على الحافة و Cloudflare Workers | Edge، Node.js |
| **Astro** | مواقع محتوى مع نقاط نهاية الخادم | Node.js، Edge |
| **SvelteKit** | تطبيقات Svelte الكاملة | Node.js |
| **Remix** | React كامل مع التوجيه المتداخل | Node.js |
| **TanStack Start** | React الكامل الآمن بالأنواع | Node.js |
| **Better Auth** | التطبيقات التي تستخدم Better Auth بالفعل | متنوع |
| **Convex** | التطبيقات التي تستخدم Convex للبنية الخلفية | Convex Runtime |
| **Bun** | تطبيقات خادم Bun الأصلية | Bun |
