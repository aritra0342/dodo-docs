## المتطلبات الأساسية

لدمج واجهة برمجة تطبيقات مدفوعات دودي، ستحتاج إلى:
- حساب تاجر مدفوعات دودي
- بيانات اعتماد API (مفتاح API ومفتاح سري للويب هوك) من لوحة التحكم

## إعداد لوحة التحكم

1. انتقل إلى [لوحة تحكم مدفوعات دودي](https://app.dodopayments.com/)

2. أنشئ منتجًا (مدفوعات لمرة واحدة أو اشتراك)

4. قم بإنشاء مفتاح API الخاص بك:
   - انتقل إلى المطور > API
   - [دليل مفصل](/api-reference/introduction#api-key-generation)
   - انسخ مفتاح API في البيئة المسماة DODO_PAYMENTS_API_KEY

5. قم بتكوين الويب هوكس:
   - انتقل إلى المطور > الويب هوكس
   - أنشئ عنوان URL للويب هوك لإشعارات الدفع
   - انسخ مفتاح الويب هوك السري في البيئة

## التكامل

### روابط الدفع

اختر مسار التكامل الذي يناسب حالة الاستخدام الخاصة بك:
- **جلسات الدفع (موصى بها)**: الأفضل لمعظم عمليات التكامل. أنشئ جلسة على خادمك وأعد توجيه العملاء إلى صفحة دفع آمنة ومستضافة.
- **الدفع المنبثق**: استخدمه عندما تحتاج إلى تجربة داخل الصفحة تفتح الدفع كنافذة منبثقة على موقعك.
- **الدفع المدمج**: قم بتضمين الدفع مباشرة في تخطيط صفحتك لتجارب دفع متكاملة ومميزة.
- **روابط الدفع الثابتة**: روابط قابلة للمشاركة على الفور بدون كود لجمع المدفوعات بسرعة.
- **روابط الدفع الديناميكية**: روابط تم إنشاؤها برمجيًا. ومع ذلك، يُوصى بجلسات الدفع لأنها توفر مزيدًا من المرونة.

#### 1. جلسات الدفع

استخدم جلسات الدفع لإنشاء تجربة دفع آمنة ومستضافة للمدفوعات لمرة واحدة أو الاشتراكات. تقوم بإنشاء جلسة على خادمك، ثم تعيد توجيه العميل إلى `checkout_url`.

<Info>
جلسات الدفع صالحة لمدة 24 ساعة بشكل افتراضي. إذا قمت بتمرير <code>confirm=true</code>، فإن الجلسات تكون صالحة لمدة 15 دقيقة ويجب توفير جميع الحقول المطلوبة.
</Info>

#### 2. الدفع المدمج (Overlay Checkout)

للحصول على تجربة دفع سلسة داخل الصفحة، استكشف تكامل [الدفع المدمج](/developer-resources/overlay-checkout) الذي يسمح للعملاء بإكمال المدفوعات دون مغادرة موقعك.

#### 3. الدفع المدمج

للحصول على تجارب دفع متكاملة مدمجة مباشرة في صفحتك، استخدم تكامل [الدفع المدمج](/developer-resources/inline-checkout). يتيح لك ذلك بناء ملخصات الطلبات المخصصة والتحكم الكامل في تخطيط الدفع بينما تتولى Dodo Payments جمع المدفوعات بشكل آمن.

#### 4. روابط الدفع الثابتة

تتيح لك روابط الدفع الثابتة قبول المدفوعات بسرعة من خلال مشاركة رابط بسيط. يمكنك تخصيص تجربة الدفع عن طريق تمرير معلمات الاستعلام لملء تفاصيل العملاء مسبقًا، والتحكم في حقول النموذج، وإضافة بيانات وصفية مخصصة.

<Steps>
<Step title="إنشاء رابط الدفع الخاص بك">
  ابدأ بعنوان URL الأساسي وأضف معرف المنتج الخاص بك:

  ```text
  https://checkout.dodopayments.com/buy/{productid}
  ```

</Step>
<Step title="إضافة المعلمات الأساسية">
  قم بتضمين معلمات الاستعلام الأساسية:
  - <ParamField query="quantity" type="integer" default="1">عدد العناصر المراد شراؤها.</ParamField>
  - <ParamField query="redirect_url" type="string" required>عنوان URL لإعادة التوجيه بعد إتمام الدفع.</ParamField>
  <Note>
  سيشمل عنوان URL لإعادة التوجيه تفاصيل الدفع كمعلمات استعلام، على سبيل المثال:<br/>
  <code>https://example.com/?payment_id=pay_ts2ySpzg07phGeBZqePbH&status=succeeded</code>
  </Note>
</Step>
<Step title="ملء معلومات العميل مسبقًا (اختياري)">
  أضف حقول العميل أو الفوترة كمعلمات استعلام لتبسيط عملية الدفع.
  <AccordionGroup>
    <Accordion title="حقول العميل المدعومة">
      - <ParamField query="fullName" type="string">الاسم الكامل للعميل (يتم تجاهله إذا تم تقديم firstName أو lastName).</ParamField>
      - <ParamField query="firstName" type="string">الاسم الأول للعميل.</ParamField>
      - <ParamField query="lastName" type="string">اسم العائلة للعميل.</ParamField>
      - <ParamField query="email" type="string">عنوان البريد الإلكتروني للعميل.</ParamField>
      - <ParamField query="country" type="string">بلد العميل.</ParamField>
      - <ParamField query="addressLine" type="string">عنوان الشارع.</ParamField>
      - <ParamField query="city" type="string">المدينة.</ParamField>
      - <ParamField query="state" type="string">الولاية أو المقاطعة.</ParamField>
      - <ParamField query="zipCode" type="string">الرمز البريدي/ZIP.</ParamField>
      - <ParamField query="showDiscounts" type="boolean">true أو false</ParamField>
    </Accordion>
  </AccordionGroup>
</Step>
<Step title="التحكم في حقول النموذج (اختياري)">
  <Info>
  يمكنك تعطيل حقول معينة لجعلها للقراءة فقط للعميل. هذا مفيد عندما يكون لديك تفاصيل العميل بالفعل (مثل المستخدمين المسجلين).
  </Info>
  لتعطيل حقل، قدم قيمته واضبط علامة <code>disable...</code> المقابلة على <code>true</code>:
  <CodeGroup>

  ```text Example
  &email=alice@example.com&disableEmail=true
  ```

  </CodeGroup>
  <Tabs>
    <Tab title="جدول علامات التعطيل">
      | الحقل         | علامة التعطيل         | المعلمة المطلوبة |
      |---------------|---------------------|-------------------|
      | الاسم الكامل     | `disableFullName`   | `fullName`        |
      | الاسم الأول    | `disableFirstName`  | `firstName`       |
      | اسم العائلة     | `disableLastName`   | `lastName`        |
      | البريد الإلكتروني         | `disableEmail`      | `email`           |
      | البلد       | `disableCountry`    | `country`         |
      | عنوان السكن  | `disableAddressLine`| `addressLine`     |
      | المدينة          | `disableCity`       | `city`            |
      | الولاية         | `disableState`      | `state`           |
      | الرمز البريدي      | `disableZipCode`    | `zipCode`         |
    </Tab>
  </Tabs>
  <Tip>
  يساعد تعطيل الحقول في منع التغييرات غير المقصودة ويضمن اتساق البيانات.
  </Tip>
  <Note>
  تعيين <code>showDiscounts=false</code> سيعطل ويخفي قسم الخصومات في نموذج الدفع. استخدم هذا إذا كنت ترغب في منع العملاء من إدخال رموز القسيمة أو الرموز الترويجية أثناء الدفع.
  </Note>

</Step>
<Step title="إضافة عناصر تحكم متقدمة (اختياري)">
  - <ParamField query="paymentCurrency" type="string">يحدد عملة الدفع. الافتراضي هو عملة بلد الفوترة.</ParamField>
  - <ParamField query="showCurrencySelector" type="boolean" default="true">عرض أو إخفاء محدد العملة.</ParamField>
  - <ParamField query="paymentAmount" type="integer">المبلغ بالسنت (لأسعار "ادفع ما تريد" فقط).</ParamField>
  - <ParamField query="metadata_*" type="string">حقول بيانات وصفية مخصصة (مثل <code>metadata_orderId=123</code>).</ParamField>
</Step>
<Step title="مشاركة الرابط">
  أرسل رابط الدفع المكتمل إلى عميلك. عندما يزورونه، يتم جمع جميع معلمات الاستعلام وتخزينها مع معرف الجلسة. ثم يتم تبسيط عنوان URL ليشمل فقط معلمة الجلسة (على سبيل المثال، <code>?session=sess_1a2b3c4d</code>). تستمر المعلومات المخزنة عبر تحديثات الصفحة ويمكن الوصول إليها طوال عملية الدفع.
  <Check>
  تجربة الدفع للعميل الآن مبسطة ومخصصة بناءً على معلماتك.
  </Check>
</Step>
</Steps>

#### 4. روابط الدفع الديناميكية
<Tip>
يفضل استخدام جلسات الدفع لمعظم حالات الاستخدام، حيث توفر مزيدًا من المرونة والتحكم.
</Tip>

تم إنشاؤها عبر استدعاء API أو SDK الخاص بنا مع تفاصيل العميل. إليك مثال:

هناك واجهتان برمجيتان لإنشاء روابط الدفع الديناميكية:

- واجهة برمجة التطبيقات لرابط الدفع لمرة واحدة [مرجع API](/api-reference/payments/post-payments)
- واجهة برمجة التطبيقات لرابط الدفع للاشتراك [مرجع API](/api-reference/subscriptions/post-subscriptions)

الدليل أدناه لإنشاء رابط الدفع لمرة واحدة.

للحصول على تعليمات مفصلة حول تكامل الاشتراكات، يرجى الرجوع إلى [دليل تكامل الاشتراك](/developer-resources/subscription-integration-guide).

export async function POST(request: NextRequest) {

    const body = await request.json();
    const { formData, cartItems } = paymentRequestSchema.parse(body);

    const response = await fetch(`${process.env.NEXT_PUBLIC_DODO_TEST_API}/payments`, {

          state: formData.state,
          street: formData.addressLine,
          zipcode: parseInt(formData.zipCode),
        },

          name: `${formData.firstName} ${formData.lastName}`,
          phone_number: formData.phoneNumber || undefined,
        },

        product_cart: cartItems.map((id) => ({
          product_id: id,
          quantity: 1,
        })),
        return_url: process.env.NEXT_PUBLIC_RETURN_URL,
      }),
    });

    if (!response.ok) {
      const errorData = await response.json().catch(() => null);
      return NextResponse.json(
        { error: "Payment link creation failed", details: errorData },
        { status: response.status }
      );
    }

    const data = await response.json();
    return NextResponse.json({ paymentLink: data.payment_link });
  } catch (err) {
    console.error("Payment error:", err);
    return NextResponse.json(
      {
        error: err instanceof Error ? err.message : "An unknown error occurred",
      },
      { status: 500 }
    );
  }
}

### تنفيذ Webhooks

قم بإعداد نقطة نهاية API لتلقي إشعارات الدفع. إليك مثال باستخدام Next.js:

```javascript
import { Webhook } from "standardwebhooks";
import { headers } from "next/headers";
import { WebhookPayload } from "@/types/api-types";

const webhook = new Webhook(process.env.DODO_WEBHOOK_KEY!); // Replace with your secret key generated from the Dodo Payments Dashboard

export async function POST(request: Request) {
  const headersList = headers();
  const rawBody = await request.text();

  const webhookHeaders = {
    "webhook-id": headersList.get("webhook-id") || "",
    "webhook-signature": headersList.get("webhook-signature") || "",
    "webhook-timestamp": headersList.get("webhook-timestamp") || "",
  };

  await webhook.verify(rawBody, webhookHeaders);
  const payload = JSON.parse(rawBody) as WebhookPayload;
  
  // Process the payload according to your business logic
}
```

تتبع تنفيذ webhook الخاص بنا مواصفات [Webhooks القياسية](https://standardwebhooks.com/). للحصول على تعريفات أنواع webhook، يرجى الرجوع إلى [دليل أحداث Webhook](/developer-resources/webhooks/intents/webhook-events-guide).
