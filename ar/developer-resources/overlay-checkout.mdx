## بدء سريع

ابدأ مع SDK للدفع عبر Dodo في بضع أسطر من التعليمات البرمجية:

```typescript
import { DodoPayments } from "dodopayments-checkout";

// Initialize the SDK
DodoPayments.Initialize({
  mode: "test", // 'test' or 'live'
  displayType: "overlay", // Optional: defaults to 'overlay' for overlay checkout
  onEvent: (event) => {
    console.log("Checkout event:", event);
  },
});

// Open checkout
DodoPayments.Checkout.open({
  checkoutUrl: "https://checkout.dodopayments.com/session/cks_123"
});
```

<Tip>
احصل على عنوان URL الخاص بصفحة الدفع من [واجهة برمجة تطبيقات إنشاء جلسة الدفع](/api-reference/checkout-sessions/create).
</Tip>

قم بإنشاء مكون يفتح واجهة الدفع:

```typescript
// components/CheckoutButton.tsx
"use client";

import { Button } from "@/components/ui/button";
import { DodoPayments } from "dodopayments-checkout";
import { useEffect, useState } from "react";

export function CheckoutButton() {
  const [isLoading, setIsLoading] = useState(false);

  useEffect(() => {
    // Initialize the SDK
    DodoPayments.Initialize({
      mode: "test",
      displayType: "overlay",
      onEvent: (event) => {
        switch (event.event_type) {
          case "checkout.opened":
            setIsLoading(false);
            break;
          case "checkout.error":
            setIsLoading(false);
            console.error("Checkout error:", event.data?.message);
            break;
        }
      },
    });
  }, []);

  const handleCheckout = async () => {
    try {
      setIsLoading(true);
      await DodoPayments.Checkout.open({
        checkoutUrl: "https://checkout.dodopayments.com/session/cks_123"
      });
    } catch (error) {
      console.error("Failed to open checkout:", error);
      setIsLoading(false);
    }
  };

  return (
    <Button 
      onClick={handleCheckout}
      disabled={isLoading}
    >
      {isLoading ? "Loading..." : "Checkout Now"}
    </Button>
  );
}
```

## مرجع API

### التكوين

#### خيارات التهيئة

```typescript
interface InitializeOptions {
  mode: "test" | "live";
  displayType?: "overlay" | "inline";
  onEvent: (event: CheckoutEvent) => void;
}
```

| الخيار | النوع | مطلوب | الوصف |
|--------|------|----------|-------------|
| `mode` | `"test" \| "live"` | نعم | وضع البيئة: `'test'` للتطوير، `'live'` للإنتاج |
| `displayType` | `"overlay" \| "inline"` | لا | نوع العرض: `'overlay'` لعرض الدفع المنبثق (الافتراضي)، `'inline'` لعرض الدفع المدمج |
| `onEvent` | `function` | نعم | دالة رد الاتصال لمعالجة أحداث الدفع |

#### خيارات الدفع

```typescript
interface CheckoutOptions {
  checkoutUrl: string;
  options?: {
    showTimer?: boolean;
    showSecurityBadge?: boolean;
    manualRedirect?: boolean;
  };
}
```

| الخيار | النوع | مطلوب | الوصف |
|--------|------|----------|-------------|
| `checkoutUrl` | `string` | نعم | عنوان جلسة الدفع من [واجهة برمجة تطبيقات إنشاء جلسة الدفع](/api-reference/checkout-sessions/create) |
| `options.showTimer` | `boolean` | لا | إظهار أو إخفاء مؤقت الدفع. الافتراضي `true`. عند تعطيله، ستتلقى حدث `checkout.link_expired` عند انتهاء الجلسة. |
| `options.showSecurityBadge` | `boolean` | لا | إظهار أو إخفاء شارة الأمان. الافتراضي `true`. |
| `options.manualRedirect` | `boolean` | لا | عند تمكينه، لن يعيد الدفع التوجيه تلقائيًا بعد الاكتمال. بدلًا من ذلك، ستتلقى أحداث `checkout.status` و`checkout.redirect_requested` لتتعامل مع التوجيه بنفسك. |

### الطرق

#### فتح الدفع

يفتح واجهة الدفع مع عنوان URL لجلسة الدفع المحدد.

```typescript
DodoPayments.Checkout.open({
  checkoutUrl: "https://checkout.dodopayments.com/session/cks_123"
});
```

يمكنك أيضًا تمرير خيارات إضافية لتخصيص سلوك الدفع:

```typescript
DodoPayments.Checkout.open({
  checkoutUrl: "https://checkout.dodopayments.com/session/cks_123",
  options: {
    showTimer: false,
    showSecurityBadge: false,
    manualRedirect: true,
  },
});
```

عند استخدام `manualRedirect`، تعامل مع اكتمال الدفع في رد الاتصال `onEvent` الخاص بك:

```typescript
DodoPayments.Initialize({
  mode: "test",
  displayType: "overlay",
  onEvent: (event) => {
    if (event.event_type === "checkout.status") {
      const status = event.data?.message?.status;
      // Handle status: "succeeded", "failed", or "processing"
    }
    if (event.event_type === "checkout.redirect_requested") {
      const redirectUrl = event.data?.message?.redirect_to;
      // Redirect the customer manually
      window.location.href = redirectUrl;
    }
    if (event.event_type === "checkout.link_expired") {
      // Handle expired checkout session
    }
  },
});
```

#### إغلاق الدفع

يغلق overlay الدفع برمجيًا.

```typescript
DodoPayments.Checkout.close();
```

#### تحقق من الحالة

يعيد ما إذا كان overlay الدفع مفتوحًا حاليًا.

```typescript
const isOpen = DodoPayments.Checkout.isOpen();
// Returns: boolean
```

### الأحداث

يوفر SDK أحداثًا في الوقت الفعلي يمكنك الاستماع إليها من خلال رد الاتصال `onEvent`:

#### بيانات حدث حالة الدفع

عند تمكين `manualRedirect`، تتلقى حدث `checkout.status` بالبيانات التالية:

```typescript
interface CheckoutStatusEventData {
  message: {
    status?: "succeeded" | "failed" | "processing";
  };
}
```

#### بيانات حدث إعادة التوجيه المطلوبة للدفع

عند تمكين `manualRedirect`، تتلقى حدث `checkout.redirect_requested` بالبيانات التالية:

```typescript
interface CheckoutRedirectRequestedEventData {
  message: {
    redirect_to?: string;
  };
}
```

```typescript
DodoPayments.Initialize({
  onEvent: (event: CheckoutEvent) => {
    switch (event.event_type) {
      case "checkout.opened":
        // Checkout overlay has been opened
        break;
      case "checkout.form_ready":
        // Checkout form is ready for user input
        break;
      case "checkout.payment_page_opened":
        // Payment page has been displayed
        break;
      case "checkout.customer_details_submitted":
        // Customer and billing details submitted
        break;
      case "checkout.closed":
        // Checkout has been closed
        break;
      case "checkout.redirect":
        // Checkout will perform a redirect
        break;
      case "checkout.error":
        // An error occurred
        console.error("Error:", event.data?.message);
        break;
      case "checkout.link_expired":
        // Checkout session has expired (only when showTimer is false)
        break;
      case "checkout.status":
        // Checkout status update (only when manualRedirect is enabled)
        const status = event.data?.message?.status;
        break;
      case "checkout.redirect_requested":
        // Redirect requested (only when manualRedirect is enabled)
        const redirectUrl = event.data?.message?.redirect_to;
        break;
    }
  }
});
```

| نوع الحدث | الوصف |
|------------|-------------|
| `checkout.opened` | تم فتح التراكب الخاص بصفحة الدفع |
| `checkout.form_ready` | نموذج الدفع جاهز لتلقي إدخال المستخدم. مفيد لإخفاء حالات التحميل. |
| `checkout.payment_page_opened` | تم عرض صفحة الدفع |
| `checkout.customer_details_submitted` | تم تقديم تفاصيل العميل والفوترة |
| `checkout.closed` | تم إغلاق التراكب الخاص بصفحة الدفع |
| `checkout.redirect` | سيقوم الدفع بإجراء إعادة توجيه |
| `checkout.error` | حدث خطأ أثناء الدفع |
| `checkout.link_expired` | يتم إطلاقه عند انتهاء صلاحية جلسة الدفع. يتم استلامه فقط عندما يتم تعيين `showTimer` على `false`. |
| `checkout.status` | يتم إطلاقه عند تمكين `manualRedirect`. يحتوي على حالة الدفع (`succeeded`، `failed`، أو `processing`). |
| `checkout.redirect_requested` | يتم إطلاقه عند تمكين `manualRedirect`. يحتوي على عنوان URL لإعادة توجيه العميل إليه. |

### تخصيص السمة

يمكنك تخصيص مظهر صفحة الدفع بتمرير كائن `themeConfig` في معلمة `options` عند فتح الدفع. يدعم تكوين السمة كلا الوضعين الفاتح والداكن، مما يتيح لك تخصيص الألوان والحدود والنصوص والأزرار ونصف قطر الحدود.

<Info>
يغطي هذا القسم تكوين السمة على **جهة العميل** باستخدام Checkout SDK. يمكنك أيضًا تكوين السمات **من جهة الخادم** عند إنشاء جلسة الدفع عبر واجهة برمجة التطبيقات باستخدام المعامل `theme_config`. راجع [تخصيص سمة الدفع](/features/checkout#checkout-theme-customization) للتكوين على مستوى API.
</Info>

## التعامل مع الأخطاء

يوفر SDK معلومات مفصلة عن الأخطاء عبر نظام الأحداث. نفّذ دائمًا معالجة مناسبة للأخطاء في رد الاتصال `onEvent` الخاص بك:

```typescript
DodoPayments.Initialize({
  mode: "test",
  displayType: "overlay",
  onEvent: (event: CheckoutEvent) => {
    if (event.event_type === "checkout.error") {
      console.error("Checkout error:", event.data?.message);
      // Handle error appropriately
      // You may want to show a user-friendly error message
      // or retry the checkout process
    }
    if (event.event_type === "checkout.link_expired") {
      // Handle expired checkout session
      console.warn("Checkout session has expired");
    }
  }
});
```

<Warning>
تعامل دائمًا مع حدث `checkout.error` لتوفير تجربة مستخدم جيدة عند حدوث أخطاء.
</Warning>

## أفضل الممارسات

1. **تهيئة مرة واحدة**: قم بتهيئة SDK مرة واحدة عند تحميل تطبيقك، وليس عند كل محاولة دفع
2. **التعامل مع الأخطاء**: نفّذ دائمًا التعامل المناسب مع الأخطاء في رد اتصال الأحداث الخاص بك
3. **وضع الاختبار**: استخدم `test` أثناء التطوير وانتقل إلى `live` فقط عندما تكون جاهزًا للإنتاج
4. **معالجة الأحداث**: عالج جميع الأحداث ذات الصلة لتوفير تجربة مستخدم كاملة
5. **عناوين URL صالحة**: استخدم دائمًا عناوين URL صالحة من واجهة برمجة تطبيقات إنشاء جلسة الدفع
6. **TypeScript**: استخدم TypeScript للحصول على أمان نوع أفضل وتجربة تطوير محسنة
7. **حالات التحميل**: اعرض حالات التحميل أثناء فتح صفحة الدفع لتحسين تجربة المستخدم
8. **إعادة التوجيه اليدوية**: استخدم `manualRedirect` عندما تحتاج إلى تحكم مخصص في التنقل بعد الدفع
9. **إدارة المؤقت**: عطل المؤقت `showTimer: false` إذا كنت تريد التعامل مع انتهاء صلاحية الجلسة يدويًا

## الدفع المتراكب مقابل الدفع المدمج

اختر نوع الدفع المناسب لحالتك:

| الميزة | الدفع المتراكب | الدفع المدمج |
|---------|-----------------|-----------------|
| عمق التكامل | نافذة منبثقة فوق الصفحة | مدمج بالكامل في الصفحة |
| التحكم في التخطيط | محدود | تحكم كامل |
| العلامة التجارية | منفصل عن الصفحة | متجانس |
| جهد التنفيذ | أقل | أعلى |
| الأفضل لـ | تكامل سريع، صفحات قائمة | صفحات دفع مخصصة، تدفقات تحويل عالية |

<Tip>
استخدم **الدفع المتراكب** للتكامل السريع مع تغييرات قليلة على صفحاتك الحالية. استخدم **الدفع المدمج** عندما تريد التحكم الأقصى في تجربة الدفع والعلامة التجارية المتجانسة.
</Tip>
