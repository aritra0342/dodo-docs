## التثبيت

<Steps>
<Step title="تثبيت الحزمة">
قم بتشغيل الأمر التالي في جذر مشروعك:

```bash
npm install @dodopayments/remix
```

</Step>
<Step title="إعداد متغيرات البيئة">
قم بإنشاء ملف <code>.env</code> في جذر مشروعك:

```env expandable
DODO_PAYMENTS_API_KEY=your-api-key
DODO_PAYMENTS_WEBHOOK_SECRET=your-webhook-secret
DODO_PAYMENTS_RETURN_URL=https://yourdomain.com/checkout/success
DODO_PAYMENTS_ENVIRONMENT="test_mode" or "live_mode"
```

<Warning>
لا تقم بالتزام ملف <code>.env</code> أو الأسرار في نظام التحكم في الإصدارات.
</Warning>
</Step>
</Steps>

    const checkoutGetHandler = Checkout({

        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,

    const checkoutPostHandler = Checkout({

## معالج مسار الدفع

<Info>
تدعم مدفوعات دودو ثلاثة أنواع من تدفقات الدفع لدمج المدفوعات في موقعك، يدعم هذا المحول جميع أنواع تدفقات الدفع.
</Info>

- **روابط الدفع الثابتة:** روابط قابلة للمشاركة على الفور لجمع المدفوعات بسرعة وبدون كود.
- **روابط الدفع الديناميكية:** توليد روابط الدفع برمجيًا مع تفاصيل مخصصة باستخدام API أو SDKs.
- **جلسات الدفع:** إنشاء تجارب دفع آمنة وقابلة للتخصيص مع عربات منتجات مُعدة مسبقًا وتفاصيل العملاء.

- يدعم كل من المدفوعات لمرة واحدة والمدفوعات المتكررة.
- للحصول على قائمة كاملة من حقول جسم POST المدعومة، راجع:
  - [جسم الطلب لمنتج الدفع لمرة واحدة](https://docs.dodopayments.com/api-reference/payments/post-payments)
  - [جسم الطلب لمنتج الاشتراك](https://docs.dodopayments.com/api-reference/subscriptions/post-subscriptions)

### تنسيق الاستجابة
يرجع الدفع الديناميكي استجابة JSON مع عنوان URL للدفع:

```json
{
  "checkout_url": "https://checkout.dodopayments.com/..."
}
```

</Accordion>
<Accordion title="جلسات الدفع (POST)">
توفر جلسات الدفع تجربة دفع أكثر أمانًا، مستضافة، تتعامل مع تدفق الدفع الكامل لكل من المشتريات لمرة واحدة والاشتراكات مع التحكم الكامل في التخصيص.

راجع [دليل تكامل جلسات الدفع](https://docs.dodopayments.com/developer-resources/checkout-session) لمزيد من التفاصيل وقائمة كاملة من الحقول المدعومة.

### تنسيق الاستجابة
ترجع جلسات الدفع استجابة JSON مع عنوان URL للدفع:

```json
{
  "checkout_url": "https://checkout.dodopayments.com/session/..."
}
```

</Accordion>

## معالج مسار بوابة العملاء

يمكنك من خلال معالج مسار بوابة العملاء دمج بوابة عملاء مدفوعات دودو بسلاسة في تطبيق ريمكس الخاص بك.

### معلمات الاستعلام
<ParamField query="customer_id" type="string" required>
معرف العميل لجلسة البوابة (على سبيل المثال، <code>?customer_id=cus_123</code>).
</ParamField>
<ParamField query="send_email" type="boolean">
إذا تم تعيينها على <code>true</code>، يتم إرسال بريد إلكتروني إلى العميل مع رابط البوابة.
</ParamField>

<Warning>
يرجع 400 إذا كان <code>customer_id</code> مفقودًا.
</Warning>

## معالج مسار ويب هوك

- **الطريقة:** يتم دعم طلبات POST فقط. ترجع الطرق الأخرى 405.
- **التحقق من التوقيع:** يتحقق من توقيع الويب هوك باستخدام <code>webhookKey</code>. يرجع 401 إذا فشل التحقق.
- **التحقق من الحمولة:** يتم التحقق منها باستخدام Zod. يرجع 400 للحمولات غير الصحيحة.
- **معالجة الأخطاء:**
  - 401: توقيع غير صالح
  - 400: حمولة غير صالحة
  - 500: خطأ داخلي أثناء التحقق
- **توجيه الأحداث:** يستدعي معالج الحدث المناسب بناءً على نوع الحمولة.
