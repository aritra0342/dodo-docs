## نظرة عامة

تضيف سينترا الذكاء والتنفيذ على قمة مجموعات دودي بايمنتس الأساسية. إنها وكيل ذكاء اصطناعي يدمج ويحلل ويتصرف في الفوترة والمدفوعات. تربط سينترا SDKs وAPIs والمحولات عبر مجموعتك (المصادقة، قاعدة البيانات، إدارة علاقات العملاء، إلخ)، وتظهر رؤى الإيرادات مثل محركات النمو، وتنفذ إجراءات العملاء مثل الاستردادات والترقيات والائتمانات.

<Info>
سينترا حاليًا في المرحلة التجريبية. قد تتغير الميزات وAPIs والتكاملات بسرعة. نرحب بالتعليقات وتقارير الأخطاء أثناء تطويرنا!
</Info>

## الإعداد

بمجرد التثبيت، تحتاج إلى تكوين سينترا باستخدام مفتاح API الخاص بدودي بايمنتس وإعدادات البيئة.

<Steps>
<Step title="احصل على مفتاح API الخاص بك">
1. قم بتسجيل الدخول إلى [لوحة معلومات دودي بايمنتس](https://dashboard.dodopayments.com).
2. انتقل إلى **الإعدادات** › **مفاتيح API**.
3. انسخ مفتاح API الخاص بك.
</Step>

<Step title="تكوين سينترا">
1. افتح بيئة التطوير المتكاملة الخاصة بك (VS Code أو Cursor أو Windsurf).
2. افتح لوحة الأوامر (`Cmd+Shift+P` على macOS / `Ctrl+Shift+P` على Windows/Linux).
3. اكتب "سينترا: تكوين" واخترها.
4. أدخل مفتاح API الخاص بدودي بايمنتس عند الطلب.

<Frame>
  <img
    src="/images/sentra/setup.png"
    alt="شاشة تكوين سينترا مع إدخال مفتاح API واختيار البيئة"
    width="50%"
  />
</Frame>
</Step>

<Step title="اختر وضع البيئة">
اختر وضع البيئة الخاص بك:
- **وضع الاختبار**: للتطوير والاختبار
- **وضع الإنتاج**: للاستخدام في الإنتاج

<Warning>
ابدأ بوضع الاختبار للتحقق من إعدادك قبل التبديل إلى وضع الإنتاج للبيانات الحقيقية.
</Warning>
</Step>

<Step title="تعيين مستوى جهد التفكير للنموذج">
اختر مستوى جهد التفكير المفضل لديك للنموذج:
- **منخفض**: استجابات أسرع، مناسبة للمهام البسيطة
- **متوسط**: أداء متوازن (موصى به)
- **مرتفع**: تحليل أكثر شمولاً، مناسب للتكاملات المعقدة
</Step>

<Step title="إكمال الإعداد">
انقر على **لنبدأ!** لإكمال الإعداد وبدء استخدام سينترا.

<Check>
تُخزن مفاتيح API الخاصة بك بأمان في تخزين الأسرار في بيئة التطوير المتكاملة الخاصة بك ولا تُعرض أبدًا في الكود الخاص بك.
</Check>
</Step>
</Steps>

## استخدام سينترا

### الأوضاع

تقدم سينترا ثلاثة أوضاع تفاعل يمكن الوصول إليها من الواجهة:

- **اسأل**: اطرح أسئلة حول دودي بايمنتس، نماذج الفوترة، استخدام API، وأفضل الممارسات
- **ابنِ**: دمج الفوترة وواجهة المستخدم في تطبيقك بمساعدة موجهة
- **خطط**: تصميم هياكل فوترة قابلة للتوسع مع إرشادات معمارية

### مهام نموذجية

إليك بعض المهام الشائعة التي يمكنك إنجازها باستخدام سينترا:

- **دمج صفحة التسعير**: إعداد صفحة تسعير كاملة مع قوائم المنتجات وإجراءات الدفع
- **تنفيذ Webhook**: تكوين معالجات webhook لأحداث الدفع
- **إضافة تدفق الاشتراك**: إنشاء إدارة الاشتراك مع قدرات الترقية/الخفض
- **إعداد فوترة قائمة على الاستخدام**: تكوين عدادات واستيعاب الأحداث لتتبع الاستخدام
- **إنشاء بوابة العملاء**: بناء بوابة خدمة ذاتية للعملاء لإدارة الفوترة

### إعدادات الموافقة التلقائية

يمكنك تكوين إعدادات الموافقة التلقائية لتبسيط سير العمل الخاص بك:

- **لا شيء**: مراجعة والموافقة على جميع التغييرات يدويًا (موصى به للإنتاج)
- **مخاطر منخفضة**: الموافقة التلقائية على التغييرات البسيطة ومنخفضة المخاطر
- **الجميع**: الموافقة التلقائية على جميع التغييرات (استخدم بحذر)

## نماذج الفوترة المدعومة

تدعم سينترا جميع نماذج الفوترة لدودي بايمنتس:

- **المدفوعات لمرة واحدة**: معاملات الشراء الفردية
- **الاشتراكات**: فوترة متكررة مع فترات متعددة
- **فوترة قائمة على الاستخدام**: فوترة مقاسة بناءً على الاستهلاك
- **فوترة قائمة على النتائج**: نماذج تسعير الدفع مقابل النتيجة

## لمن هي

تعمل سينترا على ربط الهندسة والأعمال، بحيث يمكن لكل فريق شحن العائدات دون احتكاك:

- **المطورون**: شحن العائدات بشكل أسرع داخل بيئة التطوير المتكاملة الخاصة بك
- **المؤسسون ومديرو المنتجات**: نمذجة نماذج الفوترة دون الحاجة إلى العمل العميق في البنية التحتية
- **RevOps والمالية**: استخدام إجراءات محمية دون الحاجة إلى مهندسين
- **الدعم والنجاح**: حل حالات الفوترة بأمان، مع الموافقات حيثما دعت الحاجة

## استكشاف الأخطاء وإصلاحها

<AccordionGroup>
<Accordion title="الملحق لا يظهر بعد التثبيت">
إذا لم تظهر سينترا في بيئة التطوير المتكاملة الخاصة بك بعد التثبيت، جرب ما يلي:

1. **أعد تشغيل المحرر**: أغلق وأعد فتح VS Code أو Cursor أو Windsurf تمامًا
2. **تحقق من التثبيت**: تحقق من تثبيت الملحق بالذهاب إلى الملحقات والبحث عن "سينترا"
3. **إعادة تحميل النافذة**: استخدم لوحة الأوامر (`Cmd+Shift+P` / `Ctrl+Shift+P`) وقم بتشغيل "المطور: إعادة تحميل النافذة"

<Check>
عادةً ما تحل إعادة تشغيل المحرر المشكلة في معظم الحالات.
</Check>
</Accordion>

<Accordion title="مفتاح API غير صالح أو أخطاء في المصادقة">
- تحقق من أن مفتاح API الخاص بك صحيح ونشط في لوحة معلومات دودي بايمنتس
- تأكد من أنك تستخدم وضع البيئة الصحيح (اختبار مقابل إنتاج) الذي يتطابق مع مفتاح API الخاص بك
- تحقق من أن مفتاح API الخاص بك لديه الأذونات اللازمة
- حاول إعادة توليد مفتاح API الخاص بك إذا استمرت المشكلة
</Accordion>

<Accordion title="عدم حفظ التكوين">
- تأكد من أنك قد أكملت جميع الحقول المطلوبة في شاشة التكوين
- تحقق من أن بيئة التطوير المتكاملة الخاصة بك لديها إذن للوصول إلى تخزين الأسرار
- حاول إعادة تكوين سينترا من خلال لوحة الأوامر
</Accordion>

<Accordion title="فشل توليد الخطة">
- تحقق من أن مفتاح API الخاص بك صالح وله الوصول إلى الموارد المطلوبة
- تحقق من اتصال الإنترنت الخاص بك
- تأكد من أنك تستخدم نموذج فوترة مدعوم
- حاول إعادة صياغة طلبك بمزيد من التفاصيل المحددة
</Accordion>

<Accordion title="أخطاء التنفيذ">
- راجع الخطة التي تم إنشاؤها بعناية قبل الموافقة
- تحقق من أن هيكل مشروعك يتطابق مع توقعات سينترا
- تحقق من تثبيت جميع التبعيات
- تحقق من لوحة إخراج بيئة التطوير المتكاملة للحصول على رسائل خطأ مفصلة
</Accordion>
</AccordionGroup>

<Warning>
إذا استمرت المشاكل بعد إعادة تشغيل المحرر، اتصل بالدعم مع تفاصيل حول إصدار بيئة التطوير المتكاملة الخاصة بك ورسائل الخطأ المحددة التي تراها.
</Warning>
