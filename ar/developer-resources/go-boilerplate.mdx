## نظرة عامة

يوفر نموذج Go نقطة انطلاق جاهزة للإنتاج لدمج مدفوعات دودي مع خلفية Go الخاصة بك. يتضمن هذا القالب معالجة جلسات الدفع، والتحقق من الويب هوك، ودمج بوابة العملاء، ويتبع أفضل الممارسات في Go لمساعدتك على البدء في قبول المدفوعات بسرعة.

<Info>
يستخدم هذا النموذج Go 1.21+ مع أنماط الهندسة النظيفة (`cmd`, `internal`, `templates`)، وقوالب HTML، و `dodopayments-go` SDK للتكامل السلس مع واجهة برمجة التطبيقات.
</Info>

### الميزات

- **الإعداد السريع** - ابدأ خلال أقل من 5 دقائق
- **تكامل الدفع** - تدفق الخروج معد مسبقًا باستخدام `dodopayments-go` SDK
- **واجهة حديثة** - صفحة أسعار نظيفة ذات سمة داكنة مع قوالب HTML
- **معالجة Webhook** - تحقق آمن ومعالجة لأحداث الدفع
- **بوابة العملاء** - إدارة الاشتراكات ذاتيًا
- **أفضل ممارسات Go** - هندسة نظيفة مع `cmd`، `internal`، و `templates`
- **عملية دفع معبأة مسبقًا** - يُظهر تمرير بيانات العميل لتحسين تجربة المستخدم

## المتطلبات المسبقة

قبل أن تبدأ، تأكد من أن لديك:

- **Go 1.21+**
- **حساب مدفوعات دودي** (للوصول إلى مفاتيح API وWebhook من لوحة التحكم)

## هيكل المشروع

```text
go-boilerplate/
├── cmd/
│   └── server/             # Application entry point
├── internal/
│   ├── api/                # API handlers (Checkout, Portal, Webhook)
│   ├── core/               # Configuration and system core
│   └── lib/                # Shared logic (Products, Customer utils)
├── templates/              # HTML templates
├── Makefile                # Build and run commands
├── go.mod                  # Go module definition
├── go.sum                  # Dependency checksums
├── .env.example            # Environment template
└── README.md
```

## نقاط نهاية API

يتضمن النموذج نقاط النهاية المُعدة مسبقًا التالية:

| نقطة النهاية | الطريقة | الوصف |
|--------------|--------|------------------------------|
| `/` | GET | صفحة أسعار مع قائمة بالمنتجات |
| `/api/checkout` | POST | إنشاء جلسة خروج جديدة |
| `/api/webhook` | POST | معالجة webhooks الخاصة بـ Dodo Payments |
| `/api/customer-portal` | POST | إنشاء رابط بوابة العملاء |

## التخصيص

### تحديث معلومات المنتج

قم بتحرير `internal/lib/products.go` لتعديل:
- معرفات المنتج (من لوحة Dodo الخاصة بك)
- التسعير
- الميزات
- الأوصاف

```go
var Products = []Product{
    {
        ProductID:   "pdt_001", // Replace with your product ID
        Name:        "Basic Plan",
        Description: "Get access to basic features and support",
        Price:       9999,
        Features: []string{
            "Access to basic features",
            "Email support",
            "1 Team member",
            "Basic analytics",
        },
    },
}
```

### ملء بيانات العملاء مسبقًا

في `templates/index.html`، استبدل بيانات العميل المضمنة مباشرة ببيانات المستخدم الفعلية الخاصة بك:

```javascript
const customerData = {
    name: "John Doe",       // Replace with actual logged-in user's name
    email: "john@example.com"  // Replace with actual logged-in user's email
};
```

في تطبيق الإنتاج، ستقوم بحقن هذه القيم ديناميكيًا من نظام المصادقة الخاص بك.

## أحداث الويب هوك

يوضح هذا النموذج كيفية التعامل مع أحداث webhook في `internal/api/webhook.go`. تشمل الأحداث المدعومة:

| الحدث | الوصف |
|-------|-------------|
| `subscription.active` | يتم تشغيله عندما يصبح الاشتراك فعالًا |
| `payment.succeeded` | يتم تشغيله عند نجاح الدفع |

أضف منطق عملك داخل معالج الويب هوك لـ:
- تحديث أذونات المستخدم في قاعدة بياناتك
- إرسال رسائل تأكيد عبر البريد الإلكتروني
- توفير الوصول إلى المنتجات الرقمية
- تتبع التحليلات والبيانات

## اختبار الويب هوك محليًا

للتطوير المحلي، استخدم أدوات مثل [ngrok](https://ngrok.com/) لفتح خادمك المحلي:

```bash
ngrok http 8000
```

قم بتحديث عنوان URL للويب هوك في [لوحة تحكم مدفوعات دودي](https://app.dodopayments.com/developer/webhooks):

```
https://your-ngrok-url.ngrok.io/api/webhook
```

## النشر

### بناء للإنتاج

```bash
make build
```

أو يدويًا:

```bash
go build -o bin/server cmd/server/main.go
./bin/server
```

[

![نشر مع Vercel](https://vercel.com/button)

### Docker

أنشئ `Dockerfile`:

```dockerfile
FROM golang:1.21-alpine AS builder

WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download

COPY . .
RUN go build -o bin/server cmd/server/main.go

FROM alpine:latest
WORKDIR /app
COPY --from=builder /app/bin/server .
COPY --from=builder /app/templates ./templates

EXPOSE 8000
CMD ["./server"]
```

قم بالبناء والتشغيل:

```bash
docker build -t go-dodo .
docker run -p 8000:8000 --env-file .env go-dodo
```

### اعتبارات الإنتاج

<Warning>
قبل النشر في بيئة الإنتاج:
- بدل `DODO_PAYMENTS_ENVIRONMENT` إلى `live_mode`
- استخدم مفاتيح API الخاصة بالإنتاج من لوحة التحكم
- حدّث عنوان URL لويب هوك ليتطابق مع نطاق الإنتاج لديك
- فعّل HTTPS لجميع نقاط النهاية
</Warning>
