## أحداث ويب هوك الاشتراك

الأحداث التالية متاحة لتتبع تغييرات دورة حياة الاشتراك:

| Event | Description |
|-------|-------------|
| `subscription.active` | تم تفعيل الاشتراك بنجاح |
| `subscription.updated` | تم تحديث كائن الاشتراك (أي تغيير في الحقل يؤدي إلى إطلاق هذا الحدث) |
| `subscription.on_hold` | تم تعليق الاشتراك بسبب فشل التجديد |
| `subscription.renewed` | تم تجديد الاشتراك للفترة التالية من الفوترة |
| `subscription.plan_changed` | تم ترقية أو تخفيض أو تعديل خطة الاشتراك |
| `subscription.cancelled` | تم إلغاء الاشتراك |
| `subscription.failed` | فشل إنشاء الاشتراك أثناء إنشاء التفويض |
| `subscription.expired` | وصل الاشتراك إلى نهاية مدته |

### استخدام `subscription.updated` للمزامنة في الوقت الفعلي

يتم تشغيل webhook الخاص بـ `subscription.updated` كلما تغير أي حقل في الاشتراك، مما يسمح لك بمزامنة حالة تطبيقك دون الحاجة إلى الاستقصاء:

```javascript
app.post('/webhooks/dodo', async (req, res) => {
  const event = req.body;
  
  if (event.type === 'subscription.updated') {
    const subscription = event.data;
    
    // Sync subscription changes to your database
    await syncSubscription(subscription.subscription_id, {
      status: subscription.status,
      next_billing_date: subscription.next_billing_date,
      metadata: subscription.metadata,
      // ... other fields you want to track
    });
    
    console.log(`Subscription ${subscription.subscription_id} updated`);
  }
  
  res.json({ received: true });
});
```

<Tip>
اشترك في `subscription.updated` لتلقي إشعارات في الوقت الفعلي حول أي تغييرات في الاشتراك، مما يلغي الحاجة إلى الاستقصاء عبر واجهة برمجة التطبيقات.
</Tip>
