يغطي هذا الدليل حصريًا **المدفوعات لمرة واحدة** التي تتضمن **البطاقات وApple Pay وGoogle Pay**.
</Note>

## نقاط النهاية الخاصة بواجهة برمجة التطبيقات

| البيئة | عنوان URL الأساسي | نقطة نهاية تأكيد الدفع |
| --- | --- | --- |
| **وضع الاختبار** | `https://test.dodopayments.com` | `https://api.custom.hs.dodopayments.com/payments/<payment_id>/confirm` |
| **الوضع المباشر** | `https://live.dodopayments.com` | `https://api.hs.dodopayments.com/payments/<payment_id>/confirm` |

## سير دفع البطاقة خطوة بخطوة

### الخطوة 1: الحصول على المفتاح السري

ابدأ بإنشاء دفعة باستخدام واجهة برمجة التطبيقات التالية:

- **المدفوعات**: [واجهة برمجة تطبيقات إنشاء الدفع](/api-reference/payments/post-payments)

تعيد هذه الواجهة **`secret_key`**، المطلوبة للخطوات التالية.

### الخطوة 2: تأكيد الدفع

قم بتأكيد الدفع عن طريق استدعاء نقطة النهاية `payments confirm`:

- **وضع الاختبار**: `POST https://api.custom.hs.dodopayments.com/payments/<payment_id>/confirm`
- **الوضع المباشر**: `POST https://api.hs.dodopayments.com/payments/<payment_id>/confirm`

قم بتضمين التفاصيل التالية في جسم الطلب:

- `client_secret`: المفتاح السري من الخطوة 1
- `payment_method`: اضبطه على `card`
- `payment_method_data`: تفاصيل البطاقة والفوترة
- `browser_info`: بيانات خاصة بالمتصفح (وكيل المستخدم، تفاصيل الشاشة، إلخ)
- `payment_type`: اضبطه صراحة على `normal` (للمدفوعات لمرة واحدة)

الرؤوس المطلوبة:

- احصل على مفتاحك القابل للنشر من دعم DodoPayments
- `api-key`: مفتاحك القابل للنشر (احصل عليه من فريق Dodo Payments)
- `content-type`: `application/json`

**مثال على جسم الطلب:**

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "card",
  "payment_method_data": {

      "card_network": "<network>"
    },
    "billing": {
      "address": {
        "first_name": "<first_name>",
        "last_name": "<last_name>"
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
```

### التعامل مع الاستجابة

- إذا كانت الاستجابة **`status`** هي **`requires_customer_action`**، فأعد توجيه العميل إلى عنوان URL الموجود في حقل **`next_action`** لمتابعة المصادقة.

<Warning>
تأكد من الالتزام بإرشادات الامتثال لـ PCI عند التعامل مع بيانات البطاقة الخام.
</Warning>

## دمج Apple Pay

لاستخدام Apple Pay، اضبط `payment_method` على `wallet` وأدرج بيانات Apple Pay في `payment_method_data`.

### هيكل جسم الطلب

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "wallet",
  "payment_method_data": {
    "wallet": {
      "apple_pay": {
        "payment_data": "<decrypted_payment_data>",
        "payment_method": {
          "display_name": "Visa 1234",
          "network": "Visa",
          "type": "debit"
        },
        "transaction_identifier": "UNIQUE_TRANSACTION_ID_FROM_APPLE"
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
    "time_zone": "<time_zone>",
    "java_enabled": true,
    "java_script_enabled": true,
    "ip_address": "<ip_address_of_client>",
    "device_model": "<device_model>",
    "os_type": "<os_type>",
    "os_version": "<version>"
  },
  "payment_type": "normal"
}
```

### مرجع حقول Apple Pay

| الحقل | النوع | مطلوب | الوصف |
| --- | --- | --- | --- |
| `payment_data` | string | نعم | يُستخدم هذا التعداد لتمثيل بيانات الدفع في Apple Pay، والتي يمكن أن تكون مشفرة أو غير مشفرة. |
| `payment_method.display_name` | string | نعم | اسم العرض الذي يظهر على زر Apple Pay (مثال: "Visa 1234") |
| `payment_method.network` | string | نعم | شبكة البطاقة (مثال: "Visa"، "Mastercard"، "Amex") |
| `payment_method.type` | string | نعم | نوع طريقة الدفع (مثال: "debit"، "credit") |
| `payment_method.card_exp_month` | string | لا | شهر انتهاء صلاحية البطاقة (مثال: "12") |
| `payment_method.card_exp_year` | string | لا | سنة انتهاء صلاحية البطاقة (مثال: "25") |
| `transaction_identifier` | string | نعم | معرف المعاملة الفريد من Apple Pay |

## دمج Google Pay

لاستخدام Google Pay، اضبط `payment_method` على `wallet` وأدرج بيانات Google Pay في `payment_method_data`.

### هيكل جسم الطلب

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "wallet",
  "payment_method_data": {
    "wallet": {
      "google_pay": {
        "type": "CARD",
        "description": "Visa ending in 1234",
        "info": {
          "card_network": "VISA",
          "card_details": "1234"
        },
        "tokenization_data": {
          "type": "PAYMENT_GATEWAY",
          "token": "<encrypted_payment_token>"
        }
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
    "time_zone": "<time_zone>",
    "java_enabled": true,
    "java_script_enabled": true,
    "ip_address": "<ip_address_of_client>",
    "device_model": "<device_model>",
    "os_type": "<os_type>",
    "os_version": "<version>"
  },
  "payment_type": "normal"
}
```

### مرجع حقول Google Pay

| الحقل | النوع | مطلوب | الوصف |
| --- | --- | --- | --- |
| `type` | string | نعم | نوع طريقة الدفع (عادةً "CARD") |
| `description` | string | نعم | الوصف الذي يظهر للمستخدم لطريقة الدفع |
| `info.card_network` | string | نعم | شبكة البطاقة (مثال: "VISA"، "MASTERCARD"، "AMEX") |
| `info.card_details` | string | نعم | آخر 4 أرقام من البطاقة |
| `info.assurance_details` | object | لا | تفاصيل ضمان المصادقة |
| `tokenization_data.type` | string | نعم | نوع الترميز (مثال: "PAYMENT_GATEWAY") |
| `tokenization_data.token` | string | نعم | رمز الدفع المشفر من Google Pay |
