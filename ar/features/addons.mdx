## ما هي الإضافات؟

الإضافات هي منتجات تكميلية يمكن للعملاء شراؤها جنبًا إلى جنب مع اشتراكهم الرئيسي. إنها مثالية لـ:

- **فوترة قائمة على المقاعد**: أعضاء فريق إضافيين، تراخيص مستخدمين، أو مستخدمين متزامنين
- **ترقيات الميزات**: ميزات متميزة، تحليلات متقدمة، أو دعم ذي أولوية
- **تمديدات الاستخدام**: تخزين إضافي، استدعاءات API، أو مخصصات عرض النطاق الترددي
- **إضافات الخدمة**: خدمات احترافية، تدريب، أو ساعات استشارة

## الفوائد الرئيسية

- **نماذج تسعير مرنة**: قدم خطط أساسية مع إضافات اختيارية لإنشاء هياكل تسعير متطورة. يمكنك تلبية شرائح العملاء المتنوعة مع ترقيات تنمو مع تغير احتياجات عملائك.
- **تحسين الإيرادات**: عزز متوسط الإيرادات لكل مستخدم (ARPU) من خلال تقديم إضافات ذات صلة. هذا يمكّن من فرص البيع الإضافي الطبيعية مع إضافة العملاء للميزات بمرور الوقت.
- **إدارة مبسطة**: إدارة جميع مكونات التسعير من لوحة تحكم واحدة. يتم تضمين الإضافات تلقائيًا في كل من جلسات الدفع وإدارة الاشتراكات.
- **اختيار العملاء**: السماح للعملاء بتخصيص اشتراكاتهم من خلال اختيار الإضافات التي يحتاجونها فقط، مما يعزز الرضا ويقلل من معدل الانسحاب.

## إنشاء الإضافات

يتم إنشاء الإضافات كمنتجات منفصلة في لوحة تحكم Dodo Payments ثم يتم ربطها بمنتجات الاشتراك الرئيسية الخاصة بك. تتيح لك هذه الفجوة:

- إعادة استخدام الإضافات عبر منتجات اشتراك متعددة
- إدارة التسعير بشكل مستقل
- تتبع أداء الإضافات بشكل منفصل
- تحديث الإضافات دون التأثير على الاشتراكات الأساسية

### تكوين الإضافات

عند إنشاء الإضافات، يمكنك تكوين:

- **التسعير**: تعيين تسعير لمرة واحدة أو متكرر للإضافة
- **دورة الفوترة**: مطابقة فوترة الاشتراك الخاصة بك أو استخدام دورات مختلفة
- **حدود الكمية**: تعيين الحد الأدنى والحد الأقصى من الكميات لكل عميل
- **التوافر**: التحكم في منتجات الاشتراك التي يمكنها استخدام الإضافة
- **إعدادات الضرائب**: تكوين فئات الضرائب المناسبة

### البدء

هل أنت مستعد لتنفيذ الإضافات في عملك الاشتراكي؟ إليك كيفية البدء:

<Steps>
<Step title="خطط لإضافاتك">
حدد الميزات الإضافية، أو الخدمات، أو السعة التي ستفيد عملاءك كإضافات.

اعتبر:
- ماذا يطلب العملاء بشكل متكرر؟
- ما الميزات التي يمكن تحقيق دخل منها بشكل منفصل؟
- ماذا سيخلق مسارات ترقية طبيعية؟
</Step>

<Step title="أنشئ أول إضافة لك">
استخدم لوحة تحكم Dodo Payments أو API لإنشاء أول منتج إضافة لك.

<Card title="دليل لوحة التحكم" icon="box" href="/developer-resources/seat-based-pricing">
اتبع دليلنا خطوة بخطوة لإنشاء الإضافات في لوحة التحكم.
</Card>
</Step>

<Step title="اربط بالإشتراكات">
قم بربط إضافاتك بمنتجات الاشتراك المناسبة حيث يجب أن تكون متاحة.
</Step>

<Step title="اختبر التكامل">
أنشئ جلسات دفع اختبارية مع مجموعات مختلفة من الإضافات للتأكد من أن كل شيء يعمل بشكل صحيح.
</Step>

<Step title="راقب الأداء">
تتبع معدلات اعتماد الإضافات وتأثير الإيرادات لتحسين استراتيجية التسعير الخاصة بك.
</Step>
</Steps>

## أمثلة على التكامل

### جلسات الدفع مع الإضافات
عند إنشاء جلسات الدفع، يمكنك تضمين الإضافات مع كميات مخصصة:

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    {
      product_id: 'your_subscription_id',
      quantity: 1,
      addons: [
        {
          addon_id: 'your_addon_id',
          quantity: 3 // 3 additional seats
        }
      ]
    }
  ],
  // ... other checkout options
});
```

### تغييرات الخطط مع الإضافات
قم بتعديل الاشتراكات الحالية لإضافة، أو إزالة، أو تحديث الإضافات:

```typescript
// Add add-ons to existing subscription
await client.subscriptions.changePlan('sub_123', {
  product_id: 'prod_new',
  quantity: 1,
  proration_billing_mode: 'difference_immediately',
  addons: [
    { addon_id: 'addon_123', quantity: 2 }
  ]
});

// Remove all existing add-ons
await client.subscriptions.changePlan('sub_123', {
  product_id: 'prod_new',
  quantity: 1,
  proration_billing_mode: 'difference_immediately',
  addons: [] // Empty array removes all existing add-ons
});
```

### التسعير الديناميكي
احسب التكاليف الإجمالية ديناميكيًا بناءً على اختيارات الإضافات:

```typescript
function calculateTotalCost(basePrice: number, addons: AddonSelection[]) {
  const addonTotal = addons.reduce((sum, addon) => 
    sum + (addon.price * addon.quantity), 0
  );
  return basePrice + addonTotal;
}
```

## أفضل الممارسات

- **ابدأ ببساطة**: أطلق مع 2-3 إضافات أساسية ووسع الخيارات بناءً على ملاحظات العملاء والاستخدام.
- **حافظ على وضوح التسعير**: تواصل بوضوح تسعير الإضافات وقيمتها، حتى يفهم العملاء ما يحصلون عليه مقابل التكلفة الإضافية.
- **اختبر بدقة**: تحقق من مجموعات الإضافات لضمان بقاء حسابات التسعير دقيقة وسلاسة تدفقات الدفع.

### اعتبارات التصميم

- **عرض قيمة واضح**: يجب أن تحتوي كل إضافة على فائدة واضحة يمكن للعملاء فهمها بسهولة
- **تجميع منطقي**: اجمع الإضافات ذات الصلة معًا في تدفق الدفع الخاص بك
- **كميات مرنة**: السماح للعملاء بتعديل كميات الإضافات حسب الحاجة
- **تسعير شفاف**: عرض التكاليف الإجمالية بوضوح طوال عملية الدفع

<Info>
الإضافات هي وسيلة قوية لإنشاء نماذج تسعير مرنة وقابلة للتوسع تنمو مع عملائك. ابدأ بحالات استخدام بسيطة ووسعها مع تعلم ما هو الأفضل لعملك وعملائك.
</Info>
