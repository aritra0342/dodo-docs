## ما هي الإضافات؟

الإضافات هي منتجات تكميلية يمكن للعملاء شراؤها إلى جانب اشتراكهم الرئيسي. إنها مثالية لـ:

- **الفوترة المبنية على المقاعد**: أعضاء فريق إضافيون، تراخيص مستخدمين، أو مستخدمين متزامنين
- **ترقيات الميزات**: ميزات متميزة، تحليلات متقدمة، أو دعم ذي أولوية
- **تمديدات الاستخدام**: سعة تخزين إضافية، مكالمات واجهة برمجة التطبيقات، أو بدلات عرض النطاق
- **إضافات الخدمة**: خدمات احترافية، تدريب، أو ساعات استشارية

## الفوائد الرئيسية

- **نماذج تسعير مرنة**: قدم خططًا أساسية مع إضافات اختيارية لابتكار هياكل تسعير متقدمة. يمكنك تلبية شرائح عملاء متنوعة من خلال الترقيات التي تنمو مع تغير احتياجات العملاء.
- **تحقيق أقصى استفادة من الإيرادات**: عزز متوسط الإيراد لكل مستخدم (ARPU) عبر تقديم الإضافات ذات الصلة. يتيح ذلك فرصًا طبيعية للترقية عندما يضيف العملاء ميزات مع مرور الوقت.
- **إدارة مبسطة**: ادِر جميع مكوّنات التسعير من لوحة تحكم واحدة. يتم تضمين الإضافات تلقائيًا في جلسات الدفع وإدارة الاشتراكات.
- **اختيار العميل**: اسمح للعملاء بتخصيص اشتراكاتهم من خلال اختيار الإضافات التي يحتاجونها فقط، مما يعزز الرضا ويقلل معدلات التخلّي.

## إنشاء الإضافات

يتم إنشاء الإضافات كمنتجات منفصلة في لوحة تحكم Dodo Payments، ثم يتم ربطها بمنتجات الاشتراك الرئيسية. يتيح هذا الفصل لك:

- إعادة استخدام الإضافات عبر عدة منتجات اشتراك
- إدارة التسعير بشكل مستقل
- تتبع أداء الإضافة بشكل منفصل
- تحديث الإضافات دون التأثير على الاشتراكات الأساسية

### تكوين الإضافات

عند إنشاء الإضافات، يمكنك تكوين ما يلي:

- **التسعير**: حدد تسعيرًا لمرة واحدة أو متكرر للإضافة
- **دورة الفوترة**: يجب أن تتطابق مع دورة الفوترة للاشتراك
- **حدود الكمية**: حدد الحد الأدنى والأقصى للكميات لكل عميل
- **التوافر**: تحكم في منتجات الاشتراك التي يمكنها استخدام الإضافة
- **إعدادات الضرائب**: قم بتكوين فئات الضرائب المناسبة

## أمثلة على التكامل

### جلسات الدفع مع الإضافات
عند إنشاء جلسات الدفع، يمكنك تضمين الإضافات بكميات مخصصة:

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    {
      product_id: 'your_subscription_id',
      quantity: 1,
      addons: [
        {
          addon_id: 'your_addon_id',
          quantity: 3 // 3 additional seats
        }
      ]
    }
  ],
  // ... other checkout options
});
```

### تغييرات الخطة مع الإضافات
قم بتعديل الاشتراكات الحالية لإضافة أو إزالة أو تحديث الإضافات:

```typescript
// Add add-ons to existing subscription
await client.subscriptions.changePlan('sub_123', {
  product_id: 'prod_new',
  quantity: 1,
  proration_billing_mode: 'difference_immediately',
  addons: [
    { addon_id: 'addon_123', quantity: 2 }
  ]
});

// Remove all existing add-ons
await client.subscriptions.changePlan('sub_123', {
  product_id: 'prod_new',
  quantity: 1,
  proration_billing_mode: 'difference_immediately',
  addons: [] // Empty array removes all existing add-ons
});
```

### التسعير الديناميكي
احسب التكاليف الإجمالية ديناميكيًا بناءً على اختيارات الإضافات:

```typescript
function calculateTotalCost(basePrice: number, addons: AddonSelection[]) {
  const addonTotal = addons.reduce((sum, addon) => 
    sum + (addon.price * addon.quantity), 0
  );
  return basePrice + addonTotal;
}
```

## أفضل الممارسات

- **ابدأ ببساطة**: أطلق 2-3 إضافات أساسية ووسع الخيارات بناءً على آراء العملاء والاستخدام.
- **حافظ على وضوح التسعير**: أوضح تسعير الإضافات وقيمتها، حتى يفهم العملاء ما الذي يحصلون عليه مقابل التكلفة الإضافية.
- **اختبر بشكل شامل**: تحقق من مجموعات الإضافات لضمان دقة حسابات التسعير وسلاسة تدفقات الدفع.

### اعتبارات التصميم

- **قيمة واضحة**: يجب أن يكون لكل إضافة ميزة واضحة يمكن للعملاء فهمها بسهولة
- **تجميع منطقي**: جمّع الإضافات ذات الصلة معًا في تدفق الدفع
- **كميات مرنة**: اسمح للعملاء بضبط كميات الإضافات حسب الحاجة
- **تسعير شفاف**: عرض التكاليف الإجمالية بوضوح طوال عملية الدفع

<Info>
الإضافات وسيلة قوية لإنشاء نماذج تسعير مرنة وقابلة للتوسع تنمو مع عملائك. ابدأ بحالات استخدام بسيطة ووسع نطاقها بينما تتعلم ما هو الأفضل لعملك وعملائك.
</Info>
