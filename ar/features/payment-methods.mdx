تدعم Dodo Payments مجموعة واسعة من طرق الدفع لمساعدتك في خدمة العملاء على مستوى العالم. تصبح جميع طرق الدفع المدرجة متاحة تلقائيًا بمجرد التحقق من حساب التاجر الخاص بك وتمكين وضع التشغيل المباشر.

<Info>
**التفعيل التلقائي:** لا حاجة لتكوين إضافي من جانبك. جميع طرق الدفع مُعدة مسبقًا وتديرها Dodo Payments. يمكنك اختيار تقييد الطرق التي تظهر عند الدفع باستخدام تكوين API الموضح أدناه.
</Info>

## طرق الدفع المدعومة

يوفر الجدول التالي مرجعًا سريعًا لجميع طرق الدفع المتاحة، وتوافرها الإقليمي، والقيود الرئيسية:

| طريقة الدفع              | الدول المدعومة       | العملات المدعومة     | تدعم الاشتراك         |
| :----------------------- | :------------------- | :------------------- | :------------------- |
| **بطاقات الائتمان**      | عالمي                | جميع العملات        | نعم                  |
| **بطاقات الخصم**         | عالمي                | جميع العملات        | نعم                  |
| **Apple Pay**            | عالمي (باستثناء IN)  | جميع العملات        | نعم                  |
| **Google Pay**           | عالمي (باستثناء IN)  | جميع العملات        | نعم                  |
| **Amazon Pay**           | عالمي (باستثناء IN)  | USD                  | لا                   |
| **Cash App Pay**         | الولايات المتحدة      | USD                  | لا                   |
| **Afterpay (BNPL)**      | الولايات المتحدة، المملكة المتحدة | USD، GBP             | لا                   |
| **Klarna (BNPL)**        | الولايات المتحدة، أوروبا* | USD، EUR، GBP، DKK، NOK، SEK، CZK، RON، PLN، CHF | لا                   |
| **PayPal**               | عالمي                | EUR، GBP             | لا                   |
| **UPI**                  | الهند                | INR                  | نعم*                 |
| **Rupay (بطاقات ائتمان/خصم)** | الهند                | INR                  | نعم*                 |
| **Bancontact**           | بلجيكا              | EUR                  | لا                   |
| **EPS**                  | النمسا              | EUR                  | لا                   |
| **iDEAL**                | هولندا              | EUR                  | لا                   |
| **Multibanco**           | البرتغال            | EUR                  | لا                   |

<Warning>
**تم إيقاف دعم باي بال:** تم إيقاف دعم طريقة الدفع عبر باي بال مؤقتًا. باي بال غير متاحة حاليًا للمعاملات الجديدة. سنقوم بتحديث هذه الصفحة عندما يتم استعادة دعم باي بال.
</Warning>

<Note>
يجب أن تلبي كل معاملة <strong>الحد الأدنى المطلوب لكل عملة</strong>. بالنسبة للدولار الأمريكي، الحد الأدنى هو <strong>0.50 دولار أمريكي</strong>. تختلف الحدود الدنيا للعملات الأخرى حسب المنطقة.
[انظر العملات المدعومة والحدود الدنيا](/features/adaptive-currency#supported-currencies)
</Note>

<Note>
تتطلب Klarna و Afterpay حدًا أدنى لمبلغ المعاملة قدره <strong>50.01 دولار أمريكي</strong> (أو ما يعادله في العملات المدعومة) للتأهل عند الدفع. المعاملات التي تقل عن هذا الحد لن تكون مؤهلة لهذه الخيارات للدفع بعد الشراء (BNPL).
للحصول على تفاصيل حول الدول المدعومة من Klarna، يرجى الرجوع إلى <a href="#buy-now-pay-later-bnpl">قسم BNPL</a> أدناه.
</Note>

<Note>
<strong>طرق الدفع الهندية:</strong> يتم دعم اشتراكات UPI و Rupay مع تفويضات متوافقة مع RBI. انظر قسم <a href="/features/subscription#subscriptions-with-rbi-compliant-mandates">الاشتراك مع تفويضات متوافقة مع RBI</a> للحصول على تفاصيل مهمة حول حدود التفويض، وتأخيرات المعالجة.
</Note>

## تفاصيل طريقة الدفع

<AccordionGroup>

<Accordion title="بطاقات الائتمان والخصم">
  تقبل Dodo Payments جميع شبكات بطاقات الائتمان والخصم الرئيسية عالميًا. يمكنك قبول:

  - **فيزا** – شبكة موثوقة ومقبولة على نطاق واسع عالميًا
  - **ماستركارد** – شبكة قوية وآمنة ذات انتشار عالمي
  - **أمريكان إكسبريس** – فوائد مميزة وخدمة عملاء قوية
  - **Discover** – مكافآت تنافسية، وجود قوي في الولايات المتحدة
  - **JCB** – شبكة بطاقات آسيوية رائدة مع انتشار متزايد
  - **UnionPay** – مهيمنة في الصين وآسيا، مع قبول عالمي متزايد
  - **Interac** – شبكة الدفع الآمنة في كندا
  - **Cartes Bancaires** – مستخدمة على نطاق واسع في فرنسا
  - **Diners Club** – بطاقة دولية مميزة مع فوائد حصرية
  - **بطاقات محلية كورية** – دعم لشبكات بطاقات محلية كورية
</Accordion>

<Accordion title="المحافظ الرقمية">
  قدم للعملاء تجربة دفع سريعة وآمنة باستخدام المحافظ الرقمية:

  - **Apple Pay** – الدفع بسهولة على iOS و macOS و watchOS
  - **Google Pay** – مدفوعات آمنة على Android والويب وفي المتاجر
  - **PayPal** – مدفوعات عالمية باليورو والجنيه الإسترليني (غير متاحة للاشتراكات). <strong>ملاحظة: دعم باي بال متوقف حاليًا.</strong>
  - **Cash App Pay** – مدفوعات عبر الهاتف المحمول للعملاء في الولايات المتحدة (غير متاحة للاشتراكات)
  - **Amazon Pay** – استخدم بيانات اعتماد أمازون لتجربة دفع بسيطة (غير متاحة للاشتراكات)
</Accordion>

<Accordion title="اشتر الآن وادفع لاحقًا (BNPL)">
  زيادة التحويل من خلال تقديم خيارات تقسيط مرنة:

  - **Klarna** – تمكين المدفوعات المرنة مع خيارات Klarna: الدفع على 4 أقساط، الدفع خلال 30 يومًا، أو تمويل عملية الشراء. تتوفر Klarna في الولايات المتحدة و19 دولة أوروبية، بما في ذلك النمسا وبلجيكا وجمهورية التشيك والدنمارك وفنلندا وفرنسا وألمانيا واليونان وأيرلندا وإيطاليا وهولندا والنرويج وبولندا والبرتغال ورومانيا وإسبانيا والسويد وسويسرا والمملكة المتحدة. تشمل العملات المدعومة الدولار الأمريكي واليورو والجنيه الإسترليني والكرونة الدنماركية والكرونة النرويجية والكرونة السويدية والكرونة التشيكية والليو الروماني والزلوتي البولندي والفرنك السويسري.
  - **Afterpay** – السماح للعملاء بتقسيم المشتريات إلى أربع مدفوعات بدون فوائد. تتوفر Afterpay في الولايات المتحدة والمملكة المتحدة، وتدعم المدفوعات بالدولار الأمريكي والجنيه الإسترليني.
</Accordion>

<Accordion title="UPI و Rupay (الهند)" id="upi-and-rupay-india">
  قبول خيارات الدفع الهندية الرائدة:

  - **UPI** – يقوم العملاء بمسح رمز الاستجابة السريعة أو استخدام التطبيقات (Google Pay و PhonePe و Paytm و CRED) لإجراء تحويلات مصرفية فورية وغير تلامسية
  - **بطاقات Rupay** – مدفوعات ائتمانية وخصمية آمنة عبر شبكة بطاقات محلية في الهند
</Accordion>

<Accordion title="طرق الدفع الأوروبية">
  قدم خيارات أوروبية محلية لزيادة التحويلات:

  - **iDEAL** (هولندا) – مدفوعات مصرفية هولندية فورية عبر الإنترنت
  - **Bancontact** (بلجيكا) – طريقة بطاقة الخصم الموثوقة في بلجيكا
  - **Multibanco** (البرتغال) – طريقة الدفع البرتغالية لتحويلات البنك ومدفوعات أجهزة الصراف الآلي
  - **EPS** (النمسا) – مدفوعات مصرفية مباشرة عبر الإنترنت في النمسا
</Accordion>

</AccordionGroup>

## تكوين طرق الدفع

بينما تتوفر جميع طرق الدفع تلقائيًا بناءً على موقع العميل وعملته، يمكنك اختيار تقييد خيارات الدفع التي تظهر أثناء الدفع باستخدام `allowed_payment_method_types`.

### متى يجب تكوين طرق الدفع

قد ترغب في تكوين طرق الدفع المسموح بها لـ:

- التحكم في خيارات الدفع التي تظهر للعملاء بناءً على متطلبات عملك
- تقييد طرق الدفع لأنواع معينة من المنتجات أو تدفقات المعاملات
- الحفاظ على الاتساق عبر تجارب الدفع المختلفة
- تحسين تجربة الدفع لجمهورك المستهدف

### التنفيذ

<Steps>
<Step title="إضافة معلمة إلى طلب API">
قم بتضمين حقل `allowed_payment_method_types` في طلب إنشاء الدفع أو الاشتراك الخاص بك:

```json
{
  "allowed_payment_method_types": [
    "credit",
    "debit",
    "apple_pay"
  ]
}
```

</Step>

<Step title="اختبار التكوين">
تحقق من أن طرق الدفع المحددة فقط تظهر عند الدفع.

<Tip>
اختبر توفر طرق الدفع مع مواقع العملاء المختلفة والعملات للتأكد من أن التكوين يعمل كما هو متوقع.
</Tip>
</Step>
</Steps>

### أفضل الممارسات

<AccordionGroup>
<Accordion title="السلوك الافتراضي">
إذا لم يتم تحديد `allowed_payment_method_types`، ستكون جميع طرق الدفع المؤهلة متاحة بناءً على موقع العميل، العملة، ونوع المعاملة. استخدم هذه المعلمة فقط عندما تحتاج إلى تقييد طرق الدفع.
</Accordion>

<Accordion title="الاعتبارات الإقليمية">
اعتبر سوقك المستهدف عند اختيار طرق الدفع. يمكن أن يؤدي تضمين خيارات محددة للمنطقة (مثل UPI للهند أو iDEAL لهولندا) إلى تحسين معدلات التحويل بشكل كبير في تلك الأسواق.
</Accordion>

<Accordion title="مراقبة وتحسين">
تتبع طرق الدفع التي يستخدمها عملاؤك بشكل متكرر وضبط طرقك المسموح بها بناءً على أنماط الاستخدام الفعلية وبيانات التحويل.
</Accordion>
</AccordionGroup>

### اعتبارات مهمة

تعتمد توفر طرق الدفع على عدة عوامل:
- **موقع العميل**: بعض طرق الدفع محددة بالمنطقة
- **إعدادات التاجر**: تكوين حسابك وقدراته
- **العملة**: تختلف طرق الدفع المدعومة حسب العملة
- **نوع المعاملة**: بعض الطرق لا تدعم الاشتراكات أو لديها حدود دنيا

<Note>
إضافة طريقة دفع إلى `allowed_payment_method_types` لا يضمن ظهورها - يجب أن تكون مؤهلة أيضًا بناءً على هذه العوامل.
</Note>

## استكشاف الأخطاء وإصلاحها

<AccordionGroup>
<Accordion title="طريقة الدفع لا تظهر عند الدفع">
**الأسباب المحتملة:**
- لم يتم تضمين طريقة الدفع في `allowed_payment_method_types`
- طريقة الدفع غير مدعومة لموقع العميل
- طريقة الدفع لا تدعم عملة المعاملة
- للاشتراكات: طريقة الدفع لا تدعم المدفوعات المتكررة
- مبلغ المعاملة أقل من الحد الأدنى (مثل طرق BNPL)

**الحل:** تحقق من تلبية جميع متطلبات الأهلية وتحقق من تكوين طلب API الخاص بك.
</Accordion>

<Accordion title="ظهور طرق دفع غير متوقعة">
**الأسباب المحتملة:**
- لم يتم تمرير معلمة `allowed_payment_method_types` بشكل صحيح
- قد تكون هناك تكوينات متضاربة في لوحة التحكم الخاصة بالتاجر

**الحل:** راجع حمولة طلب API الخاص بك وتأكد من أن المعلمة مصاغة بشكل صحيح كمصفوفة من السلاسل.
</Accordion>

<Accordion title="طريقة الدفع الإقليمية غير متاحة">
**الأسباب المحتملة:**
- موقع العميل لا يتطابق مع المنطقة المدعومة لطريقة الدفع
- العملة غير مدعومة من قبل طريقة الدفع الإقليمية
- قد لا يكون حساب التاجر الخاص بك مفعلًا لتلك الطريقة المحددة

**الحل:** تحقق من التوافق الإقليمي والعملة، واتصل بالدعم إذا كنت تعتقد أن الطريقة يجب أن تكون متاحة.
</Accordion>

<Accordion title="مشاكل اشتراك طريقة الدفع الهندية">
**فشل رسوم الاشتراك بعد الترقية/التخفيض:**
- قد يتجاوز مبلغ الرسوم الجديد حد التفويض الحالي (حد 15,000 روبية)
- قد يحتاج العميل إلى تحديث طريقة الدفع الخاصة به لإنشاء تفويض جديد

**الحل:** اطلب من العميل تحديث طريقة الدفع الخاصة به أو ضبط مبلغ الاشتراك للبقاء ضمن حدود التفويض.

**الاشتراك معلق بعد بدء الرسوم:**
- قد يكون العميل قد ألغى التفويض خلال نافذة المعالجة التي تستغرق 48 ساعة
- قد يكون العميل قد فشل في تفويض رسوم ذات قيمة عالية (≥15,000 روبية)

**الحل:** راقب أحداث webhook لإلغاء التفويضات وفشل المدفوعات. نفذ منطق إعادة المحاولة أو اطلب من العملاء إعادة إنشاء طريقة الدفع الخاصة بهم.

**تأخير خصم المدفوعات:**
- سلوك طبيعي: الاشتراكات عبر بطاقات الهند و UPI لديها تأخير لمدة 48 ساعة بين بدء الرسوم والخصم الفعلي
- قد يمتد هذا التأخير إلى 2-3 ساعات إضافية اعتمادًا على واجهات برمجة التطبيقات للبنك

**الحل:** هذا سلوك متوقع. لا تقم بتفعيل الفوائد أو الاعتمادات حتى يتم استلام تأكيد الدفع عبر webhooks.
</Accordion>

<Accordion title="تحديث طرق الدفع للاشتراكات">
**متى يجب تحديث طرق الدفع:**
- انتهاء صلاحية بطاقة العميل أو استبدالها
- يرغب العميل في استخدام طريقة دفع مختلفة
- الاشتراك في حالة `on_hold` بسبب فشل الدفع
- يجب تغيير طريقة الدفع لأي سبب

**بالنسبة للاشتراكات النشطة:**
تحديث طريقة الدفع للاشتراك النشط يقوم ببساطة بتحديث طريقة الدفع للتجديدات المستقبلية. لا يتم إنشاء رسوم فورية.

**بالنسبة للاشتراكات المعلقة:**
عندما يكون الاشتراك في حالة `on_hold` (بسبب فشل التجديد أو فشل رسوم تغيير الخطة)، سيؤدي تحديث طريقة الدفع إلى:
- إنشاء رسوم تلقائيًا للديون المتبقية
- إنشاء فاتورة للرسوم
- معالجة الدفع باستخدام طريقة الدفع الجديدة
- إعادة تنشيط الاشتراك إلى حالة `active` عند الدفع الناجح

<CodeGroup>

```javascript Node.js
// Update payment method for active subscription
await client.subscriptions.updatePaymentMethod('sub_123', {
  type: 'new',
  return_url: 'https://example.com/return'
});

// Or use existing payment method
await client.subscriptions.updatePaymentMethod('sub_123', {
  type: 'existing',
  payment_method_id: 'pm_abc123'
});

// For on_hold subscription - automatically creates charge
const response = await client.subscriptions.updatePaymentMethod('sub_123', {
  type: 'new',
  return_url: 'https://example.com/return'
});

if (response.payment_id) {
  // Charge created for remaining dues
  // Redirect customer to response.payment_link
  // Monitor webhooks: payment.succeeded → subscription.active
}
```

```python Python
# Update payment method for active subscription
client.subscriptions.update_payment_method(
    subscription_id="sub_123",
    type="new",
    return_url="https://example.com/return"
)

# Or use existing payment method
client.subscriptions.update_payment_method(
    subscription_id="sub_123",
    type="existing",
    payment_method_id="pm_abc123"
)

# For on_hold subscription - automatically creates charge
response = client.subscriptions.update_payment_method(
    subscription_id="sub_123",
    type="new",
    return_url="https://example.com/return"
)

if response.payment_id:
    # Charge created for remaining dues
    # Redirect customer to response.payment_link
    # Monitor webhooks: payment.succeeded → subscription.active
```

</CodeGroup>

**أحداث webhook:**
- بالنسبة للاشتراكات `on_hold`: ستتلقى `payment.succeeded` متبوعة بـ `subscription.active` بعد تحديث طريقة الدفع بنجاح
- راقب هذه الأحداث لتتبع حالة إعادة تنشيط الاشتراك
