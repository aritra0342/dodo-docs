تدعم Dodo Payments مجموعة واسعة من طرق الدفع لمساعدتك في خدمة العملاء على مستوى العالم. تصبح جميع طرق الدفع المدرجة متاحة تلقائيًا بمجرد التحقق من حساب التاجر الخاص بك وتمكين وضع التشغيل المباشر.

<Info>
**التفعيل التلقائي:** لا حاجة لتكوين إضافي من جانبك. جميع طرق الدفع مُعدة مسبقًا وتديرها Dodo Payments. يمكنك اختيار تقييد الطرق التي تظهر عند الدفع باستخدام تكوين API الموضح أدناه.
</Info>

<CardGroup cols={3}>
{/* LOCKED_PATTERN_72f2419573b9afc0696d865ec5b3104f */}
الطرق المحلية تزيد المبيعات بنسبة 20-30% في الأسواق المستهدفة.
</Card>

{/* LOCKED_PATTERN_ade9a8893e44eb87f3264d28fa873c5d */}
جميع الطرق تتم إدارتها عبر واجهة برمجة تطبيقات واحدة.
</Card>

{/* LOCKED_PATTERN_c72e46011d39da1f870ec199507415a2 */}
يتم عرض الخيارات المناسبة بناءً على الموقع والجهاز.
</Card>
</CardGroup>

<Warning>
**تم إيقاف دعم باي بال:** تم إيقاف دعم طريقة الدفع عبر باي بال مؤقتًا. باي بال غير متاحة حاليًا للمعاملات الجديدة. سنقوم بتحديث هذه الصفحة عندما يتم استعادة دعم باي بال.
</Warning>

<Note>
يجب أن تلبي كل معاملة <strong>الحد الأدنى المطلوب لكل عملة</strong>. بالنسبة للدولار الأمريكي، الحد الأدنى هو <strong>0.50 دولار أمريكي</strong>. تختلف الحدود الدنيا للعملات الأخرى حسب المنطقة.
[انظر العملات المدعومة والحدود الدنيا](/features/adaptive-currency#supported-currencies)
</Note>

<Note>
تتطلب Klarna و Afterpay حدًا أدنى لمبلغ المعاملة قدره <strong>50.01 دولار أمريكي</strong> (أو ما يعادله في العملات المدعومة) للتأهل عند الدفع. المعاملات التي تقل عن هذا الحد لن تكون مؤهلة لهذه الخيارات للدفع بعد الشراء (BNPL).
للحصول على تفاصيل حول الدول المدعومة من Klarna، يرجى الرجوع إلى <a href="#buy-now-pay-later-bnpl">قسم BNPL</a> أدناه.
</Note>

<Info>
**التفعيل التلقائي:** جميع طرق الدفع مهيأة وتديرها Dodo Payments. لا يلزم إعداد إضافي — تصبح متاحة بمجرد التحقق من حسابك وتمكين وضع الإنتاج.
</Info>

## تفاصيل طريقة الدفع

<AccordionGroup>

<Note>
*طرق الدفع الهندية (UPI، Rupay) تدعم الاشتراكات مع تفويضات متوافقة مع RBI. راجع صفحة <a href="/features/payment-methods/india">طرق الدفع في الهند</a> للحصول على تفاصيل حول تأخير المعالجة لمدة 48 ساعة ومتطلبات التفويض.*
</Note>

<Warning>
**تم إيقاف دعم PayPal مؤقتاً:** PayPal غير متاح حالياً للمعاملات الجديدة. سنقوم بتحديث هذه الصفحة عند عودته.
</Warning>

<Accordion title="طرق الدفع الأوروبية">
  قدم خيارات أوروبية محلية لزيادة التحويلات:

بشكل افتراضي، تظهر جميع طرق الدفع المؤهلة تلقائياً. استخدم `allowed_payment_method_types` لتقييد الطرق المعروضة:

<Tabs>
{/* LOCKED_PATTERN_7202147f6fc990d9a8a4b3a4cdc10768 */}

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

session = client.checkout_sessions.create(

| Type | Description |
| :--- | :---------- |
| `credit` | بطاقات الائتمان (Visa، Mastercard، Amex، إلخ) |
| `debit` | بطاقات الخصم |
| `apple_pay` | Apple Pay |
| `google_pay` | Google Pay |
| `amazon_pay` | Amazon Pay |
| `cashapp` | Cash App Pay |
| `klarna` | Klarna للدفع لاحقاً (BNPL) |
| `afterpay_clearpay` | Afterpay / Clearpay |
| `upi_collect` | UPI Collect (رمز QR) |
| `ideal` | iDEAL (هولندا) |
| `bancontact_card` | Bancontact (بلجيكا) |
| `eps` | EPS (النمسا) |
| `multibanco` | Multibanco (البرتغال) |
| `revolut_pay` | RevolutPay |
| `billie` | Billie B2B BNPL |

<Warning>
تأكد دائماً من تضمين `credit` و`debit` كخيارات احتياطية. إذا كانت جميع طرق الدفع المحددة غير متاحة لعميل، فستفشل عملية السداد.
</Warning>

<Tab title="اشتراك">

<AccordionGroup>
{/* LOCKED_PATTERN_fbf2d38e7725fb97384bef01324b59a1 */}
**تحقق من هذه العوامل:**
1. هل تم تضمين الطريقة في `allowed_payment_method_types`؟
2. هل موقع العميل يطابق المنطقة المدعومة للطريقة؟
3. هل العملة المستخدمة في المعاملة مدعومة بواسطة الطريقة؟
4. هل مبلغ المعاملة يفي بالحد الأدنى (مثلاً 50.01 دولارًا لطرق BNPL)؟
5. للاشتراكات: هل تدعم الطريقة المدفوعات المتكررة؟

**الحل:** أزل `allowed_payment_method_types` مؤقتًا لرؤية جميع الطرق المؤهلة، ثم اكتشف سبب المشكلة.
</Accordion>

{/* LOCKED_PATTERN_57385d59bd851fb1e8c26266b489efd4 */}
**الأسباب المحتملة:**
- `allowed_payment_method_types` لا يتم تمريره بشكل صحيح
- المعامل منسق بشكل غير صحيح (يجب أن يكون مصفوفة من السلاسل)

**الحل:** سجّل حمولة طلب واجهة برمجة التطبيقات وتحقق من هيكل المعامل.
</Accordion>

{/* LOCKED_PATTERN_6d1a1b1a59cbe2789bbc9434eb1ec153 */}
**تحقق:**
- عنوان فاتورة العميل يطابق منطقة الطريقة
- هل تم تعيين العملة بشكل صحيح (مثلاً EUR لـ iDEAL)
- تمكين العملة التكيفية إذا كان العميل دوليًا

**الحل:** تحقق من رمز دولة عنوان الفاتورة والعملة في طلب واجهة برمجة التطبيقات.
</Accordion>

<Accordion title="طريقة الدفع لا تظهر عند الدفع">
**الأسباب المحتملة:**
- لم يتم تضمين طريقة الدفع في `allowed_payment_method_types`
- طريقة الدفع غير مدعومة لموقع العميل
- طريقة الدفع لا تدعم عملة المعاملة
- للاشتراكات: طريقة الدفع لا تدعم المدفوعات المتكررة
- مبلغ المعاملة أقل من الحد الأدنى (مثل طرق BNPL)

**الحل:** تحقق من تلبية جميع متطلبات الأهلية وتحقق من تكوين طلب API الخاص بك.
</Accordion>
</AccordionGroup>
