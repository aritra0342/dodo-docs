## ما هي رموز الخصم؟

رموز الخصم هي رموز ترويجية تقلل من إجمالي الطلبات عند الدفع. إنها مثالية لـ:

- **حملات موسمية**: الجمعة السوداء، إطلاق المنتجات، أو anniversaries
- **عروض الاستحواذ**: حوافز للشراء الأول أو مكافآت الإحالة
- **الاحتفاظ**: مكافآت لاستعادة العملاء أو ولاء العملاء الحاليين
- **صفقات B2B**: تسعير متعاقد أو متفاوض عليه عبر رموز خاصة

## الفوائد الرئيسية

- **خصومات مرنة**: نسبة مئوية أو مبلغ ثابت
- **تحكم مستهدف**: تقييد حسب المنتج ودورات الاشتراك
- **حوكمة الحملات**: تواريخ انتهاء وحدود الاستخدام
- **تجربة دفع سلسة**: دعم واجهة المستخدم وAPI عبر جلسات الدفع

## إنشاء رموز الخصم

قم بإنشاء رموز الخصم في لوحة معلومات Dodo Payments الخاصة بك، ثم قم بتطبيقها في الدفع المستضاف أو عبر API.

### إعداد لوحة المعلومات

- **اسم الخصم** (مطلوب): الاسم الداخلي واسم العرض في لوحة المعلومات
- **الرمز** (مطلوب): السلسلة التي يدخلها العملاء عند الدفع
- **النوع والمبلغ** (مطلوب): تعيين قيمة نسبة مئوية أو مبلغ ثابت، أو إنشاء رمز عشوائي باستخدام الزر المقدم
- **تاريخ انتهاء الصلاحية** (اختياري): التاريخ الذي يصبح فيه الرمز غير صالح
- **حد الاستخدام** (اختياري): الحد الأقصى لإجمالي الاستردادات عبر جميع العملاء
- **تقييد المنتج** (اختياري): تحديد قابلية التطبيق للمنتجات المحددة
- **حد دورة الاشتراك** (اختياري): عدد دورات الفوترة التي ينطبق عليها الخصم

<Tip>
استخدم حدود الدورة للأسعار الترويجية على الاشتراكات (مثل: "50% خصم لمدة 3 أشهر").
</Tip>

## تجربة الدفع

1. يدخل المتسوقون الرمز في حقل الدفع.
2. يتم تطبيق الخصومات المؤهلة وتحديث الإجماليات على الفور.

<Info>
في جلسات الدفع، مرر `discount_code` لتطبيق الرمز مسبقًا، واضبط `feature_flags.allow_discount_code` لعرض حقل الإدخال.
</Info>

## حالات الاستخدام الشائعة

- **عروض تقديمية**: ترويجيات إطلاق محدودة الوقت للمنتجات الجديدة
- **الخصومات الكبيرة أو B2B**: خصومات متعاقد عليها لمجموعات منتجات محددة
- **استراتيجيات الاحتفاظ**: رموز استعادة العملاء في سير العمل لمنع التراجع
- **حملات موسمية**: ترويجيات قائمة على العطلات أو الأحداث

## أمثلة على التكامل

### تطبيق خصم في جلسات الدفع
قم بتطبيق خصم مسبقًا وعرض واجهة إدخال الرمز.

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    { product_id: 'prod_abc', quantity: 1 }
  ],
  discount_code: 'BLACKFRIDAY2024',
  customer: { email: 'user@example.com', name: 'Jane Doe' },
  return_url: 'https://yoursite.com/return'
});
```

### تمكين إدخال الخصم دون تطبيق مسبق
دع العملاء يدخلون رمزًا عند الدفع دون تمرير واحد مسبقًا.

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    { product_id: 'prod_abc', quantity: 1 }
  ],
  feature_flags: {
    allow_discount_code: true
  },
  return_url: 'https://yoursite.com/return'
});
```

## أفضل الممارسات

- **اسم واضح**: استخدم رموزًا قابلة للتعرف تتطابق مع أسماء الحملات
- **تحديد الوقت**: أضف تواريخ انتهاء لزيادة الإلحاح ومنع الاستخدام غير الصحيح
- **تحديد النطاق بحكمة**: حصرها على منتجات معينة لتجنب تسرب الهامش
- **التحقق مبكرًا**: تحقق من قابلية تطبيق الرمز قبل تأكيد الدفع
- **مراقبة التأثير**: تتبع الاستخدام والتحويل حسب الحملة

<Info>
رموز الخصم هي أدوات قوية للاستحواذ والاحتفاظ. ابدأ بعروض بسيطة، ذات أسماء واضحة، تحقق بدقة، وكرر بناءً على الأداء.
</Info>
