## What are Customer Wallets?

محافظ العملاء هي حسابات رصيد نقدي تحمل أموالًا حقيقية لمستخدميك. يحصل كل عميل على واحدة تلقائيًا عند إنشاء حسابه. يمكنك استخدام هذه المحافظ لـّ:

- **تخزين الأموال المدفوعة مسبقًا** لمدفوعات الاشتراك المستقبلية
- **معالجة الاستردادات** كرصيد في المحفظة بدلًا من استرداد البطاقة
- **إصدار أرصدة ترويجية** مثل مكافآت الترحيب أو مكافآت الولاء
- **تطبيق أموال المحفظة على الفواتير** تلقائيًا أثناء الفوترة
- **تتبع المعاملات النقدية** بسجل تفصيلي

<Info>
يحصل كل عميل تلقائيًا على محفظة عند إنشاء حسابه. تدعم المحافظ عملاتي USD و INR مع أرصدة نقدية منفصلة لكلٍ منهما.
</Info>

<Warning>
**محافظ العملاء ≠ الفوترة القائمة على الرصيد**

تحتفظ محافظ العملاء بـ **أرصدة نقدية حقيقية** (USD، INR) يمكن تطبيقها على الفواتير ومدفوعات الاشتراك.

إذا كنت تبحث عن تتبع وحدات استخدام افتراضية (مكالمات API، رموز، ساعات معالجة)، راجع [Credit-Based Billing](/features/credit-based-billing) بدلًا من ذلك.
</Warning>

## كيف تعمل

تحمل محافظ العملاء أموالًا يمكن للعملاء تطبيقها على مشترياتهم. عندما يكون لدى العميل فاتورة أو رسم اشتراك متكرر، يتم التحقق من رصيده في المحفظة أولاً. يتم تطبيق أي أموال متاحة تلقائيًا على الفاتورة قبل تحصيل الدفعة من وسيلة الدفع الأساسية.

### الإعداد التلقائي

عندما تنشئ عميلًا جديدًا، تنشئ Dodo Payments تلقائيًا محفظة برصيد صفر. يمكنها استقبال الأموال فورًا عبر واجهة برمجة التطبيقات الخاصة بنا.

### دعم متعدد العملات

يمكن لكل محفظة أن تحمل أرصدة بعملات مختلفة:

<ResponseField name="USD Balance" type="integer">
الرصيد بالدولار الأمريكي (يُخزن بالسنتات)
</ResponseField>

<ResponseField name="INR Balance" type="integer">
الرصيد بالروبية الهندية (يُخزن بالبائس)
</ResponseField>

<Info>
حاليًا، تتوفر الأرصدة بـ **USD** و **INR** فقط. المزيد من العملات قادمة قريبًا.
</Info>

## أمثلة من الواقع

### استرداد إلى المحفظة

عندما يطلب العميل استردادًا، يمكنك إضافة المبلغ إلى رصيده في المحفظة بدلًا من إجراء استرداد تقليدي عبر البطاقة. يحافظ هذا على الأموال ضمن نظامك لعمليات الشراء المستقبلية.

```javascript
async function refundToWallet(customerId, refundAmount, originalPaymentId) {
  await client.customers.wallets.ledgerEntries.create(customerId, {
    amount: refundAmount, // Amount in cents
    currency: 'USD',
    entry_type: 'credit',
    reason: `Refund for payment ${originalPaymentId}`,
    idempotency_key: `refund_${originalPaymentId}`
  });
}
```

### مكافأة ترحيبية / رصيد ترويجي

امنح العملاء الجدد مكافأة ترحيبية نقدية لتشجيع أول عملية شراء لهم.

```javascript
async function addWelcomeBonus(customerId) {
  await client.customers.wallets.ledgerEntries.create(customerId, {
    amount: 1000, // $10.00 promotional balance
    currency: 'USD',
    entry_type: 'credit',
    reason: 'Welcome bonus - $10 promotional balance',
    idempotency_key: `welcome_${customerId}`
  });
}
```

### دفع الاشتراك من المحفظة

خصم الأموال من المحفظة لتغطية رسوم اشتراك أو عملية شراء يدوية.

```javascript
async function deductForPurchase(customerId, purchaseAmount, purchaseId) {
  try {
    await client.customers.wallets.ledgerEntries.create(customerId, {
      amount: purchaseAmount,
      currency: 'USD',
      entry_type: 'debit',
      reason: `Subscription charge - monthly billing`,
      idempotency_key: `charge_${purchaseId}`
    });
  } catch (error) {
    if (error.status === 400) {
      console.log('Insufficient wallet balance');
    }
  }
}
```

## أفضل الممارسات

### منع المعاملات المكررة

استخدم مفاتيح عدم التكرار لضمان عدم إضافة أو خصم الأموال مرتين عن طريق الخطأ لنفس الحدث.

```javascript
async function addFundsSafely(customerId, amount, reason) {
  const idempotencyKey = `${reason}_${customerId}_${Date.now()}`;
  
  try {
    const result = await client.customers.wallets.ledgerEntries.create(customerId, {
      amount: amount,
      currency: 'USD',
      entry_type: 'credit',
      reason: reason,
      idempotency_key: idempotencyKey
    });
    
    return { success: true, wallet: result };
  } catch (error) {
    if (error.status === 409) {
      // Transaction already processed
      return { success: true, wallet: null, duplicate: true };
    }
    
    throw error;
  }
}
```

### التحقق من الأرصدة قبل التحصيل

تأكد من أن العملاء لديهم أموال كافية قبل محاولة معالجة معاملات كبيرة من المحفظة.

```javascript
async function checkBalanceBeforeOperation(customerId, requiredAmount) {
  const wallets = await client.customers.wallets.list(customerId);
  const usdWallet = wallets.items.find(w => w.currency === 'USD');
  
  if (!usdWallet || usdWallet.balance < requiredAmount) {
    throw new Error('Insufficient funds for this operation');
  }
  
  return usdWallet.balance;
}
```

## ما القادم

<Warning>
هذه الميزات مخططة للإصدارات المستقبلية:

- **انتهاء صلاحية الرصيد**: تعيين فترات انتهاء للأموال
