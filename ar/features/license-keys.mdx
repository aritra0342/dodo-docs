<Info>
قم بإصدار مفاتيح فريدة لبرامجك أو سلعك الرقمية، والتحكم في التفعيلات، وإلغاء الوصول عند الحاجة. تتناسب مفاتيح الترخيص بشكل جيد مع الاشتراكات والمشتريات لمرة واحدة.
</Info>

## ما هي مفاتيح الترخيص؟

مفاتيح الترخيص هي رموز فريدة تفوض الوصول إلى منتجك. إنها مثالية لـ:

- **ترخيص البرمجيات**: تطبيقات سطح المكتب، المكونات الإضافية، وواجهات سطر الأوامر
- **التحكم حسب المقعد**: تحديد التفعيلات لكل مستخدم أو جهاز
- **السلع الرقمية**: تأمين التنزيلات، التحديثات، أو الميزات المميزة

## الفوائد الرئيسية

- **التحكم في الوصول**: تحديد تاريخ انتهاء الصلاحية وحدود التفعيل
- **الرؤية التشغيلية**: تتبع الاستخدام، الأجهزة، والتاريخ
- **الأمان والدعم**: تعطيل، إعادة تفعيل، وتدقيق بسرعة

## إنشاء مفاتيح الترخيص

<Steps>
<Step title="إنشاء مفتاح ترخيص">
    انتقل إلى لوحة معلومات مدفوعات دودو وافتح قسم مفاتيح الترخيص. اختر الخيار لإنشاء مفتاح ترخيص جديد.
</Step>

<Step title="تكوين إعدادات مفتاح الترخيص">
    حدد كيفية إصدار واستخدام مفاتيح الترخيص الخاصة بك عن طريق تحديد هذه الحقول:
    
    - **تاريخ انتهاء صلاحية مفتاح الترخيص**: حدد المدة التي يبقى فيها المفتاح صالحًا. اختر مدة محددة (على سبيل المثال، 30 يومًا أو سنة واحدة)، أو اختر "بدون انتهاء" لمفتاح دائم.
    - **حد التفعيل**: حدد الحد الأقصى لعدد التفعيلات المتزامنة لكل مفتاح. استخدم رقمًا محددًا لسيناريوهات مثل ترخيص المستخدم الفردي (1) أو ترخيص الفريق (5)، أو اختر "غير محدود" لعدم وجود حد للتفعيل.
    - **تعليمات التفعيل**: قدم خطوات تفعيل واضحة لعملائك، والتي سيتم أيضًا إرسالها عبر البريد الإلكتروني مع مفتاح الترخيص. أمثلة: "الصق المفتاح في الإعدادات → الترخيص" أو "قم بتشغيل: `mycli activate <key>`".
</Step>

<Step title="مراجعة وحفظ">
    تحقق من جميع التفاصيل المدخلة. عند الانتهاء، احفظ تكوين مفتاح الترخيص لإنهاء إنشاء المفتاح.
    <Tip>
        استخدم تعليمات واقعية وضمن روابط الدعم لتقليل تذاكر التفعيل.
    </Tip>
</Step>
</Steps>

## إدارة ومراقبة

يوفر **قسم مفاتيح الترخيص** عرضًا شاملاً لجميع أنشطة مفاتيح الترخيص ويمكّنك من إدارة ومراقبة التراخيص بفعالية.

### معلومات المفتاح

عرض معلومات مفصلة لكل مفتاح ترخيص:

- **تفاصيل المفتاح**: قيمة مفتاح الترخيص، اسم المنتج المرتبط، تفاصيل العميل (customer_id)، وتاريخ الشراء
- **نشاط الاستخدام**: عدد التفعيلات، تواريخ وأوقات التفعيل، والحالة (نشط، منتهي، معطل)
- **الانتهاء والحدود**: تاريخ انتهاء صلاحية المفتاح، عدد التفعيلات المتبقية، وحالات التفعيل الحالية

<Frame>
    <img src="/images/license-keys/3.webp" alt="تفاصيل مفتاح الترخيص"/>
</Frame>

### الإجراءات المتاحة

يمكنك تنفيذ الإجراءات التالية على مفاتيح الترخيص:

- **تعطيل مفتاح الترخيص**: تعطيل المفتاح على الفور لمنع المزيد من الاستخدام
- **تمكين المفتاح**: إعادة تفعيل مفتاح الترخيص المعطل سابقًا
- **عرض حالات التفعيل**: رؤية جميع حالات التفعيل المرتبطة بمفتاح ترخيص معين

<Frame>
    <img src="/images/license-keys/2.webp" alt="إجراءات مفتاح الترخيص"/>
</Frame>

### الفوائد

- تتبع نشاط مفتاح الترخيص في الوقت الفعلي
- تحديد الاستخدام غير المصرح به أو التفعيلات غير المصرح بها
- تبسيط دعم العملاء من خلال توفير سجلات استخدام مفصلة في حالة وجود أي تناقض

</Info>

## أمثلة على التكامل

### تفعيل ترخيص

<Note>
لا تتطلب هذه النقاط نهاية مفتاح API ويمكن استدعاؤها مباشرة من تطبيقات العميل الخاصة بك.
</Note>

<CodeGroup>

```typescript TypeScript/Node.js
import DodoPayments from 'dodopayments';

// No API key needed for public license endpoints
const client = new DodoPayments();

const response = await client.licenses.activate({ 
  license_key: 'license_key', 
  name: 'Device Name' 
});

console.log(response.id);
```

```bash cURL
curl -X POST https://api.dodopayments.com/licenses/activate \
  -H "Content-Type: application/json" \
  -d '{
    "license_key": "your_license_key",
    "name": "Device Name"
  }'
```

</CodeGroup>

### التحقق من ترخيص

<CodeGroup>

```typescript TypeScript/Node.js
import DodoPayments from 'dodopayments';

// No API key needed for public license endpoints
const client = new DodoPayments();

const response = await client.licenses.validate({ 
  license_key: '2b1f8e2d-c41e-4e8f-b2d3-d9fd61c38f43' 
});

console.log(response.valid);
```

```bash cURL
curl -X POST https://api.dodopayments.com/licenses/validate \
  -H "Content-Type: application/json" \
  -d '{
    "license_key": "2b1f8e2d-c41e-4e8f-b2d3-d9fd61c38f43"
  }'
```

</CodeGroup>

## أفضل الممارسات

- **اجعل الحدود واضحة**: اختر افتراضيات معقولة لتواريخ الانتهاء والتفعيلات
- **إرشاد المستخدمين**: قدم تعليمات تفعيل دقيقة ووثائق ذاتية الخدمة
- **أتمتة الفحوصات**: تحقق من المفاتيح على الخادم قبل منح الوصول
- **مراقبة الأحداث**: استخدم الويب هوكس لاكتشاف الإساءة وأتمتة الإلغاءات
