## ميزات جديدة 🚀

1. **`subscription.updated` ويب هوك**  
   حدث ويب هوك جديد يتم تفعيله عند كل تحديث لكائن الاشتراك، مما يمنحك إشعارات في الوقت الفعلي حول أي تغييرات في اشتراكاتك.
   
   استخدم هذا الويب هوك لـ:
   - الحصول على إشعارات فورية عند تغيير تفاصيل الاشتراك
   - الحفاظ على حالة تطبيقك متزامنة مع مدفوعات دودي
   - التفاعل مع تحديثات الاشتراك دون الحاجة لاستطلاع واجهة برمجة التطبيقات
   
   <Tip>
   اشترك في `subscription.updated` للقضاء على الحاجة للاستطلاع والحفاظ على نظامك متزامنًا مع تغييرات الاشتراك في الوقت الفعلي.
   </Tip>
   
   تعرف على المزيد: [ويب هوك الاشتراك](/developer-resources/webhooks/intents/subscription)

2. **دعم العنوان المختصر في عملية الدفع**  

   <Frame>
   <img src="/images/changelog/minimal-address.png" alt="دعم العنوان المختصر في عملية الدفع" />
   </Frame>

   قلل من احتكاك عملية الدفع مع وضع جمع العنوان المختصر الجديد. بدلاً من جمع تفاصيل العنوان الكاملة، يقوم وضع العنوان المختصر بجمع:
   
   - **البلد**: مطلوب دائمًا لتحديد الضرائب
   - **الرمز البريدي**: يتم جمعه فقط في المناطق التي يكون فيها ضروريًا لحساب ضريبة المبيعات أو VAT أو GST
   
   <Check>
   النتيجة: إكمال عملية الدفع بشكل أسرع بكثير مع عدد أقل من حقول النموذج للعملاء.
   </Check>
   
   **تفعيل العنوان المختصر:**
   
   <Tabs>
   <Tab title="جلسات الدفع">

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [{ product_id: 'prod_123', quantity: 1 }],
     minimal_address: true
   });
   ```

   </Tab>
   
   <Tab title="روابط الدفع الثابتة">
   أضف `minimalAddress=true` إلى عنوان URL الخاص برابط الدفع الخاص بك:
   

   ```
   https://checkout.dodopayments.com/buy/prod_123?minimalAddress=true
   ```

   </Tab>
   </Tabs>
   
   <Info>
   بشكل افتراضي، تستمر عملية الدفع في جمع العناوين الكاملة. سنقوم تدريجيًا بتحويل جميع روابط الدفع إلى وضع العنوان المختصر. قم بتفعيل ميزة العلم الآن لبدء الاستفادة من تقليل احتكاك عملية الدفع.
   </Info>

3. **مفاتيح API للقراءة فقط**  
   أنشئ مفاتيح API بصلاحيات القراءة فقط لدمج أكثر أمانًا. عند إنشاء مفتاح API جديد، قم بإلغاء تحديد **تفعيل الوصول للكتابة** لإنشاء مفتاح للقراءة فقط.
   
   يمكن لمفاتيح القراءة فقط:
   - جلب بيانات الدفع والاشتراك والعميل
   - سرد المنتجات واسترجاع التفاصيل
   - الوصول إلى التقارير والتحليلات
   
   لا يمكن لمفاتيح القراءة فقط **أن**:
   - إنشاء أو تعديل المدفوعات
   - تحديث الاشتراكات
   - تغيير تكوينات المنتجات
   
   <Tip>
   قم بإلغاء تحديد "تفعيل الوصول للكتابة" لدمج لوحات التحكم، وأدوات التحليل، وأي نظام يحتاج فقط إلى عرض البيانات دون إجراء تغييرات.
   </Tip>

4. **اشتراكات UPI AutoPay وبطاقات RuPay - الإطلاق الكامل**  
   أصبحت اشتراكات UPI AutoPay وبطاقات RuPay متاحة الآن لجميع الأعمال. وهذا يمكّن المدفوعات المتكررة للعملاء الهنود مع:
   
   - **UPI AutoPay**: رسوم متكررة تلقائية عبر تفويض UPI
   - **بطاقات RuPay**: دعم الاشتراكات لشبكة البطاقات المحلية في الهند
   
   <Check>
   النتيجة: توسيع خيارات دفع الاشتراك للسوق الهندية مع معالجة تفويض متوافقة مع RBI.
   </Check>
   
   تعرف على المزيد: [الاشتراكات مع تفويضات متوافقة مع RBI](/features/subscription#subscriptions-with-rbi-compliant-mandates)

5. **واجهة برمجة التطبيقات لمعاينة تغيير الخطة**  
   نقطة نهاية جديدة لواجهة برمجة التطبيقات لمعاينة تغييرات خطة الاشتراك قبل الالتزام بها. احصل على المبلغ الدقيق للرسوم وشاهد كيف ستبدو الاشتراك بعد التغيير - دون إجراء أي تعديلات فعلية.
   

   ```javascript
   const preview = await client.subscriptions.previewChangePlan('sub_123', {
     product_id: 'prod_pro_plan',
     proration_billing_mode: 'prorated_immediately',
     quantity: 1
   });
   
   console.log('Charge amount:', preview.immediate_charge.summary);
   console.log('New plan details:', preview.new_plan);
   ```

   
   استخدم هذا لـ:
   - إظهار للعملاء بالضبط ما سيدفعونه قبل تأكيد الترقيات/الخفض
   - بناء حاسبات الأسعار وأدوات مقارنة الخطط
   - تقليل تذاكر الدعم من خلال تحسين شفافية الأسعار
   
   تعرف على المزيد: [واجهة برمجة التطبيقات لمعاينة تغيير الخطة](/api-reference/subscriptions/preview-change-plan)
