## ميزات جديدة 🚀

1. **الدفع المختلط في عربة التسوق: منتجات الدفع لمرة واحدة والاشتراكات في عربة واحدة**  
   يدعم Checkout الآن الجمع بين منتجات الدفع لمرة واحدة ومنتجات الاشتراك في سلة واحدة، مما يفتح حالات استخدام جديدة قوية للفوترة.
   
   <Frame>
   <img src="/images/changelog/mixed-checkout.png" alt="Mixed checkout with one-time and subscription products" />
   </Frame>
   
   ### ما الذي يمكّنه هذا
   
   - **رسوم الإعداد + الاشتراكات**: تحصيل رسوم إعداد لمرة واحدة إلى جانب اشتراك متكرر في عملية دفع واحدة
   - **حزم بميزات متكررة**: بيع منتج (لمرة واحدة) مرفق بوصول إلى خدمة مستمرة (اشتراك)
   - **مجموعات دورات + العضويات**: عرض شراء دورة مع اشتراك عضوية مجتمعية اختياري
   - **الوصول مدى الحياة + الاشتراكات الإضافية**: الجمع بين الوصول إلى المنتج مدى الحياة وميزات متميزة متكررة اختيارية
   
   ### كيف يعمل
   
   فقط أضف كل من المنتجات ذات الدفع لمرة واحدة والاشتراكات إلى جلسة Checkout الخاصة بك:
   

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [
       // One-time setup fee
       { product_id: 'prod_setup_fee', quantity: 1 },
       // Monthly subscription
       { product_id: 'prod_monthly_plan', quantity: 1 }
     ],
     customer: {
       email: 'customer@example.com',
       name: 'Jane Doe'
     },
     return_url: 'https://yourapp.com/success'
   });
   ```

   
   <Check>
   النتيجة: تدفق دفع واحد يُعالج الدفع لمرة واحدة ويبدأ الاشتراك — مما يقلل الاحتكاك ويزيد معدلات التحويل.
   </Check>
   
   <Info>
   عند خلط أنواع المنتجات، يتم تحصيل الدفع لمرة واحدة فوراً بينما يتبع الاشتراك دورة الفوترة التي تم تكوينها (بما في ذلك أي فترة تجريبية قمت بتعيينها).
   </Info>
   
   Learn more: [Checkout Sessions](/developer-resources/checkout-session)

2. **بوابة العملاء الموحدة**  
   
   بوابة عملاء مركزية جديدة على [customer.dodopayments.com](https://customer.dodopayments.com) حيث يمكن للعملاء عرض وإدارة جميع مشترياتهم واشتراكاتهم عبر مختلف الشركات على Dodo Payments.
   
   <Frame>
   <img src="/images/changelog/unified-customer-portal.png" alt="Unified Customer Portal" />
   </Frame>
   
   ### ميزات البوابة
   
   - **رؤية عبر الشركات**: عرض جميع المشتريات والاشتراكات من أي شركة تستخدم Dodo Payments في مكان واحد
   - **إدارة الاشتراكات المركزية**: إدارة الاشتراكات عبر تجار مختلفين من لوحة تحكم واحدة
   - **سجل فواتير موحد**: الوصول إلى الفواتير وتاريخ المدفوعات من جميع مشترياتك
   - **تجربة تسجيل دخول فردي**: سجّل الدخول مرة واحدة باستخدام بريدك الإلكتروني للوصول إلى المشتريات من جميع تجار Dodo Payments
   
   ### كيف يعمل
   
   يقوم العملاء ببساطة بزيارة [customer.dodopayments.com](https://customer.dodopayments.com) وإدخال عنوان بريدهم الإلكتروني. سيتلقون رابط تسجيل دخول آمن يمنحهم إمكانية الوصول إلى جميع مشترياتهم عبر منظومة Dodo Payments.
   
   <Tip>
   تكمل بوابة العملاء الموحدة البوابات المخصصة لكل عمل. لا يزال بإمكان العملاء الوصول إلى بوابات التجار الفردية لتجربة ذات علامة تجارية، أو استخدام البوابة الموحدة للحصول على رؤية موحدة.
   </Tip>
   
   <Note>
   تستخدم البوابة الموحدة نفس المصادقة القائمة على البريد الإلكتروني مثل البوابات الخاصة بكل عمل، مما يضمن تجربة متسقة وآمنة.
   </Note>

3. **معرف الفاتورة في استجابة الدفع**  
   تتضمن استجابة الدفع الآن حقل `invoice_id`، مما يسهل مطابقة المدفوعات مع الفواتير والتكامل مع أنظمة المحاسبة.
   

   ```json
   {
     "payment_id": "pay_2IjeQm4hqU6RA4Z4kwDee",
     "status": "succeeded",
     "invoice_id": "inv_2IsUnWGtRKFLxk7xAQeyt",
     "amount": 9900,
     "currency": "USD",
     // ... other fields
   }
   ```

   
   ### حالات الاستخدام
   
   - **التكامل المحاسبي**: ربط المدفوعات بالفواتير مباشرة في برنامج المحاسبة الخاص بك
   - **دعم العملاء**: العثور بسرعة على الفاتورة المرتبطة بمدفوع عند معالجة طلبات الدعم
   - **التسوية**: تبسيط التسوية المالية من خلال مطابقة المدفوعات بالفواتير برمجياً
   - **الامتثال**: الحفاظ على سجلات دقيقة تربط المدفوعات بالفواتير الضريبية المقابلة لها
   
   <Info>
   يستخدم الحقل `invoice_id` معرفات فواتير خاصة بالهند عند توفرها، مما يضمن الالتزام بمتطلبات الفوترة المحلية.
   </Info>
   
   Learn more: [Get Payment Detail API](/api-reference/payments/get-payments-1)
