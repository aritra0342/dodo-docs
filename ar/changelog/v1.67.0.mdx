## ميزات جديدة 🚀

1. **الدفع المختلط: منتجات الدفع لمرة واحدة والاشتراكات في سلة واحدة**  
   يدعم الدفع الآن دمج كل من منتجات الدفع لمرة واحدة ومنتجات الاشتراك في سلة واحدة، مما يفتح حالات استخدام جديدة قوية للفوترة.
   
   <Frame>
   <img src="/images/changelog/mixed-checkout.png" alt="الدفع المختلط مع منتجات الدفع لمرة واحدة والاشتراكات" />
   </Frame>
   
   ### ما الذي يتيحه هذا
   
   - **رسوم الإعداد + الاشتراكات**: فرض رسوم إعداد لمرة واحدة بجانب اشتراك متكرر في عملية دفع واحدة
   - **حزم مع مكونات متكررة**: بيع منتج (لمرة واحدة) مع الوصول إلى خدمة مستمرة (اشتراك)
   - **دورات + مجموعات العضوية**: عرض شراء دورة مع اشتراك عضوية مجتمع اختياري
   - **اشتراكات مدى الحياة + إضافات**: دمج الوصول إلى المنتج مدى الحياة مع ميزات متميزة متكررة اختيارية
   
   ### كيف يعمل
   
   ببساطة أضف كل من منتجات الدفع لمرة واحدة ومنتجات الاشتراك إلى جلسة الدفع الخاصة بك:
   

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [
       // One-time setup fee
       { product_id: 'prod_setup_fee', quantity: 1 },
       // Monthly subscription
       { product_id: 'prod_monthly_plan', quantity: 1 }
     ],
     customer: {
       email: 'customer@example.com',
       name: 'Jane Doe'
     },
     return_url: 'https://yourapp.com/success'
   });
   ```

   
   <Check>
   النتيجة: تدفق دفع واحد يعالج الدفع لمرة واحدة ويبدأ الاشتراك — مما يقلل الاحتكاك ويزيد من معدلات التحويل.
   </Check>
   
   <Info>
   عند خلط أنواع المنتجات، يتم فرض الدفع لمرة واحدة على الفور بينما يتبع الاشتراك دورة الفوترة المكونة (بما في ذلك أي فترة تجريبية قمت بتعيينها).
   </Info>
   
   تعرف على المزيد: [جلسات الدفع](/developer-resources/checkout-session)

2. **بوابة العملاء الموحدة**  
   
   بوابة عملاء مركزية جديدة في [customer.dodopayments.com](https://customer.dodopayments.com) حيث يمكن للعملاء عرض وإدارة جميع مشترياتهم واشتراكاتهم عبر مختلف الأعمال على Dodo Payments.
   
   <Frame>
   <img src="/images/changelog/unified-customer-portal.png" alt="بوابة العملاء الموحدة" />
   </Frame>
   
   ### ميزات البوابة
   
   - **رؤية عبر الأعمال**: عرض جميع المشتريات والاشتراكات من أي عمل يستخدم Dodo Payments في مكان واحد
   - **إدارة اشتراكات مركزية**: إدارة الاشتراكات عبر مختلف التجار من لوحة تحكم واحدة
   - **تاريخ فوترة موحد**: الوصول إلى الفواتير وتاريخ الدفع من جميع مشترياتك
   - **تجربة تسجيل دخول موحدة**: تسجيل الدخول مرة واحدة باستخدام بريدك الإلكتروني للوصول إلى المشتريات من جميع تجار Dodo Payments
   
   ### كيف يعمل
   
   يقوم العملاء ببساطة بزيارة [customer.dodopayments.com](https://customer.dodopayments.com) وإدخال عنوان بريدهم الإلكتروني. سيتلقون رابط تسجيل دخول آمن يمنحهم الوصول إلى جميع مشترياتهم عبر نظام Dodo Payments.
   
   <Tip>
   تكمل بوابة العملاء الموحدة بوابات العملاء الخاصة بالأعمال. لا يزال بإمكان العملاء الوصول إلى بوابات التجار الفردية لتجربة مميزة، أو استخدام البوابة الموحدة لرؤية مجمعة.
   </Tip>
   
   <Note>
   تستخدم البوابة الموحدة نفس مصادقة البريد الإلكتروني مثل البوابات الخاصة بالأعمال، مما يضمن تجربة متسقة وآمنة.
   </Note>

3. **معرف الفاتورة في استجابة الدفع**  
   تتضمن استجابة الدفع الآن حقل `invoice_id`، مما يسهل تسوية المدفوعات مع الفواتير والتكامل مع أنظمة المحاسبة.
   

   ```json
   {
     "payment_id": "pay_2IjeQm4hqU6RA4Z4kwDee",
     "status": "succeeded",
     "invoice_id": "inv_2IsUnWGtRKFLxk7xAQeyt",
     "amount": 9900,
     "currency": "USD",
     // ... other fields
   }
   ```

   
   ### حالات الاستخدام
   
   - **تكامل المحاسبة**: ربط المدفوعات مباشرة بالفواتير في برنامج المحاسبة الخاص بك
   - **دعم العملاء**: العثور بسرعة على الفاتورة المرتبطة بدفع عند التعامل مع طلبات الدعم
   - **التسوية**: تبسيط التسوية المالية من خلال مطابقة المدفوعات بالفواتير برمجياً
   - **الامتثال**: الحفاظ على سجلات دقيقة تربط المدفوعات بالفواتير الضريبية المقابلة
   
   <Info>
   يستخدم حقل `invoice_id` معرفات الفواتير الخاصة بالهند عند توفرها، مما يضمن الامتثال لمتطلبات الفوترة المحلية.
   </Info>
   
   تعرف على المزيد: [احصل على تفاصيل الدفع API](/api-reference/payments/get-payments-1)
