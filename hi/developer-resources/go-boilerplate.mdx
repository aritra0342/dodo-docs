## अवलोकन

गो बॉयलरप्लेट डोडो पेमेंट्स को आपके गो बैकएंड के साथ एकीकृत करने के लिए एक उत्पादन-तैयार प्रारंभिक बिंदु प्रदान करता है। इस टेम्पलेट में चेकआउट सत्र प्रबंधन, वेबहुक सत्यापन, ग्राहक पोर्टल एकीकरण शामिल है, और यह गो के सर्वोत्तम प्रथाओं का पालन करता है ताकि आप जल्दी से भुगतान स्वीकार करना शुरू कर सकें।

<Info>
यह बॉयलरप्लेट Go 1.21+ को clean architecture पैटर्न (`cmd`, `internal`, `templates`), HTML टेम्पलेट्स, और `dodopayments-go` SDK के साथ निर्बाध API एकीकरण के लिए उपयोग करता है.
</Info>

### विशेषताएँ

- **Quick Setup** - 5 मिनट से कम में शुरुआत करें
- **Payment Integration** - `dodopayments-go` SDK का उपयोग करके पूर्व-निर्धारित चेकआउट फ्लो
- **Modern UI** - HTML टेम्पलेट्स के साथ स्वच्छ, डार्क-थीम्ड प्राइसिंग पृष्ठ
- **Webhook Handling** - भुगतान घटनाओं को सुरक्षित रूप से सत्यापित और संसाधित करें
- **Customer Portal** - स्व-सेवा सब्सक्रिप्शन प्रबंधन
- **Go Best Practices** - clean architecture के साथ `cmd`, `internal`, और `templates`
- **Pre-filled Checkout** - उपयोगकर्ता अनुभव को बेहतर बनाने के लिए ग्राहक डेटा भेजने का प्रदर्शन

## पूर्वापेक्षाएँ

शुरू करने से पहले, सुनिश्चित करें कि आपके पास है:

- **गो 1.21+**
- **डोडो पेमेंट्स खाता** (डैशबोर्ड से API और वेबहुक कुंजी तक पहुँचने के लिए)

## परियोजना संरचना

```text
go-boilerplate/
├── cmd/
│   └── server/             # Application entry point
├── internal/
│   ├── api/                # API handlers (Checkout, Portal, Webhook)
│   ├── core/               # Configuration and system core
│   └── lib/                # Shared logic (Products, Customer utils)
├── templates/              # HTML templates
├── Makefile                # Build and run commands
├── go.mod                  # Go module definition
├── go.sum                  # Dependency checksums
├── .env.example            # Environment template
└── README.md
```

## API एंडपॉइंट्स

बॉयलरप्लेट में निम्नलिखित पूर्व-कॉन्फ़िगर किए गए एंडपॉइंट शामिल हैं:

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/` | GET | उत्पाद सूची के साथ प्राइसिंग पृष्ठ |
| `/api/checkout` | POST | एक नया चेकआउट सेशन बनाएँ |
| `/api/webhook` | POST | Dodo Payments वेबहुक को संभालें |
| `/api/customer-portal` | POST | ग्राहक पोर्टल URL जनरेट करें |

## अनुकूलन

### उत्पाद जानकारी अपडेट करें

`internal/lib/products.go` को संशोधित करें ताकि आप बदल सकें:
- उत्पाद आईडी (अपने Dodo डैशबोर्ड से)
- मूल्य निर्धारण
- विशेषताएँ
- विवरण

```go
var Products = []Product{

        Price:       9999,

            "Access to basic features",
            "Email support",

        },

```

### ग्राहक डेटा पूर्व-भरे

`templates/index.html` में, हार्डकोड किए गए ग्राहक डेटा को अपने असली उपयोगकर्ता डेटा से बदलें:


```javascript
const customerData = {
    name: "John Doe",       // Replace with actual logged-in user's name
    email: "john@example.com"  // Replace with actual logged-in user's email
};
```

एक उत्पादन ऐप में, आप अपने प्रमाणीकरण प्रणाली से इन मानों को गतिशील रूप से इंजेक्ट करेंगे।

## वेबहुक घटनाएँ

यह बॉयलरप्लेट `internal/api/webhook.go` में वेबहुक घटनाओं को संभालने का प्रदर्शन करता है। समर्थित घटनाओं में शामिल हैं:


| Event | Description |
|-------|-------------|
| `subscription.active` | जब एक सदस्यता सक्रिय हो जाती है तब ट्रिगर होता है |
| `payment.succeeded` | जब कोई भुगतान सफल होता है तब ट्रिगर होता है |

अपने व्यवसाय की लॉजिक को वेबहुक हैंडलर के अंदर जोड़ें ताकि:
- अपने डेटाबेस में उपयोगकर्ता अनुमतियों को अपडेट करें
- पुष्टि ईमेल भेजें
- डिजिटल उत्पादों तक पहुँच प्रदान करें
- विश्लेषण और मैट्रिक्स को ट्रैक करें

## स्थानीय रूप से वेबहुक का परीक्षण करना

स्थानीय विकास के लिए, अपने स्थानीय सर्वर को उजागर करने के लिए [ngrok](https://ngrok.com/) जैसे उपकरणों का उपयोग करें:

```bash
ngrok http 8000
```

अपने [डोडो पेमेंट्स डैशबोर्ड](https://app.dodopayments.com/developer/webhooks) में वेबहुक URL अपडेट करें:

```
https://your-ngrok-url.ngrok.io/api/webhook
```

## तैनाती

### उत्पादन के लिए निर्माण करें

```bash
make build
```

या मैन्युअल रूप से:

```bash
go build -o bin/server cmd/server/main.go
./bin/server
```

### वेरसेल पर तैनात करें

[

![Vercel के साथ तैनात करें](https://vercel.com/button)

](https://vercel.com/new/clone?repository-url=https://github.com/dodopayments/go-boilerplate)

### डॉकर

Create a `Dockerfile`:

```dockerfile
FROM golang:1.21-alpine AS builder

WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download

COPY . .
RUN go build -o bin/server cmd/server/main.go

FROM alpine:latest
WORKDIR /app
COPY --from=builder /app/bin/server .
COPY --from=builder /app/templates ./templates

EXPOSE 8000
CMD ["./server"]
```

बनाएं और चलाएँ:

```bash
docker build -t go-dodo .
docker run -p 8000:8000 --env-file .env go-dodo
```

### उत्पादन पर विचार

<Warning>
प्रोडक्शन में डिप्लॉय करने से पहले:
- `DODO_PAYMENTS_ENVIRONMENT` को `live_mode` में बदलें
- डैशबोर्ड से प्रोडक्शन API कीज़ का उपयोग करें
- वेबहुक URL को अपने प्रोडक्शन डोमेन पर अपडेट करें
- सभी एंडपॉइंट्स के लिए HTTPS सक्षम करें
</Warning>
