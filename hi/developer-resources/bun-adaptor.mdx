## Installation

<Steps>
<Step title="Install the package">
अपने प्रोजेक्ट रूट में निम्न कमांड चलाएँ:

```bash
bun add @dodopayments/bun
```

</Step>
<Step title="Set up environment variables">
अपने प्रोजेक्ट रूट में <code>.env</code> फ़ाइल बनाएं:

```env expandable
DODO_PAYMENTS_API_KEY=your-api-key
DODO_PAYMENTS_WEBHOOK_KEY=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT="test_mode" or "live_mode"
DODO_PAYMENTS_RETURN_URL=your-return-url
```

<Warning>
अपनी <code>.env</code> फ़ाइल या सीक्रेट्स को वर्शन कंट्रोल में कभी कमिट न करें।
</Warning>
</Step>
</Steps>

## Route Handler Examples

<Info>
सभी उदाहरण मानते हैं कि आप Bun के नेटिव सर्वर का उपयोग <code>Bun.serve()</code> के साथ कर रहे हैं।
</Info>

<Tabs>
<Tab title="Checkout Handler">
  <Info>
  अपने Bun सर्वर में Dodo Payments चेकआउट को एकीकृत करने के लिए इस हैंडलर का उपयोग करें। स्थिर (GET), डायनामिक (POST), और सेशन (POST) भुगतान प्रवाह का समर्थन करता है।
  </Info>
  <CodeGroup>

    ```typescript Bun Server Handler expandable
    import { Checkout } from '@dodopayments/bun';

    const staticCheckoutHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "static"
    });

    const sessionCheckoutHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "session"
    });

    const dynamicCheckoutHandler = Checkout({
        bearerToken: process.env.DODO_PAYMENTS_API_KEY,
        returnUrl: process.env.DODO_PAYMENTS_RETURN_URL,
        environment: process.env.DODO_PAYMENTS_ENVIRONMENT,
        type: "dynamic"
    });

    Bun.serve({
      port: 3000,
      fetch(request) {
        const url = new URL(request.url);
        
        if (url.pathname === "/api/checkout") {
          if (request.method === "GET") {
            return staticCheckoutHandler(request);
          }
          if (request.method === "POST") {
            return sessionCheckoutHandler(request);
            // or return dynamicCheckoutHandler(request);
          }
        }
        
        return new Response("Not Found", { status: 404 });
      },
    });
    ```

  </CodeGroup>

## Checkout Route Handler

<Info>
Dodo Payments आपके वेबसाइट में भुगतान एकीकृत करने के लिए तीन प्रकार के भुगतान प्रवाह का समर्थन करता है; यह एडाप्टर सभी प्रकारों का समर्थन करता है।
</Info>

- **स्थिर भुगतान लिंक:** जल्दी, नो-कोड भुगतान संग्रह के लिए तुरंत साझा करने योग्य URL।
- **डायनामिक भुगतान लिंक:** API या SDK का उपयोग करके कस्टम विवरण के साथ प्रोग्रामेटिकली भुगतान लिंक बनाएं।
- **चेकआउट सेशन्स:** पूर्व कॉन्फ़िगर उत्पाद कार्ट और ग्राहक विवरण के साथ सुरक्षित, कस्टमाइज़ किए जा सकने वाले चेकआउट अनुभव बनाएं।

Customer Portal Route Handler आपको Dodo Payments कस्टमर पोर्टल को अपने Bun सर्वर एप्लिकेशन में सहज रूप से एकीकृत करने में सक्षम बनाता है।

## Webhook Route Handler
- **Method:** केवल POST अनुरोध समर्थित हैं। अन्य तरीके 405 लौटाते हैं।
- **Signature Verification:** <code>webhookKey</code> का उपयोग करके webhook हस्ताक्षर सत्यापित करता है। यदि सत्यापन विफल होता है तो 401 लौटाता है।
- **Payload Validation:** Zod के साथ सत्यापित। अमान्य पेलोड पर 400 लौटाता है।
- **Error Handling:**
  - 401: अवैध हस्ताक्षर
  - 400: अमान्य पेलोड
  - 500: सत्यापन के दौरान आंतरिक त्रुटि
- **Event Routing:** पेलोड प्रकार के आधार पर उपयुक्त ईवेंट हैंडलर को कॉल करता है।
