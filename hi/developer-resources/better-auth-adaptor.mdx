# अवलोकन

<b>बेहतर ऑथ अडाप्टर</b> डोडो पेमेंट्स के लिए प्रदान करता है:

- साइन-अप पर स्वचालित ग्राहक निर्माण
- उत्पाद स्लग मैपिंग के साथ चेकआउट सत्र (प्राथमिकता)
- स्व-सेवा ग्राहक पोर्टल
- उपयोग-आधारित बिलिंग के लिए मीटर उपयोग इनजेशन और रिपोर्टिंग एंडपॉइंट्स
- हस्ताक्षर सत्यापन के साथ वास्तविक समय वेबहुक इवेंट प्रोसेसिंग
- पूर्ण TypeScript समर्थन

<Note>
  इस एकीकरण का उपयोग करने के लिए आपको Dodo Payments खाता और API कुंजियाँ चाहिए।
</Note>

# पूर्वापेक्षाएँ

- Node.js 20+
- आपके डोडो पेमेंट्स डैशबोर्ड तक पहुंच
- [better-auth](https://www.npmjs.com/package/better-auth) का उपयोग करने वाला मौजूदा प्रोजेक्ट

# सेटअप

<Steps>
<Step title="Configure environment variables">
अपने <code>.env</code> फ़ाइल में इन्हें जोड़ें:

```env expandable
DODO_PAYMENTS_API_KEY=your_api_key_here
DODO_PAYMENTS_WEBHOOK_SECRET=your_webhook_secret_here
BETTER_AUTH_URL=http://localhost:3000
BETTER_AUTH_SECRET=your_better_auth_secret_here
```

<Warning>
API कुंजियों या सीक्रेट्स को वर्शन कंट्रोल में कभी कमिट न करें।
</Warning>
</Step>

<Step title="Set up server-side integration">
<code>src/lib/auth.ts</code> बनाएँ या अपडेट करें:


```typescript expandable
import { betterAuth } from "better-auth";
import {
  dodopayments,
  checkout,
  portal,
  webhooks,
  usage,
} from "@dodopayments/better-auth";
import DodoPayments from "dodopayments";

export const dodoPayments = new DodoPayments({
  bearerToken: process.env.DODO_PAYMENTS_API_KEY!,
  environment: "test_mode", // or "live_mode" for production
});

export const { auth, endpoints, client } = BetterAuth({
  plugins: [
    dodopayments({
      client: dodoPayments,
      createCustomerOnSignUp: true,
      use: [
        checkout({
          products: [
            {
              productId: "pdt_xxxxxxxxxxxxxxxxxxxxx",
              slug: "premium-plan",
            },
          ],
          successUrl: "/dashboard/success",
          authenticatedUsersOnly: true,
        }),
        portal(),
        webhooks({
          webhookKey: process.env.DODO_PAYMENTS_WEBHOOK_SECRET!,
          onPayload: async (payload) => {
            console.log("Received webhook:", payload.event_type);
          },
        }),
        usage(),
      ],
    }),
  ],
});
```

<Tip>
उत्पादन के लिए <code>environment</code> को <code>live_mode</code> पर सेट करें।
</Tip>
</Step>

<Step title="Set up client-side integration">
<code>src/lib/auth-client.ts</code> बनाएँ या अपडेट करें:


```typescript expandable
import { createAuthClient } from "better-auth/react";
import { dodopaymentsClient } from "@dodopayments/better-auth";

export const authClient = createAuthClient({
  baseURL: process.env.BETTER_AUTH_URL || "http://localhost:3000",
  plugins: [dodopaymentsClient()],
});
```

</Step>
</Steps>

</Info>

<Info>
  पारंपरिक <code>checkout</code> विधि के विपरीत, <code>checkoutSession</code> को अग्रिम में बिलिंग जानकारी की आवश्यकता नहीं होती क्योंकि यह चेकआउट पृष्ठ पर उपयोगकर्ता से ली जाएगी। आप अभी भी आर्ग्युमेंट में <code>billing</code> कुंजी भेजकर इसे ओवरराइड कर सकते हैं।
</Info>

<Info>
  पारंपरिक <code>checkout</code> विधि की तरह, <code>checkoutSession</code> ग्राहक का ईमेल और नाम उनके बेहतर-ऑथ सत्र से लेता है, हालाँकि आप इसे <code>customer</code> ऑब्जेक्ट के साथ ईमेल और नाम भेजकर ओवरराइड कर सकते हैं।
</Info>

<Info>
  आप [Create Checkout Session](https://docs.dodopayments.com/api-reference/checkout-sessions/create) एंडपॉइंट के अनुरोध बॉडी में दी गई समान विकल्पों को आर्ग्युमेंट में पास कर सकते हैं।
</Info>

<Note>
  रिटर्न URL सर्वर प्लगइन में कॉन्फ़िगर किए गए <code>successUrl</code> से लिया जाता है। आपको क्लाइंट पेलोड में <code>return_url</code> शामिल करने की आवश्यकता नहीं है।
</Note>

## मीटर उपयोग को ट्रैक करना

सर्वर पर <code>usage()</code> प्लगइन सक्षम करें ताकि मीटर किए गए इवेंट कैप्चर किए जा सकें और ग्राहकों को उनके उपयोग-आधारित बिलिंग की जांच करने की अनुमति दी जा सके।

- <code>authClient.dodopayments.usage.ingest</code> साइन-इन, ईमेल-प्रमाणित उपयोगकर्ता के लिए इवेंट इनजेस्ट करता है। 
- <code>authClient.dodopayments.usage.meters.list</code> उस ग्राहक की सदस्यताओं से जुड़े मीटर के लिए हाल के उपयोग की सूची बनाता है।

```typescript expandable
// Record a metered event (e.g., an API request)
const { error: ingestError } = await authClient.dodopayments.usage.ingest({
  event_id: crypto.randomUUID(),
  event_name: "api_request",
  metadata: {
    route: "/reports",
    method: "GET",
  },
  // Optional Date; defaults to now if omitted
  timestamp: new Date(),
});

if (ingestError) {
  console.error("Failed to record usage", ingestError);
}

// List recent usage for the current customer
const { data: usage, error: usageError } =
  await authClient.dodopayments.usage.meters.list({
    query: {
      page_size: 20,
      meter_id: "mtr_yourMeterId", // optional
    },
  });

if (usage?.items) {
  usage.items.forEach((event) => {
    console.log(event.event_name, event.timestamp, event.metadata);
  });
}
```

<Tip>
  उपयोगिता इनजेस्ट करते समय एक घंटे से पुराना या पांच मिनट से अधिक भविष्य का टाइमस्टैम्प अस्वीकार कर दिया जाता है।
</Tip>
<Tip>
  यदि आप उपयोगिता मीटर सूचीबद्ध करते समय <code>meter_id</code> छोड़ देते हैं, तो ग्राहक की सब्सक्रिप्शन से जुड़े सभी मीटर लौटाए जाते हैं।
</Tip>

# वेबहुक

<Info>
  वेबहुक्स प्लगइन Dodo Payments से वास्तविक समय भुगतान घटनाओं को सुरक्षित सिग्नेचर सत्यापन के साथ प्रोसेस करता है। डिफ़ॉल्ट एंडपॉइंट <code>/api/auth/dodopayments/webhooks</code> है।
</Info>

<Steps>
<Step title="Generate and set webhook secret">
Dodo Payments डैशबोर्ड में अपने एंडपॉइंट URL (उदा., <code>https://&lt;your-domain&gt;/api/auth/dodopayments/webhooks</code>) के लिए एक वेबहुक सीक्रेट जनरेट करें और इसे अपनी <code>.env</code> फ़ाइल में सेट करें:


```env
DODO_PAYMENTS_WEBHOOK_SECRET=your_webhook_secret_here
```

</Step>

<Step title="Handle webhook events">
उदाहरण हैंडलर:


```typescript expandable
webhooks({
  webhookKey: process.env.DODO_PAYMENTS_WEBHOOK_SECRET!,
  onPayload: async (payload) => {
    console.log("Received webhook:", payload.event_type);
  },
});
```

</Step>
</Steps>

# कॉन्फ़िगरेशन संदर्भ

<AccordionGroup>
  <Accordion title="Plugin Options">
    - <b>client</b> (आवश्यक): DodoPayments क्लाइंट उदाहरण - <b>createCustomerOnSignUp</b> (वैकल्पिक): उपयोगकर्ता साइनअप पर स्वचालित रूप से ग्राहक बनाएं - <b>use</b> (आवश्यक): सक्षम करने के लिए प्लगइन्स की सूची (checkout, portal, usage, webhooks)
  </Accordion>
  <Accordion title="Checkout Plugin Options">
    - <b>products</b>: उत्पादों की सूची या असिंक्रोनस फ़ंक्शन जो उत्पाद लौटाता है - <b>successUrl</b>: सफल भुगतान के बाद रीडायरेक्ट करने का URL - <b>authenticatedUsersOnly</b>: उपयोगकर्ता प्रमाणीकरण आवश्यक करें (डिफ़ॉल्ट: false)
  </Accordion>
</AccordionGroup>

# समस्या निवारण और सुझाव

<AccordionGroup>
  <Accordion title="Common Issues">
    - <b>Invalid API key</b>: <code>.env</code> में अपना <code>DODO_PAYMENTS_API_KEY</code> दोबारा जांचें। - <b>Webhook signature mismatch</b>: सुनिश्चित करें कि वेबहुक सीक्रेट Dodo Payments डैशबोर्ड में सेट किए गए एक से मेल खाता है। - <b>Customer not created</b>: पुष्टि करें कि <code>createCustomerOnSignUp</code> <code>true</code> पर सेट है।
  </Accordion>
  <Accordion title="Best Practices">
    - सभी सीक्रेट्स और कुंजियों के लिए पर्यावरण चर का उपयोग करें। - <code>live_mode</code> पर जाने से पहले <code>test_mode</code> में परीक्षण करें। - डिबगिंग और ऑडिटिंग के लिए वेबहुक घटनाओं को लॉग करें।
  </Accordion>
</AccordionGroup>
