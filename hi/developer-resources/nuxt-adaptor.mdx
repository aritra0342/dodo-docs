## अवलोकन

<Info>
यह गाइड बताती है कि अपने Nuxt एप्लिकेशन में आधिकारिक Nuxt मॉड्यूल का उपयोग करके Dodo Payments को कैसे एकीकृत करें। आप चेकआउट, ग्राहक पोर्टल और वेबहुक API रूट सेटअप करना और सुरक्षित रूप से वातावरण चर प्रबंधित करना सीखेंगे।
</Info>

## स्थापना

<Steps>
<Step title="Nuxt मॉड्यूल स्थापित करें">
अपने प्रोजेक्ट रूट में निम्नलिखित कमांड चलाएँ:

```bash
npm install @dodopayments/nuxt
```

</Step>
<Step title="nuxt.config.ts में मॉड्यूल पंजीकृत करें">
अपने <code>modules</code> ऐरे में <code>@dodopayments/nuxt</code> जोड़ें और इसे कॉन्फ़िगर करें:

```typescript nuxt.config.ts expandable
export default defineNuxtConfig({
  modules: ["@dodopayments/nuxt"],
  devtools: { enabled: true },
  compatibilityDate: "2025-02-25",
  runtimeConfig: {
    private: {
      bearerToken: process.env.NUXT_PRIVATE_BEARER_TOKEN,
      webhookKey: process.env.NUXT_PRIVATE_WEBHOOK_KEY,
      environment: process.env.NUXT_PRIVATE_ENVIRONMENT,
      returnUrl: process.env.NUXT_PRIVATE_RETURNURL
    },
  }
});
```

</Step>
</Steps>

<Warning>
अपने <code>.env</code> फ़ाइल या रहस्यों को संस्करण नियंत्रण में कभी भी कमिट न करें।
</Warning>

## चेकआउट रूट हैंडलर

<Info>
Dodo Payments आपके वेबसाइट में भुगतान एकीकृत करने के लिए तीन प्रकार के भुगतान प्रवाह का समर्थन करता है, यह अडैप्टर सभी प्रकार के भुगतान प्रवाह का समर्थन करता है।
</Info>

- **स्थैतिक भुगतान लिंक:** त्वरित, बिना कोड के भुगतान संग्रह के लिए तुरंत साझा करने योग्य URL।
- **गतिशील भुगतान लिंक:** API या SDKs का उपयोग करके कस्टम विवरण के साथ प्रोग्रामेटिक रूप से भुगतान लिंक उत्पन्न करें।
- **चेकआउट सत्र:** पूर्व-निर्धारित उत्पाद कार्ट और ग्राहक विवरण के साथ सुरक्षित, अनुकूलन योग्य चेकआउट अनुभव बनाएं।

## वेबहुक रूट हैंडलर

- **विधि:** केवल POST अनुरोधों का समर्थन किया जाता है। अन्य विधियाँ 405 लौटाती हैं।
- **हस्ताक्षर सत्यापन:** <code>webhookKey</code> का उपयोग करके वेबहुक हस्ताक्षर की पुष्टि करता है। यदि सत्यापन विफल होता है तो 401 लौटाता है।
- **पेलोड मान्यता:** Zod के साथ मान्य किया गया। अमान्य पेलोड के लिए 400 लौटाता है।
- **त्रुटि प्रबंधन:**
  - 401: अमान्य हस्ताक्षर
  - 400: अमान्य पेलोड
  - 500: सत्यापन के दौरान आंतरिक त्रुटि
- **घटना रूटिंग:** पेलोड प्रकार के आधार पर उपयुक्त घटना हैंडलर को कॉल करता है।
