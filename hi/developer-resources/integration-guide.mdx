## पूर्वापेक्षाएँ

Dodo Payments API को एकीकृत करने के लिए, आपको आवश्यकता होगी:
- एक Dodo Payments व्यापारी खाता
- डैशबोर्ड से API क्रेडेंशियल्स (API कुंजी और वेबहुक गुप्त कुंजी)

## डैशबोर्ड सेटअप

1. [Dodo Payments डैशबोर्ड](https://app.dodopayments.com/) पर जाएं

2. एक उत्पाद बनाएं (एक बार का भुगतान या सदस्यता)

4. अपनी API कुंजी उत्पन्न करें:
   - डेवलपर > API पर जाएं
   - [विस्तृत गाइड](/api-reference/introduction#api-key-generation)
   - env में DODO_PAYMENTS_API_KEY नामक API कुंजी कॉपी करें

5. वेबहुक कॉन्फ़िगर करें:
   - डेवलपर > वेबहुक पर जाएं
   - भुगतान सूचनाओं के लिए एक वेबहुक URL बनाएं
   - env में वेबहुक गुप्त कुंजी कॉपी करें

## एकीकरण

### भुगतान लिंक

अपने उपयोग मामले के लिए उपयुक्त एकीकरण पथ चुनें:
- **चेकआउट सत्र (सिफारिश की गई)**: अधिकांश एकीकरणों के लिए सबसे अच्छा। अपने सर्वर पर एक सत्र बनाएं और ग्राहकों को एक सुरक्षित, होस्टेड चेकआउट पर रीडायरेक्ट करें।
- **ओवरले चेकआउट**: जब आपको अपने साइट पर एक मोडल ओवरले के रूप में चेकआउट खोलने का इन-पेज अनुभव चाहिए।
- **इनलाइन चेकआउट**: पूरी तरह से एकीकृत, ब्रांडेड चेकआउट अनुभव के लिए अपने पृष्ठ लेआउट में सीधे चेकआउट एम्बेड करें।
- **स्थैतिक भुगतान लिंक**: त्वरित भुगतान संग्रह के लिए बिना कोड के, तुरंत साझा करने योग्य URL।
- **गतिशील भुगतान लिंक**: प्रोग्रामेटिक रूप से बनाए गए लिंक। हालाँकि, चेकआउट सत्रों की सिफारिश की जाती है और यह अधिक लचीलापन प्रदान करते हैं।

#### 1. चेकआउट सत्र

चेकआउट सत्रों का उपयोग एक बार के भुगतान या सब्सक्रिप्शन के लिए एक सुरक्षित, होस्टेड चेकआउट अनुभव बनाने के लिए करें। आप अपने सर्वर पर एक सत्र बनाते हैं, फिर ग्राहक को लौटाए गए `checkout_url` पर रीडायरेक्ट करते हैं।

<Info>
चेकआउट सत्र डिफ़ॉल्ट रूप से 24 घंटे के लिए मान्य होते हैं। यदि आप <code>confirm=true</code> पास करते हैं, तो सत्र 15 मिनट के लिए मान्य होते हैं और सभी आवश्यक फ़ील्ड प्रदान की जानी चाहिए।
</Info>

<Steps>
<Step title="चेकआउट सत्र बनाएं">
  अपने पसंदीदा SDK का चयन करें या REST API को कॉल करें।

  <Tabs>
  <Tab title="Node.js SDK">

  ```javascript
  import DodoPayments from 'dodopayments';

  const client = new DodoPayments({
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    environment: 'test_mode', // defaults to 'live_mode'
  });

  const session = await client.checkoutSessions.create({
    product_cart: [{ product_id: 'prod_123', quantity: 1 }],
    customer: { email: 'customer@example.com', name: 'John Doe' },
    return_url: 'https://yourapp.com/checkout/success',
  });
  ```

  </Tab>

  <Tab title="Python SDK">

<Step title="ग्राहक को चेकआउट पर रीडायरेक्ट करें">
  सत्र निर्माण के बाद, होस्टेड फ्लो शुरू करने के लिए `checkout_url` पर रीडायरेक्ट करें।

  ```javascript
  // Example in a browser context
  ```

<Tip>
भुगतान लेना शुरू करने के लिए सबसे तेज़, सबसे विश्वसनीय तरीके के लिए चेकआउट सत्रों को प्राथमिकता दें। उन्नत अनुकूलन के लिए, पूर्ण <a href="/developer-resources/checkout-session">चेकआउट सत्र गाइड</a> और <a href="/api-reference/checkout-sessions/create">API संदर्भ</a> देखें।
</Tip>

#### 2. ओवरले चेकआउट

एक सहज इन-पेज चेकआउट अनुभव के लिए, हमारे [ओवरले चेकआउट](/developer-resources/overlay-checkout) एकीकरण का अन्वेषण करें जो ग्राहकों को आपकी वेबसाइट छोड़े बिना भुगतान पूरा करने की अनुमति देता है।

#### 3. इनलाइन चेकआउट

अपने पृष्ठ में सीधे एम्बेड किए गए पूरी तरह से एकीकृत चेकआउट अनुभव के लिए, हमारे [इनलाइन चेकआउट](/developer-resources/inline-checkout) एकीकरण का उपयोग करें। यह आपको कस्टम ऑर्डर सारांश बनाने और चेकआउट लेआउट पर पूरी तरह से नियंत्रण रखने की अनुमति देता है जबकि डोडो पेमेंट्स सुरक्षित रूप से भुगतान संग्रह को संभालता है।

#### 4. स्थैतिक भुगतान लिंक

स्थैतिक भुगतान लिंक आपको एक सरल URL साझा करके तेजी से भुगतान स्वीकार करने की अनुमति देते हैं। आप ग्राहक विवरण को पूर्व-भरने, फ़ॉर्म फ़ील्ड को नियंत्रित करने और कस्टम मेटाडेटा जोड़ने के लिए क्वेरी पैरामीटर पास करके चेकआउट अनुभव को अनुकूलित कर सकते हैं।

<Steps>
<Step title="अपने भुगतान लिंक का निर्माण करें">
  आधार URL से शुरू करें और अपने उत्पाद ID को जोड़ें:

  ```text
  https://checkout.dodopayments.com/buy/{productid}
  ```

</Step>
<Step title="मुख्य पैरामीटर जोड़ें">
  आवश्यक क्वेरी पैरामीटर शामिल करें:
  - <ParamField query="quantity" type="integer" default="1">खरीदने के लिए वस्तुओं की संख्या।</ParamField>
  - <ParamField query="redirect_url" type="string" required>भुगतान पूर्ण होने के बाद रीडायरेक्ट करने के लिए URL।</ParamField>
  <Note>
  रीडायरेक्ट URL में भुगतान विवरण क्वेरी पैरामीटर के रूप में शामिल होंगे, उदाहरण के लिए:<br/>
  <code>https://example.com/?payment_id=pay_ts2ySpzg07phGeBZqePbH&status=succeeded</code>
  </Note>
</Step>
<Step title="ग्राहक जानकारी पूर्व-भरें (वैकल्पिक)">
  चेकआउट को सरल बनाने के लिए ग्राहक या बिलिंग फ़ील्ड को क्वेरी पैरामीटर के रूप में जोड़ें।
  <AccordionGroup>
    <Accordion title="समर्थित ग्राहक फ़ील्ड">
      - <ParamField query="fullName" type="string">ग्राहक का पूरा नाम (यदि firstName या lastName प्रदान किया गया है तो अनदेखा किया जाएगा)।</ParamField>
      - <ParamField query="firstName" type="string">ग्राहक का पहला नाम।</ParamField>
      - <ParamField query="lastName" type="string">ग्राहक का अंतिम नाम।</ParamField>
      - <ParamField query="email" type="string">ग्राहक का ईमेल पता।</ParamField>
      - <ParamField query="country" type="string">ग्राहक का देश।</ParamField>
      - <ParamField query="addressLine" type="string">सड़क का पता।</ParamField>
      - <ParamField query="city" type="string">शहर।</ParamField>
      - <ParamField query="state" type="string">राज्य या प्रांत।</ParamField>
      - <ParamField query="zipCode" type="string">डाक/ZIP कोड।</ParamField>
      - <ParamField query="showDiscounts" type="boolean">सत्य या असत्य</ParamField>
    </Accordion>
  </AccordionGroup>
</Step>
<Step title="फ़ॉर्म फ़ील्ड को नियंत्रित करें (वैकल्पिक)">
  <Info>
  आप विशिष्ट फ़ील्ड को अक्षम कर सकते हैं ताकि वे ग्राहक के लिए केवल पढ़ने योग्य हों। यह तब उपयोगी है जब आपके पास पहले से ग्राहक का विवरण हो (जैसे, लॉगिन किए हुए उपयोगकर्ता)।
  </Info>
  एक फ़ील्ड को अक्षम करने के लिए, उसका मान प्रदान करें और संबंधित <code>disable...</code> ध्वज को <code>true</code> पर सेट करें:
  <CodeGroup>

  ```text Example
  &email=alice@example.com&disableEmail=true
  ```

  </CodeGroup>
  <Tabs>
    <Tab title="फ्लैग्स तालिका को अक्षम करें">
      | फ़ील्ड         | अक्षम फ्लैग         | आवश्यक पैरामीटर |
      |---------------|---------------------|-------------------|
      | पूरा नाम     | `disableFullName`   | `fullName`        |
      | पहला नाम    | `disableFirstName`  | `firstName`       |
      | अंतिम नाम     | `disableLastName`   | `lastName`        |
      | ईमेल         | `disableEmail`      | `email`           |
      | देश       | `disableCountry`    | `country`         |
      | पता लाइन  | `disableAddressLine`| `addressLine`     |
      | शहर          | `disableCity`       | `city`            |
      | राज्य         | `disableState`      | `state`           |
      | ज़िप कोड      | `disableZipCode`    | `zipCode`         |
    </Tab>
  </Tabs>
  <Tip>
  फ़ील्ड को अक्षम करना आकस्मिक परिवर्तनों को रोकने में मदद करता है और डेटा की स्थिरता सुनिश्चित करता है।
  </Tip>
  <Note>
  <code>showDiscounts=false</code> सेट करने से चेकआउट फॉर्म में छूट अनुभाग अक्षम और छिप जाएगा। इसका उपयोग करें यदि आप ग्राहकों को चेकआउट के दौरान कूपन या प्रोमो कोड दर्ज करने से रोकना चाहते हैं।
  </Note>

</Step>

  - <ParamField query="paymentCurrency" type="string">भुगतान मुद्रा निर्दिष्ट करता है। यह बिलिंग देश की मुद्रा पर डिफ़ॉल्ट होता है।</ParamField>

  पूरा किया गया भुगतान लिंक अपने ग्राहक को भेजें। जब वे विजिट करते हैं, तो सभी क्वेरी पैरामीटर एकत्र किए जाते हैं और एक सत्र ID के साथ संग्रहीत होते हैं। URL फिर केवल सत्र पैरामीटर को शामिल करने के लिए सरल किया जाता है (जैसे, <code>?session=sess_1a2b3c4d</code>)। संग्रहीत जानकारी पृष्ठ ताज़ा करने के दौरान बनी रहती है और चेकआउट प्रक्रिया के दौरान सुलभ होती है।
  <Check>
  ग्राहक का चेकआउट अनुभव अब आपके पैरामीटर के आधार पर सरल और व्यक्तिगत है।
  </Check>
</Step>

#### 4. गतिशील भुगतान लिंक
<Tip>
अधिकांश उपयोग मामलों के लिए चेकआउट सत्रों को प्राथमिकता दें, वे अधिक लचीलापन और नियंत्रण प्रदान करते हैं।
</Tip>

API कॉल या हमारे SDK के माध्यम से ग्राहक विवरण के साथ बनाए गए। यहाँ एक उदाहरण है:

गतिशील भुगतान लिंक बनाने के लिए दो APIs हैं:

- एक बार का भुगतान लिंक API [API संदर्भ](/api-reference/payments/post-payments)
- सदस्यता भुगतान लिंक API [API संदर्भ](/api-reference/subscriptions/post-subscriptions)

नीचे दिया गया गाइड एक बार के भुगतान लिंक निर्माण के लिए है।

सदस्यताओं को एकीकृत करने के लिए विस्तृत निर्देशों के लिए, इस [सदस्यता एकीकरण गाइड](/developer-resources/subscription-integration-guide) को देखें।

<Info>सुनिश्चित करें कि आप भुगतान लिंक प्राप्त करने के लिए `payment_link = true` पास कर रहे हैं </Info>

    ```go
    package main

import (

  "fmt"

      State: dodopayments.F("state"),
      Street: dodopayments.F("street"),
      Zipcode: dodopayments.F(int64(0)),
    }),
    Customer: dodopayments.F(dodopayments.PaymentNewParamsCustomer{
      Email: dodopayments.F("email"),
      Name: dodopayments.F("name"),

    ProductCart: dodopayments.F([]dodopayments.PaymentNewParamsProductCart{dodopayments.PaymentNewParamsProductCart{
      ProductID: dodopayments.F("product_id"),
      Quantity: dodopayments.F(int64(0)),
    }}),
  })
  if err != nil {
    panic(err.Error())
  }
  fmt.Printf("%+v\n", payment.PaymentLink)
}
    ```
