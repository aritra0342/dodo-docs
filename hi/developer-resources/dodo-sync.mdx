डोडो पेमेंट्स एक अंतर्निहित डेटाबेस सिंक सुविधा प्रदान करता है जो स्वचालित रूप से आपके भुगतान डेटा को आपके अपने डेटाबेस के साथ समन्वयित करता है। आप **भुगतान**, **ग्राहक**, **सदस्यताएँ**, और **लाइसेंस** को सिंक कर सकते हैं ताकि आपके डेटा की एक स्थानीय प्रति एनालिटिक्स, रिपोर्टिंग, या अन्य सिस्टम के साथ इंटीग्रेशन के लिए बनाए रखी जा सके।

<Info>
**कार्यान्वयन**: [npm package](https://www.npmjs.com/package/dodo-sync) के माध्यम से उपलब्ध | **स्रोत कोड**: [GitHub](https://github.com/dodopayments/dodo-sync)
</Info>

## आप क्या सिंक कर सकते हैं?

हमारी डेटाबेस सिंक सुविधा निम्नलिखित डोडो पेमेंट्स संस्थाओं को आपके डेटाबेस के साथ समन्वयित करने का समर्थन करती है:

आप इन एंटिटीज़ की किसी भी संयोजन को `scopes` पैरामीटर में निर्दिष्ट करके सिंक कर सकते हैं। सभी सिंक ऑपरेशन इनक्रिमेंटल होते हैं और बेहतर प्रदर्शन के लिए केवल नए या अपडेट किए गए रिकॉर्ड ही ट्रांसफर करते हैं।

## डेटाबेस समर्थन

हम वर्तमान में **MongoDB**, **PostgreSQL**, **MySQL**, और **ClickHouse** का समर्थन करते हैं।

हम सक्रिय रूप से समर्थन का विस्तार करने पर काम कर रहे हैं:
- **डेटाबेस**: Snowflake और अन्य।
- **पाइपलाइन्स**: ETL पाइपलाइन्स, रियलटाइम सिंक।

<Tip>
हम लगातार डेटाबेस समर्थन का विस्तार कर रहे हैं। यदि आप नए डेटाबेस इंटीग्रेशन में योगदान देना चाहते हैं, तो कृपया हमारे [GitHub repository](https://github.com/dodopayments/dodo-sync) पर Pull Request सबमिट करें।
</Tip>

## शुरू करना

आप हमारी डेटाबेस सिंक सुविधा का उपयोग **CLI** के माध्यम से या अपने **कोड** में प्रोग्रामेटिक रूप से कर सकते हैं। दोनों विधियाँ समान कार्यक्षमता प्रदान करती हैं—उसका चयन करें जो आपके कार्यप्रवाह के लिए सबसे उपयुक्त हो।

### CLI का उपयोग करना

CLI उपकरण डेटाबेस समन्वय स्थापित करने और चलाने का एक त्वरित तरीका प्रदान करता है। इसे वैश्विक रूप से स्थापित करें ताकि आप इसे अपने टर्मिनल में कहीं से भी उपयोग कर सकें:

<CodeGroup>

```bash npm
npm install -g dodo-sync
```

```bash bun
bun add -g dodo-sync
```

</CodeGroup>

#### CLI चलाना

CLI दो मोड का समर्थन करता है: **इंटरएक्टिव मोड** मार्गदर्शित सेटअप के लिए, और **मैनुअल मोड** सीधे कॉन्फ़िगरेशन के लिए।

**इंटरएक्टिव मोड**: इंटरएक्टिव सेटअप विज़ार्ड शुरू करने के लिए बिना तर्क के कमांड चलाएँ।

```bash
dodo-sync
```

**मैनुअल मोड**: विज़ार्ड को छोड़ने के लिए सीधे तर्क पास करें।

```bash
dodo-sync -i [interval] -d [database] -u [database_uri] --scopes [scopes] --api-key [api_key] --env [environment]
```

**उदाहरण:**

```bash
# MongoDB
dodo-sync -i 600 -d mongodb -u mongodb://mymongodb.url --scopes "licences,payments,customers,subscriptions" --api-key YOUR_API_KEY --env test_mode

# PostgreSQL
dodo-sync -i 600 -d postgres -u postgresql://user:password@localhost:5432/mydb --scopes "licences,payments,customers,subscriptions" --api-key YOUR_API_KEY --env test_mode

# MySQL
dodo-sync -i 600 -d mysql -u mysql://user:password@localhost:3306/mydb --scopes "licences,payments,customers,subscriptions" --api-key YOUR_API_KEY --env test_mode

# ClickHouse
dodo-sync -i 600 -d clickhouse -u http://localhost:8123 --scopes "licences,payments,customers,subscriptions" --api-key YOUR_API_KEY --env test_mode
```

#### CLI तर्क

<ParamField path="--interval" type="number" alias="-i">
सिंक अंतराल सेकंड में। यह निर्धारित करता है कि सिंक ऑपरेशन कितनी बार चलता है। यदि प्रदान नहीं किया गया है, तो सिंक एक बार चलेगा और फिर बंद हो जाएगा।
</ParamField>

<ParamField path="--database" type="string" alias="-d" required>
उपयोग करने के लिए डेटाबेस प्रकार। समर्थित मान: `"mongodb"`, `"postgres"`, `"mysql"`, या `"clickhouse"`.
</ParamField>

<ParamField path="--database-uri" type="string" alias="-u" required>
आपके डेटाबेस के लिए कनेक्शन URI:
- **MongoDB**: `mongodb://localhost:27017` या `mongodb+srv://user:pass@cluster.mongodb.net/`
- **PostgreSQL**: `postgresql://user:password@localhost:5432/mydb`
- **MySQL**: `mysql://user:password@localhost:3306/mydb`
- **ClickHouse**: `http://localhost:8123`
</ParamField>

<ParamField path="--scopes" type="string" required>
सिंक करने के लिए डेटा एंटिटीज़ की कॉमा-सेपरेटेड सूची। उपलब्ध स्कोप: `licences`, `payments`, `customers`, `subscriptions`। उदाहरण: `"payments,customers"`.
</ParamField>

<ParamField path="--api-key" type="string" required>
आपकी Dodo Payments API कुंजी। लाइव मोड के लिए `dp_live_` या टेस्ट मोड के लिए `dp_test_` से शुरू होनी चाहिए।
</ParamField>

<ParamField path="--env" type="string" required>
पर्यावरण लक्ष्य। यह `"live_mode"` या `"test_mode"` में से एक होना चाहिए। यह निर्धारित करता है कि कौन सा Dodo Payments पर्यावरण सिंक किया जाएगा।
</ParamField>

<ParamField path="--rate-limit" type="number" alias="--rl">
प्रति सेकंड अनुरोधों में रेट लिमिट। यह नियंत्रित करता है कि सिंक इंजन Dodo Payments API पर API अनुरोध कितनी तेजी से करता है ताकि API पर अत्यधिक लोड न पड़े।
</ParamField>

### अपने कोड में उपयोग करना

प्रोग्रामेटिक नियंत्रण के लिए, सिंक सुविधा को सीधे अपने एप्लिकेशन में एकीकृत करें। इसे अपने प्रोजेक्ट में एक निर्भरता के रूप में स्थापित करें:

<CodeGroup>

```bash npm
npm install dodo-sync
```

```bash bun
bun add dodo-sync
```

</CodeGroup>

#### स्वचालित सिंक (अंतर-आधारित)

जब आप चाहते हैं कि सिंक नियमित अंतराल पर लगातार चले, तो स्वचालित सिंक का उपयोग करें:

```typescript
import { DodoSync } from 'dodo-sync';

const syncDodoPayments = new DodoSync({
  interval: 60, // Sync every 60 seconds
  database: 'mongodb',
  databaseURI: process.env.MONGODB_URI, // e.g., 'mongodb://localhost:27017'
  scopes: ['licences', 'payments', 'customers', 'subscriptions'],
  dodoPaymentsOptions: {
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    environment: 'test_mode' // or 'live_mode'
  }
});

// Initialize connection
await syncDodoPayments.init();

// Start the sync loop
syncDodoPayments.start();
```

<Tip>
ऑटोमैटिक सिंकिंग के लिए `.start()` का उपयोग करते समय `interval` विकल्प आवश्यक है। सिंक निर्दिष्ट अंतराल पर लगातार चलता रहेगा जब तक प्रक्रिया को रोका न जाए।
</Tip>

#### मैनुअल सिंक

जब आप मांग पर सिंक संचालन को ट्रिगर करना चाहते हैं (जैसे, क्रोन जॉब या API एंडपॉइंट से), तो मैनुअल सिंक का उपयोग करें:

```typescript
import { DodoSync } from 'dodo-sync';

const syncDodoPayments = new DodoSync({
  database: 'mongodb',
  databaseURI: process.env.MONGODB_URI,
  scopes: ['licences', 'payments', 'customers', 'subscriptions'],
  dodoPaymentsOptions: {
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    environment: 'test_mode'
  }
});

// Initialize connection
await syncDodoPayments.init();

// Trigger a single sync operation
await syncDodoPayments.run();
```

<Tip>
मैनुअल सिंक का उपयोग करते समय, `interval` विकल्प आवश्यक नहीं है। आप जब भी सिंक ऑपरेशन करना चाहें `.run()` को कॉल कर सकते हैं।
</Tip>

## महत्वपूर्ण जानकारी

<Warning>
**MongoDB**: आपके डेटाबेस सर्वर पर `dodopayments_sync` नाम का डेटाबेस स्वचालित रूप से बनाया जाएगा। सभी सिंक डेटा वहीं संग्रहीत होंगे। यह डेटाबेस नाम वर्तमान में निश्चित है और इसे बदला नहीं जा सकता।
</Warning>

**PostgreSQL**: टेबल (`Subscriptions`, `Payments`, `Licenses`, `Customers`) आपके कनेक्शन URI में निर्दिष्ट डेटाबेस में बनाई जाएंगी। डेटा JSONB के रूप में संग्रहीत होता है।

**MySQL**: टेबल (`Subscriptions`, `Payments`, `Licenses`, `Customers`) आपके कनेक्शन URI में निर्दिष्ट डेटाबेस में बनाई जाएंगी। डेटा JSON के रूप में संग्रहीत होता है।

**ClickHouse**: टेबल (`Subscriptions`, `Payments`, `Licenses`, `Customers`) ReplacingMergeTree इंजन का उपयोग करके बनाई जाएंगी। क्वेरी करते समय डुप्लीकेट रहित परिणाम सुनिश्चित करने के लिए `FINAL` कीवर्ड का उपयोग करें।
</Warning>

<Info>
सिंक इंजन परिवर्तन ट्रैक करता है और केवल नए या अपडेट किए गए रिकॉर्ड ही सिंक करता है, जिससे बड़े डेटा सेट के साथ भी बाद के सिंक प्रभावी रहते हैं।
</Info>
