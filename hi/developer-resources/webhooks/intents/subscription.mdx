## सदस्यता वेबहुक घटनाएँ

सदस्यता जीवनचक्र परिवर्तनों को ट्रैक करने के लिए निम्नलिखित वेबहुक घटनाएँ उपलब्ध हैं:

| घटना | विवरण |
|-------|-------------|
| `subscription.active` | सदस्यता सफलतापूर्वक सक्रिय की गई |
| `subscription.updated` | सदस्यता ऑब्जेक्ट को अपडेट किया गया (किसी भी फ़ील्ड परिवर्तन से यह घटना ट्रिगर होती है) |
| `subscription.on_hold` | सदस्यता नवीनीकरण में विफलता के कारण होल्ड पर है |
| `subscription.renewed` | सदस्यता अगले बिलिंग अवधि के लिए नवीनीकरण की गई |
| `subscription.plan_changed` | सदस्यता योजना को अपग्रेड, डाउनग्रेड, या संशोधित किया गया |
| `subscription.cancelled` | सदस्यता रद्द की गई |
| `subscription.failed` | जनादेश निर्माण के दौरान सदस्यता निर्माण विफल रहा |
| `subscription.expired` | सदस्यता अपनी अवधि के अंत तक पहुँच गई |

### वास्तविक समय समन्वय के लिए `subscription.updated` का उपयोग करना

जब भी कोई सदस्यता फ़ील्ड बदलता है, तो `subscription.updated` वेबहुक सक्रिय होता है, जिससे आप बिना पोलिंग के अपने एप्लिकेशन की स्थिति को समन्वयित रख सकते हैं:

```javascript
app.post('/webhooks/dodo', async (req, res) => {
  const event = req.body;
  
  if (event.type === 'subscription.updated') {
    const subscription = event.data;
    
    // Sync subscription changes to your database
    await syncSubscription(subscription.subscription_id, {
      status: subscription.status,
      next_billing_date: subscription.next_billing_date,
      metadata: subscription.metadata,
      // ... other fields you want to track
    });
    
    console.log(`Subscription ${subscription.subscription_id} updated`);
  }
  
  res.json({ received: true });
});
```

<Tip>
`subscription.updated` पर सब्सक्राइब करें ताकि किसी भी सदस्यता परिवर्तन के बारे में वास्तविक समय में सूचनाएँ प्राप्त कर सकें, जिससे API से अपडेट के लिए पोलिंग करने की आवश्यकता समाप्त हो जाती है।
</Tip>
