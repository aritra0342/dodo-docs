## सदस्यता वेबहुक घटनाएँ

सदस्यता जीवनचक्र परिवर्तनों को ट्रैक करने के लिए निम्नलिखित वेबहुक घटनाएँ उपलब्ध हैं:

| इवेंट | विवरण |
|-------|-------------|
| `subscription.active` | सब्सक्रिप्शन सफलतापूर्वक सक्रिय हो गया |
| `subscription.updated` | सब्सक्रिप्शन ऑब्जेक्ट को अपडेट किया गया (कोई भी फ़ील्ड परिवर्तन इस इवेंट को ट्रिगर करता है) |
| `subscription.on_hold` | नवीनीकरण में असफलता के कारण सब्सक्रिप्शन को होल्ड पर रखा गया |
| `subscription.renewed` | सब्सक्रिप्शन को अगले बिलिंग अवधि के लिए नवीनीकृत किया गया |
| `subscription.plan_changed` | सब्सक्रिप्शन प्लान को अपग्रेड, डाउनग्रेड, या संशोधित किया गया |
| `subscription.cancelled` | सब्सक्रिप्शन रद्द किया गया |
| `subscription.failed` | मंदेट निर्माण के दौरान सब्सक्रिप्शन निर्माण विफल रहा |
| `subscription.expired` | सब्सक्रिप्शन अपने टर्म के अंत तक पहुंच गया |

### रीयल-टाइम सिंक के लिए `subscription.updated` का उपयोग करना

`subscription.updated` वेबहुक तब फायर होता है जब भी कोई सब्सक्रिप्शन फ़ील्ड बदलता है, जिससे आप पॉलिंग किए बिना अपने एप्लिकेशन स्थिति को सिंक में रख सकते हैं:

```javascript
app.post('/webhooks/dodo', async (req, res) => {
  const event = req.body;
  
  if (event.type === 'subscription.updated') {
    const subscription = event.data;
    
    // Sync subscription changes to your database
    await syncSubscription(subscription.subscription_id, {
      status: subscription.status,
      next_billing_date: subscription.next_billing_date,
      metadata: subscription.metadata,
      // ... other fields you want to track
    });
    
    console.log(`Subscription ${subscription.subscription_id} updated`);
  }
  
  res.json({ received: true });
});
```

<Tip>
`subscription.updated` को सब्सक्राइब करके किसी भी सब्सक्रिप्शन बदलाव के बारे में रीयल-टाइम सूचनाएँ प्राप्त करें, जिससे अपडेट्स के लिए API को पॉल करने की ज़रूरत समाप्त हो जाती है।
</Tip>
