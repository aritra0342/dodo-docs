## नई विशेषताएँ 🚀

### API सुधार
- इनवॉइस API लॉन्च किया गया, जो व्यापारियों और ग्राहकों को `payment_id` का उपयोग करके इनवॉइस डाउनलोड करने की अनुमति देता है। अधिक जानकारी के लिए [इनवॉइस API](/api-reference/payments/get-invoice) दस्तावेज़ देखें।
- सब्सक्रिप्शन नवीनीकरण सूचनाओं के लिए नया `subscription.renewed` वेबहुक इवेंट लागू किया गया।

### प्लेटफ़ॉर्म अपडेट
- डोडो पेमेंट्स [वॉल ऑफ लव](https://dodopayments.com/wall-of-love) पेश किया गया, जो ग्राहक प्रशंसापत्र को प्रदर्शित करता है।
- सफल लेनदेन के लिए सामाजिक साझाकरण कार्यक्षमता जोड़ी गई।
    <img src="/images/changelog/share-transaction.png" alt="शेयर ट्रांजेक्शन फीचर"/>
- गोपनीयता नीति, सेवा की शर्तें, और रिफंड नीति दस्तावेज़ों के स्वचालित निर्माण के लिए नीति जनरेटर उपकरण जारी किया गया। [नीति जनरेटर आजमाएँ](https://dodopayments.com/tools/policy-generator)।

### दस्तावेज़ सुधार
- व्यापक गाइड प्रकाशित किए गए:
    - [सब्सक्रिप्शन इंटीग्रेशन गाइड](/developer-resources/subscription-integration-guide)
    - [वेबहुक इवेंट्स गाइड](/developer-resources/webhooks/intents/webhook-events-guide)

## सुधार और बग फिक्स 🔧

### API अपडेट
- [क्रिएट सब्सक्रिप्शन API](/api-reference/subscriptions/post-subscriptions) में उत्पाद-स्तरीय ट्रायल अवधि को ओवरराइड करने के लिए `trial_period_days` पैरामीटर जोड़ा गया।
- ग्राहक पहचान लॉजिक में सुधार: अब एक ही ईमेल के लिए भुगतान और सब्सक्रिप्शन में लगातार `customer_id` का उपयोग किया जा रहा है।
- विभिन्न प्रदर्शन अनुकूलन लागू किए गए और छोटे बग फिक्स किए गए।
