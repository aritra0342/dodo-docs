## नई विशेषताएँ 🚀

1. **मिक्स्ड कार्ट चेकआउट: एक ही कार्ट में वन-टाइम और सब्सक्रिप्शन उत्पाद**  
   चेकआउट अब एक ही कार्ट में एक-बार का भुगतान करने वाले उत्पादों और सब्सक्रिप्शन उत्पादों को मिलाकर नए शक्तिशाली बिलिंग उपयोग केस अनलॉक करता है।
   
   <Frame>
   <img src="/images/changelog/mixed-checkout.png" alt="Mixed checkout with one-time and subscription products" />
   </Frame>
   
   ### यह क्या सक्षम करता है
   
   - **सेटअप शुल्क + सब्सक्रिप्शन**: एक ही चेकआउट में आवर्ती सब्सक्रिप्शन के साथ एक-बार का सेटअप शुल्क लें
   - **आवर्ती घटकों वाले पैकेज**: एक-बार वाले उत्पाद को चल रही सेवा एक्सेस (सब्सक्रिप्शन) के साथ पैकेज के रूप में बेचें
   - **कोर्स + सदस्यता मिश्रण**: किसी कोर्स की खरीद को वैकल्पिक समुदाय सदस्यता सब्सक्रिप्शन के साथ ऑफर करें
   - **लाइफटाइम + अतिरिक्त सब्सक्रिप्शन**: जीवन भर उत्पाद एक्सेस को वैकल्पिक आवर्ती प्रीमियम सुविधाओं के साथ जोड़ें
   
   ### यह कैसे काम करता है
   
   बस अपने चेकआउट सत्र में एक-बार और सब्सक्रिप्शन दोनों उत्पाद जोड़ें:
   

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [
       // One-time setup fee
       { product_id: 'prod_setup_fee', quantity: 1 },
       // Monthly subscription
       { product_id: 'prod_monthly_plan', quantity: 1 }
     ],
     customer: {
       email: 'customer@example.com',
       name: 'Jane Doe'
     },
     return_url: 'https://yourapp.com/success'
   });
   ```

   
   <Check>
   परिणाम: एक एकल चेकआउट फ्लो जो वन-टाइम भुगतान को प्रोसेस करता है और सब्सक्रिप्शन को प्रारंभ करता है — घर्षण को कम करता है और रूपांतरण दर बढ़ाता है।
   </Check>
   
   <Info>
   जब उत्पाद प्रकारों को मिलाया जाता है, तो एक-बार का भुगतान तुरंत लिया जाता है जबकि सब्सक्रिप्शन उससे निर्धारित बिलिंग साइकिल (जिसमें आपके द्वारा सेट किया गया कोई परीक्षण अवधि शामिल है) का पालन करता है।
   </Info>
   
   और जानें: [Checkout Sessions](/developer-resources/checkout-session)

2. **संगठित ग्राहक पोर्टल**  
   
   एक नया केंद्रीकृत ग्राहक पोर्टल [customer.dodopayments.com](https://customer.dodopayments.com) पर जहाँ ग्राहक Dodo Payments पर विभिन्न व्यवसायों में अपनी सभी खरीद और सब्सक्रिप्शन देख और प्रबंधित कर सकते हैं।
   
   <Frame>
   <img src="/images/changelog/unified-customer-portal.png" alt="Unified Customer Portal" />
   </Frame>
   
   ### पोर्टल सुविधाएँ
   
   - **विभिन्न व्यवसायों की दृश्यता**: किसी भी व्यवसाय की सभी खरीद और सब्सक्रिप्शन एक ही स्थान पर देखें जो Dodo Payments का उपयोग करता है
   - **केंद्रीकृत सब्सक्रिप्शन प्रबंधन**: विभिन्न व्यापारियों के सब्सक्रिप्शन को एक डैशबोर्ड से प्रबंधित करें
   - **संगठित बिलिंग इतिहास**: अपनी सभी खरीदारी के चालान और भुगतान इतिहास तक पहुँचें
   - **सिंगल साइन-ऑन अनुभव**: सभी Dodo Payments व्यापारियों से खरीदारी एक्सेस करने के लिए एक बार अपनी ईमेल से लॉग इन करें
   
   ### यह कैसे काम करता है
   
   ग्राहक बस [customer.dodopayments.com](https://customer.dodopayments.com) पर जाते हैं और अपनी ईमेल पता दर्ज करते हैं। उन्हें एक सुरक्षित लॉगिन लिंक प्राप्त होता है जो Dodo Payments इकोसिस्टम में उनकी सभी खरीद पर पहुंच प्रदान करता है।
   
   <Tip>
   यह Unified Customer Portal व्यवसाय-विशिष्ट ग्राहक पोर्टलों को पूरा करता है। ग्राहक अभी भी व्यक्तिगत व्यापारी पोर्टलों तक पहुँच सकते हैं एक ब्रांडेड अनुभव के लिए, या एक समेकित दृश्य के लिए संयुक्त पोर्टल का उपयोग कर सकते हैं।
   </Tip>
   
   <Note>
   यह संयुक्त पोर्टल व्यवसाय-विशिष्ट पोर्टलों की तरह ही ईमेल-आधारित प्रमाणीकरण का उपयोग करता है, जिससे एक सुसंगत और सुरक्षित अनुभव सुनिश्चित होता है।
   </Note>

3. **भुगतान प्रतिक्रिया में चालान आईडी**  
   भुगतान प्रतिक्रिया अब `invoice_id` फ़ील्ड शामिल करती है, जिससे भुगतान को चालानों के साथ मेल करना और अकाउंटिंग सिस्टम के साथ एकीकरण आसान हो जाता है।
   

   ```json
   {
     "payment_id": "pay_2IjeQm4hqU6RA4Z4kwDee",
     "status": "succeeded",
     "invoice_id": "inv_2IsUnWGtRKFLxk7xAQeyt",
     "amount": 9900,
     "currency": "USD",
     // ... other fields
   }
   ```

   
   ### उपयोग के मामले
   
   - **अकाउंटिंग एकीकरण**: अपनी अकाउंटिंग सॉफ़्टवेयर में भुगतान को सीधे चालानों से लिंक करें
   - **ग्राहक सहायता**: समर्थन अनुरोधों को संभालते समय भुगतान से संबंधित चालान जल्दी से खोजें
   - **पुनर्मिलन**: भुगतान को प्रोग्रामेटिक रूप से चालानों से मिलाकर वित्तीय पुनर्मिलन को सरल बनाएं
   - **अनुपालन**: उनके संबंधित कर चालानों से भुगतान को जोड़कर सटीक रिकॉर्ड बनाए रखें
   
   <Info>
   `invoice_id` फ़ील्ड जब उपलब्ध हो भारत-विशिष्ट चालान आईडी का उपयोग करता है, जिससे स्थानीय चालान आवश्यकताओं का अनुपालन सुनिश्चित होता है।
   </Info>
   
   और जानें: [Get Payment Detail API](/api-reference/payments/get-payments-1)
