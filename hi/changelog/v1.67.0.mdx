## नई विशेषताएँ 🚀

1. **मिश्रित कार्ट चेकआउट: एक बार और सदस्यता उत्पाद एक ही कार्ट में**  
   चेकआउट अब एक ही कार्ट में एक बार के भुगतान उत्पादों और सदस्यता उत्पादों को संयोजित करने का समर्थन करता है, जिससे शक्तिशाली नए बिलिंग उपयोग के मामले खुलते हैं।
   
   <Frame>
   <img src="/images/changelog/mixed-checkout.png" alt="एक बार और सदस्यता उत्पादों के साथ मिश्रित चेकआउट" />
   </Frame>
   
   ### यह क्या सक्षम करता है
   
   - **सेटअप शुल्क + सदस्यताएँ**: एक बार के सेटअप शुल्क के साथ एक पुनरावर्ती सदस्यता को एक ही चेकआउट में चार्ज करें
   - **पुनरावर्ती घटकों के साथ बंडल**: एक उत्पाद (एक बार) को चल रहे सेवा पहुंच (सदस्यता) के साथ बंडल करें
   - **कोर्स + सदस्यता संयोजन**: एक कोर्स खरीदने के साथ एक वैकल्पिक सामुदायिक सदस्यता सदस्यता की पेशकश करें
   - **जीवनकाल + ऐड-ऑन सदस्यताएँ**: वैकल्पिक पुनरावर्ती प्रीमियम सुविधाओं के साथ जीवनकाल उत्पाद पहुंच को संयोजित करें
   
   ### यह कैसे काम करता है
   
   बस अपने चेकआउट सत्र में एक बार और सदस्यता उत्पाद दोनों जोड़ें:
   

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [
       // One-time setup fee
       { product_id: 'prod_setup_fee', quantity: 1 },
       // Monthly subscription
       { product_id: 'prod_monthly_plan', quantity: 1 }
     ],
     customer: {
       email: 'customer@example.com',
       name: 'Jane Doe'
     },
     return_url: 'https://yourapp.com/success'
   });
   ```

   
   <Check>
   परिणाम: एकल चेकआउट प्रवाह जो एक बार के भुगतान को संसाधित करता है और सदस्यता को प्रारंभ करता है — घर्षण को कम करना और रूपांतरण दरों को बढ़ाना।
   </Check>
   
   <Info>
   उत्पाद प्रकारों को मिलाते समय, एक बार का भुगतान तुरंत लिया जाता है जबकि सदस्यता अपने निर्धारित बिलिंग चक्र का पालन करती है (जिसमें कोई भी परीक्षण अवधि शामिल है जो आपने सेट की है)।
   </Info>
   
   अधिक जानें: [चेकआउट सत्र](/developer-resources/checkout-session)

2. **एकीकृत ग्राहक पोर्टल**  
   
   [customer.dodopayments.com](https://customer.dodopayments.com) पर एक नया केंद्रीकृत ग्राहक पोर्टल जहां ग्राहक Dodo Payments पर विभिन्न व्यवसायों के बीच अपने सभी खरीद और सदस्यताओं को देख और प्रबंधित कर सकते हैं।
   
   <Frame>
   <img src="/images/changelog/unified-customer-portal.png" alt="एकीकृत ग्राहक पोर्टल" />
   </Frame>
   
   ### पोर्टल की विशेषताएँ
   
   - **क्रॉस-बिजनेस दृश्यता**: Dodo Payments का उपयोग करने वाले किसी भी व्यवसाय से सभी खरीद और सदस्यताओं को एक ही स्थान पर देखें
   - **केंद्रीकृत सदस्यता प्रबंधन**: एक ही डैशबोर्ड से विभिन्न व्यापारियों के बीच सदस्यताओं का प्रबंधन करें
   - **एकीकृत बिलिंग इतिहास**: आपकी सभी खरीद से चालान और भुगतान इतिहास तक पहुंचें
   - **एकल साइन-ऑन अनुभव**: सभी Dodo Payments व्यापारियों से खरीद तक पहुंचने के लिए एक बार अपने ईमेल के साथ लॉग इन करें
   
   ### यह कैसे काम करता है
   
   ग्राहक बस [customer.dodopayments.com](https://customer.dodopayments.com) पर जाते हैं और अपना ईमेल पता दर्ज करते हैं। उन्हें एक सुरक्षित लॉगिन लिंक प्राप्त होगा जो Dodo Payments पारिस्थितिकी तंत्र में उनके सभी खरीद तक पहुंच प्रदान करता है।
   
   <Tip>
   एकीकृत ग्राहक पोर्टल व्यवसाय-विशिष्ट ग्राहक पोर्टलों को पूरा करता है। ग्राहक अभी भी ब्रांडेड अनुभव के लिए व्यक्तिगत व्यापारी पोर्टलों तक पहुंच सकते हैं, या समेकित दृश्य के लिए एकीकृत पोर्टल का उपयोग कर सकते हैं।
   </Tip>
   
   <Note>
   एकीकृत पोर्टल व्यवसाय-विशिष्ट पोर्टलों के समान ईमेल-आधारित प्रमाणीकरण का उपयोग करता है, जिससे एक सुसंगत और सुरक्षित अनुभव सुनिश्चित होता है।
   </Note>

3. **भुगतान प्रतिक्रिया में चालान आईडी**  
   भुगतान प्रतिक्रिया अब एक `invoice_id` फ़ील्ड शामिल करता है, जिससे चालानों के साथ भुगतानों का मिलान करना और लेखांकन प्रणालियों के साथ एकीकृत करना आसान हो जाता है।
   

   ```json
   {
     "payment_id": "pay_2IjeQm4hqU6RA4Z4kwDee",
     "status": "succeeded",
     "invoice_id": "inv_2IsUnWGtRKFLxk7xAQeyt",
     "amount": 9900,
     "currency": "USD",
     // ... other fields
   }
   ```

   
   ### उपयोग के मामले
   
   - **लेखांकन एकीकरण**: अपने लेखांकन सॉफ़्टवेयर में चालानों से सीधे भुगतानों को लिंक करें
   - **ग्राहक समर्थन**: समर्थन अनुरोधों को संभालते समय भुगतान से संबंधित चालान को जल्दी से खोजें
   - **सुलह**: कार्यक्रमmatically चालानों से भुगतानों का मिलान करके वित्तीय सुलह को सरल बनाएं
   - **अनुपालन**: भुगतान को उनके संबंधित कर चालानों से जोड़ने वाले सटीक रिकॉर्ड बनाए रखें
   
   <Info>
   `invoice_id` फ़ील्ड उपलब्ध होने पर भारत-विशिष्ट चालान आईडी का उपयोग करता है, जिससे स्थानीय चालान आवश्यकताओं के साथ अनुपालन सुनिश्चित होता है।
   </Info>
   
   अधिक जानें: [भुगतान विवरण API](/api-reference/payments/get-payments-1)
