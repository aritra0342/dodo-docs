## नई विशेषताएँ 🚀

1. **`subscription.updated` वेबहुक**  
   एक नया वेबहुक इवेंट जो हर सब्सक्रिप्शन ऑब्जेक्ट अपडेट पर सक्रिय होता है, आपको अपने सब्सक्रिप्शन में किसी भी परिवर्तन के बारे में वास्तविक समय में सूचनाएँ देता है।
   
   इस वेबहुक का उपयोग करें:
   - जब सब्सक्रिप्शन विवरण बदलता है तो तात्कालिक सूचनाएँ प्राप्त करें
   - अपने एप्लिकेशन की स्थिति को Dodo Payments के साथ समन्वयित रखें
   - API को पोलिंग किए बिना सब्सक्रिप्शन अपडेट पर प्रतिक्रिया दें
   
   <Tip>
   पोलिंग की आवश्यकता को समाप्त करने और अपने सिस्टम को वास्तविक समय में सब्सक्रिप्शन परिवर्तनों के साथ समन्वयित रखने के लिए `subscription.updated` की सदस्यता लें।
   </Tip>
   
   अधिक जानें: [सब्सक्रिप्शन वेबहुक](/developer-resources/webhooks/intents/subscription)

2. **चेकआउट में न्यूनतम पता समर्थन**  

   <Frame>
   <img src="/images/changelog/minimal-address.png" alt="चेकआउट में न्यूनतम पता समर्थन" />
   </Frame>

   नए न्यूनतम पता संग्रह मोड के साथ चेकआउट में रुकावट को कम करें। पूर्ण पता विवरण एकत्र करने के बजाय, न्यूनतम पता मोड केवल एकत्र करता है:
   
   - **देश**: कर निर्धारण के लिए हमेशा आवश्यक
   - **ZIP/पोस्टल कोड**: केवल उन क्षेत्रों में एकत्र किया जाता है जहाँ यह बिक्री कर, VAT, या GST गणना के लिए आवश्यक है
   
   <Check>
   परिणाम: ग्राहकों के लिए कम फ़ॉर्म फ़ील्ड के साथ चेकआउट पूरा करने में महत्वपूर्ण तेजी।
   </Check>
   
   **न्यूनतम पता सक्षम करें:**
   
   <Tabs>
   <Tab title="चेकआउट सत्र">

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [{ product_id: 'prod_123', quantity: 1 }],
     minimal_address: true
   });
   ```

   </Tab>
   
   <Tab title="स्थिर भुगतान लिंक">
   अपने भुगतान लिंक URL में `minimalAddress=true` जोड़ें:
   

   ```
   https://checkout.dodopayments.com/buy/prod_123?minimalAddress=true
   ```

   </Tab>
   </Tabs>
   
   <Info>
   डिफ़ॉल्ट रूप से, चेकआउट पूर्ण पते एकत्र करना जारी रखता है। हम धीरे-धीरे सभी चेकआउट लिंक को न्यूनतम पता मोड में स्थानांतरित करेंगे। रुकावट को कम करने के लाभ प्राप्त करने के लिए अब फीचर फ्लैग सक्षम करें।
   </Info>

3. **केवल पढ़ने के लिए API कुंजी**  
   सुरक्षित एकीकरण के लिए केवल पढ़ने की अनुमति वाली API कुंजी बनाएं। एक नई API कुंजी बनाते समय, केवल पढ़ने की कुंजी बनाने के लिए **लिखने की पहुंच सक्षम करें** को अनचेक करें।
   
   केवल पढ़ने की कुंजी कर सकती हैं:
   - भुगतान, सब्सक्रिप्शन, और ग्राहक डेटा प्राप्त करें
   - उत्पादों की सूची बनाएं और विवरण प्राप्त करें
   - रिपोर्ट और विश्लेषण तक पहुँचें
   
   केवल पढ़ने की कुंजी **नहीं कर सकती**:
   - भुगतान बनाना या संशोधित करना
   - सब्सक्रिप्शन अपडेट करना
   - उत्पाद कॉन्फ़िगरेशन बदलना
   
   <Tip>
   डैशबोर्ड एकीकरण, विश्लेषण उपकरण, और किसी भी सिस्टम के लिए "लिखने की पहुंच सक्षम करें" को अनचेक करें जिसे केवल डेटा देखने की आवश्यकता है बिना परिवर्तन किए।
   </Tip>

4. **UPI ऑटोपे और RuPay कार्ड सब्सक्रिप्शन - पूर्ण रोलआउट**  
   UPI ऑटोपे और RuPay कार्ड सब्सक्रिप्शन अब सभी व्यवसायों के लिए उपलब्ध हैं। यह भारतीय ग्राहकों के लिए आवर्ती भुगतान सक्षम करता है:
   
   - **UPI ऑटोपे**: UPI जनादेश के माध्यम से स्वचालित आवर्ती शुल्क
   - **RuPay कार्ड**: भारत के घरेलू कार्ड नेटवर्क के लिए सब्सक्रिप्शन समर्थन
   
   <Check>
   परिणाम: RBI-अनुरूप जनादेश प्रबंधन के साथ भारतीय बाजार के लिए विस्तारित सब्सक्रिप्शन भुगतान विकल्प।
   </Check>
   
   अधिक जानें: [RBI-अनुरूप जनादेश के साथ सब्सक्रिप्शन](/features/subscription#subscriptions-with-rbi-compliant-mandates)

5. **योजना परिवर्तन पूर्वावलोकन API**  
   एक नया API एंडपॉइंट जो सब्सक्रिप्शन योजना परिवर्तनों का पूर्वावलोकन करने के लिए है, इससे पहले कि आप उन्हें लागू करें। सटीक शुल्क राशि प्राप्त करें और देखें कि परिवर्तन के बाद सब्सक्रिप्शन कैसा दिखेगा - बिना किसी वास्तविक संशोधन के।
   

   ```javascript
   const preview = await client.subscriptions.previewChangePlan('sub_123', {
     product_id: 'prod_pro_plan',
     proration_billing_mode: 'prorated_immediately',
     quantity: 1
   });
   
   console.log('Charge amount:', preview.immediate_charge.summary);
   console.log('New plan details:', preview.new_plan);
   ```

   
   इसका उपयोग करें:
   - ग्राहकों को दिखाने के लिए कि वे अपग्रेड/डाउनग्रेड की पुष्टि करने से पहले वास्तव में क्या भुगतान करेंगे
   - मूल्य निर्धारण कैलकुलेटर और योजना तुलना उपकरण बनाने के लिए
   - मूल्य निर्धारण पारदर्शिता में सुधार करके समर्थन टिकटों को कम करने के लिए
   
   अधिक जानें: [योजना परिवर्तन पूर्वावलोकन API](/api-reference/subscriptions/preview-change-plan)
