## नई विशेषताएँ 🚀

1. **`subscription.updated` Webhook**  
   सब्सक्रिप्शन ऑब्जेक्ट के प्रत्येक अपडेट पर सक्रिय होने वाला एक नया webhook इवेंट, जो आपको आपकी सब्सक्रिप्शन में किसी भी बदलाव के बारे में वास्तविक समय सूचनाएँ देता है।
   
   इस webhook का उपयोग करें:
   - जब सब्सक्रिप्शन विवरण बदलें तो त्वरित सूचनाएँ प्राप्त करने के लिए
   - अपने एप्लिकेशन स्टेट को Dodo Payments के साथ सिंक्रनाइज़ रखने के लिए
   - API को पोल किए बिना सब्सक्रिप्शन अपडेट पर प्रतिक्रिया देने के लिए
   
   <Tip>
   रियल-टाइम सब्सक्रिप्शन परिवर्तनों के साथ सिस्टम को सिंक में रखने और पोलिंग की आवश्यकता को समाप्त करने के लिए `subscription.updated` सदस्यता लें।
   </Tip>
   
   अधिक जानें: [Subscription Webhooks](/developer-resources/webhooks/intents/subscription)

2. **चेकआउट में न्यूनतम पता समर्थन**  

   <Frame>
   <img src="/images/changelog/minimal-address.png" alt="Minimal Address Support in Checkout" />
   </Frame>

   नए न्यूनतम पता संग्रह मोड के साथ चेकआउट घर्षण को कम करें। पूर्ण पते के विवरण एकत्र करने के बजाय, न्यूनतम पता मोड केवल निम्नलिखित एकत्र करता है:
   
   - **देश**: हमेशा कर निर्धारण के लिए आवश्यक
   - **ZIP/Postal code**: केवल उन क्षेत्रों में एकत्र किया जाता है जहां यह बिक्री कर, VAT, या GST की गणना के लिए आवश्यक होता है
   
   <Check>
   परिणाम: ग्राहकों के लिए कम फ़ॉर्म फ़ील्ड के साथ चेकआउट पूरा करना बहुत तेज़ हो गया।
   </Check>
   
   **न्यूनतम पता सक्षम करें:**

   <Info>

   </Info>

3. **Read-Only API Keys**  
   सुरक्षित एकीकरण के लिए केवल-पठन अनुमतियों के साथ API कुंजी बनाएं। नई API कुंजी बनाते समय, केवल-पठन कुंजी बनाने के लिए **Enable write access** को अनचेक करें।
   
   केवल-पठन कुंजियाँ निम्न कर सकती हैं:
   - भुगतान, सब्सक्रिप्शन, और ग्राहक डेटा प्राप्त करना
   - उत्पादों की सूची बनाना और विवरण प्राप्त करना
   - रिपोर्ट और विश्लेषण तक पहुंच
   
   केवल-पठन कुंजियाँ **नहीं कर सकतीं**:
   - भुगतान बनाना या संशोधित करना
   - सब्सक्रिप्शन अपडेट करना
   - उत्पाद विन्यास बदलना
   
   <Tip>
   डैशबोर्ड एकीकरण, विश्लेषण उपकरण, और किसी भी ऐसे सिस्टम के लिए "Enable write access" को अनचेक करें जिन्हें केवल डेटा देखना होता है बिना परिवर्तन किए।
   </Tip>

4. **UPI AutoPay & RuPay Card Subscriptions - Full Rollout**  
   UPI AutoPay और RuPay कार्ड सब्सक्रिप्शन अब सभी व्यवसायों के लिए उपलब्ध हैं। इससे भारतीय ग्राहकों के लिए निम्न के साथ आवर्ती भुगतान सक्षम होते हैं:
   
   - **UPI AutoPay**: UPI मेंडेट के जरिए स्वचालित आवर्ती शुल्क
   - **RuPay Cards**: भारत के घरेलू कार्ड नेटवर्क के लिए सब्सक्रिप्शन समर्थन
   
   <Check>
   परिणाम: RBI-अनुपालन वाली मेंडेट हैंडलिंग के साथ भारतीय बाजार के लिए चुनौतीपूर्ण सब्सक्रिप्शन भुगतान विकल्पों का विस्तार।
   </Check>
   
   अधिक जानें: [Subscriptions with RBI-Compliant Mandates](/features/subscription#subscriptions-with-rbi-compliant-mandates)

5. **योजना परिवर्तन पूर्वावलोकन API**  
   एक नया API एंडपॉइंट जो सब्सक्रिप्शन योजना परिवर्तनों का पूर्वावलोकन करने के लिए है, इससे पहले कि आप उन्हें लागू करें। सटीक शुल्क राशि प्राप्त करें और देखें कि परिवर्तन के बाद सब्सक्रिप्शन कैसा दिखेगा - बिना किसी वास्तविक संशोधन के।
   

   ```javascript
   const preview = await client.subscriptions.previewChangePlan('sub_123', {
     product_id: 'prod_pro_plan',
     proration_billing_mode: 'prorated_immediately',
     quantity: 1
   });
   
   console.log('Charge amount:', preview.immediate_charge.summary);
   console.log('New plan details:', preview.new_plan);
   ```

   
   इसका उपयोग करें:
   - ग्राहकों को दिखाने के लिए कि वे अपग्रेड/डाउनग्रेड की पुष्टि करने से पहले वास्तव में क्या भुगतान करेंगे
   - मूल्य निर्धारण कैलकुलेटर और योजना तुलना उपकरण बनाने के लिए
   - मूल्य निर्धारण पारदर्शिता में सुधार करके समर्थन टिकटों को कम करने के लिए
   
   अधिक जानें: [योजना परिवर्तन पूर्वावलोकन API](/api-reference/subscriptions/preview-change-plan)
