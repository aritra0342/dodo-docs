## API कुंजी प्रबंधन और प्रमाणीकरण

<Steps>
<Step title="Access API Keys">
अपनी डैशबोर्ड में **Developer → API Keys** पर जाएँ ताकि आप अपनी कुंजियों का प्रबंधन कर सकें।
</Step>

<Step title="Generate a New Key">
**Add API Key** चुनें, एक वर्णनात्मक नाम दें, और लिखने की पहुंच को कॉन्फ़िगर करें:

- **लिखने की पहुँच सक्षम करें** (चेक किया गया): सभी API संचालन के लिए पूर्ण पढ़ने और लिखने की अनुमति
- **लिखने की पहुँच सक्षम करें** (अनचेक किया गया): केवल पढ़ने की अनुमति - केवल डेटा (भुगतान, सदस्यताएँ, ग्राहक, उत्पाद) प्राप्त कर सकते हैं। संसाधनों को बनाने या संशोधित नहीं कर सकते।

<Tip>
डैशबोर्ड एकीकरणों, विश्लेषण उपकरणों, और किसी भी सिस्टम के लिए "Enable write access" को अनचेक करें जिसे केवल डेटा देखने की आवश्यकता होती है बिना कोई बदलाव किए।
</Tip>
</Step>

<Step title="Store Your Key Securely">
जनरेट किया गया कुंजी तुरंत कॉपी करें और सुनिश्चित करें कि इसे सुरक्षित रूप से संग्रहित किया गया है।
</Step>

<Step title="Authenticate Your API Requests">
सभी अनुरोधों को प्रमाणित करने के लिए अपनी API कुंजियों का उपयोग करें। निम्नलिखित प्राधिकरण प्रारूप लागू करें:

```bash 
Authorization: Bearer YOUR_API_KEY
```

<Warning>
अपने गुप्त API कुंजियों को कभी भी क्लाइंट-साइड कोड या सार्वजनिक रिपॉजिटरीज़ में उजागर न करें।
</Warning>
</Step>
</Steps>

## दर सीमाएँ

हमारा API बर्स्ट सुरक्षा के साथ द्वि-खिड़की रेट लिमिट प्रणाली का उपयोग करता है। सीमाएं आपके प्रमाणीकरण तरीके और व्यवसाय टियर के आधार पर लागू होती हैं।

### डिफ़ॉल्ट सीमाएं (टियर 0)

| विंडो | सीमा |
|--------|-------|
| प्रति सेकंड (बर्स्ट) | 40 requests |
| प्रति मिनट (टिकाऊ) | 240 requests |

### बिज़नेस टियर्स

बढ़ी हुई API आवश्यकताओं वाले व्यवसायों के लिए उच्च रेट लिमिट्स उपलब्ध हैं:

| टियर | बर्स्ट (प्रति सेकंड) | टिकाऊ (प्रति मिनट) |
|------|-------------------|------------------------|
| टियर 0 (डिफ़ॉल्ट) | 40 | 240 |
| टियर 1 | 100 | 1,000 |
| टियर 2 | 500 | 5,000 |

<Tip>
अपने व्यवसाय को उच्च रेट लिमिट टियर में अपग्रेड करने के लिए सपोर्ट से संपर्क करें।
</Tip>

### अप्रमाणित अनुरोध

वैध प्रमाणीकरण हेडर्स के बिना अनुरोधों को IP पते के अनुसार रेट लिमिट किया जाता है:

| विंडो | सीमा |
|--------|-------|
| प्रति सेकंड (बर्स्ट) | 20 requests |
| प्रति मिनट (टिकाऊ) | 100 requests |

### रेट लिमिट हेडर

इन प्रतिक्रिया हेडर्स के साथ अपनी उपयोग दर की निगरानी करें:

- `X-RateLimit-Limit` - अधिकतम अनुरोधों की अनुमति
- `X-RateLimit-Remaining` - वर्तमान विंडो में शेष अनुरोध
- `X-RateLimit-Reset` - वह समय जब रेट लिमिट रीसेट होती है

<Note>
जब आप रेट लिमिट्स पार कर लेते हैं, तो API एक `429 Too Many Requests` प्रतिक्रिया लौटाती है। अपनी पुन:प्रयास लॉजिक में घातांकीय बैकऑफ़ लागू करें।
</Note>
