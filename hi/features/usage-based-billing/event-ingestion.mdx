इवेंट उपयोग-आधारित बिलिंग की नींव हैं। बिल योग्य क्रियाएँ होने पर इवेंट भेजें, और मीटर उन्हें चार्ज में संचित करते हैं।

<Card title="API Reference - Events Ingestion" icon="code" href="/api-reference/usage-events/ingest-events">
उदाहरणों और प्रतिक्रिया कोड के साथ पूर्ण API दस्तावेज़।
</Card>

## इवेंट संरचना

{/* LOCKED_PATTERN_4d7f2e9ce45e7323dd2d626df4a7c5b8 */}
<ParamField body="event_id" type="string" required>
विशिष्ट पहचानकर्ता। UUID का उपयोग करें या ग्राहक ID + टाइमस्टैम्प + क्रिया को मिलाएं।
</ParamField>

<ParamField body="customer_id" type="string" required>
Dodo Payments ग्राहक ID। यह एक वैध मौजूदा ग्राहक होना चाहिए।
</ParamField>

<ParamField body="event_name" type="string" required>
आपके मीटर की घटना नाम से मेल खाता हुआ घटना प्रकार (केस-संवेदनशील)। उदाहरण: `api.call`, `image.generated`
</ParamField>
</Accordion>

{/* LOCKED_PATTERN_94ee55474fc7877494948c444fe08c91 */}

<ParamField body="timestamp" type="string">
ISO 8601 समय-छाप। यदि छोड़ा गया तो डिफ़ॉल्ट सर्वर समय। देरी/बैच घटनाओं के लिए सटीक बिलिंग हेतु शामिल करें।
</ParamField>

<ParamField body="metadata" type="object">
संग्रहण और फ़िल्टरिंग के लिए अतिरिक्त गुण:
- संख्यात्मक मान: `bytes`, `tokens`, `duration_ms`
- फ़िल्टर: `endpoint`, `method`, `quality`

```javascript
metadata: {
  endpoint: "/v1/orders",
  method: "POST",
  tokens: 1024
}
```

</ParamField>
</Accordion>

## इवेंट भेजना

<CodeGroup>

```javascript Single Event
await fetch('https://test.dodopayments.com/events/ingest', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.DODO_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    events: [{
      event_id: "api_call_1234",
      customer_id: "cus_abc123",
      event_name: "api.call",
      metadata: { endpoint: "/v1/orders" }
    }]
  })
});
```

```javascript Batch Events
await fetch('https://test.dodopayments.com/events/ingest', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.DODO_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    events: [
      { event_id: "call_1", customer_id: "cus_abc", event_name: "api.call" },
      { event_id: "call_2", customer_id: "cus_abc", event_name: "api.call" },
      { event_id: "call_3", customer_id: "cus_abc", event_name: "api.call" }
    ]
  })
});
```

```python Python
import requests

requests.post(
    'https://test.dodopayments.com/events/ingest',
    headers={'Authorization': f'Bearer {api_key}', 'Content-Type': 'application/json'},
    json={'events': [{'event_id': 'call_1', 'customer_id': 'cus_abc', 'event_name': 'api.call'}]}
)
```

```curl cURL
curl -X POST 'https://test.dodopayments.com/events/ingest' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{"events": [{"event_id": "call_1", "customer_id": "cus_abc", "event_name": "api.call"}]}'
```

</CodeGroup>

<Tip>
बेहतर प्रदर्शन के लिए प्रति अनुरोध 100 घटनाओं तक का बैच।
</Tip>

</Accordion>

{/* LOCKED_PATTERN_90e261cbcc49667becacc297210d42c3 */}

</Accordion>

</Accordion>
</AccordionGroup>

## समस्या निवारण

<AccordionGroup>
{/* LOCKED_PATTERN_fc1788ddf0982a3a2fe9f6e81812c541 */}
- घटना नाम को मीटर से बिल्कुल मेल करना चाहिए (केस-संवेदनशील)
- ग्राहक ID मौजूद होना चाहिए
- जांचें कि मीटर फ़िल्टर घटनाओं को बाहर नहीं कर रहे हैं
- सुनिश्चित करें कि समय-छाप हाल ही के हैं
</Accordion>

{/* LOCKED_PATTERN_1af6f154c82eaebbb38a748cef2be54a */}
API कुंजी सही है और स्वरूप का उपयोग करें: `Bearer YOUR_API_KEY`
</Accordion>

{/* LOCKED_PATTERN_bfe0427c6207c44242acb102b2467be5 */}
सुनिश्चित करें कि सभी आवश्यक फ़ील्ड मौजूद हैं: `event_id`, `customer_id`, `event_name`
</Accordion>

{/* LOCKED_PATTERN_61cd934926cd73f663eac6935ef56a1b */}
- मेटाडेटा कुंजी को मीटर के "ओवर प्रॉपर्टी" से बिल्कुल मिलना चाहिए
- स्ट्रिंग नहीं, संख्याएँ उपयोग करें: `tokens: 150` ना कि `tokens: "150"`
</Accordion>
</AccordionGroup>
