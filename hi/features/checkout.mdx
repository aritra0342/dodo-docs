## अनुकूलनशील मुद्रा

अनुकूलनशील मुद्रा ग्राहकों को उनकी पसंदीदा स्थानीय मुद्रा में भुगतान करने की अनुमति देती है, जिससे विश्वास और रूपांतरण दरें बढ़ती हैं।

### यह कैसे काम करता है

1. **सक्षम करें**: अपने व्यवसाय सेटिंग्स से अनुकूलनशील मुद्रा सक्षम करें
2. **चुनें**: ग्राहक चेकआउट पर सीधे मुद्राएँ बदल सकते हैं
3. **परिवर्तित करें**: मूल्य वास्तविक समय के FX दरों का उपयोग करके गतिशील रूप से परिवर्तित होते हैं
4. **प्रदर्शित करें**: अंतिम देय राशि भुगतान से पहले पारदर्शी रूप से दिखाई जाती है

डोडो पेमेंट्स चेकआउट पृष्ठ पर कई भाषाओं का समर्थन करता है, जिससे ग्राहक अपनी पसंदीदा भाषा में भुगतान पूरा कर सकते हैं।

## स्वचालित कर गणना

कर ग्राहक के बिलिंग स्थान के आधार पर स्वतः गणना किए जाते हैं, जिससे GST, VAT, और बिक्री कर आवश्यकताओं के साथ अनुपालन बिना मैन्युअल सेटअप के सुनिश्चित होता है।

### कर गणना कैसे काम करती है

<Steps>
<Step title="Location Detection">
  कर नियम ग्राहक के देश (और जहां लागू हो वहां क्षेत्र) के आधार पर लागू किए जाते हैं।
</Step>

<Step title="Dynamic Updates">
  कर राशि स्वतः अपडेट होती है जब:
  - देश बदलता है
  - पता अपडेट होता है
</Step>

<Step title="Transparent Display">
  भुगतान से पहले अंतिम कर विवरण स्पष्ट रूप से दिखाया जाता है।
</Step>
</Steps>

<Tip>
कर गणना पूरी तरह स्वचालित है। मानक डिजिटल सामान और SaaS उत्पादों के लिए किसी मैन्युअल कॉन्फ़िगरेशन की आवश्यकता नहीं है।
</Tip>

## व्यापार कर आईडी समर्थन

पंजीकृत व्यवसायों के लिए, चेकआउट ग्राहकों को उनकी व्यापार कर आईडी (जैसे VAT/GST नंबर) दर्ज करने की अनुमति देता है।

### जब कर आईडी दर्ज की जाती है तो क्या होता है

- कर पात्रता वास्तविक समय में सत्यापित की जाती है
- लागू कर छूट या रिवर्स-चार्ज नियम लागू किए जाते हैं
- कर राशि चेकआउट पर तुरंत अपडेट होती है

</Info>

## डिस्काउंट कोड

ग्राहक सीधे चेकआउट पृष्ठ पर डैशबोर्ड में बनाए गए डिस्काउंट या प्रोमो कोड लागू कर सकते हैं।

### चेकआउट अनुभव

1. ग्राहक डिस्काउंट कोड दर्ज करता है
2. डिस्काउंट तुरंत सत्यापित होता है
3. अद्यतित मूल्य और बचत स्पष्ट रूप से दिखाई देती है

<Frame>
  <img src="/images/checkout/c4.png" alt="चेकआउट पर डिस्काउंट कोड प्रविष्टि" style={{ width: '70%' }}/>
</Frame>

### API एकीकरण

डिस्काउंट कोड को पहले से लागू करें या डिस्काउंट इनपुट फ़ील्ड को सक्षम करें:

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    { product_id: 'prod_abc', quantity: 1 }
  ],
  discount_code: 'WELCOME20', // Pre-apply a code
  feature_flags: {
    allow_discount_code: true // Show discount input field
  },
  return_url: 'https://yoursite.com/return'
});
```

## स्मार्ट पता संग्रह

तेज़ पूरा करने के लिए चेकआउट लचीले फ़ॉर्म से पता इनपुट का समर्थन करता है।

### उपलब्ध विकल्प

| विकल्प | विवरण |
|--------|-------------|
| **Google Address Autofill** | ऑटोकम्पलीट के साथ तेज़ चयन |
| **Manual Entry** | पूर्ण पते के लिए पूरा नियंत्रण |
| **Country Selection** | कर और अनुपालन लॉजिक को संचालित करता है |

<Tip>
पता संग्रह गति, सटीकता और वैश्विक कवरेज का संतुलन करता है ताकि अनुपालन सुनिश्चित करते हुए कन्वर्ज़न अधिकतम हो सके।
</Tip>

## कस्टम फ़ील्ड

अपने चेकआउट के दौरान ग्राहकों से अतिरिक्त जानकारी एकत्र करें, जैसे कंपनी का नाम, टीम का आकार, रेफ़रल स्रोत या कोई अन्य व्यवसाय-विशिष्ट जानकारी।

### उपलब्ध फ़ील्ड प्रकार

| प्रकार | विवरण |
|------|-------------|
| `text` | एक-पंक्ति पाठ इनपुट |
| `number` | संख्यात्मक इनपुट |
| `email` | मान्यकरण के साथ ईमेल पता |
| `url` | मान्यकरण के साथ URL |
| `date` | दिनांक पिकर |
| `dropdown` | पूर्वनिर्धारित विकल्पों में से चयन |
| `boolean` | हाँ/नहीं टॉगल |

### उदाहरण

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    { product_id: 'prod_abc', quantity: 1 }
  ],
  custom_fields: [
    {
      key: 'company_name',
      label: 'Company Name',
      field_type: 'text',
      required: true
    },
    {
      key: 'team_size',
      label: 'Team Size',
      field_type: 'dropdown',
      required: true,
      options: ['1-10', '11-50', '51-200', '200+']
    }
  ],
  return_url: 'https://yoursite.com/return'
});
```

<Info>
ग्राहक प्रतिक्रियाएँ स्वतः वेबहुक पेलोड (`payment.succeeded`, `subscription.created`) और API प्रतिक्रियाओं में `custom_field_responses` ऐरे के माध्यम से शामिल होती हैं। आप प्रति चेकआउट सेशन अधिकतम 5 कस्टम फ़ील्ड परिभाषित कर सकते हैं।
</Info>

<Card title="Custom Fields Guide" icon="input-text" href="/developer-resources/checkout-session#15-collecting-custom-fields">
  कस्टम फ़ील्ड कॉन्फ़िगरेशन और प्रतिक्रियाओं तक पहुँचने के बारे में अधिक जानें।

## कलेक्शन चेकआउट

प्रोडक्ट कलेक्शन एक एकीकृत चेकआउट अनुभव देता है जहाँ ग्राहक एक ही चेकआउट में कई संबंधित उत्पाद (जैसे Starter, Pro, Enterprise प्लान) देख और चुन सकते हैं।

### यह कैसे काम करता है

1. **सभी उत्पाद दिखाए जाते हैं**: ग्राहक कलेक्शन में हर सक्रिय उत्पाद देखते हैं
2. **पहला उत्पाद पूर्व-चयनित**: कलेक्शन में पहला उत्पाद स्वतः चुना जाता है
3. **विकल्पों की तुलना**: ग्राहक चुनने से पहले मूल्य और सुविधाओं की तुलना कर सकते हैं
4. **एकल चयन**: किसी उत्पाद का चयन करने के बाद, चेकआउट सामान्य भुगतान प्रवाह के साथ आगे बढ़ता है

### कलेक्शन चेकआउट बनाना

```typescript
const session = await client.checkoutSessions.create({
  product_collection_id: 'pdc_abc123',
  product_cart: [], // Required: pass an empty array for collection checkout
  return_url: 'https://yoursite.com/return'
});
```

<Warning>
जब `product_collection_id` का उपयोग कर रहे हों, तो एक खाली `product_cart` ऐरे पास करें। डिस्काउंट कोड को सेशन निर्माण पर पूर्व-लागू नहीं किया जा सकता।
</Warning>

## चेकआउट सेशन कॉन्फ़िगरेशन

Checkout Sessions API का उपयोग करके चेकआउट व्यवहार को नियंत्रित करें:

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    { product_id: 'prod_abc', quantity: 1 }
  ],
  customer: {
    email: 'customer@example.com',
    name: 'Jane Doe'
  },
  billing_currency: 'EUR', // Set specific currency
  discount_code: 'PROMO10',
  feature_flags: {
    allow_discount_code: true
  },
  return_url: 'https://yoursite.com/return',
  metadata: {
    order_ref: 'ORD-12345'
  }
});
```

<Info>
भुगतान के बाद, ग्राहकों को उनके `return_url` पर क्वेरी पैरामीटर्स सहित स्वतः पुनर्निर्देशित किया जाता है -- जिसमें `payment_id` या `subscription_id`, `status`, `email`, और `license_key` (यदि लागू हो)। पूरी सूची के लिए [Checkout Sessions guide](/developer-resources/checkout-session#request-body) देखें।
</Info>

## चेकआउट थीम कस्टमाइज़ेशन

API के माध्यम से चेकआउट सेशन बनाते समय `customization.theme_config` पैरामीटर का उपयोग करके चेकआउट पृष्ठ की उपस्थिति को अपने ब्रांड के अनुरूप अनुकूलित करें। लाइट और डार्क मोड दोनों के लिए रंग, फॉन्ट, बॉर्डर रेडियस और बटन टेक्स्ट कॉन्फ़िगर करें।

<Frame>
  <img src="/images/checkout/theme-example.png" alt="कस्टम थीम वाला चेकआउट पृष्ठ"/>
</Frame>

<Info>
यह अनुभाग `customization.theme_config` का उपयोग करके सर्वर-साइड API थीम कॉन्फ़िगरेशन को कवर करता है। यदि आप Checkout SDK (ओवरले या इनलाइन चेकआउट) का उपयोग कर रहे हैं, तो [Overlay Checkout](/developer-resources/overlay-checkout#theme-customization) या [Inline Checkout](/developer-resources/inline-checkout#theme-customization) में थीम कस्टमाइज़ेशन अनुभाग देखें जो camelCase गुणों (जैसे `bgPrimary` के बजाय `bg_primary`) का उपयोग करते हैं।
</Info>

### थीम कॉन्फ़िगरेशन विकल्प

| गुण | विवरण |
|----------|-------------|
| `light` | लाइट मोड के लिए रंग कॉन्फ़िगरेशन |
| `dark` | डार्क मोड के लिए रंग कॉन्फ़िगरेशन |
| `font_primary_url` | प्राइमरी फॉन्ट के लिए URL |
| `font_secondary_url` | सेकेंडरी फॉन्ट के लिए URL |
| `font_size` | फॉन्ट आकार: `xs`, `sm`, `md`, `lg`, `xl`, `2xl` |
| `font_weight` | फॉन्ट वेट: `normal`, `medium`, `bold`, `extraBold` |
| `radius` | UI एलिमेंट्स के लिए बॉर्डर रेडियस (जैसे `4px`, `0.5rem`, `8px`) |
| `pay_button_text` | पे बटन के लिए कस्टम टेक्स्ट (जैसे "Complete Purchase", "Subscribe Now") |

### रंग कॉन्फ़िगरेशन (लाइट/डार्क मोड)

प्रत्येक मोड (`light` और `dark`) निम्नलिखित रंग गुणों का समर्थन करता है:

| गुण | विवरण |
|----------|-------------|
| `bg_primary` | प्राइमरी पृष्ठभूमि रंग |
| `bg_secondary` | सेकेंडरी पृष्ठभूमि रंग |
| `text_primary` | प्राइमरी टेक्स्ट रंग |
| `text_secondary` | सेकेंडरी टेक्स्ट रंग |
| `text_placeholder` | टेक्स्ट प्लेसहोल्डर रंग |
| `text_error` | टेक्स्ट त्रुटि रंग |
| `text_success` | टेक्स्ट सफलता रंग |
| `border_primary` | प्राइमरी बॉर्डर रंग |
| `border_secondary` | सेकेंडरी बॉर्डर रंग |
| `button_primary` | प्राइमरी बटन पृष्ठभूमि रंग |
| `button_primary_hover` | प्राइमरी बटन होवर रंग |
| `button_secondary` | सेकेंडरी बटन पृष्ठभूमि रंग |
| `button_secondary_hover` | सेकेंडरी बटन होवर रंग |
| `button_text_primary` | प्राइमरी बटन टेक्स्ट रंग |
| `button_text_secondary` | सेकेंडरी बटन टेक्स्ट रंग |
| `input_focus_border` | इनपुट फोकस बॉर्डर रंग |

<Info>
सभी रंग फ़ील्ड मानक CSS रंग प्रारूप स्वीकार करते हैं:
- Hex: `#fff`, `#ffffff`, `#ffffffff`
- RGB/RGBA: `rgb(255, 255, 255)`, `rgba(255, 255, 255, 0.5)`
- HSL/HSLA: `hsl(120, 100%, 50%)`, `hsla(120, 100%, 50%, 0.5)`
- Named colors: `red`, `blue`, `transparent`
</Info>

### उदाहरण

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    { product_id: 'prod_abc', quantity: 1 }
  ],
  customization: {
    theme_config: {
      // Custom fonts
      font_primary_url: 'https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600&display=swap',
      font_size: 'md',
      font_weight: 'medium',
      radius: '8px',
      pay_button_text: 'Complete Purchase',
      
      // Light mode colors
      light: {
        bg_primary: '#ffffff',
        bg_secondary: '#f5f5f5',
        text_primary: '#1a1a1a',
        text_secondary: '#666666',
        button_primary: '#0066ff',
        button_primary_hover: '#0052cc',
        button_text_primary: '#ffffff',
        border_primary: '#e0e0e0'
      },
      
      // Dark mode colors
      dark: {
        bg_primary: '#1a1a1a',
        bg_secondary: '#2d2d2d',
        text_primary: '#ffffff',
        text_secondary: '#a0a0a0',
        button_primary: '#3385ff',
        button_primary_hover: '#4d99ff',
        button_text_primary: '#ffffff',
        border_primary: '#404040'
      }
    }
  },
  return_url: 'https://yoursite.com/return'
});
```

<Tip>
आपको सभी रंग गुण निर्दिष्ट करने की आवश्यकता नहीं है। किसी भी अवर्णित गुण के लिए डिफ़ॉल्ट थीम मानों का उपयोग किया जाएगा।
</Tip>
