## ग्राहक वॉलेट क्या हैं?

ग्राहक वॉलेट वित्तीय संतुलन खाते हैं जो आपके उपयोगकर्ताओं के लिए वास्तविक धन रखते हैं। जब आप उनका खाता बनाते हैं, तो प्रत्येक ग्राहक को स्वतः एक वॉलेट मिलता है। आप इन वॉलेट का उपयोग कर सकते हैं:

- **भविष्य के सदस्यता भुगतानों के लिए पूर्व-भुगतान निधि रखें**
- **रिफंड को कार्ड रिफंड के बजाय वॉलेट शेष के रूप में संभालें**
- **वेलकम बोनस या लॉयल्टी रिवार्ड जैसे प्रचारात्मक शेष जारी करें**
- **बिलिंग के दौरान वॉलेट फंड को चालानों पर स्वचालित रूप से लागू करें**
- **विस्तृत खाता-पुस्तिका इतिहास के साथ मौद्रिक लेन-देन को ट्रैक करें**

<Info>
जब आप नया ग्राहक बनाते हैं, तो डोडो पेमेंट्स स्वतः शून्य शेष के साथ एक वॉलेट बनाता है। यह हमारी API के माध्यम से तुरंत धन प्राप्त करने के लिए तैयार है।
</Info>

<Warning>
**ग्राहक वॉलेट ≠ क्रेडिट-आधारित बिलिंग**

ग्राहक वॉलेट **वास्तविक मौद्रिक शेष** (USD, INR) रखते हैं जिन्हें चालानों और सदस्यता भुगतान पर लागू किया जा सकता है।

यदि आप वर्चुअल उपयोग इकाइयों (API कॉल, टोकन, कंप्यूट घंटे) को ट्रैक करना चाह रहे हैं, तो इसके बजाय [Credit-Based Billing](/features/credit-based-billing) देखें।
</Warning>

## यह कैसे काम करता है

ग्राहक वॉलेट ऐसे धन रखते हैं जिन्हें ग्राहक अपनी खरीदों पर लागू कर सकते हैं। जब ग्राहक के पास कोई चालान या आवर्ती सदस्यता लेन-देन होता है, तो पहले उनके वॉलेट शेष की जांच की जाती है। किसी भी उपलब्ध धन को उनके प्राथमिक भुगतान विधि को चार्ज करने से पहले स्वचालित रूप से चालान पर लागू किया जाता है।

### स्वचालित सेटअप

जब आप नया ग्राहक बनाते हैं, तो डोडो पेमेंट्स स्वतः एक वॉलेट बनाता है जिसमें शून्य शेष होता है। यह हमारी API के माध्यम से तुरंत धन प्राप्त करने के लिए तैयार है।

### बहु-मुद्रा समर्थन

प्रत्येक वॉलेट अलग-अलग मुद्राओं में शेष रख सकता है:

<ResponseField name="USD Balance" type="integer">
अमेरिकी डॉलर में शेष (सेंट में संग्रहीत)
</ResponseField>

<ResponseField name="INR Balance" type="integer">
भारतीय रुपये में शेष (पैसे में संग्रहीत)
</ResponseField>

<Info>
वर्तमान में केवल **USD** और **INR** शेष उपलब्ध हैं। और मुद्राएँ जल्द ही आ रही हैं।
</Info>

## वॉलेट के साथ कार्य करना

### ग्राहक शेष जांचें

देखें कि ग्राहकों के पास सभी मुद्राओं में कितना धन है। यह खरीद को संसाधित करने से पहले उपलब्ध शेष की जांच करने या अपने एप्लिकेशन UI में शेष दिखाने के लिए उपयोगी है।

## वास्तविक दुनिया के उदाहरण

### वॉलेट में रिफंड

जब ग्राहक रिफंड का अनुरोध करता है, तो आप राशि को उनके वॉलेट शेष में जोड़ सकते हैं बजाय पारंपरिक कार्ड रिफंड के। इससे भविष्य की खरीदों के लिए धन आपके इकोसिस्टम में रहता है।

```javascript
async function refundToWallet(customerId, refundAmount, originalPaymentId) {
  await client.customers.wallets.ledgerEntries.create(customerId, {
    amount: refundAmount, // Amount in cents
    currency: 'USD',
    entry_type: 'credit',
    reason: `Refund for payment ${originalPaymentId}`,
    idempotency_key: `refund_${originalPaymentId}`
  });
}
```

### वेलकम बोनस / प्रचारात्मक शेष

नए ग्राहकों को उनकी पहली खरीद को प्रोत्साहित करने के लिए मौद्रिक वेलकम बोनस दें।

```javascript
async function addWelcomeBonus(customerId) {
  await client.customers.wallets.ledgerEntries.create(customerId, {
    amount: 1000, // $10.00 promotional balance
    currency: 'USD',
    entry_type: 'credit',
    reason: 'Welcome bonus - $10 promotional balance',
    idempotency_key: `welcome_${customerId}`
  });
}
```

### वॉलेट से सदस्यता भुगतान

ग्राहक के वॉलेट से धन घटाकर सदस्यता शुल्क या मैनुअल खरीद को कवर करें।

```javascript
async function deductForPurchase(customerId, purchaseAmount, purchaseId) {
  try {
    await client.customers.wallets.ledgerEntries.create(customerId, {
      amount: purchaseAmount,
      currency: 'USD',
      entry_type: 'debit',
      reason: `Subscription charge - monthly billing`,
      idempotency_key: `charge_${purchaseId}`
    });
  } catch (error) {
    if (error.status === 400) {
      console.log('Insufficient wallet balance');
    }
  }
}
```

## सर्वोत्तम प्रथाएँ

### डुप्लिकेट लेन-देन रोकें

यह सुनिश्चित करने के लिए इडेम्पोटेंसी कुंजियों का उपयोग करें कि आप गलती से एक ही इवेंट के लिए दो बार धन न जोड़ें या घटाएँ।

```javascript
async function addFundsSafely(customerId, amount, reason) {
  const idempotencyKey = `${reason}_${customerId}_${Date.now()}`;
  
  try {
    const result = await client.customers.wallets.ledgerEntries.create(customerId, {
      amount: amount,
      currency: 'USD',
      entry_type: 'credit',
      reason: reason,
      idempotency_key: idempotencyKey
    });
    
    return { success: true, wallet: result };
  } catch (error) {
    if (error.status === 409) {
      // Transaction already processed
      return { success: true, wallet: null, duplicate: true };
    }
    
    throw error;
  }
}
```

### चार्ज करने से पहले शेष जांचें

बड़ा लेन-देन वॉलेट से प्रोसेस करने का प्रयास करने से पहले यह सत्यापित करें कि ग्राहकों के पास पर्याप्त धन है।

```javascript
async function checkBalanceBeforeOperation(customerId, requiredAmount) {
  const wallets = await client.customers.wallets.list(customerId);
  const usdWallet = wallets.items.find(w => w.currency === 'USD');
  
  if (!usdWallet || usdWallet.balance < requiredAmount) {
    throw new Error('Insufficient funds for this operation');
  }
  
  return usdWallet.balance;
}
```

<Warning>
इन फीचर को आगामी रिलीज़ में शामिल करने की योजना है:
</Warning>

- **शेष समाप्ति**: किसी निश्चित अवधि के बाद निधि समाप्त करने के लिए सेट करें
- **बेहतर एनालिटिक्स**: विस्तृत खर्च रिपोर्ट और शेष प्रवृत्ति
- **अधिक वेबहुक**: शेष में बदलाव और कम शेष अलर्ट के लिए वास्तविक समय सूचनाएं

<Tip>
बुनियादी फंडिंग और कटौती संचालन से शुरू करें, फिर जैसे-जैसे आपका व्यवसाय बढ़े, अधिक जटिल स्वचालित बिलिंग वर्कफ़्लो को एकीकृत करें।
</Tip>
