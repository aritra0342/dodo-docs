Dodo Payments कई प्रकार के भुगतान विधियों का समर्थन करता है ताकि आप वैश्विक ग्राहकों को सेवा प्रदान कर सकें। नए भुगतान विधियों को तुरंत चालू करें — कोई अतिरिक्त अनुबंध, इंटीग्रेशन या व्यापारी खातों की आवश्यकता नहीं है।

## कई भुगतान विधियाँ पेश करने का कारण क्या है?

## यह कैसे काम करता है

भुगतान विधियाँ स्वचालित रूप से ग्राहकों को प्रस्तुत की जाती हैं, इस आधार पर:

- **ग्राहक स्थान** — क्षेत्रीय विधियाँ जैसे iDEAL (नीदरलैंड) या UPI (भारत) केवल उन ग्राहकों के लिए प्रकट होती हैं जो उन क्षेत्रों में हैं
- **लेनदेन की मुद्रा** — भुगतान विधियाँ समर्थित मुद्राओं द्वारा फ़िल्टर की जाती हैं
- **डिवाइस प्रकार** — Apple Pay केवल सुसंगत Apple उपकरणों और ब्राउज़रों पर प्रकट होता है
- **लेनदेन का प्रकार** — कुछ विधियाँ सब्सक्रिप्शन का समर्थन नहीं करती हैं या उनमें न्यूनतम राशि होती है

<Info>
**स्वचालित सक्रियण:** सभी भुगतान विधियाँ पूर्व-संरक्षित और Dodo Payments द्वारा प्रबंधित होती हैं। अतिरिक्त सेटअप की आवश्यकता नहीं है — ये उपलब्ध हो जाती हैं जब आपका खाता सत्यापित हो जाता है और लाइव मोड सक्षम होता है।
</Info>

## भुगतान विधियाँ कॉन्फ़िगर करना

डिफ़ॉल्ट रूप से, सभी योग्य भुगतान विधियाँ स्वचालित रूप से प्रकट होती हैं। दिखाए गए विधियों को प्रतिबंधित करने के लिए `allowed_payment_method_types` का उपयोग करें:

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://test.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

### उपलब्ध विधि प्रकार

| प्रकार | विवरण |
| :--- | :---------- |
| `credit` | क्रेडिट कार्ड (Visa, Mastercard, Amex, आदि) |
| `debit` | डेबिट कार्ड |
| `apple_pay` | Apple Pay |
| `google_pay` | Google Pay |
| `amazon_pay` | Amazon Pay |
| `cashapp` | Cash App Pay |
| `klarna` | Klarna BNPL |
| `afterpay_clearpay` | Afterpay / Clearpay |
| `upi_collect` | UPI कलेक्ट (QR कोड) |
| `ideal` | iDEAL (नीदरलैंड) |
| `bancontact_card` | Bancontact (बेल्जियम) |
| `eps` | EPS (ऑस्ट्रिया) |
| `multibanco` | Multibanco (पुर्तगाल) |
| `revolut_pay` | RevolutPay |
| `billie` | Billie B2B BNPL |

<Warning>
हमेशा `credit` और `debit` को एक फॉल्बैक के रूप में शामिल करें। यदि किसी ग्राहक के लिए सभी निर्दिष्ट भुगतान विधियाँ उपलब्ध नहीं हैं, तो चेकआउट असफल हो जाएगा।
</Warning>

## सर्वोत्तम अभ्यास

<AccordionGroup>
<Accordion title="अपने बाजार से मेल खाने वाली विधियाँ शामिल करें">
अपने लक्षित बाजारों के लिए क्षेत्रीय भुगतान विधियाँ शामिल करें। iDEAL नीदरलैंड में ~60% ऑनलाइन भुगतान को कैप्चर करता है; UPI भारत में प्रखंड करता है। इन विधियों को छोड़ना कन्वर्जन को महत्वपूर्ण रूप से नुकसान पहुँचा सकता है।
</Accordion>

<Accordion title="हमेशा कार्ड फॉलबैक प्रदान करें">
विशिष्ट विधियों को लक्षित करते समय भी, `credit` और `debit` को फॉलबैक के रूप में शामिल करें। सभी ग्राहकों के पास हर भुगतान विधि तक पहुंच नहीं होती है।
</Accordion>

<Accordion title="लेनदेन न्यूनतम पर विचार करें">
BNPL विधियों (Klarna, Afterpay) के लिए न्यूनतम $50.01 आवश्यक है। निम्न-मूल्य वाली उत्पादों के लिए उन्हें शामिल न करें — वे बस प्रकट नहीं होंगे, जो ठीक है।
</Accordion>

<Accordion title="क्षेत्रों और उपकरणों पर परीक्षण करें">
लाइव होने से पहले विभिन्न ग्राहक स्थानों, मुद्राओं, और डिवाइस प्रकारों के लिए भुगतान विधियों की उपलब्धता की पुष्टि करने के लिए परीक्षण मोड का उपयोग करें।
</Accordion>
</AccordionGroup>

## समस्या निवारण

<AccordionGroup>
<Accordion title="भुगतान विधि दिखाई नहीं दे रही">
**इन कारकों की जाँच करें:**
1. क्या विधि `allowed_payment_method_types` में शामिल है?
2. क्या ग्राहक स्थान विधि के समर्थित क्षेत्र से मेल खाता है?
3. क्या लेनदेन मुद्रा विधि द्वारा समर्थित है?
4. क्या लेनदेन की राशि न्यूनतम को पूरा करती है (जैसे, BNPL के लिए $50.01)?
5. सब्सक्रिप्शन के लिए: क्या विधि आवर्ती भुगतानों का समर्थन करती है?

**हल:** सभी योग्य विधियों को देखने के लिए `allowed_payment_method_types` को अस्थायी रूप से हटा दें, फिर समस्या को संकरा करें।
</Accordion>

<Accordion title="अजीब विधियाँ दिखाई दे रही हैं">
**संभावित कारण:**
- `allowed_payment_method_types` सही तरीके से पास नहीं हो रहा है
- पैरामीटर गलत तरीके से स्वरूपित (स्ट्रिंग की सूची में होना चाहिए)

**हल:** अपने API अनुरोध का भारांक लॉग करें और पैरामीटर संरचना की पुष्टि करें।
</Accordion>

<Accordion title="क्षेत्रीय विधि उपलब्ध नहीं है">
**जाँच करें:**
- क्या ग्राहक का बिलिंग पता विधि के क्षेत्र से मेल खाता है
- क्या मुद्रा सही तरीके से सेट की गई है (जैसे, iDEAL के लिए EUR)
- क्या ग्राहक अंतर्राष्ट्रीय है तो अनुकूलनात्मक मुद्रा सक्षम है

**हल:** अपने API अनुरोध में बिलिंग पते के देश के कोड और मुद्रा की पुष्टि करें।
</Accordion>

<Accordion title="भारतीय भुगतान विधि समस्याएँ">
UPI और Rupay सब्सक्रिप्शन के लिए अनूठी आवश्यकताएँ हैं, जिनमें 48-घंटे की प्रसंस्करण देरी और RBI आदेश की सीमाएँ शामिल हैं।

**देखें:** <a href="/features/payment-methods/india">भारत भुगतान विधियाँ</a> पृष्ठ पर पूर्ण मार्गदर्शन के लिए।
</Accordion>
</AccordionGroup>
