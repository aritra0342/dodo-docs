## यह कैसे काम करता है

भुगतान विधियाँ स्वचालित रूप से ग्राहकों को प्रस्तुत की जाती हैं, इस आधार पर:

- **ग्राहक स्थान** — क्षेत्रीय विधियाँ जैसे iDEAL (नीदरलैंड) या UPI (भारत) केवल उन ग्राहकों के लिए प्रकट होती हैं जो उन क्षेत्रों में हैं
- **लेनदेन की मुद्रा** — भुगतान विधियाँ समर्थित मुद्राओं द्वारा फ़िल्टर की जाती हैं
- **डिवाइस प्रकार** — Apple Pay केवल सुसंगत Apple उपकरणों और ब्राउज़रों पर प्रकट होता है
- **लेनदेन का प्रकार** — कुछ विधियाँ सब्सक्रिप्शन का समर्थन नहीं करती हैं या उनमें न्यूनतम राशि होती है

<Info>
**स्वचालित सक्रियण:** सभी भुगतान विधियाँ पहले से कॉन्फ़िगर की गई हैं और Dodo Payments द्वारा प्रबंधित की जाती हैं। किसी अतिरिक्त सेटअप की आवश्यकता नहीं है — ये आपके खाते के सत्यापित होने और लाइव मोड सक्षम होने पर उपलब्ध हो जाती हैं।
</Info>

## भुगतान विधियाँ कॉन्फ़िगर करना

डिफ़ॉल्ट रूप से, सभी पात्र भुगतान विधियाँ स्वचालित रूप से दिखाई देती हैं। दिखने वाली विधियों को सीमित करने के लिए `allowed_payment_method_types` का उपयोग करें:

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://test.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

### उपलब्ध विधि प्रकार

| प्रकार | विवरण |
| :--- | :---------- |
| `credit` | क्रेडिट कार्ड (Visa, Mastercard, Amex, आदि) |
| `debit` | डेबिट कार्ड |
| `apple_pay` | Apple Pay |
| `google_pay` | Google Pay |
| `amazon_pay` | Amazon Pay |
| `cashapp` | Cash App Pay |
| `klarna` | Klarna BNPL |
| `afterpay_clearpay` | Afterpay / Clearpay |
| `upi_collect` | UPI संग्रह (QR कोड) |
| `ideal` | iDEAL (नीदरलैंड) |
| `bancontact_card` | Bancontact (बेल्जियम) |
| `eps` | EPS (ऑस्ट्रिया) |
| `multibanco` | Multibanco (पुर्तगाल) |
| `revolut_pay` | RevolutPay |
| `billie` | Billie B2B BNPL |

<Warning>
हमेशा `credit` और `debit` को फॉलबैक के रूप में शामिल करें। यदि सभी निर्दिष्ट भुगतान विधियाँ ग्राहक के लिए अनुपलब्ध हैं, तो चेकआउट विफल हो जाएगा।
</Warning>

## समस्या निवारण

<AccordionGroup>
{/* LOCKED_PATTERN_fbf2d38e7725fb97384bef01324b59a1 */}
**इन तत्वों की जांच करें:**
1. क्या विधि `allowed_payment_method_types` में शामिल है?
2. क्या ग्राहक स्थान विधि द्वारा समर्थित क्षेत्र से मेल खाता है?
3. क्या लेनदेन मुद्रा विधि द्वारा समर्थित है?
4. क्या लेनदेन राशि न्यूनतम को पूरा करती है (जैसे BNPL के लिए $50.01)?
5. सब्सक्रिप्शन के लिए: क्या विधि आवर्ती भुगतान का समर्थन करती है?


**समाधान:** सभी पात्र विधियाँ देखने के लिए अस्थायी रूप से `allowed_payment_method_types` को हटाएँ, फिर समस्या को 좁 करें।
</Accordion>

{/* LOCKED_PATTERN_57385d59bd851fb1e8c26266b489efd4 */}
**संभावित कारण:**
- `allowed_payment_method_types` सही ढंग से पारित नहीं हो रहा है
- पैरामीटर गलत स्वरूपित है (यह स्ट्रिंग्स की सूची होना चाहिए)


**समाधान:** अपने API अनुरोध पेलोड को लॉग करें और पैरामीटर संरचना की पुष्टि करें।
</Accordion>

{/* LOCKED_PATTERN_6d1a1b1a59cbe2789bbc9434eb1ec153 */}
**जांचें:**
- ग्राहक बिलिंग पता विधि के क्षेत्र से मेल खाता है
- मुद्रा सही ढंग से सेट है (जैसे iDEAL के लिए EUR)
- यदि ग्राहक अंतरराष्ट्रीय है तो Adaptive Currency सक्षम है


**समाधान:** अपने API अनुरोध में बिलिंग पते का देश कोड और मुद्रा सत्यापित करें।
</Accordion>

{/* LOCKED_PATTERN_52f3532b80d2def876ef3cc78b55f9f6 */}
UPI और Rupay सब्सक्रिप्शन में 48-घंटे की प्रोसेसिंग देरी और RBI आदेश सीमाएँ सहित अद्वितीय आवश्यकताएँ होती हैं।

**देखें:** <a href="/features/payment-methods/india">India Payment Methods</a> पूर्ण मार्गदर्शन के लिए।
