## छूट कोड क्या हैं?

छूट कोड प्रचारात्मक टोकन होते हैं जो चेकआउट पर ऑर्डर के कुल को कम करते हैं। ये निम्नलिखित के लिए आदर्श हैं:

- **मौसमी अभियान**: ब्लैक फ्राइडे, उत्पाद लॉन्च, या वर्षगांठ
- **अधिग्रहण प्रस्ताव**: पहले खरीद पर प्रोत्साहन या संदर्भ पुरस्कार
- **रखरखाव**: मौजूदा ग्राहकों के लिए जीतने या वफादारी पुरस्कार
- **B2B सौदे**: निजी कोड के माध्यम से अनुबंधित या बातचीत की गई कीमतें

## प्रमुख लाभ

- **लचीली छूट**: प्रतिशत या निश्चित राशि में छूट
- **लक्षित नियंत्रण**: उत्पाद और सदस्यता चक्र द्वारा प्रतिबंधित करें
- **अभियान शासन**: समाप्ति तिथियाँ और उपयोग सीमाएँ
- **सहज चेकआउट**: UI फ़ील्ड और चेकआउट सत्रों के माध्यम से API समर्थन

## छूट कोड बनाना

अपने डोडो पेमेंट्स डैशबोर्ड में छूट कोड बनाएं, फिर उन्हें होस्टेड चेकआउट या API के माध्यम से लागू करें।

### डैशबोर्ड सेटअप

- **छूट नाम** (आवश्यक): आंतरिक और डैशबोर्ड प्रदर्शन नाम
- **कोड** (आवश्यक): वह स्ट्रिंग जो ग्राहक चेकआउट पर दर्ज करते हैं
- **प्रकार और राशि** (आवश्यक): प्रतिशत या निश्चित राशि का मान सेट करें, या प्रदान किए गए बटन का उपयोग करके एक यादृच्छिक कोड उत्पन्न करें
- **समाप्ति तिथि** (वैकल्पिक): वह तिथि जिसके बाद कोड अमान्य हो जाता है
- **उपयोग सीमा** (वैकल्पिक): सभी ग्राहकों के बीच अधिकतम कुल रिडेम्प्शन
- **उत्पाद प्रतिबंध** (वैकल्पिक): चयनित उत्पादों पर लागू होने की सीमा
- **सदस्यता चक्र सीमा** (वैकल्पिक): बिलिंग चक्रों की संख्या जिस पर छूट लागू होती है

<Tip>
सब्सक्रिप्शन पर परिचयात्मक मूल्य निर्धारण के लिए साइकिल सीमाओं का उपयोग करें (जैसे, "3 महीनों के लिए 50% छूट").
</Tip>

## चेकआउट अनुभव

1. खरीदार चेकआउट फ़ील्ड में कोड दर्ज करते हैं।
2. पात्र छूट लागू होती हैं और कुल तुरंत अपडेट होते हैं।

<Info>
चेकआउट सेशनों में `discount_code` को पहले से लागू करने के लिए पास करें, और इनपुट फ़ील्ड दिखाने के लिए `feature_flags.allow_discount_code` सेट करें।
</Info>

## सामान्य उपयोग के मामले

- **प्रारंभिक प्रस्ताव**: नए उत्पादों के लिए सीमित समय के लॉन्च प्रचार
- **थोक या B2B**: चयनित उत्पाद सेट के लिए अनुबंधित छूट
- **रखरखाव खेल**: चर्न-रोकथाम कार्यप्रवाह में जीतने वाले कोड
- **मौसमी अभियान**: छुट्टियों या घटनाओं पर आधारित प्रचार

## एकीकरण उदाहरण

### चेकआउट सत्रों में छूट लागू करें
एक छूट को पूर्व-लागू करें और कोड इनपुट UI दिखाएं।

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    { product_id: 'prod_abc', quantity: 1 }
  ],
  discount_code: 'BLACKFRIDAY2024',
  customer: { email: 'user@example.com', name: 'Jane Doe' },
  return_url: 'https://yoursite.com/return'
});
```

### पूर्व-लागू किए बिना छूट प्रविष्टि सक्षम करें
ग्राहकों को चेकआउट पर एक कोड दर्ज करने दें बिना पहले से एक पास किए।

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    { product_id: 'prod_abc', quantity: 1 }
  ],
  feature_flags: {
    allow_discount_code: true
  },
  return_url: 'https://yoursite.com/return'
});
```

## सर्वोत्तम प्रथाएँ

- **स्पष्ट नाम**: पहचानने योग्य कोड का उपयोग करें जो अभियान नाम से मेल खाते हैं
- **समय-सीमा**: तात्कालिकता बढ़ाने और दुरुपयोग को रोकने के लिए समाप्तियाँ जोड़ें
- **समुचित दायरा**: विशेष उत्पादों तक सीमित करें ताकि मार्जिन लीक न हो
- **जल्दी मान्य करें**: चेकआउट की पुष्टि करने से पहले कोड की लागूता की जांच करें
- **प्रभाव की निगरानी करें**: अभियान द्वारा उपयोग और रूपांतरण को ट्रैक करें

<Info>
डिस्काउंट कोड अधिग्रहण और बनाए रखने के लिए शक्तिशाली लीवर हैं। सरल, स्पष्ट नाम वाले प्रस्तावों से शुरू करें, अच्छी तरह से जांचें, और प्रदर्शन के आधार पर सुधार करें।
</Info>
