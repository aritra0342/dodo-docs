I blueprint di ingestione sono modelli predefiniti per il tracciamento dell'uso in scenari comuni. Invece di dover capire le strutture degli eventi e le configurazioni dei misuratori da zero, inizia con un blueprint collaudato e personalizzalo per le tue esigenze.

## Cosa Sono i Blueprint?

Ogni blueprint include:
- **Raccomandazioni sulla struttura degli eventi**: Nomenclatura ottimale degli eventi e design dei metadati
- **Configurazione del misuratore**: Tipi di aggregazione e filtri suggeriti
- **Codice di esempio**: Esempi di implementazione funzionanti in più linguaggi
- **Migliori pratiche**: Suggerimenti per accuratezza, prestazioni e ottimizzazione dei costi

<Tip>
I blueprint fanno risparmiare ore di configurazione e aiutano a evitare gli errori più comuni. Parti da un blueprint e poi adattalo ai tuoi requisiti specifici.
</Tip>

## Confronto dei Blueprint

Non sei sicuro di quale blueprint si adatti alle tue esigenze? Ecco un rapido confronto:

| Caso d'uso | Blueprint | Aggregazione | Metadati Comuni |
|----------|-----------|-------------|-----------------|
| App chat AI | LLM | Somma (token) | modello, input_tokens, output_tokens |
| API REST | API Gateway | Conteggio | endpoint, metodo, status_code |
| Storage file | Object Storage | Somma (byte) | file_size, file_type, operation |
| Piattaforma video | Stream | Somma (larghezza di banda) | durata, qualità, bitrate |
| Serverless | Time Range | Somma (durata) | function_name, execution_time_ms |

## Implementazioni Personalizzate

Non vedi un blueprint per il tuo caso d'uso esatto? Nessun problema:

1. **Inizia con il blueprint più vicino**: Scegli quello più simile alle tue esigenze
2. **Adatta la struttura dell'evento**: Modifica i campi dei metadati per soddisfare le tue esigenze
3. **Regola la configurazione del misuratore**: Cambia il tipo di aggregazione o i filtri se necessario
4. **Testa a fondo**: Verifica i calcoli di fatturazione con l'uso di esempio

<Info>
Tutti i blueprint sono punti di partenza. Sentiti libero di mescolare concetti da blueprint diversi o creare implementazioni completamente personalizzate.
</Info>
