I blueprint di ingestione sono modelli predefiniti per il tracciamento dell'uso in scenari comuni. Invece di dover capire le strutture degli eventi e le configurazioni dei misuratori da zero, inizia con un blueprint collaudato e personalizzalo per le tue esigenze.

## Cosa Sono i Blueprint?

Ogni blueprint include:
- **Raccomandazioni sulla struttura degli eventi**: Nomenclatura ottimale degli eventi e design dei metadati
- **Configurazione del misuratore**: Tipi di aggregazione e filtri suggeriti
- **Codice di esempio**: Esempi di implementazione funzionanti in più linguaggi
- **Migliori pratiche**: Suggerimenti per accuratezza, prestazioni e ottimizzazione dei costi

<Tip>
I blueprint fanno risparmiare ore di tempo di configurazione e ti aiutano a evitare errori comuni. Inizia con un blueprint, poi adattalo alle tue specifiche esigenze.
</Tip>

## Blueprint Disponibili

Scegli il blueprint che corrisponde al tuo caso d'uso:

<CardGroup cols={1}>
<Card title="Blueprint LLM" icon="brain-circuit" href="/developer-resources/ingestion-blueprints/llm">
Traccia l'uso dei token AI attraverso OpenAI, Anthropic, Groq, Gemini, Vercel AI SDK e altro ancora.

**Perfetto per**: chatbot AI, generatori di codice, strumenti di creazione di contenuti, app potenziate da AI.

**Cosa traccerai**: token di input, token di output, utilizzo del modello, conteggi delle richieste.
</Card>

<Card title="Blueprint API Gateway" icon="cloud" href="/developer-resources/ingestion-blueprints/api-gateway">
Misura le richieste API a livello di gateway con supporto per filtraggio degli endpoint e limitazione della velocità.

**Perfetto per**: API REST, servizi GraphQL, fornitori di webhook, piattaforme API-first.

**Cosa traccerai**: conteggi delle richieste, utilizzo degli endpoint, metodi HTTP, tempi di risposta.
</Card>

<Card title="Blueprint Object Storage" icon="box-archive" href="/developer-resources/ingestion-blueprints/object-storage">
Traccia caricamenti di file, consumo di storage e larghezza di banda per servizi di cloud storage.

**Perfetto per**: hosting di file, piattaforme multimediali, servizi di backup, gestione documentale.

**Cosa traccerai**: byte memorizzati, caricamenti di file, download, snapshot di storage.
</Card>

<Card title="Blueprint Stream" icon="tower-broadcast" href="/developer-resources/ingestion-blueprints/stream">
Misura la larghezza di banda in streaming per video, audio e consegna di dati in tempo reale.

**Perfetto per**: streaming video, piattaforme audio, trasmissioni in diretta, flussi di dati in tempo reale.

**Cosa traccerai**: larghezza di banda consumata, durata dello stream, livelli di qualità, stream concorrenti.
</Card>

<Card title="Blueprint Time Range" icon="clock" href="/developer-resources/ingestion-blueprints/time-range">
Fattura in base al tempo trascorso per risorse di calcolo, funzioni serverless e lavori a lungo termine.

**Perfetto per**: piattaforme serverless, hosting di container, istanze di calcolo, elaborazione batch.

**Cosa traccerai**: tempo di esecuzione, ore di calcolo, invocazioni di funzioni, durata del lavoro.
</Card>
</CardGroup>

## Come Usare un Blueprint

<Steps>
<Step title="Scegli il tuo blueprint">
Seleziona il blueprint che meglio si adatta al tuo scenario di fatturazione. Ogni pagina del blueprint include guide dettagliate per l'implementazione.
</Step>

<Step title="Rivedi la struttura dell'evento">
Comprendi la nomenclatura degli eventi e il design dei metadati raccomandati. Copia la struttura dell'evento per il tuo caso d'uso.
</Step>

<Step title="Crea il misuratore">
Segui le istruzioni di configurazione del misuratore del blueprint. Usa il tipo di aggregazione e i filtri suggeriti.
</Step>

<Step title="Implementa l'invio degli eventi">
Copia il codice di esempio per il tuo linguaggio/framework. Sostituisci i valori segnaposto con le tue chiavi API e ID cliente reali.
</Step>

<Step title="Testa e verifica">
Invia eventi di test e verifica che appaiano nel tuo dashboard del misuratore con l'aggregazione corretta.
</Step>

<Step title="Personalizza se necessario">
Adatta il blueprint alle tue specifiche esigenze: aggiungi metadati personalizzati, modifica i filtri o modifica i prezzi.
</Step>
</Steps>

## Confronto dei Blueprint

Non sei sicuro di quale blueprint si adatti alle tue esigenze? Ecco un rapido confronto:

| Caso d'uso | Blueprint | Aggregazione | Metadati Comuni |
|----------|-----------|-------------|-----------------|
| App chat AI | LLM | Somma (token) | modello, input_tokens, output_tokens |
| API REST | API Gateway | Conteggio | endpoint, metodo, status_code |
| Storage file | Object Storage | Somma (byte) | file_size, file_type, operation |
| Piattaforma video | Stream | Somma (larghezza di banda) | durata, qualità, bitrate |
| Serverless | Time Range | Somma (durata) | function_name, execution_time_ms |

## Implementazioni Personalizzate

Non vedi un blueprint per il tuo caso d'uso esatto? Nessun problema:

1. **Inizia con il blueprint più vicino**: Scegli quello più simile alle tue esigenze
2. **Adatta la struttura dell'evento**: Modifica i campi dei metadati per soddisfare le tue esigenze
3. **Regola la configurazione del misuratore**: Cambia il tipo di aggregazione o i filtri se necessario
4. **Testa a fondo**: Verifica i calcoli di fatturazione con l'uso di esempio

<Info>
Tutti i blueprint sono punti di partenza. Sentiti libero di mescolare concetti di diversi blueprint o creare implementazioni completamente personalizzate.
</Info>
