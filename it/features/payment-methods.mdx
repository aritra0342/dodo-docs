## Come funziona

I metodi di pagamento vengono automaticamente presentati ai clienti in base a:

- **Posizione del cliente** — Metodi regionali come iDEAL (Paesi Bassi) o UPI (India) appaiono solo per i clienti in quelle regioni
- **Valuta della transazione** — I metodi di pagamento sono filtrati in base alle valute supportate
- **Tipo di dispositivo** — Apple Pay appare solo su dispositivi e browser Apple compatibili
- **Tipo di transazione** — Alcuni metodi non supportano iscrizioni o hanno importi minimi

<Info>
**Attivazione automatica:** Tutti i metodi di pagamento sono preconfigurati e gestiti da Dodo Payments. Non serve alcuna impostazione aggiuntiva: diventano disponibili una volta verificato il tuo account e attivata la modalità live.
</Info>

## Configurazione dei Metodi di Pagamento

Per impostazione predefinita, tutti i metodi idonei vengono visualizzati automaticamente. Usa `allowed_payment_method_types` per limitare quali metodi vengono mostrati:

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://test.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

### Tipi di Metodi Disponibili

| Type | Description |
| :--- | :---------- |
| `credit` | Carte di credito (Visa, Mastercard, Amex, ecc.) |
| `debit` | Carte di debito |
| `apple_pay` | Apple Pay |
| `google_pay` | Google Pay |
| `amazon_pay` | Amazon Pay |
| `cashapp` | Cash App Pay |
| `klarna` | Klarna BNPL |
| `afterpay_clearpay` | Afterpay / Clearpay |
| `upi_collect` | UPI Collect (codice QR) |
| `ideal` | iDEAL (Paesi Bassi) |
| `bancontact_card` | Bancontact (Belgio) |
| `eps` | EPS (Austria) |
| `multibanco` | Multibanco (Portogallo) |
| `revolut_pay` | RevolutPay |
| `billie` | Billie B2B BNPL |

<Warning>
Includi sempre `credit` e `debit` come fallback. Se tutti i metodi di pagamento specificati non sono disponibili per un cliente, il checkout fallirà.
</Warning>

## Risoluzione dei problemi

<AccordionGroup>
{/* LOCKED_PATTERN_fbf2d38e7725fb97384bef01324b59a1 */}
**Verifica questi fattori:**
1. Il metodo è incluso in `allowed_payment_method_types`?
2. La posizione del cliente corrisponde alla regione supportata dal metodo?
3. La valuta della transazione è supportata dal metodo?
4. L'importo della transazione soddisfa il minimo (es. $50,01 per i BNPL)?
5. Per gli abbonamenti: il metodo supporta pagamenti ricorrenti?

**Soluzione:** Rimuovi temporaneamente `allowed_payment_method_types` per vedere tutti i metodi idonei, poi restringi il campo per individuare il problema.
</Accordion>

{/* LOCKED_PATTERN_57385d59bd851fb1e8c26266b489efd4 */}
**Cause possibili:**
- `allowed_payment_method_types` non viene passato correttamente
- Il parametro è formattato in modo errato (deve essere un array di stringhe)

**Soluzione:** Registra il payload della richiesta API e verifica la struttura del parametro.
</Accordion>

{/* LOCKED_PATTERN_6d1a1b1a59cbe2789bbc9434eb1ec153 */}
**Verifica:**
- L'indirizzo di fatturazione del cliente corrisponde alla regione del metodo
- La valuta è impostata correttamente (es. EUR per iDEAL)
- La valuta adattiva è abilitata se il cliente è internazionale

**Soluzione:** Verifica il codice paese dell'indirizzo di fatturazione e la valuta nella tua richiesta API.
</Accordion>

{/* LOCKED_PATTERN_52f3532b80d2def876ef3cc78b55f9f6 */}
Gli abbonamenti UPI e Rupay hanno requisiti unici, inclusi ritardi di elaborazione di 48 ore e limiti di mandato della RBI.

**Vedi:** <a href="/features/payment-methods/india">India Payment Methods</a> per una guida completa.
</Accordion>
</AccordionGroup>
