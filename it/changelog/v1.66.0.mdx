## Nuove Funzionalità 🚀

1. **`subscription.updated` Webhook**  
   Un nuovo evento webhook che viene attivato a ogni aggiornamento dell’oggetto sottoscrizione, dandoti notifiche in tempo reale su qualsiasi modifica alle tue sottoscrizioni.
   
   Usa questo webhook per:
   - Ricevere notifiche istantanee quando cambiano i dettagli della sottoscrizione
   - Mantenere lo stato della tua applicazione sincronizzato con Dodo Payments
   - Reagire agli aggiornamenti delle sottoscrizioni senza interrogare periodicamente l’API
   
   <Tip>
   Iscriviti a `subscription.updated` per eliminare la necessità di polling e mantenere il sistema sincronizzato con i cambiamenti delle sottoscrizioni in tempo reale.
   </Tip>
   
   Scopri di più: [Subscription Webhooks](/developer-resources/webhooks/intents/subscription)

2. **Supporto per Indirizzo Minimo nel Checkout**  

   <Frame>
   <img src="/images/changelog/minimal-address.png" alt="Supporto per indirizzi minimi nel checkout" />
   </Frame>

   Riduci l’attrito nel checkout con la nuova modalità di raccolta degli indirizzi minima. Invece di raccogliere i dati completi dell’indirizzo, la modalità di indirizzo minimo raccoglie solo:
   
   - **Paese**: sempre richiesto per la determinazione delle tasse
   - **CAP/Codice postale**: raccolto solo nelle regioni in cui è necessario per il calcolo dell’imposta sulle vendite, dell’IVA o della GST
   
   <Check>
   Risultato: completamento del checkout significativamente più veloce con meno campi da compilare per i clienti.
   </Check>
   
   **Abilita indirizzo minimo:**
   
   <Tabs>
   <Tab title="Checkout Sessions">

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [{ product_id: 'prod_123', quantity: 1 }],
     minimal_address: true
   });
   ```

   </Tab>
   
   <Tab title="Static Payment Links">
   Aggiungi `minimalAddress=true` all’URL del tuo collegamento di pagamento:
   

   ```
   https://checkout.dodopayments.com/buy/prod_123?minimalAddress=true
   ```

   </Tab>
   </Tabs>
   
   <Info>
   Per impostazione predefinita, il checkout continua a raccogliere indirizzi completi. Trasformeremo gradualmente tutti i link di checkout nella modalità di indirizzo minimo. Abilita ora il flag della funzionalità per iniziare a beneficiare della riduzione dell’attrito nel checkout.
   </Info>

3. **Chiavi API in sola lettura**  
   Crea chiavi API con permessi di sola lettura per integrazioni più sicure. Quando crei una nuova chiave API, deseleziona **Abilita accesso in scrittura** per creare una chiave in sola lettura.
   
   Le chiavi in sola lettura possono:
   - Recuperare dati di pagamenti, sottoscrizioni e clienti
   - Elencare prodotti e recuperare dettagli
   - Accedere a report e analisi
   
   Le chiavi in sola lettura **non possono**:
   - Creare o modificare pagamenti
   - Aggiornare sottoscrizioni
   - Modificare le configurazioni dei prodotti
   
   <Tip>
   Deseleziona "Abilita accesso in scrittura" per integrazioni della dashboard, strumenti di analisi e qualsiasi sistema che deve solo visualizzare i dati senza effettuare modifiche.
   </Tip>

4. **UPI AutoPay e sottoscrizioni con carte RuPay - Rilascio completo**  
   UPI AutoPay e le sottoscrizioni con carte RuPay sono ora disponibili per tutte le aziende. Questo abilita pagamenti ricorrenti per i clienti indiani con:
   
   - **UPI AutoPay**: addebiti ricorrenti automatizzati tramite mandato UPI
   - **Carte RuPay**: supporto alle sottoscrizioni per il circuito domestico indiano
   
   <Check>
   Risultato: opzioni di pagamento per sottoscrizioni ampliate per il mercato indiano con gestione dei mandati conforme alla RBI.
   </Check>
   
   Scopri di più: [Subscriptions with RBI-Compliant Mandates](/features/subscription#subscriptions-with-rbi-compliant-mandates)

5. **API di Anteprima Cambiamento Piano**  
   Un nuovo endpoint API per visualizzare in anteprima le modifiche al piano di abbonamento prima di impegnarsi. Ottieni l'importo esatto dell'addebito e vedi come apparirà l'abbonamento dopo la modifica - senza apportare modifiche effettive.
   

   ```javascript
   const preview = await client.subscriptions.previewChangePlan('sub_123', {
     product_id: 'prod_pro_plan',
     proration_billing_mode: 'prorated_immediately',
     quantity: 1
   });
   
   console.log('Charge amount:', preview.immediate_charge.summary);
   console.log('New plan details:', preview.new_plan);
   ```

   
   Usa questo per:
   - Mostrare ai clienti esattamente quanto pagheranno prima di confermare gli aggiornamenti/downgrade
   - Costruire calcolatori di prezzo e strumenti di confronto dei piani
   - Ridurre i ticket di supporto migliorando la trasparenza dei prezzi
   
   Scopri di più: [API di Anteprima Cambiamento Piano](/api-reference/subscriptions/preview-change-plan)
