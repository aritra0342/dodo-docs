## Nuove Funzionalità 🚀

1. **`subscription.updated` Webhook**  
   Un nuovo evento webhook che si attiva ad ogni aggiornamento dell'oggetto abbonamento, fornendoti notifiche in tempo reale su qualsiasi modifica ai tuoi abbonamenti.
   
   Usa questo webhook per:
   - Ricevere notifiche istantanee quando i dettagli dell'abbonamento cambiano
   - Mantenere lo stato della tua applicazione sincronizzato con Dodo Payments
   - Reagire agli aggiornamenti degli abbonamenti senza interrogare l'API
   
   <Tip>
   Iscriviti a `subscription.updated` per eliminare la necessità di interrogare e mantenere il tuo sistema sincronizzato con le modifiche in tempo reale degli abbonamenti.
   </Tip>
   
   Scopri di più: [Webhook per Abbonamenti](/developer-resources/webhooks/intents/subscription)

2. **Supporto per Indirizzo Minimo nel Checkout**  

   <Frame>
   <img src="/images/changelog/minimal-address.png" alt="Supporto per Indirizzo Minimo nel Checkout" />
   </Frame>

   Riduci l'attrito nel checkout con la nuova modalità di raccolta indirizzo minimo. Invece di raccogliere tutti i dettagli dell'indirizzo, la modalità indirizzo minimo raccoglie solo:
   
   - **Paese**: Sempre richiesto per la determinazione delle tasse
   - **Codice Postale**: Raccolto solo nelle regioni in cui è necessario per il calcolo delle tasse sulle vendite, IVA o GST
   
   <Check>
   Risultato: Completamento del checkout significativamente più veloce con meno campi da compilare per i clienti.
   </Check>
   
   **Abilita l'indirizzo minimo:**
   
   <Tabs>
   <Tab title="Sessioni di Checkout">

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [{ product_id: 'prod_123', quantity: 1 }],
     minimal_address: true
   });
   ```

   </Tab>
   
   <Tab title="Link di Pagamento Statici">
   Aggiungi `minimalAddress=true` all'URL del tuo link di pagamento:
   

   ```
   https://checkout.dodopayments.com/buy/prod_123?minimalAddress=true
   ```

   </Tab>
   </Tabs>
   
   <Info>
   Per impostazione predefinita, il checkout continua a raccogliere indirizzi completi. Gradualmente passeremo tutti i link di checkout alla modalità indirizzo minimo. Abilita ora il flag della funzionalità per iniziare a beneficiare di un attrito ridotto nel checkout.
   </Info>

3. **Chiavi API in Sola Lettura**  
   Crea chiavi API con permessi in sola lettura per integrazioni più sicure. Quando crei una nuova chiave API, deseleziona **Abilita accesso in scrittura** per creare una chiave in sola lettura.
   
   Le chiavi in sola lettura possono:
   - Recuperare dati su pagamenti, abbonamenti e clienti
   - Elencare prodotti e recuperare dettagli
   - Accedere a report e analisi
   
   Le chiavi in sola lettura **non possono**:
   - Creare o modificare pagamenti
   - Aggiornare abbonamenti
   - Cambiare configurazioni di prodotto
   
   <Tip>
   Deseleziona "Abilita accesso in scrittura" per integrazioni con dashboard, strumenti di analisi e qualsiasi sistema che ha solo bisogno di visualizzare i dati senza apportare modifiche.
   </Tip>

4. **UPI AutoPay e Abbonamenti con Carta RuPay - Distribuzione Completa**  
   Gli abbonamenti UPI AutoPay e con carta RuPay sono ora disponibili per tutte le aziende. Questo consente pagamenti ricorrenti per i clienti indiani con:
   
   - **UPI AutoPay**: Addebiti ricorrenti automatizzati tramite mandato UPI
   - **Carte RuPay**: Supporto per abbonamenti per la rete di carte domestiche dell'India
   
   <Check>
   Risultato: Opzioni di pagamento per abbonamenti ampliate per il mercato indiano con gestione dei mandati conforme alla RBI.
   </Check>
   
   Scopri di più: [Abbonamenti con Mandati Conformi alla RBI](/features/subscription#subscriptions-with-rbi-compliant-mandates)

5. **API di Anteprima Cambiamento Piano**  
   Un nuovo endpoint API per visualizzare in anteprima le modifiche al piano di abbonamento prima di impegnarsi. Ottieni l'importo esatto dell'addebito e vedi come apparirà l'abbonamento dopo la modifica - senza apportare modifiche effettive.
   

   ```javascript
   const preview = await client.subscriptions.previewChangePlan('sub_123', {
     product_id: 'prod_pro_plan',
     proration_billing_mode: 'prorated_immediately',
     quantity: 1
   });
   
   console.log('Charge amount:', preview.immediate_charge.summary);
   console.log('New plan details:', preview.new_plan);
   ```

   
   Usa questo per:
   - Mostrare ai clienti esattamente quanto pagheranno prima di confermare gli aggiornamenti/downgrade
   - Costruire calcolatori di prezzo e strumenti di confronto dei piani
   - Ridurre i ticket di supporto migliorando la trasparenza dei prezzi
   
   Scopri di più: [API di Anteprima Cambiamento Piano](/api-reference/subscriptions/preview-change-plan)
