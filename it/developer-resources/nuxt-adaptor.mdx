## Panoramica

<Info>
Questa guida spiega come integrare Dodo Payments nella tua applicazione Nuxt utilizzando il modulo ufficiale per Nuxt. Imparerai a configurare le route API per checkout, customer portal e webhook, oltre a gestire in modo sicuro le variabili d'ambiente.
</Info>

<Steps>

Aggiungi <code>@dodopayments/nuxt</code> all'array <code>modules</code> e configuralo:

```typescript nuxt.config.ts expandable
export default defineNuxtConfig({
  modules: ["@dodopayments/nuxt"],
  devtools: { enabled: true },
  compatibilityDate: "2025-02-25",
  runtimeConfig: {
    private: {
```

## Esempi di Gestori di Route API

<Info>
Tutte le integrazioni di Dodo Payments in Nuxt vengono gestite tramite route server nella directory <code>server/routes/api/</code>.
</Info>

<Tabs>
<Tab title="Checkout API Route">
<Info>
Usa questo handler per integrare il checkout di Dodo Payments nella tua app Nuxt. Supporta i flussi di pagamento statico (GET), dinamico (POST) e sessione (POST).
</Info>

<CodeGroup>

```typescript server/routes/api/customer-portal.get.ts expandable

export default defineEventHandler((event) => {
  const {
    private: { bearerToken, environment },
  } = useRuntimeConfig();

  const handler = customerPortalHandler({
    bearerToken,
    environment: environment,
  });

  return handler(event);
});
```

**Parametri di Query:**
- <code>customer_id</code> (obbligatorio): L'ID cliente per la sessione del portale (ad es., <code>?customer_id=cus_123</code>)
- <code>send_email</code> (opzionale, booleano): Se <code>true</code>, invia un'email al cliente con il link del portale.

<Warning>
Restituisce 400 se <code>customer_id</code> manca.
</Warning>

- **Metodo:** Solo le richieste POST sono supportate. Altri metodi restituiscono 405.
- **Verifica della Firma:** Verifica la firma del webhook utilizzando <code>webhookKey</code>. Restituisce 401 se la verifica fallisce.
- **Validazione del Payload:** Validato con Zod. Restituisce 400 per payload non validi.
- **Gestione degli Errori:**
  - 401: Firma non valida
  - 400: Payload non valido
  - 500: Errore interno durante la verifica
- **Routing degli Eventi:** Chiama il gestore di eventi appropriato in base al tipo di payload.
