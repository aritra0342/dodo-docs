# Panoramica

L'<b>Better Auth Adapter</b> per Dodo Payments fornisce:

- Creazione automatica del cliente al momento della registrazione
- Sessioni di checkout (preferite) con mappatura dello slug del prodotto
- Portale clienti self-service
- Endpoint di ingestione e reporting per l'uso misurato per la fatturazione basata sull'uso
- Elaborazione degli eventi webhook in tempo reale con verifica della firma
- Supporto completo per TypeScript

<Note>
  È necessario un account Dodo Payments e chiavi API per usare questa integrazione.
</Note>

# Requisiti

- Node.js 20+
- Accesso al tuo Dashboard Dodo Payments
- Progetto esistente che utilizza [better-auth](https://www.npmjs.com/package/better-auth)

# Esempi di utilizzo

<Info>
  Preferisci <code>authClient.dodopayments.checkoutSession</code> per le nuove
  integrazioni. Il metodo legacy <code>checkout</code> è deprecato ed è mantenuto
  solo per retrocompatibilità.
</Info>

## Creazione di una Sessione di Checkout (Preferita)

<CodeGroup>

```typescript Using a slug
const { data: session, error } = await authClient.dodopayments.checkoutSession({
  // Option A: use a configured slug
  slug: "premium-plan",

// Your internal id for reference
referenceId: "order_123",
});

if (session) {
window.location.href = session.url;
}

```

`

```typescript Using product cart
const { data: session, error } = await authClient.dodopayments.checkoutSession({
  // Option B: pass a product cart/product id directly
  product_cart: [
    {
      product_id: "pdt_xxxxxxxxxxxxxxxxxxxxx",
      quantity: 1,
    }
  ],

  // Your internal id for reference
  referenceId: "order_123",
});

if (session) {
  window.location.href = session.url;
}
```

`

</CodeGroup>

<Info>
  A differenza del metodo legacy <code>checkout</code>, <code>checkoutSession</code>
  non richiede le informazioni di fatturazione all'inizio, poiché verranno raccolte
  dall'utente nella pagina di checkout. Puoi comunque forzarle passando la chiave
  <code>billing</code> nell'argomento.
</Info>

<Info>
  Simile al metodo legacy <code>checkout</code>,
  <code>checkoutSession</code> prende l'email e il nome del cliente dalla loro
  sessione better-auth, tuttavia puoi sovrascriverli passando l'oggetto
  <code>customer</code> con email e nome.
</Info>

<Info>
  Puoi passare gli stessi parametri nell'argomento come nel corpo della richiesta per [Create
  Checkout
  Session](https://docs.dodopayments.com/api-reference/checkout-sessions/create)
  endpoint.
</Info>

<Note>
  L'URL di ritorno viene preso dal <code>successUrl</code> configurato nel
  plugin server. Non è necessario includere <code>return_url</code> nel payload
  del client.
</Note>

## Monitoraggio dell'Uso Misurato

Abilita il plugin <code>usage()</code> sul server per catturare eventi misurati e consentire ai clienti di ispezionare la loro fatturazione basata sull'uso.

- <code>authClient.dodopayments.usage.ingest</code> ingesta eventi per l'utente autenticato e verificato via email.
- <code>authClient.dodopayments.usage.meters.list</code> elenca l'uso recente per i misuratori legati agli abbonamenti di quel cliente.

```typescript expandable
// Record a metered event (e.g., an API request)
const { error: ingestError } = await authClient.dodopayments.usage.ingest({
  event_id: crypto.randomUUID(),
  event_name: "api_request",
  metadata: {
    route: "/reports",
    method: "GET",
  },
  // Optional Date; defaults to now if omitted
  timestamp: new Date(),
});

if (ingestError) {
  console.error("Failed to record usage", ingestError);
}

// List recent usage for the current customer
const { data: usage, error: usageError } =
  await authClient.dodopayments.usage.meters.list({
    query: {
      page_size: 20,
      meter_id: "mtr_yourMeterId", // optional
    },
  });

if (usage?.items) {
  usage.items.forEach((event) => {
    console.log(event.event_name, event.timestamp, event.metadata);
  });
}
```

<Tip>
  I timestamp più vecchi di un'ora o con più di cinque minuti nel futuro vengono
  rifiutati durante l'ingestione dell'uso.
</Tip>
<Tip>
  Se ometti <code>meter_id</code> quando elenchi i meter di utilizzo, vengono
  restituiti tutti i meter collegati alle sottoscrizioni del cliente.
</Tip>
