Dodo Payments offre una funzionalità di sincronizzazione del database integrata che sincronizza automaticamente i tuoi dati di pagamento con il tuo database. Puoi sincronizzare **pagamenti**, **clienti**, **abbonamenti** e **licenze** per mantenere una copia locale dei tuoi dati per analisi, reportistica o integrazione con altri sistemi.

<Info>
**Implementazione**: Disponibile tramite [npm package](https://www.npmjs.com/package/dodo-sync) | **Codice sorgente**: [GitHub](https://github.com/dodopayments/dodo-sync)
</Info>

## Cosa Puoi Sincronizzare?

La nostra funzionalità di sincronizzazione del database supporta la sincronizzazione delle seguenti entità di Dodo Payments nel tuo database:

    Sincronizza i dati delle sottoscrizioni, incluse sottoscrizioni attive, cicli di fatturazione e modifiche dello stato delle sottoscrizioni.
  </Card>
  {/* LOCKED_PATTERN_195400deed106b7fc4997fd4764db9b5 */}
    Sincronizza le informazioni sulle licenze, inclusi codici di licenza, attivazioni e aggiornamenti dello stato delle licenze.

Puoi sincronizzare qualsiasi combinazione di queste entità specificandole nel parametro `scopes`. Tutte le operazioni di sincronizzazione sono incrementali e trasferiscono solo record nuovi o aggiornati per garantire prestazioni ottimali.

## Supporto per Database

Attualmente supportiamo **MongoDB**, **PostgreSQL**, **MySQL** e **ClickHouse**.

Stiamo lavorando attivamente per espandere il supporto per:
- **Database**: Snowflake e altri.
- **Pipeline**: pipeline ETL, sincronizzazione in tempo reale.

## Iniziare

Puoi utilizzare la nostra funzionalità di sincronizzazione del database tramite il **CLI** o programmaticamente nel tuo **codice**. Entrambi i metodi offrono la stessa funzionalità: scegli quello che meglio si adatta al tuo flusso di lavoro.

### Utilizzando il CLI

Lo strumento CLI fornisce un modo rapido per configurare e avviare la sincronizzazione del database. Installalo globalmente per usarlo da qualsiasi parte nel tuo terminale:

<CodeGroup>

```bash npm
npm install -g dodo-sync
```

```bash bun
bun add -g dodo-sync
```

</CodeGroup>

#### Esecuzione del CLI

Il CLI supporta due modalità: **Modalità Interattiva** per una configurazione guidata e **Modalità Manuale** per una configurazione diretta.

**Modalità Interattiva**: Esegui semplicemente il comando senza argomenti per avviare la procedura guidata di configurazione interattiva.

```bash
dodo-sync
```

**Modalità Manuale**: Passa argomenti direttamente per saltare la procedura guidata.

```bash
dodo-sync -i [interval] -d [database] -u [database_uri] --scopes [scopes] --api-key [api_key] --env [environment]
```

**Esempi:**

```bash
# MongoDB
dodo-sync -i 600 -d mongodb -u mongodb://mymongodb.url --scopes "licences,payments,customers,subscriptions" --api-key YOUR_API_KEY --env test_mode

# PostgreSQL
dodo-sync -i 600 -d postgres -u postgresql://user:password@localhost:5432/mydb --scopes "licences,payments,customers,subscriptions" --api-key YOUR_API_KEY --env test_mode

# MySQL
dodo-sync -i 600 -d mysql -u mysql://user:password@localhost:3306/mydb --scopes "licences,payments,customers,subscriptions" --api-key YOUR_API_KEY --env test_mode

# ClickHouse
dodo-sync -i 600 -d clickhouse -u http://localhost:8123 --scopes "licences,payments,customers,subscriptions" --api-key YOUR_API_KEY --env test_mode
```

#### Argomenti CLI

<ParamField path="--interval" type="number" alias="-i">
Intervallo di sincronizzazione in secondi. Determina con quale frequenza viene eseguita l'operazione di sincronizzazione. Se non fornito, la sincronizzazione verrà eseguita una sola volta e terminerà.
</ParamField>

<ParamField path="--database" type="string" alias="-d" required>
Tipo di database da utilizzare. Valori supportati: `"mongodb"`, `"postgres"`, `"mysql"` o `"clickhouse"`.
</ParamField>

<ParamField path="--database-uri" type="string" alias="-u" required>
URI di connessione per il tuo database:
- **MongoDB**: `mongodb://localhost:27017` o `mongodb+srv://user:pass@cluster.mongodb.net/`
- **PostgreSQL**: `postgresql://user:password@localhost:5432/mydb`
- **MySQL**: `mysql://user:password@localhost:3306/mydb`
- **ClickHouse**: `http://localhost:8123`
</ParamField>

<ParamField path="--scopes" type="string" required>
Elenco separato da virgole delle entità dati da sincronizzare. Scope disponibili: `licences`, `payments`, `customers`, `subscriptions`. Esempio: `"payments,customers"`.
</ParamField>

<ParamField path="--api-key" type="string" required>
La tua chiave API Dodo Payments. Deve iniziare con `dp_live_` per la modalità live o `dp_test_` per la modalità test.
</ParamField>

<ParamField path="--env" type="string" required>
Ambiente target. Deve essere `"live_mode"` o `"test_mode"`. Ciò determina da quale ambiente Dodo Payments eseguire la sincronizzazione.
</ParamField>

<ParamField path="--rate-limit" type="number" alias="--rl">
Limite di velocità in richieste al secondo. Controlla la velocità con cui il motore di sincronizzazione effettua le richieste API per evitare di sovraccaricare l'API Dodo Payments.
</ParamField>

### Utilizzando nel Tuo Codice

Per un controllo programmatico, integra la funzionalità di sincronizzazione direttamente nella tua applicazione. Installala come dipendenza nel tuo progetto:

<CodeGroup>

```bash npm
npm install dodo-sync
```

```bash bun
bun add dodo-sync
```

</CodeGroup>

#### Sincronizzazione Automatica (Basata su Intervallo)

Utilizza la sincronizzazione automatica quando desideri che la sincronizzazione venga eseguita continuamente a intervalli regolari:

```typescript
import { DodoSync } from 'dodo-sync';

const syncDodoPayments = new DodoSync({
  interval: 60, // Sync every 60 seconds
  database: 'mongodb',
  databaseURI: process.env.MONGODB_URI, // e.g., 'mongodb://localhost:27017'
  scopes: ['licences', 'payments', 'customers', 'subscriptions'],
  dodoPaymentsOptions: {
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    environment: 'test_mode' // or 'live_mode'
  }
});

// Initialize connection
await syncDodoPayments.init();

// Start the sync loop
syncDodoPayments.start();
```

<Tip>
L'opzione `interval` è obbligatoria quando si utilizza `.start()` per la sincronizzazione automatica. La sincronizzazione verrà eseguita continuamente all'intervallo specificato finché il processo non viene interrotto.
</Tip>

#### Sincronizzazione Manuale

Utilizza la sincronizzazione manuale quando desideri attivare le operazioni di sincronizzazione su richiesta (ad esempio, da un cron job o un endpoint API):

```typescript
import { DodoSync } from 'dodo-sync';

const syncDodoPayments = new DodoSync({
  database: 'mongodb',
  databaseURI: process.env.MONGODB_URI,
  scopes: ['licences', 'payments', 'customers', 'subscriptions'],
  dodoPaymentsOptions: {
    bearerToken: process.env.DODO_PAYMENTS_API_KEY,
    environment: 'test_mode'
  }
});

// Initialize connection
await syncDodoPayments.init();

// Trigger a single sync operation
await syncDodoPayments.run();
```

<Tip>
Quando si utilizza la sincronizzazione manuale, l'opzione `interval` non è richiesta. Puoi chiamare `.run()` ogni volta che devi eseguire un'operazione di sincronizzazione.
</Tip>

## Informazioni Importanti

**MongoDB**: Un database denominato `dodopayments_sync` verrà creato automaticamente sul tuo server database. Tutti i dati di sincronizzazione verranno memorizzati lì. Questo nome del database è attualmente fisso e non può essere modificato.

**PostgreSQL**: Verranno create tabelle (`Subscriptions`, `Payments`, `Licenses`, `Customers`) nel database specificato nella tua URI di connessione. I dati vengono memorizzati come JSONB.

**MySQL**: Verranno create tabelle (`Subscriptions`, `Payments`, `Licenses`, `Customers`) nel database specificato nella tua URI di connessione. I dati vengono memorizzati come JSON.

**ClickHouse**: Verranno create tabelle (`Subscriptions`, `Payments`, `Licenses`, `Customers`) utilizzando il motore ReplacingMergeTree. Quando esegui query, utilizza la parola chiave `FINAL` per assicurarti risultati deduplicati.
</Warning>

<Info>
Il motore di sincronizzazione tiene traccia delle modifiche e sincronizza solo record nuovi o aggiornati, rendendo successive sincronizzazioni efficienti anche con set di dati di grandi dimensioni.
</Info>
