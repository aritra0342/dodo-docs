## Panoramica

Un boilerplate pronto per la produzione per le sottoscrizioni utilizzando Next.js 15, React 19, Supabase, Drizzle ORM e Dodo Payments. Include Google OAuth, checkout per sottoscrizioni, gestione dei webhook, schema del database e un dashboard di base.

<Info>
Se ti servono solo gli handler delle route per un'app esistente, consulta gli adattatori dedicati: <a href="/developer-resources/nextjs-adaptor">Next.js Adaptor</a> e <a href="/developer-resources/express-adaptor">Express Adaptor</a>.
</Info>

## Requisiti

- Node.js 18+ (o Bun 1.0+)
- Progetto Supabase (URL, chiave anonima, chiave di ruolo del servizio, URL del database)
- Account Dodo Payments (chiave API, segreto webhook)
- Client OAuth di Google Cloud (ID client e segreto)

## Avvio rapido

<Steps>
{/* LOCKED_PATTERN_587a3a4c926aec89f48dfc1e7c21054e */}

```bash
git clone https://github.com/dodopayments/dodo-supabase-subscription-starter.git
cd dodo-supabase-subscription-starter
# choose one
bun install
# or
npm install
# or
pnpm install
```

</Step>

{/* LOCKED_PATTERN_ea54c61e84e28cd275c0eed6d7634e0f */}
Crea un progetto Supabase e copia:
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
- SUPABASE_SERVICE_ROLE_KEY
- DATABASE_URL (stringa di connessione)
</Step>

{/* LOCKED_PATTERN_42bf2f0f0010fd47b77c88325d34313c */}
Imposta l’URI di reindirizzamento su: `https://[your-project-ref].supabase.co/auth/v1/callback` in Google Cloud, quindi abilita il provider Google in Supabase Auth usando il tuo Client ID e Secret.
</Step>

{/* LOCKED_PATTERN_f32b65d8a917a558befa771bede82115 */}
Genera una chiave API e un secret per i webhook dalla dashboard di Dodo. Imposta l’ambiente su `test_mode` durante lo sviluppo.
</Step>

{/* LOCKED_PATTERN_a34a8e162789e5a79d108d9eb3c86df5 */}

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role

# Database
DATABASE_URL=postgresql://postgres:[password]@db.[project-ref].supabase.co:5432/postgres

# Dodo Payments
DODO_PAYMENTS_API_KEY=your-dodo-api-key
DODO_WEBHOOK_SECRET=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT=test_mode
```

<Warning>
Non commettere mai i segreti. Usa variabili d’ambiente negli ambienti di deployment.
</Warning>
</Step>

{/* LOCKED_PATTERN_c35fe33eae6aa68b4bc39bd740632964 */}

```bash
bun run db:push
# or
npm run db:push
# or
pnpm run db:push
```

<Check>
Tabelle create: `users`, `subscriptions`, `payments`.
</Check>
</Step>

{/* LOCKED_PATTERN_560d1d81fad22bd0622f0c8dcb3c706d */}

```bash
# login (one-time)
bunx supabase login
# or
npx supabase login

# deploy the edge function
bun run deploy:webhook --project-ref [your-project-ref]
# or
npm run deploy:webhook -- --project-ref [your-project-ref]
# or
pnpm run deploy:webhook --project-ref [your-project-ref]
```

```bash cURL
curl -X POST \
  'https://[your-project-ref].supabase.co/functions/v1/dodo-webhook' \
  -H 'Content-Type: application/json' \
  -H 'Dodo-Signature: <signature>' \
  -d '{"type":"payment.succeeded","data":{}}'
```

</Step>

{/* LOCKED_PATTERN_dd5db8f3734010c91f70fdf794c4e2a2 */}

```text
https://[your-project-ref].supabase.co/functions/v1/dodo-webhook
```

Seleziona eventi di pagamento e abbonamento.

L’interfaccia prezzi legge questo array `features` e lo rende dinamicamente.
</Step>

{/* LOCKED_PATTERN_e437399605995e5a4845b4d749b71d01 */}

Apri http://localhost:3000.
</Step>
</Steps>

## Cosa è incluso

- Autenticazione tramite Supabase (Google OAuth configurato)
- Checkout degli abbonamenti tramite Dodo Payments
- Edge Function Supabase per webhook (`dodo-webhook`)
- Schema e migrazioni con Drizzle ORM
- Dashboard con fatture, stato dell’abbonamento e funzionalità del piano

<Tip>
Mantieni `DODO_PAYMENTS_ENVIRONMENT` come `test_mode` finché non completi i test end-to-end.
</Tip>

## Verifica e risoluzione dei problemi

<AccordionGroup>
{/* LOCKED_PATTERN_6b286e9e2bbacd5c7684f8442203f10e */}
- Assicurati che `DODO_WEBHOOK_SECRET` corrisponda al valore dalla dashboard Dodo
- Conferma di aver distribuito l’ultima funzione `dodo-webhook`
- Verifica che il nome dell’intestazione sia corretto nella tua funzione (Dodo-Signature)
</Accordion>
{/* LOCKED_PATTERN_c270a2f8797d0168d608bb07bd40f056 */}
- Controlla la sintassi di `DATABASE_URL` e l’egress di rete Supabase
- Attendi ~2–3 minuti dopo la creazione del progetto prima del primo push
</Accordion>
{/* LOCKED_PATTERN_276d2f8137bb346c423586f69d6b7cae */}
- L’URI di reindirizzamento deve essere `https://[ref].supabase.co/auth/v1/callback`
- Assicurati che sia lo stesso in Google Cloud e nel provider Supabase Auth
</Accordion>
</AccordionGroup>
