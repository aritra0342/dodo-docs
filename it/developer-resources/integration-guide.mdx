## Requisiti

Per integrare l'API Dodo Payments, avrai bisogno di:
- Un account commerciante Dodo Payments
- Credenziali API (chiave API e chiave segreta webhook) dal dashboard

## Configurazione del Dashboard

1. Naviga al [Dashboard Dodo Payments](https://app.dodopayments.com/)

2. Crea un prodotto (pagamento una tantum o abbonamento)

4. Genera la tua chiave API:
   - Vai su Sviluppatore > API
   - [Guida dettagliata](/api-reference/introduction#api-key-generation)
   - Copia la chiave API nell'ambiente denominato DODO_PAYMENTS_API_KEY

5. Configura i webhook:
   - Vai su Sviluppatore > Webhook
   - Crea un URL webhook per le notifiche di pagamento
   - Copia la chiave segreta webhook nell'ambiente

## Integrazione

### Link di pagamento

Scegli il percorso di integrazione che si adatta al tuo caso d'uso:
- **Checkout Sessions (raccomandato)**: Migliore per la maggior parte delle integrazioni. Crea una sessione sul tuo server e reindirizza i clienti a un checkout sicuro e ospitato.
- **Overlay Checkout**: Utilizza quando hai bisogno di un'esperienza in-page che apre il checkout come un overlay modale sul tuo sito.
- **Inline Checkout**: Integra direttamente il checkout nel layout della tua pagina per esperienze di checkout completamente integrate e brandizzate.
- **Static Payment Links**: URL condivisibili istantaneamente senza codice per una rapida raccolta dei pagamenti.
- **Dynamic Payment Links**: Link creati programmaticamente. Tuttavia, le Checkout Sessions sono raccomandate e offrono maggiore flessibilità.

#### 1. Sessioni di checkout

Usa le Sessioni di Checkout per creare un'esperienza di checkout ospitata sicura per pagamenti una tantum o abbonamenti. Crei una sessione sul tuo server, quindi reindirizzi il cliente all'`checkout_url`.

<Info>
Le sessioni di checkout sono valide per 24 ore di default. Se passi <code>confirm=true</code>, le sessioni sono valide per 15 minuti e devono essere forniti tutti i campi obbligatori.
</Info>

#### 2. Checkout Overlay

Per un'esperienza di checkout in pagina senza soluzione di continuità, esplora la nostra integrazione [Checkout Overlay](/developer-resources/overlay-checkout) che consente ai clienti di completare i pagamenti senza lasciare il tuo sito web.

#### 3. Inline Checkout

Per esperienze di checkout completamente integrate incorporate direttamente nella tua pagina, utilizza la nostra integrazione [Inline Checkout](/developer-resources/inline-checkout). Questo ti consente di costruire riepiloghi degli ordini personalizzati e avere il controllo completo sul layout del checkout mentre Dodo Payments gestisce in modo sicuro la raccolta dei pagamenti.

#### 4. Static Payment Links

I link di pagamento statici ti consentono di accettare rapidamente pagamenti condividendo un semplice URL. Puoi personalizzare l'esperienza di checkout passando parametri di query per precompilare i dettagli del cliente, controllare i campi del modulo e aggiungere metadati personalizzati.

    return NextResponse.json(
      {
        error: err instanceof Error ? err.message : "An unknown error occurred",
      },
      { status: 500 }
    );
  }

### Implementazione dei Webhook

Imposta un endpoint API per ricevere notifiche di pagamento. Ecco un esempio utilizzando Next.js:

```javascript
import { Webhook } from "standardwebhooks";
import { headers } from "next/headers";
import { WebhookPayload } from "@/types/api-types";

const webhook = new Webhook(process.env.DODO_WEBHOOK_KEY!); // Replace with your secret key generated from the Dodo Payments Dashboard

export async function POST(request: Request) {
  const headersList = headers();
  const rawBody = await request.text();

  const webhookHeaders = {
    "webhook-id": headersList.get("webhook-id") || "",
    "webhook-signature": headersList.get("webhook-signature") || "",
    "webhook-timestamp": headersList.get("webhook-timestamp") || "",
  };

  await webhook.verify(rawBody, webhookHeaders);
  const payload = JSON.parse(rawBody) as WebhookPayload;
  
  // Process the payload according to your business logic
}
```

La nostra implementazione dei webhook segue la specifica [Standard Webhooks](https://standardwebhooks.com/). Per le definizioni dei tipi di webhook, fai riferimento alla nostra [Guida agli Eventi Webhook](/developer-resources/webhooks/intents/webhook-events-guide).

Puoi fare riferimento a questo progetto con implementazione demo su [GitHub](https://github.com/dodopayments/dodo-checkout-demo) utilizzando Next.js e TypeScript.
