## Gestione delle Chiavi API e Autenticazione

<Steps>
<Step title="Accedi alle Chiavi API">
Naviga su **Sviluppatore → Chiavi API** nel tuo dashboard per gestire le tue chiavi.
</Step>

<Step title="Genera una Nuova Chiave">
Seleziona **Aggiungi Chiave API**, fornisci un nome descrittivo e configura l'accesso in scrittura:

- **Abilita accesso in scrittura** (selezionato): Permessi completi di lettura e scrittura per tutte le operazioni API
- **Abilita accesso in scrittura** (non selezionato): Accesso in sola lettura - può solo recuperare dati (pagamenti, abbonamenti, clienti, prodotti). Non può creare o modificare risorse.

<Tip>
Deseleziona "Abilita accesso in scrittura" per integrazioni del dashboard, strumenti di analisi e qualsiasi sistema che ha solo bisogno di visualizzare i dati senza apportare modifiche.
</Tip>
</Step>

<Step title="Conserva la Tua Chiave in Sicurezza">
Copia immediatamente la chiave generata e assicurati che sia conservata in modo sicuro.
</Step>

<Step title="Autenticare le Tue Richieste API">
Utilizza le tue chiavi API per autenticare tutte le richieste. Applica il seguente formato di autorizzazione:

```bash 
Authorization: Bearer YOUR_API_KEY
```

<Warning>
Non esporre mai le tue chiavi API segrete nel codice lato client o in repository pubblici.
</Warning>
</Step>
</Steps>

## Formato della Risposta

<CodeGroup>

```json Success Response
{
  "id": "pay_1234567890",
  "status": "completed",
  "amount": 2999,
  "currency": "USD",
  "created_at": "2024-01-15T10:30:00Z"
}
```

```json Error Response
{
  "code": "INVALID_REQUEST",
  "message": "The request contains invalid parameters"
}
```

</CodeGroup>

## Limiti di Richiesta

- **Standard**: 1000 richieste al minuto per azienda
- **Burst**: Fino a 100 richieste al secondo

<Note>
Monitora `X-RateLimit-Remaining` header per tracciare l'uso.
</Note>
