## O que é uma migração de assinaturas?

Uma migração de assinaturas transfere seus assinantes existentes do seu processador de pagamentos legado para o Dodo Payments, fornecendo uma única fonte de verdade para todos os seus dados de clientes. Sem uma migração, os clientes que você adquiriu até agora permanecem em seu sistema legado, e apenas novos clientes seriam visíveis no Dodo Payments.

O processo de migração garante uma experiência de cobrança contínua para seus clientes—eles não precisarão reentrar seus dados de pagamento ou experimentar qualquer interrupção em suas assinaturas.

<Warning>
Atualmente, só suportamos migração para **tokens de cartão**. Apple Pay, Google Pay e outros métodos de pagamento por carteira não podem ser transferidos entre processadores de pagamento devido a restrições de tokenização impostas por esses provedores.
</Warning>

## Visão geral do processo de migração

O processo de migração de assinaturas consiste em três partes principais:

1. **Transferência de método de pagamento** — Transferência segura de dados sensíveis do cartão do seu processador atual
2. **Transferência de dados do assinante** — Exportação de detalhes do cliente, planos de assinatura e cronogramas de cobrança
3. **Importação e validação** — Unificação de métodos de pagamento e dados de assinantes no Dodo Payments

<Note>
Para iniciar uma migração de assinatura, contate nossa [equipe de suporte](mailto:support@dodopayments.com). Eles irão conectá-lo a um Engenheiro de Soluções dedicado que o guiará por todo o processo.
</Note>

## Requisitos

Antes de iniciar uma migração, certifique-se de atender aos seguintes requisitos:

<AccordionGroup>
<Accordion title="PCI DSS compliance verification">
Dodo Payments é compatível com o PCI DSS Nível 1. Para atender às obrigações de conformidade, só podemos receber dados de cartão de outro processador de pagamento compatível com o PCI DSS Nível 1.

Seu processador atual deve fornecer:
- Sua atual Declaração de Conformidade PCI (AOC), ou sua listagem no Registro Global de Provedores de Serviços da Visa
- Confirmação de que eles suportam exportações seguras de dados de cartão

Veja nossa [Declaração de Conformidade (AOC)](https://pub-2a89184e4c954b599464449d23f506bb.r2.dev/dodo-payments-pci-dss-aoc.pdf) para compartilhar com seu processador atual.
</Accordion>

<Accordion title="Dodo Payments integration">
Finalize sua integração com a Dodo Payments e assegure que todos os novos clientes estão sendo direcionados para o checkout da Dodo Payments antes de iniciar a migração. Isso previne inconsistências de dados durante o período de transição.
</Accordion>

<Accordion title="Data access authorization">
Certifique-se de que você tem a autorização necessária para solicitar uma exportação de dados do seu processador atual. Normalmente isso requer:
- Verificação do proprietário da conta
- Autorização por escrito para acesso a dados sensíveis
- Conclusão dos passos de verificação de segurança do seu processador
</Accordion>
</AccordionGroup>

## Guia de migração passo a passo

<Steps>
<Step title="Request payment method export from your current processor">
Entre em contato com a equipe de suporte do seu processador de pagamento atual e envie uma solicitação formal para uma exportação completa dos tokens de cartão dos seus clientes.

Você precisará:
- Fornecer verificação de sua identidade e autorização
- Solicitar que eles criptografem a exportação usando a chave pública PGP do Dodo Payments (fornecida abaixo)
- Pedir que eles transfiram o arquivo criptografado diretamente para nosso servidor SFTP seguro

<Info>
As transferências seguras são feitas diretamente de servidor para servidor via SFTP. Seu Engenheiro de Soluções fornecerá as credenciais do SFTP para você compartilhar com seu processador atual.
</Info>

Em média, esse processo leva cerca de **2 semanas**, embora o tempo varie de acordo com as políticas e os tempos de resposta do seu processador atual.
</Step>

<Step title="Prepare subscriber data export">
Enquanto a transferência do método de pagamento estiver em andamento, exporte os dados dos seus assinantes do seu sistema legado. Seu Engenheiro de Soluções fornecerá um modelo de CSV e orientará você quanto aos campos necessários.

### Dados de assinante necessários

| Field | Description |
|-------|-------------|
| `customer_id` | Seu identificador interno de cliente |
| `email` | Endereço de e-mail do cliente |
| `name` | Nome completo do cliente |
| `country` | Código de país de duas letras (ISO 3166-1 alfa-2) |
| `currency` | Código de moeda de três letras (ISO 4217) |
| `payment_amount` | Valor da assinatura na menor unidade da moeda |
| `payment_interval` | Frequência de cobrança (mensal, anual etc.) |
| `next_billing_date` | Próxima data programada de cobrança (ISO 8601) |
| `card_token` | Token único do seu processador atual |

<Warning>
O campo `card_token` é crítico—ele é usado para vincular os registros de assinantes aos seus métodos de pagamento durante a importação. Certifique-se de que isso corresponda exatamente ao token na exportação do método de pagamento.
</Warning>

### Formato de dados de exemplo

```json
{
  "customers": [
    {
      "customer_id": "cus_abc123",
      "email": "jane.doe@example.com",
      "name": "Jane Doe",
      "country": "US",
      "currency": "USD",
      "payment_amount": 2999,
      "payment_interval": "monthly",
      "next_billing_date": "2025-01-15T00:00:00Z",
      "card_token": "tok_xyz789",
      "metadata": {
        "plan": "pro",
        "signup_source": "website"
      }
    }
  ]
}
```

</Step>

<Step title="Validation and testing">
Antes da data de importação, nossa equipe irá:

- Verificar a integridade do arquivo e a criptografia dos dados do método de pagamento
- Validar seu CSV de assinantes para formatação correta
- Executar uma importação de teste em nosso ambiente sandbox
- Identificar e resolver quaisquer problemas de mapeamento de dados

<Check>
Você receberá um relatório de validação confirmando que tudo está pronto para a importação em produção.
</Check>
</Step>

<Step title="Production import">
Na data de importação acordada, nossa equipe de engenharia irá:

1. Importar os dados do método de pagamento em nosso Cofre de Cartões compatível com PCI
2. Importar seus dados de assinantes
3. Corresponder assinantes aos seus métodos de pagamento usando os tokens de cartão
4. Ativar todas as assinaturas migradas

<Info>
O tempo típico de processamento é de **2 a 3 dias úteis** para migrações padrão, dependendo do volume de assinantes.
</Info>
</Step>

<Step title="Post-migration verification">
Após a conclusão da importação, você receberá:

- Um relatório abrangente de migração
- Um CSV mapeando seus antigos IDs de assinatura para novos IDs do Dodo Payments
- Acesso a todos os clientes migrados em seu painel do Dodo Payments

Recomendamos:
- Verificar uma amostra das assinaturas migradas no seu painel
- Testar algumas renovações de assinatura no modo sandbox
- Atualizar as referências do seu banco de dados interno com os novos IDs
- Executar monitoramento paralelo durante o primeiro ciclo de cobrança
</Step>
</Steps>

## Dados migráveis

Os seguintes dados podem ser transferidos durante uma migração de assinaturas:

| Tipo de dado | Suportado | Notas |
|-----------|-----------|-------|
| Números de cartões de crédito/débito | ✅ | Criptografados via PGP |
| Datas de validade do cartão | ✅ | |
| Nomes dos titulares dos cartões | ✅ | |
| Endereços de cobrança | ✅ | |
| Endereços de e-mail dos clientes | ✅ | |
| Metadados dos clientes | ✅ | Pares chave-valor personalizados |
| Detalhes da assinatura | ✅ | Plano, valor, intervalo |
| Tokens do Apple Pay | ❌ | Não podem ser transferidos |
| Tokens do Google Pay | ❌ | Não podem ser transferidos |
| Outros tokens de carteira | ❌ | Não podem ser transferidos |
| Histórico de pagamentos | ❌ | Exportar do seu processador atual |

<Note>
O histórico de pagamentos e os registros de transações não podem ser migrados. Recomendamos exportar esses dados do seu processador atual para seus registros antes de concluir a transição.
</Note>

## Expectativas de cronograma

| Fase | Duração |
|-------|----------|
| Configuração inicial e planejamento | 1-2 dias |
| Transferência de método de pagamento | ~2 semanas |
| Preparação de dados do assinante | 3-5 dias (paralelo) |
| Validação e teste | 2-3 dias |
| Importação em produção | 2-3 dias úteis |
| Verificação pós-migração | 1-2 dias |

<Tip>
A migração total normalmente leva **3 a 4 semanas** do início ao fim. Começar a preparação dos dados dos assinantes enquanto a transferência do método de pagamento está em andamento ajuda a reduzir o tempo total.
</Tip>
