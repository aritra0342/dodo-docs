## O Que São Complementos?

Complementos são produtos suplementares que os clientes podem adquirir juntamente com sua assinatura principal. Eles são perfeitos para:

- **Cobrança baseada em assentos**: Membros adicionais da equipe, licenças de usuário ou usuários simultâneos
- **Upgrades de recursos**: Recursos premium, análises avançadas ou suporte prioritário
- **Extensões de uso**: Armazenamento extra, chamadas de API ou permissões de largura de banda
- **Complementos de serviço**: Serviços profissionais, treinamento ou horas de consultoria

## Principais Benefícios

- **Modelos de Preços Flexíveis**: Ofereça planos base com complementos opcionais para criar estruturas de preços sofisticadas. Você pode atender a diversos segmentos de clientes com upgrades que crescem à medida que as necessidades dos seus clientes mudam.
- **Otimização de Receita**: Aumente sua receita média por usuário (ARPU) apresentando complementos relevantes. Isso permite oportunidades naturais de upsell à medida que os clientes adicionam recursos ao longo do tempo.
- **Gerenciamento Simplificado**: Gerencie todos os componentes de preços a partir de um único painel. Os complementos são automaticamente incluídos tanto nas sessões de checkout quanto na gestão de assinaturas.
- **Escolha do Cliente**: Permita que os clientes personalizem suas assinaturas selecionando apenas os complementos que precisam, o que aumenta a satisfação e reduz a rotatividade.

## Criando Complementos

Os complementos são criados como produtos separados no seu painel do Dodo Payments e, em seguida, anexados aos seus produtos principais de assinatura. Essa separação permite que você:

- Reutilize complementos em vários produtos de assinatura
- Gerencie preços de forma independente
- Acompanhe o desempenho dos complementos separadamente
- Atualize complementos sem afetar as assinaturas base

### Configuração de Complementos

Ao criar complementos, você pode configurar:

- **Preços**: Defina preços únicos ou recorrentes para o complemento
- **Ciclo de faturamento**: Deve corresponder ao ciclo de faturamento da sua assinatura
- **Limites de quantidade**: Defina quantidades mínimas e máximas por cliente
- **Disponibilidade**: Controle quais produtos de assinatura podem usar o complemento
- **Configurações de imposto**: Configure categorias de imposto apropriadas

## Exemplos de Integração

### Sessões de Checkout com Complementos
Ao criar sessões de checkout, você pode incluir complementos com quantidades personalizadas:

```typescript
const session = await client.checkoutSessions.create({
  product_cart: [
    {
      product_id: 'your_subscription_id',
      quantity: 1,
      addons: [
        {
          addon_id: 'your_addon_id',
          quantity: 3 // 3 additional seats
        }
      ]
    }
  ],
  // ... other checkout options
});
```

### Alterações de Plano com Complementos
Modifique assinaturas existentes para adicionar, remover ou atualizar complementos:

```typescript
// Add add-ons to existing subscription
await client.subscriptions.changePlan('sub_123', {
  product_id: 'prod_new',
  quantity: 1,
  proration_billing_mode: 'difference_immediately',
  addons: [
    { addon_id: 'addon_123', quantity: 2 }
  ]
});

// Remove all existing add-ons
await client.subscriptions.changePlan('sub_123', {
  product_id: 'prod_new',
  quantity: 1,
  proration_billing_mode: 'difference_immediately',
  addons: [] // Empty array removes all existing add-ons
});
```

### Preços Dinâmicos
Calcule custos totais dinamicamente com base nas seleções de complementos:

```typescript
function calculateTotalCost(basePrice: number, addons: AddonSelection[]) {
  const addonTotal = addons.reduce((sum, addon) => 
    sum + (addon.price * addon.quantity), 0
  );
  return basePrice + addonTotal;
}
```

## Melhores Práticas

- **Comece simples**: Lance com 2-3 complementos principais e expanda as opções com base no feedback e uso dos clientes.
- **Mantenha clareza nos preços**: Comunique claramente os preços e o valor dos complementos, para que os clientes entendam o que estão recebendo pelo custo adicional.
- **Teste minuciosamente**: Valide combinações de complementos para garantir que os cálculos de preços permaneçam precisos e que os fluxos de checkout funcionem sem problemas.

### Considerações de Design

- **Proposta de Valor Clara**: Cada complemento deve ter um benefício claro que os clientes possam entender facilmente
- **Agrupamento Lógico**: Agrupe complementos relacionados juntos no seu fluxo de checkout
- **Quantidades Flexíveis**: Permita que os clientes ajustem as quantidades de complementos conforme necessário
- **Preços Transparentes**: Mostre os custos totais claramente durante todo o processo de checkout

<Info>
Add-ons são uma forma poderosa de criar modelos de precificação flexíveis e escaláveis que crescem junto com seus clientes. Comece por casos de uso simples e expanda conforme aprende o que funciona melhor para o seu negócio e seus clientes.
</Info>
