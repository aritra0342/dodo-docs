## Como Funciona

Os métodos de pagamento são apresentados automaticamente aos clientes com base em:

- **Localização do Cliente** — Métodos regionais como iDEAL (Países Baixos) ou UPI (Índia) aparecem apenas para clientes nessas regiões
- **Moeda da Transação** — Métodos de pagamento são filtrados por moedas suportadas
- **Tipo de Dispositivo** — Apple Pay aparece apenas em dispositivos e navegadores Apple compatíveis
- **Tipo de Transação** — Alguns métodos não suportam assinaturas ou têm valores mínimos

<Info>
**Ativação Automática:** Todos os métodos de pagamento são pré-configurados e gerenciados pela Dodo Payments. Não é necessário nenhum ajuste adicional — eles ficam disponíveis assim que sua conta for verificada e o modo produção estiver habilitado.
</Info>

## Configurando Métodos de Pagamento

Por padrão, todos os métodos de pagamento elegíveis aparecem automaticamente. Use `allowed_payment_method_types` para restringir quais métodos são exibidos:

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://test.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

### Tipos de Métodos Disponíveis

| Tipo | Descrição |
| :--- | :---------- |
| `credit` | Cartões de crédito (Visa, Mastercard, Amex etc.) |
| `debit` | Cartões de débito |
| `apple_pay` | Apple Pay |
| `google_pay` | Google Pay |
| `amazon_pay` | Amazon Pay |
| `cashapp` | Cash App Pay |
| `klarna` | Klarna BNPL |
| `afterpay_clearpay` | Afterpay / Clearpay |
| `upi_collect` | UPI Collect (código QR) |
| `ideal` | iDEAL (Países Baixos) |
| `bancontact_card` | Bancontact (Bélgica) |
| `eps` | EPS (Áustria) |
| `multibanco` | Multibanco (Portugal) |
| `revolut_pay` | RevolutPay |
| `billie` | Billie B2B BNPL |

<Warning>
Inclua sempre `credit` e `debit` como alternativas. Se todos os métodos especificados estiverem indisponíveis para um cliente, o checkout falhará.
</Warning>

## Melhores Práticas

<AccordionGroup>
<Accordion title="Match methods to your market">
Inclua métodos de pagamento regionais para seus mercados-alvo. iDEAL representa ~60% dos pagamentos online nos Países Baixos; UPI domina na Índia. Omitir esses métodos prejudica significativamente a conversão.
</Accordion>

<Accordion title="Always provide card fallbacks">
Mesmo quando segmentar métodos específicos, inclua `credit` e `debit` como alternativas. Nem todos os clientes têm acesso a todos os métodos de pagamento.
</Accordion>

<Accordion title="Consider transaction minimums">
Métodos BNPL (Klarna, Afterpay) exigem mínimo de $50,01. Não os inclua para produtos de baixo valor — eles simplesmente não aparecerão, o que é normal.
</Accordion>

<Accordion title="Test across regions and devices">
Use o modo de teste para verificar a disponibilidade dos métodos de pagamento para diferentes localizações de clientes, moedas e tipos de dispositivo antes de ir ao ar.
</Accordion>
</AccordionGroup>

## Solução de Problemas

<AccordionGroup>
<Accordion title="Payment method not appearing">
**Verifique estes fatores:**
1. Método incluído em `allowed_payment_method_types`?
2. Localização do cliente corresponde à região suportada pelo método?
3. Moeda da transação suportada pelo método?
4. Valor da transação atende ao mínimo (por exemplo, $50,01 para BNPL)?
5. Para assinaturas: o método suporta pagamentos recorrentes?

**Solução:** Remova temporariamente `allowed_payment_method_types` para ver todos os métodos elegíveis e identifique o problema.
</Accordion>

<Accordion title="Unexpected methods appearing">
**Possíveis causas:**
- `allowed_payment_method_types` não está sendo enviado corretamente
- Parâmetro formatado incorretamente (deve ser um array de strings)

**Solução:** Registre o payload da solicitação da API e verifique a estrutura do parâmetro.
</Accordion>

<Accordion title="Regional method not available">
**Verifique:**
- O endereço de cobrança do cliente corresponde à região do método
- A moeda está definida corretamente (por exemplo, EUR para iDEAL)
- A Moeda Adaptativa está habilitada se o cliente for internacional

**Solução:** Verifique o código do país do endereço de cobrança e a moeda na sua requisição da API.
</Accordion>

<Accordion title="Indian payment method issues">
As assinaturas UPI e Rupay têm requisitos únicos, incluindo atrasos de processamento de 48 horas e limites de mandato do RBI.

**Veja:** <a href="/features/payment-methods/india">India Payment Methods</a> para orientações completas.
</Accordion>
</AccordionGroup>
