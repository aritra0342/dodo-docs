## Como Funciona

Os métodos de pagamento são apresentados automaticamente aos clientes com base em:

- **Localização do Cliente** — Métodos regionais como iDEAL (Países Baixos) ou UPI (Índia) aparecem apenas para clientes nessas regiões
- **Moeda da Transação** — Métodos de pagamento são filtrados por moedas suportadas
- **Tipo de Dispositivo** — Apple Pay aparece apenas em dispositivos e navegadores Apple compatíveis
- **Tipo de Transação** — Alguns métodos não suportam assinaturas ou têm valores mínimos

<Info>
**Ativação Automática:** Todos os métodos de pagamento são pré-configurados e gerenciados pela Dodo Payments. Nenhuma configuração adicional é necessária — eles se tornam disponíveis assim que sua conta é verificada e o modo ao vivo é habilitado.
</Info>

## Configurando Métodos de Pagamento

Por padrão, todos os métodos de pagamento elegíveis aparecem automaticamente. Use `allowed_payment_method_types` para restringir quais métodos são mostrados:

<Tabs>
<Tab title="JavaScript">

```javascript
const session = await client.checkoutSessions.create({
  product_cart: [{ product_id: 'prod_123', quantity: 1 }],
  allowed_payment_method_types: [
    'credit',
    'debit', 
    'apple_pay',
    'google_pay'
  ],
  return_url: 'https://example.com/success'
});
```

</Tab>

<Tab title="Python">

```python
session = client.checkout_sessions.create(
    product_cart=[{"product_id": "prod_123", "quantity": 1}],
    allowed_payment_method_types=[
        "credit",
        "debit",
        "apple_pay", 
        "google_pay"
    ],
    return_url="https://example.com/success"
)
```

</Tab>

<Tab title="cURL">

```bash
curl -X POST https://api.dodopayments.com/checkout-sessions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "product_cart": [{"product_id": "prod_123", "quantity": 1}],
    "allowed_payment_method_types": ["credit", "debit", "apple_pay", "google_pay"],
    "return_url": "https://example.com/success"
  }'
```

</Tab>
</Tabs>

### Tipos de Métodos Disponíveis

| Tipo | Descrição |
| :--- | :---------- |
| `credit` | Cartões de crédito (Visa, Mastercard, Amex, etc.) |
| `debit` | Cartões de débito |
| `apple_pay` | Apple Pay |
| `google_pay` | Google Pay |
| `amazon_pay` | Amazon Pay |
| `cashapp` | Cash App Pay |
| `klarna` | Klarna BNPL |
| `afterpay_clearpay` | Afterpay / Clearpay |
| `upi_collect` | UPI Collect (QR code) |
| `ideal` | iDEAL (Países Baixos) |
| `bancontact_card` | Bancontact (Bélgica) |
| `eps` | EPS (Áustria) |
| `multibanco` | Multibanco (Portugal) |
| `revolut_pay` | RevolutPay |
| `billie` | Billie B2B BNPL |

<Warning>
Sempre inclua `credit` e `debit` como alternativas. Se todos os métodos de pagamento especificados estiverem indisponíveis para um cliente, o checkout falhará.
</Warning>

## Melhores Práticas

<AccordionGroup>
<Accordion title="Combine métodos ao seu mercado">
Inclua métodos de pagamento regionais para seus mercados-alvo. O iDEAL captura ~60% dos pagamentos online nos Países Baixos; o UPI domina na Índia. Omitir esses métodos prejudica significativamente a conversão.
</Accordion>

<Accordion title="Sempre forneça alternativas de cartão">
Mesmo ao direcionar métodos específicos, inclua `credit` e `debit` como alternativas. Nem todos os clientes têm acesso a todos os métodos de pagamento.
</Accordion>

<Accordion title="Considere mínimos de transação">
Métodos de BNPL (Klarna, Afterpay) requerem mínimo de $50.01. Não os inclua para produtos de baixo valor — eles simplesmente não aparecerão, o que é aceitável.
</Accordion>

<Accordion title="Teste entre regiões e dispositivos">
Use o modo de teste para verificar a disponibilidade do método de pagamento para diferentes localizações de clientes, moedas e tipos de dispositivos antes de entrar ao vivo.
</Accordion>
</AccordionGroup>

## Solução de Problemas

<AccordionGroup>
<Accordion title="Método de pagamento não aparecendo">
**Verifique estes fatores:**
1. Método incluído em `allowed_payment_method_types`?
2. Localização do cliente corresponde à região suportada pelo método?
3. Moeda da transação suportada pelo método?
4. Valor da transação atende ao mínimo (por exemplo, $50.01 para BNPL)?
5. Para assinaturas: método suporta pagamentos recorrentes?

**Solução:** Remova `allowed_payment_method_types` temporariamente para ver todos os métodos elegíveis, e depois reduza o problema.
</Accordion>

<Accordion title="Métodos inesperados aparecendo">
**Causas possíveis:**
- `allowed_payment_method_types` não está sendo passado corretamente
- Parâmetro formatado incorretamente (devem ser um array de strings)

**Solução:** Registre o payload da sua solicitação da API e verifique a estrutura do parâmetro.
</Accordion>

<Accordion title="Método regional não disponível">
**Verifique:**
- Endereço de cobrança do cliente corresponde à região do método
- Moeda está configurada corretamente (por exemplo, EUR para iDEAL)
- Moeda Adaptativa está habilitada se o cliente for internacional

**Solução:** Verifique o código do país do endereço de cobrança e a moeda na sua solicitação da API.
</Accordion>

<Accordion title="Problemas com métodos de pagamento indianos">
Assinaturas UPI e Rupay têm requisitos únicos, incluindo atrasos de processamento de 48 horas e limites de mandato do RBI.

**Veja:** <a href="/features/payment-methods/india">Métodos de Pagamento na Índia</a> para orientação completa.
</Accordion>
</AccordionGroup>
