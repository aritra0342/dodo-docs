Seu painel Dodo Payments, a poucos toques de distância. Use o Raycast para visualizar rapidamente pagamentos, assinaturas, clientes, produtos, códigos de desconto, chaves de licença e reembolsos sem abrir o painel.

## Instalação

<Card title="Install from Raycast Store →" icon="puzzle-piece" href="https://www.raycast.com/dodopayments/dodo-payments">
  Abra a página da extensão Dodo Payments na Raycast Store.
</Card>

<Info>
Requer Raycast no macOS e uma chave de API da Dodo Payments.
</Info>

## Configuração

<Steps>
<Step title="Get your API Key">
1. Faça login no seu painel da Dodo Payments.
2. Vá para Configurações › Chaves de API.
3. Copie sua chave de API.
</Step>

<Step title="Configure the Extension">
1. Abra o Raycast.
2. Procure por "Dodo Payments" e selecione qualquer comando.
3. Quando solicitado, insira sua chave de API no campo API Key.
4. Escolha o modo da API:
   - Test: use para desenvolvimento e testes.
   - Live: use para dados de produção.
5. Salve as configurações.
</Step>
</Steps>

<Tip>
Comece no modo Test para validar sua configuração e depois altere para Live.
</Tip>

<Warning>
Trate sua chave de API como uma senha. Não a compartilhe nem armazene em texto simples.
</Warning>

## Use a extensão

- **Pesquisar**: abra o Raycast, digite "Dodo Payments" e escolha um comando.
- **Filtrar**: refine os resultados digitando partes de IDs, e-mails, nomes ou status.
- **Abrir detalhes**: pressione Enter em um item da lista para ver mais informações.

## Solução de Problemas

<AccordionGroup>
<Accordion title="Authentication failed or 401/403 errors">
Confirme se a chave de API e o modo (Test vs Live) correspondem aos dados do seu painel. Reabra o comando e insira a chave novamente, se necessário.
</Accordion>
<Accordion title="No results appear">
Garanta que o modo selecionado contenha dados. Tente alternar entre Test e Live.
</Accordion>
<Accordion title="Rate limits or timeouts">
Aguarde um momento e tente novamente. Se o problema persistir, verifique sua conectividade de rede.
</Accordion>
</AccordionGroup>
