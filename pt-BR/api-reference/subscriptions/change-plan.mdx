## Tratamento de Falhas de Pagamento

Use o parâmetro `on_payment_failure` para controlar o que acontece quando o pagamento da alteração de plano falha:

| Value | Comportamento |
|-------|---------------|
| `prevent_change` | Mantenha a assinatura no plano atual até que o pagamento seja aprovado. A alteração de plano permanece pendente. |
| `apply_change` | Aplique a alteração de plano imediatamente independentemente do resultado do pagamento. Este é o padrão. |

<Info>
Se `on_payment_failure` não for especificado, o comportamento padrão será o definido nas configurações do nível empresarial no painel.
</Info>

<Tip>
Use `prevent_change` para atualizações críticas onde você deseja garantir o pagamento antes de conceder acesso a recursos premium.
</Tip>
