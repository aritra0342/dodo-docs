<Info>
  **Pré-requisito**: Por favor, instale o Node.js (versão 19 ou superior) antes de prosseguir.
</Info>

Siga estas etapas para instalar e executar o Mintlify em seu sistema operacional:

**Passo 1**: Instale o Mintlify:

<CodeGroup>

  ```bash npm
  npm i -g mintlify
  ```

```bash yarn
yarn global add mintlify
```

</CodeGroup>

**Passo 2**: Navegue até o diretório de docs (onde o arquivo `mint.json` está localizado) e execute o seguinte comando:

```bash
mintlify dev
```

Uma pré-visualização local de sua documentação estará disponível em `http://localhost:3000`.

### Portas Personalizadas

Por padrão, o Mintlify usa a porta 3000. Você pode personalizar a porta em que o Mintlify é executado usando a flag `--port`. Para executar o Mintlify na porta 3333, por exemplo, use este comando:

```bash
mintlify dev --port 3333
```

Se você tentar executar o Mintlify em uma porta que já está em uso, ele usará a próxima porta disponível:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Versões do Mintlify

Por favor, note que cada lançamento do CLI está associado a uma versão específica do Mintlify. Se seu site local não estiver alinhado com a versão de produção, por favor, atualize o CLI:

<CodeGroup>

```bash npm
npm i -g mintlify@latest
```

```bash yarn
yarn global upgrade mintlify
```

</CodeGroup>

## Validando Links

O CLI pode ajudar a validar links de referência feitos em sua documentação. Para identificar quaisquer links quebrados, use o seguinte comando:

```bash
mintlify broken-links
```

## Formatação de Código

Sugerimos usar extensões em seu IDE para reconhecer e formatar MDX. Se você é um usuário do VSCode, considere a [extensão MDX para VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) para destaque de sintaxe, e [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) para formatação de código.

## Solução de Problemas

<AccordionGroup>
  <Accordion title='Erro: Não foi possível carregar o módulo "sharp" usando o runtime darwin-arm64'>

    Isso pode ser devido a uma versão desatualizada do node. Tente o seguinte:
    1. Remova a versão atualmente instalada do mintlify: `npm remove -g mintlify`
    2. Atualize para o Node v19 ou superior.
    3. Reinstale o mintlify: `npm install -g mintlify`
  </Accordion>

  <Accordion title="Problema: Encontrando um erro desconhecido">
  
    Solução: Vá até a raiz do seu dispositivo e delete a pasta \~/.mintlify. Depois, execute `mintlify dev` novamente.
  </Accordion>
</AccordionGroup>

Curioso sobre o que mudou na versão do CLI? [Confira o changelog do CLI.](https://www.npmjs.com/package/mintlify?activeTab=versions)
