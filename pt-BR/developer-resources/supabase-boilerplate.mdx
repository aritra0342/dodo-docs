## Visão Geral

Um boilerplate pronto para produção para assinaturas usando Next.js 15, React 19, Supabase, Drizzle ORM e Dodo Payments. Ele vem com Google OAuth, checkout de assinatura, manipulação de webhook, esquema de banco de dados e um painel básico.

<Info>
Se você precisa apenas de manipuladores de rota para um app existente, veja os adaptadores dedicados: <a href="/developer-resources/nextjs-adaptor">Next.js Adaptor</a> e <a href="/developer-resources/express-adaptor">Express Adaptor</a>.
</Info>

## Pré-requisitos

- Node.js 18+ (ou Bun 1.0+)
- Projeto Supabase (URL, chave Anon, chave de função de serviço, URL do banco de dados)
- Conta Dodo Payments (chave API, segredo do webhook)
- Cliente OAuth do Google Cloud (ID do cliente e segredo)

## Início Rápido

<Steps>
{/* LOCKED_PATTERN_587a3a4c926aec89f48dfc1e7c21054e */}

```bash
git clone https://github.com/dodopayments/dodo-supabase-subscription-starter.git
cd dodo-supabase-subscription-starter
# choose one
bun install
# or
npm install
# or
pnpm install
```

</Step>

{/* LOCKED_PATTERN_ea54c61e84e28cd275c0eed6d7634e0f */}
Crie um projeto Supabase e copie:
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
- SUPABASE_SERVICE_ROLE_KEY
- DATABASE_URL (string de conexão)
</Step>

{/* LOCKED_PATTERN_42bf2f0f0010fd47b77c88325d34313c */}
Defina o URI de redirecionamento como `https://[your-project-ref].supabase.co/auth/v1/callback` no Google Cloud e então habilite o provedor Google no Supabase Auth usando seu Client ID e Secret.
</Step>

{/* LOCKED_PATTERN_f32b65d8a917a558befa771bede82115 */}
Gere uma chave de API e um segredo de webhook no painel da Dodo. Defina o ambiente como `test_mode` durante o desenvolvimento.
</Step>

{/* LOCKED_PATTERN_a34a8e162789e5a79d108d9eb3c86df5 */}

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role

# Database
DATABASE_URL=postgresql://postgres:[password]@db.[project-ref].supabase.co:5432/postgres

# Dodo Payments
DODO_PAYMENTS_API_KEY=your-dodo-api-key
DODO_WEBHOOK_SECRET=your-webhook-secret
DODO_PAYMENTS_ENVIRONMENT=test_mode
```

<Warning>
Nunca comprometa segredos. Use variáveis de ambiente nos ambientes de implantação.
</Warning>
</Step>

{/* LOCKED_PATTERN_c35fe33eae6aa68b4bc39bd740632964 */}

```bash
bun run db:push
# or
npm run db:push
# or
pnpm run db:push
```

<Check>
Tabelas criadas: `users`, `subscriptions`, `payments`.
</Check>
</Step>

{/* LOCKED_PATTERN_560d1d81fad22bd0622f0c8dcb3c706d */}

```bash
# login (one-time)
bunx supabase login
# or
npx supabase login

# deploy the edge function
bun run deploy:webhook --project-ref [your-project-ref]
# or
npm run deploy:webhook -- --project-ref [your-project-ref]
# or
pnpm run deploy:webhook --project-ref [your-project-ref]
```

```bash cURL
curl -X POST \
  'https://[your-project-ref].supabase.co/functions/v1/dodo-webhook' \
  -H 'Content-Type: application/json' \
  -H 'Dodo-Signature: <signature>' \
  -d '{"type":"payment.succeeded","data":{}}'
```

</Step>

{/* LOCKED_PATTERN_dd5db8f3734010c91f70fdf794c4e2a2 */}
Defina a URL do endpoint para:



Selecione eventos de pagamento e assinatura.
</Step>

{/* LOCKED_PATTERN_e437399605995e5a4845b4d749b71d01 */}

</Step>
</Steps>

## O que está incluído

- Autenticação via Supabase (Google OAuth configurado)
- Checkout de assinatura via Dodo Payments
- Função Supabase Edge para webhooks (`dodo-webhook`)
- Esquema e migrações do Drizzle ORM
- Painel com faturas, status de assinatura e recursos do plano

<Tip>
Mantenha `DODO_PAYMENTS_ENVIRONMENT` como `test_mode` até concluir os testes ponta a ponta.
</Tip>

## Verificação e solução de problemas

<AccordionGroup>
{/* LOCKED_PATTERN_6b286e9e2bbacd5c7684f8442203f10e */}
- Garanta que `DODO_WEBHOOK_SECRET` corresponda ao valor do painel da Dodo
- Confirme que você implantou a última função `dodo-webhook`
- Verifique se o nome do cabeçalho está correto na sua função (Dodo-Signature)
</Accordion>
{/* LOCKED_PATTERN_c270a2f8797d0168d608bb07bd40f056 */}
- Confira a sintaxe de `DATABASE_URL` e o egress de rede do Supabase
- Aguarde cerca de 2–3 minutos após a criação do projeto antes do primeiro push
</Accordion>
{/* LOCKED_PATTERN_276d2f8137bb346c423586f69d6b7cae */}
- O URI de redirecionamento deve ser `https://[ref].supabase.co/auth/v1/callback`
- Certifique-se de usar o mesmo valor no Google Cloud e no provedor Supabase Auth
</Accordion>
</AccordionGroup>
