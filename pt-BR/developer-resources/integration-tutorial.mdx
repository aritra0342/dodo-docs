## Pré-requisitos

Antes de começar, certifique-se de que você tenha:

- [Criado uma conta Dodo Payments](/quickstart)
- [Obtido suas chaves de API](/api-reference/introduction#authentication)
- Conhecimento básico de Next.js e React
- Node.js instalado (v14 ou superior)

## Configuração do Boilerplate

Clone nosso boilerplate Next.js para começar rapidamente:

```bash
git clone https://github.com/dodopayments/dodo-nextjs-minimal-boilerplate
cd dodo-nextjs-minimal-boilerplate
npm install
```

<Note>
O boilerplate inclui todas as dependências necessárias e a configuração básica para a integração do Dodo Payments.
</Note>

## Integração de Webhook

Configure webhooks para gerenciar eventos assíncronos:

```typescript
import { verifyWebhookSignature } from '@dodopayments/node';

export default async function handler(req, res) {
  const signature = req.headers['dodo-signature'];
  const payload = req.body;
  
  try {
    verifyWebhookSignature({
      payload,
      signature,
      secret: process.env.WEBHOOK_SECRET
    });
    
    // Handle webhook event
    switch (payload.type) {
      case 'payment.succeeded':
        // Handle successful payment
        break;
      case 'subscription.created':
        // Handle new subscription
        break;
      // ... handle other events
    }
    
    res.status(200).json({ received: true });
  } catch (err) {
    res.status(400).json({ error: 'Invalid signature' });
  }
}
```
