## Início Rápido

Comece com o rastreamento automático de tokens LLM em apenas 2 minutos:

<Steps>
{/* LOCKED_PATTERN_776027320500bde6b99bac6bed1cc64d */}
Instale os Blueprints de Ingestão da Dodo Payments:

```bash
npm install @dodopayments/ingestion-blueprints
```

</Step>

{/* LOCKED_PATTERN_e608972569617212f31c0b7d72677865 */}
Você precisará de duas chaves de API:

- **Chave de API do Dodo Payments**: Obtenha-a no [Painel do Dodo Payments](https://app.dodopayments.com/developer/api-keys)
- **Chave de API do Provedor LLM**: Do AI SDK, OpenAI, Anthropic, Groq, etc.

<Tip>
Armazene suas chaves de API com segurança em variáveis de ambiente. Nunca as comite no controle de versão.
</Tip>
</Step>

{/* LOCKED_PATTERN_3c26a22b684f0fb6b635e5bf1e61dd55 */}
Antes de rastrear o uso, crie um medidor no painel da Dodo Payments:

1. **Faça login** no [Painel da Dodo Payments](https://app.dodopayments.com/)
2. **Navegue até** Produtos → Medidores
3. **Clique** em "Criar Medidor"
4. **Configure seu medidor**:
   - **Nome do Medidor**: Escolha um nome descritivo (por exemplo, "Uso de Tokens LLM")
   - **Nome do Evento**: Defina um identificador único para o evento (por exemplo, `llm.chat_completion`)
   - **Tipo de Agregação**: Selecione `sum` para somar as contagens de tokens
   - **Propriedade Over**: Escolha o que rastrear:
     - `inputTokens` - Rastreie tokens de entrada/prompt
     - `outputTokens` - Rastreie tokens de saída/completion (inclui tokens de raciocínio quando aplicável)
     - `totalTokens` - Rastreie tokens combinados de entrada + saída

<Info>
O **Nome do Evento** definido aqui deve coincidir exatamente com o que você envia para o SDK (sensível a maiúsculas/minúsculas).
</Info>

Para instruções detalhadas, consulte o [Guia de Cobrança Baseada em Uso](/developer-resources/usage-based-billing-guide).
</Step>

## Configuração

### Configuração do Rastreador

Crie um rastreador uma vez na inicialização do aplicativo com estes parâmetros obrigatórios:

<ParamField path="apiKey" type="string" required>
Sua chave de API da Dodo Payments. Obtenha-a na [página de Chaves de API](https://app.dodopayments.com/developer/api-keys).

```javascript
apiKey: process.env.DODO_PAYMENTS_API_KEY
```

</ParamField>

<ParamField path="environment" type="string" required>
O modo de ambiente para o rastreador.

- `test_mode` - Use para desenvolvimento e testes
- `live_mode` - Use para produção

```javascript
environment: 'test_mode' // or 'live_mode'
```

<Warning>
Sempre use `test_mode` durante o desenvolvimento para evitar impactar métricas de produção.
</Warning>
</ParamField>

<ParamField path="eventName" type="string" required>
O nome do evento que dispara seu medidor. Deve corresponder exatamente ao que você configurou no medidor da Dodo Payments (sensível a maiúsculas/minúsculas).

```javascript
eventName: 'llm.chat_completion'
```

<Info>
Esse nome de evento vincula seu uso rastreado ao medidor correto para os cálculos de cobrança.
</Info>
</ParamField>

### Configuração do Wrapper

Ao envolver seu cliente LLM, forneça estes parâmetros:

<ParamField path="client" type="object" required>
Sua instância do cliente LLM (OpenAI, Anthropic, Groq, etc.).

```javascript
client: openai
```

</ParamField>

<ParamField path="customerId" type="string" required>
O identificador exclusivo do cliente para cobrança. Deve corresponder ao seu ID de cliente na Dodo Payments.

```javascript
customerId: 'customer_123'
```

<Tip>
Use o ID do usuário ou do cliente da sua aplicação para garantir cobrança precisa por cliente.
</Tip>
</ParamField>

<ParamField path="metadata" type="object">
Dados adicionais opcionais para anexar ao evento de rastreamento. Útil para filtragem e análise.

```javascript
metadata: {
  feature: 'chat',
  userTier: 'premium',
  sessionId: 'session_123',
  modelVersion: 'gpt-4'
}
```

</ParamField>

### Exemplo Completo de Configuração

<CodeGroup>

```javascript Full Configuration
import { createLLMTracker } from "@dodopayments/ingestion-blueprints";
import { generateText } from "ai";
import { google } from "@ai-sdk/google";
import "dotenv/config";

async function aiSdkExample() {
  console.log("🤖 AI SDK Simple Usage Example\n");

  try {
    // 1. Create tracker
    const llmTracker = createLLMTracker({
      apiKey: process.env.DODO_PAYMENTS_API_KEY!,
      environment: "test_mode",
      eventName: "your_meter_event_name",
    });

    // 2. Wrap the ai-sdk methods
    const client = llmTracker.wrap({
      client: { generateText },
      customerId: "customer_123",
      metadata: {
        provider: "ai-sdk",
      },
    });

    // 3. Use the wrapped function
    const response = await client.generateText({
      model: google("gemini-2.5-flash"),
      prompt: "Hello, I am a cool guy! Tell me a fun fact.",
      maxOutputTokens: 500,
    });

    console.log(response);
    console.log(response.usage);
    console.log("✅ Automatically tracked for customer\n");
  } catch (error) {
    console.error(error);
  }
}

aiSdkExample().catch(console.error);
```

### Integração da API Express.js

Exemplo completo de integração do rastreamento LLM em uma API Express.js:

<CodeGroup>

```javascript Express.js Server
import express from 'express';
import { createLLMTracker } from '@dodopayments/ingestion-blueprints';
import OpenAI from 'openai';

const app = express();
app.use(express.json());

// Initialize OpenAI client
const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

// Create tracker once at startup
const tracker = createLLMTracker({
  apiKey: process.env.DODO_PAYMENTS_API_KEY,
  environment: process.env.NODE_ENV === 'production' ? 'live_mode' : 'test_mode',
  eventName: 'api.chat_completion'
});

// Chat endpoint with automatic tracking
app.post('/api/chat', async (req, res) => {
  try {
    const { message, userId } = req.body;
    
    // Validate input
    if (!message || !userId) {
      return res.status(400).json({ error: 'Missing message or userId' });
    }
    
    // Wrap client for this specific user
    const trackedClient = tracker.wrap({
      client: openai,
      customerId: userId,
      metadata: { 
        endpoint: '/api/chat',
        timestamp: new Date().toISOString()
      }
    });
    
    // Make LLM request - automatically tracked
    const response = await trackedClient.chat.completions.create({
      model: 'gpt-4',
      messages: [{ role: 'user', content: message }],
      temperature: 0.7
    });
    
    const completion = response.choices[0].message.content;
    
    res.json({ 
      message: completion,
      usage: response.usage
    });
  } catch (error) {
    console.error('Chat error:', error);
    res.status(500).json({ error: 'Internal server error' });
  }
});

app.listen(3000, () => {
  console.log('Server running on port 3000');
});
```

</CodeGroup>

## O que é Rastreado

Cada chamada de API LLM envia automaticamente um evento de uso para o Dodo Payments com a seguinte estrutura:

<CodeGroup>

```json Event Structure
{
  "event_id": "llm_1673123456_abc123",
  "customer_id": "customer_123",
  "event_name": "llm.chat_completion",
  "timestamp": "2024-01-08T10:30:00Z",
  "metadata": {
    "inputTokens": 10,
    "outputTokens": 25,
    "totalTokens": 35,
    "model": "gpt-4",
  }
}
```

</CodeGroup>

### Campos do Evento

<ParamField path="event_id" type="string">
Identificador exclusivo para este evento específico. Gerado automaticamente pelo SDK.

Formato: `llm_[timestamp]_[random]`
</ParamField>

<ParamField path="customer_id" type="string">
O ID do cliente que você forneceu ao embrulhar o cliente. Usado para cobrança.
</ParamField>

<ParamField path="event_name" type="string">
O nome do evento que dispara seu medidor. Corresponde à configuração do seu rastreador.
</ParamField>

<ParamField path="timestamp" type="string">
Timestamp ISO 8601 quando o evento ocorreu.
</ParamField>

<ParamField path="metadata" type="object">
Uso de tokens e dados adicionais de rastreamento:

- `inputTokens` - Número de tokens de entrada/prompt usados
- `outputTokens` - Número de tokens de saída/completion usados (inclui tokens de raciocínio quando aplicável)
- `totalTokens` - Total de tokens (entrada + saída)
- `model` - O modelo LLM utilizado (por exemplo, "gpt-4")
- `provider` - O provedor LLM (se incluído nos metadados do wrapper)
- Quaisquer metadados personalizados que você forneceu ao embrulhar o cliente

<Note>
**Tokens de Raciocínio:** Para modelos com capacidades de raciocínio, `outputTokens` inclui automaticamente tanto os tokens de conclusão quanto os de raciocínio.
</Note>
</ParamField>

<Note>
**Tokens de Raciocínio:** Para modelos com capacidades de raciocínio, `outputTokens` inclui automaticamente tanto os tokens de conclusão quanto os tokens de raciocínio.
</Note>
</ParamField>

<Info>
Seu medidor da Dodo Payments usa os campos `metadata` (especialmente `inputTokens`, `outputTokens` ou `totalTokens`) para calcular o uso e a cobrança.
</Info>
