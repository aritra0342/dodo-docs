## Visão Geral

Um boilerplate pronto para produção para integrar Dodo Payments em aplicativos Expo/React Native. Este template inclui criação de sessão de checkout, manuseio de pagamentos e configuração adequada do TypeScript para ajudá-lo a começar a aceitar pagamentos em seu aplicativo móvel rapidamente.

<Info>
Este boilerplate utiliza o SDK do Expo com TypeScript e inclui implementações de exemplo para sessões de checkout e processamento de pagamentos.
</Info>

## Pré-requisitos

- Node.js 18+
- Expo CLI ou EAS CLI
- Conta Dodo Payments (chave API do painel)
- Simulador iOS ou Emulador Android (ou dispositivo físico)

## Início Rápido

<Steps>
{/* LOCKED_PATTERN_3810046f56651f3ba0e1684e009cd2e3 */}
Clone o repositório do boilerplate e navegue até o diretório do projeto:

```bash
git clone https://github.com/dodopayments/expo-boilerplate.git
cd expo-boilerplate
```

</Step>

{/* LOCKED_PATTERN_985ffb13172d607786ae5265635e02cf */}
Obtenha suas credenciais no painel da Dodo Payments no Modo Teste:
</Step>

- **Chave API**: Navegue até **Desenvolvedor → Chaves API** em seu [painel](https://app.dodopayments.com)

<Tip>
Comece com o Modo Teste para testar sua integração com segurança sem processar pagamentos reais.
</Tip>
</Step>

{/* LOCKED_PATTERN_f8f1c4a54590eca96bda1740ea80800b */}
Crie um arquivo `.env` no diretório raiz:
</Step>

```bash
touch .env
```

Adicione suas credenciais Dodo Payments:

```bash .env
EXPO_PUBLIC_DODO_API_KEY=your_test_api_key_here
```

<Warning>
Nunca inclua seu arquivo `.env` no controle de versão. Ele já está incluído em `.gitignore`.
</Warning>
</Step>

{/* LOCKED_PATTERN_5e9c44b19c1da0e28673e0b0f87e9162 */}
Instale os pacotes npm necessários:
</Step>

```bash
npm install
```

</Step>

{/* LOCKED_PATTERN_e0b1f8ca8b418cb641bfae6fc3ff28bc */}
Inicie o servidor de desenvolvimento do Expo:
</Step>

```bash
npx expo start
```

Escaneie o código QR com o app Expo Go no seu dispositivo ou pressione `i` para o simulador iOS ou `a` para o emulador Android.

<Check>
Você está pronto para começar a construir! O boilerplate inclui telas e componentes de exemplo para ajudar você a entender a integração.
</Check>
</Step>
</Steps>

## Estrutura do Projeto

```
expo-boilerplate/
├── app/
│   ├── (tabs)/
│   │   └── index.tsx          # Home screen with examples
│   └── checkout.tsx           # Checkout screen
├── components/
│   └── PaymentButton.tsx      # Payment button component
├── lib/
│   └── dodo.ts                # Dodo Payments client setup
├── .env.example               # Environment variables template
└── package.json
```

## Construindo para Produção

<Steps>
{/* LOCKED_PATTERN_8da0a64f26eca1e80276896a6999817f */}

```bash
npm install -g eas-cli
```

</Step>

<Step title="Configure EAS">

```bash
eas build:configure
```

</Step>

{/* LOCKED_PATTERN_46d8869736258162472d6c37d4e5967b */}
Para iOS:
</Step>

```bash
eas build --platform ios
```

Para Android:

```bash
eas build --platform android
```

</Step>
</Steps>
