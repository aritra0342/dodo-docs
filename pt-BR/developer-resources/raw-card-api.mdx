## API Endpoints

| Environment | Base URL | Confirm Payment Endpoint |
| --- | --- | --- |
| **Test Mode** | `https://test.dodopayments.com` | `https://api.custom.hs.dodopayments.com/payments/<payment_id>/confirm` |
| **Live Mode** | `https://live.dodopayments.com` | `https://api.hs.dodopayments.com/payments/<payment_id>/confirm` |

## Fluxo de Pagamento com Cartão Passo a Passo

### Passo 1: Obtenha a Chave Secreta

Comece criando um pagamento usando a seguinte API:

- **Payments**: [Create Payment API](/api-reference/payments/post-payments)

Essa API retorna um **`secret_key`**, necessário para as etapas subsequentes.

### Passo 2: Confirme o Pagamento

Confirme o pagamento chamando o endpoint `payments confirm`:

- **Modo de Teste**: `POST https://api.custom.hs.dodopayments.com/payments/<payment_id>/confirm`
- **Modo ao Vivo**: `POST https://api.hs.dodopayments.com/payments/<payment_id>/confirm`

Inclua os seguintes detalhes no corpo da requisição:

- `client_secret`: Chave secreta do Passo 1
- `payment_method`: Ajuste para `card`
- `payment_method_data`: Detalhes do cartão e de cobrança
- `browser_info`: Dados específicos do navegador (user agent, detalhes da tela etc.)
- `payment_type`: Defina explicitamente como `normal` (para pagamentos únicos)

Cabeçalhos obrigatórios:

- Obtenha sua chave publicável com o suporte da DodoPayments
- `api-key`: Sua chave publicável (obtenha com o time Dodo Payments)
- `content-type`: `application/json`

**Exemplo de Corpo da Requisição:**

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "card",
  "payment_method_data": {

    },
    "billing": {
      "address": {
        "first_name": "<first_name>",
        "last_name": "<last_name>"
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
```

### Lidando com a Resposta

- Se a resposta **`status`** for **`requires_customer_action`**, redirecione o cliente para a URL fornecida no campo **`next_action`** para autenticação adicional.

<Warning>
Garantir conformidade com as diretrizes PCI ao lidar com dados brutos de cartão.
</Warning>

## Integração com Apple Pay

Para usar o Apple Pay, defina `payment_method` como `wallet` e inclua os dados do Apple Pay em `payment_method_data`.

### Estrutura do Corpo da Requisição

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "wallet",
  "payment_method_data": {
    "wallet": {
      "apple_pay": {
        "payment_data": "<decrypted_payment_data>",
        "payment_method": {
          "display_name": "Visa 1234",
          "network": "Visa",
          "type": "debit"
        },
        "transaction_identifier": "UNIQUE_TRANSACTION_ID_FROM_APPLE"
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
    "time_zone": "<time_zone>",
    "java_enabled": true,
    "java_script_enabled": true,
    "ip_address": "<ip_address_of_client>",
    "device_model": "<device_model>",
    "os_type": "<os_type>",
    "os_version": "<version>"
  },
  "payment_type": "normal"
}
```

### Referência de Campos do Apple Pay

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `payment_data` | string | Yes | This enum is used to represent the Apple Pay payment data, which can either be encrypted or decrypted. |
| `payment_method.display_name` | string | Yes | Display name shown on Apple Pay button (e.g., "Visa 1234") |
| `payment_method.network` | string | Yes | Card network (e.g., "Visa", "Mastercard", "Amex") |
| `payment_method.type` | string | Yes | Payment method type (e.g., "debit", "credit") |
| `payment_method.card_exp_month` | string | No | Card expiration month (e.g., "12") |
| `payment_method.card_exp_year` | string | No | Card expiration year (e.g., "25") |
| `transaction_identifier` | string | Yes | Unique transaction identifier from Apple Pay |

## Integração com Google Pay

Para usar o Google Pay, defina `payment_method` como `wallet` e inclua os dados do Google Pay em `payment_method_data`.

### Estrutura do Corpo da Requisição

```json
{
  "client_secret": "<secret_key>",
  "payment_method": "wallet",
  "payment_method_data": {
    "wallet": {
      "google_pay": {
        "type": "CARD",
        "description": "Visa ending in 1234",
        "info": {
          "card_network": "VISA",
          "card_details": "1234"
        },
        "tokenization_data": {
          "type": "PAYMENT_GATEWAY",
          "token": "<encrypted_payment_token>"
        }
      }
    }
  },
  "browser_info": {
    "user_agent": "<user_agent>",
    "accept_header": "<accept_header>",
    "language": "<language>",
    "color_depth": "<color_depth>",
    "screen_height": "<screen_height>",
    "screen_width": "<screen_width>",
    "time_zone": "<time_zone>",
    "java_enabled": true,
    "java_script_enabled": true,
    "ip_address": "<ip_address_of_client>",
    "device_model": "<device_model>",
    "os_type": "<os_type>",
    "os_version": "<version>"
  },
  "payment_type": "normal"
}
```

### Referência de Campos do Google Pay

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `type` | string | Yes | Payment method type (typically "CARD") |
| `description` | string | Yes | User-facing description of the payment method |
| `info.card_network` | string | Yes | Card network (e.g., "VISA", "MASTERCARD", "AMEX") |
| `info.card_details` | string | Yes | Last 4 digits of the card |
| `info.assurance_details` | object | No | Authentication assurance details |
| `tokenization_data.type` | string | Yes | Tokenization type (e.g., "PAYMENT_GATEWAY") |
| `tokenization_data.token` | string | Yes | Encrypted payment token from Google Pay |
