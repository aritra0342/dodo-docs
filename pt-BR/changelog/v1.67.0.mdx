## Novos Recursos 🚀

1. **Checkout de Carrinho Misto: Produtos de Pagamento Único e Assinaturas em um Único Carrinho**  
   O checkout agora suporta a combinação de produtos de pagamento único e produtos de assinatura em um único carrinho, desbloqueando novos casos de uso de faturamento poderosos.
   
   <Frame>
   <img src="/images/changelog/mixed-checkout.png" alt="Checkout misto com produtos de pagamento único e assinatura" />
   </Frame>
   
   ### O Que Isso Permite
   
   - **Taxas de configuração + assinaturas**: Cobrar uma taxa de configuração única juntamente com uma assinatura recorrente em um único checkout
   - **Pacotes com componentes recorrentes**: Vender um produto físico (único) combinado com acesso a um serviço contínuo (assinatura)
   - **Combos de curso + associação**: Oferecer a compra de um curso com uma assinatura opcional de associação à comunidade
   - **Assinaturas vitalícias + complementares**: Combinar acesso a produtos vitalícios com recursos premium recorrentes opcionais
   
   ### Como Funciona
   
   Basta adicionar produtos de pagamento único e de assinatura à sua sessão de checkout:
   

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [
       // One-time setup fee
       { product_id: 'prod_setup_fee', quantity: 1 },
       // Monthly subscription
       { product_id: 'prod_monthly_plan', quantity: 1 }
     ],
     customer: {
       email: 'customer@example.com',
       name: 'Jane Doe'
     },
     return_url: 'https://yourapp.com/success'
   });
   ```

   
   <Check>
   Resultado: Um único fluxo de checkout que processa o pagamento único e inicia a assinatura — reduzindo a fricção e aumentando as taxas de conversão.
   </Check>
   
   <Info>
   Ao misturar tipos de produtos, o pagamento único é cobrado imediatamente enquanto a assinatura segue seu ciclo de faturamento configurado (incluindo qualquer período de teste que você tenha definido).
   </Info>
   
   Saiba mais: [Sessões de Checkout](/developer-resources/checkout-session)

2. **Portal do Cliente Unificado**  
   
   Um novo portal centralizado para clientes em [customer.dodopayments.com](https://customer.dodopayments.com) onde os clientes podem visualizar e gerenciar todas as suas compras e assinaturas em diferentes empresas no Dodo Payments.
   
   <Frame>
   <img src="/images/changelog/unified-customer-portal.png" alt="Portal do Cliente Unificado" />
   </Frame>
   
   ### Recursos do Portal
   
   - **Visibilidade entre empresas**: Visualizar todas as compras e assinaturas de qualquer empresa que use o Dodo Payments em um só lugar
   - **Gerenciamento centralizado de assinaturas**: Gerenciar assinaturas em diferentes comerciantes a partir de um único painel
   - **Histórico de faturamento unificado**: Acessar faturas e histórico de pagamentos de todas as suas compras
   - **Experiência de login único**: Fazer login uma vez com seu e-mail para acessar compras de todos os comerciantes do Dodo Payments
   
   ### Como Funciona
   
   Os clientes simplesmente visitam [customer.dodopayments.com](https://customer.dodopayments.com) e inserem seu endereço de e-mail. Eles receberão um link de login seguro que concede acesso a todas as suas compras em todo o ecossistema do Dodo Payments.
   
   <Tip>
   O Portal do Cliente Unificado complementa os portais de clientes específicos de negócios. Os clientes ainda podem acessar portais individuais de comerciantes para uma experiência de marca, ou usar o portal unificado para uma visão consolidada.
   </Tip>
   
   <Note>
   O portal unificado utiliza a mesma autenticação baseada em e-mail que os portais específicos de negócios, garantindo uma experiência consistente e segura.
   </Note>

3. **ID da Fatura na Resposta de Pagamento**  
   A resposta de pagamento agora inclui um `invoice_id` campo, facilitando a reconciliação de pagamentos com faturas e a integração com sistemas contábeis.
   

   ```json
   {
     "payment_id": "pay_2IjeQm4hqU6RA4Z4kwDee",
     "status": "succeeded",
     "invoice_id": "inv_2IsUnWGtRKFLxk7xAQeyt",
     "amount": 9900,
     "currency": "USD",
     // ... other fields
   }
   ```

   
   ### Casos de Uso
   
   - **Integração contábil**: Vincular diretamente pagamentos a faturas em seu software de contabilidade
   - **Suporte ao cliente**: Encontrar rapidamente a fatura associada a um pagamento ao lidar com solicitações de suporte
   - **Reconciliação**: Simplificar a reconciliação financeira combinando pagamentos a faturas programaticamente
   - **Conformidade**: Manter registros precisos vinculando pagamentos às suas faturas fiscais correspondentes
   
   <Info>
   O `invoice_id` campo utiliza IDs de fatura específicos da Índia quando disponíveis, garantindo conformidade com os requisitos locais de faturamento.
   </Info>
   
   Saiba mais: [Obter API de Detalhes de Pagamento](/api-reference/payments/get-payments-1)
