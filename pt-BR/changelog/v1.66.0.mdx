## Novos Recursos 🚀

1. **`subscription.updated` Webhook**  
   Um novo evento de webhook que dispara em cada atualização de objeto de assinatura, fornecendo notificações em tempo real sobre quaisquer alterações em suas assinaturas.
   
   Use este webhook para:
   - Receber notificações instantâneas quando os detalhes da assinatura mudarem
   - Manter o estado do seu aplicativo sincronizado com a Dodo Payments
   - Reagir às atualizações de assinaturas sem fazer polling na API
   
   <Tip>
   Inscreva-se no `subscription.updated` para eliminar a necessidade de polling e manter seu sistema sincronizado com as mudanças de assinatura em tempo real.
   </Tip>
   
   Saiba mais: [Subscription Webhooks](/developer-resources/webhooks/intents/subscription)

2. **Suporte a Endereço Mínimo no Checkout**  

   <Frame>
   <img src="/images/changelog/minimal-address.png" alt="Minimal Address Support in Checkout" />
   </Frame>

   Reduza o atrito no checkout com o novo modo de coleta de endereço mínimo. Em vez de coletar todos os dados de endereço, o modo de endereço mínimo coleta apenas:
   
   - **País**: Sempre necessário para determinação de impostos
   - **CEP/Código postal**: Coletado apenas em regiões onde é necessário para cálculo de imposto sobre vendas, IVA ou GST
   
   <Check>
   Resultado: conclusão do checkout significativamente mais rápida com menos campos para os clientes.
   </Check>
   
   **Ative o endereço mínimo:**
   
   <Tabs>
   {/* LOCKED_PATTERN_0090a508f27e87d00759ddcd3b9fe94c */}

   ```javascript
   const session = await client.checkoutSessions.create({
     product_cart: [{ product_id: 'prod_123', quantity: 1 }],
     minimal_address: true
   });
   ```

   </Tab>
   
   {/* LOCKED_PATTERN_ed987f4815610e04ce90e6f11a8998dc */}
   Anexe `minimalAddress=true` à URL do seu link de pagamento:
   

   ```
   https://checkout.dodopayments.com/buy/prod_123?minimalAddress=true
   ```

   </Tab>
   </Tabs>
   
   <Info>
   Por padrão, o checkout continua coletando endereços completos. Gradualmente faremos a transição de todos os links de checkout para o modo de endereço mínimo. Ative o flag de recurso agora para começar a se beneficiar da redução do atrito no checkout.
   </Info>

3. **Chaves de API Somente Leitura**  
   Crie chaves de API com permissões somente leitura para integrações mais seguras. Ao criar uma nova chave de API, desmarque **Ativar acesso de escrita** para criar uma chave somente leitura.
   
   Chaves apenas leitura podem:
   - Buscar dados de pagamentos, assinaturas e clientes
   - Listar produtos e recuperar detalhes
   - Acessar relatórios e análises
   
   Chaves somente leitura **não podem**:
   - Criar ou modificar pagamentos
   - Atualizar assinaturas
   - Alterar configurações de produtos
   
   <Tip>
   Desmarque "Ativar acesso de escrita" para integrações de painel, ferramentas de análise e qualquer sistema que precise apenas visualizar dados sem fazer alterações.
   </Tip>

4. **UPI AutoPay e Assinaturas com Cartão RuPay – Implantação Completa**  
   UPI AutoPay e assinaturas com cartão RuPay agora estão disponíveis para todos os negócios. Isso possibilita pagamentos recorrentes para clientes indianos com:
   
   - **UPI AutoPay**: Cobranças recorrentes automatizadas via mandato UPI
   - **Cartões RuPay**: Suporte a assinaturas para a rede de cartões doméstica da Índia
   
   <Check>
   Resultado: opções expandidas de pagamento de assinaturas para o mercado indiano com tratamento de mandatos compatível com o RBI.
   </Check>
   
   Saiba mais: [Subscriptions with RBI-Compliant Mandates](/features/subscription#subscriptions-with-rbi-compliant-mandates)

5. **API de Pré-visualização de Mudança de Plano**  
   Um novo endpoint de API para pré-visualizar mudanças de plano de assinatura antes de se comprometer com elas. Obtenha o valor exato da cobrança e veja como a assinatura ficará após a mudança - sem fazer modificações reais.
   

   ```javascript
   const preview = await client.subscriptions.previewChangePlan('sub_123', {
     product_id: 'prod_pro_plan',
     proration_billing_mode: 'prorated_immediately',
     quantity: 1
   });
   
   console.log('Charge amount:', preview.immediate_charge.summary);
   console.log('New plan details:', preview.new_plan);
   ```

   
   Use isso para:
   - Mostrar aos clientes exatamente o que eles pagarão antes de confirmar upgrades/downgrades
   - Construir calculadoras de preços e ferramentas de comparação de planos
   - Reduzir tickets de suporte melhorando a transparência de preços
   
   Saiba mais: [API de Pré-visualização de Mudança de Plano](/api-reference/subscriptions/preview-change-plan)
